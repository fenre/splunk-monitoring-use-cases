# Migration Status: use-cases/ → content/ — COMPLETE

This document tracked the remaining dependencies on the legacy
`use-cases/` markdown corpus during the v7→v8.2 migration. As of
**2026-05-10** the migration is **complete**:

- The legacy `use-cases/` tree has been deleted from the working tree
  and from `main` (P1 step 7 Phase C).
- Every audit, generator, build stage, CI workflow, schema, sample data
  fixture, and user-facing doc that referenced it has been rewired to
  the JSON SSOT under `content/cat-NN-<slug>/UC-X.Y.Z.json`.
- The legacy repo-root `build.py` is now a 30-line shim that delegates
  to `tools/build/build.py`; root-level `catalog.json` / `data.js` /
  `llms*.txt` are no longer committed (they live under `dist/` and are
  served from GitHub Pages).
- The legacy fallback in `tools/build/enrichment.py` and every
  `audit_*` markdown backend have been removed; the only loader is the
  SSOT loader.

The historical migration log below is preserved verbatim as the
audit-replay record. New maintainer work no longer needs this doc; go
straight to [`docs/architecture.md`](architecture.md) and
[`docs/use-cases-burndown.md`](use-cases-burndown.md) (which now
records the burndown as RETIRED) instead.

## Remaining dependencies — none

Every row that used to live here was closed during P1 steps 5–7. A
CI guard (`.github/workflows/validate.yml` "no-use-cases" step) now
fails any PR that re-introduces a `use-cases/` path or live reference.
