# Signed evidence packs (Phase C-2)

> **Status:** required for changes to
> `src/splunk_uc/generators/sign_evidence.py`,
> `src/splunk_uc/audits/evidence_signatures.py`,
> `scripts/sign_evidence_batch.py`, or
> `.github/workflows/sign-evidence.yml`.

Auditors need more than a markdown evidence pack — they need proof that the
machine-readable twin they downloaded matches what the catalogue maintainers
published, and that it was not altered after publication. Phase C-2 adds a
signing layer on top of the Phase 4.2 evidence-pack generator, mirroring the
Phase 5.4 mapping-ledger pattern documented in [`signed-provenance.md`](signed-provenance.md).

## Why we sign evidence

1. **Integrity.** Every UC `evidence` descriptor and every
   `api/v1/evidence-packs/{reg-id}.json` file carries a SHA-256 content hash.
   Mutating the underlying sidecar evidence text or compliance metadata
   invalidates the manifest.
2. **Non-repudiation.** Release and weekly workflow runs produce Sigstore
   attestations (primary) or detached GPG signatures (fallback) bound to a
   specific GitHub Actions identity or OpenPGP key.
3. **Separation of concerns.** Raw UC sidecars are **not** signed inline.
   External manifests under `dist/evidence/` aggregate evidence metadata so
   sidecar edits do not require re-signing the entire catalogue on every PR.

## What gets signed

| Artefact | Path | Notes |
| -------- | ---- | ----- |
| Per-UC manifest | `dist/evidence/UC-<id>/manifest.json` | Aggregates `evidence`, `evidenceSigning`, compliance fingerprint |
| Evidence-pack JSON | `api/v1/evidence-packs/{reg-id}.json` | Copied to `dist/evidence-packs/` for attestation |
| Pack descriptor | `dist/evidence-packs/{reg-id}.descriptor.json` | Content hash + signature envelope |
| Sigstore bundle | `dist/evidence-packs/{reg-id}.sigstore.bundle.json` | Produced by `actions/attest-build-provenance@v4` |
| GPG detached sig | `dist/evidence/.../manifest.json.sig` | Fallback when Sigstore unavailable |

## Signing methods

UC sidecars declare intent via `evidenceSigning.method`:

| Method | Primary use | Production path |
| ------ | ----------- | --------------- |
| `sigstore` | Tier-1 evidence packs | `.github/workflows/sign-evidence.yml` + OIDC |
| `rfc3161-tsa` | Timestamp authority anchoring | TSA URL recorded in manifest; token path in signed envelope |
| `gpg` | Air-gapped / fallback | `generate-evidence-signatures --gpg` |
| `none` | Manifest only, no signature | Hash-only verification |

PR CI keeps `signature.state = "unsigned"` so builds stay deterministic.
The weekly `sign-evidence` workflow promotes artefacts to `signed`.

## Verification

### Hash-only (no network)

```bash
PYTHONPATH=src python3 -m splunk_uc audit-evidence-signatures --check
```

Exit code 0 means fixture manifests recompute cleanly and any on-disk
`dist/evidence/` manifests match their content hashes.

### Sigstore (release / workflow artefacts)

```bash
gh attestation verify dist/evidence-packs/gdpr.json \
  --owner fenre \
  --bundle dist/evidence-packs/gdpr.sigstore.bundle.json

PYTHONPATH=src python3 -m splunk_uc audit-evidence-signatures \
  --check --verify-signature
```

### GPG fallback

When Sigstore is rate-limited or unavailable, the workflow falls back to an
ephemeral GPG key and publishes `dist/evidence-packs/gpg-public-key.asc`.

Local signing:

```bash
gpg --quick-gen-key "Evidence Dev <you@example.com>" default default
export GPG_KEY="$(gpg --list-secret-keys --with-colons | awk -F: '$1==\"sec\"{print $5; exit}')"
PYTHONPATH=src python3 -m splunk_uc generate-evidence-signatures --all --gpg --gpg-key "$GPG_KEY"
gpg --verify dist/evidence/UC-22.2.32/manifest.json.sig dist/evidence/UC-22.2.32/manifest.json
```

Publish the exported public key alongside release assets so auditors can
import it once and verify all detached signatures offline.

## Local development

```bash
# 1. Ensure evidence packs exist
PYTHONPATH=src python3 -m splunk_uc generate-evidence-packs

# 2. Emit unsigned manifests
PYTHONPATH=src python3 -m splunk_uc generate-evidence-signatures --all

# 3. Audit (hermetic fixtures + any dist/ output)
PYTHONPATH=src python3 -m splunk_uc audit-evidence-signatures --check

# Convenience wrapper
python3 scripts/sign_evidence_batch.py
```
