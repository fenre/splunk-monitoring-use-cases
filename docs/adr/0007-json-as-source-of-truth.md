# ADR-0007: JSON sidecars as source of truth for UC content

- **Status:** Accepted
- **Date:** 2026-05-08
- **Deciders:** Repository maintainers
- **Supersedes:** [ADR-0001](0001-markdown-as-source-of-truth.md)

## Context

ADR-0001 (2023, retroactively ratified 2026-04-16) established that UC
content lives in plain markdown under `use-cases/cat-*.md`. That decision
served the catalogue from ~6,300 to ~7,000 UCs. Two pressures forced a
revisit:

1. **Compliance metadata exploded.** Each UC now carries
   `compliance[].clause` arrays mapping to ~60 regulations, OSCAL
   component-definition IDs, MITRE ATT&CK<sup class="ref">[<a href="#ref-2">2</a>]</sup> technique IDs, peer/legal/SME
   review signoffs, evidence-pack pointers, and version-pinned data
   sources. The flat-markdown parser in [`build.py`](../../build.py)
   could no longer represent this shape without ad-hoc JSON-string
   smuggling inside markdown bullet items.
2. **Round-trip authoring tooling shifted.** Generators (`scripts/generate_*`),
   audits (`scripts/audit_*`), the MCP server, and downstream consumers
   (api/v1, OSCAL, MITRE coverage) all want a structured representation.
   Re-parsing markdown on every consumer is wasteful and error-prone.

The Phase 1.6 → 1.7 → v7.0 migration completed in 2026-04-18. Today's
content tree under `content/cat-NN-slug/UC-X.Y.Z.json` is the canonical
source; the markdown companion `UC-X.Y.Z.md` is **generated** from the
JSON by `python3 -m splunk_uc generate-md-from-json` and is kept in lock-step by a
CI `--check` gate.

[`docs/DESIGN.md`](../DESIGN.md) §5 and [`AGENTS.md`](../../AGENTS.md)
have asserted "JSON is canonical" since v7.0; ADR-0001 had simply not
been updated. This ADR closes that governance contradiction.

## Decision

**Store every UC as a JSON sidecar** at `content/cat-NN-slug/UC-X.Y.Z.json`,
validated against [`schemas/uc.schema.json`](../../schemas/uc.schema.json)
v1.6.x. The sidecar is the authoring surface and the source of truth for
every downstream artefact:

| Artefact | Generated by | Contract |
|---|---|---|
| `content/.../UC-X.Y.Z.md` | `python3 -m splunk_uc generate-md-from-json` | Round-trip; `--check` blocks drift. |
| `catalog.json` | `tools/build/build.py` (legacy `build.py` writer until P1.2) | Stable abbreviated keys; see `docs/catalog-schema.md`. |
| `data.js` | same | JS-wrapped `catalog.json`; retired in P5 (frontend rebuild). |
| `api/v1/**.json` | `python3 -m splunk_uc generate-api-surface` | OpenAPI 3.1 + JSON-LD; v1 contract is additive-only. |
| `dist/uc/UC-X.Y.Z/` | `tools/build/build.py` | SSG; permalinks are stable per [ADR-0005](0005-uc-id-x-y-z-scheme.md). |
| `llms.txt`, `llms-full.txt` | `tools/build/build.py` | LLM-readable corpus; chunked variant in P17. |

If a generated artefact disagrees with its source JSON sidecar, the
generated artefact is wrong; rerun the generator.

## Consequences

**Positive:**

- One canonical shape powers the build, audits, MCP, and api/v1.
- `additionalProperties: false` on `uc.schema.json` v1.6.1 catches
  typos and drift at PR time.
- Every UC is round-trippable (JSON ↔ markdown ↔ rendered HTML), and
  the round-trip has a CI gate.
- Compliance metadata (clause arrays, signoff ledgers, OSCAL refs) lives
  natively rather than encoded inside markdown bullets.
- Typed-pipeline work (P4 of the repo-overhaul plan) becomes possible:
  the JSON shape generates Pydantic models that replace 50+ sites of
  `uc.get("kfp", "")` dict-of-strings access.

**Negative:**

- Authors who prefer reading prose in PR diffs see a JSON diff for
  cat-22 / Phase-2.x edits. Mitigation: the generated `.md` companion
  shows up in the same PR and gives a human-readable second view.
- Any malformed JSON is rejected at PR time with no graceful
  degradation. Mitigation: the pre-commit hook (`python3 -m splunk_uc validate-uc-schema-staged`,
  Phase 0) catches schema violations locally.
- Two artefacts per UC (`.json` + `.md`) means 7,657 × 2 = 15,314 files
  in `content/`. Mitigation: the markdown companion is auto-generated;
  authors only edit the JSON.

## Alternatives considered

- **Stick with markdown as canonical** (status quo per ADR-0001).
  Rejected: compliance metadata cannot be expressed cleanly; downstream
  consumers re-parse markdown anyway and frequently disagree.
- **Embed JSON inside YAML front-matter** in markdown. Rejected: YAML
  syntax errors in 7,657 files would be a constant maintenance load;
  multi-line SPL inside YAML strings requires escaping that is hostile
  to authors.
- **A single SQLite `catalog.db`**. Rejected: opaque to PR review; forks
  would need infrastructure; the static-site contract (ADR-0002) would
  break.

## Migration

The migration completed in v7.0 with these moves:

1. `scripts/archive/migrate_uc_markdown_to_json.py` lifted every UC
   heading from `use-cases/cat-*.md` into a JSON sidecar.
2. `python3 -m splunk_uc generate-md-from-json` regenerated the `.md` companion
   alongside each sidecar.
3. The legacy `use-cases/cat-*.md` tree remains in the repo until v8.1
   (per repo-overhaul plan P1 step 7) and is read-only; only one audit
   (`python3 -m splunk_uc audit-uc-structure`) still references it.
4. The audit migrates to the JSON tree in P1 step 3.

After P1 step 3 lands, no live code depends on `use-cases/`. After
v8.0 ships, the legacy tree moves to `content-legacy/` for one minor
version, then is deleted.
