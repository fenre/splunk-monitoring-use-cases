# AGENTS.md — AI Agent Entrypoint

> Start here if you are an AI agent (Cursor, Claude Code, Codex, Copilot,
> or any MCP-capable client) working with this repository.

## What this repository is

A curated, machine-readable, versioned catalog of **7,300+ IT infrastructure
monitoring use cases** for Splunk, organized across 23 technology domains.
Each use case carries a ready-to-run SPL query, CIM data model mappings,
criticality/difficulty ratings, implementation guidance, and plain-language
explanations.

The catalog is **content-only** — it does not execute or schedule searches
against a live Splunk instance.

## Content layout

Source of truth: `content/cat-NN-slug/UC-X.Y.Z.json` (one directory per category, one JSON file per UC).
Build: `make build` (or `python3 tools/build/build.py --out dist`) → `dist/` (static site + API + exports).

There is one build pipeline. Read [`docs/DESIGN.md`](docs/DESIGN.md) §6
for the build stages.

## Identity scheme

- **UC-ID:** `UC-X.Y.Z` (Category.Subcategory.Index). Unique repo-wide, gap-free within subcategories.
- **Category folders:** `cat-NN-kebab-case` (zero-padded category number).
- **JSON sidecar IDs:** `X.Y.Z` (no `UC-` prefix in the `id` field; the prefix is display-only).

## Catalog JSON — field abbreviations

`catalog.json` uses abbreviated keys for compactness. The full mapping is in
[`docs/catalog-schema.md`](docs/catalog-schema.md). The most common:

| Key | Meaning | Key | Meaning |
|-----|---------|-----|---------|
| `i` | ID | `n` | Name/title |
| `c` | Criticality | `f` | Difficulty |
| `v` | Value | `ge` | Plain-language explanation |
| `t` | App/TA | `d` | Data sources |
| `q` | SPL query | `qs` | CIM/tstats SPL |
| `s` | Subcategories | `u` | Use cases |
| `e` | Equipment IDs | `em` | Equipment model IDs |
| `wv` | Wave (crawl/walk/run) | `pre` | Prerequisite UC IDs |

## Authoring rules

1. **Schema:** Every UC JSON sidecar must validate against `schemas/uc.schema.json`.
2. **Quality tiers:** Gold / Silver / Bronze — see `.cursor/rules/gold-standard-authoring.mdc` and `docs/gold-standard-template.md`.
3. **Required fields (JSON):** `id`, `title`, `criticality`, `difficulty`, `monitoringType`, `value`, `app`, `dataSources`, `spl`, `implementation`, `visualization`, `cimModels`, `grandmaExplanation`.
4. **grandmaExplanation:** Every UC carries a plain-language `ge` field; generate with `scripts/generate_grandma_explanations.py`.
5. **Validation:** Run `make audit` (or `python3 scripts/audit_uc_structure.py --full`) and `make build` before committing.

## How an AI agent should consume this catalog

Pick the access pattern that matches your tooling, then read
[`AGENTS-EXAMPLES.md`](AGENTS-EXAMPLES.md) for copy-paste recipes:

1. **MCP-connected agent** (Cursor, Claude Code, Codex, custom MCP client)
   — call `search_use_cases` first, then `get_use_case` (or
   `get_use_case_markdown` when you want plain text) to drill into one UC.
2. **JSON-API consumer** (RAG pipelines, automation scripts) — fetch
   `catalog.json` once, then `/uc/UC-X.Y.Z/index.json` per UC. Stable
   URLs are frozen at v7.0.0.
3. **Bare LLM with browse / fetch** (ChatGPT, Perplexity, Gemini) — start
   at `/llms.txt`, then fetch `/uc/UC-X.Y.Z/uc.md` for pure-markdown UC
   detail. No JSON parsing required.

Every UC URL exposes three coordinated representations:
`index.html` (humans), `index.json` (machines), `uc.md` (LLMs). They
share the same identity, last-modified stamp, and catalogue version.

## CI gates

All audits are in `.github/workflows/validate.yml`. Key steps:

- UC ID uniqueness and gap-free ordering
- UC structure and required fields
- Non-technical view sync
- Changelog cross-references
- Catalog schema shape
- Prerequisite graph (cycles, unknown IDs, wave monotonicity)
- SPL hallucination audit (unknown commands, invalid eval functions)
- Build freshness (regenerate and diff)
- Version triple consistency (`VERSION` ↔ `CHANGELOG.md` ↔ `index.html`)
- Roadmap consistency (`scripts/audit_roadmap_consistency.py --check`)
- License inventory (`scripts/audit_license_inventory.py --check`)
- Metrics history snapshot (`scripts/snapshot_metrics.py --check` — fails when `VERSION` is bumped without a matching `data/metrics-history/<VERSION>.json`)
- Metrics shape (`dist/metrics.json` validates against `schemas/v2/metrics.schema.json` on every reproducible build)
- Stewardship digest schema (PR smoke-test of `scripts/generate_stewardship_digest.py` against `schemas/v2/stewardship-digest.schema.json`; weekly refresh in `.github/workflows/stewardship.yml`)
- Build reproducibility (nightly + build-pipeline PRs: `python -m splunk_uc audit-reproducibility --keep` runs two consecutive `--reproducible` builds and asserts `dist/integrity.json` byte-identical; see `.github/workflows/build-reproducibility.yml`)
- `splunk_uc` dispatcher smoke (per-PR: `python -m splunk_uc --help` and `--version` succeed; pinned by `tests/splunk_uc/test_dispatcher.py`)

## Quick commands

```bash
make build                                          # rebuild full site into dist/
make audit                                          # run core audit checks
make audit-full                                     # run ALL audit checks
make test                                           # unit tests + build validation
make audit-roadmap                                  # validate ROADMAP.md structure + links
make export-roadmap                                 # emit Project-board JSON snapshot
make audit-license-inventory                        # validate dependency licenses against allowlist
make write-license-inventory                        # regenerate data/license-inventory.json baseline
make audit-metrics-snapshot                         # ensure release-time metrics snapshot exists
make snapshot-metrics                               # write data/metrics-history/<VERSION>.json from dist/metrics.json
make stewardship-digest                             # generate dist/stewardship-digest.{json,md}
make audit-reproducibility                          # two --reproducible builds must match (~90s)
make audit-reproducibility-fast                     # single --reproducible build smoke (~30s)
make splunk-uc-help                                 # show the python -m splunk_uc CLI help
PYTHONPATH=src python3 -m splunk_uc --help          # canonical splunk_uc dispatcher entry point (P6)
python3 scripts/generate_grandma_explanations.py    # fill missing plain-language fields
python3 scripts/splunk_fortune.py                   # random UC "fortune cookie"
PYTHONPATH=src python3 -m splunk_uc audit-prerequisites --check  # validate implementation ordering
```
