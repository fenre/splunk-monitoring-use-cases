# UC-16.1.24 · ServiceNow Bidirectional Incident Sync

## Description

Bidirectional sync between ITSI episodes and ServiceNow incidents eliminates manual ticket creation and ensures incident status is consistent across platforms.

## Value

Bidirectional sync between ITSI episodes and ServiceNow incidents eliminates manual ticket creation and ensures incident status is consistent across platforms.

## Implementation

Configure ServiceNow integration in ITSI: map episode severity to ServiceNow priority, define assignment groups, and enable bidirectional status updates. Episodes auto-create incidents; ServiceNow resolution closes episodes. Monitor sync latency and failure rate. Requires Splunk Add-on for ServiceNow 5.5+.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsi_grouped_alerts status!=5
| eval has_snow_ticket=if(isnotnull(snow_incident_number), "synced", "unsynced")
| stats count by has_snow_ticket severity
| sort severity
```
