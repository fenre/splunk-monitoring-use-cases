# UC-16.2.7 · Asset Discovery Reconciliation

## Description

Reconciles discovery source of truth against CMDB for asset inventory — variance by source, age, and confidence.

## Value

Reconciles discovery source of truth against CMDB for asset inventory — variance by source, age, and confidence.

## Implementation

Normalize hostnames (FQDN strip). Map discovery tool serial/IP to CMDB. Report `stale_or_missing` counts weekly. Drive CMDB update tasks for unmatched discovery rows.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: ServiceNow Discovery, SCCM, or network scan feeds.
• Ensure the following data sources are available: `sourcetype=snow:discovery_model` or custom `discovery:asset`, `sourcetype=snow:cmdb_ci`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize hostnames (FQDN strip). Map discovery tool serial/IP to CMDB. Report `stale_or_missing` counts weekly. Drive CMDB update tasks for unmatched discovery rows.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=discovery sourcetype="discovery:asset" earliest=-1d
| eval host_key=lower(mvindex(split(hostname,"."),0))
| stats latest(_time) as last_seen by host_key, serial_number
| join type=left max=1 host_key [ search index=itsm sourcetype="snow:cmdb_ci" | eval host_key=lower(mvindex(split(name,"."),0)) | table host_key, sys_id, last_discovered ]
| eval match_state=if(isnotnull(sys_id) AND abs(last_seen-last_discovered)<86400,"synced","stale_or_missing")
| stats count by match_state
```
