# UC-16.3.6 · Major Incident and Post-Mortem Tracking

## Description

Tracking major incidents and post-mortem completion ensures learning and accountability. Supports compliance and continuous improvement.

## Value

Tracking major incidents and post-mortem completion ensures learning and accountability. Supports compliance and continuous improvement.

## Implementation

Ingest major incident and post-mortem status. Alert when post-mortem is overdue. Report on major incident count, MTTR, and post-mortem completion rate. Track root cause categories.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: ITSM major incident, post-mortem records.
• Ensure the following data sources are available: Major incident flag, post-mortem due and completed dates.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest major incident and post-mortem status. Alert when post-mortem is overdue. Report on major incident count, MTTR, and post-mortem completion rate. Track root cause categories.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="incident"
| where major_incident=1 AND is_open=0
| eval pm_due=closed_time + (7*86400)
| where now() > pm_due AND post_mortem_completed=0
| table number, short_description, closed_time, pm_due, post_mortem_completed
```
