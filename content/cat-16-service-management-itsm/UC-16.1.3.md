# UC-16.1.3 · MTTR by Category

## Description

MTTR per category identifies where process improvements or automation would have the greatest impact.

## Value

MTTR per category identifies where process improvements or automation would have the greatest impact.

## Implementation

Calculate MTTR from incident open to resolution timestamps. Break down by category, subcategory, and assignment group. Track trends over time. Set MTTR targets per category and report on achievement.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_snow`.
• Ensure the following data sources are available: Incident lifecycle data (open, assigned, resolved timestamps).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Calculate MTTR from incident open to resolution timestamps. Break down by category, subcategory, and assignment group. Track trends over time. Set MTTR targets per category and report on achievement.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="snow:incident" state="resolved"
| eval mttr_hours=round((resolved_at-opened_at)/3600,1)
| stats avg(mttr_hours) as avg_mttr, median(mttr_hours) as median_mttr by category
| sort -avg_mttr
```
