# UC-16.1.10 · Service Request Fulfillment Time

## Description

Fulfillment time metrics drive service catalog optimization and customer satisfaction. Slow fulfillment reduces adoption.

## Value

Fulfillment time metrics drive service catalog optimization and customer satisfaction. Slow fulfillment reduces adoption.

## Implementation

Track service request lifecycle from submission to fulfillment. Calculate fulfillment time per catalog item. Identify items with slow fulfillment for automation opportunities. Report on catalog efficiency.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_snow`.
• Ensure the following data sources are available: Service request data.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track service request lifecycle from submission to fulfillment. Calculate fulfillment time per catalog item. Identify items with slow fulfillment for automation opportunities. Report on catalog efficiency.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="snow:sc_request"
| eval fulfillment_hours=round((closed_at-opened_at)/3600,1)
| stats avg(fulfillment_hours) as avg_hours, median(fulfillment_hours) as median_hours by cat_item
| sort -avg_hours
```
