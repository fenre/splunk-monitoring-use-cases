# UC-16.3.10 · Business Service Availability (Composite SLA)

## Description

Rolls up component availability into a single business-service SLA percentage (weighted or “all essential up”) for customer-facing reporting — beyond host-level heatmaps.

## Value

Rolls up component availability into a single business-service SLA percentage (weighted or “all essential up”) for customer-facing reporting — beyond host-level heatmaps.

## Implementation

Define `business_service_map.csv` with components, optional weights, and `is_essential` (any essential down forces SLA=0 or “breach”). Ingest normalized availability per component every 5 minutes; backfill from summary index for monthly SLA. Align with contract SLAs (e.g., 99.9% monthly). ITSI can replace this with service KPIs and composite service health.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI, or custom lookups + summary indexing.
• Ensure the following data sources are available: `index=monitoring` normalized health events, `business_service_map.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Define `business_service_map.csv` with components, optional weights, and `is_essential` (any essential down forces SLA=0 or “breach”). Ingest normalized availability per component every 5 minutes; backfill from summary index for monthly SLA. Align with contract SLAs (e.g., 99.9% monthly). ITSI can replace this with service KPIs and composite service health.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=monitoring sourcetype IN (server_health, app_health, db_health) earliest=-24h
| eval up=if(status="ok" OR status="up",1,0)
| stats latest(up) as up by component_name
| lookup business_service_map.csv component_name OUTPUT business_service, weight, is_essential
| eval w=coalesce(weight,1)
| stats max(eval(if((is_essential=1 OR lower(is_essential)="true") AND up=0,1,0))) as essential_down
       sum(eval(w*up)) as weighted_up
       sum(w) as total_weight by business_service
| eval composite_sla=if(essential_down>0, 0, round(100*weighted_up/total_weight,2))
| where composite_sla < 99.9 OR essential_down>0
| table business_service, composite_sla, essential_down
```
