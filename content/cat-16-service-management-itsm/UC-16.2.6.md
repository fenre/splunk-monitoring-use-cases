# UC-16.2.6 · CI Relationship Drift

## Description

Detects when CI relationships change unexpectedly versus a baseline — supporting impact analysis integrity and unauthorized dependency changes.

## Value

Detects when CI relationships change unexpectedly versus a baseline — supporting impact analysis integrity and unauthorized dependency changes.

## Implementation

Schedule a nightly `outputlookup cmdb_rel_baseline.csv` from `cmdb_rel_ci` and compare with `| diff` or `join` on parent+child+type for strict drift. The SPL above flags relationships with activity on multiple days in a week (churn). Alert on new parent/child pairs against a saved baseline lookup when available.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_snow`.
• Ensure the following data sources are available: `sourcetype=snow:cmdb_rel_ci` or relationship table snapshots.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Schedule a nightly `outputlookup cmdb_rel_baseline.csv` from `cmdb_rel_ci` and compare with `| diff` or `join` on parent+child+type for strict drift. The SPL above flags relationships with activity on multiple days in a week (churn). Alert on new parent/child pairs against a saved baseline lookup when available.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="snow:cmdb_rel_ci" earliest=-7d@d
| bucket _time span=1d
| stats values(_time) as days by parent, child, relationship_type
| eval change_days=mvcount(days)
| where change_days>1
| table parent, child, relationship_type, change_days
```
