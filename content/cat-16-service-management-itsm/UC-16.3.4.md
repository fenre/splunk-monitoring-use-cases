# UC-16.3.4 · Escalation and Handoff Latency

## Description

Long escalation or handoff times delay resolution. Monitoring handoff duration supports SLA and identifies bottlenecks.

## Value

Long escalation or handoff times delay resolution. Monitoring handoff duration supports SLA and identifies bottlenecks.

## Implementation

Ingest assignment and state change history. Compute time between assignments. Alert when average handoff exceeds threshold. Report on escalation paths and slow handoffs.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: ITSM workflow logs, incident history.
• Ensure the following data sources are available: Assignment change events, timestamps per group.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest assignment and state change history. Compute time between assignments. Alert when average handoff exceeds threshold. Report on escalation paths and slow handoffs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="incident:history"
| search type=assignment_change
| eval handoff_hrs=(next_assignment_time - prev_assignment_time)/3600
| stats avg(handoff_hrs) as avg_handoff, max(handoff_hrs) as max_handoff by from_group, to_group
| where avg_handoff > 2
```
