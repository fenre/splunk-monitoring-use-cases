# UC-16.1.16 · Ticket Aging by Priority

## Description

Distribution of open ticket age by priority highlights backlog imbalance (e.g., many old P3s) and supports capacity and escalation decisions.

## Value

Distribution of open ticket age by priority highlights backlog imbalance (e.g., many old P3s) and supports capacity and escalation decisions.

## Implementation

Normalize priority labels. Bucket age for open tickets only. Report stacked bar or pivot table (priority × age bucket). Alert when count in `30d+` for P1/P2 exceeds policy.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_snow`.
• Ensure the following data sources are available: Open incidents (`sourcetype=snow:incident`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize priority labels. Bucket age for open tickets only. Report stacked bar or pivot table (priority × age bucket). Alert when count in `30d+` for P1/P2 exceeds policy.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="snow:incident" state IN ("new","in_progress","on_hold")
| eval age_days=round((now()-opened_at)/86400,1)
| eval bucket=case(age_days<=1,"0-1d", age_days<=7,"2-7d", age_days<=30,"8-30d", true(),"30d+")
| eval pri=case(priority=1,"P1", priority=2,"P2", priority=3,"P3", priority=4,"P4", true(),"Other")
| stats count by pri, bucket
| sort pri, bucket
```
