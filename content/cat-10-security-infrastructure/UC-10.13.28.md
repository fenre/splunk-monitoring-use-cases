# UC-10.13.28 · Asset Risk Scoring (CIM)

## Description

Aggregate open vulns per `dest` weighted by CVSS for top-risk hosts.

## Value

Aggregate open vulns per `dest` weighted by CVSS for top-risk hosts.

## Implementation

Blend with threat intel for exploited-in-wild CVEs (2× weight).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Vulnerabilities CIM.
• Ensure the following data sources are available: `dest`, `severity`, `cve`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Blend with threat intel for exploited-in-wild CVEs (2× weight).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=vulnerability status=open
| eval w=case(severity="critical",10, severity="high",7, severity="medium",4, 1)
| stats sum(w) as risk_score, dc(cve) as cve_count by dest
| sort -risk_score
| head 100
```
