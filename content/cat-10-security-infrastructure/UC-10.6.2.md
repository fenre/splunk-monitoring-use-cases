# UC-10.6.2 · Mean Time to Remediation

## Description

MTTR measures remediation efficiency. Long MTTR indicates process bottlenecks or resource constraints requiring management attention.

## Value

MTTR measures remediation efficiency. Long MTTR indicates process bottlenecks or resource constraints requiring management attention.

## Implementation

Track first_seen and fixed_date for each vulnerability. Calculate MTTR by severity. Report against SLA targets (Critical: 7d, High: 30d, Medium: 90d). Identify teams with consistently high MTTR for process improvement.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Vuln management TA.
• Ensure the following data sources are available: Scan results with first_seen and last_seen dates.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track first_seen and fixed_date for each vulnerability. Calculate MTTR by severity. Report against SLA targets (Critical: 7d, High: 30d, Medium: 90d). Identify teams with consistently high MTTR for process improvement.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vulnerability sourcetype="tenable:vuln" state="Fixed"
| eval mttr_days=round((fixed_date-first_seen)/86400)
| stats avg(mttr_days) as avg_mttr, median(mttr_days) as median_mttr by severity
```
