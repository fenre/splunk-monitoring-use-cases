# UC-10.11.101 · CrowdStrike Identity Protection Risk Scores

## Description

Identity risk scores prioritize account takeover investigations.

## Value

Identity risk scores prioritize account takeover investigations.

## Implementation

Ingest Falcon Identity Protection events. Escalate high-risk users to IdP step-up.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `TA-crowdstrike-falcon`.
• Ensure the following data sources are available: `sourcetype=crowdstrike:identity`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest Falcon Identity Protection events. Escalate high-risk users to IdP step-up.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=identity sourcetype="crowdstrike:identity"
| stats max(risk_score) as r by user_principal
| where r>70
```
