# UC-10.11.36 · Check Point VPN Tunnel Status

## Description

VPN status protects site-to-site and remote user connectivity.

## Value

VPN status protects site-to-site and remote user connectivity.

## Implementation

Parse VPN tunnel monitoring from gateway logs. Alert on tunnels down past grace period.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_checkpoint`.
• Ensure the following data sources are available: `sourcetype=cp:vpn`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Parse VPN tunnel monitoring from gateway logs. Alert on tunnels down past grace period.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="cp:vpn"
| stats latest(tunnel_state) as st by peer, community
| search st IN ("Down","Init","Problem")
```
