# UC-10.14.11 · Safety Instrumented System (SIS) Bypass

## Description

Monitors mode changes, jumpering, or overrides on SIS (T0800), directly tied to process safety and regulatory expectations.

## Value

Monitors mode changes, jumpering, or overrides on SIS (T0800), directly tied to process safety and regulatory expectations.

## Implementation

Use Splunk ES and OT Safety data integrations. Require MOC/permit correlation. Escalate immediately to process safety and T0800 workflows.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OT Security Add-on for Splunk (Splunkbase 5151).
• Ensure the following data sources are available: `index=ot` `sourcetype="ics:sis"` safety PLC diagnostics, alarm historians.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Use Splunk ES and OT Safety data integrations. Require MOC/permit correlation. Escalate immediately to process safety and T0800 workflows.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="ics:sis"
| where match(status_message, "(?i)bypass|override|maintenance|manual|inhibit")
| lookup ot_sis_authorized_changes loop_id user OUTPUT permit_id valid_until
| eval permit_ok=if(!isnull(permit_id) AND _time<=valid_until, 1, 0)
| where permit_ok=0
| stats earliest(_time) as first_seen values(status_message) as states by loop_id, asset, user
```
