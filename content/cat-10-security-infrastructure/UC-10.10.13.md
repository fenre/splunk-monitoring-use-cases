# UC-10.10.13 · YARA false positive analysis

## Description

Correlates YARA matches with EDR verdicts to label likely FPs.

## Value

Correlates YARA matches with EDR verdicts to label likely FPs.

## Implementation

Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Custom TA / HEC.
• Ensure the following data sources are available: YARA + EDR reputation lookup.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=yara sourcetype=yara:scan match="true"
| lookup edr_file_reputation hash_sha256 OUTPUT verdict
| stats count by rule_name, verdict
| where verdict="clean" OR verdict="unknown"
```
