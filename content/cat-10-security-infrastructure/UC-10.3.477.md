# UC-10.3.477 · Windows Impair Defense Delete Win Defender Context Menu

## Description

The following analytic detects the deletion of the Windows Defender context menu entry from the registry. It leverages data from the Endpoint datamodel, specifically monitoring registry actions where the path includes "*\\shellex\\ContextMenuHandlers\\EPP" and the action is 'deleted'. This activity is significant as it is commonly associated with Remote Access Trojan (RAT) malware attempting to disable security features. ## Detailed Implementation

Enterprise Security Detection Rule

"Windows Impair Defense Delete Win Defender Context Menu" is a Hunting detection designed for analyst-driven threat hunting rather than automated alerting, sourced from the Splunk Enterprise Security Content Update (ESCU). Hunting detections are not intended for automated alerting. They are run on-demand or on a low-frequency schedule by security analysts investigating specific threat hypotheses.

Prerequisites

• Splunk Enterprise Security 7.x or later with the ES Content Update (ESCU) app installed and up to date.
• Data sources: Sysmon EventID 13. Must be ingested into Splunk and normalized to the Common Information Model (CIM) via the appropriate Technology Add-on.
• MITRE ATT&CK: T1562.001. Review the ATT&CK matrix for adjacent techniques to identify detection coverage gaps in your environment.

Deployment

1. In Enterprise Security, navigate to Configure → Content → Content Management.
2. Search for "Windows Impair Defense Delete Win Defender Context Menu" or filter by Analytic Story.
3. Hunting detections are typically left disabled for automated scheduling. Instead, run them on-demand from the Search bar or schedule at low frequency (daily or weekly).
4. Review results manually — hunting detections cast a wider net and expect analyst judgment to separate signal from noise.
5. When results warrant further investigation, create a Notable Event manually or initiate your incident response workflow.

Tuning and False Positive Management

Known false positives for this detection: It is unusual to turn this feature off a Windows system since it is a default security control, although it is not rare for some policies to disable it. Although no false positives have been identified, use the provided filter macro to tune the search.

• Hunting detections are expected to produce broader result sets that require analyst interpretation. Focus on refining the search scope (time range, specific hosts or users) rather than eliminating all noise.
