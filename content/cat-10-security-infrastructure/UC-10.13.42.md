# UC-10.13.42 · Configuration Drift Analysis

## Description

Repeated `change_type=update` on same `object_id` without matching change record — drift vs. process.

## Value

Repeated `change_type=update` on same `object_id` without matching change record — drift vs. process.

## Implementation

Integrate Ansible Tower job IDs as authorized change keys.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Change CIM, config mgmt.
• Ensure the following data sources are available: `object_id`, `object_attrs`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Integrate Ansible Tower job IDs as authorized change keys.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=change change_type=update
| stats count by object_id, user
| where count>5
| lookup snow_changes object_id OUTPUT ticket
| where isnull(ticket)
```
