# UC-10.12.41 · NIST 800-53 Control Validation

## Description

Operationalizes AU, AC, SI, and IR families via automated checks against ingested security telemetry.

## Value

Operationalizes AU, AC, SI, and IR families via automated checks against ingested security telemetry.

## Implementation

Map each control to correlation searches; feed failures into GRC; annual assessor exports.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ES, compliance frameworks.
• Ensure the following data sources are available: Mixed — auth, change, vuln indexes tagged `control_id`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map each control to correlation searches; feed failures into GRC; annual assessor exports.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=nist_800_53
| lookup nist_controls control_id OUTPUT family automated
| where automated="true" AND result="fail"
| stats latest(_time) as last_fail by control_id, host
| sort control_id
```

**Pipeline walkthrough**

• Scopes the data: index=security, tags. Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
• Enriches events using `lookup` (lookup definition + optional OUTPUT fields).
• Filters the current rows with `where automated="true" AND result="fail"` — typically the threshold or rule expression for this monitoring goal.
• `stats` rolls up events into metrics; results are split **by control_id, host** so each row reflects one combination of those dimensions (useful for per-host, per-user, or per-entity comparisons for this use case).
• Orders rows with `sort` — combine with `head`/`tail` for top-N patterns.
