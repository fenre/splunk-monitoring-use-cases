# UC-10.10.5 · MITRE ATT&CK technique coverage gap analysis

## Description

Compares technique IDs from fired notables against desired coverage matrix.

## Value

Compares technique IDs from fired notables against desired coverage matrix.

## Implementation

Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security / custom.
• Ensure the following data sources are available: Notable events with MITRE annotations, lookup of desired coverage.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup desired_mitre_coverage.csv
| fields technique_id
| join type=left max=0 technique_id [
    search `notable` earliest=-365d
    | mvexpand annotations.mitre_attack.mitre_attack_id limit=500
    | rename annotations.mitre_attack.mitre_attack_id as technique_id
    | stats dc(rule_name) as rule_count by technique_id
  ]
| where isnull(rule_count)
```
