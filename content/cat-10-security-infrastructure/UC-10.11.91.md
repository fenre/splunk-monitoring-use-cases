# UC-10.11.91 · FortiAnalyzer Log Forwarding Gap Detection

## Description

Log forwarding gaps mean SOC loses visibility until collectors resume.

## Value

Log forwarding gaps mean SOC loses visibility until collectors resume.

## Implementation

Monitor FortiAnalyzer receive timestamps per device. Alert when expected firewalls stop forwarding.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_fortinet`.
• Ensure the following data sources are available: `sourcetype=fortinet:faz`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Monitor FortiAnalyzer receive timestamps per device. Alert when expected firewalls stop forwarding.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="fortinet:faz"
| stats latest(last_recv) as lr by device
| where now()-lr > 3600
```
