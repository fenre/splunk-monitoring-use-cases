# UC-10.11.54 · ZPA Application Segment Health

## Description

Segment health ensures zero trust access to private apps remains available.

## Value

Segment health ensures zero trust access to private apps remains available.

## Implementation

Ingest ZPA admin and connector telemetry. Alert when application segments degrade or disconnect.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler`.
• Ensure the following data sources are available: `sourcetype=zscaler:zpa`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest ZPA admin and connector telemetry. Alert when application segments degrade or disconnect.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="zscaler:zpa"
| stats latest(segment_status) as st by app_segment
| search st!="HEALTHY"
```
