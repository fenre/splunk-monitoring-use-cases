# UC-10.13.4 · Service Account Interactive Login (CIM)

## Description

Service accounts should not have interactive / remote-interactive logons; indicates credential theft or misconfiguration.

## Value

Service accounts should not have interactive / remote-interactive logons; indicates credential theft or misconfiguration.

## Implementation

Maintain `lookup service_accounts_noninteractive.csv`; exclude break-glass; alert on first-seen interactive.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Windows TA → Authentication.
• Ensure the following data sources are available: Event 4624 with logon type mapped to `signature` or `user`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Maintain `lookup service_accounts_noninteractive.csv`; exclude break-glass; alert on first-seen interactive.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=authentication user="svc_*" OR user="*-svc"
| search signature="*Interactive*" OR app="WinLogon:Interactive"
| stats count by user, src, host
```
