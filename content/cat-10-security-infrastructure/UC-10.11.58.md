# UC-10.11.58 · Zscaler Cloud Firewall Rule Hit Analysis

## Description

Cloud firewall hits extend NGFW-style control to roaming and branch users.

## Value

Cloud firewall hits extend NGFW-style control to roaming and branch users.

## Implementation

Use cloud firewall logs for non-web TCP/UDP. Identify noisy rules and geo anomalies.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler`.
• Ensure the following data sources are available: `sourcetype=zscaler:fw`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Use cloud firewall logs for non-web TCP/UDP. Identify noisy rules and geo anomalies.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=firewall sourcetype="zscaler:fw"
| stats count by rule_label, action, dest_country
| sort -count
```
