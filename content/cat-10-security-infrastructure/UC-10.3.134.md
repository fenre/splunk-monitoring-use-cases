# UC-10.3.134 · Get WMIObject Group Discovery with Script Block Logging

## Description

The following analytic detects the execution of the `Get-WMIObject Win32_Group` command using PowerShell Script Block Logging (EventCode=4104). This method captures the full command sent to PowerShell, allowing for detailed analysis. Identifying group information on an endpoint is not inherently malicious but can be suspicious based on context such as time, endpoint, and user. ## Detailed Implementation

Enterprise Security Detection Rule

"Get WMIObject Group Discovery with Script Block Logging" is a Hunting detection designed for analyst-driven threat hunting rather than automated alerting, sourced from the Splunk Enterprise Security Content Update (ESCU). Hunting detections are not intended for automated alerting. They are run on-demand or on a low-frequency schedule by security analysts investigating specific threat hypotheses.

Prerequisites

• Splunk Enterprise Security 7.x or later with the ES Content Update (ESCU) app installed and up to date.
• Data sources: Powershell Script Block Logging 4104. Must be ingested into Splunk and normalized to the Common Information Model (CIM) via the appropriate Technology Add-on.
• MITRE ATT&CK: T1069.001. Review the ATT&CK matrix for adjacent techniques to identify detection coverage gaps in your environment.

Deployment

1. In Enterprise Security, navigate to Configure → Content → Content Management.
2. Search for "Get WMIObject Group Discovery with Script Block Logging" or filter by Analytic Story.
3. Hunting detections are typically left disabled for automated scheduling. Instead, run them on-demand from the Search bar or schedule at low frequency (daily or weekly).
4. Review results manually — hunting detections cast a wider net and expect analyst judgment to separate signal from noise.
5. When results warrant further investigation, create a Notable Event manually or initiate your incident response workflow.

## SPL

```spl
`powershell` EventCode=4104 ScriptBlockText = "*Get-WMIObject*" AND ScriptBlockText = "*Win32_Group*"
      | fillnull
      | stats count min(_time) as firstTime max(_time) as lastTime
        BY dest signature signature_id
           user_id vendor_product EventID
           Guid Opcode Name
           Path ProcessID ScriptBlockId
           ScriptBlockText
      | `security_content_ctime(firstTime)`
      | `security_content_ctime(lastTime)`
      | `get_wmiobject_group_discovery_with_script_block_logging_filter`
```
