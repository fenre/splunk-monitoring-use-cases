# UC-10.13.29 · Vulnerability Scan Gap Detection

## Description

Assets in CMDB without recent vuln assessment event in CIM — scanning blind spot.

## Value

Assets in CMDB without recent vuln assessment event in CIM — scanning blind spot.

## Implementation

Exclude decommissioned; alert on prod tier >14d gap.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Vulnerabilities CIM + inventory.
• Ensure the following data sources are available: Last seen scan per `dest`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Exclude decommissioned; alert on prod tier >14d gap.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=vulnerability
| stats latest(_time) as last_scan by dest
| eval gap_days=(now()-last_scan)/86400
| where gap_days>30
| lookup cmdb_servers hostname as dest OUTPUT owner
```
