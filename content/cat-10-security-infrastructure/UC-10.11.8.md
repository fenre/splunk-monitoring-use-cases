# UC-10.11.8 · FortiGate HA Failover Events

## Description

Unexpected HA events can indicate split-brain risk or hardware stress on clustered firewalls.

## Value

Unexpected HA events can indicate split-brain risk or hardware stress on clustered firewalls.

## Implementation

Parse HA role change, sync loss, and failover messages. Alert on unplanned failovers and track frequency for stability reviews.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_fortinet`.
• Ensure the following data sources are available: `sourcetype=fortinet:fortigate_event`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Parse HA role change, sync loss, and failover messages. Alert on unplanned failovers and track frequency for stability reviews.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="fortinet:fortigate_event" (logdesc="*HA*" OR subtype="ha")
| stats count by action, group_id
| sort -_time
```
