# UC-10.11.100 · Check Point Mobile Access VPN Session Anomalies

## Description

Mobile VPN anomalies may indicate account sharing or credential theft.

## Value

Mobile VPN anomalies may indicate account sharing or credential theft.

## Implementation

Detect impossible travel for mobile VPN users. Correlate with IdP signals.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_checkpoint`.
• Ensure the following data sources are available: `sourcetype=cp:mobile`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Detect impossible travel for mobile VPN users. Correlate with IdP signals.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="cp:mobile"
| stats dc(public_ip) as locs by user
| where locs>3
```
