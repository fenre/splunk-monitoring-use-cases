# UC-10.17.6 · SonicWall SSL-VPN Concurrent Sessions vs Licensed Capacity (SonicWall)

## Description

VPN license exhaustion blocks remote workers during incidents or weather events. Session counts should be tracked like any other capacity metric.

## Value

Prevents emergency license purchases and documents growth for budget planning.

## Implementation

Enable SSL-VPN session logging. If only available via SNMP/API, ingest via scripted input into `sourcetype=sonicwall:vpn:metrics`.

## SPL

```spl
index=netops sourcetype="dell:sonicwall" earliest=-4h
| where match(_raw,"(?i)ssl-vpn|NetExtender|session")
| rex field=_raw "(?i)active[\s_]*sessions[\s:=]+(?<sessions>\d+)"
| stats latest(sessions) as active_sessions by host
| eval license_limit=250
| eval pct=round(100*active_sessions/license_limit,1)
| where pct>=80
```

## Visualization

Single value (sessions vs cap), timechart (concurrent sessions).
