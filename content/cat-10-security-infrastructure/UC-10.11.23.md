# UC-10.11.23 · Palo Alto Data Filtering Policy Violations

## Description

Data filtering catches sensitive file egress at the NGFW edge.

## Value

Data filtering catches sensitive file egress at the NGFW edge.

## Implementation

Enable data filtering profiles on sensitive rules. Log file transfers with patterns matched. Alert on blocks to critical destinations.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_paloalto`.
• Ensure the following data sources are available: `sourcetype=pan:data`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Enable data filtering profiles on sensitive rules. Log file transfers with patterns matched. Alert on blocks to critical destinations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=pan sourcetype="pan:data" (action=alert OR action=block)
| stats count by filter, file_type, user
| sort -count
```
