# UC-10.11.74 · Tenable Remediation SLA Monitoring

## Description

SLA monitoring drives accountability for patch and mitigation timelines.

## Value

SLA monitoring drives accountability for patch and mitigation timelines.

## Implementation

Join vulnerability records to SLA lookup by severity. Escalate overdue remediations.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_nessus`.
• Ensure the following data sources are available: `sourcetype=tenable:remediation`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Join vulnerability records to SLA lookup by severity. Escalate overdue remediations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vuln sourcetype="tenable:remediation"
| eval breach=if((now()-detected_time) > 604800,1,0)
| stats sum(breach) as breaches by owner
| where breaches>0
```
