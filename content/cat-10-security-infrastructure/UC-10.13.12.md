# UC-10.13.12 · Internal-to-External Traffic Ratio (CIM)

## Description

Sudden shift in egress ratio can signal C2 or misrouted backups.

## Value

Sudden shift in egress ratio can signal C2 or misrouted backups.

## Implementation

Alert on ratio change >30% vs. weekly baseline.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: NetFlow / firewall CIM.
• Ensure the following data sources are available: `src`/`dest` with RFC1918 classification.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Alert on ratio change >30% vs. weekly baseline.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=network
| eval src_int=if(cidrmatch(src,"10.0.0.0/8") OR cidrmatch(src,"192.168.0.0/16"),1,0)
| eval dest_ext=if(NOT cidrmatch(dest,"10.0.0.0/8"),1,0)
| timechart span=15m sum(eval(src_int=1 AND dest_ext=1)) as egress sum(bytes) as total
| eval ratio=egress/total
```
