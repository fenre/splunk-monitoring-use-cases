# UC-10.14.3 · Default or Shared Credentials on OT Devices

## Description

Surfaces authentication using vendor default or shared accounts on PLCs, RTUs, and HMIs (T0812), enabling trivial takeover and process manipulation.

## Value

Surfaces authentication using vendor default or shared accounts on PLCs, RTUs, and HMIs (T0812), enabling trivial takeover and process manipulation.

## Implementation

Integrate OT authentication logs with ES. Pair with password policy and account governance. Escalate repeated default-credential use referencing T0812.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OT Security Add-on for Splunk (Splunkbase 5151).
• Ensure the following data sources are available: `index=ot_security` `sourcetype="ics:auth"` RADIUS/TACACS+, HMI/SCADA login logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Integrate OT authentication logs with ES. Pair with password policy and account governance. Escalate repeated default-credential use referencing T0812.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_security sourcetype="ics:auth"
| eval user=lower(trim(user))
| lookup ot_default_creds user OUTPUT is_default
| where is_default=1 OR match(user, "admin|root|operator|guest")
| stats count by src, dest_host, user, vendor
| sort - count
```
