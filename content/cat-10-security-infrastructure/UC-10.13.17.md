# UC-10.13.17 · Recurring Infection (Same Host)

## Description

Repeat malware on same `dest` indicates failed remediation or persistent foothold.

## Value

Repeat malware on same `dest` indicates failed remediation or persistent foothold.

## Implementation

Auto-ticket reimage workflow when count > threshold in 7d.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Malware CIM.
• Ensure the following data sources are available: `Malware_Attacks.dest`, `signature`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Auto-ticket reimage workflow when count > threshold in 7d.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=malware
| stats count by dest, signature
| where count>3
```
