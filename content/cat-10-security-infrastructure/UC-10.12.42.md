# UC-10.12.42 · CAC Authentication Monitoring

## Description

Tracks smart card logon failures, PIN lockouts, and certificate issues for DOD/IC access.

## Value

Tracks smart card logon failures, PIN lockouts, and certificate issues for DOD/IC access.

## Implementation

Correlate with HSM and CRL availability; help desk runbooks for YubiKey/CAC readers.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Windows Security, PKI.
• Ensure the following data sources are available: `sourcetype="WinEventLog:Security"` EventCode=4768,4769.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Correlate with HSM and CRL availability; help desk runbooks for YubiKey/CAC readers.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ad sourcetype="WinEventLog:Security" EventCode IN (4768,4769,4776)
| search Certificate_Information="*"
| eval fail=if(match(Status,"0x[1-9a-f]"),1,0)
| timechart span=15m sum(fail) as card_failures, count as attempts
| where card_failures/attempts > 0.1
```
