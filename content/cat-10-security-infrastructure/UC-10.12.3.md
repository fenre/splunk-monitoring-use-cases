# UC-10.12.3 · Credit Card Velocity Checks

## Description

Detects card testing and enumeration via rapid auth attempts across merchants or regions.

## Value

Detects card testing and enumeration via rapid auth attempts across merchants or regions.

## Implementation

Tokenize PANs; never log full track data. Thresholds tuned per BIN and channel. Feed declines into case management for issuer contact.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Payment processor TA, gateway logs.
• Ensure the following data sources are available: `sourcetype="card:authorization"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Tokenize PANs; never log full track data. Thresholds tuned per BIN and channel. Feed declines into case management for issuer contact.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=payments sourcetype="card:authorization"
| bin _time span=15m
| stats count, dc(merchant_id) as merchants, dc(auth_result) as outcomes by card_token, _time
| where count>=8 AND merchants>=4
| sort -count
```
