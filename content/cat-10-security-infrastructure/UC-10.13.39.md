# UC-10.13.39 · Change Velocity Tracking

## Description

Count of `All_Changes` per day — spikes suggest emergency change storms or automation gone wrong.

## Value

Count of `All_Changes` per day — spikes suggest emergency change storms or automation gone wrong.

## Implementation

Compare to CAB-approved schedule; flag off-window bursts.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: ServiceNow, Ansible → Change.
• Ensure the following data sources are available: `All_Changes` with `change_type`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Compare to CAB-approved schedule; flag off-window bursts.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=change
| timechart span=1d count by change_type
```
