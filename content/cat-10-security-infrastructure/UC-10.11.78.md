# UC-10.11.78 · Tenable Asset Discovery Reconciliation

## Description

Asset reconciliation aligns VM scope with actual infrastructure for accurate risk.

## Value

Asset reconciliation aligns VM scope with actual infrastructure for accurate risk.

## Implementation

Compare Tenable-discovered assets to CMDB. Find missing or duplicate records.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_nessus`.
• Ensure the following data sources are available: `sourcetype=tenable:asset`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Compare Tenable-discovered assets to CMDB. Find missing or duplicate records.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vuln sourcetype="tenable:asset"
| stats values(ip) as ips by hostname
| lookup cmdb_hosts hostname OUTPUT expected_ip
| where isnull(expected_ip) OR mvfind(ips, expected_ip)<0
```
