# UC-10.16.5 · Phishing Attempt Volume Trending

## Description

Weekly volume of blocked phishing attempts indicates campaign intensity and user targeting. Spikes justify awareness programs, mailbox rule reviews, and gateway tuning without waiting for a successful compromise.

## Value

Weekly volume of blocked phishing attempts indicates campaign intensity and user targeting. Spikes justify awareness programs, mailbox rule reviews, and gateway tuning without waiting for a successful compromise.

## Implementation

Normalize vendor-specific threat classifications into a `threat_type` field at ingest. Deduplicate by message ID where possible. Correlate with `index=notable` for user-reported phishing that bypassed automation. Refresh weekly in SOC metrics reviews alongside click-through simulations.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365, Proofpoint TA, Cisco ESA TA, Splunk Enterprise Security.
• Ensure the following data sources are available: `index=email` or secure email gateway sourcetypes, `Alerts` data model–eligible email threat events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize vendor-specific threat classifications into a `threat_type` field at ingest. Deduplicate by message ID where possible. Correlate with `index=notable` for user-reported phishing that bypassed automation. Refresh weekly in SOC metrics reviews alongside click-through simulations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=email OR index=main (sourcetype="ms:o365:reporting:messagetrace" OR sourcetype="proofpoint:ppt:msg" OR sourcetype="cisco:esa:mail")
| search threat_type="*phish*" OR category="phishing" OR verdict="malicious"
| bin _time span=1w
| stats count as blocked_phish by _time
| timechart span=1w sum(blocked_phish) as phishing_blocked
```
