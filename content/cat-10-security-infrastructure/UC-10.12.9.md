# UC-10.12.9 · KYC Continuous Monitoring

## Description

Detects adverse media, sanctions hits, or identity churn post-onboarding.

## Value

Detects adverse media, sanctions hits, or identity churn post-onboarding.

## Implementation

Refresh high-risk customers on schedule; integrate watchlist updates; escalate unresolved alerts > SLA.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: KYC vendor API logs, case management.
• Ensure the following data sources are available: `sourcetype="kyc:screening"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Refresh high-risk customers on schedule; integrate watchlist updates; escalate unresolved alerts > SLA.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=compliance sourcetype="kyc:screening"
| where match(alert_type,"(sanctions|pep|adverse)") AND disposition="open"
| stats latest(_time) as last_alert, values(match_score) as scores by customer_id, case_id
| where last_alert < relative_time(now(),"-90d@d")
| sort last_alert
```
