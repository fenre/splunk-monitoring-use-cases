# UC-10.11.126 · Check Point Threat Prevention Policy Layer Effectiveness

## Description

Check Point Threat Prevention operates through policy layers (IPS, Anti-Virus, Anti-Bot, Threat Emulation, Threat Extraction). Measuring detection and prevention rates per layer validates that each security function is contributing and identifies layers that may need tuning.

## Value

Check Point Threat Prevention operates through policy layers (IPS, Anti-Virus, Anti-Bot, Threat Emulation, Threat Extraction). Measuring detection and prevention rates per layer validates that each security function is contributing and identifies layers that may need tuning.

## Implementation

Ensure all Threat Prevention blades log to Splunk. Compare prevention rates across blades. Alert when any blade's detection count drops to zero (possibly disabled). Report on blade effectiveness for security investment justification and compliance.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_checkpoint`.
• Ensure the following data sources are available: `sourcetype=cp_log` (all Threat Prevention blade logs).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ensure all Threat Prevention blades log to Splunk. Compare prevention rates across blades. Alert when any blade's detection count drops to zero (possibly disabled). Report on blade effectiveness for security investment justification and compliance.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security sourcetype="cp_log" earliest=-30d
| where match(lower(product),"(?i)ips|anti.?virus|anti.?bot|threat.?emulation|threat.?extraction")
| eval blade=lower(product)
| eval prevented=if(match(lower(action),"(?i)prevent|block|drop"),1,0)
| stats count sum(prevented) as prevents by blade
| eval prevention_rate=round(100*prevents/count,1)
| sort -count
```
