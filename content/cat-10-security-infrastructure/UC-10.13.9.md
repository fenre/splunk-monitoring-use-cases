# UC-10.13.9 · Top Talkers Analysis (CIM)

## Description

Baselines bytes and sessions by `src`/`dest` for capacity and exfiltration hunting.

## Value

Baselines bytes and sessions by `src`/`dest` for capacity and exfiltration hunting.

## Implementation

Use summaries; split internal vs. external via `dest_zone` or RFC1918 lookups.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: NetFlow, firewall TAs → Network_Traffic.
• Ensure the following data sources are available: `All_Traffic` CIM-tagged events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Use summaries; split internal vs. external via `dest_zone` or RFC1918 lookups.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=network tag=communicate
| stats sum(bytes) as b by src dest
| sort -b
| head 100
```

Understanding this SPL

**Top Talkers Analysis (CIM)** — Baselines bytes and sessions by `src`/`dest` for capacity and exfiltration hunting.

Documented **Data sources**: `All_Traffic` CIM-tagged events. **App/TA** (typical add-on context): NetFlow, firewall TAs → Network_Traffic. The SPL below should target the same indexes and sourcetypes you configured for that feed—rename `index=` / `sourcetype=` if your deployment differs.

The first pipeline stage scopes events using **index**: security.

**Pipeline walkthrough**

• Scopes the data: index=security, tags. Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
• `stats` rolls up events into metrics; results are split **by src dest** so each row reflects one combination of those dimensions (useful for per-host, per-user, or per-entity comparisons for this use case).
• Orders rows with `sort` — combine with `head`/`tail` for top-N patterns.
• Limits the number of rows with `head`.

Optional CIM / accelerated variant (same use case, normalized fields via Common Information Model):

```spl
| tstats `summariesonly` sum(All_Traffic.bytes_in) as bytes_in, sum(All_Traffic.bytes_out) as bytes_out from datamodel=Network_Traffic.All_Traffic by All_Traffic.src, All_Traffic.dest
| eval bytes=bytes_in+bytes_out
| sort -bytes
| head 100
```
