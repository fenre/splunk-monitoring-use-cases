# UC-10.12.6 · Card-Not-Present Transaction Spikes

## Description

E-commerce CNP spikes often precede credential stuffing or BIN attacks.

## Value

E-commerce CNP spikes often precede credential stuffing or BIN attacks.

## Implementation

Segment by merchant and MCC; seasonal baselines for retail peaks. Correlate with 3DS failures and chargeback velocity.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Gateway TA, e-commerce platform.
• Ensure the following data sources are available: `sourcetype="gateway:txn"`, `card_present="false"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Segment by merchant and MCC; seasonal baselines for retail peaks. Correlate with 3DS failures and chargeback velocity.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=payments sourcetype="gateway:txn" card_present="false"
| timechart span=1h sum(amount) as gmv, count as txns
| eventstats avg(txns) as baseline, stdev(txns) as s
| eval z=(txns-baseline)/s
| where abs(z)>3
```

**Pipeline walkthrough**

• `timechart` plots the metric over time using **span=1h** buckets — ideal for trending and alerting on this use case.
• `eventstats` aggregates the pipeline (counts, distinct values, sums, percentiles, etc.) into fewer rows.
• `eval` defines or adjusts **z** — often to normalize units, derive a ratio, or prepare for thresholds.
• Filters the current rows with `where abs(z)>3` — typically the threshold or rule expression for this monitoring goal.
