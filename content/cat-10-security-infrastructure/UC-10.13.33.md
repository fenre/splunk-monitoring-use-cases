# UC-10.13.33 · Delivery Failure Trending

## Description

Rises in `action` bounce/drop indicate reputation issues, DNS problems, or attacks.

## Value

Rises in `action` bounce/drop indicate reputation issues, DNS problems, or attacks.

## Implementation

Correlate with SPF/DKIM failures in same window.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Email CIM with `action` mapped.
• Ensure the following data sources are available: DSN / MTA logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Correlate with SPF/DKIM failures in same window.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=email action=bounced OR status=deferred
| timechart span=15m count by signature
```

Understanding this SPL

**Delivery Failure Trending** — Rises in `action` bounce/drop indicate reputation issues, DNS problems, or attacks.

Documented **Data sources**: DSN / MTA logs. **App/TA** (typical add-on context): Email CIM with `action` mapped. The SPL below should target the same indexes and sourcetypes you configured for that feed—rename `index=` / `sourcetype=` if your deployment differs.

The first pipeline stage scopes events using **index**: security.

**Pipeline walkthrough**

• Scopes the data: index=security, tags. Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
• `timechart` plots the metric over time using **span=15m** buckets with a separate series **by signature** — ideal for trending and alerting on this use case.

Optional CIM / accelerated variant (same use case, normalized fields via Common Information Model):

