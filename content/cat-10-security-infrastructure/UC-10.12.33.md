# UC-10.12.33 · E-Commerce Bot Detection

## Description

Protects inventory, loyalty points, and checkout from scalping and credential stuffing.

## Value

Protects inventory, loyalty points, and checkout from scalping and credential stuffing.

## Implementation

Layer bot management (JS challenge, CAPTCHA); rate limit by ASN and fingerprint.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: CDN/WAF, application logs.
• Ensure the following data sources are available: `sourcetype="nginx:access"`, `sourcetype="akamai:siem"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Layer bot management (JS challenge, CAPTCHA); rate limit by ASN and fingerprint.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=web sourcetype="nginx:access" uri="/checkout*"
| eval bot_score=if(match(http_user_agent,"(?i)(bot|crawler|python)"),1,0)
| stats count, dc(session_id) as sess, avg(request_time_ms) as avg_rt by src
| where count>200 AND sess<5 AND avg_rt<50
| sort -count
```
