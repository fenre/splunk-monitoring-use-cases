# UC-10.10.11 · YARA scan coverage tracking

## Description

Percentage of endpoints or paths scanned per interval vs inventory.

## Value

Percentage of endpoints or paths scanned per interval vs inventory.

## Implementation

Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Custom TA / HEC.
• Ensure the following data sources are available: YARA scans plus CMDB lookup.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=yara sourcetype=yara:scan
| stats dc(host) as hosts_scanned
| append [ | inputlookup server_inventory.csv | stats dc(host) as hosts_total ]
| eval coverage_pct=round(100*hosts_scanned/hosts_total,2)
| table hosts_scanned, hosts_total, coverage_pct
```
