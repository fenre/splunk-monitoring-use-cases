# UC-10.11.93 · Palo Alto Certificate Inspection and Untrusted Issuer Alerts

## Description

Certificate anomalies highlight decryption failures and potential adversary-in-the-middle.

## Value

Certificate anomalies highlight decryption failures and potential adversary-in-the-middle.

## Implementation

Review untrusted or expired certificates in decryption logs. May indicate MITM or misconfig.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_paloalto`.
• Ensure the following data sources are available: `sourcetype=pan:decryption`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Review untrusted or expired certificates in decryption logs. May indicate MITM or misconfig.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=pan sourcetype="pan:decryption" cert_status!="trusted"
| stats count by issuer, dest
| sort -count
```
