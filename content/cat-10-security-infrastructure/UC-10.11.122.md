# UC-10.11.122 · Check Point ThreatCloud IOC Match Rate

## Description

ThreatCloud is Check Point's global threat intelligence feed powering all prevention blades. Match rate trending validates that protections are active and that feed updates are being applied. A drop in match rate may indicate feed delivery failure or policy misconfiguration.

## Value

ThreatCloud is Check Point's global threat intelligence feed powering all prevention blades. Match rate trending validates that protections are active and that feed updates are being applied. A drop in match rate may indicate feed delivery failure or policy misconfiguration.

## Implementation

Baseline daily detection count per blade (IPS, Anti-Virus, Anti-Bot, Threat Emulation). Alert when daily count drops >50% vs 7-day average — indicates possible feed or blade issue. Report on detection mix for security posture assessment.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_checkpoint`.
• Ensure the following data sources are available: `sourcetype=cp_log` (IPS/AV/Anti-Bot logs with ThreatCloud indicators).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Baseline daily detection count per blade (IPS, Anti-Virus, Anti-Bot, Threat Emulation). Alert when daily count drops >50% vs 7-day average — indicates possible feed or blade issue. Report on detection mix for security posture assessment.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security sourcetype="cp_log" earliest=-7d
| where isnotnull(protection_name) AND match(lower(action),"(?i)prevent|block|detect|drop")
| timechart span=1d count by product
```
