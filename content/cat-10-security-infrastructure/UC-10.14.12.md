# UC-10.14.12 · Unauthorized VPN Tunnel into OT Network

## Description

Detects VPN or encrypted sessions into OT that violate perimeter policy (T0886), supporting regulatory and segmentation requirements.

## Value

Detects VPN or encrypted sessions into OT that violate perimeter policy (T0886), supporting regulatory and segmentation requirements.

## Implementation

Maintain approved vendor and employee VPN profiles. Alert on split tunnel to OT, new device IDs. Map to T0886.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OT Security Add-on for Splunk (Splunkbase 5151).
• Ensure the following data sources are available: `index=ot_security` VPN concentrator logs, firewall sessions.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Maintain approved vendor and employee VPN profiles. Alert on split tunnel to OT, new device IDs. Map to T0886.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_security sourcetype="ot:security:vpn" OR (index=ot sourcetype="ics:firewall" dest_port IN (443, 1194, 4500))
| lookup ot_approved_vpn_peers remote_ip user OUTPUT approved_site
| where isnull(approved_site)
| stats earliest(_time) as start latest(_time) as end by remote_ip, user, assigned_ip, gateway
```
