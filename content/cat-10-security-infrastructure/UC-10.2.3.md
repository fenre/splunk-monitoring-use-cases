# UC-10.2.3 · Signature Coverage Gaps

## Description

Identifying network segments without IDS coverage ensures comprehensive threat detection across the infrastructure.

## Value

Identifying network segments without IDS coverage ensures comprehensive threat detection across the infrastructure.

## Implementation

Maintain network segment inventory with expected IDS sensor mapping. Compare against actual alert data. Alert when a segment has no IDS events for >7 days (sensor may be down or misconfigured).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: IDS sensor health monitoring.
• Ensure the following data sources are available: Sensor health reports, network segment inventory.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Maintain network segment inventory with expected IDS sensor mapping. Compare against actual alert data. Alert when a segment has no IDS events for >7 days (sensor may be down or misconfigured).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup network_segments.csv
| join type=left max=1 segment_name
    [search index=ids sourcetype="snort:alert" earliest=-7d
     | stats count by sensor, segment_name]
| where isnull(count) OR count=0
| table segment_name, expected_sensor, count
```
