# UC-10.8.38 · Data Loss Prevention Policy Violation Trending

## Description

DLP violations indicate policy gaps or insider risk. Trending by policy, user, and channel supports tuning and incident prioritization.

## Value

DLP violations indicate policy gaps or insider risk. Trending by policy, user, and channel supports tuning and incident prioritization.

## Implementation

Ingest DLP alerts from email, endpoint, and cloud channels. Classify by policy and severity. Alert on spike in violations or repeated offenders. Report on top policies and channels.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: DLP product logs, email/cloud TAs.
• Ensure the following data sources are available: DLP alert logs, policy evaluation events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest DLP alerts from email, endpoint, and cloud channels. Classify by policy and severity. Alert on spike in violations or repeated offenders. Report on top policies and channels.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=dlp sourcetype="dlp:alert"
| timechart span=1h count by policy_name
| where count > 0
```
