# UC-10.12.1 · ATM Fraud Pattern Detection

## Description

Correlates card-present withdrawals with geographic velocity and device tamper signals to catch skimming and cash-out campaigns before losses spread.

## Value

Correlates card-present withdrawals with geographic velocity and device tamper signals to catch skimming and cash-out campaigns before losses spread.

## Implementation

Ingest ATM host and authorization logs with terminal ID, amount, response code, and geo. Baseline per-terminal throughput; alert on velocity across many ATMs or impossible travel vs. last chip read.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: ATM switch logs, core banking TA, ISO 8583 parsers.
• Ensure the following data sources are available: `sourcetype="atm:transaction"`, `sourcetype="iso8583:auth"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest ATM host and authorization logs with terminal ID, amount, response code, and geo. Baseline per-terminal throughput; alert on velocity across many ATMs or impossible travel vs. last chip read.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=payments sourcetype IN ("atm:transaction","iso8583:auth") txn_type="withdrawal"
| eval hour=strftime(_time,"%H")
| stats count, dc(atm_id) as atms, values(card_hash) as cards by src_merchant_id, hour
| where count>20 AND atms>5
| sort -count
```
