# UC-10.13.25 · Vulnerability Age Tracking

## Description

Time open since `first_found` drives SLA and board reporting.

## Value

Time open since `first_found` drives SLA and board reporting.

## Implementation

Join CVSS for prioritization; exclude accepted risks with GRC ID.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Tenable, Qualys → Vulnerabilities.
• Ensure the following data sources are available: `Vulnerabilities` CIM with `status`, `cve`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Join CVSS for prioritization; exclude accepted risks with GRC ID.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=vulnerability status=open
| eval age_days=(now()-strptime(first_found,"%Y-%m-%d"))/86400
| where age_days>90
| stats sum(age_days) as total_age by dest, cve
```
