# UC-10.11.67 · Carbon Black Sensor Communication Health

## Description

Sensor communication health preserves visibility and response capability.

## Value

Sensor communication health preserves visibility and response capability.

## Implementation

Monitor sensor check-in and upgrade state. Alert on sensors not communicating >24h.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_vmware_carbonblack`.
• Ensure the following data sources are available: `sourcetype=carbonblack:sensor`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Monitor sensor check-in and upgrade state. Alert on sensors not communicating >24h.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=endpoint sourcetype="carbonblack:sensor"
| stats latest(comm_status) as cs by hostname
| search cs!="active"
```
