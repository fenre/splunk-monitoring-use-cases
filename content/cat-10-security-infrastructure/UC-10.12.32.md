# UC-10.12.32 · Gift Card Fraud Patterns

## Description

Detects bulk activation, balance draining, and reseller abuse patterns.

## Value

Detects bulk activation, balance draining, and reseller abuse patterns.

## Implementation

Velocity limits per device; block serial activations from single payment instrument across stores.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Gift card processor.
• Ensure the following data sources are available: `sourcetype="giftcard:event"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Velocity limits per device; block serial activations from single payment instrument across stores.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=retail sourcetype="giftcard:event"
| search event IN ("ACTIVATE","REDEEM","TRANSFER")
| stats sum(amount) as flow, dc(card_id) as cards by customer_hash, _time
| bin _time span=1h
| where flow>5000 AND cards>10
| sort -flow
```

**Pipeline walkthrough**

• Scopes the data: index=retail, sourcetype="giftcard:event". Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
• Applies an explicit `search` filter to narrow the current result set.
• `stats` rolls up events into metrics; results are split **by customer_hash, _time** so each row reflects one combination of those dimensions (useful for per-host, per-user, or per-entity comparisons for this use case).
• Discretizes time or numeric ranges with `bin`/`bucket`.
• Filters the current rows with `where flow>5000 AND cards>10` — typically the threshold or rule expression for this monitoring goal.
• Orders rows with `sort` — combine with `head`/`tail` for top-N patterns.
