# UC-10.10.3 · Stale detection identification (rules never firing 90+ days)

## Description

Lists correlation searches with zero or near-zero hits over 90 days for retirement review.

## Value

Lists correlation searches with zero or near-zero hits over 90 days for retirement review.

## Implementation

Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security / custom.
• Ensure the following data sources are available: Summary index populated from savedsearch scheduler or ES introspection.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=summary sourcetype=detection_metrics earliest=-90d
| stats sum(hit_count) as hits by rule_name
| where hits=0 OR hits<1
| sort rule_name
```
