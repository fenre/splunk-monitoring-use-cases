# UC-10.11.51 · ZIA Web Policy Violation Trending

## Description

Web policy trending shows how often controls block risky or non-compliant traffic.

## Value

Web policy trending shows how often controls block risky or non-compliant traffic.

## Implementation

Ingest ZIA NSS or log streaming. Normalize rule and URL category. Tune noisy categories.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler`.
• Ensure the following data sources are available: `sourcetype=zscaler:web`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest ZIA NSS or log streaming. Normalize rule and URL category. Tune noisy categories.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=proxy sourcetype="zscaler:web" action=BLOCKED
| timechart span=1h count by rule_name
| where count>0
```
