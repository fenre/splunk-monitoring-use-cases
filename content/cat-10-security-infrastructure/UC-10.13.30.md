# UC-10.13.30 · Email Volume Anomaly

## Description

Spike in `Email` model message count may indicate spam flood, phish campaign, or misconfigured relay.

## Value

Spike in `Email` model message count may indicate spam flood, phish campaign, or misconfigured relay.

## Implementation

Seasonal business hours in `predict`; whitelist marketing bursts via subject lookup.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Exchange, Proofpoint → Email.
• Ensure the following data sources are available: `Email` CIM.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Seasonal business hours in `predict`; whitelist marketing bursts via subject lookup.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=email
| timechart span=1h count
| predict count as prediction algorithm=LLP future_timespan=0
| where count > 2*prediction
```
