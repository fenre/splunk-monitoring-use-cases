# UC-10.13.37 · Bandwidth by Category

## Description

When `app`/`url` category exists in CIM or via lookup — top GB by category.

## Value

When `app`/`url` category exists in CIM or via lookup — top GB by category.

## Implementation

Normalize CDN cache hits; exclude software update domains.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Proxy → Web / Network_Traffic.
• Ensure the following data sources are available: `bytes_in`, `bytes_out` on Web events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize CDN cache hits; exclude software update domains.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=web
| eval bytes=bytes_in+bytes_out
| lookup proxy_category url OUTPUT category
| stats sum(bytes) as b by category
| sort -b
```

Understanding this SPL

**Bandwidth by Category** — When `app`/`url` category exists in CIM or via lookup — top GB by category.

The first pipeline stage scopes events using **index**: security.
