# UC-10.13.43 · Change Window Violations

## Description

Production changes outside approved windows — `_time` vs. lookup `allowed_window`.

## Value

Production changes outside approved windows — `_time` vs. lookup `allowed_window`.

## Implementation

Emergency flag in ticket bypasses alert.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Change CIM.
• Ensure the following data sources are available: `result`, `_time`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Emergency flag in ticket bypasses alert.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=change dest_zone="prod"
| eval dow=strftime(_time,"%u"), hr=strftime(_time,"%H")
| lookup change_windows dow hr OUTPUT allowed
| where allowed!="true"
| stats count by user, object_id
```
