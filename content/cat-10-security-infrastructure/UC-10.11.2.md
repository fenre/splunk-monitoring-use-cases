# UC-10.11.2 · FortiGate IPS Event Trending

## Description

IPS trending highlights exploit attempts and helps tune profiles and exception lists.

## Value

IPS trending highlights exploit attempts and helps tune profiles and exception lists.

## Implementation

Enable IPS logging on relevant policies. Normalize severity and attack name fields from the TA. Trend top signatures and alert on critical severity bursts.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_fortinet`.
• Ensure the following data sources are available: `sourcetype=fortinet:fortigate_ips`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Enable IPS logging on relevant policies. Normalize severity and attack name fields from the TA. Trend top signatures and alert on critical severity bursts.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=firewall sourcetype="fortinet:fortigate_ips"
| stats count by attack, severity, src
| sort 100 -count
```
