# UC-10.12.17 · HIPAA Transmission Security

## Description

Detects cleartext PHI on protocols or TLS versions below policy (e.g., TLS 1.0).

## Value

Detects cleartext PHI on protocols or TLS versions below policy (e.g., TLS 1.0).

## Implementation

Block weak ciphers at edge; exception process for legacy devices with compensating controls.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Load balancer, TLS inspection (where lawful).
• Ensure the following data sources are available: `sourcetype="lb:access"`, `sourcetype="zeek:ssl"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Block weak ciphers at edge; exception process for legacy devices with compensating controls.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="zeek:ssl" (server_name="*fhir*" OR server_name="*ccd*")
| where version IN ("TLSv1","TLSv1.1") OR match(cipher, "RC4|DES")
| stats count by src, dest, version, cipher
| sort -count
```
