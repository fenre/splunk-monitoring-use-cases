# UC-10.10.6 · Custom detection vs ESCU baseline comparison

## Description

Side-by-side hit volume and severity for custom rules vs ESCU-packaged searches.

## Value

Side-by-side hit volume and severity for custom rules vs ESCU-packaged searches.

## Implementation

Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security / custom.
• Ensure the following data sources are available: Summary metrics keyed by rule_name.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=summary sourcetype=detection_metrics
| eval family=if(match(rule_name, "(?i)escu"), "ESCU", "Custom")
| timechart span=1d sum(hit_count) by family
```
