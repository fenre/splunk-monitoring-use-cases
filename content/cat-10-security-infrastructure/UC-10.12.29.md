# UC-10.12.29 · Clinical Alert Fatigue Analysis

## Description

Quantifies alert acknowledgement times and duplicate fires to guide CDS tuning and Joint Commission alarm safety.

## Value

Quantifies alert acknowledgement times and duplicate fires to guide CDS tuning and Joint Commission alarm safety.

## Implementation

Deprecate low-value rules; correlate with nurse staffing ratios from census feeds.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: EHR alerting, middleware.
• Ensure the following data sources are available: `sourcetype="epic:alert"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Deprecate low-value rules; correlate with nurse staffing ratios from census feeds.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=clinical sourcetype="epic:alert"
| eval ack_sec=ack_time-_time
| stats median(ack_sec) as med_ack, count as fires, dc(alert_id) as rules by unit
| eval fires_per_bed=fires/bed_count
| sort -fires
```
