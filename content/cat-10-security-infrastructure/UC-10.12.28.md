# UC-10.12.28 · Pharmacy Dispensing Audit

## Description

Tracks controlled substance dispensing overrides and after-hours fills for diversion programs.

## Value

Tracks controlled substance dispensing overrides and after-hours fills for diversion programs.

## Implementation

Align with state PDMP reporting; peer review for high override rates.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Pharmacy system (Pyxis, Epic Willow).
• Ensure the following data sources are available: `sourcetype="pharmacy:dispense"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Align with state PDMP reporting; peer review for high override rates.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=clinical sourcetype="pharmacy:dispense" dea_schedule IN ("2","3","4")
| search override_flag="true" OR fill_hour<6 OR fill_hour>22
| stats count, sum(quantity) as units by drug_name, pharmacist_id, patient_hash
| sort -units
```
