# UC-10.6.7 · New Vulnerability Detection

## Description

Newly discovered critical vulnerabilities require immediate triage. Alerting ensures rapid response to emerging risks.

## Value

Newly discovered critical vulnerabilities require immediate triage. Alerting ensures rapid response to emerging risks.

## Implementation

After each scan, identify new critical/high vulnerabilities (first_seen within scan window). Alert immediately on new critical findings. Include CVE details and affected hosts. Integrate with ticketing for automated remediation tracking.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Vuln management TA.
• Ensure the following data sources are available: Scan results (first_seen within last scan window).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
After each scan, identify new critical/high vulnerabilities (first_seen within scan window). Alert immediately on new critical findings. Include CVE details and affected hosts. Integrate with ticketing for automated remediation tracking.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vulnerability sourcetype="tenable:vuln" severity="Critical"
| where first_seen > relative_time(now(), "-24h")
| table hostname, cve_id, plugin_name, severity, first_seen
| sort -first_seen
```
