# UC-10.11.11 · FortiGate UTM Threat Correlation

## Description

Multi-module UTM hits on one session often indicate automated or staged attacks.

## Value

Multi-module UTM hits on one session often indicate automated or staged attacks.

## Implementation

Normalize UTM subtypes (av, ips, webfilter, appctrl) on shared session keys. Flag sessions hitting multiple UTM modules as higher fidelity incidents.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_fortinet`.
• Ensure the following data sources are available: `sourcetype=fortinet:fortigate_utm`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize UTM subtypes (av, ips, webfilter, appctrl) on shared session keys. Flag sessions hitting multiple UTM modules as higher fidelity incidents.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=firewall sourcetype="fortinet:fortigate_utm"
| stats values(subtype) as utm_types, count by src, dst_ip, session_id
| where mvcount(utm_types)>2
```
