# UC-10.11.106 · Zscaler ZTNA Application Access Denials

## Description

ZTNA denials reveal unauthorized private application access attempts.

## Value

ZTNA denials reveal unauthorized private application access attempts.

## Implementation

Track ZTNA denials for policy versus user error. Investigate repeated denials as potential probing.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler`.
• Ensure the following data sources are available: `sourcetype=zscaler:ztna`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track ZTNA denials for policy versus user error. Investigate repeated denials as potential probing.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl

| stats count by app_name, user, reason
| sort -count
```
