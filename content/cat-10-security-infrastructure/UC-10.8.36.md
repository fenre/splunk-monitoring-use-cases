# UC-10.8.36 · Certificate Transparency Log Monitoring

## Description

Unauthorized or fraudulent certificate issuance for your domains can enable phishing or MITM. CT log monitoring detects new certs for your domains and supports rapid revocation.

## Value

Unauthorized or fraudulent certificate issuance for your domains can enable phishing or MITM. CT log monitoring detects new certs for your domains and supports rapid revocation.

## Implementation

Subscribe to CT log streams (e.g., crt.sh) for your domains. Maintain domain watchlist. Alert on new certificates for critical domains. Correlate with internal PKI and public CAs.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: CT log ingestion, custom scripted input.
• Ensure the following data sources are available: Certificate Transparency log streams, domain watchlist.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Subscribe to CT log streams (e.g., crt.sh) for your domains. Maintain domain watchlist. Alert on new certificates for critical domains. Correlate with internal PKI and public CAs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=certs sourcetype="ct:log"
| lookup domain_watchlist.csv domain OUTPUT domain as watched
| where watched=*
| stats latest(_time) as issued, values(issuer_cn) as issuers by domain, cn
| table domain, cn, issued, issuers
```
