# UC-10.16.8 · Endpoint Protection Coverage Trending

## Description

Percentage of endpoints with current antivirus or EDR definitions and healthy agent status demonstrates control effectiveness for audits and cyber insurance. Downward trends often track OS rebuilds, mergers, or agent failures rather than policy alone.

## Value

Percentage of endpoints with current antivirus or EDR definitions and healthy agent status demonstrates control effectiveness for audits and cyber insurance. Downward trends often track OS rebuilds, mergers, or agent failures rather than policy alone.

## Implementation

Map vendor-specific “last update” and protection-state fields; some use epoch seconds. Join to CMDB or ES `assets` to exclude decommissioned hosts from the denominator. Alert when coverage drops more than a few points week over week. Document exceptions (OT, lab) via a lookup to avoid false remediation tickets.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=main (sourcetype="crowdstrike:hosts" OR sourcetype="ms:defender:machineinfo" OR sourcetype="tanium:endpoint")
| eval defs_age_hours=(now()-strptime(last_av_update,"%Y-%m-%dT%H:%M:%S"))/3600
| eval current=if(isnotnull(defs_age_hours) AND defs_age_hours<=48 AND match(status,"normal|active|enabled"),1,0)
| bin _time span=1w
| stats dc(host) as fleet, sum(current) as compliant by _time
| eval coverage_pct=round(100*compliant/nullif(fleet,0),2)
| timechart span=1w first(coverage_pct) as av_edr_coverage_pct
```
