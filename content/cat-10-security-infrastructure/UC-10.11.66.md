# UC-10.11.66 · Carbon Black Alert Severity Distribution

## Description

Severity distribution balances SOC workload and SLA design.

## Value

Severity distribution balances SOC workload and SLA design.

## Implementation

Normalize CB alert schema. Route criticals to on-call. Report mean time to triage.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_vmware_carbonblack`.
• Ensure the following data sources are available: `sourcetype=carbonblack:alert`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize CB alert schema. Route criticals to on-call. Report mean time to triage.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=endpoint sourcetype="carbonblack:alert"
| stats count by severity, type
| sort -severity
```
