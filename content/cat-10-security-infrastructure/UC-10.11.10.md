# UC-10.11.10 · FortiManager Configuration Compliance

## Description

Centralized compliance reduces misconfiguration-driven gaps in UTM and policy.

## Value

Centralized compliance reduces misconfiguration-driven gaps in UTM and policy.

## Implementation

Ingest FortiManager audit and revision logs. Compare device config revisions to approved baselines via lookup. Alert on drift.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_fortinet`.
• Ensure the following data sources are available: `sourcetype=fortinet:fortimanager`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest FortiManager audit and revision logs. Compare device config revisions to approved baselines via lookup. Alert on drift.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security sourcetype="fortinet:fortimanager" fmgt_event="config"
| stats latest(revision) as rev by adom, device_name
| join type=left max=1 device_name [| inputlookup fortigate_golden_config]
| where rev!=expected_rev OR isnull(expected_rev)
```
