# UC-10.14.20 · OT Network Baseline Deviation Detection

## Description

Detects new protocols, ports, or connections relative to learned OT baselines (T0840), highlighting shadow connectivity and preparatory attack activity.

## Value

Detects new protocols, ports, or connections relative to learned OT baselines (T0840), highlighting shadow connectivity and preparatory attack activity.

## Implementation

Populate baselines via OT Security Add-on batch jobs after stable learning windows. Use Splunk ES for notable generation on first-seen edges. Pair with T0840 in threat hunting.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OT Security Add-on for Splunk (Splunkbase 5151).
• Ensure the following data sources are available: `index=ot` NetFlow or Zeek `sourcetype="ics:flow"`, baseline KV store.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Populate baselines via OT Security Add-on batch jobs after stable learning windows. Use Splunk ES for notable generation on first-seen edges. Pair with T0840 in threat hunting.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="ics:flow"
| eval sig=src."|".dest."|".dest_port."|".proto
| lookup ot_traffic_baseline sig OUTPUT first_seen_baseline
| where isnull(first_seen_baseline)
| bin _time span=1d
| stats earliest(_time) as first_seen dc(dest_port) as new_ports by src, dest, proto
| lookup ot_asset_inventory src OUTPUT zone
| where zone="OT"
```
