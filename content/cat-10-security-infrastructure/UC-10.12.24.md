# UC-10.12.24 · HL7 Message Delivery Monitoring

## Description

ADT/ORM gaps break registration, billing, and care coordination workflows.

## Value

ADT/ORM gaps break registration, billing, and care coordination workflows.

## Implementation

Track message control ID retries; alert on NAK storms or queue depth from engine metrics.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Interface engine (Mirth, Cloverleaf).
• Ensure the following data sources are available: `sourcetype="hl7:v2"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track message control ID retries; alert on NAK storms or queue depth from engine metrics.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=interfaces sourcetype="hl7:v2"
| eval ack_ok=if(ACK_Code="AA",1,0)
| timechart span=15m sum(ack_ok) as ok, count as total
| eval ack_rate=round(ok/total*100,2)
| where ack_rate < 99
```
