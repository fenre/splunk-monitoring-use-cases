# UC-10.11.4 · FortiGate Web Filter Category Blocks

## Description

Category blocks expose risky browsing, policy misuse, and potential malware download attempts.

## Value

Category blocks expose risky browsing, policy misuse, and potential malware download attempts.

## Implementation

Ingest explicit-proxy or flow-mode web filter logs. Join category IDs to human-readable names. Report top blocked categories and repeat offenders.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_fortinet`.
• Ensure the following data sources are available: `sourcetype=fortinet:fortigate_webfilter`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest explicit-proxy or flow-mode web filter logs. Join category IDs to human-readable names. Report top blocked categories and repeat offenders.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=web sourcetype="fortinet:fortigate_webfilter" action=blocked
| stats count by category, user, src
| sort -count
```
