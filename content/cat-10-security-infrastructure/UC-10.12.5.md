# UC-10.12.5 · ACH Origination Anomalies

## Description

Surfaces unusual ACH batches (debit concentration, new originators) that may indicate fraud or operational error.

## Value

Surfaces unusual ACH batches (debit concentration, new originators) that may indicate fraud or operational error.

## Implementation

Validate company_id against registered originators; alert on new SEC codes or return spikes after origination.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: ACH/NACHA ingestion, treasury workstation.
• Ensure the following data sources are available: `sourcetype="ach:origination"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Validate company_id against registered originators; alert on new SEC codes or return spikes after origination.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=payments sourcetype="ach:origination" sec_code="WEB"
| stats sum(amount) as batch_total, dc(individual_id) as distinct_receivers by company_id, file_id
| eventstats avg(batch_total) as avg_batch by company_id
| where batch_total > 3*avg_batch OR distinct_receivers > 500
| sort -batch_total
```
