# UC-10.12.19 · Minimum Necessary Access Validation

## Description

Identifies users with broader EHR rights than their role requires (privilege creep).

## Value

Identifies users with broader EHR rights than their role requires (privilege creep).

## Implementation

Quarterly recertification; break-glass accounts heavily monitored; integrate with provisioning.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: IAM, EHR role extracts.
• Ensure the following data sources are available: `sourcetype="epic:security"`, HR feed.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Quarterly recertification; break-glass accounts heavily monitored; integrate with provisioning.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=healthcare sourcetype="epic:security"
| lookup hr_job_role user_id OUTPUT job_family expected_epic_role
| where epic_role!=expected_epic_role AND NOT match(epic_role,"(train|breakglass)")
| stats values(epic_role) as roles by user_id, job_family
```

Understanding this SPL

**Minimum Necessary Access Validation** — Identifies users with broader EHR rights than their role requires (privilege creep).

**Pipeline walkthrough**

• Scopes the data: index=healthcare, sourcetype="epic:security". Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
