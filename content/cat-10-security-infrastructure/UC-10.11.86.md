# UC-10.11.86 · Tanium Threat Response Action Audit

## Description

Threat response audits ensure automated remediation executes safely at scale.

## Value

Threat response audits ensure automated remediation executes safely at scale.

## Implementation

Capture automated Threat Response actions from Tanium. Investigate failures to restore trust.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_tanium`.
• Ensure the following data sources are available: `sourcetype=tanium:threatresponse`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Capture automated Threat Response actions from Tanium. Investigate failures to restore trust.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=audit sourcetype="tanium:threatresponse"
| stats count by playbook, outcome, hash
| search outcome IN ("failed","partial")
```
