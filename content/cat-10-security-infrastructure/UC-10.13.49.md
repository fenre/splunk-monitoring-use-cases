# UC-10.13.49 · Endpoint Resource Utilization (CIM)

## Description

Sustained high CPU on many endpoints may indicate cryptomining or ransomware — use `Performance` DM when TAs map `cpu_load_percent` (field name varies by TA).

## Value

Sustained high CPU on many endpoints may indicate cryptomining or ransomware — use `Performance` DM when TAs map `cpu_load_percent` (field name varies by TA).

## Implementation

Prefer `mstats` for metrics indexes; join CIM `Performance.host` to CMDB tier.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Perfmon, Telegraf → Performance CIM.
• Ensure the following data sources are available: Normalized performance metrics per `host`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Prefer `mstats` for metrics indexes; join CIM `Performance.host` to CMDB tier.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=perf tag=endpoint metric_name="cpu_pct"
| bin _time span=5m
| stats perc95(value) as cpu_p95 by _time, host
| where cpu_p95>90
```
