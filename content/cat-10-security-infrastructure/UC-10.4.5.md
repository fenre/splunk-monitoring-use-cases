# UC-10.4.5 · Spoofed Email Detection

## Description

DMARC/SPF/DKIM failures indicate email spoofing attempts. Monitoring validates email authentication configuration.

## Value

DMARC/SPF/DKIM failures indicate email spoofing attempts. Monitoring validates email authentication configuration.

## Implementation

Configure DMARC reporting (aggregate to a designated mailbox). Ingest DMARC XML reports. Track authentication failures by sending domain. Alert on spoofing of your own domains. Move toward DMARC p=reject for full protection.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Email security TA.
• Ensure the following data sources are available: DMARC aggregate reports, email authentication logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Configure DMARC reporting (aggregate to a designated mailbox). Ingest DMARC XML reports. Track authentication failures by sending domain. Alert on spoofing of your own domains. Move toward DMARC p=reject for full protection.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=email sourcetype="dmarc:aggregate"
| where dkim_result!="pass" OR spf_result!="pass"
| stats count by src, header_from, dkim_result, spf_result, disposition
| sort -count
```
