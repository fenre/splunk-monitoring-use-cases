# UC-10.14.1 · OT Security Add-on Health and Configuration Status

## Description

Ensures OT Security Add-on correlation searches, lookups, and MITRE mappings stay healthy so process-safety detections do not silently fail.

## Value

Ensures OT Security Add-on correlation searches, lookups, and MITRE mappings stay healthy so process-safety detections do not silently fail.

## Implementation

Deploy Splunk Enterprise Security and the OT Security Add-on. Schedule a daily check for disabled or failing correlation searches. Alert when any OT-prefixed saved search has failed or been disabled.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OT Security Add-on for Splunk (Splunkbase 5151).
• Ensure the following data sources are available: `index=_internal` scheduler/search logs, `index=ot_security` `sourcetype="ot:security:health"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Deploy Splunk Enterprise Security and the OT Security Add-on. Schedule a daily check for disabled or failing correlation searches. Alert when any OT-prefixed saved search has failed or been disabled.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal source=*scheduler.log status=failed savedsearch_name="OT*"
| stats count by savedsearch_name, status
| append [
    | rest /servicesNS/-/-/saved/searches search="OT*"
    | where disabled=1
    | table title disabled
  ]
| table savedsearch_name, title, status, disabled, count
```
