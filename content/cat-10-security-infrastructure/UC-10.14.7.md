# UC-10.14.7 · Rogue Engineering Tool Execution

## Description

Spots unauthorized engineering software sessions to field devices (T0843), reducing risk of malicious reprogramming.

## Value

Spots unauthorized engineering software sessions to field devices (T0843), reducing risk of malicious reprogramming.

## Implementation

Requires Splunk ES, OT Security Add-on, and workstation inventory. Tune for sanctioned tools and vendor remote support. Correlate with T0843 and change records.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OT Security Add-on for Splunk (Splunkbase 5151).
• Ensure the following data sources are available: `index=ot_security` host-based logs from engineering workstations, EDR forwarded to OT index.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Requires Splunk ES, OT Security Add-on, and workstation inventory. Tune for sanctioned tools and vendor remote support. Correlate with T0843 and change records.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_security sourcetype="ot:security:process"
| lookup ot_approved_eng_tools process_name OUTPUT approved_tool
| where isnull(approved_tool) AND match(process_name, "(?i)studio|logix|step7|tia|codesys|unity")
| stats earliest(_time) as first_seen values(user) as users dc(host) as hosts by process_name
| lookup ot_eng_workstations host OUTPUT approved_role
| where isnull(approved_role)
```
