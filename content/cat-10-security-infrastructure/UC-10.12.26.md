# UC-10.12.26 · EHR Login Anomaly Detection

## Description

Unusual geography, time, or volume of EHR logins may indicate credential theft or insider snooping.

## Value

Unusual geography, time, or volume of EHR logins may indicate credential theft or insider snooping.

## Implementation

Step-up auth from new countries; peer-group baseline per department; VIP patient list access rules.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: EHR SSO logs, IdP.
• Ensure the following data sources are available: `sourcetype="epic:login"`, `sourcetype="okta:systemlog"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Step-up auth from new countries; peer-group baseline per department; VIP patient list access rules.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=healthcare sourcetype="epic:login" outcome="success"
| iplocation src
| stats dc(Country) as countries, count by user_id
| where countries>2 OR count>100
| sort -count
```
