# UC-10.11.108 · Zscaler DNS Security Filtering

## Description

DNS filtering stops malicious resolutions before HTTP or TLS sessions start.

## Value

DNS filtering stops malicious resolutions before HTTP or TLS sessions start.

## Implementation

Use ZIA DNS security logs. Chain to endpoint for malware families using DNS tunneling.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler`.
• Ensure the following data sources are available: `sourcetype=zscaler:dns`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Use ZIA DNS security logs. Chain to endpoint for malware families using DNS tunneling.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=dns sourcetype="zscaler:dns" action=blocked
| stats count by threat_category, domain
| sort -count
```
