# UC-10.10.8 · Notable event triage time by detection rule

## Description

Average time from notable creation to first owner assignment or close by rule.

## Value

Average time from notable creation to first owner assignment or close by rule.

## Implementation

Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security / custom.
• Ensure the following data sources are available: Notable event fields (custom field names may vary by ES version).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize field names to your Splunk ES version and index names. Use summary indexing or ES introspection where `detection_metrics` is not present. For YARA, deploy a scripted input or HEC to ingest scanner JSON.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
`notable`
| eval triage_sec=if(isnotnull(first_owner_assignment_time), first_owner_assignment_time-_time, now()-_time)
| stats avg(triage_sec) as avg_triage_sec perc95(triage_sec) as p95_triage_sec by rule_name
| sort - avg_triage_sec
```
