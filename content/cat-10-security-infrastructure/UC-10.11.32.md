# UC-10.11.32 · Check Point Anti-Bot Detection Events

## Description

Anti-bot events reveal C2 and automated malware behavior.

## Value

Anti-bot events reveal C2 and automated malware behavior.

## Implementation

Enable anti-bot blade logging. Map bot families to MITRE. Block or monitor per policy.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_checkpoint`.
• Ensure the following data sources are available: `sourcetype=cp:antibot`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Enable anti-bot blade logging. Map bot families to MITRE. Block or monitor per policy.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security sourcetype="cp:antibot"
| stats count by bot_name, action, src
| sort -count
```
