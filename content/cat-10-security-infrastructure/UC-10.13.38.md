# UC-10.13.38 · Web Application Error Rate

## Description

Application-level errors in `status` 500 series or custom `error` field in CIM extensions.

## Value

Application-level errors in `status` 500 series or custom `error` field in CIM extensions.

## Implementation

Group by `app` token in URL path; tie to deployment markers.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Web CIM.
• Ensure the following data sources are available: `status=500`, `url`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Group by `app` token in URL path; tie to deployment markers.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=web status>=500
| timechart span=5m count by url
| where count>100
```
