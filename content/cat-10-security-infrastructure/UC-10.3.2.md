# UC-10.3.2 · Quarantine Action Monitoring

## Description

Failed quarantine means malware remains active on the endpoint. Monitoring ensures automated remediation is working.

## Value

Failed quarantine means malware remains active on the endpoint. Monitoring ensures automated remediation is working.

## Implementation

Track EDR remediation actions (quarantine, kill process, isolate). Calculate quarantine success rate. Alert on failed quarantine actions. Follow up on "allowed" malware detections to ensure analyst review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: EDR TA.
• Ensure the following data sources are available: EDR remediation/action logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track EDR remediation actions (quarantine, kill process, isolate). Calculate quarantine success rate. Alert on failed quarantine actions. Follow up on "allowed" malware detections to ensure analyst review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=edr sourcetype="crowdstrike:detection"
| stats count(eval(action="quarantined")) as quarantined, count(eval(action="allowed")) as allowed by severity
| eval quarantine_rate=round(quarantined/(quarantined+allowed)*100,1)
```
