# UC-10.11.9 · FortiGate SSL Inspection Bypass

## Description

SSL bypass monitoring balances privacy with ensuring encrypted threats are still inspected where required.

## Value

SSL bypass monitoring balances privacy with ensuring encrypted threats are still inspected where required.

## Implementation

Log SSL inspection decisions including bypass reasons (user, category, tech). Review for excessive bypasses that reduce visibility.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_fortinet`.
• Ensure the following data sources are available: `sourcetype=fortinet:fortigate_ssl`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Log SSL inspection decisions including bypass reasons (user, category, tech). Review for excessive bypasses that reduce visibility.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl

| stats count by bypass_reason, policyid, user
| sort -count
```
