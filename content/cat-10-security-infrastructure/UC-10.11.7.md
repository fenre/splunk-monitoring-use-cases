# UC-10.11.7 · FortiGate SD-WAN Tunnel Health

## Description

SD-WAN tunnel health ensures branch connectivity and path failover behave as designed.

## Value

SD-WAN tunnel health ensures branch connectivity and path failover behave as designed.

## Implementation

Ingest event logs for SD-WAN and IPsec state changes. Build expected neighbor inventory. Page on tunnel down beyond SLA.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_fortinet`.
• Ensure the following data sources are available: `sourcetype=fortinet:fortigate_event`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest event logs for SD-WAN and IPsec state changes. Build expected neighbor inventory. Page on tunnel down beyond SLA.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="fortinet:fortigate_event" (subtype="vpn" OR logdesc="*SD-WAN*")
| stats latest(state) as state by interface, neighbor
| search state IN ("down","dead","inactive")
```
