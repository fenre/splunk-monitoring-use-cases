# UC-10.11.115 · Tenable.io Agent and Linked Scanner Health

## Description

Agent health maintains continuous assessment without network-only blind spots.

## Value

Agent health maintains continuous assessment without network-only blind spots.

## Implementation

Monitor Tenable.io linked agents for two-week silence. Redeploy or fix connectivity.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_nessus`.
• Ensure the following data sources are available: `sourcetype=tenable:io`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Monitor Tenable.io linked agents for two-week silence. Redeploy or fix connectivity.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vuln sourcetype="tenable:io"
| stats latest(last_scan) as ls by agent_uuid
| where now()-ls > 1209600
```
