# UC-10.13.5 ¬∑ Authentication Source Diversity

## Description

Many distinct `src` addresses for one `user` in a short window suggests shared credentials or stuffing.

## Value

Many distinct `src` addresses for one `user` in a short window suggests shared credentials or stuffing.

## Implementation

Lower threshold for privileged users; combine with threat intel on `src`.

## Detailed Implementation

Prerequisites
‚Ä¢ Install and configure the required add-on or app: Any Authentication TA.
‚Ä¢ Ensure the following data sources are available: CIM-authenticated VPN, cloud, or SSO logs.
‚Ä¢ For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 ‚Äî Configure data collection
Lower threshold for privileged users; combine with threat intel on `src`.

Step 2 ‚Äî Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=authentication action=failure OR action=success
| bin _time span=1h
| stats dc(src) as src_diversity by user, _time
| where src_diversity>15
```
