# UC-10.11.16 · Palo Alto WildFire Malware Verdict Trending

## Description

WildFire trending shows evolving malware volume and sandbox pipeline health.

## Value

WildFire trending shows evolving malware volume and sandbox pipeline health.

## Implementation

Ensure WildFire submissions are fully parsed. Map verdict integers to labels in eval. Trend malicious versus benign over time.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_paloalto`.
• Ensure the following data sources are available: `sourcetype=pan:wildfire`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl

| timechart span=1d count by verdict
| fillnull value=0
```
