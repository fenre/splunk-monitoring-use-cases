# UC-10.11.125 · Check Point CloudGuard Network Security Events

## Description

CloudGuard Network extends Check Point Quantum prevention to cloud workloads in AWS, Azure, and GCP. Security events from cloud-deployed gateways provide the same IPS, threat prevention, and access control visibility as on-prem — critical for hybrid environments.

## Value

CloudGuard Network extends Check Point Quantum prevention to cloud workloads in AWS, Azure, and GCP. Security events from cloud-deployed gateways provide the same IPS, threat prevention, and access control visibility as on-prem — critical for hybrid environments.

## Implementation

Deploy CloudGuard Network in AWS VPC, Azure VNET, or GCP VPC and forward logs to Smart-1 Cloud or on-prem Log Server. Normalize `cloud_provider` from SIC name or tags. Alert on critical threats in cloud environments. Report on cloud vs on-prem detection ratios for security posture.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_checkpoint`.
• Ensure the following data sources are available: `sourcetype=cp_log` (CloudGuard logs via Log Exporter or Smart-1 Cloud).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Deploy CloudGuard Network in AWS VPC, Azure VNET, or GCP VPC and forward logs to Smart-1 Cloud or on-prem Log Server. Normalize `cloud_provider` from SIC name or tags. Alert on critical threats in cloud environments. Report on cloud vs on-prem detection ratios for security posture.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cloud sourcetype="cp_log" earliest=-24h
| where match(lower(product),"(?i)cloudguard|cloud.?network") OR match(lower(origin_sic_name),"(?i)aws|azure|gcp")
| stats count by product, action, protection_name, cloud_provider
| sort -count
```
