# UC-10.13.6 · MFA Failure Rate Trending (CIM)

## Description

MFA fatigue attacks and token issues both raise failure rates; CIM unifies IdP vendors.

## Value

MFA fatigue attacks and token issues both raise failure rates; CIM unifies IdP vendors.

## Implementation

Alert when `mfa_fail_pct` doubles vs. same hour prior week.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Okta, Azure AD, Duo TAs → Authentication.
• Ensure the following data sources are available: MFA challenge events mapped with `action=failure` and `reason` or `signature`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Alert when `mfa_fail_pct` doubles vs. same hour prior week.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=authentication "mfa" OR signature="*MFA*"
| timechart span=1h count(eval(action="failure")) as mfa_fail count as mfa_total
| eval mfa_fail_pct=round(mfa_fail/mfa_total*100,2)
```
