# UC-10.12.13 · Algorithmic Trading Circuit Breaker

## Description

Ensures volatility/gateway breakers trip correctly and strategies are flattened when limits hit.

## Value

Ensures volatility/gateway breakers trip correctly and strategies are flattened when limits hit.

## Implementation

Log all manual overrides; post-mortem each halt with P&L and message counts.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Risk engine, exchange gateway.
• Ensure the following data sources are available: `sourcetype="risk:circuit"`, `sourcetype="fix:session"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Log all manual overrides; post-mortem each halt with P&L and message counts.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=trading sourcetype="risk:circuit"
| search state IN ("HALT","THROTTLE","KILL")
| stats latest(_time) as tripped, values(strategy_id) as strategies by gateway, state
| join max=1 gateway [| search index=trading sourcetype="fix:session" | stats latest(heartbeat_gap) as gap by gateway]
| where gap>30 OR isnotnull(tripped)
```
