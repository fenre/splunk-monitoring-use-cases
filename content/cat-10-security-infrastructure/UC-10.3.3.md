# UC-10.3.3 · Agent Health Monitoring

## Description

Endpoints without healthy EDR agents are blind spots. Gap detection ensures comprehensive coverage.

## Value

Endpoints without healthy EDR agents are blind spots. Gap detection ensures comprehensive coverage.

## Implementation

Poll EDR agent status API daily. Identify agents offline >24 hours, outdated versions, or degraded status. Cross-reference with CMDB for full coverage analysis. Alert on critical servers with unhealthy agents.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: EDR TA, scripted input.
• Ensure the following data sources are available: EDR agent status API, last check-in timestamps.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll EDR agent status API daily. Identify agents offline >24 hours, outdated versions, or degraded status. Cross-reference with CMDB for full coverage analysis. Alert on critical servers with unhealthy agents.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=edr sourcetype="crowdstrike:sensor_health"
| eval hours_since_checkin=round((now()-last_seen_epoch)/3600,1)
| where hours_since_checkin > 24 OR sensor_version < "6.50"
| table hostname, os, sensor_version, hours_since_checkin, status
```
