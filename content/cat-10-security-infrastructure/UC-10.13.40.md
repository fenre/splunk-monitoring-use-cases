# UC-10.13.40 · Unauthorized Change Detection

## Description

Changes without ticket ID or with `user` not in CAB roster — map `object_attrs` to CIM.

## Value

Changes without ticket ID or with `user` not in CAB roster — map `object_attrs` to CIM.

## Implementation

Auto-severity for production `object_category`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Change CIM + ITSM.
• Ensure the following data sources are available: `object_id`, `user`, `change_type`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Auto-severity for production `object_category`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=change
| lookup cab_authorized_users user OUTPUT authorized
| where authorized!="true" OR isnull(change_ticket)
| stats count by user, object_id
```
