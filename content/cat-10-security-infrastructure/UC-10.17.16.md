# UC-10.17.16 · SonicWall Connection Table Saturation and Max-Session Pressure (SonicWall)

## Description

State table exhaustion causes random TCP failures during DNS storms or SYN floods even if bandwidth is fine.

## Value

Triggers timely firmware tuning, connection limit adjustments, or DDoS escalation.

## Implementation

Ingest SNMP `currConn` or syslog threshold traps. Compare to licensed maximum.

## SPL

```spl
index=netops sourcetype="dell:sonicwall" earliest=-4h
| where match(_raw,"(?i)connection.*table|max connection|session.*limit|resource.*exhaust")
| rex field=_raw "(?i)(?<curr_conn>\d+)\s*(active )?connections"
| timechart span=5m max(curr_conn) by host
```

## Visualization

Timechart (connections), gauge vs maximum.
