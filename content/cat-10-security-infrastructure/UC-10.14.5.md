# UC-10.14.5 · Data Historian Compromise Indicators

## Description

Detects anomalous queries, bulk exports, or new clients against historian servers (T0815), which can hide sabotage or exfiltrate operational data.

## Value

Detects anomalous queries, bulk exports, or new clients against historian servers (T0815), which can hide sabotage or exfiltrate operational data.

## Implementation

Whitelist backup and reporting hosts. Create thresholds for off-hours bulk reads and new ODBC/JDBC clients. Tag with T0815 for threat intel alignment.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OT Security Add-on for Splunk (Splunkbase 5151).
• Ensure the following data sources are available: `index=ot_security` `sourcetype="ics:historian"` DB/OS logs for historian hosts.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Whitelist backup and reporting hosts. Create thresholds for off-hours bulk reads and new ODBC/JDBC clients. Tag with T0815 for threat intel alignment.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_security sourcetype="ics:historian"
| bin _time span=1h
| stats count sum(rows_returned) as rows dc(client_ip) as clients by _time, historian_host
| eventstats avg(count) as avg_count by historian_host
| where count>avg_count*3 OR rows>10000 OR clients>5
| table _time, historian_host, count, rows, clients, avg_count
```
