# UC-10.13.27 · CVSS Distribution

## Description

Histogram of `severity` / CVSS for backlog risk profile.

## Value

Histogram of `severity` / CVSS for backlog risk profile.

## Implementation

Filter internet-exposed assets via lookup for criticality weighting.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Vulnerabilities CIM.
• Ensure the following data sources are available: `severity` field (critical/high/medium/low) or numeric `cvss`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Filter internet-exposed assets via lookup for criticality weighting.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security tag=vulnerability status=open
| stats count by severity
| sort severity
```
