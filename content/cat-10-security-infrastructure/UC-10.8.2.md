# UC-10.8.2 · Certificate Issuance Audit

## Description

Unauthorized certificate issuance from internal CAs can enable man-in-the-middle attacks. Audit trail supports compliance.

## Value

Unauthorized certificate issuance from internal CAs can enable man-in-the-middle attacks. Audit trail supports compliance.

## Implementation

Forward CA server audit logs to Splunk (Event ID 4887 for AD CS). Track all certificate issuance events. Alert on issuance from non-standard templates or by unauthorized requesters. Report on issuance volume and template usage.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: CA server log forwarding.
• Ensure the following data sources are available: CA audit logs (Microsoft AD CS, EJBCA, HashiCorp Vault PKI).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert

```spl

| search EventCode=4887
| table _time, RequesterName, CertificateTemplate, SerialNumber, SubjectCN
```
