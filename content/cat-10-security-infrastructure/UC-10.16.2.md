# UC-10.16.2 · SIEM Alert-to-Incident Ratio Trending

## Description

Comparing ES notables to analyst-confirmed incidents shows whether the SOC is drowning in noise or under-escalating real cases. A falling ratio over 90 days may mean tuning drift; a rising ratio can indicate improved correlation or staffing effects worth validating.

## Value

Comparing ES notables to analyst-confirmed incidents shows whether the SOC is drowning in noise or under-escalating real cases. A falling ratio over 90 days may mean tuning drift; a rising ratio can indicate improved correlation or staffing effects worth validating.

## Implementation

Align `status` values with your ES workflow (Closed - True Positive / False Positive). If you use formal incident tickets, join notables to `incident_id` from a lookup populated by SOAR or ITSM export. Review weekly with detection engineering to separate tuning opportunities from genuine threat volume changes. Exclude test rules via `rule_name` lookup.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security.
• Ensure the following data sources are available: `index=notable`, disposition fields (`status`, `urgency`, `owner`), optional `index=itsi` if incidents sync externally.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Align `status` values with your ES workflow (Closed - True Positive / False Positive). If you use formal incident tickets, join notables to `incident_id` from a lookup populated by SOAR or ITSM export. Review weekly with detection engineering to separate tuning opportunities from genuine threat volume changes. Exclude test rules via `rule_name` lookup.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=notable earliest=-90d
| eval confirmed=if(match(status,"Closed - True Positive"),1,0)
| eval triaged_incident=if(match(status,"(New|In Progress|Pending)") OR match(owner,"SOC|Incident"),1,0)
| bin _time span=1w
| stats count as alerts sum(confirmed) as true_positives sum(triaged_incident) as in_flight by _time
| eval tp_rate_pct=round(100*true_positives/nullif(alerts,0),2)
| timechart span=1w first(alerts) as weekly_alerts first(tp_rate_pct) as true_positive_rate_pct
```
