# UC-17.1.6 · MAC Authentication Bypass (MAB)

## Description

MAB devices bypass 802.1X and rely on MAC address only. Monitoring for unauthorized MACs prevents rogue device access.

## Value

MAB devices bypass 802.1X and rely on MAC address only. Monitoring for unauthorized MACs prevents rogue device access.

## Implementation

Maintain whitelist of approved MAB devices (printers, IP phones, IoT). Alert on unknown MAC addresses authenticating via MAB. Track MAB device population. Report on MAB vs 802.1X ratio for security posture.

## Detailed Implementation

• Install and configure the required add-on or app: `Splunk_TA_cisco-ise`, HPE Aruba ClearPass syslog, Forescout CounterACT syslog.

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=nac sourcetype="cisco:ise:auth" auth_method="MAB"
| lookup approved_mab_devices.csv mac_address OUTPUT device_description, approved
| where isnull(approved) OR approved!="Yes"
| table _time, endpoint_mac, switch, port, location
```
