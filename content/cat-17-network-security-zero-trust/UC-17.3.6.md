# UC-17.3.6 · Policy Drift Detection

## Description

Zero-trust policies require continuous validation. Drift from baseline configuration introduces security gaps.

## Value

Zero-trust policies require continuous validation. Drift from baseline configuration introduces security gaps.

## Implementation

Track all ZT policy changes via audit logs. Compare current configuration against approved baseline. Alert on unauthorized modifications. Require change management approval for policy changes. Report on policy change frequency.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler`, Netskope Add-on for Splunk (Splunkbase 3808), `Splunk_TA_paloalto` (Prisma Access), `TA-fortinet_fortigate` (FortiSASE), Check Point App for Splunk (Splunkbase 4293).
• Ensure the following data sources are available: ZT policy audit logs, configuration snapshots.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track all ZT policy changes via audit logs. Compare current configuration against approved baseline. Alert on unauthorized modifications. Require change management approval for policy changes. Report on policy change frequency.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=zt sourcetype="zt:admin_audit"
| search action IN ("policy_modified","rule_added","rule_deleted","rule_disabled")
| table _time, admin, action, policy_name, details
| sort -_time
```
