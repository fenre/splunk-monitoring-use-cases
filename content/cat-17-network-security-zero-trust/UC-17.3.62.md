# UC-17.3.62 · Versa SASE Security and Access Events (Versa Networks)

## Description

Versa SASE unifies SD-WAN, SWG, CASB, ZTNA, and FWaaS in a single platform. Security event analysis across these functions provides unified threat visibility and policy enforcement validation.

## Value

Versa SASE unifies SD-WAN, SWG, CASB, ZTNA, and FWaaS in a single platform. Security event analysis across these functions provides unified threat visibility and policy enforcement validation.

## Implementation

Configure Versa Analytics to stream logs to Splunk via UDP/TCP syslog in key-value format. Install the Versa Splunk App for pre-built reports. Map `rule_name` to policy intent. Alert on threat and firewall blocks. Report on SASE enforcement across sites.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Versa Networks Splunk App (via Versa Analytics syslog integration).
• Ensure the following data sources are available: Versa Analytics logs (syslog key-value pairs: alarm, event, flow, firewall, threat logs).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Configure Versa Analytics to stream logs to Splunk via UDP/TCP syslog in key-value format. Install the Versa Splunk App for pre-built reports. Map `rule_name` to policy intent. Alert on threat and firewall blocks. Report on SASE enforcement across sites.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sase sourcetype="versa:sase" earliest=-24h
| where match(lower(action),"(?i)block|deny|drop|alert")
| stats count by log_type, rule_name, src_ip, action
| sort -count
```
