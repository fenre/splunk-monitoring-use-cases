# UC-17.4.8 · Cloudflare Access Zero Trust Policy Deny Events by User and Application (`cloudflare:json`)

## Description

Access denials highlight over-tight policies, terminated employees still trying SSO, or credential stuffing against SaaS.

## Value

IAM teams tune policies without blind lifts; security spots brute-force patterns.

## Implementation

Configure Logpush jobs per dataset; route all into a unified `cloudflare:json` sourcetype or use transforms to normalize. Use `spath` for PascalCase fields. Create macros per zone name.

## SPL

```spl
index=zt sourcetype="cloudflare:json" earliest=-24h
| where match(_raw,"(?i)Access|JWT|identity")
| spath UserEmail
| spath AppUUID
| spath Action
| where match(Action,"(?i)deny|block")
| stats count by UserEmail, AppUUID, IPCountry
| sort -count
```

## Visualization

Table (user, app, country), timechart (deny rate).
