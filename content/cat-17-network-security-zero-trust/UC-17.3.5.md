# UC-17.3.5 · Posture Assessment Trending

## Description

Endpoint posture compliance rates over time measure security improvement and identify persistent non-compliance areas.

## Value

Endpoint posture compliance rates over time measure security improvement and identify persistent non-compliance areas.

## Implementation

Track posture assessment results over time by check type (AV, encryption, OS patch, firewall). Report on compliance improvement trends. Alert when compliance drops below target. Identify persistent non-compliance patterns.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler`, Netskope Add-on for Splunk (Splunkbase 3808), `Splunk_TA_paloalto` (Prisma Access), `Splunk_TA_microsoft-cloudservices` (Entra ID / Intune).
• Ensure the following data sources are available: ZT posture assessment data.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track posture assessment results over time by check type (AV, encryption, OS patch, firewall). Report on compliance improvement trends. Alert when compliance drops below target. Identify persistent non-compliance patterns.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=zt sourcetype="zt:posture"
| timechart span=1d avg(compliance_pct) as compliance by check_type
```
