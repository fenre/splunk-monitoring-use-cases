# UC-17.2.20 · VPN Bandwidth Utilization Trending

## Description

Time-series bandwidth per headend and user cohort — complements UC-17.2.7 top talkers with **trend** and **gateway** dimension.

## Value

Time-series bandwidth per headend and user cohort — complements UC-17.2.7 top talkers with **trend** and **gateway** dimension.

## Implementation

If bytes not in syslog, use SNMP interface counters or NetFlow `exporter=VPN`. Alert on sustained >80% of circuit.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-asa`, `Splunk_TA_paloalto`, `TA-fortinet_fortigate`, NetFlow.
• Ensure the following data sources are available: VPN accounting, NetFlow from VPN interface.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
If bytes not in syslog, use SNMP interface counters or NetFlow `exporter=VPN`. Alert on sustained >80% of circuit.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="cisco:asa" earliest=-7d
| bin _time span=1h
| stats sum(bytes_in) as in_b, sum(bytes_out) as out_b by _time, host
| eval gbps=round((in_b+out_b)*8/3600/1000000000,3)
| timechart span=1h avg(gbps) by host
```
