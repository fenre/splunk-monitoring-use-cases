# UC-17.3.12 · Zscaler ZIA Policy Violation Trends

## Description

Time-series of blocked violations per URL category and rule — tunes SWG policy and spots sudden policy drift.

## Value

Time-series of blocked violations per URL category and rule — tunes SWG policy and spots sudden policy drift.

## Implementation

Map `rule_name` / `policy` from ZIA. Alert when daily blocks for a rule exceed 2× 7-day average (possible mis-tuned category).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Zscaler TA.
• Ensure the following data sources are available: `sourcetype=zscaler:web` or `zscaler:zia`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map `rule_name` / `policy` from ZIA. Alert when daily blocks for a rule exceed 2× 7-day average (possible mis-tuned category).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=proxy sourcetype="zscaler:web" earliest=-30d
| where action="blocked" OR threat_score>0
| timechart span=1d count by rule_name
```
