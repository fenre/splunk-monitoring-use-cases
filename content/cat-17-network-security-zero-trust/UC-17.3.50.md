# UC-17.3.50 · Akamai Guardicore Reveal Map Anomalies

## Description

Guardicore's Reveal feature maps all process-level communication between workloads. New or unexpected connections that deviate from the learned application dependency map may indicate lateral movement, compromised workloads, or configuration drift.

## Value

Guardicore's Reveal feature maps all process-level communication between workloads. New or unexpected connections that deviate from the learned application dependency map may indicate lateral movement, compromised workloads, or configuration drift.

## Implementation

Export the Guardicore application dependency map as a lookup. Run anomaly detection against new flows. Alert on never-before-seen process-port combinations to critical segments. Tune for deployment rollouts.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Akamai Guardicore Add-on for Splunk (Splunkbase 7426).
• Ensure the following data sources are available: Akamai Guardicore flow/connection data.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Export the Guardicore application dependency map as a lookup. Run anomaly detection against new flows. Alert on never-before-seen process-port combinations to critical segments. Tune for deployment rollouts.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=microseg sourcetype="guardicore:network" earliest=-24h
| eval flow_key=src_asset.":".tostring(src_port)."->".dst_asset.":".tostring(dst_port)
| lookup known_flows.csv flow_key OUTPUT known
| where isnull(known)
| stats count by src_asset, dst_asset, dst_port, process_name
| where count > 5
| sort -count
```
