# UC-17.3.18 · Device Trust Score Trending

## Description

Fleet-level and cohort trend of device trust scores — extends point-in-time UC-17.3.2.

## Value

Fleet-level and cohort trend of device trust scores — extends point-in-time UC-17.3.2.

## Implementation

Ensure `trust_score` is numeric 0–100. Alert when 7-day moving average drops >10 points for Windows corporate fleet.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler`, `Splunk_TA_microsoft-cloudservices` (Entra ID), `TA-crowdstrike-falcon`.
• Ensure the following data sources are available: `zscaler:device_posture`, `zt:device_trust`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ensure `trust_score` is numeric 0–100. Alert when 7-day moving average drops >10 points for Windows corporate fleet.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=zt sourcetype="zscaler:device_posture" earliest=-30d
| timechart span=1d avg(trust_score) as avg_trust by os_type
```
