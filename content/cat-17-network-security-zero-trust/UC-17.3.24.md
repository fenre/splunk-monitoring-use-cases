# UC-17.3.24 · Conditional Access Policy Enforcement (Entra ID)

## Description

Volume of grants vs blocks per named CA policy — complements generic UC-17.3.1 with Microsoft-specific policy dimension.

## Value

Volume of grants vs blocks per named CA policy — complements generic UC-17.3.1 with Microsoft-specific policy dimension.

## Implementation

Include `failureReason` for blocks. Alert when block rate for a policy jumps without change ticket.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Azure / Entra TA.
• Ensure the following data sources are available: `sourcetype=azure:signin` with `conditional_access_status`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Include `failureReason` for blocks. Alert when block rate for a policy jumps without change ticket.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=identity sourcetype="azure:signin" earliest=-7d
| where isnotnull(conditional_access_policy_name)
| stats count by conditional_access_policy_name, conditional_access_status
| sort -count
```
