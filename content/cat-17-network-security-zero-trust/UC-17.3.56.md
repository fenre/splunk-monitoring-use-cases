# UC-17.3.56 · Cloudflare Access (ZTNA) Policy Enforcement

## Description

Cloudflare Access replaces VPN with per-application zero-trust access policies based on identity, device posture, and network context. Tracking allow/block decisions validates that access policies are correctly enforced.

## Value

Cloudflare Access replaces VPN with per-application zero-trust access policies based on identity, device posture, and network context. Tracking allow/block decisions validates that access policies are correctly enforced.

## Implementation

Configure Cloudflare Logpush to push Access logs to Splunk via HEC or S3. Map `app_name` to internal application names. Alert when block rate spikes for specific applications. Report on access patterns by geography.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Cloudflare App for Splunk (Splunkbase 4501).
• Ensure the following data sources are available: Cloudflare Logpush (Access logs), `sourcetype=cloudflare:access`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Configure Cloudflare Logpush to push Access logs to Splunk via HEC or S3. Map `app_name` to internal application names. Alert when block rate spikes for specific applications. Report on access patterns by geography.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=zt sourcetype="cloudflare:access" earliest=-7d
| stats count by action, app_name, user_email, country
| sort -count
```
