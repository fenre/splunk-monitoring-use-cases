# UC-17.1.2 · Endpoint Posture Failures

## Description

Non-compliant endpoints accessing the network pose security risks. Posture tracking ensures endpoint hygiene enforcement.

## Value

Non-compliant endpoints accessing the network pose security risks. Posture tracking ensures endpoint hygiene enforcement.

## Implementation

Ingest ISE posture assessment results. Track compliance rates per policy (AV status, patch level, disk encryption). Alert on critical endpoints failing posture (exec laptops, admin workstations). Report on remediation effectiveness.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-ise`, HPE Aruba ClearPass syslog, Forescout CounterACT syslog.
• Ensure the following data sources are available: ISE posture assessment logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=nac sourcetype="cisco:ise:posture"
| where posture_status="NonCompliant"
| stats count by endpoint_mac, posture_policy, failure_reason
| sort -count
```
