# UC-17.2.8 · Simultaneous Session Detection

## Description

A single user with simultaneous VPN sessions from different locations strongly indicates credential compromise.

## Value

A single user with simultaneous VPN sessions from different locations strongly indicates credential compromise.

## Implementation

Track active VPN sessions per user. Alert when a user has concurrent sessions from different IPs. Whitelist known scenarios (multiple devices). Trigger automated investigation including password reset.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-asa`, `Splunk_TA_paloalto` (GlobalProtect), `TA-fortinet_fortigate`, `Splunk_TA_juniper`.
• Ensure the following data sources are available: VPN session logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track active VPN sessions per user. Alert when a user has concurrent sessions from different IPs. Whitelist known scenarios (multiple devices). Trigger automated investigation including password reset.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="cisco:asa" action="session_connect"
| stats dc(src) as unique_ips, values(src) as ips by user
| where unique_ips > 1
```
