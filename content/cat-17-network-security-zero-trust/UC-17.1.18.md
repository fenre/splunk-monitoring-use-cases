# UC-17.1.18 · NAC Policy Compliance Trending

## Description

Daily percentage of authentications that receive “permit” vs “deny” vs “redirect” per authorization policy — trending for policy drift and rollout validation.

## Value

Daily percentage of authentications that receive “permit” vs “deny” vs “redirect” per authorization policy — trending for policy drift and rollout validation.

## Implementation

Normalize `authorization_result` from your ISE field set. Alert when `deny` share increases >2× baseline week-over-week.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk_TA_cisco-ise.
• Ensure the following data sources are available: `cisco:ise:auth`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize `authorization_result` from your ISE field set. Alert when `deny` share increases >2× baseline week-over-week.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=nac sourcetype="cisco:ise:auth" earliest=-30d
| eval outcome=case(match(lower(authorization_result),"(?i)permit|access.accept"),"permit", match(lower(authorization_result),"(?i)deny|reject"),"deny", true(),"other")
| timechart span=1d count by outcome
```
