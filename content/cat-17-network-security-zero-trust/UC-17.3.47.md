# UC-17.3.47 · Check Point Harmony SASE Admin Audit (Check Point)

## Description

Centralized SASE policy changes affect all connected users and sites. Audit logs enable compliance, change correlation, and accountability.

## Value

Centralized SASE policy changes affect all connected users and sites. Audit logs enable compliance, change correlation, and accountability.

## Implementation

Forward audit logs via Log Exporter or Smart-1 Cloud. Alert on after-hours changes or policy publishes without change ticket. Correlate access anomalies with recent policy changes.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Check Point App for Splunk (Splunkbase 4293).
• Ensure the following data sources are available: `sourcetype=cp_log` (audit/admin logs).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Forward audit logs via Log Exporter or Smart-1 Cloud. Alert on after-hours changes or policy publishes without change ticket. Correlate access anomalies with recent policy changes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=checkpoint sourcetype="cp_log" product="SmartConsole" earliest=-30d
| where match(lower(operation),"(?i)create|modify|delete|publish|install")
| stats count earliest(_time) as first latest(_time) as last values(operation) as ops by administrator, object_name
| sort -last
```
