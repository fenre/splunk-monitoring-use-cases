# UC-17.2.14 · Geo-Impossible VPN Connections

## Description

Detects logins from two distant countries faster than plausible travel — complements static geo allowlists (UC-17.2.3).

## Value

Detects logins from two distant countries faster than plausible travel — complements static geo allowlists (UC-17.2.3).

## Implementation

Tune time window (e.g., 4–8h) and distance (optional `haversine` if lat/long from enriched data). Whitelist mobile users and split tunnel carrier NAT.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-asa`, `Splunk_TA_paloalto` (GlobalProtect), GeoIP.
• Ensure the following data sources are available: VPN session connect with `src`, `user`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Tune time window (e.g., 4–8h) and distance (optional `haversine` if lat/long from enriched data). Whitelist mobile users and split tunnel carrier NAT.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="cisco:asa" action="session_connect" earliest=-24h
| iplocation src
| eval country=Country
| sort user _time
| streamstats current=f last(_time) as prev_time last(Country) as prev_country by user
| eval gap_hrs=round((_time-prev_time)/3600,2)
| where isnotnull(prev_country) AND country!=prev_country AND gap_hrs < 6
| table _time, user, prev_country, country, gap_hrs, src
```
