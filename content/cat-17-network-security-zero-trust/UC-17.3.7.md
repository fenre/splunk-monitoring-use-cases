# UC-17.3.7 · Device Certificate Expiration and Renewal

## Description

Expired device certificates break ZTNA and VPN access. Monitoring expiration and renewal success ensures continuous access and avoids outages.

## Value

Expired device certificates break ZTNA and VPN access. Monitoring expiration and renewal success ensures continuous access and avoids outages.

## Implementation

Ingest device certificate inventory and renewal events. Alert when cert expires in <30 days or renewal fails. Report on cert distribution and renewal success rate. Automate renewal where possible.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: PKI/certificate inventory, `Splunk_TA_zscaler`, Netskope Add-on for Splunk (Splunkbase 3808), `Splunk_TA_paloalto` (Prisma Access).
• Ensure the following data sources are available: Certificate validity, renewal requests, enrollment events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest device certificate inventory and renewal events. Alert when cert expires in <30 days or renewal fails. Report on cert distribution and renewal success rate. Automate renewal where possible.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=zt sourcetype="device:cert"
| eval days_left=floor((expiry_time-now())/86400)
| where days_left < 30 OR renewal_status="failed"
| table device_id, cn, expiry_time, days_left, renewal_status
| sort days_left
```
