# UC-17.2.21 · SSL VPN Certificate Compliance

## Description

Tracks server certificate expiry and chain errors on SSL VPN / GlobalProtect portals from TLS handshake logs.

## Value

Tracks server certificate expiry and chain errors on SSL VPN / GlobalProtect portals from TLS handshake logs.

## Implementation

Prefer proactive cert inventory from PKI; this search catches client-reported errors. Alert on any `expired` match on production gateways.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-asa`, `Splunk_TA_paloalto`.
• Ensure the following data sources are available: SSL/TLS syslog, management logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Prefer proactive cert inventory from PKI; this search catches client-reported errors. Alert on any `expired` match on production gateways.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn (sourcetype="cisco:asa" OR sourcetype="pan:system") earliest=-30d
| search "certificate" AND ("expired" OR "not trusted" OR "invalid")
| stats count by host, cert_cn, message
| sort -count
```
