# UC-17.4.5 · Cloudflare Page Rules and Redirect Chain Depth Analysis (`cloudflare:json`)

## Description

Redirect loops and excessive hops hurt SEO and mobile latency. HTTP request logs expose chains that configuration UIs hide.

## Value

Marketing and platform teams fix mis-issued page rules before search ranking drops.

## Implementation

Configure Logpush jobs per dataset; route all into a unified `cloudflare:json` sourcetype or use transforms to normalize. Use `spath` for PascalCase fields. Create macros per zone name.

## SPL

```spl
index=cdn sourcetype="cloudflare:json" earliest=-7d
| spath ClientRequestURI
| spath EdgeResponseStatus
| where EdgeResponseStatus IN (301,302,307,308)
| eval hops=1
| stats sum(hops) as redirects by ClientRequestHost, ClientRequestURI
| where redirects>3
| sort -redirects
```

## Visualization

Sankey or table (from→to status), histogram (hop count).
