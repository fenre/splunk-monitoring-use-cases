# UC-17.4.9 · Cloudflare DNS Query Volume and NXDOMAIN Ratio (`cloudflare:json`)

## Description

NXDOMAIN storms often precede DGA malware or misconfigured applications hammering bad suffixes.

## Value

Early warning for botnet infections and DNS-based data exfil attempts.

## Implementation

Configure Logpush jobs per dataset; route all into a unified `cloudflare:json` sourcetype or use transforms to normalize. Use `spath` for PascalCase fields. Create macros per zone name.

## SPL

```spl
index=dns sourcetype="cloudflare:json" earliest=-24h
| where match(_raw,"(?i)DNS")
| spath QueryType
| spath ResponseCode
| bin _time span=15m
| stats count count(eval(ResponseCode="NXDOMAIN")) as nx by _time
| eval nx_pct=round(100*nx/count,2)
| where nx_pct>15
```
