# UC-17.3.13 · ZPA Application Segment Health

## Description

Tracks connector health, app segment reachability, and error rates for ZPA-published apps.

## Value

Tracks connector health, app segment reachability, and error rates for ZPA-published apps.

## Implementation

Normalize `app_segment` and latency fields from your ZPA TA. Alert on connector group with >5% error rate vs prior week.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Zscaler TA.
• Ensure the following data sources are available: `sourcetype=zscaler:zpa`, connector telemetry.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize `app_segment` and latency fields from your ZPA TA. Alert on connector group with >5% error rate vs prior week.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=zt sourcetype="zscaler:zpa" earliest=-24h
| where match(lower(status),"(?i)fail|error|down") OR latency_ms>2000
| stats count by app_segment, connector_group, error_code
| sort 30 -count
```
