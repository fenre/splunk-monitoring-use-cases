# UC-17.3.22 · ZTNA Application Access Latency

## Description

p95 latency per published application for user experience SLAs on ZTNA paths.

## Value

p95 latency per published application for user experience SLAs on ZTNA paths.

## Implementation

Segment by connector group and region. Compare before/after app migrations.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler` (ZPA), Cloudflare Logpush integration.
• Ensure the following data sources are available: `zscaler:zpa` access logs with `latency_ms`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Segment by connector group and region. Compare before/after app migrations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=zt sourcetype="zscaler:zpa" earliest=-24h
| stats perc95(latency_ms) as p95_ms, count by application
| where p95_ms > 800
| sort -p95_ms
```
