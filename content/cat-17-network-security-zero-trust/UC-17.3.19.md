# UC-17.3.19 · Continuous Authentication Compliance

## Description

Tracks step-up auth, re-auth, and session risk evaluation outcomes for policies requiring continuous verification.

## Value

Tracks step-up auth, re-auth, and session risk evaluation outcomes for policies requiring continuous verification.

## Implementation

Map Entra `riskLevelDuringSignIn` and CA grant controls. Report MFA completion rate when risk elevated.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_microsoft-cloudservices` (Entra ID Protection), `Splunk_TA_okta`, Zscaler ZPA TA.
• Ensure the following data sources are available: `sourcetype=azure:signin`, `okta:system`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map Entra `riskLevelDuringSignIn` and CA grant controls. Report MFA completion rate when risk elevated.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=identity sourcetype="azure:signin" earliest=-7d
| where risk_level!="none" OR authentication_requirement="multiFactorAuthentication"
| stats count by user, risk_detail, result
| sort 40 -count
```
