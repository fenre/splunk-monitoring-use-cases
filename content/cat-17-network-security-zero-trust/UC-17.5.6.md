# UC-17.5.6 · Forcepoint ZTNA Private Application Access Latency (`forcepoint:one`)

## Description

High latency on private apps drives users to bypass ZTNA with split tunnel tricks.

## Value

Network teams optimize connector placement before shadow paths appear.

## Implementation

Install the Insights SIEM app, configure HEC inputs per product line, and align indexes (`proxy`, `email`, `dlp`, `sase`). Normalize usernames to enterprise SAMAccountName.

## SPL

```spl
index=sase sourcetype="forcepoint:one" earliest=-24h
| where match(lower(event_type),"(?i)ztna|private app")
| stats perc95(latency_ms) as p95_ms by app_name, connector_name
| where p95_ms>300
| sort -p95_ms
```

## Visualization

Bar chart (p95 latency), map (connector region).
