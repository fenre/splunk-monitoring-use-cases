# UC-17.3.4 · ZTNA Application Access

## Description

Per-application access patterns in ZTNA reveal usage trends, security risks, and application performance issues.

## Value

Per-application access patterns in ZTNA reveal usage trends, security risks, and application performance issues.

## Implementation

Track application access through ZTNA per user and device. Identify unused applications for decommissioning. Monitor access patterns for anomalies. Report on application adoption and usage.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_zscaler` (ZPA), Netskope Add-on for Splunk (Splunkbase 3808), `Splunk_TA_paloalto` (Prisma Access), `TA-fortinet_fortigate` (FortiSASE), Check Point App for Splunk (Splunkbase 4293).
• Ensure the following data sources are available: ZTNA access logs (application, user, device, action).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track application access through ZTNA per user and device. Identify unused applications for decommissioning. Monitor access patterns for anomalies. Report on application adoption and usage.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=zt sourcetype="zscaler:zpa"
| stats dc(user) as unique_users, count as total_access by application
| sort -unique_users
```
