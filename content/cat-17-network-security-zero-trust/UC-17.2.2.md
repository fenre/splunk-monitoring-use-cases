# UC-17.2.2 · VPN Authentication Failures

## Description

Repeated VPN auth failures indicate credential attacks against the remote access perimeter, a primary attack vector.

## Value

Repeated VPN auth failures indicate credential attacks against the remote access perimeter, a primary attack vector.

## Implementation

Track VPN authentication failures by user and source IP. Alert on >5 failures per user per 15 minutes. Correlate with AD lockout events. Block source IPs with excessive failures. Report on attack patterns.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-asa`, `Splunk_TA_paloalto` (GlobalProtect), `TA-fortinet_fortigate`, `Splunk_TA_juniper`.
• Ensure the following data sources are available: VPN authentication logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track VPN authentication failures by user and source IP. Alert on >5 failures per user per 15 minutes. Correlate with AD lockout events. Block source IPs with excessive failures. Report on attack patterns.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="cisco:asa" action="authentication_failed"
| stats count by user, src
| where count > 5
| sort -count
```
