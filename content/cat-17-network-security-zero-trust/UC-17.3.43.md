# UC-17.3.43 · FortiSASE Admin Configuration Audit (Fortinet)

## Description

FortiSASE policies are centrally managed and affect all connected users globally. Configuration audit logs enable change tracking, compliance, and root-cause analysis when policies cause access issues.

## Value

FortiSASE policies are centrally managed and affect all connected users globally. Configuration audit logs enable change tracking, compliance, and root-cause analysis when policies cause access issues.

## Implementation

Forward FortiSASE management audit events. Alert on after-hours changes or bulk policy modifications. Correlate with ITSM change tickets.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `TA-fortinet_fortigate` (FortiSASE admin audit).
• Ensure the following data sources are available: `sourcetype=fgt_event` subtype=system, logid related to admin/config.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Forward FortiSASE management audit events. Alert on after-hours changes or bulk policy modifications. Correlate with ITSM change tickets.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sase sourcetype="fgt_event" subtype="system" earliest=-30d
| where match(lower(msg),"(?i)config|policy|rule|admin.*login|object.*modified")
| stats count earliest(_time) as first latest(_time) as last values(msg) as actions by user, ui
| sort -last
```
