# UC-17.2.12 · VPN Concentrator Capacity

## Description

Tracks session count and CPU/memory against platform limits to avoid remote-access brownouts during peaks.

## Value

Tracks session count and CPU/memory against platform limits to avoid remote-access brownouts during peaks.

## Implementation

Prefer vendor metrics (e.g., AnyConnect session count OID). Alert when CPU >80% sustained or sessions >85% of license. Simplify if only session logs: use UC-17.2.1 trend + license field.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-asa`, `Splunk_TA_paloalto` (GlobalProtect), SNMP TA.
• Ensure the following data sources are available: SNMP OIDs, `cisco:asa` system events, vendor metrics API.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Prefer vendor metrics (e.g., AnyConnect session count OID). Alert when CPU >80% sustained or sessions >85% of license. Simplify if only session logs: use UC-17.2.1 trend + license field.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=snmp sourcetype="snmp:cpu" host="vpn-headend-*" earliest=-24h
| timechart span=15m avg(cpu_utilization) as avg_cpu by host
```
