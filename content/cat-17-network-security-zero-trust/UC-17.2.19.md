# UC-17.2.19 · Always-On VPN Enforcement

## Description

Identifies corporate assets connecting without Always-On (pre-login) VPN when policy requires it.

## Value

Identifies corporate assets connecting without Always-On (pre-login) VPN when policy requires it.

## Implementation

Prefer explicit field from ASA if available. Join MDM “managed device” list for `requires_aov`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk_TA_cisco-asa, endpoint inventory.
• Ensure the following data sources are available: VPN logs with `always_on` flag, MDM compliance.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Prefer explicit field from ASA if available. Join MDM “managed device” list for `requires_aov`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="cisco:asa" action="session_connect" earliest=-24h
| eval aov=if(match(lower(_raw),"(?i)always.?on|pre.?login"),1,0)
| lookup corp_laptops.csv hostname OUTPUT requires_aov
| where requires_aov=1 AND aov=0
| stats count by user, hostname, src
```
