# UC-17.3.39 · FortiSASE SWG Policy Violation Trends (Fortinet)

## Description

FortiSASE cloud-delivered SWG enforces web filtering for remote and branch users through FortiGate NGFW policies in the cloud. Trending blocked categories and rules identifies policy drift and shadow IT.

## Value

FortiSASE cloud-delivered SWG enforces web filtering for remote and branch users through FortiGate NGFW policies in the cloud. Trending blocked categories and rules identifies policy drift and shadow IT.

## Implementation

Forward FortiSASE logs via syslog to Splunk HF. The `TA-fortinet_fortigate` parses FortiSASE traffic identically to on-prem FortiGate. Alert when daily blocks exceed 2× 7-day baseline. Segment by FortiSASE PoP region for regional policy analysis.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `TA-fortinet_fortigate` (FortiSASE logs use FortiGate syslog format).
• Ensure the following data sources are available: `sourcetype=fgt_utm` (FortiSASE web filter logs via syslog).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Forward FortiSASE logs via syslog to Splunk HF. The `TA-fortinet_fortigate` parses FortiSASE traffic identically to on-prem FortiGate. Alert when daily blocks exceed 2× 7-day baseline. Segment by FortiSASE PoP region for regional policy analysis.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=proxy sourcetype="fgt_utm" subtype="webfilter" earliest=-7d
| where action="blocked"
| stats count by catdesc, user, policyname
| sort -count
```
