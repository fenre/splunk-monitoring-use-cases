# UC-17.2.15 · VPN Tunnel Keepalive Failure Analysis

## Description

Tracks DPD/keepalive failures and tunnel teardown reasons for site-to-site and remote-access — isolates path MTU, NAT, and idle timeout issues.

## Value

Tracks DPD/keepalive failures and tunnel teardown reasons for site-to-site and remote-access — isolates path MTU, NAT, and idle timeout issues.

## Implementation

Normalize `message_signature` with `rex` or `cluster` on raw. Correlate with UC-17.2.5 stability metrics.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-asa`, `Splunk_TA_paloalto`.
• Ensure the following data sources are available: VPN/IKE syslog (`cisco:asa`, `pan:system`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize `message_signature` with `rex` or `cluster` on raw. Correlate with UC-17.2.5 stability metrics.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn (sourcetype="cisco:asa" OR sourcetype="pan:system") earliest=-24h
| search "IKE" OR "keepalive" OR "DPD" OR "dead peer"
| stats count by tunnel_id, peer_ip, message_signature
| sort 40 -count
```
