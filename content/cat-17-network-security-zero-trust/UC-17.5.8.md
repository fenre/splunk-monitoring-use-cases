# UC-17.5.8 · Forcepoint User Risk Score Changes Over Time (`forcepoint:one`)

## Description

Step changes in risk score often precede account compromise or policy violations.

## Value

SOC can trigger step-up MFA or manager notification automatically.

## Implementation

Install the Insights SIEM app, configure HEC inputs per product line, and align indexes (`proxy`, `email`, `dlp`, `sase`). Normalize usernames to enterprise SAMAccountName.

## SPL

```spl
index=sase sourcetype="forcepoint:one" earliest=-7d
| where isnotnull(user_risk_score)
| bin _time span=1h
| stats latest(user_risk_score) as risk by user, _time
| streamstats window=2 global=f latest(risk) as prev by user
| eval delta=risk-prev
| where delta>=20
```

## Visualization

Line chart (risk trajectory), alert on delta.
