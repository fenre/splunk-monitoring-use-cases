# UC-17.3.61 · SonicWall Cloud SWG and SMA Access Events (SonicWall)

## Description

SonicWall Cloud SWG and Secure Mobile Access (SMA) provide cloud-delivered web security and ZTNA for remote and branch users. Tracking blocked connections, authentication events, and policy actions validates secure access enforcement.

## Value

SonicWall Cloud SWG and Secure Mobile Access (SMA) provide cloud-delivered web security and ZTNA for remote and branch users. Tracking blocked connections, authentication events, and policy actions validates secure access enforcement.

## Implementation

Forward SonicWall Cloud SWG/SMA logs via syslog or use the SMA 1000 TA. Map `cat` (web category) to acceptable-use policy. Alert on authentication failures from SMA. Report on policy enforcement effectiveness.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: SonicWall SMA 1000 TA (Splunkbase 6670), `dell:sonicwall:firewall` syslog.
• Ensure the following data sources are available: `sourcetype=dell:sonicwall:firewall` (syslog), SMA 1000 logs via TA.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Forward SonicWall Cloud SWG/SMA logs via syslog or use the SMA 1000 TA. Map `cat` (web category) to acceptable-use policy. Alert on authentication failures from SMA. Report on policy enforcement effectiveness.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=proxy sourcetype="dell:sonicwall:firewall" earliest=-7d
| where fw_action IN ("block","deny","drop")
| stats count by cat, usr, fw_action, rule
| sort -count
```
