# UC-17.2.7 · VPN Bandwidth Consumption

## Description

Per-user bandwidth tracking identifies heavy users, guides capacity planning, and detects potential data exfiltration.

## Value

Per-user bandwidth tracking identifies heavy users, guides capacity planning, and detects potential data exfiltration.

## Implementation

Track VPN session byte counters per user. Alert on users with excessive upload (potential data exfiltration). Report on bandwidth distribution for capacity planning. Identify optimization opportunities (video offload, split-tunnel).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-asa`, `Splunk_TA_paloalto` (GlobalProtect), `TA-fortinet_fortigate`, `Splunk_TA_juniper`, RADIUS accounting.
• Ensure the following data sources are available: VPN session accounting (bytes in/out).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track VPN session byte counters per user. Alert on users with excessive upload (potential data exfiltration). Report on bandwidth distribution for capacity planning. Identify optimization opportunities (video offload, split-tunnel).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="cisco:asa"
| stats sum(bytes_in) as bytes_in, sum(bytes_out) as bytes_out by user
| eval total_gb=round((bytes_in+bytes_out)/1073741824,2)
| sort -total_gb
| head 20
```
