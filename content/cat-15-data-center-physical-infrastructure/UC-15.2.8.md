# UC-15.2.8 · CRAC Unit Failure and Alarm State

## Description

Explicit CRAC fault/alarm bits catch compressor and fan failures not visible in temperature alone.

## Value

Explicit CRAC fault/alarm bits catch compressor and fan failures not visible in temperature alone.

## Implementation

Map vendor alarm codes to plain text. Page on any fault. Track MTTR for CRAC repairs.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: BMS, SNMP (Liebert/Vertiv MIBs).
• Ensure the following data sources are available: `sourcetype="bms:crac"` (unit_alarm, compressor_fault, fan_fault).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map vendor alarm codes to plain text. Page on any fault. Track MTTR for CRAC repairs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cooling sourcetype="bms:crac"
| where unit_alarm=1 OR compressor_fault=1 OR fan_fault=1 OR unit_status="fault"
| table _time, unit_name, unit_alarm, compressor_fault, fan_fault, alarm_text
```
