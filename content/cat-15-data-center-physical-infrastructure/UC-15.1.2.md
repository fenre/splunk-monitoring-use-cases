# UC-15.1.2 · PDU Power per Rack

## Description

Per-rack power monitoring prevents circuit overloads and enables efficient rack placement for new equipment.

## Value

Per-rack power monitoring prevents circuit overloads and enables efficient rack placement for new equipment.

## Implementation

Poll PDU metrics via SNMP. Track per-outlet and per-circuit power. Alert when any circuit exceeds 80% capacity. Report on rack power distribution for capacity planning. Track power trends per rack.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: SNMP TA (PDU-MIB), vendor API.
• Ensure the following data sources are available: Smart PDU per-outlet and per-circuit metrics.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll PDU metrics via SNMP. Track per-outlet and per-circuit power. Alert when any circuit exceeds 80% capacity. Report on rack power distribution for capacity planning. Track power trends per rack.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=power sourcetype="snmp:pdu"
| eval pct_capacity=round(current_amps/rated_amps*100,1)
| where pct_capacity > 80
| table rack_id, pdu_name, circuit, current_amps, rated_amps, pct_capacity
```
