# UC-15.2.6 · Cooling Capacity Planning

## Description

Trending cooling load vs capacity ensures adequate cooling for current and planned equipment deployments.

## Value

Trending cooling load vs capacity ensures adequate cooling for current and planned equipment deployments.

## Implementation

Calculate cooling load from IT power consumption (1 watt IT ≈ 3.41 BTU/h heat). Compare against total cooling capacity. Track utilization percentage. Alert when approaching 80% capacity. Plan for seasonal variations.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: BMS metrics.
• Ensure the following data sources are available: CRAC/CRAH cooling output, IT heat load calculations.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Calculate cooling load from IT power consumption (1 watt IT ≈ 3.41 BTU/h heat). Compare against total cooling capacity. Track utilization percentage. Alert when approaching 80% capacity. Plan for seasonal variations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cooling sourcetype="bms:cooling_capacity"
| timechart span=1d avg(cooling_output_kw) as output, avg(cooling_capacity_kw) as capacity
| eval utilization_pct=round(output/capacity*100,1)
```
