# UC-15.1.18 · Generator Start Failure

## Description

Failed cranks or inability to reach rated speed leave the site on battery-only until resolved.

## Value

Failed cranks or inability to reach rated speed leave the site on battery-only until resolved.

## Implementation

Alert immediately on failed start during tests or utility loss. Track battery, starter, and fuel subsystem codes.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Generator controller.
• Ensure the following data sources are available: `sourcetype="generator:events"` (start_result, crank_count, fault_code).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Alert immediately on failed start during tests or utility loss. Track battery, starter, and fuel subsystem codes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=power sourcetype="generator:events"
| search start_result IN ("fail","abort") OR crank_count > 3
| table _time, generator_id, start_result, crank_count, fault_code
| sort -_time
```
