# UC-15.3.16 · Camera Feed Loss and Recording Gap Detection

## Description

Detects loss of RTSP/HLS stream and gaps in NVR recording separately from generic camera “offline” pings.

## Value

Detects loss of RTSP/HLS stream and gaps in NVR recording separately from generic camera “offline” pings.

## Implementation

Poll VMS for per-camera FPS and last recorded frame time. Alert on stream loss or gap >1 min for critical zones.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: VMS API, stream health probes.
• Ensure the following data sources are available: `sourcetype="vms:stream_health"` (fps, keyframes, recording_continuity).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll VMS for per-camera FPS and last recorded frame time. Alert on stream loss or gap >1 min for critical zones.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype="vms:stream_health"
| where fps=0 OR recording_gap_sec > 60 OR stream_state="lost"
| stats latest(recording_gap_sec) as gap by camera_id, site
| sort -gap
```
