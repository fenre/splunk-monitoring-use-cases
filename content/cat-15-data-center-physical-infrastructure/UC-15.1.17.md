# UC-15.1.17 · UPS Self-Test Failure

## Description

Failed self-tests indicate battery or inverter faults before a real outage.

## Value

Failed self-tests indicate battery or inverter faults before a real outage.

## Implementation

Ingest results from scheduled and manual tests. Alert on any failure. Force battery replacement workflow per vendor guidance.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: UPS SNMP (upsTestResultsSummary).
• Ensure the following data sources are available: `sourcetype="snmp:ups:selftest"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest results from scheduled and manual tests. Alert on any failure. Force battery replacement workflow per vendor guidance.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=power sourcetype="snmp:ups:selftest"
| where test_result!="passed" OR upsTestResultsSummary!="donePass"
| table _time, ups_name, test_type, test_result, upsTestResultsSummary
```
