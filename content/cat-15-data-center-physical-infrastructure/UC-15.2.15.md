# UC-15.2.15 · Economizer Mode Utilization

## Description

Low economizer hours may indicate stuck dampers or bad enthalpy programming—wasting energy.

## Value

Low economizer hours may indicate stuck dampers or bad enthalpy programming—wasting energy.

## Implementation

Compare economizer hours to weather bin data. Investigate AHU with low free-cooling in mild weather.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: BMS.
• Ensure the following data sources are available: `sourcetype="bms:economizer"` (mode, oa_temp_f, enthalpy_ok).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Compare economizer hours to weather bin data. Investigate AHU with low free-cooling in mild weather.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cooling sourcetype="bms:economizer"
| timechart span=1d sum(eval(if(mode="economizer",1,0))) as econ_hours by ahu_id
| where econ_hours < 4
```
