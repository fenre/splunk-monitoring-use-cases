# UC-15.3.8 · Raised Floor and Cable Management Events

## Description

Floor tile removal or cable strain events can indicate unauthorized work or trip hazards. Monitoring supports physical security and change audit.

## Value

Floor tile removal or cable strain events can indicate unauthorized work or trip hazards. Monitoring supports physical security and change audit.

## Implementation

Deploy sensors for critical floor tiles and cable runs. Forward events to Splunk. Alert on tile removal or strain above threshold. Correlate with change tickets. Report on access and strain history.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Floor/cable sensors, DCIM.
• Ensure the following data sources are available: Tile position, cable tension or movement sensors.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Deploy sensors for critical floor tiles and cable runs. Forward events to Splunk. Alert on tile removal or strain above threshold. Correlate with change tickets. Report on access and strain history.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype="floor:sensor"
| search (tile_removed="true" OR cable_strain > 80)
| table _time, location, tile_id, cable_strain, operator
| sort -_time
```
