# UC-15.2.17 · Cooling Redundancy Status

## Description

N+1 cooling requires knowing when one CRAC is down and remaining capacity still covers peak IT load.

## Value

N+1 cooling requires knowing when one CRAC is down and remaining capacity still covers peak IT load.

## Implementation

Set `required_tons` from peak IT load × safety factor. Alert when redundancy lost. Block new installs if margin negative.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: BMS, cooling plant model.
• Ensure the following data sources are available: `sourcetype="bms:cooling_redundancy"` (online_tons, required_tons, units_down).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Set `required_tons` from peak IT load × safety factor. Alert when redundancy lost. Block new installs if margin negative.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cooling sourcetype="bms:cooling_redundancy"
| where online_tons < required_tons * 1.1 OR units_down > 0
| table _time, zone, online_tons, required_tons, units_down, unit_list
```
