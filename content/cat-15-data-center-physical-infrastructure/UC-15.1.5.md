# UC-15.1.5 · PUE Calculation

## Description

Power Usage Effectiveness is the primary data center efficiency metric. Trending drives energy optimization and sustainability goals.

## Value

Power Usage Effectiveness is the primary data center efficiency metric. Trending drives energy optimization and sustainability goals.

## Implementation

Aggregate total facility power and IT equipment power from PDU/UPS/BMS data. Calculate PUE hourly and daily. Track seasonal variation. Report monthly to operations and sustainability teams. Target PUE <1.5.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Aggregate power metrics from PDU/UPS/BMS.
• Ensure the following data sources are available: Total facility power, IT load power.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Aggregate total facility power and IT equipment power from PDU/UPS/BMS data. Calculate PUE hourly and daily. Track seasonal variation. Report monthly to operations and sustainability teams. Target PUE <1.5.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=power sourcetype="power:aggregate"
| timechart span=1h avg(total_facility_kw) as facility, avg(it_load_kw) as it_load
| eval pue=round(facility/it_load,2)
```
