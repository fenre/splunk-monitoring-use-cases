# UC-15.1.14 · Power Factor Monitoring

## Description

Low PF increases utility demand charges and heats conductors; correction banks need verification.

## Value

Low PF increases utility demand charges and heats conductors; correction banks need verification.

## Implementation

Poll main switchboard meters. Alert when PF drops below utility contract threshold. Correlate with capacitor bank status if monitored.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Power meter SNMP (PQ meters).
• Ensure the following data sources are available: `sourcetype="power_meter:pq"` (pf, thd, kw, kvar).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll main switchboard meters. Alert when PF drops below utility contract threshold. Correlate with capacitor bank status if monitored.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=power sourcetype="power_meter:pq"
| where pf < 0.92 OR thd_pct > 8
| timechart span=15m avg(pf) as avg_pf by feed_id
```
