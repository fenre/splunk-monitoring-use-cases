# UC-14.7.2 ¬∑ PLC Tag Data Ingestion Validation

## Description

Missing or silent tag streams break dashboards, historians, and ML models; validating expected tags catches connector or network regressions before production impact.

## Value

Missing or silent tag streams break dashboards, historians, and ML models; validating expected tags catches connector or network regressions before production impact.

## Implementation

Normalize tag events so each sample carries gateway_id, tag_name, and source_device. Replace thresholds with per-device baselines from a lookup. For stricter checks, join to a required-tag lookup.

## Detailed Implementation

Prerequisites
‚ÄĘ Install and configure the required add-on or app: Litmus Edge (Splunk HEC connector).
‚ÄĘ Ensure the following data sources are available: `index=ot` `sourcetype="litmus:tag"` fields `gateway_id`, `tag_name`, `source_device`.
‚ÄĘ For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 ‚ÄĒ Configure data collection
Normalize tag events so each sample carries gateway_id, tag_name, and source_device. Replace thresholds with per-device baselines from a lookup. For stricter checks, join to a required-tag lookup.

Step 2 ‚ÄĒ Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="litmus:tag"
| bin _time span=5m
| stats count as events, dc(tag_name) as distinct_tags by gateway_id, source_device, _time
| where events < 1 OR distinct_tags < 1
| table _time, gateway_id, source_device, events, distinct_tags
```
