# UC-14.2.7 · Modbus TCP Anomaly Detection

## Description

Unusual read/write rates or register ranges can indicate process upset or malicious manipulation.

## Value

Unusual read/write rates or register ranges can indicate process upset or malicious manipulation.

## Implementation

Baseline requests per 5m per unit and function code. Use MLTK or `eventstats` for median. Alert on spikes without corresponding maintenance window.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk OT Intelligence, Modbus TA.
• Ensure the following data sources are available: `sourcetype="modbus:traffic"` or `modbus:gateway`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Baseline requests per 5m per unit and function code. Use MLTK or `eventstats` for median. Alert on spikes without corresponding maintenance window.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="modbus:traffic"
| bin _time span=5m
| stats count by _time, unit_id, function_code, src
| eventstats median(count) as med by unit_id, function_code
| where count > med * 5 AND count > 100
```
