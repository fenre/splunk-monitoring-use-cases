# UC-14.2.16 · Process Variable Trending

## Description

Slow drift before alarm limits supports predictive maintenance and energy optimization.

## Value

Slow drift before alarm limits supports predictive maintenance and energy optimization.

## Implementation

Use `predict` for critical tags. Alert when forecast crosses limits before physical alarm. Tune per process area.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OPC-UA historian.
• Ensure the following data sources are available: `sourcetype="opcua:process"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Use `predict` for critical tags. Alert when forecast crosses limits before physical alarm. Tune per process area.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="opcua:process"
| timechart span=1h avg(value) as avg_val by tag_name
| predict avg_val as pred future_timespan=24
| lookup ot_tag_limits.csv tag_name OUTPUT high_limit, low_limit
| where pred > high_limit OR pred < low_limit
```
