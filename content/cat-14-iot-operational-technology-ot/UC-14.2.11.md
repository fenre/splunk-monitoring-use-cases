# UC-14.2.11 · NERC CIP Compliance Checks

## Description

Evidence of electronic access controls, logging, and change management for bulk electric systems.

## Value

Evidence of electronic access controls, logging, and change management for bulk electric systems.

## Implementation

Tag in-scope assets and controls. Use saved searches per CIP requirement (e.g., access logging, 30-day log retention). Document in Splunk as authoritative evidence store.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Custom (CIP evidence packs), Splunk Enterprise Security.
• Ensure the following data sources are available: Firewall, VPN, AD, asset logs tagged `nerc_cip`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Tag in-scope assets and controls. Use saved searches per CIP requirement (e.g., access logging, 30-day log retention). Document in Splunk as authoritative evidence store.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security sourcetype IN ("vpn:log","firewall:traffic") nerc_cip=1
| where action="deny" OR match(user,"(?i)orphan")
| stats count by asset_id, control_id, evidence_type
```

Understanding this SPL
