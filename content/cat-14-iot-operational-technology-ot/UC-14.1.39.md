# UC-14.1.39 · Water Consumption Trending and Anomaly Detection

## Description

Undetected water leaks can waste thousands of liters daily and cause structural damage. Continuous flow monitoring detects leaks, running toilets, and irrigation faults within hours rather than the weeks it takes for a utility bill to reveal problems.

## Value

Undetected water leaks can waste thousands of liters daily and cause structural damage. Continuous flow monitoring detects leaks, running toilets, and irrigation faults within hours rather than the weeks it takes for a utility bill to reveal problems.

## Implementation

Install smart water meters on main feeds and key branches. Monitor flow during unoccupied hours — sustained flow indicates leaks or running fixtures. Alert when unoccupied consumption exceeds 3x baseline.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Smart water meters (Modbus, pulse), IoT flow sensors, building management API.
• Ensure the following data sources are available: `sourcetype=bms:water`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Install smart water meters on main feeds and key branches. Monitor flow during unoccupied hours — sustained flow indicates leaks or running fixtures. Alert when unoccupied consumption exceeds 3x baseline.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=building sourcetype="bms:water" meter_type="main_feed"
| bin _time span=1h
| stats sum(liters) as hourly_liters by building_name, _time
| eval hour=strftime(_time, "%H"), dow=strftime(_time, "%u")
| eval is_occupied=if(hour>=7 AND hour<=19 AND dow<=5, 1, 0)
| where is_occupied=0 AND hourly_liters > 50
| eventstats avg(hourly_liters) as baseline_unoccupied by building_name
| eval anomaly_ratio=round(hourly_liters/baseline_unoccupied, 1)
| where anomaly_ratio > 3
| table _time, building_name, hourly_liters, baseline_unoccupied, anomaly_ratio
```
