# UC-14.7.6 · Edge Data Transformation Error Rate

## Description

Transformation and normalization failures silently drop or corrupt OT fields; tracking error rates ties engineering changes to ingestion health.

## Value

Transformation and normalization failures silently drop or corrupt OT fields; tracking error rates ties engineering changes to ingestion health.

## Implementation

Route Litmus pipeline diagnostics to Splunk. Alert when error rate exceeds SLO (e.g., >1% over 15 minutes).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Litmus Edge (Splunk HEC connector).
• Ensure the following data sources are available: `index=ot` `sourcetype="litmus:edge"` error fields or message text.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Route Litmus pipeline diagnostics to Splunk. Alert when error rate exceeds SLO (e.g., >1% over 15 minutes).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="litmus:edge"
| eval is_error=if(match(lower(_raw), "transform error|mapping error|parse error|conversion failed"), 1, 0)
| bin _time span=15m
| stats sum(is_error) as errors, count as total by gateway_id, _time
| eval error_rate_pct=round(100*errors/total,2)
| where errors>0
| table _time, gateway_id, errors, total, error_rate_pct
```
