# UC-14.4.2 · Environmental Monitoring

## Description

Distributed environmental sensors provide early warning of conditions that could damage equipment or inventory.

## Value

Distributed environmental sensors provide early warning of conditions that could damage equipment or inventory.

## Implementation

Deploy environmental sensors in server rooms, warehouses, and facilities. Ingest via MQTT or API. Alert immediately on water leak or smoke detection. Track temperature/humidity trends per location.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: MQTT input, IoT platform API.
• Ensure the following data sources are available: Environmental sensor data (temperature, humidity, water leak, smoke).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Deploy environmental sensors in server rooms, warehouses, and facilities. Ingest via MQTT or API. Alert immediately on water leak or smoke detection. Track temperature/humidity trends per location.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iot sourcetype="sensor:environmental"
| where water_detected="true" OR smoke_detected="true" OR temp_f > 90
| table _time, sensor_id, location, alert_type, value
```
