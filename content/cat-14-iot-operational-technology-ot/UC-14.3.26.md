# UC-14.3.26 · Edge Analytics Container for Rolling Statistics & Threshold Logic

## Description

Computes advanced analytics (moving averages, percentiles, trend detection) locally, reducing cloud computation burden.

## Value

Computes advanced analytics (moving averages, percentiles, trend detection) locally, reducing cloud computation burden.

## Implementation

Build custom ARM64 container with NumPy/Pandas libraries (may require multi-stage build to reduce image size). Container implements rolling window statistics (5/15/60-minute moving averages) via gRPC sensor stream. Compute percentiles, trend lines, and detect threshold crossings locally. Publish results as new metrics to MQTT (Advanced Settings → MQTT Publish) or directly to Splunk via gRPC SDK. Store raw + computed metrics locally (3M backlog) for redundancy. Configure resource limits: memory 512MB, CPU 1.5 cores.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=edge-hub-data metric_name=temperature
| timechart avg(temperature) as temp_avg by host
| delta temp_avg as temp_trend
| stats avg(temp_trend) as avg_trend, stdev(temp_trend) as trend_std by host
| eval trend_anomaly=if(abs(temp_trend) > (avg_trend + (2*trend_std)), "YES", "NO")
```
