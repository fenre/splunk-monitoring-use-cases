# UC-14.9.12 · New Communication Flow Detection

## Description

In a stable OT environment, the set of communication flows between devices should be predictable. OT security platforms generate alerts when a previously unseen protocol flow appears between two components. New FTP, HTTP, or SSH flows in the control network may indicate lateral movement, data exfiltration, or unauthorized remote access tools.

## Value

In a stable OT environment, the set of communication flows between devices should be predictable. OT security platforms generate alerts when a previously unseen protocol flow appears between two components. New FTP, HTTP, or SSH flows in the control network may indicate lateral movement, data exfiltration, or unauthorized remote access tools.

## Implementation

Both platforms detect new communication flows after an initial learning period. Alert on new flows. Prioritize IT protocols appearing in OT zones (FTP, SSH, HTTP, RDP, SMB). Correlate with baseline deviations. Investigate source and destination to determine if the flow is legitimate.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=nozomi sourcetype="nozomi:link_events" status="new"
| bin _time span=1d
| stats count as new_flows, dc(protocols) as protocol_count, values(protocols) as protocol_list by _time
| eval concern=if(new_flows > 10, "High — investigate burst of new flows", "Normal")
| table _time, new_flows, protocol_count, protocol_list, concern
```
