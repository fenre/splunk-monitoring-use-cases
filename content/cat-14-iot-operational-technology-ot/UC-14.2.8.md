# UC-14.2.8 · OPC-UA Session Abuse

## Description

Excessive sessions or anonymous binds from new clients can indicate scanning or unauthorized access.

## Value

Excessive sessions or anonymous binds from new clients can indicate scanning or unauthorized access.

## Implementation

Ingest OPC-UA audit events. Whitelist known engineering hosts. Alert on anonymous session creation or high rejection rate.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OPC-UA server audit logs, Edge Hub.
• Ensure the following data sources are available: `sourcetype="opcua:session"` (server audit events).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest OPC-UA audit events. Whitelist known engineering hosts. Alert on anonymous session creation or high rejection rate.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="opcua:session"
| where event_type IN ("CreateSession","ActivateSession") AND (is_anonymous=1 OR rejected=1)
| bin _time span=1h
| stats dc(session_id) as sessions, dc(client_ip) as clients by server_endpoint, _time
| where sessions > 50 OR clients > 10
```
