# UC-14.1.12 · Firmware Version Compliance Across Fleet

## Description

Known-bad IOS/NX-OS/IOS-XE builds expose the network to CVEs; compliance reporting supports change windows.

## Value

Known-bad IOS/NX-OS/IOS-XE builds expose the network to CVEs; compliance reporting supports change windows.

## Implementation

Poll ENTITY-MIB / vendor firmware revision OIDs on a weekly schedule. Maintain CSV of approved builds per platform. Drive remediation tickets for non-compliant devices.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: SNMP TA (ENTITY-MIB, vendor OIDs), SolarWinds/Prime export.
• Ensure the following data sources are available: `sourcetype="snmp:inventory"` (sysDescr, entPhysicalFirmwareRev).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll ENTITY-MIB / vendor firmware revision OIDs on a weekly schedule. Maintain CSV of approved builds per platform. Drive remediation tickets for non-compliant devices.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="snmp:inventory"
| stats latest(firmware_version) as fw by device_name, model
| lookup approved_network_firmware.csv model OUTPUT approved_fw
| where fw!=approved_fw
| table device_name, model, fw, approved_fw
```
