# UC-14.4.13 · Firmware Update Compliance

## Description

OTA campaigns must complete; stragglers remain vulnerable to known exploits.

## Value

OTA campaigns must complete; stragglers remain vulnerable to known exploits.

## Implementation

Compare device inventory to OTA target per wave. Escalate devices >7 days behind.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: IoT DM, OTA job logs.
• Ensure the following data sources are available: `sourcetype="iot:ota"` (job_id, status, version).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Compare device inventory to OTA target per wave. Escalate devices >7 days behind.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iot sourcetype="iot:ota"
| stats latest(target_version) as tgt by device_id
| join max=1 device_id [
  search index=iot sourcetype="iot_platform:devices"
  | stats latest(firmware_version) as cur by device_id
]
| where cur!=tgt
| table device_id, cur, tgt
```

Understanding this SPL

• Joins to a subsearch with `join` — set `max=` to match cardinality and avoid silent truncation.
