# UC-14.4.10 · IoT Data Pipeline Throughput and Latency

## Description

Pipeline latency or drop in throughput affects real-time dashboards and alerts. Monitoring supports SLA and troubleshooting of ingestion and processing.

## Value

Pipeline latency or drop in throughput affects real-time dashboards and alerts. Monitoring supports SLA and troubleshooting of ingestion and processing.

## Implementation

Ingest pipeline stage metrics (rate, latency, backlog). Alert when rate drops or latency/backlog exceeds threshold. Report on throughput by stage and trend. Correlate with gateway and cloud health.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Pipeline metrics, message broker logs.
• Ensure the following data sources are available: Ingestion rate, end-to-end latency, backlog size.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest pipeline stage metrics (rate, latency, backlog). Alert when rate drops or latency/backlog exceeds threshold. Report on throughput by stage and trend. Correlate with gateway and cloud health.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iot sourcetype="pipeline:metrics"
| bin _time span=5m
| stats avg(ingestion_rate) as rate, avg(latency_ms) as latency, max(backlog) as backlog by pipeline_stage, _time
| where rate < 100 OR latency > 5000 OR backlog > 50000
| table pipeline_stage, rate, latency, backlog
```
