# UC-14.5.5 · Edge Hub to Cloud HEC Forwarding Backlog

## Description

When Edge Hub loses connectivity to Splunk, data backs up locally (e.g. 3M sensor points in SQLite). Backlog growth and drain rate indicate risk of data loss and recovery time.

## Value

When Edge Hub loses connectivity to Splunk, data backs up locally (e.g. 3M sensor points in SQLite). Backlog growth and drain rate indicate risk of data loss and recovery time.

## Implementation

Edge Hub reports backlog and HEC connection state to edge-hub-health. Ingest and alert when backlog exceeds threshold (e.g. 10K events) or connected=0. Track backlog drain rate after reconnect to estimate catch-up time.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Edge Hub (system health).
• Ensure the following data sources are available: `index=edge-hub-health sourcetype=edge_hub` (backlog, queue depth).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Edge Hub reports backlog and HEC connection state to edge-hub-health. Ingest and alert when backlog exceeds threshold (e.g. 10K events) or connected=0. Track backlog drain rate after reconnect to estimate catch-up time.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=edge-hub-health sourcetype=edge_hub
| stats latest(backlog_count) as backlog, latest(hec_connected) as connected by host
| where backlog > 10000 OR connected != 1
| table host backlog connected _time
```
