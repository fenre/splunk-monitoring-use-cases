# UC-14.2.13 · Safety Instrumented System Monitoring

## Description

SIS trips, bypasses, and diagnostics demand immediate visibility and audit trails.

## Value

SIS trips, bypasses, and diagnostics demand immediate visibility and audit trails.

## Implementation

Segregate SIS data per safety policy. Never route writes from IT networks. Alert on any bypass or fault.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: SIS vendor DCS export, OPC-UA alarms.
• Ensure the following data sources are available: `sourcetype="sis:event"` (trip, bypass, fault).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Segregate SIS data per safety policy. Never route writes from IT networks. Alert on any bypass or fault.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="sis:event"
| where event_type IN ("trip","bypass","fault","demand")
| table _time, sis_tag, event_type, cause, sil_level
| sort -_time
```
