# UC-14.2.10 · ICS Protocol Violation Alerts

## Description

Malformed DNP3/Modbus/Profinet frames or wrong L4 ports indicate misconfiguration or attacks.

## Value

Malformed DNP3/Modbus/Profinet frames or wrong L4 ports indicate misconfiguration or attacks.

## Implementation

Normalize IDS fields into Splunk. Tune for false positives on legacy equipment. Route critical to SOC and OT jointly.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Industrial IDS, Zeek ICS parsers.
• Ensure the following data sources are available: `sourcetype="ics:protocol"` IDS alerts.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize IDS fields into Splunk. Tune for false positives on legacy equipment. Route critical to SOC and OT jointly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="ics:protocol"
| where severity IN ("high","critical") OR match(message,"(?i)(malformed|illegal|out.of.range)")
| stats count by protocol, src, dest, signature
| sort -count
```

Understanding this SPL
