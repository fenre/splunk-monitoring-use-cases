# UC-14.7.4 · Production Sensor Data Completeness Audit

## Description

Stale or missing sensor readings invalidate safety and quality analytics; completeness audits align telemetry coverage with regulatory expectations for critical measurements.

## Value

Stale or missing sensor readings invalidate safety and quality analytics; completeness audits align telemetry coverage with regulatory expectations for critical measurements.

## Implementation

Set stale_threshold per tag class. Optionally join to a required-tag lookup and alert when any required tag is absent entirely.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Litmus Edge (Splunk HEC connector).
• Ensure the following data sources are available: `index=ot` `sourcetype="litmus:tag"` fields `gateway_id`, `tag_name`, optional `quality`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Set stale_threshold per tag class. Optionally join to a required-tag lookup and alert when any required tag is absent entirely.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="litmus:tag"
| stats latest(_time) as last_seen by gateway_id, tag_name
| eval stale_sec=now()-last_seen
| where stale_sec > 300
| eval stale_minutes=round(stale_sec/60,1)
| table gateway_id, tag_name, last_seen, stale_minutes
| sort - stale_minutes
```
