# UC-14.4.19 · BLE/Zigbee Gateway Health

## Description

Mesh coordinators and gateways aggregate low-power sensors; their health affects entire buildings.

## Value

Mesh coordinators and gateways aggregate low-power sensors; their health affects entire buildings.

## Implementation

Poll hub API for mesh depth, neighbor loss, and MQTT uplink. Alert on coordinator down or offline child count spike.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Zigbee2MQTT, Home Assistant, vendor hub.
• Ensure the following data sources are available: `sourcetype="zigbee:gateway"` OR `sourcetype="ble:gateway"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll hub API for mesh depth, neighbor loss, and MQTT uplink. Alert on coordinator down or offline child count spike.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iot sourcetype IN ("zigbee:gateway","ble:gateway")
| where coordinator_status!="OK" OR offline_devices > 5 OR mqtt_connected=0
| table _time, gateway_id, coordinator_status, offline_devices, mqtt_connected
```
