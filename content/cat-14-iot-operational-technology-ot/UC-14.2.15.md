# UC-14.2.15 · Control Loop Deviation

## Description

PV diverging from SP with saturated output indicates tuning issues or actuator faults.

## Value

PV diverging from SP with saturated output indicates tuning issues or actuator faults.

## Implementation

Define deadband per loop. Alert when sustained error exceeds threshold or output pegs. Integrate with maintenance CMMS.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OPC-UA process tags.
• Ensure the following data sources are available: `sourcetype="opcua:control"` (pv, sp, out).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Define deadband per loop. Alert when sustained error exceeds threshold or output pegs. Integrate with maintenance CMMS.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="opcua:control"
| eval err=abs(pv-sp)
| where err > deadband*5 OR (abs(out)>95 AND err>deadband)
| timechart span=1m avg(err) by loop_id
```
