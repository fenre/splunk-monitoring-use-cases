# UC-14.4.17 · Edge-to-Cloud Sync Failures

## Description

Store-and-forward gaps mean stale dashboards and missed alerts for remote sites.

## Value

Store-and-forward gaps mean stale dashboards and missed alerts for remote sites.

## Implementation

Parse sync success, backoff, and queue depth. Alert on failure or growing backlog. Correlate with WAN outages.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Edge agent logs (Azure IoT Edge, Greengrass).
• Ensure the following data sources are available: `sourcetype="edge:sync"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Parse sync success, backoff, and queue depth. Alert on failure or growing backlog. Correlate with WAN outages.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iot sourcetype="edge:sync"
| where status="failed" OR backlog_mb > 100 OR last_success_age_sec > 600
| stats latest(backlog_mb) as backlog, latest(status) as st by edge_id, cloud_endpoint
| sort -backlog
```
