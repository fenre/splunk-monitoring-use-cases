# UC-14.5.6 · MQTT Retain and Last Will Message Audit

## Description

Retained messages and Last Will payloads can contain sensitive state. Auditing who published retain/LWT and when supports change control and security review.

## Value

Retained messages and Last Will payloads can contain sensitive state. Auditing who published retain/LWT and when supports change control and security review.

## Implementation

Enable broker audit or access logging for MQTT publish (include retain flag and will payload if logged). Forward to Splunk. Alert on new retain on sensitive topics or LWT changes. Dashboard retain/LWT events by client and topic.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: MQTT broker (access log, audit log).
• Ensure the following data sources are available: Broker audit log (publish with retain=1, will payload).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Enable broker audit or access logging for MQTT publish (include retain flag and will payload if logged). Forward to Splunk. Alert on new retain on sensitive topics or LWT changes. Dashboard retain/LWT events by client and topic.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype=mqtt_broker_audit (retain=1 OR "last_will" OR "will_message")
| bin _time span=1h
| stats count by client_id, topic, action, _time
| where count > 0
| table _time client_id topic action count
```
