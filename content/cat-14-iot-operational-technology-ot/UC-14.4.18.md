# UC-14.4.18 · IoT Device Provisioning Audit

## Description

Unauthorized provisioning events create shadow devices and billing abuse.

## Value

Unauthorized provisioning events create shadow devices and billing abuse.

## Implementation

Compare provisioning rate to approved baseline. Alert on new template or IAM principal. Cross-check with HR for contractor access. For AWS, prefer CloudTrail (`sourcetype="aws:cloudtrail"`) with `eventSource="iot.amazonaws.com"` (and relevant `eventName` values) instead of a non-standard `cloudtrail:iot` sourcetype; keep `iot:provisioning` only if that is your normalized pipeline.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=audit sourcetype="iot:provisioning"
| where action IN ("RegisterThing","CreateCertificate","AttachPolicy")
| stats count by actor, device_template, src
| where count > 50
| sort -count
```
