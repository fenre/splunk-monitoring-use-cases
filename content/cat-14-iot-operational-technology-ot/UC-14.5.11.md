# UC-14.5.11 · MQTT Authentication Failure and ACL Denials

## Description

Failed MQTT logins and publish/subscribe denials indicate credential abuse, misconfiguration, or attack. Monitoring supports access control and incident response.

## Value

Failed MQTT logins and publish/subscribe denials indicate credential abuse, misconfiguration, or attack. Monitoring supports access control and incident response.

## Implementation

Enable broker authentication and ACL logging. Forward to Splunk. Alert when failure count from a single client or IP exceeds threshold. Dashboard failures by client and topic.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: MQTT broker (auth and ACL logs).
• Ensure the following data sources are available: Broker access log (auth failure, ACL deny).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Enable broker authentication and ACL logging. Forward to Splunk. Alert when failure count from a single client or IP exceeds threshold. Dashboard failures by client and topic.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype=mqtt_broker_log ("auth failed" OR "ACL deny" OR "access denied" OR "unauthorized")
| bin _time span=15m
| stats count by client_id, src, reason, _time
| where count > 5
| sort -count
```
