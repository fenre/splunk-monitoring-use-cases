# UC-14.6.3 · S7comm CPU State Change Detection

## Description

CPU stop/start or mode transitions can halt a line or leave a PLC in an unsafe state; detecting them from the wire supports both troubleshooting and detection of malicious or accidental operational disruption.

## Value

CPU stop/start or mode transitions can halt a line or leave a PLC in an unsafe state; detecting them from the wire supports both troubleshooting and detection of malicious or accidental operational disruption.

## Implementation

Place Zeek on taps facing S7 controllers and HMIs; normalize `subfunction_name`/`rosctr_name` strings from production captures. Alert on stop/start patterns outside approved maintenance; correlate with MES/SCADA alarms for the same asset.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: TA for Zeek (Splunkbase 5466), Corelight App for Splunk (Splunkbase 3884).
• Ensure the following data sources are available: `index=ot` `sourcetype="zeek:s7comm:json"` (`rosctr_name`, `subfunction_name`, `function_name`, `error_class`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Place Zeek on taps facing S7 controllers and HMIs; normalize `subfunction_name`/`rosctr_name` strings from production captures. Alert on stop/start patterns outside approved maintenance; correlate with MES/SCADA alarms for the same asset.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="zeek:s7comm:json"
| where match(subfunction_name,"(?i)STOP|START|RESUME") OR match(rosctr_name,"(?i)STOP|RUN|HOLD") OR match(function_name,"(?i)PLC|mode|cpu")
| stats count by source_h destination_h rosctr_name subfunction_name function_name
| sort - count
| head 100
```
