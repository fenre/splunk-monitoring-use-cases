# UC-19.1.35 · IOM / FEX to FI Link Flap Events

## Description

Repeated link flaps between IOM/FEX and FI cause micro-outages for blade traffic and complicate troubleshooting with intermittent CRCs. Counting flaps per uplink highlights bad optics or mis-seated cables before full path loss.

## Value

Repeated link flaps between IOM/FEX and FI cause micro-outages for blade traffic and complicate troubleshooting with intermittent CRCs. Counting flaps per uplink highlights bad optics or mis-seated cables before full path loss.

## Implementation

(1) Forward UCSM FI and FEX syslog with UTC timestamps; (2) tune keywords for your transceiver vendor messages; (3) alert on sustained flap count and create cable/optics work order.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_ucs sourcetype="cisco:ucs:syslog" earliest=-24h
| search (link DOWN OR "link down" OR "LOS" OR "SFP" OR flap) AND (iom OR fex OR "eth uplink")
| stats count as flap_events by chassis_id, port, severity
| where flap_events>=5
| sort -flap_events
```
