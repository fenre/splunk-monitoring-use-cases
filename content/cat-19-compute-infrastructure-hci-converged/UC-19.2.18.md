# UC-19.2.18 · Cluster Expansion Events

## Description

Audits node add/remove, disk group expand, and maintenance mode during cluster scale events.

## Value

Audits node add/remove, disk group expand, and maintenance mode during cluster scale events.

## Implementation

Normalize messages from Prism and vCenter. Correlate with change tickets. Alert on unplanned expansion.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `TA-nutanix`, vCenter events.
• Ensure the following data sources are available: `hci:cluster_events`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize messages from Prism and vCenter. Correlate with change tickets. Alert on unplanned expansion.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="hci:cluster_events" earliest=-30d
| search "node added" OR "add node" OR "remove node" OR "expand"
| table _time, cluster, user, action, details
| sort -_time
```
