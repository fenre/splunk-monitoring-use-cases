# UC-19.1.29 · Blade Server ECC Memory Error Rate (Cisco UCS)

## Description

Correctable ECC errors often precede uncorrectable failures and guest crashes. Trending per-blade memory error rates lets you schedule DIMM replacement during maintenance windows instead of reacting to sudden hardware loss.

## Value

Correctable ECC errors often precede uncorrectable failures and guest crashes. Trending per-blade memory error rates lets you schedule DIMM replacement during maintenance windows instead of reacting to sudden hardware loss.

## Implementation

(1) Ensure UCS Manager faults or CIMC memory events forward to Splunk with consistent `dn` naming; (2) baseline normal correctable rates per chassis; (3) alert when per-blade counts exceed baseline or severity is major/critical.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-ucs`, UCS Manager faults and SEL.
• Ensure the following data sources are available: `index=cisco_ucs` `sourcetype="cisco:ucs:faults"` with fields `dn`, `cause`, `code`, `severity`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ensure UCS Manager faults or CIMC memory events forward to Splunk with consistent `dn` naming; (2) baseline normal correctable rates per chassis; (3) alert when per-blade counts exceed baseline or severity is major/critical.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_ucs sourcetype="cisco:ucs:faults" earliest=-24h
| search dn="sys/chassis-*/blade-*" AND (like(lower(cause),"%ecc%") OR like(lower(cause),"%memory%") OR like(lower(cause),"%dimm%"))
| stats count as fault_events by dn, code, severity
| sort -fault_events
```
