# UC-19.1.21 · Intersight HCL Compliance Status

## Description

Hardware Compatibility List compliance ensures OS, driver, and firmware combinations are Cisco-validated. Non-HCL configurations risk unpredictable failures and void support entitlements.

## Value

Hardware Compatibility List compliance ensures OS, driver, and firmware combinations are Cisco-validated. Non-HCL configurations risk unpredictable failures and void support entitlements.

## Implementation

Poll Intersight HCL status via the add-on. Alert when servers move out of Validated status. Correlate with planned OS or driver upgrades.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Cisco Intersight Add-on`.
• Ensure the following data sources are available: `cisco:intersight:compute` (HCL status fields).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll Intersight HCL status via the add-on. Alert when servers move out of Validated status. Correlate with planned OS or driver upgrades.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_intersight sourcetype="cisco:intersight:compute" object_type="cond.HclStatus"
| stats latest(status) as hcl_status latest(reason) as reason by server_name, model
| where hcl_status!="Validated"
| table server_name, model, hcl_status, reason
```
