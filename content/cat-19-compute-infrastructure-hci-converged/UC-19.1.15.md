# UC-19.1.15 · Chassis PSU Redundancy

## Description

Detects loss of N+1 PSU redundancy in chassis before full power loss.

## Value

Detects loss of N+1 PSU redundancy in chassis before full power loss.

## Implementation

Map PSU fields from API. Page on non-redundant state.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-ucs`.
• Ensure the following data sources are available: `cisco:ucs:environmental`, PSU inventory.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map PSU fields from API. Page on non-redundant state.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_ucs sourcetype="cisco:ucs:environmental" metric_type="psu"
| where oper_state!="ok" OR redundancy_state!="redundant"
| stats count by chassis_id, psu_slot, oper_state, redundancy_state
```
