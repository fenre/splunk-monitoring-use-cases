# UC-19.2.27 · vSAN Disk Group Capacity Headroom and Mount State

## Description

Disk groups nearing full capacity slow resyncs and increase rebuild time, which extends windows of reduced fault tolerance. Monitoring per–disk group free space and mount state prevents surprise admission control during failures.

## Value

Disk groups nearing full capacity slow resyncs and increase rebuild time, which extends windows of reduced fault tolerance. Monitoring per–disk group free space and mount state prevents surprise admission control during failures.

## Implementation

(1) Ingest vSAN disk group metrics from RVC or vSAN SDK exporter; (2) warn at 80% used and critical at 90%; (3) page on any disk group not mounted; (4) correlate with physical disk SMART (UC-19.2.5).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: VMware vSAN health via vCenter TA or custom API collector.
• Ensure the following data sources are available: `index=hci` `sourcetype="vsan:diskgroup"` with fields `cluster`, `host`, `dg_name`, `used_pct`, `state`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="vsan:diskgroup" earliest=-1h
| stats latest(used_pct) as used_pct, latest(state) as st by cluster, host, dg_name
| where used_pct>80 OR st!="mounted"
| sort -used_pct
| table cluster, host, dg_name, used_pct, st
```
