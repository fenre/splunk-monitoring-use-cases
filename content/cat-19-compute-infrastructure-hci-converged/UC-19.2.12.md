# UC-19.2.12 · HCI Prism Central and Management Plane Health

## Description

Prism Central (PC) failure affects visibility and automation. Monitoring PC and management plane ensures operations and alerting remain functional.

## Value

Prism Central (PC) failure affects visibility and automation. Monitoring PC and management plane ensures operations and alerting remain functional.

## Implementation

Poll Prism Central health and API metrics. Alert on unhealthy status, high API latency, or backed-up task queue. Report on PC availability and performance trend. Maintain HA for PC where available.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Prism Central API, management node metrics.
• Ensure the following data sources are available: PC service status, API latency, task queue depth.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll Prism Central health and API metrics. Alert on unhealthy status, high API latency, or backed-up task queue. Report on PC availability and performance trend. Maintain HA for PC where available.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="nutanix:prism_central"
| stats latest(status) as status, latest(api_latency_ms) as latency, latest(task_queue) as queue by pc_instance
| where status!="healthy" OR latency > 5000 OR queue > 100
| table pc_instance, status, latency, queue
```
