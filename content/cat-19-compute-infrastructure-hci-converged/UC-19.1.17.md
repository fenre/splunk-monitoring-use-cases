# UC-19.1.17 · BIOS Policy Compliance

## Description

Verifies server BIOS settings match service profile BIOS policy (VT-x, power management, boot mode).

## Value

Verifies server BIOS settings match service profile BIOS policy (VT-x, power management, boot mode).

## Implementation

Extract BIOS tokens from inventory poll. Reconcile with expected policy per SP.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-ucs`.
• Ensure the following data sources are available: `cisco:ucs:bios`, service profile.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Extract BIOS tokens from inventory poll. Reconcile with expected policy per SP.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_ucs sourcetype="cisco:ucs:bios"
| lookup ucs_bios_policy.csv sp_name OUTPUT require_vt, require_boot_mode
| where vt_enabled!=require_vt OR boot_mode!=require_boot_mode
| table server_dn, sp_name, vt_enabled, boot_mode, require_vt, require_boot_mode
```
