# UC-19.3.5 · Windows Admin Center Connection and Gateway Audit Events

## Description

Windows Admin Center is often the break-glass console for HCI operations. Auditing gateway authentication failures and session spikes detects credential attacks or misconfigured smart-card rules before admins lose access during incidents.

## Value

Windows Admin Center is often the break-glass console for HCI operations. Auditing gateway authentication failures and session spikes detects credential attacks or misconfigured smart-card rules before admins lose access during incidents.

## Implementation

(1) Populate `wac_gateway_hosts` lookup with gateway computer names; (2) tune out known scanner subnets with `src_ip` exclusions; (3) alert on repeated 4625 failures to RDP/WinRM ports; (4) correlate with VPN and conditional access logs for investigations.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows, HEC JSON from WAC gateway.
• Ensure the following data sources are available: `index=wineventlog` `sourcetype="WinEventLog:Microsoft-Windows-Security-Auditing"` with fields `host`, `EventCode`, `user`, `src_ip`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate `wac_gateway_hosts` lookup with gateway computer names; (2) tune out known scanner subnets with `src_ip` exclusions; (3) alert on repeated 4625 failures to RDP/WinRM ports; (4) correlate with VPN and conditional access logs for investigations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wineventlog sourcetype="WinEventLog:Microsoft-Windows-Security-Auditing" earliest=-24h EventCode=4625 Logon_Type IN (3,10)
| lookup wac_gateway_hosts host OUTPUT is_gateway
| where is_gateway="true" OR like(host,"%wac-gw%")
| stats count as failed_logons by host, src_ip, user
| where failed_logons>=8
| sort -failed_logons
```
