# UC-19.2.22 · HPE dHCI Tier Health

## Description

HPE disaggregated HCI storage tier latency, capacity, and replication lag between compute and storage.

## Value

HPE disaggregated HCI storage tier latency, capacity, and replication lag between compute and storage.

## Implementation

Map vendor tier IDs. Alert on latency or replication lag SLO breach.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HPE OneView, dHCI metrics.
• Ensure the following data sources are available: `hpe:dhci:tier`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map vendor tier IDs. Alert on latency or replication lag SLO breach.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="hpe:dhci:tier" earliest=-4h
| where tier_latency_ms>5 OR capacity_pct>85 OR repl_lag_sec>30
| stats max(tier_latency_ms) as lat, max(repl_lag_sec) as lag by cluster, tier_name
| sort -lat
```
