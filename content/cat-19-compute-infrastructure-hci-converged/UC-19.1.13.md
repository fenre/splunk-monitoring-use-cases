# UC-19.1.13 · FI Port Channel Member Errors and CRCs

## Description

Per-member link errors and CRCs on FI port-channels — augments aggregate PC status (UC-19.1.5).

## Value

Per-member link errors and CRCs on FI port-channels — augments aggregate PC status (UC-19.1.5).

## Implementation

Ingest per-member counters. Alert on CRC growth or member down inside operational PC.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-ucs`.
• Ensure the following data sources are available: `cisco:ucs:fi_stats` port members.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest per-member counters. Alert on CRC growth or member down inside operational PC.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_ucs sourcetype="cisco:ucs:fi_stats" object_type="port"
| where crc_errors>0 OR link_state!="up"
| stats sum(crc_errors) as crc by fi_id, pc_id, port_id
| where crc>0
| sort -crc
```
