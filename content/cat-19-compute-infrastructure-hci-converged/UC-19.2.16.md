# UC-19.2.16 · HCI Node Failure Domain Risk

## Description

Validates RF/FTM rules so critical VM replicas don’t share the same fault domain (rack/block) — risk scoring when domains are imbalanced.

## Value

Validates RF/FTM rules so critical VM replicas don’t share the same fault domain (rack/block) — risk scoring when domains are imbalanced.

## Implementation

Build fault domain from rack metadata. Flag Tier-0 VMs without FD separation. Use for BCP testing.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Nutanix Prism, vSAN stretched cluster APIs.
• Ensure the following data sources are available: `hci:fd`, host-to-disk mapping.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Build fault domain from rack metadata. Flag Tier-0 VMs without FD separation. Use for BCP testing.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="hci:fault_domain" earliest=-7d
| stats dc(node) as nodes_in_fd by cluster, fault_domain_name
| where nodes_in_fd>3
| lookup hci_fd_risk.csv cluster fault_domain_name OUTPUT risk_score
| where risk_score>70
| table cluster, fault_domain_name, nodes_in_fd, risk_score
```
