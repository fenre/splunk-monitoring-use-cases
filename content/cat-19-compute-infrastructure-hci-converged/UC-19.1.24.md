# UC-19.1.24 · Intersight Contract and Warranty Compliance

## Description

Monitoring support contract and warranty expiration across the compute fleet prevents coverage gaps that delay RMA and increase risk during hardware failures.

## Value

Monitoring support contract and warranty expiration across the compute fleet prevents coverage gaps that delay RMA and increase risk during hardware failures.

## Implementation

Poll contract information weekly. Alert at 90, 60, and 30 day thresholds. Generate a monthly report for procurement.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Cisco Intersight Add-on`.
• Ensure the following data sources are available: `cisco:intersight:contracts`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll contract information weekly. Alert at 90, 60, and 30 day thresholds. Generate a monthly report for procurement.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_intersight sourcetype="cisco:intersight:contracts"
| eval days_to_expiry=round((strptime(contract_end_date,"%Y-%m-%dT%H:%M:%S")-now())/86400)
| where days_to_expiry < 90 OR contract_status!="Active"
| table server_name, serial, contract_status, contract_end_date, days_to_expiry
| sort days_to_expiry
```
