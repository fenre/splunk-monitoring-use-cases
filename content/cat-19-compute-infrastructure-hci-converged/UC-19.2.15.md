# UC-19.2.15 · Storage Pool Rebalance Monitoring

## Description

Tracks Curator/Planned Outage rebalance and disk usage skew during rebalance operations on Nutanix/vSAN.

## Value

Tracks Curator/Planned Outage rebalance and disk usage skew during rebalance operations on Nutanix/vSAN.

## Implementation

Map Curator task fields. Alert when rebalance stalls or ETA exceeds policy. Report impact on I/O (UC-19.2.3).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Nutanix Prism, vSAN health.
• Ensure the following data sources are available: `nutanix:curator`, `vsan:rebalance`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map Curator task fields. Alert when rebalance stalls or ETA exceeds policy. Report impact on I/O (UC-19.2.3).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="nutanix:curator" earliest=-24h
| where status="running" OR rebalance_pct>0
| stats latest(rebalance_pct) as pct, latest(eta_min) as eta by cluster, task_id
| table cluster, task_id, pct, eta
```
