# UC-19.2.23 · vSAN Witness Appliance Health

## Description

Stretched vSAN witness availability and quorum for split-brain prevention.

## Value

Stretched vSAN witness availability and quorum for split-brain prevention.

## Implementation

Poll witness health API. Page immediately if witness disconnected or quorum lost.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: vCenter, witness VM metrics.
• Ensure the following data sources are available: `vsan:witness`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll witness health API. Page immediately if witness disconnected or quorum lost.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="vsan:witness" earliest=-24h
| where witness_state!="connected" OR quorum!="met"
| stats latest(witness_state) as ws, latest(quorum) as q by cluster
| table cluster, ws, q
```
