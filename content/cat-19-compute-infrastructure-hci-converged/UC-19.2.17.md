# UC-19.2.17 · vSAN Disk Group Health

## Description

Disk group mount state, component health, and checksum errors for VMware vSAN.

## Value

Disk group mount state, component health, and checksum errors for VMware vSAN.

## Implementation

Ingest vSAN health JSON. Page on unhealthy disk group or rising checksums. Correlate with physical disk (UC-19.2.5).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: vSAN health API, vCenter TA.
• Ensure the following data sources are available: `vsan:diskgroup`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest vSAN health JSON. Page on unhealthy disk group or rising checksums. Correlate with physical disk (UC-19.2.5).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="vsan:diskgroup" earliest=-4h
| where state!="healthy" OR checksum_errors>0 OR component_state!="active"
| stats latest(state) as st, sum(checksum_errors) as checksums by cluster, host, dg_name
| sort -checksums
```
