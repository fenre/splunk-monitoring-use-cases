# UC-19.2.28 · Nutanix Storage Pool Erasure Coding vs RF Footprint

## Description

Erasure coding reduces physical footprint but increases rebuild amplification on dense clusters. Tracking EC-enabled containers versus replication factor helps right-size policies and avoid capacity cliffs during failure events.

## Value

Erasure coding reduces physical footprint but increases rebuild amplification on dense clusters. Tracking EC-enabled containers versus replication factor helps right-size policies and avoid capacity cliffs during failure events.

## Implementation

(1) Poll storage pool summary including EC flags; (2) baseline overhead ratio per pool design; (3) alert when overhead spikes vs 30-day median indicating rebuild or mis-tuned EC stripe width.

## Detailed Implementation

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="nutanix:storage_pool" earliest=-4h
| stats latest(ec_enabled) as ec, latest(rf) as rf, latest(used_logical_tb) as log_tb, latest(used_physical_tb) as phys_tb by cluster, storage_pool
| eval overhead_ratio=round(phys_tb/nullif(log_tb,0), 2)
| where ec=="true" OR ec=="1"
| table cluster, storage_pool, rf, log_tb, phys_tb, overhead_ratio
| sort -overhead_ratio
```
