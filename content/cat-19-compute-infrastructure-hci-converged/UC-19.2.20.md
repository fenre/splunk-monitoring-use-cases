# UC-19.2.20 · SimpliVity Backup Efficiency

## Description

Backup job success, dedupe ratio, and store utilization for HPE SimpliVity.

## Value

Backup job success, dedupe ratio, and store utilization for HPE SimpliVity.

## Implementation

Map OmniStack API fields. Alert on failed backup or low dedupe vs baseline.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HPE SimpliVity REST API.
• Ensure the following data sources are available: `simplivity:backup`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map OmniStack API fields. Alert on failed backup or low dedupe vs baseline.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="simplivity:backup" earliest=-7d
| where status!="success" OR dedupe_ratio<3
| stats latest(status) as st, latest(dedupe_ratio) as dr by cluster, policy_name
| table cluster, policy_name, st, dr
```
