# UC-19.2.25 · Nutanix Cluster Health Score and Critical Services

## Description

A single degraded critical service (Stargate, Cassandra, Curator) can silently erode I/O quality before user-visible alerts fire. Tracking cluster health score and service map together shortens time to stabilize the control plane.

## Value

A single degraded critical service (Stargate, Cassandra, Curator) can silently erode I/O quality before user-visible alerts fire. Tracking cluster health score and service map together shortens time to stabilize the control plane.

## Implementation

(1) Ingest Prism cluster health JSON on a 1–5 minute cadence; (2) normalize service status casing; (3) alert when score drops below SLO or any core service is not up; (4) link to Nutanix support bundle collection runbook.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `TA-nutanix`, Prism Element API.
• Ensure the following data sources are available: `index=hci` `sourcetype="nutanix:cluster_health"` with fields `cluster`, `health_score`, `service_name`, `service_status`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest Prism cluster health JSON on a 1–5 minute cadence; (2) normalize service status casing; (3) alert when score drops below SLO or any core service is not up; (4) link to Nutanix support bundle collection runbook.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="nutanix:cluster_health" earliest=-2h
| stats latest(health_score) as score by cluster
| join type=left cluster [
    search index=hci sourcetype="nutanix:cluster_health" earliest=-2h service_name=*
    | where service_status!="UP" AND service_status!="up"
    | stats values(service_name) as bad_services by cluster
  ]
| where score<90 OR isnotnull(bad_services)
| table cluster, score, bad_services
| sort score
```
