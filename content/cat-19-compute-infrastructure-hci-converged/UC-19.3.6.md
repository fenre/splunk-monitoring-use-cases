# UC-19.3.6 · Cluster-Aware Updating Run Status and Node Drain Failures

## Description

Failed CAU runs leave nodes partially patched or stuck outside maintenance mode, which breaks uniform firmware and OS compliance. Tracking per-node CAU stages surfaces stuck updates before the next failure domain event.

## Value

Failed CAU runs leave nodes partially patched or stuck outside maintenance mode, which breaks uniform firmware and OS compliance. Tracking per-node CAU stages surfaces stuck updates before the next failure domain event.

## Implementation

(1) Emit structured JSON from `Get-CauRunHistory` after each CAU wave; (2) alert on failed or rolled-back runs; (3) join with Windows Update EventCode 19/20 success for cross-check; (4) attach remediation KB links in alert payload.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Windows Event Forwarding, PowerShell automation logs.
• Ensure the following data sources are available: `index=azure_stack_hci` `sourcetype="azurestackhci:cau"` with fields `cluster_name`, `run_id`, `node`, `stage`, `status`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=azure_stack_hci sourcetype="azurestackhci:cau" earliest=-7d
| where status!="Succeeded" AND status!="InProgress"
| stats latest(status) as st, latest(stage) as stage, values(node) as nodes by cluster_name, run_id
| table cluster_name, run_id, stage, st, nodes
| sort -run_id
```
