# UC-19.2.10 · HCI Hypervisor and AOS Version Compliance

## Description

Mixed hypervisor or AOS versions can cause compatibility and support issues. Tracking version compliance supports upgrade planning and support eligibility.

## Value

Mixed hypervisor or AOS versions can cause compatibility and support issues. Tracking version compliance supports upgrade planning and support eligibility.

## Implementation

Ingest cluster and node version data. Maintain baseline by environment. Alert on version drift. Report on compliance percentage and nodes due for upgrade.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Nutanix Prism, vSphere/vCenter API.
• Ensure the following data sources are available: Node AOS version, hypervisor version, compliance baseline.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest cluster and node version data. Maintain baseline by environment. Alert on version drift. Report on compliance percentage and nodes due for upgrade.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hci sourcetype="nutanix:cluster"
| stats latest(aos_version) as aos, latest(hypervisor_version) as hv by node
| lookup hci_compliance_baseline.csv env OUTPUT target_aos, target_hv
| where aos!=target_aos OR hv!=target_hv
| table node, aos, target_aos, hv, target_hv
```
