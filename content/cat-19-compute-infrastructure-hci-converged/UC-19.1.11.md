# UC-19.1.11 · Service Profile Association Failures (Cisco UCS)

## Description

Failed or stuck service profile associations block server bring-up and maintenance — complements compliance state (UC-19.1.2).

## Value

Failed or stuck service profile associations block server bring-up and maintenance — complements compliance state (UC-19.1.2).

## Implementation

Tune search to UCS fault codes for association. Alert on any failed association in production orgs.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_cisco-ucs`.
• Ensure the following data sources are available: `cisco:ucs:config`, UCS faults.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Tune search to UCS fault codes for association. Alert on any failed association in production orgs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_ucs (sourcetype="cisco:ucs:config" OR sourcetype="cisco:ucs:faults")
| search assoc_state="failed" OR match(lower(descr),"(?i)association.*fail")
| stats count by sp_name, server_dn, descr
| sort -count
```
