# UC-11.2.13 · Groups Membership Changes

## Description

Sensitive Google Groups (all-staff, external partners) membership changes can broadly expose mail and Drive.

## Value

Sensitive Google Groups (all-staff, external partners) membership changes can broadly expose mail and Drive.

## Implementation

Tag high-impact groups in lookup. Alert on adds to groups with external posting or shared drives.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_GoogleWorkspace`.
• Ensure the following data sources are available: Groups audit (`ADD_GROUP_MEMBER`, `REMOVE_GROUP_MEMBER`, `CREATE_GROUP`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Tag high-impact groups in lookup. Alert on adds to groups with external posting or shared drives.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=gws sourcetype="gws:groups" OR (sourcetype="gws:admin" event_name="GROUP_*")
| search event_name IN ("ADD_GROUP_MEMBER","REMOVE_GROUP_MEMBER","CREATE_GROUP")
| lookup sensitive_groups.csv group_email OUTPUT tier
| where tier="high" OR like(group_email,"*external*")
| table _time, actor.email, event_name, group_email, member_email
```
