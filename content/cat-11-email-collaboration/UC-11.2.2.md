# UC-11.2.2 · Gmail Message Flow

## Description

Email delivery monitoring and DLP enforcement protects sensitive data and ensures business communication reliability.

## Value

Email delivery monitoring and DLP enforcement protects sensitive data and ensures business communication reliability.

## Implementation

Ingest Gmail logs. Track message delivery rates, spam filtering effectiveness, and DLP triggers. Alert on delivery failures or increased spam rates. Report on email security posture.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_GoogleWorkspace`.
• Ensure the following data sources are available: Gmail logs via BigQuery export or Reports API.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest Gmail logs. Track message delivery rates, spam filtering effectiveness, and DLP triggers. Alert on delivery failures or increased spam rates. Report on email security posture.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=gws sourcetype="gws:gmail"
| stats count by message_info.disposition
| eval pct=round(count/sum(count)*100,1)
```
