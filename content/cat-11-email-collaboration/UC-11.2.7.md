# UC-11.2.7 · Drive External Sharing Alerts

## Description

Alerts on files and folders shared outside the primary domain or with `anyone with link`—complements anomaly baselines (UC-11.2.3).

## Value

Alerts on files and folders shared outside the primary domain or with `anyone with link`—complements anomaly baselines (UC-11.2.3).

## Implementation

Tune domain pattern via lookup. Alert on shares to competitor domains or public links on confidential folders (path regex).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_GoogleWorkspace`.
• Ensure the following data sources are available: Drive audit log (`change_user_access`, `shared_drive_settings_change`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Tune domain pattern via lookup. Alert on shares to competitor domains or public links on confidential folders (path regex).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=gws sourcetype="gws:drive" event_name="change_user_access"
| where new_value="people_with_link" OR NOT match(target_user_email, ".*@yourdomain\\.com")
| table _time, actor.email, doc_title, target_user_email, new_value
| sort -_time
```
