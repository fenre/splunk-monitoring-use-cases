# UC-11.2.14 · Cloud Identity Device Management

## Description

Endpoint visibility for mobile and desktop enrolled in Endpoint Verification / MDM—lost devices and OS drift.

## Value

Endpoint visibility for mobile and desktop enrolled in Endpoint Verification / MDM—lost devices and OS drift.

## Implementation

Ingest device inventory daily. Alert on lost/stolen, rooted/jailbroken, or encryption-off. Integrate with Chrome management (UC-11.2.10).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_GoogleWorkspace`.
• Ensure the following data sources are available: Endpoint audit (`REGISTER_DEVICE`, `SYNC_DEVICE`), Reports API mobile devices.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest device inventory daily. Alert on lost/stolen, rooted/jailbroken, or encryption-off. Integrate with Chrome management (UC-11.2.10).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=gws sourcetype="gws:endpoint" OR sourcetype="gws:mobile"
| where compliance_state!="COMPLIANT" OR device_compromised="true" OR status="LOST"
| stats count by device_id, user_email, compliance_state
| sort -count
```
