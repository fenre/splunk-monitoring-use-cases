# UC-11.4.5 · Mail Delivery Rate and Bounce Rate by Domain

## Description

Sudden drop in delivery rate or spike in bounces for a domain indicates reputation or configuration issues. Trending supports deliverability and capacity planning.

## Value

Sudden drop in delivery rate or spike in bounces for a domain indicates reputation or configuration issues. Trending supports deliverability and capacity planning.

## Implementation

Parse delivery and bounce events by recipient domain. Compute hourly delivery and bounce rate. Alert when bounce rate exceeds 5% or delivery volume drops significantly for critical domains.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Mail server logs, bounce logs.
• Ensure the following data sources are available: Postfix/Sendmail delivery status, bounce messages, Exchange tracking logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Parse delivery and bounce events by recipient domain. Compute hourly delivery and bounce rate. Alert when bounce rate exceeds 5% or delivery volume drops significantly for critical domains.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mail sourcetype=mail_delivery
| bin _time span=1h
| stats count(eval(status="delivered")) as delivered, count(eval(status="bounce")) as bounces by domain, _time
| eval bounce_rate=round(bounces/(delivered+bounces)*100, 2)
| where bounce_rate > 5 OR delivered < 10
| table domain delivered bounces bounce_rate
```
