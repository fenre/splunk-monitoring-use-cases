# UC-11.4.6 · Outbound Mail Volume and Recipient Anomaly

## Description

Unusual outbound volume or new bulk recipients may indicate compromised account or phishing campaign. Baseline and anomaly detection support incident response.

## Value

Unusual outbound volume or new bulk recipients may indicate compromised account or phishing campaign. Baseline and anomaly detection support incident response.

## Implementation

Ingest outbound send events. Baseline message count and unique recipients per sender (hourly). Alert when volume or recipient count exceeds 3 standard deviations or recipient count >100 in one hour.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Mail server logs.
• Ensure the following data sources are available: Postfix/Sendmail/Exchange outbound logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest outbound send events. Baseline message count and unique recipients per sender (hourly). Alert when volume or recipient count exceeds 3 standard deviations or recipient count >100 in one hour.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mail sourcetype=mail_send
| bin _time span=1h
| stats dc(recipient) as recipients, count as msg_count by sender, _time
| eventstats avg(msg_count) as avg_count, stdev(msg_count) as std_count by sender
| eval z_score=if(std_count>0, (msg_count-avg_count)/std_count, 0)
| where z_score > 3 OR recipients > 100
| table _time sender msg_count recipients z_score
```
