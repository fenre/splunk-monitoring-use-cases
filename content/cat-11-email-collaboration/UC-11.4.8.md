# UC-11.4.8 ¬∑ SMTP Relay Monitoring

## Description

Tracks messages relayed through internal SMTP gateways vs policy ‚Äî unexpected relay volume or open relay abuse paths.

## Value

Tracks messages relayed through internal SMTP gateways vs policy ‚Äî unexpected relay volume or open relay abuse paths.

## Implementation

Parse relay lines for authorized vs denied. Alert on high relay denied from single IP (scanning) or high accepted relay to external domains (misconfiguration).

## Detailed Implementation

Prerequisites
‚Ä¢ Install and configure the required add-on or app: `Splunk_TA_syslog`, Postfix/Exchange logs.
‚Ä¢ Ensure the following data sources are available: `postfix:syslog` `relay=`, `status=sent`, `reject` relay attempts.
‚Ä¢ For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 ‚Äî Configure data collection
Parse relay lines for authorized vs denied. Alert on high relay denied from single IP (scanning) or high accepted relay to external domains (misconfiguration).

Step 2 ‚Äî Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mail sourcetype="postfix:syslog" OR sourcetype=syslog process=postfix
| search relay=* OR "relay access denied"
| stats count by relay_domain, action, src
| where count > 500
```
