# UC-11.2.9 · Gmail Suspicious Forwarding

## Description

Detects auto-forwarding and delegation to external addresses—common post-compromise behavior.

## Value

Detects auto-forwarding and delegation to external addresses—common post-compromise behavior.

## Implementation

Ingest email settings changes. Alert on new forwarding to non-corporate domains. Weekly report of active forwards from `gmail:user_settings` export.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_GoogleWorkspace`.
• Ensure the following data sources are available: Email settings audit (`CHANGE_EMAIL_SETTING`, Gmail routing / forwarding flags in Reports API export).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest email settings changes. Alert on new forwarding to non-corporate domains. Weekly report of active forwards from `gmail:user_settings` export.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=gws sourcetype="gws:admin" event_name="CHANGE_EMAIL_SETTING"
| search forward OR delegate OR filter
| where NOT match(setting_value, ".*@yourdomain\\.com")
| table _time, actor.email, target_user, setting_name, setting_value
```
