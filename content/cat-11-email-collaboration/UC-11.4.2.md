# UC-11.4.2 · POP3 / IMAP Mail Retrieval Service Availability

## Description

POP3 and IMAP services allow mail clients to retrieve messages. Even when delivery works correctly, a crashed Dovecot or Cyrus daemon prevents users from reading email, appearing as a total mail outage. Nagios `check_pop` and `check_imap` monitor these ports directly; Splunk replicates availability detection through daemon log analysis.

## Value

## Implementation

Forward Dovecot or Cyrus IMAP logs via Universal Forwarder. Dovecot logs login events, failed auth, and daemon lifecycle events continuously during normal operation. Zero events for >10 minutes on a mail host indicates a process crash or service failure. Alert after 2 consecutive empty windows. Cross-correlate with auth failures (could indicate process restart loops). For comprehensive coverage, deploy a scripted TCP probe on ports 143 (IMAP), 993 (IMAPS), 110 (POP3), 995 (POP3S).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mail sourcetype=syslog (process=dovecot OR process=imap OR process=pop3)
| bucket _time span=5m
| stats count as imap_events by host, _time
| where isnull(imap_events) OR imap_events=0
| eval status="IMAP_POP3_DOWN"
| table _time, host, status
```
