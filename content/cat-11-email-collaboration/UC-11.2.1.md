# UC-11.2.1 · Admin Console Audit

## Description

Admin actions in Google Workspace affect all users. Audit trail supports compliance and detects unauthorized changes.

## Value

Admin actions in Google Workspace affect all users. Audit trail supports compliance and detects unauthorized changes.

## Implementation

Configure Google Workspace TA to ingest admin audit logs via Reports API. Track user management, policy changes, and configuration modifications. Alert on sensitive operations (role changes, 2FA disablement).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_GoogleWorkspace`.
• Ensure the following data sources are available: Google Workspace Admin audit log.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Configure Google Workspace TA to ingest admin audit logs via Reports API. Track user management, policy changes, and configuration modifications. Alert on sensitive operations (role changes, 2FA disablement).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=gws sourcetype="gws:admin" event_name IN ("CREATE_USER","DELETE_USER","CHANGE_ADMIN_ROLE")
| table _time, actor.email, event_name, target_user
```
