# UC-11.3.4 · Trunk Utilization (Cisco CUCM)

## Description

Trunk capacity limits cause busy signals and missed calls. Monitoring prevents capacity-related service degradation.

## Value

Trunk capacity limits cause busy signals and missed calls. Monitoring prevents capacity-related service degradation.

## Implementation

Track concurrent calls per trunk group from CDR data. Alert when utilization exceeds 80% of capacity. Monitor for trunk failures and failover events. Report on peak utilization for capacity planning.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: UCM CDR, gateway logs.
• Ensure the following data sources are available: CDR records, gateway/trunk metrics.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track concurrent calls per trunk group from CDR data. Alert when utilization exceeds 80% of capacity. Monitor for trunk failures and failover events. Report on peak utilization for capacity planning.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=voip sourcetype="cisco:ucm:cdr"
| bin _time span=15m
| stats dc(globalCallID_callId) as concurrent_calls by _time, trunk_group
| where concurrent_calls > 20
```
