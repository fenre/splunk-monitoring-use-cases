# UC-11.2.12 · Workspace Marketplace App Review

## Description

New marketplace app installs can expose Drive/Gmail data; tracks installs and OAuth scopes.

## Value

New marketplace app installs can expose Drive/Gmail data; tracks installs and OAuth scopes.

## Implementation

Maintain allowlist of approved apps. Alert on new apps with sensitive scopes. Quarterly access review with app owners.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_GoogleWorkspace`.
• Ensure the following data sources are available: Token audit (`authorize`), admin audit (`ADD_APPLICATION`, `INSTALL_MARKETPLACE_APP`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Maintain allowlist of approved apps. Alert on new apps with sensitive scopes. Quarterly access review with app owners.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=gws sourcetype="gws:token" event_name="authorize"
| stats values(scope) as scopes by app_name, actor.email
| mvexpand scopes limit=500
| search scopes="https://www.googleapis.com/auth/drive" OR scopes="*gmail*"
| sort app_name
```

Understanding this SPL

**Workspace Marketplace App Review** — New marketplace app installs can expose Drive/Gmail data; tracks installs and OAuth scopes.

Documented **Data sources**: Token audit (`authorize`), admin audit (`ADD_APPLICATION`, `INSTALL_MARKETPLACE_APP`). **App/TA** (typical add-on context): `Splunk_TA_GoogleWorkspace`. The SPL below should target the same indexes and sourcetypes you configured for that feed—rename `index=` / `sourcetype=` if your deployment differs.

The first pipeline stage scopes events using **index**: gws; **sourcetype**: gws:token. If that sourcetype is not mentioned in Data sources, double-check parsing or update the documentation to match the feed you actually ingest.

**Pipeline walkthrough**

• Scopes the data: index=gws, sourcetype="gws:token". Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
• `stats` rolls up events into metrics; results are split **by app_name, actor.email** so each row reflects one combination of those dimensions (useful for per-host, per-user, or per-entity comparisons for this use case).
• Expands multivalue fields with `mvexpand` — use `limit=` to cap row explosion.
• Applies an explicit `search` filter to narrow the current result set.
• Orders rows with `sort` — combine with `head`/`tail` for top-N patterns.
