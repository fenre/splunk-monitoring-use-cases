# UC-11.5.5 · Webex Room System Uptime

## Description

Room system availability underpins executive and boardroom SLAs; uptime trending supports hardware refresh and network path decisions.

## Value

Room system availability underpins executive and boardroom SLAs; uptime trending supports hardware refresh and network path decisions.

## Implementation

Derive online state from heartbeat or Control Hub connectivity. Compute rolling uptime per room vs. expected business hours. Alert on devices below 99% weekly uptime or prolonged offline spans.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `ta_cisco_webex_add_on_for_splunk`.
• Ensure the following data sources are available: `sourcetype=webex:device`, device online/offline events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Derive online state from heartbeat or Control Hub connectivity. Compute rolling uptime per room vs. expected business hours. Alert on devices below 99% weekly uptime or prolonged offline spans.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=webex sourcetype="webex:device"
| eval up=if(connection_state="connected",1,0)
| bin _time span=1h
| stats avg(up) as uptime_ratio by _time, device_id
| where uptime_ratio < 0.99
| table _time, device_id, uptime_ratio
```
