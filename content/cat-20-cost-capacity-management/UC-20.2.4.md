# UC-20.2.4 · License Utilization Tracking

## Description

Software licenses represent significant IT spend. Tracking usage vs entitlements identifies under-licensed risks (compliance violations) and over-licensed waste (unnecessary spend). Right-sizing licenses can save 15-30% of software costs.

## Value

Software licenses represent significant IT spend. Tracking usage vs entitlements identifies under-licensed risks (compliance violations) and over-licensed waste (unnecessary spend). Right-sizing licenses can save 15-30% of software costs.

## Implementation

Collect license usage data from license servers (FlexLM, RLM) and vendor APIs. Maintain entitlement records in a lookup table. Track daily peak concurrent usage. Alert at 90% consumption (buy more) and flag <50% utilization (optimize). Generate quarterly true-up reports.

Step 1 — Configure data collection
Collect license usage data from license servers (FlexLM, RLM) and vendor APIs. Maintain entitlement records in a lookup table. Track daily peak concurrent usage. Alert at 90% consumption (buy more) and flag <50% utilization (optimize). Generate quarterly true-up reports.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=licenses sourcetype="license:usage"
| stats latest(used_licenses) as used, latest(total_licenses) as total by product, vendor, license_type
| eval utilization_pct=round((used/total)*100, 1)
| eval status=case(utilization_pct>=95, "At Risk", utilization_pct>=80, "High Use", utilization_pct<50, "Underutilized", 1==1, "Healthy")
| sort -utilization_pct
| table product, vendor, license_type, used, total, utilization_pct, status
```
