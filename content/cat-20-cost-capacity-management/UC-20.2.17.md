# UC-20.2.17 · Storage Capacity Forecast by Tier

## Description

Forecast days-to-full per storage tier (flash vs capacity) — extends pool forecasting (UC-20.2.2) with **tier** dimension for procurement.

## Value

Forecast days-to-full per storage tier (flash vs capacity) — extends pool forecasting (UC-20.2.2) with **tier** dimension for procurement.

## Implementation

Map array vendor tiers. Alert when 60d forecast crosses 90% for any tier.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Storage TA, SNMP.
• Ensure the following data sources are available: `storage:capacity` with `tier`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map array vendor tiers. Alert when 60d forecast crosses 90% for any tier.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=storage sourcetype="storage:capacity" earliest=-90d
| timechart span=1d latest(used_pct) as used_pct by storage_system, tier
| predict used_pct algorithm=LLP5 future_timespan=60
```
