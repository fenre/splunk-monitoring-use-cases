# UC-20.3.3 · Subscription Renewal Forecasting

## Description

Forecasting renewal cash-out dates and contract values avoids surprise budget hits and gives procurement time to negotiate or consolidate vendors.

## Value

Forecasting renewal cash-out dates and contract values avoids surprise budget hits and gives procurement time to negotiate or consolidate vendors.

## Implementation

Load subscription end dates and annual amounts from CLM or reseller exports; for AWS/Azure/GCP, tag marketplace subscriptions. Alert 90/60/30 days before renewal. Combine with utilization metrics to decide downgrade options before signing.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Contract repository export (ServiceNow SPM, Ariba), marketplace billing.
• Ensure the following data sources are available: `sourcetype=license:usage`, `sourcetype=aws:billing:cur`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Load subscription end dates and annual amounts from CLM or reseller exports; for AWS/Azure/GCP, tag marketplace subscriptions. Alert 90/60/30 days before renewal. Combine with utilization metrics to decide downgrade options before signing.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
(index=contracts sourcetype="license:usage") OR (index=cloud_billing sourcetype="aws:billing:cur")
| eval renewal_epoch=strptime(renewal_date,"%Y-%m-%d"), amount=tonumber(annual_cost_usd)
| where renewal_epoch > relative_time(now(),"+30d@d") AND renewal_epoch < relative_time(now(),"+365d@d")
| eval days_until=round((renewal_epoch-now())/86400,0)
| stats sum(amount) as renewal_spend by vendor, renewal_date, cost_center
| sort renewal_date
```
