# UC-20.2.22 · Disk IOPS Saturation Trending

## Description

Time-series of `iops_utilization_pct` or latency vs IOP limit for SAN/NVMe — storage performance bottleneck before capacity full.

## Value

Time-series of `iops_utilization_pct` or latency vs IOP limit for SAN/NVMe — storage performance bottleneck before capacity full.

## Implementation

Map vendor IOPS cap. Alert on sustained >80% util or latency SLO breach. Scale pool or move workload.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Storage TA.
• Ensure the following data sources are available: `storage:performance`, array metrics.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Map vendor IOPS cap. Alert on sustained >80% util or latency SLO breach. Scale pool or move workload.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=storage sourcetype="storage:performance" earliest=-7d
| timechart span=1h avg(iops_util_pct) as iops_util avg(read_latency_ms) as lat by volume_id
| where iops_util>80 OR lat>10
```
