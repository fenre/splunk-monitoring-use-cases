# UC-20.1.11 · Spot Instance Interruption Rate

## Description

Interruptions per 1,000 instance-hours by pool and AZ — extends raw event counts (UC-20.1.7) with a **rate** for SLO tracking.

## Value

Interruptions per 1,000 instance-hours by pool and AZ — extends raw event counts (UC-20.1.7) with a **rate** for SLO tracking.

## Implementation

Align instance-hour denominator from CUR `NormalizedUsageAmount`. Alert when `intr_per_1k` exceeds baseline for stateful tiers.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk Add-on for AWS`, CloudTrail.
• Ensure the following data sources are available: `aws:cloudtrail` Spot events, instance-hours from CUR.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Align instance-hour denominator from CUR `NormalizedUsageAmount`. Alert when `intr_per_1k` exceeds baseline for stateful tiers.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=aws sourcetype="aws:cloudtrail" earliest=-30d
| search eventName="SpotInstanceInterruption"
| stats count as intr
| append [ search index=cloud_billing sourcetype="aws:billing:cur" earliest=-30d
  | search lineItem_UsageType="*SpotUsage*"
  | stats sum(lineItem_NormalizedUsageAmount) as instance_hours ]
| stats sum(intr) as interruptions sum(instance_hours) as ih
| eval intr_per_1k=round(1000*interruptions/nullif(ih,0),2)
```
