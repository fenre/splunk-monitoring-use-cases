# UC-20.2.30 · Search and Analytics Cluster Disk Watermark Risk

## Description

OpenSearch and Elasticsearch clusters stop indexing when disk watermarks are breached, breaking log and application pipelines. Tracking used shard store versus cluster capacity predicts when frozen tiers or data nodes must be added.

## Value

OpenSearch and Elasticsearch clusters stop indexing when disk watermarks are breached, breaking log and application pipelines. Tracking used shard store versus cluster capacity predicts when frozen tiers or data nodes must be added.

## Implementation

(1) Poll `_cluster/stats` or Elastic Cloud metrics daily. (2) Align thresholds with `cluster.routing.allocation.disk.watermark` settings. (3) Integrate with storage forecasting (UC-20.2.2) when forecast crosses seventy-five percent within thirty days.

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=observability sourcetype="elastic:cluster_stats" earliest=-30d
| eval used=tonumber(coalesce(store_size_bytes, total_used_bytes))
| eval total=tonumber(coalesce(total_capacity_bytes, disk_total_bytes))
| eval used_pct=round(100*used/nullif(total,0),2)
| bin _time span=1d
| stats latest(used_pct) as used_pct by _time, cluster_name
| predict used_pct as forecast_pct algorithm=LLP5 future_timespan=30
| where 'forecast_pct+30d' > 75
| table cluster_name, used_pct, 'forecast_pct+30d'
```
