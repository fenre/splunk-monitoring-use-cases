# UC-20.2.13 · Cost Forecast vs Budget and Variance Alert

## Description

Forecast over budget risks overspend; large variance indicates model or usage change. Monitoring forecast vs budget supports proactive control and reforecasting.

## Value

Forecast over budget risks overspend; large variance indicates model or usage change. Monitoring forecast vs budget supports proactive control and reforecasting.

## Implementation

Ingest forecast and budget. Compute variance. Alert when forecast exceeds budget by threshold or variance is large. Report on forecast accuracy and budget burn rate. Integrate with finance.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Cost forecasting tool, budget data.
• Ensure the following data sources are available: Monthly forecast, budget, actuals to date.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest forecast and budget. Compute variance. Alert when forecast exceeds budget by threshold or variance is large. Report on forecast accuracy and budget burn rate. Integrate with finance.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cloud_cost sourcetype="cost:forecast"
| stats latest(forecast_total) as forecast, latest(budget) as budget, latest(actual_ytd) as actual by account_id, month
| eval variance_pct=round((forecast-budget)/budget*100, 1)
| where variance_pct > 10 OR variance_pct < -20
| table account_id, month, forecast, budget, actual, variance_pct
```
