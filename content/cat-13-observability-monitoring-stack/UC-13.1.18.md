# UC-13.1.18 ¬∑ Typing Queue Health (_internal)

## Description

The typing queue applies rules to parsed events; backlog here delays field extraction and routing to indexes.

## Value

The typing queue applies rules to parsed events; backlog here delays field extraction and routing to indexes.

## Implementation

Monitor typing queue fill and blocked state. Tune props/transforms if chronic backlog. Correlate with high-cardinality lookups or expensive `EVAL` in transforms.

Step 1 ‚Äî Configure data collection
Monitor typing queue fill and blocked state. Tune props/transforms if chronic backlog. Correlate with high-cardinality lookups or expensive `EVAL` in transforms.

Step 2 ‚Äî Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=splunkd group=queue name=*typing*
| eval fill_pct=if(max_size>0, round(current_size/max_size*100,1), null())
| timechart span=5m max(fill_pct) as max_fill by host
| where max_fill > 65
```
