# UC-13.3.7 · Distributed Trace Sampling and Coverage

## Description

Trace sampling rate and coverage affect observability. Monitoring sampling decisions and trace completeness supports tuning and gap detection.

## Value

Trace sampling rate and coverage affect observability. Monitoring sampling decisions and trace completeness supports tuning and gap detection.

## Implementation

Ingest trace metadata and sampling rates. Alert when sampling drops below target or trace completeness (spans per trace) is low for critical services. Report on coverage by service and env.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: APM/tracing TAs (Jaeger, Tempo, OTLP).
• Ensure the following data sources are available: Trace metadata, sampling flags, span counts per trace.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest trace metadata and sampling rates. Alert when sampling drops below target or trace completeness (spans per trace) is low for critical services. Report on coverage by service and env.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=traces sourcetype="trace:span"
| bin _time span=1h
| stats count as spans, dc(trace_id) as traces, avg(sample_rate) as avg_sample by service, _time
| eval spans_per_trace=spans/traces
| where spans_per_trace < 5 OR avg_sample < 0.01
```
