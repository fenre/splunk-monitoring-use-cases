# UC-13.1.16 · Parsing Queue Health (_internal)

## Description

A saturated parsing queue delays ingestion and increases `_indextime` lag. Tracking fill ratio and blocked state isolates pipeline bottlenecks before data loss.

## Value

A saturated parsing queue delays ingestion and increases `_indextime` lag. Tracking fill ratio and blocked state isolates pipeline bottlenecks before data loss.

## Implementation

Filter `metrics.log` queue metrics for parsing queue names. Alert on sustained fill >70% or `is_blocked`. Correlate with new sourcetypes, regex-heavy props, or indexer CPU.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console.
• Ensure the following data sources are available: `index=_internal` `sourcetype=splunkd` `group=queue` (parsingqueue).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Filter `metrics.log` queue metrics for parsing queue names. Alert on sustained fill >70% or `is_blocked`. Correlate with new sourcetypes, regex-heavy props, or indexer CPU.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=splunkd group=queue name=*parsing*
| eval fill_pct=if(max_size>0, round(current_size/max_size*100,1), null())
| where fill_pct > 70 OR is_blocked=1
| timechart span=5m max(fill_pct) as max_fill by host, name
```
