# UC-13.1.34 ¬∑ Real-Time Search Resource Consumption (_internal)

## Description

Real-time searches reserve cores and memory; runaway RT jobs degrade interactive and scheduled workload.

## Value

Real-time searches reserve cores and memory; runaway RT jobs degrade interactive and scheduled workload.

## Implementation

Baseline RT search counts per SH. Alert on unusual RT concurrency. Identify dashboards or users with many concurrent RT panels.

## Detailed Implementation

Prerequisites
‚Ä¢ Install and configure the required add-on or app: Monitoring Console.
‚Ä¢ Ensure the following data sources are available: `index=_internal` `sourcetype=splunkd` `group=search_concurrency` / `search_process`.
‚Ä¢ For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 ‚Äî Configure data collection
Baseline RT search counts per SH. Alert on unusual RT concurrency. Identify dashboards or users with many concurrent RT panels.

Step 2 ‚Äî Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=splunkd group=search_concurrency
| stats max(active_rt_searches) as rt_active, max(active_hist_searches) as hist by host
| eval rt_ratio=if((rt_active+hist)>0, round(100*rt_active/(rt_active+hist),1), null())
| where rt_active > 20 OR rt_ratio > 40
```
