# UC-13.2.3 · Episode Volume and MTTR

## Description

Episode volume and resolution time measure IT operations effectiveness. Trending drives process improvement.

## Value

Episode volume and resolution time measure IT operations effectiveness. Trending drives process improvement.

## Implementation

Track episode creation, severity distribution, and time-to-resolution. Monitor episode assignment and owner workload. Alert on episode volume spikes. Report on MTTR by severity for management.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI.
• Ensure the following data sources are available: `itsi_grouped_alerts` index.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track episode creation, severity distribution, and time-to-resolution. Monitor episode assignment and owner workload. Alert on episode volume spikes. Report on MTTR by severity for management.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsi_grouped_alerts
| stats count as episodes, avg(duration) as avg_duration_sec by severity
| eval avg_mttr_min=round(avg_duration_sec/60,1)
```

**Pipeline walkthrough**
