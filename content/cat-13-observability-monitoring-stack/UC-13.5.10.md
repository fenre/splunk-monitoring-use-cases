# UC-13.5.10 · Splunk RUM JavaScript Error Rate by Page

## Description

Frontend JavaScript errors — unhandled exceptions, failed API calls, resource loading failures — directly degrade user experience but are invisible to backend monitoring. RUM captures these errors with full stack traces, page URL, browser, and user session context. Tracking JS error rate by page detects regressions after deployments, identifies browser-specific bugs, and quantifies the user impact of frontend failures that backend health checks miss entirely.

## Implementation

Splunk RUM captures JavaScript errors including: uncaught exceptions, unhandled promise rejections, resource loading failures (img, script, CSS 404s), and fetch/XHR errors. Each error event includes stack trace, page URL, browser, OS, and session ID. Calculate the percentage of user sessions experiencing errors per page — this measures user impact more accurately than raw error count (one broken page viewed by 100 users = 100 errors but 100% session impact). Alert when error session percentage exceeds 5% for any page with >100 views. Compare error rates by browser to identify browser-specific regressions. Link RUM errors to backend traces via trace_id to identify full-stack root causes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=observability sourcetype="signalfx:rum:errors"
| eval error_type=coalesce(exception_type, "UnknownError")
| bin _time span=1h
| stats count as error_count, dc(session_id) as affected_sessions, values(error_type) as error_types by _time, page_url, browser_name
| join type=left _time page_url [search index=observability sourcetype="signalfx:rum:metrics"
    | bin _time span=1h
    | stats dc(session_id) as total_sessions by _time, page_url]
| eval error_session_pct=round(affected_sessions*100/nullif(total_sessions, 0), 1)
| where error_count > 10 OR error_session_pct > 5
| table _time, page_url, browser_name, error_count, affected_sessions, error_session_pct, error_types
| sort -error_session_pct
```
