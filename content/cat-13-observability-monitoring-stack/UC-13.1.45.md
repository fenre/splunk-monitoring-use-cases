# UC-13.1.45 ┬╖ Forwarder Version Compliance (operational inventory)

## Description

Old universal forwarders miss TLS fixes and acknowledgment behaviors required by security policy.

## Value

Old universal forwarders miss TLS fixes and acknowledgment behaviors required by security policy.

## Implementation

Maintain CSV of approved forwarder builds. Supplement with DS client list. Drive upgrades via DS server classes.

## Detailed Implementation

Prerequisites
тАв Install and configure the required add-on or app: Deployment Server, `splunkd` phone-home.
тАв Ensure the following data sources are available: `index=_internal` `group=deploymentclient` / `sourcetype=splunk:forwarder_inventory`.
тАв For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 тАФ Configure data collection
Maintain CSV of approved forwarder builds. Supplement with DS client list. Drive upgrades via DS server classes.

Step 2 тАФ Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=splunkd group=deploymentclient
| stats latest(version) as uf_version by hostname
| lookup approved_uf_versions.csv version OUTPUT approved
| where isnull(approved)
| table hostname, uf_version
```
