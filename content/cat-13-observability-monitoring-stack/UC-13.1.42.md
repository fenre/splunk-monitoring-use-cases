# UC-13.1.42 · I/O Wait Bottleneck Detection (_introspection)

## Description

High I/O wait on indexers correlates with slow searches, bucket replication, and ingestion lag.

## Value

High I/O wait on indexers correlates with slow searches, bucket replication, and ingestion lag.

## Implementation

Field names depend on platform; normalize in `props`. Correlate with storage latency metrics from SAN/NVMe. Alert when sustained `io_wait` exceeds threshold.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console, OTel/node_exporter.
• Ensure the following data sources are available: `index=_introspection` `sourcetype=splunk_resource_usage` (disk I/O fields) / host metrics.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Field names depend on platform; normalize in `props`. Correlate with storage latency metrics from SAN/NVMe. Alert when sustained `io_wait` exceeds threshold.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_introspection sourcetype=splunk_resource_usage
| where data.io_wait_pct > 25 OR data.disk_busy_pct > 80
| timechart span=5m avg(data.io_wait_pct) by host
```
