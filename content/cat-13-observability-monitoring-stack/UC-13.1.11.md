# UC-13.1.11 · Indexer Cluster Bucket Replication

## Description

Under-replicated buckets mean data is at risk of loss. Monitoring ensures the replication factor is maintained.

## Value

Under-replicated buckets mean data is at risk of loss. Monitoring ensures the replication factor is maintained.

## Implementation

Monitor cluster master/manager REST endpoints. Track replication and search factor compliance. Alert on any buckets not meeting the configured factor. Investigate cause (indexer down, disk full, network issues).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console.
• Ensure the following data sources are available: `_internal` (CM logs, REST endpoints).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Monitor cluster master/manager REST endpoints. Track replication and search factor compliance. Alert on any buckets not meeting the configured factor. Investigate cause (indexer down, disk full, network issues).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| rest /services/cluster/master/buckets
| where search_factor_met=0 OR replication_factor_met=0
| stats count as non_compliant_buckets
```

Understanding this SPL

**Indexer Cluster Bucket Replication** — Under-replicated buckets mean data is at risk of loss. Monitoring ensures the replication factor is maintained.

Documented **Data sources**: `_internal` (CM logs, REST endpoints). **App/TA** (typical add-on context): Monitoring Console. The SPL below should target the same indexes and sourcetypes you configured for that feed—rename `index=` / `sourcetype=` if your deployment differs.
