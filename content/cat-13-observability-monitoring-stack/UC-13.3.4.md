# UC-13.3.4 · Monitoring Coverage Gap Detection

## Description

Hosts not covered by any monitoring tool are blind spots. Detection ensures comprehensive infrastructure visibility.

## Value

Hosts not covered by any monitoring tool are blind spots. Detection ensures comprehensive infrastructure visibility.

## Implementation

Cross-reference CMDB with all monitoring tool inventories (Splunk forwarders, EDR agents, SNMP targets). Identify assets not monitored by any tool. Alert on new unmonitored assets. Track coverage percentage as a KPI.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Cross-tool asset correlation.
• Ensure the following data sources are available: CMDB + all monitoring tool inventories.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Cross-reference CMDB with all monitoring tool inventories (Splunk forwarders, EDR agents, SNMP targets). Identify assets not monitored by any tool. Alert on new unmonitored assets. Track coverage percentage as a KPI.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup cmdb_hosts.csv
| join type=left max=1 hostname [search index=_internal group=tcpin_connections | stats latest(_time) as splunk_last by hostname]
| join type=left max=1 hostname [search index=edr sourcetype="*sensor*" | stats latest(_time) as edr_last by hostname]
| where isnull(splunk_last) AND isnull(edr_last)
| table hostname, os, department
```
