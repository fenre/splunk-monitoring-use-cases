# UC-13.3.14 · OpenTelemetry Collector Dropped Spans and Metrics

## Description

Collector queue pressure causing data loss.

## Value

Collector queue pressure causing data loss.

## Implementation

Scrape OpenTelemetry Collector's internal metrics endpoint (default :8888/metrics) via Prometheus or OTLP. Ingest `otelcol_exporter_send_failed_spans`, `otelcol_processor_dropped_metric_points`, `otelcol_exporter_send_failed_metric_points`, and `otelcol_processor_dropped_spans`. Alert when any dropped/failed count >0 for critical pipelines. Correlate with `otelcol_receiver_accepted_spans` and queue depth metrics to identify backpressure. Tune batch size, retries, or add more collector replicas.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Custom (OTel Collector metrics).
• Ensure the following data sources are available: OTel Collector internal metrics (otelcol_exporter_send_failed_spans, otelcol_processor_dropped_metric_points).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Scrape OpenTelemetry Collector's internal metrics endpoint (default :8888/metrics) via Prometheus or OTLP. Ingest `otelcol_exporter_send_failed_spans`, `otelcol_processor_dropped_metric_points`, `otelcol_exporter_send_failed_metric_points`, and `otelcol_processor_dropped_spans`. Alert when any dropped/failed count >0 for critical pipelines. Correlate with `otelcol_receiver_accepted_spans` and queue depth metrics to identify backpressure. Tune batch size, retries, or add more collector replicas.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| mstats avg(_value) WHERE index=otel_collector metric_name IN ("otelcol_exporter_send_failed_spans", "otelcol_processor_dropped_metric_points", "otelcol_exporter_send_failed_metric_points") BY metric_name, exporter, processor span=5m
| where _value > 0
| timechart span=5m sum(_value) as dropped by metric_name
```
