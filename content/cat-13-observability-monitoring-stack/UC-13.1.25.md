# UC-13.1.25 · Cluster Bundle Push Failures (_internal)

## Description

Failed bundle distribution leaves indexers with stale `indexes.conf`, apps, or peer configs, causing search/replication skew.

## Value

Failed bundle distribution leaves indexers with stale `indexes.conf`, apps, or peer configs, causing search/replication skew.

## Implementation

Monitor CM and peer logs for bundle apply failures. Alert immediately. Verify disk space on peers and CM connectivity.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console, Cluster Manager.
• Ensure the following data sources are available: `index=_internal` `sourcetype=splunkd` (CM, bundle replication).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Monitor CM and peer logs for bundle apply failures. Alert immediately. Verify disk space on peers and CM connectivity.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=splunkd (bundle OR BundleReplication)
| search (log_level=ERROR OR "bundle.*fail" OR "Failed to apply")
| stats count by host, peer, message
| sort -count
```
