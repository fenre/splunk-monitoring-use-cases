# UC-13.2.30 · Configuration Assistant Recommendations Tracking

## Description

The Configuration Assistant (ITSI 4.20+) provides AI-powered optimization recommendations. Tracking which recommendations are implemented vs ignored ensures continuous ITSI health improvement.

## Value

The Configuration Assistant (ITSI 4.20+) provides AI-powered optimization recommendations. Tracking which recommendations are implemented vs ignored ensures continuous ITSI health improvement.

## Implementation

Review Configuration Assistant recommendations weekly. Categorize by type: threshold tuning, KPI consolidation, entity rule optimization, base search performance. Track implementation rate and measure impact (reduced skipped searches, fewer false positives, improved health score stability). Prioritize recommendations that affect critical services.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI 4.20+.
• Ensure the following data sources are available: `index=_internal sourcetype=itsi_internal_log`, Configuration Assistant UI.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Review Configuration Assistant recommendations weekly. Categorize by type: threshold tuning, KPI consolidation, entity rule optimization, base search performance. Track implementation rate and measure impact (reduced skipped searches, fewer false positives, improved health score stability). Prioritize recommendations that affect critical services.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=itsi_internal_log component=ConfigurationAssistant
| stats count by recommendation_type action_taken
| eval implementation_rate=round(count/sum(count)*100, 1)
```
