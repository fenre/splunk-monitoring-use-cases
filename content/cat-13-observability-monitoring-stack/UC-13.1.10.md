# UC-13.1.10 · Search Head Cluster Status

## Description

SHC member failures affect user access and search capacity. Captain election issues can cause complete SHC outage.

## Value

SHC member failures affect user access and search capacity. Captain election issues can cause complete SHC outage.

## Implementation

Monitor SHC member health via REST API. Track captain status and election events. Alert on member disconnection or replication failures. Monitor artifact replication lag between members.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console.
• Ensure the following data sources are available: `_internal` (SHC logs, REST endpoints).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Monitor SHC member health via REST API. Track captain status and election events. Alert on member disconnection or replication failures. Monitor artifact replication lag between members.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| rest /services/shcluster/member/members
| table label, status, last_heartbeat, replication_count
| eval heartbeat_age=now()-last_heartbeat
| where status!="Up" OR heartbeat_age > 300
```
