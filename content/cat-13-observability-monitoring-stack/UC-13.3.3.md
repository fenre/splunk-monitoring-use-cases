# UC-13.3.3 · PagerDuty/Opsgenie Integration

## Description

Tracking alert lifecycle and on-call response metrics ensures incident response SLAs are met and identifies process improvements.

## Value

Tracking alert lifecycle and on-call response metrics ensures incident response SLAs are met and identifies process improvements.

## Implementation

Poll PagerDuty/Opsgenie API for incident data. Track acknowledgment time, resolution time, and escalation rates. Report on on-call workload distribution. Alert when acknowledgment SLA is breached.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: PagerDuty API input.
• Ensure the following data sources are available: PagerDuty incidents API, Opsgenie alerts API.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll PagerDuty/Opsgenie API for incident data. Track acknowledgment time, resolution time, and escalation rates. Report on on-call workload distribution. Alert when acknowledgment SLA is breached.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=pagerduty sourcetype="pagerduty:incident"
| eval ack_time_min=round((acknowledged_at_epoch-created_at_epoch)/60,1)
| stats avg(ack_time_min) as avg_ack, avg(resolved_at_epoch-created_at_epoch)/60 as avg_resolve by service
```

Understanding this SPL

**Pipeline walkthrough**

• Scopes the data: index=pagerduty, sourcetype="pagerduty:incident". Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
