# UC-13.3.6 · SLO Burn Rate and Error Budget Tracking

## Description

Burn rate and error budget show how fast SLO is consumed. Tracking supports proactive action before SLA breach and prioritization of reliability work.

## Value

Burn rate and error budget show how fast SLO is consumed. Tracking supports proactive action before SLA breach and prioritization of reliability work.

## Implementation

Compute SLO compliance and error budget from availability/latency data. Ingest into Splunk. Alert on burn rate above threshold or error budget exhaustion. Report on remaining budget by service.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: ITSI, custom SLO metrics, APM data.
• Ensure the following data sources are available: SLO compliance metrics, error budget calculations.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Compute SLO compliance and error budget from availability/latency data. Ingest into Splunk. Alert on burn rate above threshold or error budget exhaustion. Report on remaining budget by service.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=slos sourcetype="slo:compliance"
| eval burn_rate=1-(success_count/(success_count+failure_count))
| bin _time span=1h
| stats avg(burn_rate) as avg_burn, sum(error_budget_consumed) as consumed by service, slo_name, _time
| where avg_burn > 0.1
```
