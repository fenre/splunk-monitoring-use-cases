# UC-13.2.19 · Multi-Tier Application Service Tree Modeling

## Description

Service trees link infrastructure KPIs to business outcomes, enabling impact analysis that shows which component failure affects which customer-facing service.

## Value

Service trees link infrastructure KPIs to business outcomes, enabling impact analysis that shows which component failure affects which customer-facing service.

## Implementation

Model services top-down: business service → application tier → middleware → infrastructure. Use entity rules with host/IP aliases to dynamically bind entities. Define dependency relationships so parent health reflects child degradation. Use service templates for repeatable patterns across environments.

| rename title as service_name
