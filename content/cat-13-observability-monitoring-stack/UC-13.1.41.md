# UC-13.1.41 · Dispatch Directory Size (_introspection)

## Description

Large dispatch directories fill disk and slow search teardown—common after runaway searches or stuck jobs.

## Value

Large dispatch directories fill disk and slow search teardown—common after runaway searches or stuck jobs.

## Implementation

Nightly scripted input: `du -sm` on `$SPLUNK_HOME/var/run/splunk/dispatch`, emit JSON. Alert on rapid growth. Automate cleanup of orphaned jobs per support guidance.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console, scripted disk check.
• Ensure the following data sources are available: Scripted input / HEC: `sourcetype=splunk:dispatch_stats` on `index=main` (dispatch dir size MB per SH).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Nightly scripted input: `du -sm` on `$SPLUNK_HOME/var/run/splunk/dispatch`, emit JSON. Alert on rapid growth. Automate cleanup of orphaned jobs per support guidance.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=main sourcetype="splunk:dispatch_stats"
| eval size_gb=round(dispatch_dir_size_mb/1024,2)
| where size_gb > 50
| stats max(size_gb) as max_gb by host
```
