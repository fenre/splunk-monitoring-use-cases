# UC-13.1.32 · Search Scheduler Fill Ratio (_internal)

## Description

Scheduler at capacity skips searches—blind spots for alerts and summaries.

## Value

Scheduler at capacity skips searches—blind spots for alerts and summaries.

## Implementation

Track skipped vs completed over sliding windows. Break down by app and user. Add concurrency or split heavy searches when fill ratio grows.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console.
• Ensure the following data sources are available: `index=_internal` `sourcetype=scheduler` / `splunkd` `group=scheduler`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track skipped vs completed over sliding windows. Break down by app and user. Add concurrency or split heavy searches when fill ratio grows.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=scheduler
| stats count(eval(status="skipped")) as skipped, count as total
| eval fill_skip_pct=round(100*skipped/total,2)
| where fill_skip_pct > 5
```
