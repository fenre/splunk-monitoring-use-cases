# UC-13.1.44 · App Version Consistency Across SHC (operational inventory)

## Description

Mixed app versions on SHC members cause bundle skew and intermittent UI or search errors.

## Value

Mixed app versions on SHC members cause bundle skew and intermittent UI or search errors.

## Implementation

Push inventory script via `runshellscript` or external job. Alert on mismatch for production apps. Exclude dev-only apps via lookup.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: SHC, REST.
• Ensure the following data sources are available: `| rest /services/apps/local` per member, `sourcetype=splunk:shc_app_inventory`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Push inventory script via `runshellscript` or external job. Alert on mismatch for production apps. Exclude dev-only apps via lookup.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=inventory sourcetype="splunk:shc_app_inventory"
| stats values(app_version) as ver by app_name, member
| eventstats dc(ver) as ver_count by app_name
| where ver_count > 1
| table app_name, member, ver
```
