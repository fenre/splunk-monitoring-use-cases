# UC-13.1.1 · Indexer Queue Fill Ratio

## Description

Backed-up indexing queues cause data loss or delay. Detection enables immediate investigation of ingestion bottlenecks.

## Value

Backed-up indexing queues cause data loss or delay. Detection enables immediate investigation of ingestion bottlenecks.

## Implementation

Monitor parsing, merging, and typing queues via `_internal`. Alert when any queue exceeds 70% fill ratio. Investigate source of data surge (new data source, burst events). Consider parallel pipelines or additional indexers.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console (built-in).
• Ensure the following data sources are available: `_internal` (metrics.log, queue metrics).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Monitor parsing, merging, and typing queues via `_internal`. Alert when any queue exceeds 70% fill ratio. Investigate source of data surge (new data source, burst events). Consider parallel pipelines or additional indexers.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=splunkd group=queue
| eval fill_pct=round(current_size/max_size*100,1)
| where fill_pct > 70
| timechart span=5m max(fill_pct) as queue_pct by name
```
