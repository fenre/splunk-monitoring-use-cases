# UC-13.4.9 · LLM Prompt Injection Attempt Detection

## Description

Injection attempts try to override system instructions or exfiltrate context; logging and alerting limits abuse before secrets or policies are bypassed.

## Value

Injection attempts try to override system instructions or exfiltrate context; logging and alerting limits abuse before secrets or policies are bypassed.

## Implementation

Log truncated or hashed prompts server-side only (privacy review required). Use regex and optional ML classifiers for injection patterns. Route critical hits to SOC. Do not index full PII-heavy prompts without policy. Pair with response policy blocks.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: LLM gateway with prompt logging (redacted), DLP on egress.
• Ensure the following data sources are available: `sourcetype=openai:api`, `sourcetype=azure:openai`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Log truncated or hashed prompts server-side only (privacy review required). Use regex and optional ML classifiers for injection patterns. Route critical hits to SOC. Do not index full PII-heavy prompts without policy. Pair with response policy blocks.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ai_ops sourcetype IN ("openai:api","azure:openai")
| search match(lower(prompt_preview),"(ignore|disregard|system prompt|jailbreak|sudo mode|base64)")
| eval severity=if(match(prompt_preview,"(?i)(password|secret|api[_-]?key)"),"critical","high")
| stats count by user_id, app_id, severity
| where count>=3
| sort -count
```
