# UC-13.1.31 · Workload Management Pool Saturation (_internal)

## Description

WLM pools prevent search starvation; saturation delays critical searches and alerts.

## Value

WLM pools prevent search starvation; saturation delays critical searches and alerts.

## Implementation

Define pool limits per SLA. Alert on rejections or sustained queue depth. Correlate with ad-hoc search storms.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Monitoring Console.
• Ensure the following data sources are available: `index=_internal` `sourcetype=splunkd` (WorkloadManager, `workload_pool`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Define pool limits per SLA. Alert on rejections or sustained queue depth. Correlate with ad-hoc search storms.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal sourcetype=splunkd (WorkloadManager OR "workload_pool")
| search ("saturated" OR "rejected" OR "queue" OR log_level=ERROR)
| stats count by host, pool_name, message
| sort -count
```
