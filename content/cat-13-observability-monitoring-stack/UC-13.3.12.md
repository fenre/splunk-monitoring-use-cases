# UC-13.3.12 · Prometheus TSDB Compaction Failures

## Description

Storage engine issues impacting metric retention.

## Value

Storage engine issues impacting metric retention.

## Implementation

Poll Prometheus `/api/v1/status/tsdb` for head stats, block count, and retention info. Ingest Prometheus server logs (stderr) for compaction-related errors. Parse TSDB head chunk count and series count for anomaly detection. Alert on compaction failure messages or when head series count grows abnormally (potential compaction backlog). Correlate with disk I/O and storage capacity metrics.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=prometheus (sourcetype="prometheus:tsdb" OR sourcetype="prometheus:log")
| search "compaction" AND ("failed" OR "error" OR "panic")
| stats count, latest(_time) as last_occurrence by host, message
| eval last_occurrence_human=strftime(last_occurrence,"%Y-%m-%d %H:%M:%S")
| table host, message, count, last_occurrence_human
| sort -count
```
