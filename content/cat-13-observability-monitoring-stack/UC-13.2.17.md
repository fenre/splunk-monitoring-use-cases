# UC-13.2.17 · Notable Event Suppression Audit

## Description

Over-suppression hides incidents; audit ensures suppress rules and analyst actions are justified.

## Value

Over-suppression hides incidents; audit ensures suppress rules and analyst actions are justified.

## Implementation

Ingest notable audit events with user, rule, and reason. Alert on high-volume suppression by single user or new rule. Review monthly for policy compliance.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI, ES correlation (if linked).
• Ensure the following data sources are available: `index=itsi_notable:audit` / notable audit logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest notable audit events with user, rule, and reason. Alert on high-volume suppression by single user or new rule. Review monthly for policy compliance.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsi_notable:audit OR index=notable sourcetype="itsi:notable_audit"
| search action IN ("suppress","close","suppress_episode")
| stats count by user, rule_id, reason
| sort -count
```
