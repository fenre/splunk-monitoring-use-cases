# 22.30.23 — Security control testing outcomes aggregation

**Criticality:** low | **Difficulty:** advanced | **Status:** N/A

## Description

Supports APRA CPS 234 supervisory examination readiness for security control testing outcomes aggregation.

## Value

Gives the responsible team auditor-ready evidence for APRA CPS 234 compliance by automating regulatory compliance evidence using indexes test and sourcetypes cps234:penetration. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=test sourcetype="cps234:penetration" earliest=-730d
| stats latest(remediation_pct) as p by engagement_id
| where p<100
```

## CIM SPL

```spl
| tstats summariesonly=t latest(All_Changes.status) as agg_value from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - agg_value
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=test`; Sourcetypes: `sourcetype=cps234:penetration`.
