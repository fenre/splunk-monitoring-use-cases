# UC-22.5.19 · MiFID II Market Abuse — Insider List Access Log Correlation Before Price-Sensitive Events

## Description

MAR requires insider lists. Correlating document or deal-room access with corporate calendar events surfaces potential leaks when access clusters immediately before sensitive milestones.

## Value

MAR requires insider lists. Correlating document or deal-room access with corporate calendar events surfaces potential leaks when access clusters immediately before sensitive milestones.

## Implementation

(1) Heuristic only—human investigation required; (2) align ObjectId to deal dataroom IDs; (3) legal privilege workflow; (4) integrate insider list membership via `lookup`; (5) document SMF16 review cadence.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Cloud Services (Splunkbase 3110), DLP.
• Ensure the following data sources are available: `index=o365` `sourcetype="ms:o365:management"` (UserId, ObjectId, Operation); `index=corporate` `sourcetype="deal:insider_list"` (project_id, user_id, added_date); `index=corporate` `sourcetype="deal:milestone"` (project_id, milestone_type, milestone_date).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Heuristic only—human investigation required; (2) align ObjectId to deal dataroom IDs; (3) legal privilege workflow; (4) integrate insider list membership via `lookup`; (5) document SMF16 review cadence.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=corporate sourcetype="deal:milestone" milestone_type="earnings_release" earliest=-180d
| eval ms=strptime(milestone_date,"%Y-%m-%d")
| join type=left project_id [
    search index=o365 sourcetype="ms:o365:management" Operation="FileDownloaded" earliest=-180d
    | rename UserId as user_id
    | stats earliest(_time) as first_dl by user_id, ObjectId
  ]
| where first_dl<relative_time(ms,"-14d@d") AND first_dl>relative_time(ms,"-60d@d")
| table project_id, milestone_date, user_id, first_dl
```
