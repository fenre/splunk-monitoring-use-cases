# UC-22.14.69 · Network Disconnect for Inactive Sessions on Admin Services

## Description

Measures idle durations on administrative listeners for SC-10 network disconnect controls (proxy perspective).

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes proxy and sourcetypes zscaler:web, zscaler:web`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Validate `transaction` keys; (2) align idle timeout to policy minutes; (3) alert on excessive admin idle tunnels; (4) document VPN vs ZTNA differences. Indexes required: `index=proxy`; Sourcetypes: `sourcetype=zscaler:web`, `sourcetype=zscaler:web``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: F5 BIG-IP APM or Zscaler logs via TA.
• Ensure the following data sources are available: `sourcetype=zscaler:web` or `f5:apm:session`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Validate `transaction` keys; (2) align idle timeout to policy minutes; (3) alert on excessive admin idle tunnels; (4) document VPN vs ZTNA differences.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=proxy sourcetype=zscaler:web earliest=-24h app_class="Admin"
| transaction user maxpause=30m maxspan=24h
| eval idle_mins=round(duration/60,1)
| where idle_mins>240
```
