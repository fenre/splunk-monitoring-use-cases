# UC-22.11.30 · Malware Detection Volume Trend by Store and Server Tier

## Description

Trends malware blocks and quarantines where payment systems exist so regional infection campaigns are visible before they spread into the CDE.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes edr and sourcetypes crowdstrike:detection. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map CrowdStrike `aid` to store metadata. (2) Filter out benign test detections via `severity` threshold. (3) Compare to seasonal retail baseline. (4) Include in monthly security committee pack. Indexes required: `index=edr`; Sourcetypes: `sourcetype=crowdstrike:detection`; Lookups: `pci_site_codes`, `pci_site_codes.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for CrowdStrike FDR (5082), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=edr` `sourcetype="crowdstrike:detection"`; ES `malware` data model if accelerated.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map CrowdStrike `aid` to store metadata. (2) Filter out benign test detections via `severity` threshold. (3) Compare to seasonal retail baseline. (4) Include in monthly security committee pack.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=edr sourcetype="crowdstrike:detection" earliest=-30d
| lookup pci_site_codes.csv aid OUTPUT store_region, pci_tier
| where pci_tier IN ("pos","store_server","corp_payment")
| bin _time span=1d
| stats count by _time, store_region, pci_tier
| eventstats sum(count) as daily_total by _time
| eval share_pct=if(daily_total>0, round(100*count/daily_total,2), null())
| sort _time, store_region
```

## SPL

| lookup pci_site_codes.csv aid OUTPUT store_region, pci_tier
| where pci_tier IN ("pos","store_server","corp_payment")
| bin _time span=1d
| stats count by _time, store_region, pci_tier
| eventstats sum(count) as daily_total by _time
| eval share_pct=if(daily_total>0, round(100*count/daily_total,2), null())
