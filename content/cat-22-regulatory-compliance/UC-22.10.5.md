# UC-22.10.5 ┬Ę Termination ŌĆö Access Revocation Within Policy Window

## Description

Detects logons, VPN sessions, or EHR access after HR termination timestamp so access revocation SLAs are provably enforced.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security, NIST 800-53 compliance by automating regulatory compliance evidence using indexes citrix, epic, hr and sourcetypes epic:audit`, workday:termination. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ingest authoritative termination feed within 15 minutes SLA; (2) tune out break-glass shared accounts; (3) auto-disable via SOAR optional; (4) weekly HIPAA access committee digest. Indexes required: `index=citrix`, `index=epic`, `index=hr`, `index=windows`; Sourcetypes: `sourcetype=epic:audit``, `sourcetype=workday:termination`.

Step 2 ŌĆö Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr sourcetype="workday:termination" earliest=-30d
| eval term_epoch=strptime(term_date,"%Y-%m-%d %H:%M:%S")
| table samaccountname, term_epoch
| rename samaccountname as user
| join type=inner user [
    search index=windows EventCode="4624" earliest=-30d
    | eval user=mvindex(Security_ID,0)
    | rex field=user "\\\\(?<sam>[^\\\\]+)$"
    | eval user=coalesce(sam,user)
    | table _time, user, WorkstationName, Logon_Type ]
| where _time > term_epoch
| eval hours_after_term=round((_time-term_epoch)/3600,2)
```
