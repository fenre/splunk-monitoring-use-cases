# UC-22.10.21 · Access Control — Role-Based Violations (Coder Accessing Medication Admin)

## Description

Detects EHR workspace or activity codes inconsistent with HR job role, supporting least-privilege access control for ePHI workflows.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security, HIPAA Security Rule compliance by automating regulatory compliance evidence using indexes epic and sourcetypes epic:audit, epic:audit`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Refresh role mapping nightly from IAM; (2) tune with clinical float pools; (3) integrate with provisioning QA before go-live. Indexes required: `index=epic`; Sourcetypes: `sourcetype=epic:audit`, `sourcetype=epic:audit``; Lookups: `epic_user_roles`, `epic_user_roles.csv`.

Step 1 — Configure data collection
(1) Refresh role mapping nightly from IAM; (2) tune with clinical float pools; (3) integrate with provisioning QA before go-live.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=epic sourcetype="epic:audit" earliest=-24h
| lookup epic_user_roles.csv USER_ID OUTPUT hr_role, allowed_activities mvsep="|"
| eval allowed=if(isnull(allowed_activities), "*", allowed_activities)
| where isnotnull(hr_role) AND NOT match(Activity, allowed)
| stats count values(Activity) as bad_activities by USER_ID, hr_role
| sort - count
```
