# 22.10.22 — Remote ePHI Access — MFA Gap for VPN + O365 Clinical Mail

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Surfaces successful VPN or Exchange Online logons without MFA claim for users in ePHI-privileged groups so remote access meets organizational authentication policy.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes o365, vpn and sourcetypes ms:o365:management, ms:o365:reporting`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=o365 sourcetype="ms:o365:management" Workload="AzureActiveDirectory" Operation="UserLoggedIn" earliest=-24h
| eval mfa=if(match(ExtendedProperties,"MFA\s*completed|multi-factor"),1,0)
| lookup hipaa_ephi_users.csv user OUTPUT in_ephi_group
| where in_ephi_group="true" AND mfa=0
| stats count by user, ClientIP, ApplicationId
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.user | sort - count
```

## Implementation

(1) Confirm `ExtendedProperties` field names for your tenant; (2) include Conditional Access exclusions report; (3) feed to IAM for forced enrollment. Indexes required: `index=o365`, `index=vpn`; Sourcetypes: `sourcetype=ms:o365:management`, `sourcetype=ms:o365:reporting``; Lookups: `hipaa_ephi_users`, `hipaa_ephi_users.csv`.

## Visualization
