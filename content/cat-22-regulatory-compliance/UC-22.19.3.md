# UC-22.19.3 · STIG file-integrity hash mismatch (FISMA / FedRAMP)

## Description

Strengthens agency or cloud service continuous authorization evidence under FISMA / FedRAMP by detecting drift between deployed file-integrity hashes and the STIG baseline across cloud, endpoint, and GRC feeds.

## Value

Gives the responsible team auditor-ready evidence for FISMA compliance by automating regulatory compliance evidence using indexes os and sourcetypes stash:file_integrity. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=os`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os sourcetype=stash:file_integrity earliest=-7d
| eval uc_tag="22.19.3"
| lookup stig_baseline.csv path OUTPUT expected_hash
| where sha256!=expected_hash
| stats count by host, path
```
