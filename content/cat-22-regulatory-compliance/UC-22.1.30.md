# UC-22.1.30 · GDPR Data Processing Agreement Obligation Control Matrix

## Description

Article 28(3) spells out mandatory terms that must bind processors. This use case tracks which DPA clauses (sub-processing, assistance with DSARs, deletion/return, audit rights) are marked as included per processor and flags incomplete DPAs before go-live of new integrations.

## Value

Article 28(3) spells out mandatory terms that must bind processors. This use case tracks which DPA clauses (sub-processing, assistance with DSARs, deletion/return, audit rights) are marked as included per processor and flags incomplete DPAs before go-live of new integrations.

## Implementation

(1) Encode Art. 28(3) obligations as rows in `gdpr_dpa_obligation_matrix.csv`; (2) integrate legal review workflow to set `included` and evidence URLs; (3) alert when any processor has missing mandatory clauses; (4) block go-live in ServiceNow when `live_without_dpa=1`; (5) refresh after each DPA amendment.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup gdpr_dpa_obligation_matrix.csv
| stats dc(eval(if(included="false", clause_id, null()))) as missing_clauses,
        values(eval(if(included="false", clause_id, null()))) as missing_list
  by processor_name
| where missing_clauses > 0
| join processor_name [
    | inputlookup gdpr_processor_register.csv
    | fields processor_name, go_live_date, dpa_signed
]
| eval live_without_dpa=if(dpa_signed!="true" AND isnotnull(go_live_date), 1, 0)
| table processor_name, missing_clauses, missing_list, dpa_signed, go_live_date, live_without_dpa
```
