# UC-22.14.68 · Transmission Confidentiality and Integrity — TLS Policy Downgrades

## Description

Detects servers offering weak TLS versions or cipher suites for SC-8 transmission confidentiality and integrity.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes net and sourcetypes stream:tls. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Enrich with asset owner; (2) emergency exceptions with expiry; (3) track remediation in RA-7; (4) validate after patch window. Indexes required: `index=net`; Sourcetypes: `sourcetype=stream:tls`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Stream (Splunkbase 1809) or certificate inventory.
• Ensure the following data sources are available: `stream:tls` or `tenable:` plugin SSL findings.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enrich with asset owner; (2) emergency exceptions with expiry; (3) track remediation in RA-7; (4) validate after patch window.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=net sourcetype=stream:tls earliest=-7d version IN ("SSLv3","TLS1.0","TLS1.1")
| stats dc(src_ip) as clients, values(cipher) as ciphers by dest_ip, dest_port
| sort - clients
```
