# 22.46.5 — Developer data-handling training — prod-access engineers lacking training

**Criticality:** high | **Difficulty:** intermediate | **Status:** draft

## Description

Prevents the 'developers with production access but stale training' anti-pattern that surfaces in GDPR Art.32 enforcement cases.

## Value

Links IdP posture to training posture — the evidence auditors actually ask for.

## SPL

```spl
| rest /services/identity/prod-access-groups splunk_server=*
| fields user group scope
| where scope="regulated"
| join user [search index=lms sourcetype=completion course_id=data_handling_v* earliest=-365d | stats max(_time) as last_completion by user]
| where (now()-last_completion)>31536000 OR isnull(last_completion)
| table user group scope last_completion
```

## Implementation

(1) Tag IdP groups for prod access with regulated scope; (2) publish DPO-approved course IDs; (3) run weekly; (4) auto-revoke access after configured overdue window. Indexes required: `index=lms`; Sourcetypes: `sourcetype=completion`.

## Visualization

Table of engineers without training, bar chart by team, single-value 'prod-access engineers with current training'. Key fields: `group`, `last_completion`, `scope`, `user`, `user]`.

## Compliance

- **GDPR** 2016/679: Art.32(4) (partial) — Art.32(4) requires awareness of personnel processing personal data.
- **HIPAA** 2013-final: §164.308(a)(5) (partial) — HIPAA workforce training applies to workforce with ePHI access.
- **ISO-27001** 2022: A.6.3 (full) — A.6.3 information-security awareness, education, and training.
- **UK GDPR** post-Brexit: Art.32(4) (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.32(4).

## Known False Positives

Freshly-onboarded engineers in the 30-day grace window should be allow-listed by onboarding_date.
