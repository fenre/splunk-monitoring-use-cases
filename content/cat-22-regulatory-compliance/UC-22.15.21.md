# UC-22.15.21 · Forced Remote Engineering Session Termination After Hours

## Description

Ensures vendor and internal remote sessions to OT are closed at shift end by correlating still-active tunnels with schedule.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes fw and sourcetypes pan:traffic. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Configure firewall idle timeouts; (2) CyberArk max session duration; (3) validate OT maintenance windows; (4) alert if sessions exceed 8h. Indexes required: `index=fw`; Sourcetypes: `sourcetype=pan:traffic`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Palo Alto (2757), CyberArk (4295).
• Ensure the following data sources are available: `pan:traffic` app=ssl,sslvpn; `cyberark:psm` session end.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Configure firewall idle timeouts; (2) CyberArk max session duration; (3) validate OT maintenance windows; (4) alert if sessions exceed 8h.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=fw sourcetype="pan:traffic" earliest=-2d
| where app IN ("ssl","sslvpn","gp-gateway") AND dest_zone="OT-Jump"
| eval sess_key=concat(src,"-",dest_ip)
| stats min(_time) as start, max(_time) as end by sess_key, user
| eval span_h=(end-start)/3600
| eval ended_after_shift=if((strftime(end,"%H")>"22" OR strftime(end,"%H")<"05") AND span_h>4,1,0)
| where ended_after_shift=1
| table start, end, span_h, sess_key, user
```
