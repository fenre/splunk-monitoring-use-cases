# UC-22.15.49 · Safety Zone Isolation Verification — Non-Safety OPC on SIS VLAN

## Description

Identifies OPC-UA or HMI sessions sourced from assets tagged non-safety on safety-controller VLANs.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_sec and sourcetypes claroty:session. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align CMDB safety flags; (2) re-cable or re-ACL; (3) monthly scan; (4) document compensating monitoring. Indexes required: `index=ot_sec`; Sourcetypes: `sourcetype=claroty:session`; Lookups: `asset_safety_class`, `asset_safety_class.csv`, `vlan_role`, `vlan_role.csv`.

Step 1 — Configure data collection
(1) Align CMDB safety flags; (2) re-cable or re-ACL; (3) monthly scan; (4) document compensating monitoring.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_sec sourcetype="claroty:session" earliest=-7d
| lookup asset_safety_class.csv asset_id OUTPUT safety_class
| lookup vlan_role.csv vlan OUTPUT vlan_role
| where vlan_role="SIS" AND safety_class!="SIS"
| stats dc(asset_id) as assets, count by vlan, protocol
| sort - count
```
