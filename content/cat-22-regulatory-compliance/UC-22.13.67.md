# 22.13.67 — Supply Chain Incident Response Task Tracking

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We monitor open tasks for supply chain incidents affecting registered vendor components used in BES Cyber Systems.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes supply_chain_incident, supply_chain_incident`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=itsm sourcetype=supply_chain_incident earliest=-180d status!="Closed"
| eval due=strptime(remediation_due,"%Y-%m-%d")
| where due < now()
| stats values(task) as open_tasks by incident_id, vendor, component
| sort - incident_id
```

## Implementation

(1) Create ticket type for supply chain cases. (2) Link tasks to asset inventory rows. (3) Daily stand-up dashboard. (4) Close with compensating control evidence if needed. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=supply_chain_incident`, `sourcetype=supply_chain_incident``.

## Visualization

Table (overdue incidents), Column chart (by vendor), Single value (open)

## Compliance

- **NERC CIP** current: CIP-013-1 R4 Part 4.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-013-1 R4 Part 4.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
