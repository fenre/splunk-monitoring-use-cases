# 22.30.1 — Technology risk management governance metrics

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Supports MAS TRM supervisory examination readiness for technology risk management governance metrics.

## Value

Gives the responsible team auditor-ready evidence for MAS TRM compliance by automating regulatory compliance evidence using indexes bank and sourcetypes core:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=bank sourcetype="core:audit" regulator="MAS" earliest=-37h
| stats count by function_id, user, action
| lookup mas_trm_critical_functions.csv function_id OUTPUT tier
| where tier="Material" AND action="FUNDS_TRANSFER"
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.user, All_Changes.action | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=bank`; Sourcetypes: `sourcetype=core:audit`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones).

## Compliance

- **MAS TRM** 2021: §4.1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for §4.1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
