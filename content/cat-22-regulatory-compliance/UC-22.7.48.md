# UC-22.7.48 · Multi-Region Failover — Health-Check Driven DNS Answer Changes

## Description

Monitors GSLB or Route53 answer sets proving multi-region recovery execution.

## Value

Monitors GSLB or Route53 answer sets proving multi-region recovery execution.

## Implementation

(1) Document expected flip patterns; (2) correlate with incident tickets; (3) tabletop validation.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Route53 query logs.
• Ensure the following data sources are available: `route53`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Document expected flip patterns; (2) correlate with incident tickets; (3) tabletop validation.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=dns sourcetype=route53 earliest=-30d
| where match(query_name,"www\.prod\.example\.com") AND message_type="RESPONSE"
| stats values(rdata) as answers by bin(_time,5m)
| where mvcount(answers)>1
```
