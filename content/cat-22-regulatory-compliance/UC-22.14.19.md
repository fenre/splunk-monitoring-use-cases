# UC-22.14.19 · Least Privilege — Excessive Cloud IAM Permissions

## Description

Reviews AWS CloudTrail `AttachUserPolicy` events for high-risk managed policies supporting AC-6 least privilege.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes cloud and sourcetypes aws:cloudtrail. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Use `spath` or JSON KV mode consistently; (2) maintain deny-list of risky policy ARNs; (3) auto-ticket attachments; (4) quarterly access review export. Indexes required: `index=cloud`; Sourcetypes: `sourcetype=aws:cloudtrail`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for AWS (Splunkbase 1876).
• Ensure the following data sources are available: `aws:cloudtrail` management events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Use `spath` or JSON KV mode consistently; (2) maintain deny-list of risky policy ARNs; (3) auto-ticket attachments; (4) quarterly access review export.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cloud sourcetype=aws:cloudtrail earliest=-7d eventName IN ("AttachUserPolicy","PutUserPolicy","AttachRolePolicy")
| spath path=eventSource output=eventSource
| spath path=requestParameters output=requestParameters
| search requestParameters="*AdministratorAccess*" OR requestParameters="*IAMFullAccess*"
| stats count by userName, eventName, requestParameters, aws_account_id
```
