# UC-22.14.15 · Cross-Organizational Audit Forwarding Health to SIEM

## Description

Validates receipt rates from partner or subsidiary syslog streams against SLAs for AU-16 cross-organizational auditing agreements.

## Value

Validates receipt rates from partner or subsidiary syslog streams against SLAs for AU-16 cross-organizational auditing agreements.

## Implementation

(1) Tag events with `org_id` at ingest; (2) baseline median EPS per hour/day; (3) alert on sustained drops; (4) document contractual EPS and retention in the interconnection agreement (CA-9).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform, Splunk Connect for Syslog.
• Ensure the following data sources are available: `index=partner*` OR `host=partner_*` sourcetype=syslog or HEC tokens per org.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag events with `org_id` at ingest; (2) baseline median EPS per hour/day; (3) alert on sustained drops; (4) document contractual EPS and retention in the interconnection agreement (CA-9).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=partner earliest=-24h
| bin _time span=1h
| stats count by _time, org_id, sourcetype
| eventstats median(count) as med by org_id, sourcetype
| eval drop=if(count < med*0.5, 1, 0)
| where drop=1
```
