# UC-22.10.10 · Business Associate Contracts — BAA Coverage for Connected Systems

## Description

Highlights new interfaces or SaaS tenants talking to ePHI systems without a registered BAA so legal can close contract gaps before production traffic carries PHI.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes epic and sourcetypes epic:interface, epic:interface`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Populate `baa_register.csv` from contract repository; (2) integrate ServiceNow vendor risk tasks; (3) block firewall exceptions until BAA recorded (process outside Splunk). Indexes required: `index=epic`; Sourcetypes: `sourcetype=epic:interface`, `sourcetype=epic:interface``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk DB Connect (2686).
• Ensure the following data sources are available: `index=epic` `sourcetype=epic:interface` (remote host, interface name), `lookup baa_register.csv` (`vendor`, `hostname`, `baa_status`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate `baa_register.csv` from contract repository; (2) integrate ServiceNow vendor risk tasks; (3) block firewall exceptions until BAA recorded (process outside Splunk).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=epic sourcetype="epic:interface" earliest=-7d
| stats latest(_time) as last_seen values(interface_name) as ifaces by remote_host
| lookup baa_register.csv hostname as remote_host OUTPUT vendor, baa_status, baa_expiry
| where isnull(baa_status) OR baa_status!="Active"
| eval days_since=round((now()-last_seen)/86400,1)
| sort - last_seen
```
