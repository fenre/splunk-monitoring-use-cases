# UC-22.10.44 ¬∑ Breach Risk Assessment ‚Äî Four-Factor Documentation Tracking

## Description

Stores structured answers for nature/extent of PHI, unauthorized recipient, actual acquisition, and mitigation so breach vs non-breach determinations are auditable.

## Value

Stores structured answers for nature/extent of PHI, unauthorized recipient, actual acquisition, and mitigation so breach vs non-breach determinations are auditable.

Step 1 ‚Äî Configure data collection
(1) Emit JSON from GRC web form via HEC; (2) retain immutable snapshots on decision changes; (3) legal owns field definitions.

Step 2 ‚Äî Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="hipaa:breach_ra" earliest=-365d
| eval complete=if(isnotnull(factor1) AND isnotnull(factor2) AND isnotnull(factor3) AND isnotnull(factor4),"Yes","No")
| where complete="No" OR isnull(preliminary_result)
| eval days_open=round((now()-_time)/86400,1)
| table incident_id, days_open, reviewer, factor1, factor2, factor3, factor4
| sort - days_open
```
