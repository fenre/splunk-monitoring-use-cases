# UC-22.14.47 · Incident Reporting to Authorities — Regulatory Timer Watch

## Description

Adds timer fields for regulatory reporting obligations tied to incidents for IR-6 incident reporting.

## Value

Adds timer fields for regulatory reporting obligations tied to incidents for IR-6 incident reporting.

## Implementation

(1) Standardize tags for regulated incidents; (2) integrate legal workflow fields; (3) alert before deadlines; (4) document reporting chains per jurisdiction.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263).
• Ensure the following data sources are available: `` `notable` `` with `category` or `tag` for regulated incidents.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Standardize tags for regulated incidents; (2) integrate legal workflow fields; (3) alert before deadlines; (4) document reporting chains per jurisdiction.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
`notable` earliest=-30d tag="regulated_breach"
| eval hours_open=round((now()-_time)/3600,1)
| where hours_open>72 AND status!="Closed"
| table _time, rule_name, owner, status, hours_open
```
