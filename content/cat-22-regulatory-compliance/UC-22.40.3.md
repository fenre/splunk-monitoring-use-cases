# UC-22.40.3 · Periodic access review SLA — stale certifications by control owner

## Description

Tracks the age of the most recent access review per application and control owner. Applications falling outside the 90-day cadence drive follow-up tickets.

## Value

Access review is the #1 ITGC deficiency in typical public-company audits; turning it into a green/red KPI changes the operating rhythm.

## Implementation

(1) Ingest review-completion events from the GRC tool; (2) refresh app_inventory.csv weekly; (3) run daily.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: See subcategory preamble..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest review-completion events from the GRC tool; (2) refresh app_inventory.csv weekly; (3) run daily.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype=iam:access_review earliest=-365d
| stats max(_time) as last_review by app_id reviewer
| eval days_since=round((now()-last_review)/86400,0)
| where days_since>90
| lookup app_inventory.csv app_id OUTPUT owner criticality
| table owner app_id criticality last_review days_since
```
