# UC-22.11.72 · Log Source Coverage Gaps — Expected Sourcetypes with Zero Events

## Description

Compares a reference list of required sourcetypes for PCI monitoring to last-hour ingestion so missing log categories are detected before an audit sample fails completeness tests.

## Value

Compares a reference list of required sourcetypes for PCI monitoring to last-hour ingestion so missing log categories are detected before an audit sample fails completeness tests.

## Implementation

(1) Curate lookup with owner contact per sourcetype. (2) Run hourly; exclude maintenance windows via secondary lookup. (3) Auto-ticket to platform ops. (4) Attach zero-gap monthly PDF for PCI evidence.

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Curate lookup with owner contact per sourcetype. (2) Run hourly; exclude maintenance windows via secondary lookup. (3) Auto-ticket to platform ops. (4) Attach zero-gap monthly PDF for PCI evidence.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup pci_required_sourcetypes.csv
| fields required_sourcetype, required_index
| join type=left required_sourcetype [
    | metadata type=sourcetypes index=pci index=cde
    | rename sourcetype as required_sourcetype
    | eval seen_recent=if(recentTime>relative_time(now(),"-2h"),1,0)
    | fields required_sourcetype, seen_recent, totalCount
  ]
| where isnull(seen_recent) OR seen_recent=0
| table required_index, required_sourcetype, seen_recent, totalCount
```
