# UC-22.15.27 · OPC-UA Message Tampering Indicators via Signature Failures

## Description

Raises alerts when OPC-UA signing/encryption modes downgrade or signature checks fail, indicating integrity issues on industrial sessions.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_protocol and sourcetypes edge_hub_opcua. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Disable None except lab VLANs; (2) baseline each server; (3) alert on new None percentage; (4) correlate with firewall TLS inspection if used. Indexes required: `index=ot_protocol`; Sourcetypes: `sourcetype=edge_hub_opcua`.

Step 1 — Configure data collection
(1) Disable None except lab VLANs; (2) baseline each server; (3) alert on new None percentage; (4) correlate with firewall TLS inspection if used.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_protocol sourcetype="edge_hub_opcua" earliest=-24h
| eval mode=coalesce(opcua_security_mode,"None")
| stats count by opcua_server_endpoint, mode, opcua_security_policy_uri
| eventstats sum(count) as tot by opcua_server_endpoint
| eval pct=round(100*count/tot,2)
| where mode="None" AND pct>1
| sort - pct
```
