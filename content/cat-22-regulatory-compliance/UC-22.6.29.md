# UC-22.6.29 · Capacity Management — Disk Utilization Forecast Breach in 14 Days

## Description

Projects disk full dates from growth rate for servers supporting ISMS scope to meet capacity management.

## Value

Projects disk full dates from growth rate for servers supporting ISMS scope to meet capacity management.

## Implementation

(1) Collect daily samples; (2) alert predicted pct>90; (3) capacity committee review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Infrastructure monitoring or `os:df`.
• Ensure the following data sources are available: `os:df` or `metrics`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Collect daily samples; (2) alert predicted pct>90; (3) capacity committee review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os sourcetype=os:df (mount="/var" OR mount="/data") earliest=-30d
| timechart span=1d latest(UsePercent) as pct by host
| predict pct as future algorithm=LLP future_timespan=14
```
