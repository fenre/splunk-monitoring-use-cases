# UC-22.3.43 · DORA Art.8 — ICT risk identification: newly discovered high-severity exposure on critical financial services

## Description

Identifies newly-discovered CVSS>=7 exposures on hosts supporting a DORA-critical financial service. Measures whether the finding is registered in the ICT risk register within the 24h target implied by Art.8 'continuous identification'.

## Value

Produces per-finding evidence that critical-service exposure is identified and logged within the DORA-aligned window, rather than relying on monthly scanner review meetings.

## Implementation

(1) Onboard scanner output; (2) tag hosts with ict_service + criticality_tier nightly; (3) schedule UC every 15m; (4) each hit opens a risk-register entry (ServiceNow) within 24h; (5) weekly report to ICT risk committee; (6) breach of 24h SLA escalates to the Head of IR and Board/Audit Committee.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security, Splunk Add-on for Tenable, Splunk Add-on for Qualys.
• Ensure the following data sources are available: `index=vm` sourcetype=tenable:sc:vuln OR sourcetype=qualys:host; `index=cmdb` sourcetype=cmdb:ci_ict_service; DORA critical-service lookup `dora_critical_services.csv`..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Onboard scanner output; (2) tag hosts with ict_service + criticality_tier nightly; (3) schedule UC every 15m; (4) each hit opens a risk-register entry (ServiceNow) within 24h; (5) weekly report to ICT risk committee; (6) breach of 24h SLA escalates to the Head of IR and Board/Audit Committee.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vm sourcetype IN (tenable:sc:vuln,qualys:host) cvss>=7 earliest=-15m
| rename dest AS hostname
| lookup dora_critical_services.csv hostname OUTPUT ict_service criticality_tier
| where criticality_tier="critical"
| eval registration_sla_met=if(relative_time(now(),"-24h")<=_time,1,0)
| stats count BY ict_service cvss cve hostname registration_sla_met
```
