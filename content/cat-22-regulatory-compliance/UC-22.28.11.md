# 22.28.11 — BSI module compliance tracking

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Demonstrates alignment with BSI IT-Grundschutz expectations for bsi module compliance tracking using auditable Splunk evidence.

## Value

Gives the responsible team auditor-ready evidence for IT-Grundschutz compliance by automating regulatory compliance evidence using indexes grc and sourcetypes grundschutz:module. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=grc sourcetype="grundschutz:module" earliest=-730d
| stats latest(status) as st by module_id, system
| where st IN ("Open","Partial")
| lookup bsi_module_catalog.csv module_id OUTPUT layer
| stats count by layer, st
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Vulnerabilities.Vulnerabilities by Vulnerabilities.severity, Vulnerabilities.signature, Vulnerabilities.dest | sort - count
```

## Implementation

(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners. Indexes required: `index=grc`; Sourcetypes: `sourcetype=grundschutz:module`.
