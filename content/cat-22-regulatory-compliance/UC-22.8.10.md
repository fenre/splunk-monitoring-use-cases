# UC-22.8.10 · SOC 2 CC2 — Ethical Conduct and Acceptable-Use Violation Monitoring

## Description

CC2 addresses integrity and ethical values. Trending acceptable-use policy breaches—harassment keywords, unauthorized software installs, or policy bypass attempts—shows the tone at the top is reinforced with detective controls and corrective action.

## Value

CC2 addresses integrity and ethical values. Trending acceptable-use policy breaches—harassment keywords, unauthorized software installs, or policy bypass attempts—shows the tone at the top is reinforced with detective controls and corrective action.

## Implementation

(1) Align `Operation` filters with HR/legal acceptable-use categories; (2) pseudonymize `UserId` in shared dashboards where required; (3) route high-volume users to People Ops for coaching before escalation; (4) retain 13 months for annual SOC evidence; (5) document disciplinary outcomes separately in HRIS—link only by ticket id in Splunk notes.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365 (Splunkbase 4055), Splunk Add-on for Microsoft Windows (Splunkbase 742).
• Ensure the following data sources are available: `index=o365` `sourcetype="ms:o365:management"` (Operation, Workload, UserId); `index=endpoint` `sourcetype IN ("WinEventLog:Security","crowdstrike:json")` for policy-related detections.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align `Operation` filters with HR/legal acceptable-use categories; (2) pseudonymize `UserId` in shared dashboards where required; (3) route high-volume users to People Ops for coaching before escalation; (4) retain 13 months for annual SOC evidence; (5) document disciplinary outcomes separately in HRIS—link only by ticket id in Splunk notes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=o365 sourcetype="ms:o365:management" Workload IN ("Security","Compliance") earliest=-90d
| where like(Operation,"%policy%") OR like(Operation,"%DLP%")
| stats count as events by UserId, Operation
| eventstats sum(events) as user_total
| where user_total>25
| sort - user_total
```
