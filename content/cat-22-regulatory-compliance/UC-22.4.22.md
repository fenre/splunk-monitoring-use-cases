# UC-22.4.22 · CCPA/CPRA — Cross-Context Behavioral Advertising — SSP Auction Requests After “Limit Use of Sensitive PI” Flag

## Description

Limiting use of sensitive PI for advertising must be operationally enforced. Programmatic auction requests that still attach sensitive audience segments after the limit flag is set indicate pipeline defects with high regulatory salience.

## Value

Gives the responsible team auditor-ready evidence for CCPA/CPRA compliance by automating regulatory compliance evidence using indexes ads, privacy and sourcetypes consent:state, prebid:auction. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Legal defines `segments` taxonomy; (2) strip segment contents in long-term storage—hash or bucketize for Splunk; (3) block supply in ad server when possible; (4) daily compliance digest; (5) vendor notification workflow. Indexes required: `index=ads`, `index=privacy`; Sourcetypes: `sourcetype=consent:state`, `sourcetype=prebid:auction`.

## Detailed Implementation

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Legal defines `segments` taxonomy; (2) strip segment contents in long-term storage—hash or bucketize for Splunk; (3) block supply in ad server when possible; (4) daily compliance digest; (5) vendor notification workflow.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="consent:state" limit_sensitive_ad_use=true earliest=-30d
| eval eff=strptime(updated_ts,"%Y-%m-%dT%H:%M:%SZ")
| join type=left user_id [
    search index=ads sourcetype="prebid:auction" earliest=-30d
    | eval has_sensitive=if(match(segments,"(?i)(health|finance|precise_geo)"),1,0)
    | rename _time as auction_time
    | fields user_id, auction_time, has_sensitive, segments
  ]
| where auction_time>eff AND has_sensitive=1
| stats count by user_id
| sort - count
```
