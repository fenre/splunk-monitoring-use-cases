# 22.13.61 — Control Center Real-Time Assessment TLS Cipher and Certificate Health

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

We monitor TLS handshakes or certificate metadata for inter-control-center assessment links to catch weak ciphers or expiring certificates.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes cert and sourcetypes cert:inventory, cert:inventory`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=cert sourcetype=cert:inventory earliest=-7d
| lookup control_center_links.csv fqdn OUTPUT link_name cip012_scope
| where cip012_scope="true"
| eval exp_epoch=strptime(not_after,"%Y-%m-%d")
| eval days_left=round((exp_epoch-now())/86400,0)
| where days_left < 45 OR match(cipher_suite,"(?i)RC4|DES")
| table link_name, fqdn, days_left, cipher_suite, issuer
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Certificates.All_Certificates by All_Certificates.ssl_subject_common_name, All_Certificates.dest | sort - count
```

## Implementation

(1) Populate inventory from scanners or ACME logs. (2) Alert at forty-five and fourteen days to expiry. (3) Track cipher suite deprecations. (4) Pair with firewall allowlist verification. Indexes required: `index=cert`; Sourcetypes: `sourcetype=cert:inventory`, `sourcetype=cert:inventory``; Lookups: `control_center_links`, `control_center_links.csv`.

## Visualization

Table (at-risk links), Single value (expiring count), Timeline. Key fields: `cipher_suite`, `days_left`, `exp_epoch`, `fqdn`, `issuer`, `link_name`.

## Compliance

- **NERC CIP** current: CIP-012-1 R1 Part 1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-012-1 R1 Part 1.1.

## Known False Positives
