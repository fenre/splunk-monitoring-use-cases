# 22.13.32 — Malware Detection Events on BES Cyber Systems

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We centralize detected-malicious-code events from BES assets so alerts and evidence meet system security management expectations.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes endpoint and sourcetypes WinDefender:Threat, ms:defender:alert. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=endpoint (sourcetype=WinDefender:Threat OR sourcetype="ms:defender:alert") earliest=-7d
| lookup bes_cyber_asset_inventory.csv host OUTPUT bes_system impact_rating
| where isnotnull(bes_system)
| stats count earliest(_time) as first latest(_time) as last by host, threat_name, action
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Malware.Malware_Attacks by Malware_Attacks.dest, Malware_Attacks.signature, Malware_Attacks.action | sort - count
```

## Implementation

(1) Fix sourcetype list to your EDR. (2) Map actions to blocked vs quarantined. (3) Create ES notable with CIP-008 link. (4) Retain chain-of-custody fields if exported. Indexes required: `index=endpoint`; Lookups: `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.
