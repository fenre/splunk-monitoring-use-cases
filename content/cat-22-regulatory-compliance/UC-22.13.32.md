# UC-22.13.32 · Malware Detection Events on BES Cyber Systems

## Description

We centralize detected-malicious-code events from BES assets so alerts and evidence meet system security management expectations.

## Value

We centralize detected-malicious-code events from BES assets so alerts and evidence meet system security management expectations.

## Implementation

(1) Fix sourcetype list to your EDR. (2) Map actions to blocked vs quarantined. (3) Create ES notable with CIP-008 link. (4) Retain chain-of-custody fields if exported.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=endpoint` Defender or EDR sourcetypes, `bes_cyber_asset_inventory.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Fix sourcetype list to your EDR. (2) Map actions to blocked vs quarantined. (3) Create ES notable with CIP-008 link. (4) Retain chain-of-custody fields if exported.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=endpoint (sourcetype=WinDefender:Threat OR sourcetype="ms:defender:alert") earliest=-7d
| lookup bes_cyber_asset_inventory.csv host OUTPUT bes_system impact_rating
| where isnotnull(bes_system)
| stats count earliest(_time) as first latest(_time) as last by host, threat_name, action
| sort - count
```
