# UC-22.8.22 · SOC 2 CC7 — Operational Anomaly Detection on Critical Batch and API SLOs

## Description

CC7 includes monitoring for anomalies. Statistical deviation on transaction volumes, error rates, or job durations for financially relevant pipelines shows detective monitoring beyond static thresholds—supporting availability and processing-integrity narratives.

## Value

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk IT Service Intelligence (Splunkbase 1841) or MLTK.
• Ensure the following data sources are available: `index=app` `sourcetype="api:gateway"` (service, status, latency_ms, _time); or `sourcetype="batch:summary"` (job_name, duration_sec, records_out).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Scope `service` to in-scope subsystems; (2) tune z-score window; (3) suppress known marketing events via lookup; (4) feed notable to on-call; (5) document model refresh cadence for auditors.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=app sourcetype="api:gateway" service="payments_api" earliest=-60d
| timechart span=1h perc95(latency_ms) as p95_lat, sum(eval(if(status>=500,1,0))) as err, count as vol
| eventstats median(vol) as med_vol, stdev(vol) as sd_vol
| eval z=if(sd_vol>0, (vol-med_vol)/sd_vol, 0)
| where abs(z)>4
| table _time, vol, med_vol, z, p95_lat, err
```
