# UC-22.4.6 · CCPA Global Privacy Control and “Do Not Sell or Share” Signal Enforcement

## Description

Opt-out preference signals including GPC must be honored where required. Aggregating API-side opt-out application outcomes demonstrates that technical controls downstream of the browser actually suppress sale/share processing, not only that marketing pages were visited.

## Value

Gives the responsible team auditor-ready evidence for CCPA/CPRA compliance by automating regulatory compliance evidence using indexes privacy and sourcetypes _json. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Emit one event per profile/channel when consent middleware finishes applying opt-out; (2) set `gpc_received` from upstream headers; (3) alert if `fail_rate>1` percent for any `channel`; (4) join failures to application logs via `profile_id`; (5) document rollback procedures for bad releases. Indexes required: `index=privacy`; Sourcetypes: `sourcetype=_json`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk HTTP Event Collector (core platform).
• Ensure the following data sources are available: `index=privacy` `sourcetype="_json"` `source="http:ccpa_optout_apply"` (profile_id, channel, gpc_received, opt_out_applied, failure_reason, _time).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Emit one event per profile/channel when consent middleware finishes applying opt-out; (2) set `gpc_received` from upstream headers; (3) alert if `fail_rate>1` percent for any `channel`; (4) join failures to application logs via `profile_id`; (5) document rollback procedures for bad releases.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="_json" source="http:ccpa_optout_apply" earliest=-24h
| eval failed=if(opt_out_applied="false" OR isnotnull(failure_reason), 1, 0)
| stats count as attempts, sum(failed) as failures, sum(eval(gpc_received="true")) as gpc_context by channel
| eval fail_rate=round(100*failures/attempts, 2)
| sort - failures
```
