# 22.13.56 — Baseline Update Documentation — Config Baseline Version Changes

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We audit when official configuration baseline versions change and require an authorizing record for each update.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes grc and sourcetypes cip010_baseline_version, cip010_baseline_version`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=grc sourcetype=cip010_baseline_version earliest=-365d
| sort 0 - _time
| streamstats current=f latest(baseline_version) as prev_version by platform
| where baseline_version!=prev_version
| eval has_auth=if(isnotnull(auth_record_id),1,0)
| where has_auth=0
| table _time, platform, baseline_version, prev_version, owner, auth_record_id
```

## Implementation

(1) Emit version bump events from CMDB or GitOps. (2) Require `auth_record_id` from CAB approval. (3) Alert compliance on missing linkage. (4) Archive prior baselines offline. Indexes required: `index=grc`; Sourcetypes: `sourcetype=cip010_baseline_version`, `sourcetype=cip010_baseline_version``.

## Visualization

Timeline (version bumps), Table (missing auth), Single value (violations). Key fields: `auth_record_id`, `baseline_version`, `has_auth`, `owner`, `platform`, `prev_version`.

## Compliance

- **NERC CIP** current: CIP-010-4 R1 Part 1.3 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-010-4 R1 Part 1.3.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
