# 22.15.35 — Mixed Safety and Non-Safety Traffic on Same VLAN Evidence

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

Correlates asset roles from inventory with L2/L3 observations to highlight application partitioning violations between safety and non-safety systems.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_assets, ot_net and sourcetypes claroty:asset, nozomi:assets. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_net sourcetype="claroty:asset" OR sourcetype="nozomi:assets" earliest=-7d
| eval vlan=vlan_id
| join type=left name [ search index=ot_assets | rename asset_name as name | fields name, safety_related, vlan_expected ]
| where safety_related="true" AND vlan!=vlan_expected
| stats values(vlan) as vlans_seen by name, safety_related
```

## Implementation

(1) Tag SIS vs BPCS in CMDB; (2) enforce separate VLANs; (3) migration plan for mixed assets; (4) document compensating controls if any. Indexes required: `index=ot_assets`, `index=ot_net`; Sourcetypes: `sourcetype=claroty:asset`, `sourcetype=nozomi:assets`.

## Visualization

Table, Diagram export, Single value (violations). Key fields: `name`, `safety_related`, `vlan`, `vlans_seen`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 5.4 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 5.4.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
