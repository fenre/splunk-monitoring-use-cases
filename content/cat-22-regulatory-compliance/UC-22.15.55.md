# 22.15.55 — Zone Trust Level Verification — SL-T vs Observed Purdue Layer

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Joins SL-T/Criticality from inventory to observed Purdue level to find assets operating below required trust level for their criticality.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_assets, ot_topology. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_assets OR index=ot_topology earliest=-1d
| eval id=coalesce(asset_id, device_id)
| stats latest(sl_t) as required_tier, latest(observed_purdue_level) as observed by id
| eval mismatch=case(required_tier="SL-3" AND observed>2,1, required_tier="SL-2" AND observed>3,1, true(),0)
| where mismatch=1
| table id, required_tier, observed
```

## Implementation

(1) Refresh topology weekly from IDS; (2) reconcile moves with CMDB; (3) migration plan to correct zone; (4) risk acceptance workflow for exceptions. Indexes required: `index=ot_assets`, `index=ot_topology`.

## Visualization

Table, Scatter (tier vs level), Single value (count). Key fields: `id`, `mismatch`, `observed`, `required_tier`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
