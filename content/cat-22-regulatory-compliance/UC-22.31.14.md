# UC-22.31.14 · Information security roles and responsibilities attestations

## Description

Evidences APRA CPS 234 obligations for information security roles and responsibilities attestations using endpoint, identity, and cloud telemetry.

## Value

Gives the responsible team auditor-ready evidence for APRA CPS 234 compliance by automating regulatory compliance evidence using indexes audit and sourcetypes cps234:test. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ingest CrowdStrike policy posture via TA 5082 and map hosts to NDB and NZISM tiers. (2) Tune alerts with ACSD Essential Eight and APRA-aligned severities. (3) Coordinate privacy and security ownership for dual-reporting workflows. (4) Monthly export for privacy officer and prudential working groups. Indexes required: `index=audit`; Sourcetypes: `sourcetype=cps234:test`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742), Splunk Add-on for CrowdStrike FDR (5082), Splunk ITSI (1841), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: Windows Security Event Logs, endpoint protection logs, cloud audit trails, vulnerability scan outputs, email security logs, network infrastructure logs, identity management systems, incident response records.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=audit sourcetype="cps234:test" control_family="Roles" earliest=-180d
| stats latest(result) as r by control_id, third_party
| where r!="Pass"
```
