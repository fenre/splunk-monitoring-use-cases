# UC-22.14.73 · Protection of Information at Rest — Storage Encryption Misconfigurations

## Description

Finds newly created volumes or buckets without encryption-at-rest enabled for SC-28 protection of information at rest.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes cloud and sourcetypes aws:cloudtrail. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Tune regex to your JSON shape; (2) auto-open CM-3 change if intentional; (3) integrate AWS Config rules for continuous check; (4) evidence for assessor sampling. Indexes required: `index=cloud`; Sourcetypes: `sourcetype=aws:cloudtrail`.

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cloud sourcetype=aws:cloudtrail earliest=-7d eventName IN ("CreateVolume","CreateBucket")
| spath path=responseElements output=responseElements
| regex responseElements="(?i)encrypted.?false|Encryption:?None|null"
| stats count by userName, eventName, aws_account_id
```
