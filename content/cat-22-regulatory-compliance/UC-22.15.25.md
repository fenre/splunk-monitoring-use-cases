# 22.15.25 — Clock Skew on OT Hosts vs NTP Stratum

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Surfaces OT servers and RTUs with large `_indextime` vs `_time` deltas undermining trustworthy audit timestamps.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_infra. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_infra earliest=-24h
| eval skew=abs(_indextime-_time)
| where skew>120
| stats max(skew) as max_skew, perc95(skew) as p95_skew by host, sourcetype
| sort - p95_skew
```

## Implementation

(1) Enforce PTP/NTP on all IEDs; (2) fix dual-homed syslog paths; (3) alert >120s; (4) document stratum hierarchy per site. Indexes required: `index=ot_infra`.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
