# UC-22.15.37 · Unauthorized Firmware Upload Attempt on RTU

## Description

Detects firmware or OS image transfer sessions to RTUs from non-engineering hosts, testing authorization enforcement on components.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes fw, ot_sec and sourcetypes claroty:alert, pan:traffic. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Block firmware protocols except via jump host; (2) require signed images; (3) alert any ftp to field; (4) integrate vendor secure boot logs if available. Indexes required: `index=fw`, `index=ot_sec`; Sourcetypes: `sourcetype=claroty:alert`, `sourcetype=pan:traffic`; Lookups: `ot_engineering_nets`, `ot_engineering_nets.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Dragos/Claroty alerts + firewall.
• Ensure the following data sources are available: `claroty:alert` category=firmware; `pan:traffic` ftp/scp.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Block firmware protocols except via jump host; (2) require signed images; (3) alert any ftp to field; (4) integrate vendor secure boot logs if available.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_sec sourcetype="claroty:alert" category="firmware" earliest=-14d
| append [ search index=fw sourcetype="pan:traffic" app IN ("ftp","scp","sftp") dest_zone="OT-RTU" earliest=-14d
    | lookup ot_engineering_nets.csv src OUTPUT role | where role!="ews" ]
| stats earliest(_time) as t0, values(src) as sources by dest, signature, sourcetype
| sort - t0
```
