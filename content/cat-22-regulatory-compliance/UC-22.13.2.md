# 22.13.2 — BES Cyber System Impact Rating Drift Detection

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We highlight when operational tags on hosts disagree with the registered high or medium impact rating so impact determinations stay defensible.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes ot_security and sourcetypes ot:asset. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
| inputlookup bes_cyber_system_categorization.csv
| eval key=lower(bes_asset_hostname)
| join type=left key [
    search index=ot_security sourcetype="ot:asset" earliest=-24h
    | eval key=lower(hostname)
    | stats latest(impact_tag) as observed_impact by key
  ]
| where lower(observed_impact)!=lower(impact_rating)
| table bes_system, impact_rating, observed_impact, key
```

## Implementation

(1) Populate `impact_tag` from OT Security Add-on or CMDB sync. (2) Tune mapping for staging assets. (3) Monthly governance report. (4) Drive formal CIP-002 review updates when drift persists. Indexes required: `index=ot_security`; Sourcetypes: `sourcetype=ot:asset`; Lookups: `bes_cyber_system_categorization`, `bes_cyber_system_categorization.csv`.

## Visualization

Table (mismatches), Single value (drift count), Column chart (by site). Key fields: `bes_system`, `impact_rating`, `key`, `observed_impact`.

## Compliance

- **NERC CIP** current: CIP-002-6 R2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-002-6 R2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
