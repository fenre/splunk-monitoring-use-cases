# UC-22.13.2 · BES Cyber System Impact Rating Drift Detection

## Description

We highlight when operational tags on hosts disagree with the registered high or medium impact rating so impact determinations stay defensible.

## Value

We highlight when operational tags on hosts disagree with the registered high or medium impact rating so impact determinations stay defensible.

## Implementation

(1) Populate `impact_tag` from OT Security Add-on or CMDB sync. (2) Tune mapping for staging assets. (3) Monthly governance report. (4) Drive formal CIP-002 review updates when drift persists.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk OT Security Add-on (5151).
• Ensure the following data sources are available: `index=ot_security` asset records, `bes_cyber_system_categorization.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate `impact_tag` from OT Security Add-on or CMDB sync. (2) Tune mapping for staging assets. (3) Monthly governance report. (4) Drive formal CIP-002 review updates when drift persists.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup bes_cyber_system_categorization.csv
| eval key=lower(bes_asset_hostname)
| join type=left key [
    search index=ot_security sourcetype="ot:asset" earliest=-24h
    | eval key=lower(hostname)
    | stats latest(impact_tag) as observed_impact by key
  ]
| where lower(observed_impact)!=lower(impact_rating)
| table bes_system, impact_rating, observed_impact, key
```
