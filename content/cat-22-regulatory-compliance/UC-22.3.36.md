# UC-22.3.36 · DORA Exit Strategy — Data Portability Test Evidence and Export Volume Integrity

## Description

Exiting a provider is impossible without data. Logging export job byte counts, checksums, and completion times for annual portability drills evidences technical feasibility of contractual exit clauses.

## Value

Gives the responsible team auditor-ready evidence for DORA compliance by automating regulatory compliance evidence using indexes dataops and sourcetypes exit:export_baseline, exit:export_test. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Establish baselines after first full export; (2) run tests in non-prod mirror where possible; (3) classify logs without customer payloads; (4) store checksum manifests in WORM; (5) map to exit playbook section IDs. Indexes required: `index=dataops`; Sourcetypes: `sourcetype=exit:export_baseline`, `sourcetype=exit:export_test`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=dataops sourcetype="exit:export_test" earliest=-400d
| stats sum(bytes_out) as total_bytes, max(duration_sec) as max_dur by provider_id, test_id
| join type=left provider_id [
    search index=dataops sourcetype="exit:export_baseline" earliest=-400d
    | fields provider_id, expected_min_bytes
  ]
| where total_bytes < expected_min_bytes*0.95
| table provider_id, test_id, total_bytes, expected_min_bytes, max_dur
```
