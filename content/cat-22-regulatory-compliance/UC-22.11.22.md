# UC-22.11.22 · TLS 1.2 Minimum Version Violations on Payment APIs

## Description

Surfaces handshakes negotiated below TLS 1.2 on in-scope listeners so weak transmission protections are remediated before external vulnerability scanning deadlines.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes aws, stream and sourcetypes aws:elb:accesslogs, stream:tls. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize TLS version strings per data source. (2) Map `dest_ip` to payment VLANs. (3) Exclude lab subnets via lookup. (4) Feed to vulnerability management for ASV correlation. Indexes required: `index=aws`, `index=stream`; Sourcetypes: `sourcetype=aws:elb:accesslogs`, `sourcetype=stream:tls`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Stream (1809), Splunk Add-on for AWS (1876).
• Ensure the following data sources are available: `index=stream` `sourcetype="stream:tls"`; `index=aws` `sourcetype="aws:elb:accesslogs"` with `ssl_protocol`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize TLS version strings per data source. (2) Map `dest_ip` to payment VLANs. (3) Exclude lab subnets via lookup. (4) Feed to vulnerability management for ASV correlation.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
(index=stream sourcetype="stream:tls" earliest=-24h)
 OR (index=aws sourcetype="aws:elb:accesslogs" earliest=-24h)
| eval tls_ver=coalesce(tls_version, ssl_protocol)
| lookup pci_payment_endpoints.csv dest_ip OUTPUT in_scope
| where in_scope="true"
| where tls_ver IN ("TLSv1","TLSv1.0","TLSv1.1","SSLv3") OR match(tls_ver,"(?i)^ssl")
| stats count by tls_ver, dest_ip, src_ip, server_name
| sort - count
```
