# 22.21.8 — Training Data Lineage Event Completeness (Art. 13)

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Validates dataset_id, snapshot_hash, and license_tag on each training run completion for dataset lineage traceability.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ml_train and sourcetypes kubeflow:run, kubeflow:run`, mlflow:run. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ml_train (sourcetype="kubeflow:run" OR sourcetype="mlflow:run") event_type="completed" earliest=-7d
| eval lineage_ok=if(isnotnull(dataset_id) AND isnotnull(snapshot_hash) AND isnotnull(license_tag),1,0)
| stats count as runs, sum(lineage_ok) as ok by experiment, pipeline
| eval ok_pct=round(100*ok/runs,2)
| where ok_pct < 100
| sort + ok_pct
```

## Implementation

(1) Add lineage fields at pipeline DSL; (2) join catalog table via DB Connect for license_tag validation; (3) block promotion if lineage_ok=0; (4) export monthly compliance CSV. Indexes required: `index=ml_train`; Sourcetypes: `sourcetype=kubeflow:run`, `sourcetype=kubeflow:run``, `sourcetype=mlflow:run`, `sourcetype=mlflow:run``.

## Visualization

Bar chart (ok_pct by pipeline), Table (failed runs), Single value (runs missing license_tag)
