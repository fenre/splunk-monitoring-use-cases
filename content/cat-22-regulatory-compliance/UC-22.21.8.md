# UC-22.21.8 · Training Data Lineage Event Completeness (Art. 13)

## Description

Validates dataset_id, snapshot_hash, and license_tag on each training run completion for dataset lineage traceability.

## Value

Validates dataset_id, snapshot_hash, and license_tag on each training run completion for dataset lineage traceability.

## Implementation

(1) Add lineage fields at pipeline DSL; (2) join catalog table via DB Connect for license_tag validation; (3) block promotion if lineage_ok=0; (4) export monthly compliance CSV.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686), Splunk Machine Learning Toolkit (2890).
• Ensure the following data sources are available: `index=ml_train` `sourcetype=kubeflow:run` OR `sourcetype=mlflow:run`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Add lineage fields at pipeline DSL; (2) join catalog table via DB Connect for license_tag validation; (3) block promotion if lineage_ok=0; (4) export monthly compliance CSV.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ml_train (sourcetype="kubeflow:run" OR sourcetype="mlflow:run") event_type="completed" earliest=-7d
| eval lineage_ok=if(isnotnull(dataset_id) AND isnotnull(snapshot_hash) AND isnotnull(license_tag),1,0)
| stats count as runs, sum(lineage_ok) as ok by experiment, pipeline
| eval ok_pct=round(100*ok/runs,2)
| where ok_pct < 100
| sort + ok_pct
```
