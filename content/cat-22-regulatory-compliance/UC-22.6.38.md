# UC-22.6.38 · Logging — Forwarder Stopped or CrashLoop on Security-Relevant Hosts

## Description

Detects universal forwarder or agent stopped states on in-scope systems so logging obligations stay met.

## Value

Detects universal forwarder or agent stopped states on in-scope systems so logging obligations stay met.

## Implementation

(1) Maintain tier lookup; (2) auto-restart playbook; (3) ISO evidence for logging availability.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk `_internal`.
• Ensure the following data sources are available: `splunkd` logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain tier lookup; (2) auto-restart playbook; (3) ISO evidence for logging availability.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal source=*splunkd.log* earliest=-24h
| rex field=_raw "(?i)Stopping all processes|crashed"
| stats count by host
| lookup security_tier_hosts.csv host OUTPUT tier
| where tier IN ("tier0","tier1")
```
