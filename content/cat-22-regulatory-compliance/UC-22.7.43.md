# UC-22.7.43 · Executive Paging Lag After Sev-1 Playbook Start

## Description

Verifies paging actions fired within policy window after high-severity SOAR playbook start.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes soar and sourcetypes phantom:action_run, phantom:playbook_run. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Standardize severity; (2) test quarterly; (3) document comms tree. Indexes required: `index=soar`; Sourcetypes: `sourcetype=phantom:action_run`, `sourcetype=phantom:playbook_run`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk SOAR, PagerDuty.
• Ensure the following data sources are available: `phantom:action_run`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Standardize severity; (2) test quarterly; (3) document comms tree.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=soar sourcetype=phantom:action_run earliest=-90d action="send pager"
| stats earliest(_time) as page_time by playbook_run_id
| join playbook_run_id [| search index=soar sourcetype=phantom:playbook_run severity=1 | rename _time as pb_time | fields playbook_run_id, pb_time]
| eval lag_sec=page_time-pb_time
| where lag_sec>1800
```
