# UC-22.11.1 · Scheduled Firewall Rule Review Evidence for CDE NSCs

## Description

Produces a repeatable inventory of firewall policy changes touching the cardholder data environment so assessors can see that network security controls are formally reviewed at least every six months.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes netfw and sourcetypes cisco:asa, pan:config, pan:system. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain `pci_cde_device_groups.csv` mapping Panorama device groups or firewall hostnames to `in_cde`. (2) Align the search window to your documented six-month review cycle. (3) Export the stats table monthly and attach the export ID to the GRC ticket. (4) Exclude template pushes that do not alter enforced policy using a `commit_id` dedupe if available. Indexes required: `index=netfw`; Sourcetypes: `sourcetype=cisco:asa`, `sourcetype=pan:config`, `sourcetype=pan:system`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=netfw` `sourcetype IN ("pan:config","pan:system")` — commits, admin user, rule object names; optional `sourcetype="cisco:asa"` configuration events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `pci_cde_device_groups.csv` mapping Panorama device groups or firewall hostnames to `in_cde`. (2) Align the search window to your documented six-month review cycle. (3) Export the stats table monthly and attach the export ID to the GRC ticket. (4) Exclude template pushes that do not alter enforced policy using a `commit_id` dedupe if available.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=netfw sourcetype IN ("pan:config","pan:system") earliest=-185d
| where match(_raw,"(?i)commit|policy|rule|security|nat")
| lookup pci_cde_device_groups.csv device_group OUTPUT in_cde
| where in_cde="true"
| rex field=_raw "(?i)admin@?(?<fw_admin>\\S+)"
| bin _time span=30d
| stats values(fw_admin) as reviewers, dc(_raw) as change_events by _time, host
| sort - _time
```
