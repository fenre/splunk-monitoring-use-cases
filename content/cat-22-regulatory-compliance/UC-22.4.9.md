# UC-22.4.9 ¬∑ CCPA/CPRA ‚Äî Sensitive PI ‚Äî Precise Geolocation Collection Stop Signal After User Opt-Out

## Description

Sensitive personal information includes precise geolocation. Detecting mobile or web telemetry that continues after an opt-out timestamp demonstrates downstream systems honor consumer choices and reduces enforcement exposure under ¬ß1798.121.

## Value

Gives the responsible team auditor-ready evidence for CCPA/CPRA compliance by automating regulatory compliance evidence using indexes app, privacy and sourcetypes consent:state, mobile:location_event. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align `accuracy_m` threshold with your ‚Äúprecise‚Äù definition; (2) hash `profile_id` in shared dashboards if required; (3) route to privacy engineering for pipeline fixes; (4) document lawful exceptions (e.g., emergency services) in a lookup; (5) retain evidence per records schedule. Indexes required: `index=app`, `index=privacy`; Sourcetypes: `sourcetype=consent:state`, `sourcetype=mobile:location_event`.

Step 1 ‚Äî Configure data collection
(1) Align `accuracy_m` threshold with your ‚Äúprecise‚Äù definition; (2) hash `profile_id` in shared dashboards if required; (3) route to privacy engineering for pipeline fixes; (4) document lawful exceptions (e.g., emergency services) in a lookup; (5) retain evidence per records schedule.

Step 2 ‚Äî Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="consent:state" geo_collection_allowed=false earliest=-30d
| eval opt=strptime(updated_ts,"%Y-%m-%dT%H:%M:%SZ")
| join type=left profile_id [
    search index=app sourcetype="mobile:location_event" earliest=-30d
    | where accuracy_m<=50
    | fields profile_id, _time, accuracy_m
  ]
| where _time>opt
| stats count by profile_id, accuracy_m
| sort - count
```
