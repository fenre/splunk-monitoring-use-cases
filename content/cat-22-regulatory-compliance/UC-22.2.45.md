# UC-22.2.45 · NIS2 CSIRT Notification Compliance and Channel Health

## Description

Notifications must reach the competent CSIRT through approved channels. This use case monitors successful delivery of structured incident notifications (webhook/API/email handoff), template versioning, and cryptographic signing validation where used — reducing silent failures at the last mile.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes itsm, reg_notify and sourcetypes csirt:delivery. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Instrument notification orchestrator to emit delivery events; (2) version PDF/XML templates per national CSIRT schema updates; (3) alert legal/IR on any non-2xx response or signature validation failure; (4) test channels quarterly with synthetic non-production payloads; (5) correlate with UC-22.2.33 deadlines. Indexes required: `index=itsm`, `index=reg_notify`; Sourcetypes: `sourcetype=csirt:delivery`; Lookups: `nis2_csirt_endpoint_catalog`, `nis2_csirt_endpoint_catalog.csv`.

Step 1 — Configure data collection
(1) Instrument notification orchestrator to emit delivery events; (2) version PDF/XML templates per national CSIRT schema updates; (3) alert legal/IR on any non-2xx response or signature validation failure; (4) test channels quarterly with synthetic non-production payloads; (5) correlate with UC-22.2.33 deadlines.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=reg_notify sourcetype="csirt:delivery" earliest=-90d
| lookup nis2_csirt_endpoint_catalog.csv csirt_id OUTPUT required_template_version
| eval template_ok=if(template_version=required_template_version,1,0)
| eval delivery_ok=if(match(lower(delivery_status),"success|accepted|202"),1,0)
| where delivery_ok=0 OR template_ok=0
| stats count by csirt_id delivery_status template_version required_template_version
| sort - count
```
