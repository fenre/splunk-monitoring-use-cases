# UC-22.13.49 · Baseline Configuration Deviation — Key OS and Application Settings

## Description

We compare configuration snapshots from servers and appliances to approved baselines so unauthorized drift on BES Cyber Assets is visible.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes config and sourcetypes config:snapshot, config:snapshot`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Collect snapshots via scripted input or Chef/Ansible exports. (2) Version baselines with change tickets. (3) Alert on critical settings only first. (4) Feed drift into change review board. Indexes required: `index=config`; Sourcetypes: `sourcetype=config:snapshot`, `sourcetype=config:snapshot``; Lookups: `bes_config_baseline`, `bes_config_baseline.csv`, `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=config sourcetype=config:snapshot earliest=-24h
| lookup bes_cyber_asset_inventory.csv host OUTPUT bes_system
| where isnotnull(bes_system)
| lookup bes_config_baseline.csv host, setting_name OUTPUT expected_value
| where expected_value!=current_value
| table host, setting_name, expected_value, current_value, bes_system
```
