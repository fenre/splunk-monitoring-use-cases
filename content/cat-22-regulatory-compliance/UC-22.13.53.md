# 22.13.53 — Transient Cyber Asset Connection Logging to ESP Jump Zones

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We log and summarize laptops or vendor devices connecting transiently inside ESP jump zones for transient cyber asset visibility.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes network and sourcetypes pan:traffic, pan:traffic`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=network sourcetype=pan:traffic dest_zone="ESP_JUMP" earliest=-24h
| eval tca=if(match(src,"dhcp_pool"),1,0)
| lookup dhcp_leases.csv mac AS mac OUTPUT hostname
| stats earliest(_time) as first latest(_time) as last sum(bytes) as bytes by src, hostname, user
| sort - last
```

## CIM SPL

```spl
| tstats summariesonly=t sum(All_Traffic.bytes_in) as agg_value from datamodel=Network_Traffic.All_Traffic by All_Traffic.src, All_Traffic.dest | sort - agg_value
```

## Implementation

(1) Correlate DHCP MAC to user if available. (2) Maintain TCA register with expected devices. (3) Alert on unknown TCA over one hour. (4) Scan-before-connect workflow tracked separately. Indexes required: `index=network`; Sourcetypes: `sourcetype=pan:traffic`, `sourcetype=pan:traffic``; Lookups: `dhcp_leases`, `dhcp_leases.csv`.

## Visualization

Table (TCA sessions), Timeline, Map (optional). Key fields: `bytes`, `first`, `hostname`, `last`, `src`, `tca`.

## Compliance

- **NERC CIP** current: CIP-010-4 R5 Part 5.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-010-4 R5 Part 5.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
