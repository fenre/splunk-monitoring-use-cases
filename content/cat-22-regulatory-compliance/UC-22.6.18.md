# UC-22.6.18 · Security Awareness Completion Rate by Department

## Description

Aggregates LMS training completions tagged as security awareness for workforce coverage metrics.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes hr and sourcetypes lms:completion. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Refresh department from HR nightly; (2) escalate managers below threshold; (3) document exceptions. Indexes required: `index=hr`; Sourcetypes: `sourcetype=lms:completion`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: LMS CSV or Workday Learning HEC.
• Ensure the following data sources are available: `lms:completion`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Refresh department from HR nightly; (2) escalate managers below threshold; (3) document exceptions.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr sourcetype=lms:completion course_tag=security_awareness earliest=-365d
| stats latest(status) as last_status by user_id, department
| eval complete=if(last_status="passed",1,0)
| stats sum(complete) as passed, count as enrolled by department
| eval pct=round(100*passed/enrolled,1)
| where pct<95
```
