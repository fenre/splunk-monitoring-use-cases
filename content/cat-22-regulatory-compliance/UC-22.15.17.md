# UC-22.15.17 · OT Access from Untrusted Networks Without Split Tunnel Block

## Description

Flags VPN or agent sessions originating from high-risk geos or residential ASNs while accessing OT jump portals.

## Value

Flags VPN or agent sessions originating from high-risk geos or residential ASNs while accessing OT jump portals.

## Implementation

(1) Maintain country allowlist aligned to business; (2) require hardware token from untrusted regions; (3) integrate threat intel ASN feed; (4) auto-disable accounts on repeat hits.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757).
• Ensure the following data sources are available: `pan:globalprotect` with `public_ip`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain country allowlist aligned to business; (2) require hardware token from untrusted regions; (3) integrate threat intel ASN feed; (4) auto-disable accounts on repeat hits.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="pan:globalprotect" earliest=-7d
| iplocation public_ip
| lookup ot_trusted_country.csv Country OUTPUT trust
| where trust!="trusted" OR like(ASN,"*RESIDENTIAL*")
| lookup ot_jump_portals.csv portal OUTPUT sl_t
| stats earliest(_time) as first, latest(_time) as last, values(public_ip) as ips by user, portal, Country
| sort - last
```
