# 22.28.17 — IT operations governance evidence

**Criticality:** medium | **Difficulty:** expert | **Status:** N/A

## Description

Demonstrates alignment with BAIT/KAIT expectations for it operations governance evidence using auditable Splunk evidence.

## Value

Gives the responsible team auditor-ready evidence for BAIT/KAIT compliance by automating regulatory compliance evidence using indexes itsi_summary, ops and sourcetypes linux:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=itsi_summary OR index=ops sourcetype="linux:audit" earliest=-24h
| stats count by host, sourcetype
| lookup bait_it_ops_scope.csv host OUTPUT critical
| where critical="true" AND count<10
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Vulnerabilities.Vulnerabilities by Vulnerabilities.dest | sort - count
```

## Implementation

(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners. Indexes required: `index=itsi_summary`, `index=ops`; Sourcetypes: `sourcetype=linux:audit`.
