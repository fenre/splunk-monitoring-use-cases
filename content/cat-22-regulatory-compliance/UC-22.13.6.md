# 22.13.6 — Cyber Security Plan Technical Control Attestation — MFA for Privileged Access

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

We compare multifactor authentication signals for privileged BES accounts against the cyber security plan so documented controls match authentication reality.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode=4624 earliest=-24h
| lookup bes_privileged_users.csv Account_Name AS user OUTPUT requires_mfa bes_system
| where requires_mfa="true"
| eval strong_pkg=if(match(AuthenticationPackageName,"(?i)Negotiate|Kerberos") AND LogonType IN ("10","3"),1,0)
| stats count as logons sum(strong_pkg) as strong_signals by user, bes_system
| eval ratio=round(strong_signals/logons,3)
| where ratio < 0.9
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.user | sort - count
```
