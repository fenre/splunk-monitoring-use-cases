# UC-22.15.39 · Secure Boot or Image Hash Mismatch on Industrial Appliance

## Description

Correlates component attestation failures with subsequent process alarms to prioritize firmware integrity incidents.

## Value

Correlates component attestation failures with subsequent process alarms to prioritize firmware integrity incidents.

## Implementation

(1) Collect TPM/UEFI events from industrial gateways; (2) isolate device on mismatch; (3) compare to gold hash registry; (4) vendor RMA workflow.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HEC from appliance vendor or Edge Hub.
• Ensure the following data sources are available: `index=ot_edge` `sourcetype="edge:attest"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Collect TPM/UEFI events from industrial gateways; (2) isolate device on mismatch; (3) compare to gold hash registry; (4) vendor RMA workflow.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_edge sourcetype="edge:attest" earliest=-7d
| where result!="OK" OR match(_raw,"(?i)hash mismatch|secure boot fail")
| join type=left device_id [ search index=ot_hist sourcetype="hist:alarm" earliest=-7d | stats count as alarms by device_id ]
| table _time, device_id, result, alarms
```
