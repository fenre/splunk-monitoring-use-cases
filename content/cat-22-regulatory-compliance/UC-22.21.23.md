# UC-22.21.23 · Model Withdrawal / Recall Execution Audit (Art. 12/72)

## Description

Verifies recall_command acknowledged by each edge node within SLA for model withdrawal evidence.

## Value

Verifies recall_command acknowledged by each edge node within SLA for model withdrawal evidence.

## Implementation

(1) Maintain authoritative node inventory; (2) alert pct < 100 after 4h; (3) ITSI episode per recall_id; (4) store signed ack externally.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI (1841), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=ai_edge` `sourcetype=ai:recall:ack` — recall_id, node_id, ack_time.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain authoritative node inventory; (2) alert pct < 100 after 4h; (3) ITSI episode per recall_id; (4) store signed ack externally.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ai_edge sourcetype="ai:recall:ack" earliest=-30d
| stats min(ack_time) as first_ack, dc(node_id) as nodes_acked by recall_id
| join type=left recall_id [
    search index=ai_edge sourcetype="ai:recall:target" earliest=-30d
    | stats dc(node_id) as nodes_total by recall_id
  ]
| eval pct=round(100*nodes_acked/nodes_total,2)
| where pct < 100
| table recall_id, nodes_acked, nodes_total, pct, first_ack
```
