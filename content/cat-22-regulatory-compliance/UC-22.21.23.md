# 22.21.23 — Model Withdrawal / Recall Execution Audit (Art. 12/72)

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Verifies recall_command acknowledged by each edge node within SLA for model withdrawal evidence.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_edge and sourcetypes ai:recall:ack, ai:recall:ack`, ai:recall:target. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_edge sourcetype="ai:recall:ack" earliest=-30d
| stats min(ack_time) as first_ack, dc(node_id) as nodes_acked by recall_id
| join type=left recall_id [
    search index=ai_edge sourcetype="ai:recall:target" earliest=-30d
    | stats dc(node_id) as nodes_total by recall_id
  ]
| eval pct=round(100*nodes_acked/nodes_total,2)
| where pct < 100
| table recall_id, nodes_acked, nodes_total, pct, first_ack
```

## Implementation

(1) Maintain authoritative node inventory; (2) alert pct < 100 after 4h; (3) ITSI episode per recall_id; (4) store signed ack externally. Indexes required: `index=ai_edge`; Sourcetypes: `sourcetype=ai:recall:ack`, `sourcetype=ai:recall:ack``, `sourcetype=ai:recall:target`.
