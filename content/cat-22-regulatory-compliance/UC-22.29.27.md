# UC-22.29.27 · Opt-in and opt-out preference tracking

## Description

Provides APAC privacy supervisory evidence for opt-in and opt-out preference tracking aligned with PIPL; PDPA.

## Value

Provides APAC privacy supervisory evidence for opt-in and opt-out preference tracking aligned with PIPL; PDPA.

## Implementation

(1) Maintain `apac_transfer_register.csv` and consent or DPIA lookups referenced in runbooks. (2) Map technical identifiers to records of processing activities with legal review. (3) Restrict indexes to regional privacy roles with least privilege. (4) Quarterly DPIA and consent linkage review with local counsel.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742), Splunk Add-on for Microsoft Office 365 (4055), Splunk Add-on for AWS (1876), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: Windows Security Event Logs, cloud audit trails, DLP system outputs, consent management platforms, data classification tools, cross-border transfer logs, privacy impact assessments, breach notification records.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `apac_transfer_register.csv` and consent or DPIA lookups referenced in runbooks. (2) Map technical identifiers to records of processing activities with legal review. (3) Restrict indexes to regional privacy roles with least privilege. (4) Quarterly DPIA and consent linkage review with local counsel.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=web sourcetype="access_combined" earliest=-24h
| search uri="*optin*" OR uri="*opt-out*"
| stats count by uri, status
| sort - count
```
