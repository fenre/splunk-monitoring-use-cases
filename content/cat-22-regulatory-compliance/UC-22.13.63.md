# 22.13.63 — Communication Path Integrity — Unexpected Route or ASN Changes

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

We detect when BGP or path monitoring shows control center interconnects traversing unexpected autonomous systems or countries.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes netflow and sourcetypes flow:internet, flow:internet`, thousandeyes:tests`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=netflow sourcetype=flow:internet cip012_path=1 earliest=-24h
| stats values(dest_asn) as asn_list by src_cc, dest_cc, hour
| lookup cip012_approved_asn.csv src_cc, dest_cc, dest_asn OUTPUT approved
| mvexpand asn_list
| rename asn_list as dest_asn
| lookup cip012_approved_asn.csv src_cc, dest_cc, dest_asn OUTPUT approved
| where isnull(approved)
| table src_cc, dest_cc, dest_asn, hour
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```

## Implementation

(1) Build approved ASN matrix with carriers. (2) Enrich flows with GeoIP cautiously. (3) Alert NOC on drift. (4) Document maintenance reroutes in lookup. Indexes required: `index=netflow`; Sourcetypes: `sourcetype=flow:internet`, `sourcetype=flow:internet``, `sourcetype=thousandeyes:tests``.

## Visualization

Table (unexpected ASNs), Map (optional), Timeline. Key fields: `asn_list`, `dest_asn`, `dest_cc`, `hour`, `src_cc`.

## Compliance

- **NERC CIP** current: CIP-012-1 R3 Part 3.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-012-1 R3 Part 3.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
