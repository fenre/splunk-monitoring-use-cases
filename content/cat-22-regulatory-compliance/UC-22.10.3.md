# UC-22.10.3 · Information System Activity Review — Cross-Source ePHI Access Summary

## Description

Produces a daily leadership-ready rollup of who touched ePHI across EHR, VPN, and databases so workforce activity review obligations are operationally met, not ad hoc.

## Value

Produces a daily leadership-ready rollup of who touched ePHI across EHR, VPN, and databases so workforce activity review obligations are operationally met, not ad hoc.

## Implementation

(1) Align user ID formats via `identity_lookup.csv`; (2) restrict index ACLs to privacy/security; (3) schedule daily PDF/dashboard to HIPAA committee; (4) retain per legal record retention policy.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686), Splunk Add-on for Citrix (2757), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: `index=epic` `sourcetype=epic:audit`, `index=citrix` `sourcetype=citrix:session`, `index=vpn` `sourcetype=paloalto:globalprotect`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align user ID formats via `identity_lookup.csv`; (2) restrict index ACLs to privacy/security; (3) schedule daily PDF/dashboard to HIPAA committee; (4) retain per legal record retention policy.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
(index=epic sourcetype="epic:audit" AccessType="PatientChart") OR (index=cerner sourcetype="cerner:audit" Action="PatientChartView") OR (index=citrix sourcetype="citrix:session" Application="*Hyperspace*") OR (index=vpn sourcetype="paloalto:globalprotect" status="success")
| eval user=coalesce(USER_ID, PractitionerId, user, src_user)
| eval PAT_ID=coalesce(PAT_ID, PatientId)
| bin _time span=1d
| stats dc(PAT_ID) as patients_touched dc(host) as systems by _time, user
| sort - patients_touched
```
