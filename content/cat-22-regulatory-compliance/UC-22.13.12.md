# 22.13.12 — Post-Termination Access Revocation within 24 Hours

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

We measure hours from HR termination timestamp to Active Directory account disable for BES personnel, evidencing the one-day removal expectation.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes hr, wineventlog and sourcetypes WinEventLog:Security, hr:termination, hr:termination`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=hr sourcetype=hr:termination earliest=-14d
| eval term_epoch=strptime(term_time,"%Y-%m-%d %H:%M:%S")
| rename employee_id as user
| join type=left user [
    search index=wineventlog sourcetype="WinEventLog:Security" EventCode=4726 earliest=-14d
    | stats earliest(_time) as disable_epoch by TargetUserName
    | rename TargetUserName as user
  ]
| eval delta_h=round((disable_epoch-term_epoch)/3600,2)
| where isnull(disable_epoch) OR delta_h>24
| table user, term_time, disable_epoch, delta_h
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Ensure 4726 from all in-scope domains. (2) Normalize time zones to UTC. (3) Page IAM when SLA missed. (4) Optionally join CyberArk disable events as supplemental proof. Indexes required: `index=hr`, `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`, `sourcetype=hr:termination`, `sourcetype=hr:termination``.

## Visualization
