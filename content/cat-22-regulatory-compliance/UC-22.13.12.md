# UC-22.13.12 · Post-Termination Access Revocation within 24 Hours

## Description

We measure hours from HR termination timestamp to Active Directory account disable for BES personnel, evidencing the one-day removal expectation.

## Value

We measure hours from HR termination timestamp to Active Directory account disable for BES personnel, evidencing the one-day removal expectation.

## Implementation

(1) Ensure 4726 from all in-scope domains. (2) Normalize time zones to UTC. (3) Page IAM when SLA missed. (4) Optionally join CyberArk disable events as supplemental proof.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=hr` `sourcetype=hr:termination`, `index=wineventlog` `EventCode=4726`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ensure 4726 from all in-scope domains. (2) Normalize time zones to UTC. (3) Page IAM when SLA missed. (4) Optionally join CyberArk disable events as supplemental proof.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr sourcetype=hr:termination earliest=-14d
| eval term_epoch=strptime(term_time,"%Y-%m-%d %H:%M:%S")
| rename employee_id as user
| join type=left user [
    search index=wineventlog sourcetype="WinEventLog:Security" EventCode=4726 earliest=-14d
    | stats earliest(_time) as disable_epoch by TargetUserName
    | rename TargetUserName as user
  ]
| eval delta_h=round((disable_epoch-term_epoch)/3600,2)
| where isnull(disable_epoch) OR delta_h>24
| table user, term_time, disable_epoch, delta_h
```
