# UC-22.7.25 · Phishing Simulation Clicks vs Security Awareness Completion

## Description

Joins simulated phishing results with LMS completion records to evidence awareness training effectiveness for PR.AT.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes security and sourcetypes knowbe4:phish. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize user keys; (2) assign remedial training; (3) export for HR compliance; (4) refresh after campaigns. Indexes required: `index=security`; Sourcetypes: `sourcetype=knowbe4:phish`; Lookups: `lms_training_completion`, `lms_training_completion.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: KnowBe4 or O365 ingest.
• Ensure the following data sources are available: `knowbe4:phish`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize user keys; (2) assign remedial training; (3) export for HR compliance; (4) refresh after campaigns.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=security sourcetype=knowbe4:phish earliest=-90d
| stats latest(clicked) as clicked by user
| lookup lms_training_completion.csv user OUTPUT sec_aware_complete
| eval gap=if(clicked="Yes" AND sec_aware_complete!="true",1,0)
| where gap=1
```
