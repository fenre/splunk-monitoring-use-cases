# UC-22.14.4 · Response to Audit Logging Failures and Forwarder Gaps

## Description

Correlates forwarder offline events, TCP routing failures, and HEC 4xx/5xx spikes so operators can respond before AU-5 audit logging failures silently drop evidence.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes _internal and sourcetypes `splunkd`, http_event_collector_metrics, splunkd. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Enable verbose forwarder logging only on troubleshooting windows to avoid noise; (2) route `_internal` to a security operations index with RBAC; (3) alert on sustained `failure_signals`; (4) document compensating controls when maintenance pauses logging. Indexes required: `index=_internal`; Sourcetypes: `sourcetype=`splunkd``, `sourcetype=http_event_collector_metrics`, `sourcetype=splunkd`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Universal Forwarder / Splunk Cloud Platform.
• Ensure the following data sources are available: `_internal` sourcetype=`splunkd` OR `http_event_collector_metrics`, deployment server phonehome metrics.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enable verbose forwarder logging only on troubleshooting windows to avoid noise; (2) route `_internal` to a security operations index with RBAC; (3) alert on sustained `failure_signals`; (4) document compensating controls when maintenance pauses logging.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal earliest=-4h (sourcetype=splunkd OR sourcetype=http_event_collector_metrics)
| bin _time span=15m
| stats count(eval(match(_raw,"(?i)blocked|pause|queue|full|failure"))) as failure_signals,
        dc(host) as hosts by _time, sourcetype
| where failure_signals>0
| sort - failure_signals
```

**Pipeline walkthrough**
