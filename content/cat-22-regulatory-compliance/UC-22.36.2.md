# 22.36.2 — Right-to-erasure propagation completeness across downstream systems

**Criticality:** critical | **Difficulty:** advanced | **Status:** verified

## Description

Enforces that every registered downstream system reports a deletion acknowledgement for each accepted erasure request within seven days. A subject_token that persists past the SLA in any system is a direct Art.17 / CPRA §1798.105 finding.

## Value

Moves the erasure claim from 'we believe it propagated' to 'we proved it propagated, system by system' — exactly the evidence supervisory authorities ask for on first inspection.

## SPL

```spl
index=dsar subject_token=* action=erasure earliest=-180d
| stats values(system) as acked_systems by subject_token ticket_id
| lookup ropa_system_map.csv ticket_id OUTPUT systems_in_scope
| eval missing=mvfilter(NOT match(acked_systems, system))
| where mvcount(missing)>0
| table ticket_id subject_token systems_in_scope acked_systems missing
```

## Implementation

(1) Publish a ropa_system_map.csv lookup keyed by ticket_id → list of systems-in-scope; (2) ingest deletion-ack events from each system (DB TTL jobs, DWH deletion manifests, app-level hooks); (3) run hourly; (4) integrate with SOAR to auto-reopen tickets with missing acks. Indexes required: `index=dsar`.

## Visualization

Matrix of ticket_id × system with green/red cells; aging histogram; single-value for 'in-scope systems with ack in SLA'.

## Compliance

- **GDPR** 2016/679: Art.17 (full) — Art.17 right to erasure requires that the data be erased 'without undue delay'. The UC tracks completion across every downstream system listed in the ROPA.
- **GDPR** 2016/679: Art.17(2) (partial) — Art.17(2) requires reasonable measures to inform third parties; this UC surfaces the subset of third-party processors that acknowledged the erasure, but the DPO must verify the remainder manually.
- **CCPA/CPRA** CPRA (as amended): §1798.105 (full) — §1798.105 consumer right to delete; the UC tracks deletion across every system registered in the business's consumer-facing record.
- **UK GDPR** post-Brexit: Art.17 (partial) — Derived from GDPR@2016/679 Art.17 (parent assurance='full'). UK GDPR preserves parent clause numbering; inherited assurance degraded one step.
- **UK GDPR** post-Brexit: Art.17(2) (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.17(2).

## Known False Positives

Systems that hold data under a legitimate retention obligation (tax law, medical records) will 'miss' an ack. Use erasure_exceptions.csv to exclude them per ticket.
