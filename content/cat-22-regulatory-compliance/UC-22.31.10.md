# 22.31.10 — Operating system patching latency monitoring

**Criticality:** medium | **Difficulty:** beginner | **Status:** N/A

## Description

Evidences ASD Essential Eight obligations for operating system patching latency monitoring using endpoint, identity, and cloud telemetry.

## Value

Gives the responsible team auditor-ready evidence for ASD E8 compliance by automating regulatory compliance evidence using indexes os and sourcetypes package. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=os sourcetype=package host="AU-*" earliest=-24h
| stats latest(version) as ver by package, host
| lookup au_patch_baseline.csv package OUTPUT min_version
| where ver<min_version
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.dest | sort - count
```

Indexes required: `index=os`; Sourcetypes: `sourcetype=package`; Lookups: `au_patch_baseline`, `au_patch_baseline.csv`.
