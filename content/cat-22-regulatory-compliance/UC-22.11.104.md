# UC-22.11.104 · PCI-DSS 11.4 — Intrusion detection: IDS signature/health attestation + untuned alert monitoring

## Description

PCI-DSS 11.4 requires IDS/IPS use. This UC attests the sensors are alive and their signatures are fresh. Sensor gaps are a top-3 audit finding.

## Value

Transforms annual sign-off into 30-minute ground-truth. IR team sees stale signatures before they become an audit issue.

## Implementation

(1) Maintain pci_ids_sensors.csv; (2) onboard sensor telemetry; (3) schedule UC every 30m; (4) non-healthy state opens a P1/P2; (5) monthly CISO review. Indexes required: `index=ids`; Sourcetypes: `sourcetype=palo:threat`, `sourcetype=snort:ids`, `sourcetype=suricata:eve`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup pci_ids_sensors.csv
| join type=outer sensor
  [search index=ids earliest=-30m
   | stats latest(_time) AS last_seen latest(sig_version) AS sig_version BY sensor]
| eval age_d=round((now()-strptime(sig_version,"%Y-%m-%d"))/86400,0)
| eval state=case(isnull(last_seen),"offline",last_seen<now()-1800,"stale",age_d>7,"signatures-stale",true(),"healthy")
| where state!="healthy"
| table sensor last_seen sig_version age_d state
```
