# UC-22.14.48 · Incident Response Assistance — External IR Firm Access Auditing

## Description

Monitors VPN and jump host logons for third-party IR accounts for IR-7 incident response assistance access.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes net and sourcetypes cisco:asa. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Tag IR vendor accounts in IdM and forward to Splunk; (2) alert on access outside incident windows; (3) session recording references in tickets; (4) revoke after IR-4 closure. Indexes required: `index=net`; Sourcetypes: `sourcetype=cisco:asa`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Cisco ASA (Splunkbase 1620), Splunk Add-on for Unix and Linux (Splunkbase 833).
• Ensure the following data sources are available: `cisco:asa` VPN, `linux:secure` sshd.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag IR vendor accounts in IdM and forward to Splunk; (2) alert on access outside incident windows; (3) session recording references in tickets; (4) revoke after IR-4 closure.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=net sourcetype=cisco:asa earliest=-30d group_policy="IR_VENDOR"
| stats earliest(_time) as first, latest(_time) as last, values(src_ip) as sources by user, dest
| eval duration_h=round((last-first)/3600,2)
```
