# UC-22.13.23 · Physical Access Outside Approved Maintenance Window at PSP

## Description

We alert on badge-granted entries at Physical Security Perimeters outside documented maintenance windows for follow-up with operations.

## Value

We alert on badge-granted entries at Physical Security Perimeters outside documented maintenance windows for follow-up with operations.

## Implementation

(1) Encode maintenance windows in `psp_maint_windows.csv`. (2) Add holiday exceptions. (3) Route alerts to physical security. (4) Correlate with work orders.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk OT Security Add-on (5151).
• Ensure the following data sources are available: `index=physical` `sourcetype=pacs:access`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Encode maintenance windows in `psp_maint_windows.csv`. (2) Add holiday exceptions. (3) Route alerts to physical security. (4) Correlate with work orders.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype=pacs:access result="granted" earliest=-7d
| lookup cip_psp_register reader_id OUTPUT psp_name maint_cron_ok
| eval hour=strftime(_time,"%H")
| eval dow=strftime(_time,"%w")
| lookup psp_maint_windows.csv psp_name dow hour OUTPUT in_window
| where in_window!="true"
| table _time, employee_id, psp_name, reader_id, in_window
```
