# 22.13.23 — Physical Access Outside Approved Maintenance Window at PSP

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We alert on badge-granted entries at Physical Security Perimeters outside documented maintenance windows for follow-up with operations.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes physical and sourcetypes pacs:access, pacs:access`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=physical sourcetype=pacs:access result="granted" earliest=-7d
| lookup cip_psp_register reader_id OUTPUT psp_name maint_cron_ok
| eval hour=strftime(_time,"%H")
| eval dow=strftime(_time,"%w")
| lookup psp_maint_windows.csv psp_name dow hour OUTPUT in_window
| where in_window!="true"
| table _time, employee_id, psp_name, reader_id, in_window
```

## Implementation

(1) Encode maintenance windows in `psp_maint_windows.csv`. (2) Add holiday exceptions. (3) Route alerts to physical security. (4) Correlate with work orders. Indexes required: `index=physical`; Sourcetypes: `sourcetype=pacs:access`, `sourcetype=pacs:access``.
