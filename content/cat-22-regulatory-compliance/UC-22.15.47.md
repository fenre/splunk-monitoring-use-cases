# 22.15.47 — Conduit Traffic Allowlist Enforcement — Unexpected DNP3 Functions

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Detects DNP3 function codes on encrypted OT tunnels that are not in the approved conduit function matrix.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_protocol and sourcetypes edge_hub_dnp3. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_protocol sourcetype="edge_hub_dnp3" conduit_id="C-DMZ-OT1" earliest=-24h
| lookup dnp3_conduit_matrix.csv conduit_id function_code OUTPUT allowed
| where allowed!="true"
| stats count by src_ip, dest_ip, function_code, conduit_id
| sort - count
```

## Implementation

(1) Document matrix per conduit; (2) test in lab; (3) alert and optional block at gateway; (4) annual matrix review with engineering. Indexes required: `index=ot_protocol`; Sourcetypes: `sourcetype=edge_hub_dnp3`; Lookups: `dnp3_conduit_matrix`, `dnp3_conduit_matrix.csv`.

## Visualization

Heatmap (function×conduit), Table, Time chart. Key fields: `conduit_id`, `dest_ip`, `function_code`, `src_ip`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 5.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 5.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
