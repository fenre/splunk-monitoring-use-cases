# UC-22.11.54 · Service Account Inventory Reconciliation — Unexpected SPN or Delegation Changes

## Description

Tracks modifications to Kerberos SPNs and constrained delegation on payment-related service accounts so non-human identities stay inventoried and cannot silently gain broader authentication rights.

## Value

Tracks modifications to Kerberos SPNs and constrained delegation on payment-related service accounts so non-human identities stay inventoried and cannot silently gain broader authentication rights.

## Implementation

(1) Maintain `pci_service_accounts.csv` from CMDB. (2) Map EventCode descriptions for auditors. (3) Require CAB for SPN edits. (4) Daily diff email to identity team.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=windows` EventCode 4742,4738,5136 (Directory Service Changes for `servicePrincipalName`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `pci_service_accounts.csv` from CMDB. (2) Map EventCode descriptions for auditors. (3) Require CAB for SPN edits. (4) Daily diff email to identity team.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=windows sourcetype="WinEventLog:Security" EventCode IN (4742,4738) earliest=-7d
| where match(_raw,"(?i)servicePrincipalName|TrustedToAuthForDelegation|msDS-AllowedToDelegateTo)")
| lookup pci_service_accounts.csv SamAccountName OUTPUT pci_svc
| where pci_svc="true"
| stats earliest(_time) as first_change, values(EventCode) as codes by SamAccountName, SubjectUserName
| sort - first_change
```
