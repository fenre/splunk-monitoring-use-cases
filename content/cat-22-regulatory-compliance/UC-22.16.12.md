# 22.16.12 — Break-glass vault usage correlated to active P1 incidents

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "Break-glass vault usage correlated to active P1 incidents".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes itsm, pam and sourcetypes cyberark:pta, snow:incident. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=pam sourcetype=cyberark:pta earliest=-90d reason="*break*glass*"
| eval uc_tag="22.16.12"
| join type=left user [
    search index=itsm sourcetype=snow:incident priority="1" earliest=-90d | eval inc_start=strptime(opened_at,"%Y-%m-%d %H:%M:%S") | fields number user inc_start
  ]
| eval aligned=if(_time>=relative_time(inc_start,"-15m") AND _time<=relative_time(inc_start,"+8h"),1,0)
| where aligned=0
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=itsm`, `index=pam`.
