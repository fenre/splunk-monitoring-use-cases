# UC-22.10.45 · Individual Notification — Letter Generation & Mailing Evidence

## Description

Tracks generation, print, and certified mail tracking numbers for breach letters within the 60-day window to support individual notification compliance.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes privacy and sourcetypes hipaa:breach_notice, hipaa:breach_notice`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Integrate print vendor webhooks; (2) never index full SSN—use tokenized IDs; (3) legal sign-off field in same event. Indexes required: `index=privacy`; Sourcetypes: `sourcetype=hipaa:breach_notice`, `sourcetype=hipaa:breach_notice``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector (HEC), Splunk Add-on for ServiceNow (Splunkbase 1928) if tickets drive mailhouse.
• Ensure the following data sources are available: `index=privacy` `sourcetype=hipaa:breach_notice` (`incident_id`, `patient_token`, `letter_status`, `mail_tracking_id`, `mailed_dttm`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Integrate print vendor webhooks; (2) never index full SSN—use tokenized IDs; (3) legal sign-off field in same event.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="hipaa:breach_notice" earliest=-120d
| eval mailed_epoch=strptime(mailed_dttm,"%Y-%m-%d %H:%M:%S")
| eval breach_discovered=strptime(breach_discovered_dttm,"%Y-%m-%d %H:%M:%S")
| eval days_to_mail=round((mailed_epoch-breach_discovered)/86400,1)
| where letter_status!="Mailed" OR days_to_mail>60 OR isnull(mail_tracking_id)
| stats count by incident_id, letter_status
| sort - count
```
