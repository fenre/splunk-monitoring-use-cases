# UC-22.4.14 · CCPA/CPRA — Automated Decision Profiling — Opt-Out for Automated Decisioning Honored in Scoring API

## Description

Consumers may limit certain automated decisioning. Correlating scoring API calls with consent flags catches services still requesting model scores after an opt-out—reducing risk of statutory breaches tied to profiling choices.

## Value

Consumers may limit certain automated decisioning. Correlating scoring API calls with consent flags catches services still requesting model scores after an opt-out—reducing risk of statutory breaches tied to profiling choices.

## Implementation

(1) Align `consumer_id` keys across systems; (2) return 403 at gateway when possible—Splunk verifies drift; (3) redact payloads—use metadata only; (4) integrate with DSAR tooling; (5) legal defines `api_product` in-scope for “significant” decisioning.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="consent:state" limit_automated_decisioning=true earliest=-30d
| eval eff=strptime(updated_ts,"%Y-%m-%dT%H:%M:%SZ")
| join type=left consumer_id [
    search index=app sourcetype="apigee:access" api_product="risk_score_v2" earliest=-30d
    | rename developer_app_id as consumer_id
    | fields consumer_id, _time, response_code
  ]
| where _time>eff AND response_code=200
| stats count by consumer_id, api_product
| sort - count
```
