# 22.13.20 — ESP Security Rule Commit Without Approved Change Record

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We flag firewall commits affecting ESP rules when no linked change ticket exists, supporting controlled configuration of the security perimeter.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes itsm, network and sourcetypes chg:firewall, chg:firewall`, pan:config. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=network sourcetype=pan:config earliest=-7d
| where match(cmd,"(?i)commit")
| join type=left max=0 admin [
    search index=itsm sourcetype=chg:firewall earliest=-30d
    | stats latest(change_id) as change_id by submitter
    | rename submitter as admin
  ]
| where isnull(change_id)
| table _time, admin, object, cmd, change_id
```

## CIM SPL

```spl
| tstats summariesonly=t latest(All_Changes.status) as agg_value from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - agg_value
```

## Implementation

(1) Map Panorama admin IDs to ITSM submitter IDs. (2) Require change_id in commit messages where supported. (3) Weekly governance report. (4) Escalate repeat offenders. Indexes required: `index=itsm`, `index=network`; Sourcetypes: `sourcetype=chg:firewall`, `sourcetype=chg:firewall``, `sourcetype=pan:config`, `sourcetype=pan:config``.

## Visualization
