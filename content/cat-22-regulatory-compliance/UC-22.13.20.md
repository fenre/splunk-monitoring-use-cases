# UC-22.13.20 · ESP Security Rule Commit Without Approved Change Record

## Description

We flag firewall commits affecting ESP rules when no linked change ticket exists, supporting controlled configuration of the security perimeter.

## Value

We flag firewall commits affecting ESP rules when no linked change ticket exists, supporting controlled configuration of the security perimeter.

## Implementation

(1) Map Panorama admin IDs to ITSM submitter IDs. (2) Require change_id in commit messages where supported. (3) Weekly governance report. (4) Escalate repeat offenders.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757).
• Ensure the following data sources are available: `index=network` `sourcetype=pan:config`, `index=itsm` `sourcetype=chg:firewall`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map Panorama admin IDs to ITSM submitter IDs. (2) Require change_id in commit messages where supported. (3) Weekly governance report. (4) Escalate repeat offenders.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype=pan:config earliest=-7d
| where match(cmd,"(?i)commit")
| join type=left max=0 admin [
    search index=itsm sourcetype=chg:firewall earliest=-30d
    | stats latest(change_id) as change_id by submitter
    | rename submitter as admin
  ]
| where isnull(change_id)
| table _time, admin, object, cmd, change_id
```
