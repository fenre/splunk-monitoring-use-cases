# 22.21.21 — Post-Market Surveillance Signal Detection (Art. 72)

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

Uses z-score on daily serious_complaint signals per model_id for early post-market surveillance escalation.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_pms and sourcetypes ai:complaint:serious, ai:complaint:serious`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_pms sourcetype="ai:complaint:serious" earliest=-120d
| timechart span=1d count by model_id
| untable _time model_id count
| eventstats avg(count) as mu, stdev(count) as sd by model_id
| eval z=(count-mu)/nullif(sd,0)
| where z>3
| table _time, model_id, count, mu, z
```

## Implementation

(1) Define serious_complaint taxonomy; (2) dedupe by ticket_id; (3) alert z>3 for 2 consecutive days; (4) feed pharmacovigilance-style review where applicable. Indexes required: `index=ai_pms`; Sourcetypes: `sourcetype=ai:complaint:serious`, `sourcetype=ai:complaint:serious``.
