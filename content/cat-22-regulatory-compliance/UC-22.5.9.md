# UC-22.5.9 · MiFID II Algo Trading — Per-Instrument Circuit Breaker Trigger Frequency and Cooling-Off Compliance

## Description

Circuit breakers protect market integrity. Monitoring how often firm algos hit exchange or broker breaker thresholds, and whether mandatory cooling-off periods were observed before re-enable, supports Art. 17 governance of algorithmic activity.

## Value

Gives the responsible team auditor-ready evidence for MiFID II compliance by automating regulatory compliance evidence using indexes trading and sourcetypes oms:circuit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize timestamps to UTC; (2) map `algo_id` to registered strategy in inventory; (3) escalate to compliance officer on any early resume; (4) exclude test venues via lookup; (5) retain for regulatory interview packs. Indexes required: `index=trading`; Sourcetypes: `sourcetype=oms:circuit`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector from OMS/EMS, market access gateway logs.
• Ensure the following data sources are available: `index=trading` `sourcetype="oms:circuit"` (algo_id, symbol, venue, trigger_ts, breaker_type, cooldown_until, resumed_ts).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize timestamps to UTC; (2) map `algo_id` to registered strategy in inventory; (3) escalate to compliance officer on any early resume; (4) exclude test venues via lookup; (5) retain for regulatory interview packs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=trading sourcetype="oms:circuit" earliest=-90d
| eval cool=strptime(cooldown_until,"%Y-%m-%dT%H:%M:%SZ"), res=strptime(resumed_ts,"%Y-%m-%dT%H:%M:%SZ")
| where isnotnull(res) AND res<cool
| stats count as early_resumes by algo_id, venue, symbol
| sort - early_resumes
```
