# 22.2.43 — NIS2 Annual Security Assessment Completion Tracking

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Effectiveness assessments and broader security assessments are recurring obligations. This use case tracks annual independent assessments, red-team summaries, and control testing sign-offs — ensuring each NIS2-scoped entity has a current assessment on file.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes itsm. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
| inputlookup nis2_annual_assessment_register.csv
| eval due_epoch=strptime(next_assessment_due,"%Y-%m-%d")
| eval overdue=if(now()>due_epoch,1,0)
| join legal_entity_id [
    search index=itsm short_description="*NIS2*annual*assessment*" earliest=-400d
    | stats latest(closed_at) as last_closed by company
    | rename company as legal_entity_id
]
| where overdue=1
| table legal_entity_id, assessment_type, next_assessment_due, last_closed, overdue
```

## Implementation

(1) Define assessment types (internal audit, external cert, penetration retest); (2) sync due dates from GRC; (3) alert CFO/legal when entity-level assessment is overdue; (4) attach assessment reports in controlled repository; (5) link findings to remediation in ServiceNow. Indexes required: `index=itsm`; Lookups: `nis2_annual_assessment_register`, `nis2_annual_assessment_register.csv`.

## Visualization

Table (overdue assessments), Timeline (by entity), Single value (overdue count), Bar chart (assessment type mix).

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(f) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(f) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
