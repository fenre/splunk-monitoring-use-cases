# UC-22.11.43 · Least-Privilege Validation — Interactive Logons to Database Tier from Workstations

## Description

Surfaces database interactive logins from non-administrator workstations that may violate least privilege for personnel access to cardholder data stores.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes db and sourcetypes mssql:audit, oracle:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Populate jump host egress IPs. (2) Exclude application service accounts via lookup. (3) Pair with bastion session logs for false positives. (4) Quarterly access review input. Indexes required: `index=db`; Sourcetypes: `sourcetype=mssql:audit`, `sourcetype=oracle:audit`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742), Splunk Add-on for Microsoft SQL Server (2648) if audit ingested.
• Ensure the following data sources are available: `index=db` `mssql:audit` action="AUDIT SESSION" or `sourcetype="oracle:audit"`; `lookup pci_jump_hosts.csv` client IP.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate jump host egress IPs. (2) Exclude application service accounts via lookup. (3) Pair with bastion session logs for false positives. (4) Quarterly access review input.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=db sourcetype="mssql:audit" earliest=-24h
| where match(action,"(?i)login|authentication")
| lookup pci_jump_hosts.csv ip AS client_ip OUTPUT is_jump
| where isnull(is_jump) OR is_jump="false"
| lookup pci_db_tier.csv server_name OUTPUT tier
| where tier="carddata_db"
| stats count by client_ip, server_name, database_principal_name
| sort - count
```
