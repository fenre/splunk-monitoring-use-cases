# 22.15.29 — Unexpected PLC Logic CRC Change

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Compares reported logic checksums or project CRCs to last approved baseline to detect unauthorized software changes.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_eng and sourcetypes plc:crc. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_eng sourcetype="plc:crc" earliest=-7d
| lookup approved_plc_crc.csv controller program_name OUTPUT expected_crc
| where isnotnull(expected_crc) AND crc!=expected_crc
| stats earliest(_time) as first_bad by controller, program_name, crc, expected_crc, user
| sort - first_bad
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.user | sort - count
```

## Implementation

(1) Export CRC after each validated download; (2) store gold images securely; (3) auto-ticket on mismatch; (4) offline forensic for first event. Indexes required: `index=ot_eng`; Sourcetypes: `sourcetype=plc:crc`; Lookups: `approved_plc_crc`, `approved_plc_crc.csv`.

## Visualization

Table, Timeline, Single value (mismatches/day). Key fields: `controller`, `crc`, `expected_crc`, `first_bad`, `program_name`, `user`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 3.4 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 3.4.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
