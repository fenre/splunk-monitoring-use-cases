# UC-22.15.29 · Unexpected PLC Logic CRC Change

## Description

Compares reported logic checksums or project CRCs to last approved baseline to detect unauthorized software changes.

## Value

Compares reported logic checksums or project CRCs to last approved baseline to detect unauthorized software changes.

## Implementation

(1) Export CRC after each validated download; (2) store gold images securely; (3) auto-ticket on mismatch; (4) offline forensic for first event.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Historian or engineering export.
• Ensure the following data sources are available: `index=ot_eng` CRC events; `approved_plc_crc.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Export CRC after each validated download; (2) store gold images securely; (3) auto-ticket on mismatch; (4) offline forensic for first event.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_eng sourcetype="plc:crc" earliest=-7d
| lookup approved_plc_crc.csv controller program_name OUTPUT expected_crc
| where isnotnull(expected_crc) AND crc!=expected_crc
| stats earliest(_time) as first_bad by controller, program_name, crc, expected_crc, user
| sort - first_bad
```
