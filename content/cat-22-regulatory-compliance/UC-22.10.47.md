# UC-22.10.47 · Media Notification — Large-State Resident Threshold Tracking

## Description

Computes per-state affected resident counts to trigger prominent media notice obligations when thresholds in a jurisdiction or overall breach size demand public notice.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes privacy and sourcetypes hipaa:breach_subject, hipaa:breach_subject`, hipaa:media_notice. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Legal provides threshold table including state mini-HIPAA laws; (2) do not store full addresses in Splunk; (3) coordinate with comms team workflow IDs. Indexes required: `index=privacy`; Sourcetypes: `sourcetype=hipaa:breach_subject`, `sourcetype=hipaa:breach_subject``, `sourcetype=hipaa:media_notice`; Lookups: `state_media_threshold`, `state_media_threshold.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / HEC.
• Ensure the following data sources are available: `index=privacy` `sourcetype=hipaa:breach_subject` (`incident_id`, `state`, `media_required`), `lookup state_media_threshold.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Legal provides threshold table including state mini-HIPAA laws; (2) do not store full addresses in Splunk; (3) coordinate with comms team workflow IDs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="hipaa:breach_subject" earliest=-365d included_in_count="true"
| stats dc(subject_token) as affected by incident_id, state
| lookup state_media_threshold.csv state OUTPUT threshold
| eval media_needed=if(affected>=threshold,"Yes","No")
| where media_needed="Yes"
| join incident_id [
    search index=privacy sourcetype="hipaa:media_notice" earliest=-365d
    | stats latest(published_dttm) as published by incident_id, state ]
| where isnull(published)
| table incident_id, state, affected, threshold
```
