# UC-22.13.59 · Information Handling Procedure Compliance — Email DLP for BCSI

## Description

We track Data Loss Prevention blocks on messages tagged as BCSI to show handling rules are enforced in email channels.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes email and sourcetypes mimecast:dlp, mimecast:dlp`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map DLP policies to BCSI categories. (2) Differentiate block vs notify. (3) Weekly compliance metrics. (4) Investigate repeated violators via HR process. Indexes required: `index=email`; Sourcetypes: `sourcetype=mimecast:dlp`, `sourcetype=mimecast:dlp``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=email` `sourcetype=mimecast:dlp` or `ms:o365:dlp`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map DLP policies to BCSI categories. (2) Differentiate block vs notify. (3) Weekly compliance metrics. (4) Investigate repeated violators via HR process.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=email sourcetype=mimecast:dlp earliest=-7d policy="BCSI"
| stats count by action, sender, recipient
| sort - count
```
