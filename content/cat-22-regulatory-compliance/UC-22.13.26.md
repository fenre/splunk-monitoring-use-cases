# 22.13.26 — Physical Access Log Ingest Continuity for Retention Evidence

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We detect multi-day gaps in PACS indexing that could undermine evidence of physical access monitoring and review.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes physical and sourcetypes pacs:access, pacs:access`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=physical sourcetype=pacs:access earliest=-60d
| timechart span=1d count as events
| streamstats window=7 avg(events) as baseline
| eval gap=if(events<baseline*0.15 OR events=0,1,0)
| where gap=1
| table _time, events, baseline
```

## Implementation

(1) Exclude planned outages via lookup. (2) Alert physical security and Splunk admins. (3) Track forwarder uptime. (4) Export gap analysis for audits. Indexes required: `index=physical`; Sourcetypes: `sourcetype=pacs:access`, `sourcetype=pacs:access``.

## Visualization

Time chart (daily volume), Table (gap days), Single value (max gap hours)
