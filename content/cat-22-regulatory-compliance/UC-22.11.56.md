# UC-22.11.56 · Physical Badge Access to Data Center Containing Cardholder Systems

## Description

Tracks granted and denied door reads for suites housing payment hardware so physical entry to sensitive areas is logged and reviewable for onsite PCI assessments.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes physical and sourcetypes pacs:door. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map reader IDs to PCI zones from facility diagrams. (2) De-identify `badge_id` in dashboards if required. (3) Alert on sustained deny spikes (tailgating). (4) Quarterly reviewer export. Indexes required: `index=physical`; Sourcetypes: `sourcetype=pacs:door`; Lookups: `pci_dc_readers`, `pci_dc_readers.csv`.

• Ensure the following data sources are available: `index=physical` `sourcetype="pacs:door"` — `reader_id`, `badge_id`, `result`, `site`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map reader IDs to PCI zones from facility diagrams. (2) De-identify `badge_id` in dashboards if required. (3) Alert on sustained deny spikes (tailgating). (4) Quarterly reviewer export.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype="pacs:door" earliest=-7d
| lookup pci_dc_readers.csv reader_id OUTPUT cde_dc_zone
| where cde_dc_zone="true"
| eval denied=if(match(result,"(?i)deny|reject|unknown"),1,0)
| timechart span=1d sum(denied) as denied_reads, count as total_reads
| eval deny_rate=if(total_reads>0, round(100*denied_reads/total_reads,2), null())
```
