# 22.32.20 — CUI incident response evidence

**Criticality:** critical | **Difficulty:** beginner | **Status:** N/A

## Description

Supports CMMC compliance evidence for cui incident response evidence in Splunk dashboards and scheduled reports.

## Value

Gives the responsible team auditor-ready evidence for CMMC compliance by automating regulatory compliance evidence using indexes edr and sourcetypes crowdstrike:detection. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=edr sourcetype="crowdstrike:detection" tag=cui earliest=-7d
| stats count by tactic, hostname
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.dest | sort - count
```

## Implementation

(1) Maintain `amer_authorization_boundary.csv` with ATO, CUI, and CJIS scope flags. (2) Integrate POA&M exports via scheduled output and lookup refresh for ISSM workflows. (3) Separate LGPD and federal datasets with index- and role-level controls. (4) Quarterly control mapping workshop with ISSM, ISO, and privacy counsel. Indexes required: `index=edr`; Sourcetypes: `sourcetype=crowdstrike:detection`.

## Visualization

Stacked bar (control family), Table (users and events), Single value (open findings). Key fields: `hostname`, `tactic`.

## Compliance

- **CMMC** 2.0: IR.L2-3.6.1 (contributing) — CUI incident response evidence directly supports incident handling capability.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
