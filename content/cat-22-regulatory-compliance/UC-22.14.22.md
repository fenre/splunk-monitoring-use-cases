# UC-22.14.22 · Session Lock Events for Workstation Inactivity Policy

## Description

Monitors workstation lock (4800) and unlock (4801) ratio for AC-11 session lock enforcement.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes sec and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Validate ComputerName field extraction; (2) exclude kiosk OU via lookup; (3) sample manual validation on outliers; (4) integrate with MDM compliance scores if available. Indexes required: `index=sec`; Sourcetypes: `sourcetype=WinEventLog:Security`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (Splunkbase 742).
• Ensure the following data sources are available: `WinEventLog:Security` 4800/4801.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Validate ComputerName field extraction; (2) exclude kiosk OU via lookup; (3) sample manual validation on outliers; (4) integrate with MDM compliance scores if available.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sec sourcetype=WinEventLog:Security EventCode IN (4800,4801) earliest=-7d
| eval evt=if(EventCode=4800,"lock","unlock")
| stats count by evt, ComputerName
| xyseries ComputerName evt count
| eval lock_ratio=round(lock/(lock+unlock+0.001),3)
| where lock_ratio<0.2
```
