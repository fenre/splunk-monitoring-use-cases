# UC-22.7.39 · IR Ticket Stuck in Containment Beyond SLA

## Description

Measures security incidents stuck in containment state beyond SLA for incident management effectiveness.

## Value

Measures security incidents stuck in containment state beyond SLA for incident management effectiveness.

## Implementation

(1) Map state numbers to your instance; (2) page IR lead; (3) document blockers.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (Splunkbase 1928).
• Ensure the following data sources are available: `snow:incident`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map state numbers to your instance; (2) page IR lead; (3) document blockers.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype=snow:incident category=security earliest=-30d state="3"
| eval hours_in_state=(now()-strptime(u_containment_start,"%Y-%m-%d %H:%M:%S"))/3600
| where isnotnull(u_containment_start) AND hours_in_state>8
```
