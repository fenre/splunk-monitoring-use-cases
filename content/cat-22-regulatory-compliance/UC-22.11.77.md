# UC-22.11.77 · IDS and IPS Alert Volume Baseline and Spike Detection

## Description

Applies moving baselines to IDS/IPS alert rates on CDE perimeter sensors so intrusion-detection failures or sudden drops in alerting—often a sign of sensor issues—are visible alongside spikes.

## Value

Applies moving baselines to IDS/IPS alert rates on CDE perimeter sensors so intrusion-detection failures or sudden drops in alerting—often a sign of sensor issues—are visible alongside spikes.

## Implementation

(1) Tune `med>10` for low-noise signatures. (2) Pair drops with health checks. (3) Feed spikes to SOC shift lead. (4) Include in monthly IDS tuning minutes.

## Detailed Implementation

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tune `med>10` for low-noise signatures. (2) Pair drops with health checks. (3) Feed spikes to SOC shift lead. (4) Include in monthly IDS tuning minutes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ids sourcetype IN ("pan:threat","suricata:alert") earliest=-30d
| lookup pci_sensor_zones.csv sensor OUTPUT cde_adjacent
| where cde_adjacent="true"
| bin _time span=1d
| stats count by _time, signature
| eventstats median(count) as med by signature
| eval spike=if(count>med*4 AND med>10,1,0)
| where spike=1
| table _time, signature, count, med
| sort - count
```
