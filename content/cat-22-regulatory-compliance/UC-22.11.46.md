# UC-22.11.46 · Vendor Remote Access Sessions into CDE Jump Hosts

## Description

Lists vendor identities and source IPs using remote access paths into the CDE so third-party access is monitored consistently with service provider due diligence expectations.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes netfw, windows and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map vendor accounts to contracts in GRC. (2) Cross-check active maintenance windows. (3) Alert outside window. (4) Include in third-party access quarterly attestations. Indexes required: `index=netfw`, `index=windows`; Sourcetypes: `sourcetype=WinEventLog:Security`; Lookups: `pci_jump_hosts`, `pci_jump_hosts.csv`, `pci_vendor_accounts`, `pci_vendor_accounts.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757), Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=windows` EventCode 4624 LogonType 10; `index=netfw` GlobalProtect logs with vendor realm.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map vendor accounts to contracts in GRC. (2) Cross-check active maintenance windows. (3) Alert outside window. (4) Include in third-party access quarterly attestations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=windows sourcetype="WinEventLog:Security" EventCode=4624 LogonType=10 earliest=-7d
| lookup pci_jump_hosts.csv ComputerName OUTPUT jump_host
| where jump_host="true"
| lookup pci_vendor_accounts.csv TargetUserName OUTPUT vendor_name
| where isnotnull(vendor_name)
| stats earliest(_time) as first_seen, latest(_time) as last_seen by TargetUserName, vendor_name, WorkstationName, IpAddress
| sort vendor_name
```
