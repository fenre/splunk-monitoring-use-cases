# 22.13.19 — EACMS and IRA Authentication Failure Concentration

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We detect brute-force patterns against Electronic Access Control and Monitoring Systems that gate ESP remote access.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes network, os and sourcetypes eacms:auth, eacms:auth`, linux:secure. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
(index=network sourcetype=eacms:auth result="failure") OR (index=os sourcetype=linux:secure "Failed password")
earliest=-24h
| eval user=coalesce(user, account)
| lookup eacms_asset_map host OUTPUT eacms_id
| stats count by user, src_ip, host, eacms_id
| where count>25
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.user, Authentication.src, Authentication.dest | sort - count
```

## Implementation

(1) Normalize vendor syslog into `eacms:auth`. (2) Distinguish IRA accounts from vendors. (3) Integrate with ES adaptive response. (4) Retain raw events per records retention. Indexes required: `index=network`, `index=os`; Sourcetypes: `sourcetype=eacms:auth`, `sourcetype=eacms:auth``, `sourcetype=linux:secure`, `sourcetype=linux:secure``; Lookups: `eacms_asset_map`.

## Visualization

Table (top sources), Time chart (failures), Single value (locked-out systems). Key fields: `eacms_id`, `host`, `src_ip`, `user`.

## Compliance

- **NERC CIP** current: CIP-005-6 R4 Part 4.2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-005-6 R4 Part 4.2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
