# 22.15.24 — Forwarder Stops — Audit Pipeline Failure Response

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Detects universal forwarder or OT collector outages that stop security-relevant events reaching the SIEM.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes _internal and sourcetypes splunkd. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=_internal source=*metrics.log group=tcpout_connections earliest=-4h
| stats latest(kb) as kb by host
| join type=left host [ search index=_internal sourcetype=splunkd earliest=-4h "Closing tcp" OR "ERROR TcpOutputFd" | stats count as errs by host ]
| where isnull(kb) OR errs>10
| table host, kb, errs
```

## Implementation

(1) Enable MC alerts; (2) map hosts to OT sites; (3) redundant intermediate HF; (4) local buffering on UF until restored. Indexes required: `index=_internal`; Sourcetypes: `sourcetype=splunkd`.

## Visualization

Table (affected sites), Single value (hosts down), Time chart.

## Compliance

- **IEC 62443** 2013-ongoing: SR 2.10 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 2.10.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
