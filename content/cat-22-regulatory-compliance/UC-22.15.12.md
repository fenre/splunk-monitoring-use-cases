# 22.15.12 — Non-Interactive Industrial Protocol Clients Without Device Identity

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

Finds Modbus/OPC-UA sessions where client identity cannot be tied to an approved automation asset, weakening software process and device identification.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_protocol and sourcetypes edge_hub_modbus, edge_hub_opcua. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_protocol (sourcetype="edge_hub_modbus" OR sourcetype="edge_hub_opcua") earliest=-24h
| eval client_key=coalesce(client_ip, opcua_client_appuri, modbus_unit_id)
| lookup ot_approved_clients.csv client_key OUTPUT asset_name, owner
| where isnull(asset_name)
| stats dc(function_code) as fc_dc, dc(node_id) as nodes, count by client_key, sourcetype
| sort - count
```

## Implementation

(1) Build authoritative client inventory from engineering CMDB; (2) enrich Edge Hub events with IP→asset; (3) alert on new clients; (4) certificate-pin OPC-UA where possible. Indexes required: `index=ot_protocol`; Sourcetypes: `sourcetype=edge_hub_modbus`, `sourcetype=edge_hub_opcua`; Lookups: `ot_approved_clients`, `ot_approved_clients.csv`.

## Visualization

Table (unknown clients), Map (optional), Time chart (first seen). Key fields: `client_key`, `fc_dc`, `nodes`, `sourcetype`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 1.2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 1.2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
