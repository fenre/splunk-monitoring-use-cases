# UC-22.6.23 · Removable Storage — USB Mount Events on Engineering Workstations

## Description

Uses endpoint logs for removable media mounts in restricted VLANs to enforce media handling procedures.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes endpoint and sourcetypes XmlWinEventLog:Microsoft-Windows-Sysmon/Operational. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) USB policy allowlist; (2) DLP correlation optional; (3) quarterly sample for auditors. Indexes required: `index=endpoint`; Sourcetypes: `sourcetype=XmlWinEventLog:Microsoft-Windows-Sysmon/Operational`; Lookups: `restricted_subnets`, `restricted_subnets.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Sysmon or CrowdStrike.
• Ensure the following data sources are available: Sysmon EventCode=22.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) USB policy allowlist; (2) DLP correlation optional; (3) quarterly sample for auditors.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=endpoint sourcetype=XmlWinEventLog:Microsoft-Windows-Sysmon/Operational EventCode=22 earliest=-30d
| lookup restricted_subnets.csv src_ip OUTPUT zone
| where zone="engineering_secure"
| stats count by ComputerName, Image
```
