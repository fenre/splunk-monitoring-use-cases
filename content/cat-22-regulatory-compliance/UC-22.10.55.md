# 22.10.55 — Cloud Service Provider — ePHI Hosting Admin Actions in AWS or Azure Audit

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Tracks CSP administrator and break-glass activity against accounts or subscriptions tagged as hosting replicated ePHI backups or analytics, supporting BA and cloud subcontractor oversight.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes aws and sourcetypes aws:cloudtrail, aws:cloudtrail`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=aws sourcetype="aws:cloudtrail" earliest=-7d
| rename userIdentity.arn as user_arn
| eval account=coalesce(recipientAccountId, aws_account_id)
| lookup aws_account_ephi_tags.csv account OUTPUT contains_ephi_backup
| where contains_ephi_backup="true"
| search eventName IN ("AssumeRole","GetObject","PutObject","DeleteObject","CreateSnapshot","RunInstances","ModifyDBInstance")
| eval principal=mvindex(split(user_arn,"/"),-1)
| stats count values(eventName) as actions by principal, account, sourceIPAddress
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.user | sort - count
```

## Implementation

(1) Tag AWS accounts in `aws_account_ephi_tags.csv` from enterprise architecture; (2) use organization CloudTrail with integrity validation; (3) route root and break-glass usage to SOC and privacy; (4) map ARNs to named individuals via IAM Identity Center export. Indexes required: `index=aws`; Sourcetypes: `sourcetype=aws:cloudtrail`, `sourcetype=aws:cloudtrail``.

## Visualization

Table (principal, actions, account), Timeline (snapshot or delete spikes), Single value (break-glass events).

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
