# UC-22.10.29 · Device & Media Controls — USB Mass Storage on ePHI Workstations

## Description

Captures removable media mount attempts on endpoints that access Hyperspace so device and media control policies for ePHI workstations are enforceable and auditable.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes endpoint, windows and sourcetypes WinEventLog:Microsoft-Windows-Kernel-PnP/Configuration, crowdstrike:usb`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Enforce BitLocker + DLP on exceptions; (2) require temporary USB grants via ServiceNow with ticket ID in log annotation optional; (3) quarterly exception review. Indexes required: `index=endpoint`, `index=windows`; Sourcetypes: `sourcetype=WinEventLog:Microsoft-Windows-Kernel-PnP/Configuration`, `sourcetype=crowdstrike:usb``; Lookups: `clinical_workstations`, `clinical_workstations.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=windows sourcetype="WinEventLog:Microsoft-Windows-Kernel-PnP/Configuration" EventCode="400" earliest=-7d
| where match(ClassName,"USBSTOR|Disk&Ven_")
| lookup clinical_workstations.csv ComputerName OUTPUT contains_ephi_workstation
| where contains_ephi_workstation="true"
| stats count by ComputerName, ClassName, DeviceID
| sort - count
```
