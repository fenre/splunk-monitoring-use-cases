# UC-22.7.15 · Supply Chain — New Package Installs in CI Against Approved Registry

## Description

Monitors CI pipelines for package installs that bypass internal registry mirrors, aligning with GV.SC supply chain risk management.

## Value

Monitors CI pipelines for package installs that bypass internal registry mirrors, aligning with GV.SC supply chain risk management.

## Implementation

(1) Seed approved internal registry mirror packages; (2) scope to release branches; (3) block on alert in pipeline optional; (4) quarterly supplier software review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: GitHub Enterprise audit or Jenkins.
• Ensure the following data sources are available: `github:audit` or `jenkins:console`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Seed approved internal registry mirror packages; (2) scope to release branches; (3) block on alert in pipeline optional; (4) quarterly supplier software review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cicd (sourcetype=github:audit OR sourcetype=jenkins:console) earliest=-7d "npm install" OR "pip install"
| rex field=_raw "(?i)(npm|pip)\s+install\s+(?<pkg>[^\s]+)"
| lookup approved_build_packages.csv pkg OUTPUT approved
| where isnull(approved)
```
