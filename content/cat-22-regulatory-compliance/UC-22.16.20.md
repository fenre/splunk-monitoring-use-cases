# 22.16.20 — Documented subnets vs observed NetFlow peers

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "Documented subnets vs observed NetFlow peers".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes netflow and sourcetypes flow:ipfix. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=netflow sourcetype=flow:ipfix earliest=-7d
| eval uc_tag="22.16.20"
| eval dst_subnet=mvindex(split(dst,"."),0).".".mvindex(split(dst,"."),1).".".mvindex(split(dst,"."),2)".0/24"
| stats sum(bytes) as b by dst_subnet
| lookup pipeline_documented_subnets.csv dst_subnet OUTPUT zone
| where isnull(zone)
| sort - b
```

## CIM SPL

```spl
| tstats summariesonly=t sum(All_Traffic.bytes_in) as agg_value from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - agg_value
```
