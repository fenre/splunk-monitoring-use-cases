# UC-22.4.23 · CCPA/CPRA — Cross-Context Behavioral Advertising — Household Device Graph Linking Without Aggregated Opt-Out Propagation

## Description

Household-level graphs can undermine individual opt-outs if not synchronized. Monitoring graph edges created linking a opted-out device to new cookies without a refreshed household consent record evidences governance gaps in cross-device advertising.

## Value

Gives the responsible team auditor-ready evidence for CCPA/CPRA compliance by automating regulatory compliance evidence using indexes id, privacy and sourcetypes consent:state, graph:edge. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain `household_consent_floor.csv` (household_id, min_consent_version) updated when household policies change; (2) pseudonymize `household_id`; (3) vendor SLA for propagation latency; (4) legal defines household scope; (5) integrate with periodic reconciliation job. Indexes required: `index=id`, `index=privacy`; Sourcetypes: `sourcetype=consent:state`, `sourcetype=graph:edge`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Identity graph vendor logs via HEC.
• Ensure the following data sources are available: `index=id` `sourcetype="graph:edge"` (household_id, device_a, device_b, created_ts, consent_version); `index=privacy` `sourcetype="consent:state"` (device_id, sale_opt_out, updated_ts).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `household_consent_floor.csv` (household_id, min_consent_version) updated when household policies change; (2) pseudonymize `household_id`; (3) vendor SLA for propagation latency; (4) legal defines household scope; (5) integrate with periodic reconciliation job.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="consent:state" sale_opt_out=true earliest=-30d
| eval eff=strptime(updated_ts,"%Y-%m-%dT%H:%M:%SZ")
| rename device_id as device_a
| join type=left device_a [
    search index=id sourcetype="graph:edge" earliest=-30d
    | eval ct=strptime(created_ts,"%Y-%m-%dT%H:%M:%SZ")
    | fields household_id, device_a, device_b, ct, consent_version
  ]
| lookup household_consent_floor.csv household_id OUTPUT min_consent_version as required_consent_version
| where ct>eff AND consent_version!=required_consent_version
| table household_id, device_a, device_b, ct, consent_version, required_consent_version
```
