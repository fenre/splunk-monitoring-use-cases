# 22.13.57 — BES Cyber System Information Access via Share and Web Downloads

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

We monitor access to file shares and web paths classified as BES Cyber System Information for inappropriate bulk retrieval.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode=5145 earliest=-24h
| lookup bcsi_path_tags.csv ShareName, RelativeTargetName OUTPUT classification
| where classification="BCSI"
| stats sum(TransferredBytes) as bytes by Account_Name, ShareName, ComputerName
| where bytes > 104857600
| sort - bytes
```

## Implementation

(1) Tag BCSI paths in lookup from information protection register. (2) Enable detailed file share auditing. (3) Alert on high-byte transfers. (4) Integrate with insider threat program. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`.

## Visualization

Table (large transfers), Bar chart (by user), Single value (events). Key fields: `Account_Name`, `ComputerName`, `ShareName`, `bytes`.

## Compliance

- **NERC CIP** current: CIP-011-3 R1 Part 1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-011-3 R1 Part 1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
