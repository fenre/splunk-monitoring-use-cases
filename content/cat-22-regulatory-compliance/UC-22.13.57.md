# UC-22.13.57 · BES Cyber System Information Access via Share and Web Downloads

## Description

We monitor access to file shares and web paths classified as BES Cyber System Information for inappropriate bulk retrieval.

## Value

We monitor access to file shares and web paths classified as BES Cyber System Information for inappropriate bulk retrieval.

## Implementation

(1) Tag BCSI paths in lookup from information protection register. (2) Enable detailed file share auditing. (3) Alert on high-byte transfers. (4) Integrate with insider threat program.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=wineventlog` `EventCode=5145` or proxy logs, `bcsi_path_tags.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag BCSI paths in lookup from information protection register. (2) Enable detailed file share auditing. (3) Alert on high-byte transfers. (4) Integrate with insider threat program.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode=5145 earliest=-24h
| lookup bcsi_path_tags.csv ShareName, RelativeTargetName OUTPUT classification
| where classification="BCSI"
| stats sum(TransferredBytes) as bytes by Account_Name, ShareName, ComputerName
| where bytes > 104857600
| sort - bytes
```
