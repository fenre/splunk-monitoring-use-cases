# 22.13.24 — Unauthorized Physical Access Attempts at PSP

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We aggregate denied, forced, and tailgate physical access events for rapid triage and evidence of physical perimeter monitoring.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes physical and sourcetypes pacs:access, pacs:access`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=physical sourcetype=pacs:access earliest=-24h
    (result="denied" OR event_type IN ("forced","tailgate","passback"))
| lookup cip_psp_register reader_id OUTPUT psp_name bes_site
| stats count by event_type, psp_name, bes_site
| sort - count
```

## Implementation

(1) Map readers to PSP inventory. (2) Push high severity to SOC bridge. (3) Optional join to VMS syslog. (4) Weekly trending for reliability. Indexes required: `index=physical`; Sourcetypes: `sourcetype=pacs:access`, `sourcetype=pacs:access``; Lookups: `cip_psp_register`.

## Visualization

Stacked bar (event types), Table (raw), Single value (tailgates). Key fields: `bes_site`, `event_type`, `psp_name`.

## Compliance

- **NERC CIP** current: CIP-006-6 R2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-006-6 R2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
