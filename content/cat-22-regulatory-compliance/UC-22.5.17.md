# UC-22.5.17 · MiFID II Conflicts of Interest — Gift and Entertainment Threshold Breach Trending

## Description

Inducements create conflicts. Aggregating declared gifts per employee and counterparty against policy caps evidences proactive monitoring of inducement risk alongside gifts-and-entertainment registers.

## Value

Inducements create conflicts. Aggregating declared gifts per employee and counterparty against policy caps evidences proactive monitoring of inducement risk alongside gifts-and-entertainment registers.

## Implementation

(1) Set EUR cap per policy; (2) multi-currency conversion upstream; (3) route to COO office; (4) quarterly attestations; (5) map to inducements policy section.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect to ethics register, or `inputlookup`.
• Ensure the following data sources are available: `index=compliance` `sourcetype="ethics:gift"` (employee_id, counterparty_id, value_eur, gift_date, approved_flag).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Set EUR cap per policy; (2) multi-currency conversion upstream; (3) route to COO office; (4) quarterly attestations; (5) map to inducements policy section.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=compliance sourcetype="ethics:gift" earliest=-365d
| eval gd=strptime(gift_date,"%Y-%m-%d")
| bin gd span=90d
| stats sum(value_eur) as rolling_value by employee_id, gd
| where rolling_value>1000 AND approved_flag!="true"
| sort - rolling_value
```
