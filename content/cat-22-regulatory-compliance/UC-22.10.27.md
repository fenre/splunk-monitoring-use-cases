# UC-22.10.27 · Integrity — Caboodle / Clarity ETL Job Failures & Partial Loads

## Description

Detects failed or partial analytics warehouse loads that could silently skew quality reporting built on ePHI, supporting integrity of data used for care and compliance analytics.

## Value

Detects failed or partial analytics warehouse loads that could silently skew quality reporting built on ePHI, supporting integrity of data used for care and compliance analytics.

## Implementation

(1) Map job names per environment; (2) create ITSI KPI for critical ETL health; (3) correlate with downstream dashboard freshness checks.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686), Splunk ITSI (1841).
• Ensure the following data sources are available: `index=etl` `sourcetype=sqlserver:agent_job` or Informatica/SSIS logs, `index=epic` Caboodle job status HEC.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map job names per environment; (2) create ITSI KPI for critical ETL health; (3) correlate with downstream dashboard freshness checks.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=etl sourcetype="sqlserver:agent_job" earliest=-48h (job_name="*CABOODLE*" OR job_name="*CLARITY*")
| eval ok=if(match(lower(run_status),"succeeded|success"),1,0)
| stats latest(run_status) as last_status latest(_time) as last_run sum(ok) as ok_runs count as runs by job_name, server_name
| eval health=if(ok_runs=runs,"OK","DEGRADED")
| where health!="OK"
| sort last_run
```
