# UC-22.10.24 · Medical Device Integration — Unapproved HL7 Feeds to EMPI

## Description

Detects new sending facilities or IP addresses pushing ADT/ORM traffic into the integration engine without change control, protecting access control around device-mediated ePHI flows.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes hl7 and sourcetypes hl7:receive, hl7:receive`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize HL7 with TA or `rex` consistently; (2) integrate Mirth/Interface engine connection table; (3) auto-ticket network security for rogue feeds. Indexes required: `index=hl7`; Sourcetypes: `sourcetype=hl7:receive`, `sourcetype=hl7:receive``; Lookups: `approved_hl7_senders`, `approved_hl7_senders.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Stream (1809) or `Splunk_TA_f5` / load balancer logs, Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=hl7` `sourcetype=hl7:receive` (`MSH-4`, `MSH-3`, `sending_ip`), `lookup approved_hl7_senders.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize HL7 with TA or `rex` consistently; (2) integrate Mirth/Interface engine connection table; (3) auto-ticket network security for rogue feeds.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hl7 sourcetype="hl7:receive" earliest=-7d
| rex field=_raw "^MSH\\|[^|]*\\|[^|]*\\|(?<sending_app>[^|]*)\\|(?<sending_facility>[^|]*)"
| stats earliest(_time) as first_seen latest(_time) as last_seen dc(PID) as patients by sending_ip, sending_facility, sending_app
| lookup approved_hl7_senders.csv sending_ip sending_facility OUTPUT approved
| where isnull(approved) OR approved="false"
| sort - patients
```
