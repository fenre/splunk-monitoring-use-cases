# 22.31.12 — Daily backup verification and restore test evidence

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Evidences ASD Essential Eight obligations for daily backup verification and restore test evidence using endpoint, identity, and cloud telemetry.

## Value

Gives the responsible team auditor-ready evidence for ASD E8 compliance by automating regulatory compliance evidence using indexes backup and sourcetypes rubrik:backup, veeam:job. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=backup sourcetype="veeam:job" OR sourcetype="rubrik:backup" earliest=-2d
| stats latest(job_status) as js by host, job_name
| where js!="Success"
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.dest | sort - count
```

## Implementation

(1) Ingest CrowdStrike policy posture via TA 5082 and map hosts to NDB and NZISM tiers. (2) Tune alerts with ACSD Essential Eight and APRA-aligned severities. (3) Coordinate privacy and security ownership for dual-reporting workflows. (4) Monthly export for privacy officer and prudential working groups. Indexes required: `index=backup`; Sourcetypes: `sourcetype=rubrik:backup`, `sourcetype=veeam:job`.
