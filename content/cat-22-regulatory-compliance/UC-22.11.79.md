# UC-22.11.79 · Quarterly Internal Scan Remediation Aging Buckets

## Description

Buckets open internal scan findings by age so quarterly rescan and fix cycles for high-risk issues remain on schedule and visible to management dashboards.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes vm and sourcetypes qualys:host. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Confirm Qualys timestamp format for `strptime`. (2) Map `SEVERITY_LEVEL` to PCI risk. (3) Executive stack chart weekly. (4) Align `90+d` alerts with remediation charter. Indexes required: `index=vm`; Sourcetypes: `sourcetype=qualys:host`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Qualys (2964).
• Ensure the following data sources are available: `index=vm` `qualys:host` with `FIRST_FOUND_DATETIME`, `STATUS`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Confirm Qualys timestamp format for `strptime`. (2) Map `SEVERITY_LEVEL` to PCI risk. (3) Executive stack chart weekly. (4) Align `90+d` alerts with remediation charter.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vm sourcetype="qualys:host" earliest=-120d
| where match(TAGS,"(?i)PCI|CDE)")
| where match(STATUS,"(?i)active|new|reopened)")
| eval age_days=round((now()-strptime(FIRST_FOUND_DATETIME,"%Y-%m-%dT%H:%M:%SZ"))/86400,1)
| eval bucket=case(age_days<=30,"0-30d", age_days<=60,"31-60d", age_days<=90,"61-90d",1=1,"90+d")
| stats count by bucket, SEVERITY_LEVEL
| sort bucket
```
