# UC-22.1.36 · GDPR Transfer Impact Assessment (TIA) Status for Third-Country Transfers

## Description

Following Schrems II, controllers must assess supplementary measures for transfers not covered by adequacy. This use case tracks whether a TIA exists, is current, and matches actual data flows observed in network telemetry — highlighting transfers to jurisdictions without completed TIAs.

## Value

Gives the responsible team auditor-ready evidence for GDPR compliance by automating regulatory compliance evidence. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Build `gdpr_transfer_register.csv` keyed by external service domains with TIA and mechanism fields; (2) enrich egress with country; (3) exclude adequacy-listed countries per legal list; (4) alert on high-volume transfers with missing TIA; (5) integrate with privacy office for TIA workflow updates.

## Detailed Implementation

Prerequisites

• Ensure the following data sources are available: `gdpr_transfer_register.csv`, CIM Network_Traffic, geolocation enrichment.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Build `gdpr_transfer_register.csv` keyed by external service domains with TIA and mechanism fields; (2) enrich egress with country; (3) exclude adequacy-listed countries per legal list; (4) alert on high-volume transfers with missing TIA; (5) integrate with privacy office for TIA workflow updates.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| tstats `summariesonly` sum(All_Traffic.bytes_out) as bytes_out
  from datamodel=Network_Traffic.All_Traffic
  where earliest=-30d NOT All_Traffic.dest_category IN ("eu_eea","uk_adequate")
  by All_Traffic.dest
| rename All_Traffic.* as *
| iplocation dest
| eval country=Country
| lookup gdpr_transfer_register.csv dest_domain OUTPUT tia_status, transfer_mechanism, data_categories
| where isnull(tia_status) OR tia_status!="completed"
| sort - bytes_out
| table dest, country, bytes_out, tia_status, transfer_mechanism, data_categories
```
