# 22.20.4 — CMMC AU.L2-3.3.2 — User-to-action traceability on CUI systems

**Criticality:** critical | **Difficulty:** intermediate | **Status:** verified

## Description

Ensures unique user traceability on CUI-classified systems by correlating authentication events (logon, explicit credential use, privilege escalation) to individual user accounts. CMMC AU.L2-3.3.2 requires that actions can be traced to individual users so that they can be held accountable.

## Value

Maps every action on CUI systems to a named individual user, enabling the organisation to demonstrate that shared/generic accounts are not used for CUI access and that all activity is attributable.

## SPL

```spl
index=wineventlog sourcetype=WinEventLog:Security EventCode IN (4624, 4648, 4672) earliest=-24h
| lookup cui_hosts.csv ComputerName AS dest OUTPUT cui_flag
| where cui_flag="true"
| eval user=coalesce(TargetUserName, SubjectUserName)
| eval logon_type=case(Logon_Type="2","Interactive", Logon_Type="3","Network", Logon_Type="10","RemoteInteractive", true(),Logon_Type)
| stats dc(dest) AS systems_accessed count AS actions values(logon_type) AS logon_types BY user
| where user!="-" AND user!="SYSTEM" AND user!="*$"
| sort - actions
| table user, systems_accessed, actions, logon_types
```

## CIM SPL

```spl
| tstats count from datamodel=Authentication WHERE Authentication.dest IN (cui_hosts_lookup) BY Authentication.user, Authentication.dest, Authentication.action | sort - count
```

## Implementation

(1) Verify data sources: `WinEventLog:Security`, `cui_hosts.csv` lookup; (2) Maintain CUI-scoped lookups; (3) Schedule search at compliance-aligned interval; (4) Route findings to ticketing queue; (5) Retain exports per records management policy.

## Visualization

Table of users with system count and action count. Bar chart of logon types per user. Single value for total unique users on CUI systems.

## Compliance

- **CMMC** 2.0: AU.L2-3.3.2 (partial) — Directly maps user identities to system actions on CUI hosts — the core requirement of AU.L2-3.3.2 unique user traceability.

## Known False Positives

Service accounts and machine accounts (ending with $) generate high volumes of logon events — these are filtered by the search. Scheduled tasks running under named accounts are expected.
