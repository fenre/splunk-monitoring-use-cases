# UC-22.22.25 · Major Incident Notification Readiness to NCA (Payment Service)

## Description

Supports incident governance, customer communications evidence, and supervisory reporting workflows associated with payment service continuity and fraud statistics.

## Value

Gives the responsible team auditor-ready evidence for PSD2 compliance by automating regulatory compliance evidence using indexes payments and sourcetypes pay:incident:milestone. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review. Indexes required: `index=payments`; Sourcetypes: `sourcetype=pay:incident:milestone`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk ITSI (1841).
• Ensure the following data sources are available: `index=payments` `pay:incident:milestone` — incident_id, milestone, severity, impacted_psu_estimate.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=payments sourcetype="pay:incident:milestone" earliest=-90d
| eval inc_tag="nca"
| stats earliest(eval(if(milestone=="detected",_time,null()))) as t0,
        earliest(eval(if(milestone=="nca_notified",_time,null()))) as t1,
        dc(incident_id) as incidents by inc_tag, severity
| eval hours_to_nca=round((t1-t0)/3600,2)
| eval uc_id="22.22.25"
| where inc_tag="nca" AND (isnull(t1) OR hours_to_nca>24)
| sort - hours_to_nca
| table uc_id, severity, incidents, t0, t1, hours_to_nca
```
