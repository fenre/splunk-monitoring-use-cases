# UC-22.14.72 · Session Authenticity — Token Replay Across Geographies

## Description

Detects same refresh token used from distant geolocations within short windows for SC-23 session authenticity.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes o365 and sourcetypes ms:o365:management. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Confirm operation strings; (2) reduce noise with corporate travel; (3) integrate with IR-4; (4) force re-auth playbooks. Indexes required: `index=o365`; Sourcetypes: `sourcetype=ms:o365:management`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365 (Splunkbase 4055).
• Ensure the following data sources are available: `ms:o365:management` token events if available, or proxy OAuth logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=o365 sourcetype=ms:o365:management earliest=-24d Operation="Token issued*"
| iplocation ClientIP
| stats earliest(_time) as t1, latest(_time) as t2, values(Country) as countries, values(ClientIP) as ips by UserId, SessionId
| eval dt=t2-t1
| where mvcount(countries)>1 AND dt < 7200
```
