# UC-22.14.17 · Access Enforcement — Unauthorized Access Attempts to Sensitive Shares

## Description

Surfaces repeated failed object access attempts that indicate AC-3 access enforcement gaps or misconfigured ACLs.

## Value

Surfaces repeated failed object access attempts that indicate AC-3 access enforcement gaps or misconfigured ACLs.

## Implementation

(1) Enable granular SACLs on crown-jewel shares; (2) normalize `ShareName` fields; (3) tune threshold per sensitivity; (4) integrate with case management for access reviews.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (Splunkbase 742).
• Ensure the following data sources are available: `WinEventLog:Security` EventCode 4656/4663 (object access) when auditing enabled.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enable granular SACLs on crown-jewel shares; (2) normalize `ShareName` fields; (3) tune threshold per sensitivity; (4) integrate with case management for access reviews.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sec sourcetype=WinEventLog:Security EventCode IN (4656,4663) earliest=-24h
| search Object_Type="File" ShareName="\\*SECRET*"
| stats count by user, ShareName, ObjectName, host
| where count>25
```
