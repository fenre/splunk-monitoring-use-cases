# UC-22.14.11 · Audit Generation Coverage for Critical Network Controls

## Description

Verifies that boundary devices continue generating audit records for allow/deny decisions, evidencing AU-12 audit generation for network events.

## Value

Verifies that boundary devices continue generating audit records for allow/deny decisions, evidencing AU-12 audit generation for network events.

## Implementation

(1) Fix timestamp parsing per TA — sample `last` field; (2) tune gap threshold per HA pair; (3) alert if firewall stops logging; (4) map to firewall inventory CM-8.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Palo Alto Networks Add-on for Splunk (Splunkbase 2757), Splunk Common Information Model Add-on (Splunkbase 1621).
• Ensure the following data sources are available: `pan:traffic`, `pan:threat` or CIM Network_Traffic / Intrusion_Detection sources.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Fix timestamp parsing per TA — sample `last` field; (2) tune gap threshold per HA pair; (3) alert if firewall stops logging; (4) map to firewall inventory CM-8.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=net (sourcetype=pan:traffic OR sourcetype=pan:threat) earliest=-24h
| stats earliest(_time) as first, latest(_time) as last, count by host, sourcetype
| eval gap_minutes=round((now()-strptime(last,"%Y-%m-%d %H:%M:%S %z"))/60,1)
| eval gap_minutes=if(isnull(gap_minutes), round((now()-last)/60,1), gap_minutes)
| where gap_minutes>30
```
