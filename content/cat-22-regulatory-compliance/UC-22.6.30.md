# UC-22.6.30 · Malware Protection — AV Engine Disabled or Out of Date Events

## Description

Flags endpoints reporting disabled protection or stale pattern files for malware control assurance.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes edr and sourcetypes defender:health. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize health vocabulary; (2) auto-remediate via SOAR; (3) weekly compliance export. Indexes required: `index=edr`; Sourcetypes: `sourcetype=defender:health`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Microsoft Defender, CrowdStrike.
• Ensure the following data sources are available: `defender:health` or `crowdstrike:sensor`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize health vocabulary; (2) auto-remediate via SOAR; (3) weekly compliance export.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=edr sourcetype=defender:health earliest=-1d
| where protection_status!="Active" OR signature_age_days>7
| stats latest(protection_status) as status, max(signature_age_days) as sig_age by host
```
