# UC-22.14.34 · Flaw Remediation SLA Tracking from Vulnerability Scans

## Description

Measures age of open critical vulnerabilities for SI-2 flaw remediation timeliness.

## Value

Measures age of open critical vulnerabilities for SI-2 flaw remediation timeliness.

## Implementation

(1) Normalize `first_found` timestamp format from your TA; (2) join CMDB owner; (3) alert on SLA breach; (4) feed POA&M (CA-5).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Tenable Add-On for Splunk (Splunkbase 4060), Splunk Enterprise Security (Splunkbase 263).
• Ensure the following data sources are available: `tenable:sc` or `tenable:vuln` plugin outputs, ES `index=vulnerabilities` if normalized.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize `first_found` timestamp format from your TA; (2) join CMDB owner; (3) alert on SLA breach; (4) feed POA&M (CA-5).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vulns sourcetype=tenable:vuln earliest=-7d severity IN ("Critical","High")
| eval age_days=round((now()-strptime(first_found,"%Y-%m-%d"))/86400,1)
| where age_days>30 AND state!="Fixed"
| stats max(age_days) as oldest_open, values(plugin_id) as plugins by host, severity
```
