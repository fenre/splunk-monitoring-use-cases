# UC-22.10.52 · BAA Compliance Evidence — Control Attestations vs Technical Telemetry

## Description

Joins annual BA questionnaire answers (encryption, logging) with Splunk-ingested technical proof points so subcontractor compliance is evidence-based, not checkbox-only.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes metrics, privacy and sourcetypes hipaa:ba_control_metric, hipaa:ba_control_metric`, hipaa:baa_attestation. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map each attestation row to an automated metric (for example TLS coverage percent from vulnerability scans); (2) legal sets materiality thresholds; (3) feed gaps into vendor risk reviews. Indexes required: `index=metrics`, `index=privacy`; Sourcetypes: `sourcetype=hipaa:ba_control_metric`, `sourcetype=hipaa:ba_control_metric``, `sourcetype=hipaa:baa_attestation`, `sourcetype=hipaa:baa_attestation``.

## Detailed Implementation

Prerequisites

• Ensure the following data sources are available: `index=privacy` `sourcetype=hipaa:baa_attestation` (`vendor_id`, `control_id`, `answer`), `index=metrics` `sourcetype=hipaa:ba_control_metric` (`vendor_id`, `control_id`, `measured_value`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map each attestation row to an automated metric (for example TLS coverage percent from vulnerability scans); (2) legal sets materiality thresholds; (3) feed gaps into vendor risk reviews.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="hipaa:baa_attestation" earliest=-365d
| where answer IN ("Yes","FullyImplemented")
| join vendor_id, control_id [
    search index=metrics sourcetype="hipaa:ba_control_metric" earliest=-30d
    | stats latest(measured_value) as measured by vendor_id, control_id ]
| eval gap=if(isnull(measured) OR measured<95,"Evidence_Gap","OK")
| where gap!="OK"
| table vendor_id, control_id, answer, measured, gap
```
