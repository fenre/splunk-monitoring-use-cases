# UC-22.10.2 · HIPAA Risk Management — Control Deficiency Tracking

## Description

Tracks open HIPAA control gaps from assessments and pen tests through remediation so risk is reduced to reasonable and appropriate levels, not just documented once.

## Value

Tracks open HIPAA control gaps from assessments and pen tests through remediation so risk is reduced to reasonable and appropriate levels, not just documented once.

## Implementation

(1) Define control_family values mapping to 164.308/310/312 families; (2) ingest weekly GRC CSV/HEC; (3) alert on `overdue_controls>0`; (4) require closure notes referencing implemented safeguard.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `` `notable` `` or `index=grc` HEC JSON (`control_id`, `framework`, `status`, `due_date`, `owner`) from GRC exports.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Define control_family values mapping to 164.308/310/312 families; (2) ingest weekly GRC CSV/HEC; (3) alert on `overdue_controls>0`; (4) require closure notes referencing implemented safeguard.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype="grc:hipaa_control" earliest=-90d framework="HIPAA_Security_Rule"
| eval overdue=if(status!="Closed" AND now()>strptime(due_date,"%Y-%m-%d"),1,0)
| stats count as open_controls sum(overdue) as overdue_controls by owner, control_family
| sort - overdue_controls
```
