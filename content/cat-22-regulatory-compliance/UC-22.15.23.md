# UC-22.15.23 · Audit Index Growth vs Licensed Retention (Capacity Risk)

## Description

Forecasts `ot_audit` disk usage to avoid silent loss of security audit records before policy retention is met.

## Value

Forecasts `ot_audit` disk usage to avoid silent loss of security audit records before policy retention is met.

## Implementation

(1) Set `maxTotalDataSizeMB` with headroom; (2) route verbose protocols to warm/cold tier; (3) expand frozen path; (4) alert if `days_to_full`<90.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Cloud REST.
• Ensure the following data sources are available: `/services/data/indexes` for `ot_audit`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Set `maxTotalDataSizeMB` with headroom; (2) route verbose protocols to warm/cold tier; (3) expand frozen path; (4) alert if `days_to_full`<90.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| rest /services/data/indexes splunk_server=local count=0 title=ot_audit
| eval gb=currentDBSizeMB/1024, days=round(frozenTimePeriodInSecs/86400,1)
| eval daily_gb=gb/30
| eval days_to_full=if(daily_gb>0, round((maxTotalDataSizeMB/1024)/daily_gb,1), null())
| table title, gb, days, daily_gb, days_to_full
```
