# 22.15.23 — Audit Index Growth vs Licensed Retention (Capacity Risk)

**Criticality:** medium | **Difficulty:** intermediate | **Status:** verified

## Description

Forecasts `ot_audit` disk usage to avoid silent loss of security audit records before policy retention is met.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
| rest /services/data/indexes splunk_server=local count=0 title=ot_audit
| eval gb=currentDBSizeMB/1024, days=round(frozenTimePeriodInSecs/86400,1)
| eval daily_gb=gb/30
| eval days_to_full=if(daily_gb>0, round((maxTotalDataSizeMB/1024)/daily_gb,1), null())
| table title, gb, days, daily_gb, days_to_full
```

## Implementation

(1) Set `maxTotalDataSizeMB` with headroom; (2) route verbose protocols to warm/cold tier; (3) expand frozen path; (4) alert if `days_to_full`<90.

## Visualization

Single value, Gauge, Table. Key fields: `daily_gb`, `days`, `days_to_full`, `gb`, `title`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 2.9 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 2.9.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
