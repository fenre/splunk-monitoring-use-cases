# UC-22.2.38 · NIS2 Industrial Control System Patching and Change Evidence

## Description

Patch and change management for ICS must balance availability with security. This use case tracks firmware updates, PLC logic downloads, and antivirus signature updates for OT endpoints — flagging systems past approved patch cycles or changes without linked change records.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes itsm, ot_changes and sourcetypes snow:change_request. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Forward vendor-specific OT change events via HEC; (2) require ServiceNow change tickets for logic downloads; (3) alert when observed firmware diverges from approved baseline; (4) coordinate outage windows with operations; (5) retain evidence for sector regulator inspections. Indexes required: `index=itsm`, `index=ot_changes`; Sourcetypes: `sourcetype=snow:change_request`; Lookups: `nis2_ot_patch_policy`, `nis2_ot_patch_policy.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_changes earliest=-90d
| stats latest(firmware_version) as observed_fw latest(_time) as last_change by asset_id
| lookup nis2_ot_patch_policy.csv asset_id OUTPUT required_fw_version last_approved_change_epoch
| eval fw_compliant=if(observed_fw=required_fw_version,1,0)
| eval change_stale=if(last_change < last_approved_change_epoch,1,0)
| where fw_compliant=0 OR change_stale=1
| table asset_id, observed_fw, required_fw_version, last_change, change_stale
```
