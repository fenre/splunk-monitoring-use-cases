# UC-22.11.52 · Generic Account Prohibition — `admin` / `root` Interactive Success on CDE

## Description

Alerts on successful interactive use of generic superuser accounts on in-scope Linux payment hosts, enforcing unique IDs and eliminating shared root for routine administration.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes linux and sourcetypes linux:secure. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Prefer `auditd` TYPE=USER_LOGIN for richer fields if available. (2) Allow break-glass from bastion subnet only via `where`. (3) Immediate SOC page. (4) Quarterly attestation of zero violations goal. Indexes required: `index=linux`; Sourcetypes: `sourcetype=linux:secure`; Lookups: `pci_linux_payment_hosts`, `pci_linux_payment_hosts.csv`.

## Detailed Implementation

Step 1 — Configure data collection
(1) Prefer `auditd` TYPE=USER_LOGIN for richer fields if available. (2) Allow break-glass from bastion subnet only via `where`. (3) Immediate SOC page. (4) Quarterly attestation of zero violations goal.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=linux sourcetype="linux:secure" earliest=-24h
| where match(_raw,"Accepted password|Accepted publickey")
| rex "for (?<acct>\\S+) from (?<src_ip>\\d+\\.\\d+\\.\\d+\\.\\d+)"
| where acct IN ("root","admin","Administrator")
| lookup pci_linux_payment_hosts.csv host OUTPUT pci_host
| where pci_host="true"
| stats count by host, acct, src_ip
| sort - count
```
