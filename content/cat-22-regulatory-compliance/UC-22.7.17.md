# UC-22.7.17 · Software Bill of Materials Signals — Container Image Digests

## Description

Correlates Kubernetes workload image digests to approved golden images for containerized asset inventory depth.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes containers and sourcetypes kube:*. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Export digest list from artifact registry; (2) alert on unknown digests; (3) feed asset register; (4) map to business owner namespace labels. Indexes required: `index=containers`; Sourcetypes: `sourcetype=kube:*`; Lookups: `approved_container_images`, `approved_container_images.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: OTel k8s attributes or `kube:audit`.
• Ensure the following data sources are available: `kube:*` sourcetype.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Export digest list from artifact registry; (2) alert on unknown digests; (3) feed asset register; (4) map to business owner namespace labels.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=containers sourcetype=kube:* earliest=-6h
| stats latest(image_id) as image by pod, namespace
| lookup approved_container_images.csv image OUTPUT tier
| where isnull(tier)
```
