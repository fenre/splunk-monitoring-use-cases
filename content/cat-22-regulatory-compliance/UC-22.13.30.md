# UC-22.13.30 · Patch Installation or Mitigation within 35-Day Window

## Description

We highlight applicable patches past thirty-five days from completed evaluation without apply or approved mitigation plan.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes patch and sourcetypes patch:status, patch:status`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align `eval_completed` with official evaluation logs. (2) Import mitigation plans from GRC. (3) Daily alert to server owners. (4) Ticket exceptions through CIP-003. Indexes required: `index=patch`; Sourcetypes: `sourcetype=patch:status`, `sourcetype=patch:status``; Lookups: `cip007_mitigation_plans`, `cip007_mitigation_plans.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=patch` `sourcetype=patch:status`, `cip007_mitigation_plans.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align `eval_completed` with official evaluation logs. (2) Import mitigation plans from GRC. (3) Daily alert to server owners. (4) Ticket exceptions through CIP-003.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=patch sourcetype=patch:status bes_asset=1 earliest=-120d
| eval eval_epoch=strptime(eval_completed,"%Y-%m-%d")
| eval due_epoch=eval_epoch+(35*86400)
| eval applied=if(match(status,"(?i)installed"),1,0)
| lookup cip007_mitigation_plans.csv patch_id OUTPUT plan_id approved_until
| eval mitigated=if(isnotnull(plan_id),1,0)
| where applied=0 AND mitigated=0 AND now()>due_epoch
| table host, patch_id, eval_completed, due_epoch, status
```
