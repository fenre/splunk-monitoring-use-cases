# 22.37.4 — Purpose-limitation enforcement — processing not matching declared purpose

**Criticality:** high | **Difficulty:** advanced | **Status:** draft

## Description

Observes access tokens carrying a declared processing purpose and compares that purpose to the declared purposes of the system being accessed. Any mismatch is evidence that the purpose-limitation principle is not being enforced.

## Value

Converts an otherwise-abstract regulatory principle into a continuously-measurable control with evidentiary value.

## SPL

```spl
index=app_activity earliest=-1h token_purpose=*
| lookup purpose_catalogue.csv system_id OUTPUT allowed_purposes
| eval ok=if(match(allowed_purposes,token_purpose),1,0)
| where ok=0
| stats count values(user) as users values(subject_token) as subjects by system_id token_purpose allowed_purposes
| table system_id token_purpose allowed_purposes count users subjects
```

## Implementation

(1) Mint OAuth/OIDC tokens with a purpose claim; (2) publish purpose_catalogue.csv keyed by system_id; (3) run hourly; (4) alert DPO on mismatches and drive remediation via SOAR. Indexes required: `index=app_activity`.

## Visualization

Sankey of token-purpose → system, heatmap of mismatches by system, single-value 'mismatch events (24h)'.

## Compliance

- **GDPR** 2016/679: Art.5(1)(b) (partial) — Art.5(1)(b) 'purpose limitation' requires technical enforcement wherever feasible; this UC detects observable mismatches.
- **CCPA** CPRA (as amended): §1798.100(b) (partial) — CPRA §1798.100(b) requires use limitation to disclosed purposes; a mismatch is direct evidence of a deficiency.
- **LGPD** Lei nº 13.709/2018: Art.6 (partial) — LGPD Art.6(I) 'finalidade' mirrors purpose limitation; same surveillance value.
- **UK GDPR** post-Brexit: Art.5(1)(b) (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.5(1)(b).

## Known False Positives

Shared service accounts without per-request purpose context cannot be evaluated; filter them out via service_account_allowlist.csv.
