# UC-22.15.13 · Stale or Shared OT Service Accounts

## Description

Surfaces OT accounts with weak rotation, excessive group breadth, or shared naming patterns inconsistent with IACS account management practice.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ad_snapshot, windows and sourcetypes ad:user. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ingest periodic LDAP snapshots to CSV; (2) define password rotation SLAs by tier; (3) eliminate shared accounts; (4) track exceptions with expiry dates. Indexes required: `index=ad_snapshot`, `index=windows`; Sourcetypes: `sourcetype=ad:user`.

## Detailed Implementation

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=windows EventCode IN ("4723","4724") earliest=-180d
| eval account=Target_User_Name
| where match(account,"(?i)scada|plc|hmi|svc_ot|batch")
| stats latest(_time) as last_pw_change by account
| eval days_since=round((now()-last_pw_change)/86400,0)
| where days_since>90 OR isnull(last_pw_change)
| join type=left account [ search index=ad_snapshot sourcetype="ad:user" | fields account, memberOf | eval grp_count=mvcount(memberOf) ]
| table account, days_since, grp_count
```
