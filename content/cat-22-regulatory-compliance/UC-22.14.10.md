# UC-22.14.10 · Audit Record Retention Compliance vs Policy

## Description

Compares configured frozen retention to policy targets for systems of record logs supporting AU-11 audit record retention.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Build `nist_retention_policy.csv` with index or prefix to required days; (2) reconcile with offline archive retention; (3) alert on non-compliance; (4) export for POA&M tracking.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: `/services/data/indexes`, lookup `nist_retention_policy.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Build `nist_retention_policy.csv` with index or prefix to required days; (2) reconcile with offline archive retention; (3) alert on non-compliance; (4) export for POA&M tracking.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| rest /services/data/indexes splunk_server=local count=0
| search disabled=0
| eval retention_days=round(frozenTimePeriodInSecs/86400,1)
| lookup nist_retention_policy.csv title OUTPUT required_min_days, system_category
| eval compliant=if(retention_days+0>=required_min_days+0,"Yes","No")
| where compliant="No" OR isnull(required_min_days)
```
