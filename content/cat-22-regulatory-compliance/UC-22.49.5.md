# UC-22.49.5 · Cryptographic erasure attestation — per-asset destruction evidence

## Description

Correlates asset disposal events with the matching KMS key-destruction events to provide verifiable erasure evidence.

## Value

Solves the 'how do I prove we actually erased that data?' question that every DPO faces.

## Implementation

(1) Tag asset-disposal events with asset_id and key_id; (2) ingest KMS DeleteKey; (3) run daily.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: Asset disposal workflow events, KMS key destruction events..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag asset-disposal events with asset_id and key_id; (2) ingest KMS DeleteKey; (3) run daily.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=assets sourcetype=asset:disposal earliest=-7d method=cryptographic_erasure
| join asset_id [search index=kms action=DeleteKey earliest=-8d | fields asset_id key_id _time | rename _time as key_deleted_at]
| eval delta=abs(_time-key_deleted_at)
| where isnull(key_deleted_at) OR delta>86400
| table asset_id key_id _time key_deleted_at delta
```
