# UC-22.4.4 · CCPA Right to Correct Inaccurate Personal Information

## Description

Businesses must use commercially reasonable efforts to correct inaccurate personal information upon a verifiable request. This search tracks correction tickets from intake through closure so you can prove timely handling and reduce risk of complaints to the California Attorney General for mishandled consumer rights workflows.

## Value

Gives the responsible team auditor-ready evidence for CCPA/CPRA compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:sc_req_item. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align `cat_item` and keyword filters with your ServiceNow CCPA correction catalog items; (2) route agent queues into `assignment_group` for accountability dashboards; (3) join optional `consumer_id_hash` field if present for deduplication; (4) schedule daily and alert when `nearing_breach>0`; (5) export monthly evidence for privacy counsel. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:sc_req_item`.

## Detailed Implementation

Step 1 — Configure data collection
(1) Align `cat_item` and keyword filters with your ServiceNow CCPA correction catalog items; (2) route agent queues into `assignment_group` for accountability dashboards; (3) join optional `consumer_id_hash` field if present for deduplication; (4) schedule daily and alert when `nearing_breach>0`; (5) export monthly evidence for privacy counsel.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="snow:sc_req_item" earliest=-90d
    (short_description="*correct*" OR short_description="*inaccurate*" OR cat_item="*CCPA*Correct*")
| eval opened_epoch=strptime(opened_at, "%Y-%m-%d %H:%M:%S")
| eval closed_epoch=if(isnotnull(closed_at), strptime(closed_at, "%Y-%m-%d %H:%M:%S"), null())
| eval age_days=round((now()-opened_epoch)/86400, 1)
| eval sla_days=45
| eval at_risk=if(isnull(closed_epoch) AND age_days>(sla_days-7), 1, 0)
| stats count as requests, sum(at_risk) as nearing_breach by assignment_group, state
| sort - requests
```
