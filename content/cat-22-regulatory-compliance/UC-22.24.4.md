# UC-22.24.4 · Conformity Assessment Evidence Index Freshness

## Description

Supports qualified trust service auditing, wallet security monitoring, and PKI integrity evidence expected under eIDAS trust frameworks.

## Value

Supports qualified trust service auditing, wallet security monitoring, and PKI integrity evidence expected under eIDAS trust frameworks.

## Implementation

(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=trust` `qtsp:conformity:record` — trust_service, serial, not_after, ocsp_url, http_status, wallet_id.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=trust sourcetype="qtsp:conformity:record" earliest=-7d
| eval eidas_tag="ca"
| eval exp_epoch=strptime(not_after,"%Y-%m-%dT%H:%M:%SZ")
| eval days_to_exp=round((exp_epoch-now())/86400,1)
| stats count as events, min(days_to_exp) as min_days, sum(eval(http_status>=400)) as http_err by trust_service, eidas_tag
| eval uc_id="22.24.4"
| where min_days < 30 OR http_err>0 OR eidas_tag="susp"
| sort min_days
| head 200
```
