# 22.15.41 — Duplicate IP or MAC on OT Switch — Segmentation Violation

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Surfaces ARP/ND anomalies that indicate bridging or misconfiguration undermining component-level network segmentation.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes net, ot_sec and sourcetypes cisco:ios, claroty:alert. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=net sourcetype="cisco:ios" "%Duplicate address%" OR "%ARP collision%" earliest=-7d
| append [ search index=ot_sec sourcetype="claroty:alert" signature="*ARP*" earliest=-7d ]
| stats count by host, vlan, signature, src_ip
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.dest, All_Traffic.src | sort - count
```

## Implementation

(1) Enable DHCP snooping; (2) BPDU guard on access ports; (3) correlate with change tickets; (4) physical inspection for rogue bridges. Indexes required: `index=net`, `index=ot_sec`; Sourcetypes: `sourcetype=cisco:ios`, `sourcetype=claroty:alert`.

## Visualization

Table, Topology panel (if ITSI), Time chart. Key fields: `host`, `signature`, `src_ip`, `vlan`.

## Compliance

- **IEC 62443** 2013-ongoing: CR 5.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CR 5.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
