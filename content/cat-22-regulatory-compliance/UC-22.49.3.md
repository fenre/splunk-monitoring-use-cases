# UC-22.49.3 · Litigation-hold override audit — holds applied/released without ticket

## Description

Detects unauthorised application or release of litigation holds — both of which are common sources of adverse inference in discovery.

## Value

Prevents silent tampering with evidence preservation.

## Implementation

(1) Ingest hold-change audit events from data stores and from the legal-hold tool; (2) run hourly.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: See subcategory preamble..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest hold-change audit events from data stores and from the legal-hold tool; (2) run hourly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=datastore sourcetype=hold:change earliest=-24h
| join hold_id [search index=legal sourcetype=hold:ticket | fields hold_id ticket_id requested_by]
| where isnull(ticket_id)
| table _time hold_id record_id action ticket_id
```
