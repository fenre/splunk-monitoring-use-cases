# UC-22.3.32 · DORA Outsourcing Registers — Sub-Processor Notification Lag vs Contractual Notice Period

## Description

Outsourcing arrangements require visibility into sub-processing changes. Comparing provider sub-processor announcement dates to your internal register update timestamps shows contractual notification periods were honored and documented.

## Value

Outsourcing arrangements require visibility into sub-processing changes. Comparing provider sub-processor announcement dates to your internal register update timestamps shows contractual notification periods were honored and documented.

## Implementation

(1) Set `14` to your contractual days; (2) legal owns register updates—Splunk flags drift; (3) exclude immaterial vendors via lookup; (4) integrate CLM tool when available; (5) export for RoI reconciliation.

## Detailed Implementation

Prerequisites

• Ensure the following data sources are available: `index=vendor` `sourcetype="vendor:subproc_notice"` (provider_id, subprocessor_name, effective_date, notice_received_at); `roi_subprocessor.csv` (provider_id, subprocessor_name, registered_at).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Set `14` to your contractual days; (2) legal owns register updates—Splunk flags drift; (3) exclude immaterial vendors via lookup; (4) integrate CLM tool when available; (5) export for RoI reconciliation.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vendor sourcetype="vendor:subproc_notice" earliest=-400d
| eval notice=strptime(notice_received_at,"%Y-%m-%d"), eff=strptime(effective_date,"%Y-%m-%d")
| join type=left provider_id, subprocessor_name [
    | inputlookup roi_subprocessor.csv
    | eval reg=strptime(registered_at,"%Y-%m-%d")
    | fields provider_id, subprocessor_name, reg
  ]
| eval lag_days=round((reg-notice)/86400,1)
| where lag_days>14 OR isnull(reg)
| table provider_id, subprocessor_name, notice_received_at, registered_at, lag_days
```
