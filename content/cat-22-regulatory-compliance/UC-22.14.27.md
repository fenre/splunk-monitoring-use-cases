# UC-22.14.27 · Device Identification for Corporate-Managed Endpoints

## Description

Correlates certificate- or MDM-based device IDs with authentication events for IA-3 device identification and compliance.

## Value

Correlates certificate- or MDM-based device IDs with authentication events for IA-3 device identification and compliance.

## Implementation

(1) Build `corp_managed_devices.csv` from Intune/JAMF export; (2) refresh nightly; (3) alert on unknown DeviceId with privileged users; (4) document attribute mapping in SSP.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365 (Splunkbase 4055), Splunk Add-on for Microsoft Windows (Splunkbase 742).
• Ensure the following data sources are available: `ms:o365:management` device fields, `WinEventLog:Security` 4768 (Kerberos) with device claims if enabled.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Build `corp_managed_devices.csv` from Intune/JAMF export; (2) refresh nightly; (3) alert on unknown DeviceId with privileged users; (4) document attribute mapping in SSP.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=o365 sourcetype=ms:o365:management earliest=-24h DeviceId=*
| stats latest(OperatingSystem) as os, dc(UserId) as users by DeviceId
| lookup corp_managed_devices.csv DeviceId OUTPUT owner, compliance_state
| where compliance_state!="compliant" OR isnull(compliance_state)
```
