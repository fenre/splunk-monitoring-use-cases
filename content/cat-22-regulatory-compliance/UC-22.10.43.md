# UC-22.10.43 · Breach Discovery — Time-to-Detect from First PHI Indicator

## Description

Measures elapsed time between earliest technical indicator of a potential PHI incident and formal incident creation so discovery timelines support Breach Notification Rule evidence.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes dlp and sourcetypes ms:o365:management`, o365:dlp. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Tag correlation searches that create HIPAA candidate notables; (2) normalize UTC; (3) executive dashboard for legal discovery SLA. Indexes required: `index=dlp`; Sourcetypes: `sourcetype=ms:o365:management``, `sourcetype=o365:dlp`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `` `notable` `` (`_time`, `rule_name`, `status`), `index=dlp` `sourcetype=ms:o365:management` or `ms:o365:dlp` (`IncidentId`, `CreatedTime`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag correlation searches that create HIPAA candidate notables; (2) normalize UTC; (3) executive dashboard for legal discovery SLA.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
`notable` HIPAA_Breach_Candidate=1 earliest=-90d
| eval detect_time=_time
| join IncidentId type=left [
    search index=dlp sourcetype="o365:dlp" earliest=-90d
    | eval first_indicator=strptime(CreatedTime,"%Y-%m-%dT%H:%M:%SZ")
    | stats min(first_indicator) as first_dlp by IncidentId ]
| eval hours_to_detect=round((detect_time-first_dlp)/3600,2)
| table IncidentId, first_dlp, detect_time, hours_to_detect, rule_name, status
| sort - hours_to_detect
```
