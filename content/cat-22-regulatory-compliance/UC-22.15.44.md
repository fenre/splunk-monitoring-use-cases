# 22.15.44 — Redundant Controller Failover During Attack Window

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Correlates IDS alerts with DCS redundancy state changes to evidence availability protections via backup paths.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes hist_alarms, ot_sec and sourcetypes dcs:redundancy, nozomi:alerts. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
(index=hist_alarms sourcetype="dcs:redundancy" message="*failover*") OR (index=ot_sec sourcetype="nozomi:alerts" (severity="high" OR severity="critical" OR match(severity,"(?i)^high")))
| eval key=coalesce(controller_pair, dest_ip)
| transaction key maxspan=10m
| where mvcount(sourcetype)>1
| table _time, key, duration, sourcetype
```

## Implementation

(1) Normalize timestamps; (2) validate expected failover tests; (3) exclude maintenance windows; (4) post-incident review template. Indexes required: `index=hist_alarms`, `index=ot_sec`; Sourcetypes: `sourcetype=dcs:redundancy`, `sourcetype=nozomi:alerts`.

## Visualization

Timeline, Table, Single value (correlated events). Key fields: `duration`, `key`, `sourcetype`.

## Compliance

- **IEC 62443** 2013-ongoing: CR 7.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CR 7.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
