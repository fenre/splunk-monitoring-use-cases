# 22.13.11 — Electronic Access Authorization Record Coverage for PAM Sessions

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We verify CyberArk sessions target systems that have a valid, unexpired access authorization on file for the user.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes pam and sourcetypes cyberark:ptalog, cyberark:ptalog`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=pam sourcetype=cyberark:ptalog action="Logon" earliest=-24h
| lookup cip004_access_authorizations.csv user, address OUTPUT auth_id expires
| eval exp_epoch=strptime(expires,"%Y-%m-%d")
| where isnull(auth_id) OR now()>exp_epoch
| stats count by user, address, safe
| sort - count
```

## Implementation

(1) Normalize `address` to the same key as the authorization register. (2) Nightly reconciliation job. (3) Feed hits to access governance. (4) Attach export to quarterly access review. Indexes required: `index=pam`; Sourcetypes: `sourcetype=cyberark:ptalog`, `sourcetype=cyberark:ptalog``; Lookups: `cip004_access_authorizations`, `cip004_access_authorizations.csv`.

## Visualization
