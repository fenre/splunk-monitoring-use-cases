# UC-22.15.6 · Control System Continuity — Historian Export and OT Backup Health

## Description

Detects missed historian snapshots or failed OT-tier backups that undermine recovery time objectives for control systems.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes backup, ot_hist and sourcetypes commvault:job, hist:export_status, veeam:JobSession. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Emit structured export rows from each historian interface; (2) tag backup jobs covering OT VMs; (3) tune thresholds to RPO; (4) link runbook for cold standby DCS. Indexes required: `index=backup`, `index=ot_hist`; Sourcetypes: `sourcetype=commvault:job`, `sourcetype=hist:export_status`, `sourcetype=veeam:JobSession`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Universal Forwarder on backup/historian bridge hosts.
• Ensure the following data sources are available: `index=ot_hist` export status; `index=backup` Commvault/Veeam.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Emit structured export rows from each historian interface; (2) tag backup jobs covering OT VMs; (3) tune thresholds to RPO; (4) link runbook for cold standby DCS.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_hist sourcetype="hist:export_status" earliest=-72h
| stats latest(_time) as last_ok, latest(status) as st by plant, historian_id
| eval hours_since=round((now()-last_ok)/3600,1)
| where st!="SUCCESS" OR hours_since>26
| append [ search index=backup sourcetype="veeam:JobSession" OR sourcetype="commvault:job" earliest=-72h | where match(JobName,"(?i)OT|Historian|VMware_OT") | stats latest(Result) as st by JobName | where st!="Success" ]
| sort - hours_since
```
