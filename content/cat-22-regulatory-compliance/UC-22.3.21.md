# UC-22.3.21 · DORA ICT Concentration — Single-Provider Spend and Workload Share Thresholds

## Description

Concentration in a single ICT provider creates systemic dependency. Tracking spend, contract value, and mapped production workloads against internal limits evidences ongoing measurement of concentration risk for management and supervisors.

## Value

Concentration in a single ICT provider creates systemic dependency. Tracking spend, contract value, and mapped production workloads against internal limits evidences ongoing measurement of concentration risk for management and supervisors.

## Implementation

(1) Set `share_pct` threshold per board risk appetite; (2) join `cmdb_provider_map` to count critical CIs per vendor; (3) quarterly board pack export; (4) reconcile vendor_id to Register of Information keys; (5) document mitigation (secondary vendor) in GRC.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (Splunkbase 2686), HTTP Event Collector from FinOps.
• Ensure the following data sources are available: `index=vendor` `sourcetype="finops:invoice"` (vendor_id, amount_usd, period); `cmdb_provider_map.csv` (ci_id, vendor_id, criticality).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Set `share_pct` threshold per board risk appetite; (2) join `cmdb_provider_map` to count critical CIs per vendor; (3) quarterly board pack export; (4) reconcile vendor_id to Register of Information keys; (5) document mitigation (secondary vendor) in GRC.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vendor sourcetype="finops:invoice" period=strftime(now(),"%Y-%m") earliest=-400d
| stats sum(amount_usd) as spend by vendor_id, period
| eventstats sum(spend) as total_spend by period
| eval share_pct=round(100*spend/total_spend,2)
| where share_pct>25
| sort period, - share_pct
```
