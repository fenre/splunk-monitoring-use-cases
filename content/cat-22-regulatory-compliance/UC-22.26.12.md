# UC-22.26.12 · Safety-critical system integrity

## Description

Flags overdue security patches and unsigned firmware changes on safety-related hosts.

## Value

Gives the responsible team auditor-ready evidence for NO Petroleumsforskriften compliance by automating regulatory compliance evidence using indexes os and sourcetypes WinEventLog:Application, drilling:app:status. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain supporting lookups (classified hosts, sectors, transfers) aligned to Norwegian regulatory language. (2) Schedule searches with Europe/Oslo time semantics for reporting cutoffs. (3) Route ES notables to the correct national security, OT, or privacy queues. (4) Quarterly false-positive tuning with HR, operations, and legal stakeholders. Indexes required: `index=os`; Sourcetypes: `sourcetype=WinEventLog:Application`, `sourcetype=drilling:app:status`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os sourcetype IN ("drilling:app:status","WinEventLog:Application") host="DRILL-*" earliest=-24h
| rex field=_raw "Version=(?<app_ver>\d+\.\d+\.\d+)"
| stats latest(app_ver) as observed by host
| lookup drilling_control_baseline.csv host OUTPUT expected_ver
| where isnotnull(expected_ver) AND observed!=expected_ver
```
