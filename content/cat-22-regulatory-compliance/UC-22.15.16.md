# UC-22.15.16 · Certificate-Based OPC-UA Logon Failures and Weak Trust Stores

## Description

Tracks OPC-UA secure channel failures and untrusted issuer patterns to validate strength of public-key authentication for industrial servers.

## Value

Tracks OPC-UA secure channel failures and untrusted issuer patterns to validate strength of public-key authentication for industrial servers.

## Implementation

(1) Deploy UA GDS or centralized PKI; (2) map status codes to runbooks; (3) alert on spike per server; (4) rotate OPC-UA app certs on schedule.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Edge Hub / unified `index=ot_protocol`.
• Ensure the following data sources are available: `edge_hub_opcua` or unified sourcetype with `security_policy`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Deploy UA GDS or centralized PKI; (2) map status codes to runbooks; (3) alert on spike per server; (4) rotate OPC-UA app certs on schedule.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_protocol sourcetype="edge_hub_opcua" earliest=-7d
| search status!="Good" OR match(_raw,"(?i)BadCertificate|issuer|untrusted")
| eval server=coalesce(opcua_server_endpoint, dest_host)
| stats count by server, status, opcua_security_mode, opcua_user_token_type
| sort - count
```
