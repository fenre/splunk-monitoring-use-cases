# 22.15.16 — Certificate-Based OPC-UA Logon Failures and Weak Trust Stores

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

Tracks OPC-UA secure channel failures and untrusted issuer patterns to validate strength of public-key authentication for industrial servers.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_protocol and sourcetypes edge_hub_opcua. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_protocol sourcetype="edge_hub_opcua" earliest=-7d
| search status!="Good" OR match(_raw,"(?i)BadCertificate|issuer|untrusted")
| eval server=coalesce(opcua_server_endpoint, dest_host)
| stats count by server, status, opcua_security_mode, opcua_user_token_type
| sort - count
```

## Implementation

(1) Deploy UA GDS or centralized PKI; (2) map status codes to runbooks; (3) alert on spike per server; (4) rotate OPC-UA app certs on schedule. Indexes required: `index=ot_protocol`; Sourcetypes: `sourcetype=edge_hub_opcua`.

## Visualization

Table, Time chart, Single value (failed channels). Key fields: `opcua_security_mode`, `opcua_user_token_type`, `server`, `status`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 1.9 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 1.9.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
