# UC-22.14.25 · Use of External Systems — Unmanaged SaaS OAuth Grants

## Description

Highlights high-risk OAuth consent grants to external applications for AC-20 use of external systems governance.

## Value

Highlights high-risk OAuth consent grants to external applications for AC-20 use of external systems governance.

## Implementation

(1) Build `risky_oauth_apps.csv` from security reviews; (2) integrate CASB signals if forwarded; (3) quarterly user attestation; (4) auto-revoke workflow outside Splunk with ticket link.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365 (Splunkbase 4055).
• Ensure the following data sources are available: `ms:o365:management` OAuth2 / consent operations.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Build `risky_oauth_apps.csv` from security reviews; (2) integrate CASB signals if forwarded; (3) quarterly user attestation; (4) auto-revoke workflow outside Splunk with ticket link.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=o365 sourcetype=ms:o365:management earliest=-30d Operation="Consent to application"
| stats values(UserId) as users, latest(_time) as last_seen by ObjectId
| lookup risky_oauth_apps.csv ObjectId OUTPUT app_risk
| where app_risk="high" OR match(ObjectId,"unknown")
```
