# UC-22.1.2 · GDPR Data Subject Access Request Fulfillment Tracking

## Description

Measures DSAR ticket lifecycle from opened to closed against a 30-calendar-day SLA so privacy and audit teams can evidence timely handling of access, rectification, erasure, and portability requests.

## Value

Gives the responsible team auditor-ready evidence for GDPR compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:incident, snow:sc_req_item. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Install Splunk Add-on for ServiceNow (1928) with sc_req_item and incident inputs enabled; (2) align `cat_item`/`short_description` filters with your DSAR catalogue naming; (3) confirm timestamp format in `opened_at`/`closed_at` and adjust `strptime` format if needed; (4) schedule daily and alert on `open_breach`. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:incident`, `sourcetype=snow:sc_req_item`.

## Detailed Implementation

Prerequisites

• Ensure the following data sources are available: `index=itsm` `sourcetype="snow:sc_req_item"` (number, cat_item, opened_at, closed_at, state, short_description) or `sourcetype="snow:incident"` (number, category, opened_at, closed_at, short_description, priority).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Install Splunk Add-on for ServiceNow (1928) with sc_req_item and incident inputs enabled; (2) align `cat_item`/`short_description` filters with your DSAR catalogue naming; (3) confirm timestamp format in `opened_at`/`closed_at` and adjust `strptime` format if needed; (4) schedule daily and alert on `open_breach`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm (sourcetype="snow:sc_req_item" OR sourcetype="snow:incident")
    (cat_item="*Subject Access*" OR short_description="*DSAR*" OR short_description="*data subject*")
| eval opened_epoch=strptime(opened_at, "%Y-%m-%d %H:%M:%S")
| eval closed_epoch=if(isnotnull(closed_at), strptime(closed_at, "%Y-%m-%d %H:%M:%S"), null())
| eval age_days=round((now()-opened_epoch)/86400, 1)
| eval sla_met=if(isnotnull(closed_epoch) AND (closed_epoch-opened_epoch)<=2592000, "Met", "Missed")
| eval open_breach=if(isnull(closed_epoch) AND age_days>30, "Open_SLA_Breach", null())
| table _time, number, sourcetype, state, age_days, sla_met, open_breach, short_description
| sort - age_days
```
