# UC-22.13.9 · Security Awareness Training Completion for BES Personnel

## Description

We list personnel with active BES access who have not completed the current awareness module so training evidence is current before audits.

## Value

We list personnel with active BES access who have not completed the current awareness module so training evidence is current before audits.

## Implementation

(1) Sync LMS completions nightly. (2) Align `employee_id` with directory accounts. (3) Alert managers fourteen days before the annual due wave. (4) Export CSV for the training coordinator.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=hr` `sourcetype=lms:completion`, `bes_privileged_users.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Sync LMS completions nightly. (2) Align `employee_id` with directory accounts. (3) Alert managers fourteen days before the annual due wave. (4) Export CSV for the training coordinator.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup bes_privileged_users.csv
| join type=left employee_id [
    search index=hr sourcetype=lms:completion course_id="CIP_SECURITY_AWARENESS" earliest=-400d
    | stats latest(completed_at) as completed_at by employee_id
  ]
| eval completed_epoch=strptime(completed_at,"%Y-%m-%d")
| eval stale=if(isnull(completed_epoch) OR completed_epoch<relative_time(now(),"-350d@d"),1,0)
| where stale=1
| table employee_id, role, completed_at
```
