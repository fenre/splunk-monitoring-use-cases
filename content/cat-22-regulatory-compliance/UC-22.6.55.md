# UC-22.6.55 · ISO/IEC 27001:2022 Clause 8.1 — Operational planning: change advisory board (CAB) approval evidence

## Description

Flags implemented changes that were not approved by an authorised CAB member prior to implementation. Clause 8.1 requires the organisation to plan, implement and control processes to meet information-security requirements — CAB approval is the direct control.

## Value

Turns the 'post-implementation review' into a pre-emptive signal. The Head of IT Operations sees the breach list within the hour.

## Implementation

(1) Export ServiceNow change events; (2) maintain cab_roles.csv listing authorised approvers; (3) schedule UC hourly; (4) approval_met=0 opens a deficiency record with 7d remediation; (5) monthly Audit Committee report. Indexes required: `index=change`; Sourcetypes: `sourcetype=servicenow:change`, `sourcetype=servicenow:change;`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow.
• Ensure the following data sources are available: `index=change` sourcetype=servicenow:change; approver-role lookup `cab_roles.csv`..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Export ServiceNow change events; (2) maintain cab_roles.csv listing authorised approvers; (3) schedule UC hourly; (4) approval_met=0 opens a deficiency record with 7d remediation; (5) monthly Audit Committee report.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=change sourcetype=servicenow:change state IN (implemented,closed) earliest=-7d
| stats latest(type) AS type latest(_time) AS implemented_at latest(approval_status) AS approval_status latest(approver) AS approver latest(approval_date) AS cab_approved_at BY change_id
| lookup cab_roles.csv approver OUTPUT is_authorised_cab
| eval approval_met=if(type="emergency",1,if(approval_status="approved" AND is_authorised_cab=1 AND cab_approved_at<=implemented_at,1,0))
| where approval_met=0
| table change_id type implemented_at cab_approved_at approver approval_met
```

## Known False Positives

Emergency changes legitimately bypass CAB; their follow-up retrospective approval must exist separately — this UC excludes type=emergency by design.
