# 22.13.3 — ESP Boundary Change Detection via Firewall Configuration Events

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

We tie ESP security rule, zone, and interface edits to change records so boundary changes to BES Cyber Systems are visible and attributable.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes itsm, network and sourcetypes chg:firewall, chg:firewall`, pan:config. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=network (sourcetype=pan:config OR sourcetype=pan:system) earliest=-7d
| where match(object,"(?i)security|zone|interface") OR match(cmd,"(?i)commit")
| lookup esp_object_inventory object OUTPUT esp_segment_id
| join type=left max=0 object [
    search index=itsm sourcetype=chg:firewall earliest=-30d
    | stats latest(approved) as approved latest(change_id) as change_id by firewall_object
    | rename firewall_object as object
  ]
| where approved!="true" OR isnull(change_id)
| table _time, admin, object, esp_segment_id, change_id, approved
```

## CIM SPL

```spl
| tstats summariesonly=t latest(All_Changes.status) as agg_value from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - agg_value
```

## Implementation

(1) Forward Panorama or device config logs. (2) Map changed objects to ESP segments in `esp_object_inventory`. (3) Require ITSM linkage for production commits. (4) Weekly compliance dashboard export. Indexes required: `index=itsm`, `index=network`; Sourcetypes: `sourcetype=chg:firewall`, `sourcetype=chg:firewall``, `sourcetype=pan:config`, `sourcetype=pan:config``; Lookups: `esp_object_inventory`.

## Visualization

Timeline (changes), Table (unauthorized edits), Bar chart (by ESP segment). Key fields: `admin`, `approved`, `change_id`, `esp_segment_id`, `object`.

## Compliance

- **NERC CIP** current: CIP-002-6 R1 Part 1.2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-002-6 R1 Part 1.2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
