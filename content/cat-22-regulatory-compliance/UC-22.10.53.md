# UC-22.10.53 · Subcontractor Access — Downstream API Keys Touching ePHI Interfaces

## Description

Monitors first-seen API keys or OAuth clients used by BA subcontractors against an approved key registry so the extended BAA chain does not introduce unreviewed ePHI paths.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes interop and sourcetypes apigee:proxy, apigee:proxy`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Hash static API keys at ingest with `sha256(api_key)` for display fields; (2) require procurement to register subcontractor keys before production; (3) auto-revoke via API management integration on critical alerts. Indexes required: `index=interop`; Sourcetypes: `sourcetype=apigee:proxy`, `sourcetype=apigee:proxy``; Lookups: `ba_subcontractor_clients`, `ba_subcontractor_clients.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector (HEC), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=interop` `sourcetype=apigee:proxy` or AWS API Gateway logs (`client_id`, `api_key`, `consumer`), `lookup ba_subcontractor_clients.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Hash static API keys at ingest with `sha256(api_key)` for display fields; (2) require procurement to register subcontractor keys before production; (3) auto-revoke via API management integration on critical alerts.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=interop sourcetype="apigee:proxy" uri_path="/fhir/*" earliest=-30d
| stats earliest(_time) as first_seen latest(_time) as last_seen dc(status) as status_codes by client_id, api_key
| eval key_or_client=coalesce(client_id, api_key)
| lookup ba_subcontractor_clients.csv key_or_client OUTPUT approved_vendor, subcontractor_name
| where isnull(approved_vendor)
| sort first_seen
```
