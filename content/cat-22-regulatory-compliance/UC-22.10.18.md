# UC-22.10.18 · Integrity — Unexpected UPDATE/DELETE on PHI Tables (Clarity/Caboodle)

## Description

Alerts on bulk or off-hours SQL changes to tables holding patient demographics or clinical facts so improper alteration of ePHI is detected and investigated.

## Value

Alerts on bulk or off-hours SQL changes to tables holding patient demographics or clinical facts so improper alteration of ePHI is detected and investigated.

## Implementation

(1) Enable SQL Server Audit to SIEM; (2) map ETL service accounts to allow list; (3) pair with change tickets from ServiceNow; (4) retain immutable copy to WORM storage if required.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686), Splunk Add-on for Microsoft SQL Server (2648).
• Ensure the following data sources are available: `index=mssql` `sourcetype=mssql:audit` (`statement`, `schema_name`, `object_name`, `application_name`, `session_server_principal_name`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enable SQL Server Audit to SIEM; (2) map ETL service accounts to allow list; (3) pair with change tickets from ServiceNow; (4) retain immutable copy to WORM storage if required.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mssql sourcetype="mssql:audit" earliest=-24h action_class="DML"
| where match(lower(object_name),"pat|patient|phi|epic") AND match(lower(statement),"update|delete")
| eval is_bulk=if(match(statement,"(?i)where\s+1=1|truncate|delete\s+from"),1,0)
| where is_bulk=1 OR (match(lower(application_name),"microsoft sql server management studio") AND _time < relative_time(now(),"@d+18h"))
| stats count by session_server_principal_name, object_name, application_name
| sort - count
```
