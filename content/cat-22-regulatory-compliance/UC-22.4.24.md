# UC-22.4.24 · CCPA/CPRA — Correction/Deletion Verification — Downstream Data Warehouse Row Still Present After Deletion Certificate

## Description

Deletion rights require proof that replicas disappear. Comparing deletion job completion events to late-arriving warehouse snapshots (row presence tests) shows the firm validates end-state—not only API 200 responses from a primary system.

## Value

Gives the responsible team auditor-ready evidence for CCPA/CPRA compliance by automating regulatory compliance evidence using indexes dataops, privacy and sourcetypes dq:row_probe, privacy:delete_job. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Use salted `subject_id` only; (2) probes must be authorized read-only queries; (3) redact outputs; (4) auto-reopen deletion jobs; (5) document in ROPA technical measures. Indexes required: `index=dataops`, `index=privacy`; Sourcetypes: `sourcetype=dq:row_probe`, `sourcetype=privacy:delete_job`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (read-only probes), deletion orchestrator HEC.
• Ensure the following data sources are available: `index=privacy` `sourcetype="privacy:delete_job"` (subject_id, job_id, completed_ts, systems_touched); `index=dataops` `sourcetype="dq:row_probe"` (subject_id, dataset, found_flag, probe_ts).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Use salted `subject_id` only; (2) probes must be authorized read-only queries; (3) redact outputs; (4) auto-reopen deletion jobs; (5) document in ROPA technical measures.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="privacy:delete_job" earliest=-90d
| eval done=strptime(completed_ts,"%Y-%m-%dT%H:%M:%SZ")
| join type=left subject_id [
    search index=dataops sourcetype="dq:row_probe" earliest=-90d
    | eval pt=strptime(probe_ts,"%Y-%m-%dT%H:%M:%SZ")
    | where found_flag=true
    | fields subject_id, dataset, found_flag, pt
  ]
| where pt>done
| stats values(dataset) as datasets by subject_id, job_id
| sort subject_id
```
