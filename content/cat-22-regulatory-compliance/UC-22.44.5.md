# UC-22.44.5 · SBOM attestation completeness — critical vendors without signed SBOM

## Description

Surfaces critical vendors whose SBOM is absent, unsigned, or older than policy cadence (default 180 days).

## Value

Makes the CRA/NIS2 SBOM requirement a tracked operational metric.

## Implementation

(1) Onboard vendor SBOMs through procurement workflow; (2) validate signatures with sigstore or equivalent; (3) run weekly. Lookups: `vendor_sbom_attestations`, `vendor_sbom_attestations.csv`, `vrm_attestations`, `vrm_attestations.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: Vendor SBOM register (vendor_sbom.csv), signature-validation tooling outputs, VRM criticality..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Onboard vendor SBOMs through procurement workflow; (2) validate signatures with sigstore or equivalent; (3) run weekly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup vrm_attestations.csv
| where criticality="critical"
| lookup vendor_sbom_attestations.csv vendor OUTPUT sbom_sha256 signed signed_by signed_at
| eval age_days=round((now()-strptime(signed_at,"%Y-%m-%d"))/86400,0)
| where isnull(sbom_sha256) OR signed!="true" OR age_days>180
| table vendor criticality sbom_sha256 signed signed_by age_days
```
