# UC-22.2.1 · NIS2 Incident Detection and 24-Hour Early Warning Reporting

## Description

Measures detection-to-response progress on high-urgency ES notables to support early-warning obligations and internal crisis reporting within the first 24 hours of awareness.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map ES `urgency` values to your NIS2 incident classes; (2) require analysts to transition `status`/`status_description` at acknowledgement and containment; (3) alert on `t_minus_4h` for CSIRT/legal escalation; (4) export `past_24h_open` rows into crisis-management runbooks and regulatory reporting drafts.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263).
• Ensure the following data sources are available: `` `notable` `` macro (rule_name, urgency, status, owner, status_description, _time).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map ES `urgency` values to your NIS2 incident classes; (2) require analysts to transition `status`/`status_description` at acknowledgement and containment; (3) alert on `t_minus_4h` for CSIRT/legal escalation; (4) export `past_24h_open` rows into crisis-management runbooks and regulatory reporting drafts.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
`notable` urgency IN ("high","critical") earliest=-3d
| eval hours_open=round((now()-_time)/3600, 2)
| eval t_minus_4h=if(hours_open>=20 AND hours_open<24 AND status!="Closed", 1, 0)
| eval past_24h_open=if(hours_open>24 AND status!="Closed", 1, 0)
| table _time, rule_name, urgency, status, owner, status_description, hours_open, t_minus_4h, past_24h_open
| where t_minus_4h=1 OR past_24h_open=1
| sort - past_24h_open, - hours_open
```
