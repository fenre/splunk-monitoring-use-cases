# UC-22.13.13 · Quarterly Access Review Dataset — Interactive Sessions by System

## Description

We assemble quarterly interactive access by user and BES asset to accelerate access certification and attestation.

## Value

We assemble quarterly interactive access by user and BES asset to accelerate access certification and attestation.

## Implementation

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for CyberArk (4295), Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=pam` `sourcetype=cyberark:ptalog`, `bes_cyber_asset_inventory.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Define quarter macros for reproducible windows. (2) Export to GRC certification tool. (3) Restrict index ACLs. (4) Store signed attestation back via HEC if desired.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=pam sourcetype=cyberark:ptalog action="Logon" earliest=-90d@d latest=@d
| lookup bes_cyber_asset_inventory.csv address AS address OUTPUT bes_system impact_rating
| where isnotnull(bes_system)
| stats earliest(_time) as first latest(_time) as last dc(address) as targets by user, bes_system, impact_rating
| sort impact_rating, user
```
