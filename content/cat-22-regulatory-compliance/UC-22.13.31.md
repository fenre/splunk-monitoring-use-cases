# 22.13.31 — Malware Prevention Agent Coverage on BES Cyber Assets

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We list in-scope hosts that have not reported antivirus or EDR health within the required interval.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes endpoint and sourcetypes WinDefender:Status, WinDefender:Status`, crowdstrike:hosts. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
| inputlookup bes_cyber_asset_inventory.csv
| rename hostname as host
| join type=left host [
    search index=endpoint (sourcetype=WinDefender:Status OR sourcetype=crowdstrike:hosts) earliest=-48h
    | stats latest(_time) as last_report by host
  ]
| eval stale=if(isnull(last_report) OR (now()-last_report)>172800,1,0)
| where stale=1
| table host, bes_system, last_report
```

## Implementation

(1) Normalize host naming between inventory and endpoint tools. (2) Set stale threshold per vendor guidance. (3) Weekly report to operations. (4) Track remediation tickets. Indexes required: `index=endpoint`; Sourcetypes: `sourcetype=WinDefender:Status`, `sourcetype=WinDefender:Status``, `sourcetype=crowdstrike:hosts`, `sourcetype=crowdstrike:hosts``; Lookups: `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.
