# 22.13.44 — Backup Job Success and Failure for BES Databases and Configuration Stores

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We verify backup jobs for systems supporting BES recovery completed successfully so recovery planning rests on reliable restore points.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes backup and sourcetypes commvault:job, veeam:job, veeam:job`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=backup (sourcetype=veeam:job OR sourcetype=commvault:job) earliest=-48h
| lookup bes_cyber_asset_inventory.csv job_target AS host OUTPUT bes_system
| where isnotnull(bes_system)
| stats latest(result) as last_result latest(_time) as last_run by host, job_name
| where last_result!="Success"
| table host, job_name, last_result, last_run, bes_system
```

## Implementation

(1) Map backup job targets to BES inventory. (2) Alert on failures and missed schedules. (3) Pair with ticket automation. (4) Retain success summaries for auditors. Indexes required: `index=backup`; Sourcetypes: `sourcetype=commvault:job`, `sourcetype=veeam:job`, `sourcetype=veeam:job``; Lookups: `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.

## Visualization

Table (failed jobs), Single value (failure count), Time chart (success rate). Key fields: `bes_system`, `host`, `job_name`, `last_result`, `last_run`.

## Compliance

- **NERC CIP** current: CIP-009-6 R1 Part 1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-009-6 R1 Part 1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
