# UC-22.39.3 · Data-subject breach communication timeline tracker (Art.34 / §164.404)

## Description

Measures every open high-risk-breach communication campaign against the statutory clocks (GDPR 'without undue delay', HIPAA 60 days). Campaigns falling behind the completion-rate threshold are surfaced to the DPO for intervention.

## Value

Data-subject communication is routinely cited in enforcement decisions (Meta, Uber, British Airways); measuring it is the only credible evidence of 'taken reasonable efforts'.

## Implementation

(1) Ingest campaign events from the messaging platform; (2) join to breach_campaigns.csv with classification timestamps; (3) run daily; (4) escalate via SOAR. Indexes required: `index=comms`; Sourcetypes: `sourcetype=breach:notification`.

Step 1 — Configure data collection
(1) Ingest campaign events from the messaging platform; (2) join to breach_campaigns.csv with classification timestamps; (3) run daily; (4) escalate via SOAR.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=comms sourcetype=breach:notification earliest=-90d
| stats count(eval(status="delivered")) as delivered count(eval(status="bounced_handled")) as fallback count as total by campaign_id
| eval ok=(delivered+fallback)/total
| lookup breach_campaigns.csv campaign_id OUTPUT classification classified_at
| eval age_days=round((now()-strptime(classified_at,"%Y-%m-%dT%H:%M:%SZ"))/86400,0)
| where age_days>=50 AND ok<0.95
| table campaign_id classified_at age_days total delivered fallback ok
```

## Visualization

Per-campaign completion funnel, aging bar chart, single-value '% of campaigns on track'. Key fields: `age_days`, `campaign_id`, `classified_at`, `delivered`, `fallback`, `ok`.

## Known False Positives

Subjects without deliverable contact info require the fall-back channel; bounces without fall-back handling should be followed up manually.
