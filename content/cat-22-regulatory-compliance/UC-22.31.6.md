# 22.31.6 — Application control allowlist drift monitoring

**Criticality:** medium | **Difficulty:** beginner | **Status:** N/A

## Description

Evidences ASD Essential Eight obligations for application control allowlist drift monitoring using endpoint, identity, and cloud telemetry.

## Value

Gives the responsible team auditor-ready evidence for ASD E8 compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode=4688 earliest=-24h
| stats count by New_Process_Name, host
| lookup asd_e8_scope.csv host OUTPUT in_scope
| where in_scope="true" AND NOT match(New_Process_Name,"(?i)\\\\Program Files")
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.dest | sort - count
```

## Implementation

(1) Ingest CrowdStrike policy posture via TA 5082 and map hosts to NDB and NZISM tiers. (2) Tune alerts with ACSD Essential Eight and APRA-aligned severities. (3) Coordinate privacy and security ownership for dual-reporting workflows. (4) Monthly export for privacy officer and prudential working groups. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`; Lookups: `asd_e8_scope`, `asd_e8_scope.csv`.
