# UC-22.11.70 · File Integrity Monitoring Alerts on Payment Web Roots

## Description

Surfaces Splunk Universal Forwarder or OSSEC/AIDE change alerts on directories serving checkout static content so unauthorized modification of critical files is detected promptly.

## Value

Surfaces Splunk Universal Forwarder or OSSEC/AIDE change alerts on directories serving checkout static content so unauthorized modification of critical files is detected promptly.

## Implementation

(1) Scope FIM to PCI web roots only to reduce noise. (2) Correlate with deploy pipeline user. (3) Page on off-hours changes. (4) Store weekly digest for change-detection evidence (Req 11 overlap).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Unix and Linux (833), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=os` `sourcetype="ossec"` or `auditd` paths; `sourcetype="stash"` from `monitor://` with `hash` in `change_management`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os (sourcetype="linux:audit" OR sourcetype="ossec") earliest=-24h
| where match(name,"(?i)/var/www/payment|/opt/checkout/dist)")
| where match(key,"(?i)write|attrib|chmod|unlink|create)")
| stats earliest(_time) as first_change, values(key) as ops by host, name
| sort - first_change
```
