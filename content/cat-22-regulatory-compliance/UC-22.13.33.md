# 22.13.33 — Security Event Log Generation Validation for Windows BES Hosts

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We detect Windows systems where the Security channel stopped forwarding while the host remains online, risking missing auditable events.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security, WinEventLog:Security`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" earliest=-48h
| lookup bes_cyber_asset_inventory.csv ComputerName AS host OUTPUT bes_system
| where isnotnull(bes_system)
| stats latest(_time) as last_sec by host
| eval gap_h=round((now()-last_sec)/3600,2)
| where gap_h>24
| table host, bes_system, last_sec, gap_h
```

## Implementation

(1) Tune gap threshold per risk. (2) Exclude maintenance windows. (3) Alert server owners and Splunk admins. (4) Track ticket closure with log resume proof. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`, `sourcetype=WinEventLog:Security``; Lookups: `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.
