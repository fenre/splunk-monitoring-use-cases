# UC-22.10.33 · Minimum Necessary — Access Outside Active Care Team

## Description

Compares chart opens to on-service treatment team membership so accesses without an active treatment relationship are surfaced for minimum necessary review.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes epic and sourcetypes epic:audit, epic:audit`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Build care team extract from Clarity `PAT_ENC`/`TEAM` tables into lookup; (2) exclude HIM release of information users; (3) integrate privacy queue workflow. Indexes required: `index=epic`; Sourcetypes: `sourcetype=epic:audit`, `sourcetype=epic:audit``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=epic` `sourcetype=epic:audit` (`USER_ID`, `PAT_ID`, `AccessType`), nightly roster `epic_active_careteam.csv` (`PAT_ID`, `member_id`, `relation`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Build care team extract from Clarity `PAT_ENC`/`TEAM` tables into lookup; (2) exclude HIM release of information users; (3) integrate privacy queue workflow.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=epic sourcetype="epic:audit" AccessType="PatientChart" earliest=-24h
| lookup epic_active_careteam.csv PAT_ID USER_ID as member_id OUTPUT relation
| where isnull(relation)
| stats count dc(PAT_ID) as patients by USER_ID
| where count>=5
| sort - count
```
