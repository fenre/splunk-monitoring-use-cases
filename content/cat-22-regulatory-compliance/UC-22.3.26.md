# UC-22.3.26 · DORA TLPT — Tester Independence and Conflict-of-Interest Attestation Log

## Description

Independence of testers from operational teams is a supervisory expectation. A tamper-evident log of tester employer changes, asset holdings declarations, and recusal events supports governance of TLPT quality.

## Value

Gives the responsible team auditor-ready evidence for DORA compliance by automating regulatory compliance evidence using indexes grc and sourcetypes tlpt:independence. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Require attestation before VPN tokens issued; (2) integrate MSSP tester roster; (3) alert on `conflict_flag=true` until cleared by legal; (4) retain 7 years per records policy; (5) hash payloads for integrity. Indexes required: `index=grc`; Sourcetypes: `sourcetype=tlpt:independence`.

## Detailed Implementation

• Ensure the following data sources are available: `index=grc` `sourcetype="tlpt:independence"` (tester_id, engagement_id, attestation_type, signed_date, conflict_flag).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype="tlpt:independence" earliest=-400d conflict_flag=true
| stats latest(signed_date) as last_signed, values(attestation_type) as types by tester_id, engagement_id
| sort tester_id
```
