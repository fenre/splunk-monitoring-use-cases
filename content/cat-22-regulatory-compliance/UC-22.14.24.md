# UC-22.14.24 · Remote Access Anomalies — Geo-Velocity and Off-Hours VPN

## Description

Detects impossible travel and off-hours remote access for AC-17 remote access monitoring.

## Value

Detects impossible travel and off-hours remote access for AC-17 remote access monitoring.

## Implementation

(1) Validate `UserLoggedIn` operation naming for tenant; (2) pair with MFA status field if present; (3) reduce noise with corporate travel calendar lookup optional; (4) document tuning in SSP.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365 (Splunkbase 4055), Splunk Add-on for Cisco ASA (Splunkbase 1620).
• Ensure the following data sources are available: `ms:o365:management` SignIns, `cisco:asa` VPN logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Validate `UserLoggedIn` operation naming for tenant; (2) pair with MFA status field if present; (3) reduce noise with corporate travel calendar lookup optional; (4) document tuning in SSP.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=o365 sourcetype=ms:o365:management Workload=AzureActiveDirectory Operation=UserLoggedIn earliest=-24h
| iplocation extended_field=ClientIP
| stats earliest(_time) as t1, latest(_time) as t2, values(Country) as countries by UserId, ClientIP
| eval delta_h=round((t2-t1)/3600,2)
| where mvcount(countries)>1 AND delta_h<2
```
