# UC-22.11.27 · Anti-Malware Agent Coverage Gaps on CDE Windows Servers

## Description

Compares CrowdStrike or legacy AV presence signals to the PCI server inventory so every in-scope system is actively protected against malware as required for all system components.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes edr, windows and sourcetypes crowdstrike:hosts. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize hostname case with `lower()`. (2) Reconcile VMs cloned without sensor. (3) Auto-ticket missing agents within SLA. (4) Monthly attestation export for Req 5. Indexes required: `index=edr`, `index=windows`; Sourcetypes: `sourcetype=crowdstrike:hosts`; Lookups: `pci_windows_cde_assets`, `pci_windows_cde_assets.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=edr sourcetype="crowdstrike:hosts" earliest=-24h
| lookup pci_windows_cde_assets.csv hostname OUTPUT cde_member
| where cde_member="true"
| eval protected=if(match(agent_status,"(?i)normal|protected|enabled"),1,0)
| where protected=0 OR isnull(agent_status)
| stats values(agent_status) as status_vals by hostname, local_ip
| sort hostname
```
