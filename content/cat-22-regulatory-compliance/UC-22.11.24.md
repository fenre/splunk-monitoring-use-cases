# UC-22.11.24 · Certificate Expiry Risk for Public-Facing Payment Hostnames

## Description

Counts days-to-expiry for certificates protecting CHD transmission paths so renewals happen before outages or downgrade attacks during renewal gaps.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes vm and sourcetypes tenable:sc:vuln. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map Tenable `dns` to customer-facing FQDNs. (2) Ingest `notAfter_epoch` via TA field alias. (3) Page at 30/14/7 days thresholds. (4) Document emergency renewal procedure linkage. Indexes required: `index=vm`; Sourcetypes: `sourcetype=tenable:sc:vuln`; Lookups: `pci_public_payment_hosts`, `pci_public_payment_hosts.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Tenable (4060), Splunk Add-on for Qualys (2964).
• Ensure the following data sources are available: `index=vm` `tenable:sc:vuln` plugin 51192; `qualys:host` cert fields if mapped.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map Tenable `dns` to customer-facing FQDNs. (2) Ingest `notAfter_epoch` via TA field alias. (3) Page at 30/14/7 days thresholds. (4) Document emergency renewal procedure linkage.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vm sourcetype="tenable:sc:vuln" earliest=-24h
| where pluginID=51192 OR match(plugin_name,"(?i)certificate.*expir")
| lookup pci_public_payment_hosts.csv dns_name AS dns OUTPUT in_business_scope
| where in_business_scope="true"
| eval days_left=round((notAfter_epoch-now())/86400,1)
| where days_left<45 AND days_left>0
| stats min(days_left) as soonest_expiry by dns, ip
| sort soonest_expiry
```
