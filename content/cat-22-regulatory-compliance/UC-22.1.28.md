# UC-22.1.28 · GDPR Sub-Processor Change Monitoring

## Description

Processors must not engage another processor without prior specific or general written authorisation of the controller. This use case detects new destination domains, SaaS tenants, or infrastructure regions receiving personal data from approved processors — flagging unapproved sub-processing paths inferred from network and SaaS audit telemetry.

## Value

Processors must not engage another processor without prior specific or general written authorisation of the controller. This use case detects new destination domains, SaaS tenants, or infrastructure regions receiving personal data from approved processors — flagging unapproved sub-processing paths inferred from network and SaaS audit telemetry.

## Implementation

(1) Build allowlist of approved processor and sub-processor domains from DPAs; (2) classify egress traffic by processor parent using DNS and TLS SNI metadata; (3) alert on high-volume flows to unlisted destinations; (4) require procurement approval workflow to update allowlist; (5) document investigations for Art. 28 accountability.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| tstats `summariesonly` sum(All_Traffic.bytes_out) as bytes_out dc(All_Traffic.src) as src_count
  from datamodel=Network_Traffic.All_Traffic
  where earliest=-7d All_Traffic.action="allowed"
  by All_Traffic.dest
| rename All_Traffic.* as *
| eval dest_domain=replace(dest,"^\d+\.\d+\.\d+\.\d+$","")
| lookup gdpr_processor_subprocessor_allowlist.csv dest_domain OUTPUT primary_processor, approval_status
| where isnotnull(primary_processor) AND (approval_status!="approved" OR isnull(approval_status))
| sort - bytes_out
| table dest, dest_domain, primary_processor, approval_status, bytes_out, src_count
```
