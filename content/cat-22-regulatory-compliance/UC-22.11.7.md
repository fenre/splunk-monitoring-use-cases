# UC-22.11.7 · Outbound Service Allow-List Violations from CDE Servers

## Description

Compares observed egress from in-scope servers against the documented approved service list, proving that outbound traffic from the CDE is limited to what the standard permits.

## Value

Compares observed egress from in-scope servers against the documented approved service list, proving that outbound traffic from the CDE is limited to what the standard permits.

## Implementation

(1) Build the egress lookup from the PCI network standard appendix. (2) Refresh when new SaaS dependencies are onboarded. (3) Weekly ops review of the stats output. (4) Store signed CSV with each policy version.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757).
• Ensure the following data sources are available: `index=netfw` `sourcetype="pan:traffic"`; `lookup pci_cde_server_subnets.csv`, `lookup pci_approved_egress.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Build the egress lookup from the PCI network standard appendix. (2) Refresh when new SaaS dependencies are onboarded. (3) Weekly ops review of the stats output. (4) Store signed CSV with each policy version.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=netfw sourcetype="pan:traffic" action=allowed earliest=-7d
| lookup pci_cde_server_subnets.csv subnet AS src OUTPUT cde_server
| where cde_server="true"
| eval svc=lower(concat(dest_port,"#",app))
| lookup pci_approved_egress.csv svc OUTPUT approved
| where isnull(approved) OR approved="false"
| stats count by src, dest, dest_port, app, dest_zone
| sort - count
```
