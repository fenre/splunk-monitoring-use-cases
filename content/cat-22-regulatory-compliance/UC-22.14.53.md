# UC-22.14.53 ┬Ę Configuration Change Control ŌĆö Unauthorized Firewall Rule Adds

## Description

Detects firewall commits outside maintenance windows without change tickets for CM-3 configuration change control.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes net and sourcetypes pan:system. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize PAN commit events; (2) build `recent_changes.csv` from SNOW scheduled output; (3) alert on orphan commits; (4) emergency change tagging. Indexes required: `index=net`; Sourcetypes: `sourcetype=pan:system`.

Step 1 ŌĆö Configure data collection

Step 2 ŌĆö Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=net sourcetype=pan:system earliest=-7d (status="commit" OR match(_raw,"(?i)commit"))
| eval change_window=if(_time % 86400 > 72000 OR _time % 86400 < 18000,1,0)
| lookup recent_changes.csv device commit_id OUTPUT ticket
| where isnull(ticket) AND change_window=0
```
