# UC-22.38.4 · Transfer Impact Assessment currency — stale Schrems II assessments

## Description

Makes TIA currency a visible metric rather than a yearly memory item. Stale TIAs are the most common Schrems II finding by supervisory authorities.

## Value

Converts a calendar reminder into a demonstrable control with evidence per-transfer.

## Implementation

(1) Maintain tia_register.csv; (2) run weekly; (3) on miss, DPO re-opens the TIA review workflow.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: TIA register (tia_register.csv), transfer_register.csv, DPO review events..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain tia_register.csv; (2) run weekly; (3) on miss, DPO re-opens the TIA review workflow.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup tia_register.csv
| eval age_days=round((now()-strptime(last_reviewed,"%Y-%m-%d"))/86400,0)
| where age_days>cadence_days
| lookup transfer_register.csv tia_id OUTPUT transfer_id destination
| table tia_id destination last_reviewed age_days cadence_days transfer_id
```
