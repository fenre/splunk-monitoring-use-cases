# 22.34.1 — SWIFT secure zone logical isolation monitoring

**Criticality:** medium | **Difficulty:** intermediate | **Status:** N/A

## Description

Produces SWIFT Customer Security Programme evidence for swift secure zone logical isolation monitoring ahead of independent assessment.

## Value

Gives the responsible team auditor-ready evidence for SWIFT CSP compliance by automating regulatory compliance evidence using indexes network, swift and sourcetypes pan:traffic, swift:zone:flow. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=swift sourcetype="swift:zone:flow" OR index=network sourcetype="pan:traffic" tag=swift earliest=-24h
| stats sum(bytes) as b by src_zone, dest_zone
| where src_zone!="SWIFT_DMZ" AND dest_zone="SWIFT_SECURE"
| sort - b
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Segment SWIFT CSP evidence into dedicated indexes with strict role-based access. (2) Join operators to HR screening and vendor risk lookups refreshed weekly. (3) Validate MFA and PAM sourcetypes are CIM-mapped for Authentication models. (4) Retain evidence for at least 13 months aligned to attestation cycles. Indexes required: `index=network`, `index=swift`.

## Visualization

Table (operators), Single value (sessions), Bar chart (CSCF controls).

## Compliance

- **SWIFT CSP** CSCF v2025: 6.4 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 6.4.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
