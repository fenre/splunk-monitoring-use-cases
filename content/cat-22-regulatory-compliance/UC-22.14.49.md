# UC-22.14.49 · Incident Response Plan Test Evidence from Scheduled Tabletop Tags

## Description

Aggregates tagged tabletop exercise events for IR-8 incident response plan testing evidence.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes grc and sourcetypes ir:tabletop, ir:tabletop`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Emit structured HEC after each exercise; (2) define semi-annual threshold; (3) alert GRC owners; (4) attach lessons learned links. Indexes required: `index=grc`; Sourcetypes: `sourcetype=ir:tabletop`, `sourcetype=ir:tabletop``.

## Detailed Implementation

Prerequisites

• Ensure the following data sources are available: `index=grc` `sourcetype=ir:tabletop` JSON.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Emit structured HEC after each exercise; (2) define semi-annual threshold; (3) alert GRC owners; (4) attach lessons learned links.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype=ir:tabletop earliest=-365d
| stats latest(exercise_date) as last_ex, values(scenario_id) as scenarios by plan_version, participant_team
| eval overdue=if(now()-strptime(last_ex,"%Y-%m-%d")>15552000,1,0)
| where overdue=1
```
