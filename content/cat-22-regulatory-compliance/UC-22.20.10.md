# 22.20.10 — CMMC Level 2 practice evidence — CUI control area 10

**Criticality:** medium | **Difficulty:** expert | **Status:** N/A

## Description

Demonstrates contractor cybersecurity hygiene for "CMMC Level 2 practice evidence — CUI control area 10" using CUI-scoped telemetry mapped to CMMC 2.0 Level 2 expectations.

## Value

Gives the responsible team auditor-ready evidence for CMMC compliance by automating regulatory compliance evidence using indexes os and sourcetypes stash:file_integrity. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=os sourcetype=stash:file_integrity earliest=-7d
| eval uc_tag="22.20.10"
| lookup cui_hosts.csv host OUTPUT cui_flag
| where cui_flag="true"
| stats dc(path) as changed_files by host
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.dest | sort - count
```

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=os`.

## Visualization

Stacked bar, Table, Time chart, Single value. Key fields: `changed_files`, `host`, `uc_tag`.

## Compliance

- **CMMC** 2.0: CM.L2-3.4.1 (contributing) — File integrity monitoring on CUI hosts supports baseline configuration verification.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
