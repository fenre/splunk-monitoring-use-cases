# UC-22.6.14 · Business Continuity — RTO Breach Signals from ITSI Service Degradation

## Description

Uses service health episodes breaching RTO thresholds as objective continuity monitoring beyond paper plans.

## Value

Uses service health episodes breaching RTO thresholds as objective continuity monitoring beyond paper plans.

## Implementation

(1) Calibrate duration field to your ITSI KPIs; (2) align with BIA tiers; (3) BC manager monthly review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI (Splunkbase 1841).
• Ensure the following data sources are available: `itsi_summary`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Calibrate duration field to your ITSI KPIs; (2) align with BIA tiers; (3) BC manager monthly review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsi_summary earliest=-90d is_service_max_severity_event=1
| lookup service_rto_minutes.csv serviceid OUTPUT rto_minutes
| eval breach=if(severity>=5 AND duration>rto_minutes,1,0)
| where breach=1
| stats count by serviceid
```
