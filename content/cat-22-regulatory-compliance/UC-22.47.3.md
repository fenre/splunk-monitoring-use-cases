# UC-22.47.3 · Control owner attestation freshness

## Description

Elevates owner attestation from an email thread to a tracked GRC metric. Owner attestation freshness is a bedrock SOC 2 CC1.3 criterion.

## Value

Delivers the internal-audit team a continuous view of 'who has signed what'.

## Implementation

(1) Emit owner_attestations.csv from GRC workflow; (2) run weekly.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: GRC owner attestation ledger, control_inventory.csv..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Emit owner_attestations.csv from GRC workflow; (2) run weekly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup control_inventory.csv
| lookup owner_attestations.csv control_id OUTPUT last_attested signed_by
| eval age_days=round((now()-strptime(last_attested,"%Y-%m-%d"))/86400,0)
| where age_days>attestation_cadence_days OR isnull(last_attested)
| table control_id owner last_attested age_days attestation_cadence_days
```
