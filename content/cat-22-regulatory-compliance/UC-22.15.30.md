# UC-22.15.30 · Out-of-Range SCADA Setpoints and Invalid Control Commands

## Description

Validates control writes against engineering limits and recipe tables to catch dangerous setpoint or mode commands.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes scada and sourcetypes hist:control. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Publish limits from P&ID digital twin; (2) test alarms in shadow mode; (3) integrate with MES recipes; (4) alert operations supervisor. Indexes required: `index=scada`; Sourcetypes: `sourcetype=hist:control`; Lookups: `setpoint_limits`, `setpoint_limits.csv`.

Step 1 — Configure data collection
(1) Publish limits from P&ID digital twin; (2) test alarms in shadow mode; (3) integrate with MES recipes; (4) alert operations supervisor.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=scada sourcetype="hist:control" earliest=-24h
| lookup setpoint_limits.csv tag OUTPUT min_sp, max_sp, allowed_modes
| eval bad=if(value<min_sp OR value>max_sp OR !mvfind(allowed_modes, mode),1,0)
| where bad=1
| table _time, tag, value, min_sp, max_sp, mode, operator
```
