# UC-22.14.63 · Continuous Monitoring Control Health Scores

## Description

Rolls up data source freshness, detection coverage, and vuln SLA into a CA-7 continuous monitoring scorecard.

## Value

Rolls up data source freshness, detection coverage, and vuln SLA into a CA-7 continuous monitoring scorecard.

## Implementation

(1) Populate `ca7:kpi` via scheduled searches; (2) weight factors by risk; (3) monthly AO briefing; (4) tie to CA-8 test windows.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263).
• Ensure the following data sources are available: Multiple summary indexes: `summary:ingest_health`, `summary:vuln_sla`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate `ca7:kpi` via scheduled searches; (2) weight factors by risk; (3) monthly AO briefing; (4) tie to CA-8 test windows.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=summary sourcetype=ca7:kpi earliest=-1d
| stats latest(ingest_score) as ingest, latest(detection_score) as det, latest(vuln_score) as vuln by _time env
| eval overall=round((ingest+det+vuln)/3,1)
| where overall < 85
```
