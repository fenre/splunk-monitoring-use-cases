# UC-22.49.4 · Retention policy drift — system config vs policy catalogue

## Description

Ensures system-level retention configuration stays aligned to the authoritative policy catalogue.

## Value

Turns retention policy adherence into a continuously-measured metric per system.

## Implementation

(1) Export retention configuration nightly; (2) maintain retention_policy.csv; (3) run daily.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: System retention configuration snapshots, retention_policy.csv..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Export retention configuration nightly; (2) maintain retention_policy.csv; (3) run daily.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=datastore sourcetype=retention:config earliest=-1d
| lookup retention_policy.csv data_domain OUTPUT policy_days
| eval drift_days=abs(configured_days-policy_days)
| where drift_days>0
| table store data_domain configured_days policy_days drift_days
```
