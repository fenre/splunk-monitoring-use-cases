# 22.30.6 — Incident notification to MAS timeline tracking

**Criticality:** medium | **Difficulty:** expert | **Status:** N/A

## Description

Supports MAS TRM supervisory examination readiness for incident notification to mas timeline tracking.

## Value

Gives the responsible team auditor-ready evidence for MAS TRM compliance by automating regulatory compliance evidence using indexes test and sourcetypes mas:security_test. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=test sourcetype="mas:security_test" earliest=-730d
| stats latest(result) as r by test_id, application
| where r!="Pass"
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=test`; Sourcetypes: `sourcetype=mas:security_test`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones).

## Compliance

- **MAS TRM** 2021: §8.1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for §8.1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
