# 22.15.7 — OT Security Audit Population Sampling Dashboard

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Delivers repeatable audit samples of OT security-relevant events across zones for internal and external assessments of the security program.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_audit earliest=-90d
| lookup ot_zones.csv cidr OUTPUT zone, sl_t
| stats count as evt, dc(user) as users, dc(dest_ip) as peers by zone, sourcetype
| sort zone, - evt
| head 300
```

## CIM SPL

```spl
| tstats summariesonly=t dc(All_Changes.dest) as agg_value from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - agg_value
```

## Implementation

(1) Land firewall, HMI, PAM, IDS into one auditable index with RBAC; (2) maintain CIDR→zone lookup; (3) export monthly CSV to auditors; (4) document sampling method in scope letter. Indexes required: `index=ot_audit`.

## Visualization

Heatmap (zone × sourcetype), Table, Bar (evt by zone).

## Compliance

- **IEC 62443** 2013-ongoing: 4.4.3 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 4.4.3.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
