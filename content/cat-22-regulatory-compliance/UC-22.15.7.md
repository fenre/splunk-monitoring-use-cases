# UC-22.15.7 · OT Security Audit Population Sampling Dashboard

## Description

Delivers repeatable audit samples of OT security-relevant events across zones for internal and external assessments of the security program.

## Value

Delivers repeatable audit samples of OT security-relevant events across zones for internal and external assessments of the security program.

## Implementation

(1) Land firewall, HMI, PAM, IDS into one auditable index with RBAC; (2) maintain CIDR→zone lookup; (3) export monthly CSV to auditors; (4) document sampling method in scope letter.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk OT Security Add-on (5151), CIM Add-on (1621).
• Ensure the following data sources are available: `index=ot_audit` unified; `ot_zones.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Land firewall, HMI, PAM, IDS into one auditable index with RBAC; (2) maintain CIDR→zone lookup; (3) export monthly CSV to auditors; (4) document sampling method in scope letter.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_audit earliest=-90d
| lookup ot_zones.csv cidr OUTPUT zone, sl_t
| stats count as evt, dc(user) as users, dc(dest_ip) as peers by zone, sourcetype
| sort zone, - evt
| head 300
```
