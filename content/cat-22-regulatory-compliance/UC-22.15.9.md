# UC-22.15.9 · Physical Access vs OT Network First-Seen Asset

## Description

Relates physical access events to new or unknown OT assets appearing on protected VLANs to strengthen environmental controls narrative.

## Value

Relates physical access events to new or unknown OT assets appearing on protected VLANs to strengthen environmental controls narrative.

## Implementation

(1) Sync clocks between PACS and NMS; (2) enrich asset owner field; (3) filter tailgating test doors; (4) integrate physical security escalation path.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Nozomi Networks TA (6905), HEC from PACS.
• Ensure the following data sources are available: `index=pacs` `sourcetype="pacs:door"`; `index=ot_net` asset updates.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Sync clocks between PACS and NMS; (2) enrich asset owner field; (3) filter tailgating test doors; (4) integrate physical security escalation path.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=pacs sourcetype="pacs:door" door_id IN ("OT-MCC-A","OT-SERVER-1") earliest=-24h
| eval t_phys=_time, who=badge_id
| join max=0 type=inner who [
  search index=ot_net sourcetype="nozomi:assets" earliest=-24h
  | where first_seen_time>_time-86400
  | eval who=owner
  | stats min(first_seen_time) as fs by asset_name, vlan, who
]
| eval delta_sec=fs-t_phys
| where delta_sec>120 OR isnull(who)
| table _time, door_id, badge_id, asset_name, vlan, delta_sec
```
