# 22.15.9 — Physical Access vs OT Network First-Seen Asset

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Relates physical access events to new or unknown OT assets appearing on protected VLANs to strengthen environmental controls narrative.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_net, pacs and sourcetypes nozomi:assets, pacs:door. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=pacs sourcetype="pacs:door" door_id IN ("OT-MCC-A","OT-SERVER-1") earliest=-24h
| eval t_phys=_time, who=badge_id
| join max=0 type=inner who [
  search index=ot_net sourcetype="nozomi:assets" earliest=-24h
  | where first_seen_time>_time-86400
  | eval who=owner
  | stats min(first_seen_time) as fs by asset_name, vlan, who
]
| eval delta_sec=fs-t_phys
| where delta_sec>120 OR isnull(who)
| table _time, door_id, badge_id, asset_name, vlan, delta_sec
```

## Implementation

(1) Sync clocks between PACS and NMS; (2) enrich asset owner field; (3) filter tailgating test doors; (4) integrate physical security escalation path. Indexes required: `index=ot_net`, `index=pacs`; Sourcetypes: `sourcetype=nozomi:assets`, `sourcetype=pacs:door`.

## Visualization

Table, Map (optional), Time chart. Key fields: `asset_name`, `badge_id`, `delta_sec`, `door_id`, `fs`, `t_phys`.

## Compliance

- **IEC 62443** 2013-ongoing: 4.3.3.3 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 4.3.3.3.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
