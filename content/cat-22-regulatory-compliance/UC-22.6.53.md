# 22.6.53 — ISO/IEC 27001:2022 Clause 7.2 — Competence evidence: role-based training completion

**Criticality:** medium | **Difficulty:** beginner | **Status:** verified

## Description

Joins the role-to-module matrix against LMS completions to report each person's coverage. Clause 7.2 requires the organisation to determine and maintain competence.

## Value

Replaces the annual LMS export with a daily compliance signal. HR can notify managers proactively rather than after the audit finding.

## SPL

```spl
index=hr sourcetype=hr:role_assignment earliest=-30d
| stats latest(role) AS role BY user
| lookup iso_27001_training_matrix.csv role OUTPUT required_modules
| eval required_modules=split(required_modules,",")
| join user
  [search index=hr sourcetype=lms:completion status=completed earliest=-365d
   | stats values(module) AS completed_modules BY user]
| eval covered=mvmap(required_modules, if(isnotnull(mvfind(completed_modules,x)),1,0))
| eval compliance_pct=round(100*sum(covered)/mvcount(required_modules),0)
| where compliance_pct<100
| table user role required_modules completed_modules compliance_pct
```

## Implementation

(1) Onboard LMS completions; (2) maintain iso_27001_training_matrix.csv per role; (3) schedule UC daily; (4) compliance_pct<100 triggers an HR email; (5) quarterly report to ISMS committee. Indexes required: `index=hr`; Sourcetypes: `sourcetype=hr:role_assignment`, `sourcetype=hr:role_assignment;`, `sourcetype=lms:completion`, `sourcetype=lms:completion;`.

## Visualization

Stacked bar by role, table of sub-100% users, single value 'users below 100%'.

## Compliance

- **ISO/IEC 27001** 2022: 7.2 (full) — Clause 7.2 Competence — directly evidenced by role-based module completion.
- **HIPAA Security Rule** 2013-final: §164.308(a)(5) (full) — Uplifted to full: SPL provides comprehensive, standalone evidence for §164.308(a)(5) with specific data sources and actionable thresholds.
- **PCI-DSS** v4.0: 12.6 (partial) — PCI-DSS 12.6 Security awareness programme — contributes.

## Known False Positives

New hires in their grace window (30d) appear as non-compliant — exclude by hire_date.
