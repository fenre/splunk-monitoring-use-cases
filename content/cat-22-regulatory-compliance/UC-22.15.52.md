# UC-22.15.52 · Historian-to-Corporate Data Flow Audit — Large ODBC Without SPN

## Description

Tracks large ODBC/JDBC pulls from OT historian to corporate subnets missing service principal authentication.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes hist_queries and sourcetypes hist:sql. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Require Kerberos or cert auth for service accounts; (2) throttle BI extracts; (3) DLP review for tag exports; (4) document data classification. Indexes required: `index=hist_queries`; Sourcetypes: `sourcetype=hist:sql`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Historian vendor logs.
• Ensure the following data sources are available: `hist:sql` or `pi:audit`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require Kerberos or cert auth for service accounts; (2) throttle BI extracts; (3) DLP review for tag exports; (4) document data classification.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hist_queries sourcetype="hist:sql" earliest=-7d
| eval bytes_mb=round(bytes/1048576,2)
| where bytes_mb>500 AND match(client_ip,"^10\.12\.") AND isnull(service_principal)
| stats sum(bytes_mb) as total_mb by client_app, client_ip, historian_cluster
| sort - total_mb
```
