# 22.29.18 — Access control validation for personal data stores

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Provides APAC privacy supervisory evidence for access control validation for personal data stores aligned with PDPA SG; PIPL; K-ISMS.

## Value

Gives the responsible team auditor-ready evidence for SG PDPA compliance by automating regulatory compliance evidence using indexes linux and sourcetypes linux:secure. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=linux sourcetype="linux:secure" earliest=-24h
| search appname="sudo"
| stats count by user, command
| lookup apac_privileged_cmd_allowlist.csv command OUTPUT allowed
| where allowed!="true"
```

## Implementation

(1) Maintain `apac_transfer_register.csv` and consent or DPIA lookups referenced in runbooks. (2) Map technical identifiers to records of processing activities with legal review. (3) Restrict indexes to regional privacy roles with least privilege. (4) Quarterly DPIA and consent linkage review with local counsel. Indexes required: `index=linux`; Sourcetypes: `sourcetype=linux:secure`.

## Visualization

Stacked bar (jurisdiction x action), Table (open reviews), Single value (exports without mechanism). Key fields: `command`, `user`.
