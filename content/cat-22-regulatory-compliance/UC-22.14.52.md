# UC-22.14.52 · Baseline Configuration Drift vs Gold Build

## Description

Compares endpoint configuration snapshots to approved baselines for CM-2 baseline configuration.

## Value

Compares endpoint configuration snapshots to approved baselines for CM-2 baseline configuration.

## Implementation

(1) Create periodic inventory scripted input; (2) refresh gold CSV from SOE team; (3) alert on drift; (4) map to CM-6 parameters.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (Splunkbase 742), custom scripted baseline forwarder.
• Ensure the following data sources are available: `WinEventLog:Microsoft-Windows-GroupPolicy/Operational` or inventory JSON.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Create periodic inventory scripted input; (2) refresh gold CSV from SOE team; (3) alert on drift; (4) map to CM-6 parameters.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os sourcetype=win_build_inventory earliest=-24h
| lookup gold_windows_build.csv build_number OUTPUT approved_patch_level
| eval drift=if(patch_level<approved_patch_level,"behind","ok")
| where drift!="ok"
```
