# UC-22.14.75 · Vulnerability Monitoring — Exploitable in the Wild Prioritization

## Description

Joins vuln scan data with threat intel tags like KEV for RA-5 vulnerability monitoring.

## Value

Joins vuln scan data with threat intel tags like KEV for RA-5 vulnerability monitoring.

## Implementation

(1) Sync KEV catalog regularly; (2) align SLA to policy; (3) alert owners; (4) integrate patch change windows CM-3.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Tenable Add-On for Splunk (Splunkbase 4060), threat intel lookups.
• Ensure the following data sources are available: `tenable:vuln` with `plugin_id`, `kev_catalog.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Sync KEV catalog regularly; (2) align SLA to policy; (3) alert owners; (4) integrate patch change windows CM-3.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vulns sourcetype=tenable:vuln earliest=-7d state!="Fixed"
| lookup kev_catalog.csv plugin_id OUTPUT in_kev, due_date
| where in_kev="true"
| stats max(patch_due_sla_days) as sla by host, plugin_id
```
