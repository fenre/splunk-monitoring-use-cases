# UC-22.14.64 · Penetration Test Windows and Detected Activities

## Description

Correlates scheduled pen-test IP ranges with IDS alerts for CA-8 penetration testing visibility.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes sec and sourcetypes pan:threat, suricata. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain `pentest_windows.csv` with IPs and dates; (2) reduce false IR-4 by tagging engagements; (3) store ROE in GRC; (4) after-action detection gap review. Indexes required: `index=sec`; Sourcetypes: `sourcetype=pan:threat`, `sourcetype=suricata`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263), IDS sourcetypes.
• Ensure the following data sources are available: `notable` OR `suricata`, `snort`, `pan:threat`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `pentest_windows.csv` with IPs and dates; (2) reduce false IR-4 by tagging engagements; (3) store ROE in GRC; (4) after-action detection gap review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sec (sourcetype=suricata OR sourcetype=pan:threat) earliest=-30d
| lookup pentest_windows.csv src_ip OUTPUT engagement_id, tester
| where isnotnull(engagement_id)
| stats count by engagement_id, signature, dest
| sort - count
```
