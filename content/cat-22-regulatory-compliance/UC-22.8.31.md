# 22.8.31 — SOC 2 CC6.6 — Encryption-in-transit validation: cleartext protocols crossing the trust boundary

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Detects sessions crossing trust-zone boundaries that negotiated plaintext or deprecated TLS (pre-TLS1.2). Under CC6.6 the service organisation must 'implement logical access security measures to protect against threats from sources outside its system boundaries'; plaintext at the boundary is the failure mode.

## Value

Replaces annual TLS-compliance scans with a continuous signal, so drift (a new container accidentally exposing http) is caught within 30 minutes.

## SPL

```spl
index=stream sourcetype=stream:tls earliest=-30m
| rename src_ip AS src dest_ip AS dest
| lookup trust_zones.csv ip=src OUTPUT zone AS src_zone
| lookup trust_zones.csv ip=dest OUTPUT zone AS dest_zone
| where src_zone!=dest_zone
| eval is_deprecated=if(tls_version IN ("SSLv3","TLS1.0","TLS1.1"),1,0)
| where is_deprecated=1 OR protocol IN ("http","ftp","telnet","smtp")
| stats count BY src dest dest_port protocol tls_version is_deprecated
```

## Implementation

(1) Onboard Splunk Stream TLS data; (2) maintain trust_zones.csv; (3) schedule UC every 30m; (4) each hit opens a Platform ticket with src/dest/dest_port; (5) SLA: remediate within 24h. Indexes required: `index=stream`; Sourcetypes: `sourcetype=stream:http`, `sourcetype=stream:tcp`, `sourcetype=stream:tls`, `sourcetype=stream:tls;`.

## Visualization

Sankey of src_zone → dest_zone by protocol, table of deprecated sessions, single value 'deprecated TLS sessions right now'.

## Compliance

- **SOC-2** 2017 TSC: CC6.6 (full) — CC6.6 Logical access security measures protect the boundary — cleartext crossing a trust boundary is the direct failure detected.
- **PCI-DSS** v4.0: 4.2.1 (full) — PCI-DSS 4.2.1 strong cryptography for transmission of cardholder data across open networks — directly evidenced.
- **ISO/IEC 27001** 2022: A.8.24 (full) — ISO/IEC 27001:2022 A.8.24 Use of cryptography — directly evidenced.
- **HIPAA Security Rule** 2013-final: §164.312(e)(1) (full) — Uplifted to full: SPL provides comprehensive, standalone evidence for §164.312(e)(1) with specific data sources and actionable thresholds.

## Known False Positives

Explicit plaintext management LANs (backup target, ILO) are allowlisted in trust_zones.csv via category='management-cleartext-allowed'.
