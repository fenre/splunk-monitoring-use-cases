# 22.21.6 — Transparency — Explainability Payload Presence (Art. 13)

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Confirms user-facing responses include required explanation fields (summary, confidence, limitations) for transparency obligations under Art. 13.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_ux and sourcetypes ai:chat:response, ai:chat:response`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_ux sourcetype="ai:chat:response" earliest=-24h
| eval has_sum=if(len(explanation_summary)>20,1,0)
| eval has_conf=if(isnum(confidence),1,0)
| eval has_lim=if(len(limitations)>10,1,0)
| eval art13_ok=if(has_sum+has_conf+has_lim=3,1,0)
| stats count as n, sum(art13_ok) as ok by product, model_id
| eval ok_pct=round(100*ok/n,2)
| where ok_pct < 100
| sort + ok_pct
```

## Implementation

(1) Instrument API gateway to log structured explainability fields; (2) redact PII in explanations; (3) weekly report to product/legal; (4) tie to conformity documentation version. Indexes required: `index=ai_ux`; Sourcetypes: `sourcetype=ai:chat:response`, `sourcetype=ai:chat:response``.
