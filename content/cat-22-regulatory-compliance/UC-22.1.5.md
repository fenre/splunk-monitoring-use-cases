# UC-22.1.5 · GDPR Consent Management Audit Trail

## Description

Preserves a searchable trail of consent grant, refuse, and withdraw events from web applications for accountability and consent withdrawal parity requirements.

## Value

Preserves a searchable trail of consent grant, refuse, and withdraw events from web applications for accountability and consent withdrawal parity requirements.

## Implementation

(1) Ingest Apache/nginx access logs via TA 3186 or Universal Forwarder file inputs; (2) for richer evidence, emit HEC JSON from the consent microservice with explicit `action`, `purpose_id`, and hashed subject ID fields; (3) map URIs to consent purposes via a lookup `consent_uri_map.csv`; (4) restrict index ACLs to privacy teams; (5) schedule weekly reporting for consent withdrawal ratio monitoring.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Apache Web Server (Splunkbase 3186), HTTP Event Collector (HEC — platform capability for structured JSON from consent APIs).
• Ensure the following data sources are available: `index=web` `sourcetype="access_combined"` (clientip, uri, method, status, useragent) for consent page interactions; or custom HEC JSON events with explicit consent fields.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest Apache/nginx access logs via TA 3186 or Universal Forwarder file inputs; (2) for richer evidence, emit HEC JSON from the consent microservice with explicit `action`, `purpose_id`, and hashed subject ID fields; (3) map URIs to consent purposes via a lookup `consent_uri_map.csv`; (4) restrict index ACLs to privacy teams; (5) schedule weekly reporting for consent withdrawal ratio monitoring.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=web sourcetype="access_combined" earliest=-7d
    (uri="*/consent*" OR uri="*/privacy-preferences*")
| rex field=uri_query "action=(?<consent_action>[^&]+)"
| eval consent_event=coalesce(consent_action, if(status=200, "page_view", "error"))
| stats count by clientip, uri, consent_event, status
| sort - count
```
