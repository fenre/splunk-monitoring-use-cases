# 22.2.25 — NIS2 Supply Chain Security Assessment Coverage

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Beyond detecting abnormal supplier sessions (UC-22.2.2), entities must assess and document supply chain risk. This use case tracks completion of security questionnaires, on-site assessments, and software bill of materials (SBOM) reviews for suppliers touching NIS2-scoped services.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes github, itsm and sourcetypes dependabot_alert. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
| inputlookup nis2_supplier_assessment_register.csv
| eval assessment_due=strptime(next_assessment_date,"%Y-%m-%d")
| eval overdue=if(now()>assessment_due,1,0)
| join supplier_id [
    search index=github sourcetype="dependabot_alert" earliest=-30d
    | stats dc(alert_number) as open_supply_chain_alerts by repo
    | rename repo as supplier_code_repo
]
| where overdue=1 OR open_supply_chain_alerts > 10
| table supplier_name, tier, next_assessment_date, overdue, open_supply_chain_alerts
```

## Implementation

(1) Classify suppliers by tier in the register; (2) sync assessment due dates from GRC or ServiceNow; (3) correlate critical repos with supplier-maintained code; (4) escalate overdue tier-1 assessments to procurement and CISO; (5) document compensating controls when assessments slip. Indexes required: `index=github`, `index=itsm`; Sourcetypes: `sourcetype=dependabot_alert`; Lookups: `nis2_supplier_assessment_register`, `nis2_supplier_assessment_register.csv`.

## Visualization

Table (overdue assessments), Bar chart (alerts by supplier), Single value (tier-1 gaps).

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(d) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(d) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
