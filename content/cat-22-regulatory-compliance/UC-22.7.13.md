# UC-22.7.13 · Documented Baseline Drift — Firewall Rule Adds Outside CAB Window

## Description

Flags new allow rules created outside change windows to prove security policies embedded in change management are monitored.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes network and sourcetypes pan:config. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize config sourcetype for your firewall; (2) ingest change calendar; (3) tune for automation accounts; (4) attach to policy exception workflow. Indexes required: `index=network`; Lookups: `change_calendar`, `change_calendar.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (Splunkbase 2757).
• Ensure the following data sources are available: `pan:config` or syslog config events (rule, admin, cmd).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize config sourcetype for your firewall; (2) ingest change calendar; (3) tune for automation accounts; (4) attach to policy exception workflow.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype=pan:config earliest=-30d cmd="set" object="rule"
| lookup change_calendar.csv _time OUTPUT cab_window
| where cab_window!="open" OR isnull(cab_window)
```
