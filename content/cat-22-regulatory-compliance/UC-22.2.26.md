# 22.2.26 — NIS2 Network Security Monitoring Coverage by Segment

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Network security policies must be enforced uniformly. This use case measures log source coverage and IDS inspection rates per network segment defined in the security architecture — identifying blind spots where NIS2-scoped traffic is not visible to Splunk.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes netfw. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
| inputlookup nis2_segment_coverage_expectations.csv
| join segment_id [
    | tstats `summariesonly` count where index=netfw earliest=-24h by index
    | eval segment_id=case(
        match(index,"(?i)corp"),"CORP",
        match(index,"(?i)dmz"),"DMZ",
        1=1,"OTHER")
    | stats sum(count) as events by segment_id
]
| eval expected_eps=expected_events_per_day/86400*3600
| eval coverage_gap=if(events < expected_eps*0.5, 1, 0)
| where coverage_gap=1
| table segment_id, events, expected_eps, nis2_scope
```

## CIM SPL

```spl
| tstats summariesonly=t sum(All_Traffic.bytes_in) as agg_value from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - agg_value
```

## Implementation

(1) Define minimum event rates per segment from baselines; (2) tag segments as in-scope for NIS2; (3) alert when collectors fail or indexes stop receiving; (4) integrate with ES data source monitoring; (5) quarterly architecture review of blind spots. Indexes required: `index=netfw`; Lookups: `nis2_segment_coverage_expectations`, `nis2_segment_coverage_expectations.csv`.

## Visualization

Table (segments with gaps), Single value (blind segments), Heatmap (EPS by hour), Map (site coverage). Key fields: `coverage_gap`, `events`, `expected_eps`, `nis2_scope`, `segment_id`.

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(a) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(a) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
