# 22.11.92 — PCI-DSS 2.2 — Secure configuration baseline: drift from approved hardening template

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Baseline compliance for CDE hosts. PCI-DSS 2.2 requires configuration standards that address known security vulnerabilities; drift from the approved baseline is the direct failure.

## Value

Turns the quarterly 'baseline assessment' into a 30-minute signal. Platform team sees regressions before the QSA.

## SPL

```spl
index=configmgmt sourcetype IN (chef:compliance,ansible:callback) earliest=-30m
| stats latest(role) AS role latest(baseline_version) AS baseline_version latest(resource_failed) AS resource_failed latest(_time) AS last_converged BY hostname
| eval drift_count=coalesce(resource_failed,0)
| where drift_count>0 OR last_converged<now()-86400
| table hostname role baseline_version drift_count last_converged
```

## Implementation

(1) Enforce Chef/Ansible on all CDE hosts; (2) emit convergence events; (3) maintain pci_baselines.csv with the current approved baseline_version per role; (4) schedule UC every 30m; (5) drift opens a Platform ticket. Indexes required: `index=configmgmt`; Sourcetypes: `sourcetype=ansible:callback`, `sourcetype=ansible:callback;`, `sourcetype=chef:compliance`.

## Visualization

Bar of drift_count by role, table of top-drift hosts, single value 'hosts in drift'.

## Compliance

- **PCI-DSS** v4.0: 2.2 (full) — PCI-DSS 2.2 Configuration standards — directly evidenced.
- **ISO/IEC 27001** 2022: A.8.9 (full) — ISO/IEC 27001:2022 A.8.9 Configuration management — directly evidenced.
- **SOC-2** 2017 TSC: CC8.1 (partial) — SOC 2 CC8.1 — contributing.
- **NIST 800-53** Rev. 5: CM-6 (full) — Uplifted to full: SPL provides comprehensive, standalone evidence for CM-6 with specific data sources and actionable thresholds.

## Known False Positives

Baseline upgrades take effect over a staged rollout — support multiple active baseline_versions in the lookup.
