# UC-22.11.92 · PCI-DSS 2.2 — Secure configuration baseline: drift from approved hardening template

## Description

Baseline compliance for CDE hosts. PCI-DSS 2.2 requires configuration standards that address known security vulnerabilities; drift from the approved baseline is the direct failure.

## Value

Turns the quarterly 'baseline assessment' into a 30-minute signal. Platform team sees regressions before the QSA.

## Implementation

(1) Enforce Chef/Ansible on all CDE hosts; (2) emit convergence events; (3) maintain pci_baselines.csv with the current approved baseline_version per role; (4) schedule UC every 30m; (5) drift opens a Platform ticket.

## Detailed Implementation

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enforce Chef/Ansible on all CDE hosts; (2) emit convergence events; (3) maintain pci_baselines.csv with the current approved baseline_version per role; (4) schedule UC every 30m; (5) drift opens a Platform ticket.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=configmgmt sourcetype IN (chef:compliance,ansible:callback) earliest=-30m
| stats latest(role) AS role latest(baseline_version) AS baseline_version latest(resource_failed) AS resource_failed latest(_time) AS last_converged BY hostname
| eval drift_count=coalesce(resource_failed,0)
| where drift_count>0 OR last_converged<now()-86400
| table hostname role baseline_version drift_count last_converged
```
