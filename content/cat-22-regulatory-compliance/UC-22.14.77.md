# UC-22.14.77 · Threat Hunting Outcomes Logged for Repeatable Hunts

## Description

Structures hunt hypothesis, queries, and findings for RA-10 threat hunting (as organizational capability evidence).

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes thunt and sourcetypes hunt:record, hunt:record`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Require hunt closure fields; (2) link to IR tickets when escalated; (3) quarterly metrics for leadership; (4) library of proven SPL in repo outside Splunk. Indexes required: `index=thunt`; Sourcetypes: `sourcetype=hunt:record`, `sourcetype=hunt:record``.

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require hunt closure fields; (2) link to IR tickets when escalated; (3) quarterly metrics for leadership; (4) library of proven SPL in repo outside Splunk.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=thunt sourcetype=hunt:record earliest=-90d
| stats values(hypothesis) as hyps, latest(outcome) as outcome, max(duration_min) as duration by hunt_id, hunter
| where outcome="inconclusive" OR duration>480
```
