# UC-22.14.30 · Authentication Feedback — Credential Stuffing via Login Failures

## Description

Uses failed authentication ratios without exposing secrets to tune IA-6 authentication feedback monitoring at the aggregate level.

## Value

Uses failed authentication ratios without exposing secrets to tune IA-6 authentication feedback monitoring at the aggregate level.

## Implementation

(1) Do not log raw passwords — rely on platform signals only; (2) integrate threat intel on ClientIP; (3) rate-limit alerts; (4) feed SOAR password spray playbooks.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365 (Splunkbase 4055).
• Ensure the following data sources are available: `ms:o365:management` sign-in failures.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Do not log raw passwords — rely on platform signals only; (2) integrate threat intel on ClientIP; (3) rate-limit alerts; (4) feed SOAR password spray playbooks.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=o365 sourcetype=ms:o365:management earliest=-24h ResultStatus="Failed"
| stats count by ClientIP, UserId
| where count>50
| sort - count
```
