# 22.21.18 — EU Database Registration Event Audit (Art. 49)

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Tracks registration_id, status, and error codes from EU database API client logs for conformity registration evidence.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_registry and sourcetypes eu:ai:db:client, eu:ai:db:client`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_registry sourcetype="eu:ai:db:client" earliest=-30d
| stats count as calls, sum(eval(http_status>=400)) as errors by operation, registration_id
| eval err_rate=round(100*errors/calls,2)
| where err_rate>5 OR errors>0 AND calls<20
| sort - err_rate
```

## Implementation

(1) Mask secrets in _raw; (2) alert on sustained 5xx; (3) reconcile with legal registry export weekly; (4) retain API idempotency keys. Indexes required: `index=ai_registry`; Sourcetypes: `sourcetype=eu:ai:db:client`, `sourcetype=eu:ai:db:client``.

## Visualization

Time chart (errors), Table (registration_id with errors), Single value (global err_rate)
