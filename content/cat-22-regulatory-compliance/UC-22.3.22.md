# UC-22.3.22 · DORA ICT Concentration — Critical Service Dependency Fan-In by Provider

## Description

Multiple critical services routing through one provider’s shared edge or identity layer is hidden concentration. Fan-in metrics highlight when too many important business functions share a single third-party control point.

## Value

Multiple critical services routing through one provider’s shared edge or identity layer is hidden concentration. Fan-in metrics highlight when too many important business functions share a single third-party control point.

## Implementation

(1) Normalize `provider_id` to legal entity; (2) refresh dependency graph weekly; (3) cross-check with contractual sub-processors list; (4) alert architecture review board; (5) attach to ICT concentration register annex.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: CMDB export via Splunk DB Connect, custom sourcetype.
• Ensure the following data sources are available: `index=cmdb` `sourcetype="cmdb:dependency"` (service_id, upstream_ci, provider_id, tier).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize `provider_id` to legal entity; (2) refresh dependency graph weekly; (3) cross-check with contractual sub-processors list; (4) alert architecture review board; (5) attach to ICT concentration register annex.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cmdb sourcetype="cmdb:dependency" tier="critical" earliest=-1d
| stats dc(service_id) as dependent_services by provider_id, upstream_ci
| where dependent_services>15
| sort - dependent_services
```
