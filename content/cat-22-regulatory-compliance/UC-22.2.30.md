# 22.2.30 — NIS2 Human Resources Security Measures Evidence

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Policies on human resources security — screening, training, and termination procedures — are part of holistic risk management. This use case ingests LMS completion for privileged-role staff, background-check status flags (non-PII aggregates), and leaver access revocation timing for NIS2-scoped roles.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes training. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
| inputlookup nis2_privileged_role_roster.csv
| join user_id [
    search index=training earliest=-365d
    | stats latest(completion_date) as last_training by user_id
]
| eval training_age=round((now()-strptime(last_training,"%Y-%m-%d"))/86400,0)
| where training_age > 365 OR isnull(last_training)
| table role_name, user_id, last_training, training_age
```

## Implementation

(1) Hash user identifiers in training feeds where possible; (2) require annual secure-development and IR training for NIS2 privileged roles; (3) alert managers on overdue training; (4) correlate HR termination dates with Okta deprovision latency (aggregate only); (5) export compliance percentages for audits. Indexes required: `index=training`; Lookups: `nis2_privileged_role_roster`, `nis2_privileged_role_roster.csv`.

## Visualization

Table (overdue training), Single value (% compliant), Bar chart (by role family), Timeline (completion waves). Key fields: `last_training`, `role_name`, `training_age`, `user_id`.

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(i) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(i) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
