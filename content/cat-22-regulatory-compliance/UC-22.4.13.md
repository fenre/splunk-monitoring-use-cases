# UC-22.4.13 · CCPA/CPRA — Data Broker Registry — Opt-Out Propagation Latency to Downstream Data Partners

## Description

When consumers opt out of sale/share, partners must stop promptly. Measuring time from opt-out receipt to partner acknowledgment API or file manifest update demonstrates operational effectiveness beyond UI banners.

## Value

Gives the responsible team auditor-ready evidence for CCPA/CPRA compliance by automating regulatory compliance evidence using indexes privacy and sourcetypes consent:optout, partner:ack. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Set `72` hours per contractual SLA; (2) include GPC batch jobs as `channel`; (3) legal escalation for chronic laggards; (4) exclude partners in wind-down via lookup; (5) evidence for CPRA “do not sell/share” audits. Indexes required: `index=privacy`; Sourcetypes: `sourcetype=consent:optout`, `sourcetype=partner:ack`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector from consent platform and partner webhooks.
• Ensure the following data sources are available: `index=privacy` `sourcetype="consent:optout"` (consumer_id, optout_ts, channel); `index=privacy` `sourcetype="partner:ack"` (consumer_id, partner_id, ack_ts).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Set `72` hours per contractual SLA; (2) include GPC batch jobs as `channel`; (3) legal escalation for chronic laggards; (4) exclude partners in wind-down via lookup; (5) evidence for CPRA “do not sell/share” audits.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="consent:optout" earliest=-90d
| eval oo=strptime(optout_ts,"%Y-%m-%dT%H:%M:%SZ")
| join type=left consumer_id [
    search index=privacy sourcetype="partner:ack" earliest=-90d
    | eval ak=strptime(ack_ts,"%Y-%m-%dT%H:%M:%SZ")
    | stats min(ak) as first_ack by consumer_id, partner_id
  ]
| eval lag_h=if(isnotnull(first_ack), round((first_ack-oo)/3600,2), null())
| where isnull(first_ack) OR lag_h>72
| stats count by partner_id
| sort - count
```
