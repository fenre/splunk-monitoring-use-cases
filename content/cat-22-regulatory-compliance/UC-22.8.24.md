# UC-22.8.24 · SOC 2 CC8 — Infrastructure-as-Code Drift vs Approved Terraform Modules

## Description

CC8 governs change to infrastructure. Comparing live cloud resource tags and configurations to approved module signatures detects shadow infrastructure that bypassed pipeline controls—supporting change integrity for cloud environments.

## Value

CC8 governs change to infrastructure. Comparing live cloud resource tags and configurations to approved module signatures detects shadow infrastructure that bypassed pipeline controls—supporting change integrity for cloud environments.

## Implementation

(1) Adjust `spath` to your Config JSON shape; (2) maintain golden baseline per account; (3) integrate with Terraform Cloud audit API for module version; (4) route to cloud governance; (5) document exceptions in change register only.

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cloud sourcetype="aws:config_snapshot" resourceType="AWS::EC2::Instance" earliest=-1d
| spath path=configuration path=output
| eval module_tag=mvindex('tags{}.value', mvfind('tags{}.key', "TerraformModule"))
| lookup approved_modules.csv resourceType OUTPUT required_tag_module
| where isnull(module_tag) OR module_tag!=required_tag_module
| stats count by module_tag, resourceType, awsAccountId
| sort - count
```
