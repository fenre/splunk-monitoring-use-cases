# 22.33.9 — Secure system development lifecycle evidence

**Criticality:** low | **Difficulty:** advanced | **Status:** N/A

## Description

Aligns SAMA Cyber Security Framework supervisory expectations with measurable Splunk evidence for secure system development lifecycle evidence.

## Value

Gives the responsible team auditor-ready evidence for SAMA CSF compliance by automating regulatory compliance evidence using indexes sdlc and sourcetypes jira:issue. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=sdlc sourcetype="jira:issue" project="SEC" earliest=-90d
| search labels="*SAMA*" AND status!="Done"
| stats count by assignee, status
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```

## Implementation

(1) Normalize bilingual field extractions in props.conf where required. (2) Map IP ranges and business units to national critical infrastructure and banking registers. (3) Route SAMA, QCB, NESA, and PDPL alerts to regional bilingual runbooks. (4) Weekly leadership digest with regulator-specific views. Indexes required: `index=sdlc`; Sourcetypes: `sourcetype=jira:issue`.

## Visualization

Bar chart (sector), Table (top flows), Single value (policy denies). Key fields: `assignee`, `status`.

## Compliance

- **SAMA CSF** v1.0 (2017): 3.1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 3.1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
