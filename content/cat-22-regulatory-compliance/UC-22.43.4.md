# UC-22.43.4 · Scanner coverage gap — regulated hosts without a recent scan

## Description

Turns scanner coverage into a continuously-measured KPI instead of a quarterly spreadsheet.

## Value

Surfaces silent scanner onboarding failures before they become audit findings.

## Implementation

(1) Maintain asset_inventory.csv; (2) export scanner state to scan_history.csv; (3) run daily.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: Vulnerability scanner inventory, asset_inventory.csv with regulated flag, per-tier SLA lookup..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain asset_inventory.csv; (2) export scanner state to scan_history.csv; (3) run daily.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup asset_inventory.csv
| where regulated="true"
| lookup scan_history.csv host OUTPUT last_scan
| eval days_since=round((now()-strptime(last_scan,"%Y-%m-%d"))/86400,0)
| lookup tier_sla.csv tier OUTPUT scan_sla_days
| where isnull(last_scan) OR days_since>scan_sla_days
| table tier host last_scan days_since scan_sla_days
```
