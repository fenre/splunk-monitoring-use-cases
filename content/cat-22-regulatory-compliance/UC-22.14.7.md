# UC-22.14.7 ┬Ę Time Synchronization and Clock Skew for Audit Timestamps

## Description

Surfaces hosts whose event `_time` diverges from indexer time beyond tolerance, evidencing AU-8 time stamp synchronization.

## Value

Surfaces hosts whose event `_time` diverges from indexer time beyond tolerance, evidencing AU-8 time stamp synchronization.

## Implementation

(1) Standardize NTP/chrony on all forwarders; (2) tune thresholds per sourcetype (batch logs vs real-time); (3) alert Tier0 assets on breach; (4) attach remediation runbook for clock drift.

Step 2 ŌĆö Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os earliest=-24h
| eval skew_sec=abs(_indextime-_time)
| stats perc95(skew_sec) as p95_skew, max(skew_sec) as max_skew by host, sourcetype
| where p95_skew>120 OR max_skew>300
```
