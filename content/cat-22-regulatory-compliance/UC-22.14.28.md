# UC-22.14.28 · Identifier Management — Non-Human Service Account Sprawl

## Description

Tracks creation of service principals and machine accounts for IA-4 identifier management discipline.

## Value

Tracks creation of service principals and machine accounts for IA-4 identifier management discipline.

## Implementation

(1) Maintain authoritative registry CSV or KVStore; (2) integrate IAM ticketing for approvals; (3) monthly cleanup report; (4) align naming standards with CM-8 tags.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (Splunkbase 742), Splunk Add-on for AWS (Splunkbase 1876).
• Ensure the following data sources are available: `WinEventLog:Security` 4741/4742, `aws:cloudtrail` `CreateUser`/`CreateRole`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain authoritative registry CSV or KVStore; (2) integrate IAM ticketing for approvals; (3) monthly cleanup report; (4) align naming standards with CM-8 tags.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sec sourcetype=WinEventLog:Security EventCode IN (4741,4742) earliest=-30d
| stats latest(_time) as last_seen by Account_Name, EventCode
| lookup svc_account_registry.csv Account_Name OUTPUT owner_team, approved
| where approved!="true" OR isnull(approved)
```
