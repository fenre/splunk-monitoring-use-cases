# UC-22.12.36 · SOX-ITGC AccessMgmt.Provisioning — Financial-system user provisioning SLA & workflow adherence

## Description

For every add to a financial-system group, assert an approved ServiceNow request exists and provisioning happened within 24h of approval.

## Value

Closes the #1 SOX deficiency 'access provisioned without an approved ticket'. Internal audit gets a daily report rather than a quarterly spot-check.

## Implementation

(1) Maintain financial_systems_groups.csv; (2) enforce ServiceNow workflow; (3) schedule UC daily; (4) sla_met=0 opens a SOX deficiency record; (5) monthly CFO report. Indexes required: `index=iam`, `index=ticketing`; Sourcetypes: `sourcetype=ad:group_change;`, `sourcetype=okta:system_log`, `sourcetype=servicenow:request`, `sourcetype=servicenow:request.`.

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain financial_systems_groups.csv; (2) enforce ServiceNow workflow; (3) schedule UC daily; (4) sla_met=0 opens a SOX deficiency record; (5) monthly CFO report.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iam eventType=group.user_membership.add earliest=-7d
| lookup financial_systems_groups.csv group OUTPUT in_scope
| where in_scope="yes"
| rename actor AS added_by target.id AS user
| eval added_at=_time
| join type=outer user group
  [search index=ticketing sourcetype=servicenow:request request_type=access_request state=approved earliest=-30d
   | stats latest(ticket_id) AS ticket_id latest(approved_at) AS approved_at BY user target_group
   | rename target_group AS group]
| eval sla_met=if(isnotnull(approved_at) AND added_at<=approved_at+24*3600,1,0)
| where sla_met=0
| table group user added_at ticket_id approved_at sla_met
```
