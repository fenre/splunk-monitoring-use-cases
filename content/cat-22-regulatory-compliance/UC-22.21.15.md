# 22.21.15 — Operator Session Segregation for High-Risk AI Consoles (Art. 14)

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Detects concurrent high-risk console sessions per operator_id to support segregation of duties and oversight controls.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_console and sourcetypes ai:admin:session, ai:admin:session`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_console sourcetype="ai:admin:session" earliest=-24h
| bin _time span=5m
| stats dc(session_id) as concurrent_sessions by _time, operator_id
| where concurrent_sessions > 1
| timechart span=1h max(concurrent_sessions) by operator_id
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src span=1h | sort - count
```

## Implementation

(1) Ingest IdP session logs if console lacks them; (2) tune threshold per role; (3) ES correlation for shared accounts; (4) policy attestation annually. Indexes required: `index=ai_console`; Sourcetypes: `sourcetype=ai:admin:session`, `sourcetype=ai:admin:session``.

## Visualization

Time chart (concurrent_sessions), Table (operator_id peaks), Single value (distinct operators affected)

## Compliance

- **EU AI Act** Regulation (EU) 2024/1689: Art.14 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.14.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
