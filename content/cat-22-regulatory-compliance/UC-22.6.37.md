# UC-22.6.37 · Redundancy of IT — Cluster Node Loss Events for Critical Databases

## Description

Tracks loss of quorum or node eviction in database clusters supporting redundancy requirements.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes database and sourcetypes mssql:ag. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map AG to business service; (2) ITSI optional correlation; (3) DR failover evidence. Indexes required: `index=database`; Sourcetypes: `sourcetype=mssql:ag`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: SQL Server AG logs.
• Ensure the following data sources are available: `mssql:ag`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map AG to business service; (2) ITSI optional correlation; (3) DR failover evidence.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=database sourcetype=mssql:ag earliest=-90d
| where match(message,"(?i)evict|offline|lost.quorum")
| stats count by ag_name, replica_server_name
```
