# UC-22.14.44 · Incident Response Training Completion Tracking

## Description

Joins LMS completion events with HR role for IR-2 incident response training evidence.

## Value

Joins LMS completion events with HR role for IR-2 incident response training evidence.

## Implementation

(1) Hash user identifiers consistently; (2) refresh role lookup from IdM; (3) quarterly compliance report; (4) store exports for assessors.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector (HEC), Splunk Add-on for ServiceNow (Splunkbase 1928).
• Ensure the following data sources are available: `lms:completion` HEC JSON or `snow:training_task` if modeled.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Hash user identifiers consistently; (2) refresh role lookup from IdM; (3) quarterly compliance report; (4) store exports for assessors.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr_training sourcetype=lms:completion earliest=-365d course_id="IR-101"
| stats latest(completed_at) as completed by user_id
| inputlookup append=t ir_required_roles.csv
| stats first(completed) as completed, values(role) as roles by user_id
| where isnull(completed) AND roles="SOC"
```
