# UC-22.15.5 · IACS Incident Response Containment Interval

## Description

Quantifies detection-to-containment for OT cyber incidents and separates drills from production to evidence an exercised IR plan.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_incidents and sourcetypes snow:incident. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Require containment milestone in workflow; (2) align timestamps to OT SOC clock; (3) compare drill vs live separately; (4) feed p95 into program KPIs. Indexes required: `index=ot_incidents`; Sourcetypes: `sourcetype=snow:incident`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for ServiceNow (1928).
• Ensure the following data sources are available: `index=ot_incidents` ServiceNow `snow:incident` or SOAR with `contained_at`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_incidents sourcetype="snow:incident" earliest=-180d
| where category="OT/Cyber" OR match(short_description,"(?i)ics|scada|plc")
| eval opened=strptime(opened_at,"%Y-%m-%d %H:%M:%S"), contained=strptime(contained_at,"%Y-%m-%d %H:%M:%S")
| eval is_drill=if(match(tag,"drill") OR match(short_description,"(?i)drill|tabletop"),1,0)
| eval contain_min=round((contained-opened)/60,1)
| where isnotnull(contain_min)
| stats median(contain_min) as p50, perc95(contain_min) as p95, count by is_drill
```
