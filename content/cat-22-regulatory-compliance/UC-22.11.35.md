# UC-22.11.35 · Web Application Firewall Blocks and Anomalies on Checkout URIs

## Description

Aggregates ModSecurity, AWS WAF, or commercial WAF denies on payment paths so public-facing web applications retain active attack detection between code releases.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes waf and sourcetypes aws:waf:logs, modsec:json. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize field names with CIM aliases. (2) Enrich `clientIp` with threat intel. (3) Tune out scanner noise using rate limits. (4) Feed spikes to application on-call. Indexes required: `index=waf`; Sourcetypes: `sourcetype=aws:waf:logs`, `sourcetype=modsec:json`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=waf (sourcetype="aws:waf:logs" OR sourcetype="modsec:json") earliest=-24h
| where match(coalesce(uri,httpRequest.uri),"(?i)/checkout|/payment|/cart|/token")
| where action IN ("BLOCK","block","deny","403")
| stats count by terminatingRuleId, coalesce(uri,httpRequest.uri) as path, clientIp
| sort - count
| head 200
```
