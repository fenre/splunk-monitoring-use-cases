# 22.21.17 — Technical Documentation Completeness Checklist (Art. 11/Annex IV)

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Scores required documentation sections present per model_id for Annex IV technical documentation completeness.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_docs and sourcetypes ai:tdoc:status. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_docs sourcetype="ai:tdoc:status" earliest=-7d
| stats values(section_id) as sections, sum(present) as present_cnt by model_id
| eval required=12
| eval completeness=round(100*present_cnt/required,2)
| where completeness < 100
| sort + completeness
```

## Implementation

(1) Define canonical section_id list; (2) integrate CMS webhook; (3) block EU deploy if completeness < 100; (4) store PDF hashes externally. Indexes required: `index=ai_docs`; Sourcetypes: `sourcetype=ai:tdoc:status`.

## Visualization

Gauge-style single value (avg completeness), Table (missing sections), Bar chart (by model_id)

## Compliance

- **EU AI Act** Regulation (EU) 2024/1689: Art.13 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.13.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
