# UC-22.21.22 · Serious Incident Report Pack Timeliness (Art. 73)

## Description

Tracks time from serious_incident_detected to regulator_packet_submitted for serious incident reporting workflows.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_incident and sourcetypes ai:serious:incident. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Legal defines milestones; (2) immutable append-only index ACL; (3) alert at 60h; (4) attach packet hash in external DMS. Indexes required: `index=ai_incident`; Sourcetypes: `sourcetype=ai:serious:incident`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk DB Connect (2686).
• Ensure the following data sources are available: `index=ai_incident` — incident_id, milestone, _time.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Legal defines milestones; (2) immutable append-only index ACL; (3) alert at 60h; (4) attach packet hash in external DMS.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ai_incident sourcetype="ai:serious:incident" earliest=-180d
| eval m_detect=if(milestone="detected",_time,null())
| eval m_submit=if(milestone="regulator_submitted",_time,null())
| stats min(m_detect) as t0, min(m_submit) as t1 by incident_id
| eval hours_to_submit=round((t1-t0)/3600,2)
| where isnull(t1) OR hours_to_submit>72
| table incident_id, t0, t1, hours_to_submit
```
