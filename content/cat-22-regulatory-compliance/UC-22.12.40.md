# UC-22.12.40 · SOX-ITGC Operations.JobSchedule — Batch-schedule monitoring: financial-job exception visibility

## Description

Measures per-job batch exceptions and requires operator acknowledgement within 4h. SOX-ITGC Operations.JobSchedule requires exception monitoring for financial batch jobs.

## Value

Makes the typically-spreadsheet-based batch monitoring queryable — Ops has a live queue, auditors have a history.

## Implementation

(1) Route Autosys/Control-M events; (2) maintain financial_batch_jobs.csv; (3) schedule UC every 30m; (4) unacked exceptions open an incident; (5) monthly CFO + Head of IT Ops review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security.
• Ensure the following data sources are available: `index=batch` sourcetype=autosys:event OR sourcetype=controlm:job; operator ack `index=ticketing` sourcetype=servicenow:incident (category=batch-ack)..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Route Autosys/Control-M events; (2) maintain financial_batch_jobs.csv; (3) schedule UC every 30m; (4) unacked exceptions open an incident; (5) monthly CFO + Head of IT Ops review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=batch sourcetype IN (autosys:event,controlm:job) earliest=-30m
| lookup financial_batch_jobs.csv job_id OUTPUT in_scope expected_window
| where in_scope="yes"
| stats latest(state) AS state latest(_time) AS run_at BY job_id
| where state IN ("FAILED","LATE","WARNING")
| join type=outer job_id
  [search index=ticketing sourcetype=servicenow:incident category=batch-ack earliest=-24h
   | stats latest(assigned_to) AS ack_operator latest(_time) AS ack_at BY job_id]
| eval ack_met=if(isnotnull(ack_operator) AND ack_at>=run_at AND ack_at<=run_at+4*3600,1,0)
| where ack_met=0
| table job_id run_at state ack_operator ack_at ack_met
```
