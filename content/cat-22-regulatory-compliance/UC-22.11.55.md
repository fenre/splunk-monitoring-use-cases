# UC-22.11.55 · Session Timeout Enforcement on Payment Web Admin Consoles

## Description

Measures idle session duration before logout on administrative payment portals using HTTP session cookies or app logs so inactive sessions re-authenticate within policy limits.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes web and sourcetypes access_combined. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align `maxpause` with configured server session timeout. (2) Mask `session_id` in dashboards. (3) Tune for long-running report downloads. (4) Pair with WAF session fixation checks. Indexes required: `index=web`; Sourcetypes: `sourcetype=access_combined`.

## Detailed Implementation

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=web sourcetype="access_combined" earliest=-24h
| where match(uri,"(?i)/admin|/portal/manage")
| transaction session_id maxpause=30m maxspan=8h
| eval idle_mins=round(duration/60,2)
| where idle_mins>15
| stats max(idle_mins) as max_idle by session_id, clientip
| sort - max_idle
```
