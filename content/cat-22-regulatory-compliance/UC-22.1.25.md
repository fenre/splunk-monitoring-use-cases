# UC-22.1.25 · GDPR Security Incident Tracking Linked to Personal Data Impact

## Description

Article 32(1)(c) requires the ability to restore availability and access in a timely manner, and incident handling underpins the integrity and resilience of processing. This use case tags security incidents with personal-data impact classification, tracks containment and eradication milestones, and aligns with breach notification workflows — extending generic SOC metrics with GDPR-specific impact labels.

## Value

Article 32(1)(c) requires the ability to restore availability and access in a timely manner, and incident handling underpins the integrity and resilience of processing. This use case tags security incidents with personal-data impact classification, tracks containment and eradication milestones, and aligns with breach notification workflows — extending generic SOC metrics with GDPR-specific impact labels.

## Implementation

(1) Enrich notables with asset lookups that flag personal-data processing; (2) require analysts to set `status_description` with PII impact keywords when known; (3) alert DPO workflow when `CONFIRMED_PII`; (4) correlate with UC-22.1.3 for 72h authority notification clocks; (5) retain closed-incident exports for accountability.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263).

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enrich notables with asset lookups that flag personal-data processing; (2) require analysts to set `status_description` with PII impact keywords when known; (3) alert DPO workflow when `CONFIRMED_PII`; (4) correlate with UC-22.1.3 for 72h authority notification clocks; (5) retain closed-incident exports for accountability.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
`notable` earliest=-14d
| lookup gdpr_system_access_matrix.csv system_hostname AS dest OUTPUT data_classification
| eval gdpr_impact=case(
    isnotnull(data_classification) AND match(lower(status_description),"(?i)pii|personal|breach|exfil"), "CONFIRMED_PII",
    isnotnull(data_classification), "POTENTIAL_PII_SYSTEM",
    1=1, "UNKNOWN")
| stats latest(status) as status latest(owner) as owner max(_time) as last_event by rule_name, gdpr_impact
| where gdpr_impact IN ("CONFIRMED_PII","POTENTIAL_PII_SYSTEM")
| table rule_name, gdpr_impact, status, owner, last_event
```
