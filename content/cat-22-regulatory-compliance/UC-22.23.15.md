# UC-22.23.15 · Actively Exploited Vulnerability Notification Window (24h evidence)

## Description

Supports CRA product security, vulnerability handling, and transparency obligations with auditable telemetry from engineering and incident processes.

## Value

Gives the responsible team auditor-ready evidence for EU CRA compliance by automating regulatory compliance evidence using indexes product_security and sourcetypes sec:incident:notify. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review. Indexes required: `index=product_security`; Sourcetypes: `sourcetype=sec:incident:notify`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for GitHub (5765).
• Ensure the following data sources are available: `index=product_security` `sec:incident:notify` — product_id, release_id, cve_id, status, result, enisa_ticket_id.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=product_security sourcetype="sec:incident:notify" actively_exploited=true earliest=-30d
| eval cra_tag="active24"
| eval uc_id="22.23.15"
| stats earliest(_time) as t_detect, latest(notification_sent_epoch) as t_sent by product_id, cve_id
| eval hours_to_notify=if(isnotnull(t_sent), round((t_sent-t_detect)/3600,2), null())
| where isnull(t_sent) OR hours_to_notify>24
| sort - hours_to_notify
| table uc_id, product_id, cve_id, t_detect, t_sent, hours_to_notify
```
