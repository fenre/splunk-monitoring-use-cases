# UC-22.10.31 · Facility vs Logical Access — Badge-In Without VPN/SSO for Remote Roles

## Description

Correlates physical badge events with absence of corresponding VPN or SSO session for users tagged as onsite-only, highlighting impossible travel or credential sharing.

## Value

Correlates physical badge events with absence of corresponding VPN or SSO session for users tagged as onsite-only, highlighting impossible travel or credential sharing.

## Implementation

(1) Normalize time zones for multi-campus; (2) tune for vendor remote workers; (3) integrate HR work location attribute.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), HTTP Event Collector (HEC).
• Ensure the following data sources are available: `index=pacs` or `index=physical` `sourcetype=genetec:access` (`badge_id`, `reader`, `result`), `index=vpn` GlobalProtect, `lookup badge_to_user.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize time zones for multi-campus; (2) tune for vendor remote workers; (3) integrate HR work location attribute.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype="genetec:access" result="Granted" earliest=-1d
| lookup badge_to_user.csv badge_id OUTPUT samaccountname, work_mode
| where work_mode="OnsiteOnly"
| eval day=strftime(_time,"%Y-%m-%d")
| join type=left samaccountname, day [
    search index=vpn sourcetype="paloalto:globalprotect" earliest=-1d
    | eval day=strftime(_time,"%Y-%m-%d")
    | stats earliest(_time) as vpn_first by src_user, day
    | rename src_user as samaccountname ]
| where isnull(vpn_first)
| stats count by samaccountname, reader, day
```
