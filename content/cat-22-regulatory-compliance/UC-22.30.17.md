# 22.30.17 — CERT-In incident reporting tracking

**Criticality:** high | **Difficulty:** beginner | **Status:** verified

## Description

Supports RBI cyber security framework supervisory examination readiness for cert-in incident reporting tracking.

## Value

Gives the responsible team auditor-ready evidence for RBI Cyber compliance by automating regulatory compliance evidence using indexes grc and sourcetypes cert_in:report. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=grc sourcetype="cert_in:report" earliest=-30d
| where isnull(submitted_at) AND severity IN ("High","Critical")
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=grc`; Sourcetypes: `sourcetype=cert_in:report`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones).

## Compliance

- **RBI Cyber** 2016 (as amended): Annex-B (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Annex-B.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
