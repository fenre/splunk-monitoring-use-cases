# UC-22.11.78 · Segmentation Control Validation — Scanner IP Blocked at CDE Border as Expected

## Description

Confirms that active segmentation tests from non-CDE scanner IPs are denied at the NSC while still logged, producing evidence that untrusted networks cannot reach CHD without passing controls.

## Value

Confirms that active segmentation tests from non-CDE scanner IPs are denied at the NSC while still logged, producing evidence that untrusted networks cannot reach CHD without passing controls.

## Implementation

(1) Coordinate scanner IP list with annual segmentation test vendor. (2) Alert if `deny_ratio` drops (misconfiguration). (3) Attach results to PCI evidence library. (4) Run after major firewall pushes.

## Detailed Implementation

• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757).

Step 1 — Configure data collection
(1) Coordinate scanner IP list with annual segmentation test vendor. (2) Alert if `deny_ratio` drops (misconfiguration). (3) Attach results to PCI evidence library. (4) Run after major firewall pushes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=netfw sourcetype="pan:traffic" earliest=-24h
| lookup pci_segmentation_scanner_ips.csv ip AS src OUTPUT is_scanner
| lookup pci_cde_subnets.csv subnet AS dest OUTPUT in_cde
| where is_scanner="true" AND in_cde="true"
| stats count by action, rule, src, dest
| eval expected_deny=if(action IN ("deny","drop","reset-client"),1,0)
| stats sum(expected_deny) as deny_hits, count as total by src
| eval deny_ratio=if(total>0, round(100*deny_hits/total,2), null())
| where deny_ratio<95
| sort src
```
