# UC-22.7.16 · Hardware Asset Coverage — Agents Missing on In-Scope Servers

## Description

Shows servers in CMDB without forwarder heartbeat so asset management inventories are complete for CSF Identify.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes _internal. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align host naming; (2) weekly diff to CMDB owners; (3) track closure in ID.AM improvement backlog. Indexes required: `index=_internal`; Lookups: `cmdb_servers`, `cmdb_servers.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Universal Forwarder `_internal`, CMDB lookup.
• Ensure the following data sources are available: `_internal` metrics; `cmdb_servers.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align host naming; (2) weekly diff to CMDB owners; (3) track closure in ID.AM improvement backlog.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| metadata type=hosts index=*
| lookup cmdb_servers.csv host OUTPUT in_scope
| where in_scope="true"
| join type=left host [| search index=_internal source=*metrics.log* group=tcpin_connections earliest=-24h | stats dc(sourceHost) as reporting by sourceHost | rename sourceHost as host]
| where isnull(reporting)
```
