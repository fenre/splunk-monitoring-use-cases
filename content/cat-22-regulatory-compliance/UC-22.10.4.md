# 22.10.4 — Workforce Clearance — HR Hire vs AD Account Creation

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Flags Active Directory accounts created or enabled before HR clearance effective date, supporting workforce clearance procedures for roles with ePHI access.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes hr, windows and sourcetypes workday:worker, workday:worker`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
(index=windows EventCode IN ("4720","4722") TargetUserName="*") OR (index=hr sourcetype="workday:worker")
| eval emp_id=coalesce(employee_id, TargetUserName)
| eval event_type=if(EventCode="4720","AccountCreated",if(EventCode="4722","AccountEnabled","HRRecord"))
| stats earliest(_time) as first_ad latest(_time) as last_hr by emp_id, event_type
| join type=outer emp_id [
    search index=hr sourcetype="workday:worker" earliest=-180d
    | eval clearance_epoch=strptime(effective_date,"%Y-%m-%d")
    | table employee_id, clearance_status, clearance_epoch
    | rename employee_id as emp_id ]
| eval violation=if(first_ad < clearance_epoch AND clearance_status="Cleared","PreClearance_AD",null())
| where isnotnull(violation)
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Normalize timestamps to UTC; (2) map service accounts to exclusion lookup; (3) route violations to IAM and HR; (4) quarterly attestation report. Indexes required: `index=hr`, `index=windows`; Sourcetypes: `sourcetype=workday:worker`, `sourcetype=workday:worker``.

## Visualization

Table (account, violation, timestamps), Single value (violations 30d).

## Compliance

- **HIPAA Security** 2013-final: §164.308(a)(3) (partial) — Uplifted from contributing: SPL directly addresses §164.308(a)(3) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
