# 22.32.23 — Advanced authentication for CJI sessions

**Criticality:** low | **Difficulty:** intermediate | **Status:** N/A

## Description

Supports CJIS Security Policy compliance evidence for advanced authentication for cji sessions in Splunk dashboards and scheduled reports.

## Value

Gives the responsible team auditor-ready evidence for CJIS compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode IN (4624,4648) earliest=-24h
| lookup cjis_workstation.csv host OUTPUT cjis_flag
| where cjis_flag="true"
| eval mfa=if(match(AuthenticationPackage,"Negotiate"),1,0)
| where mfa=0
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Maintain `amer_authorization_boundary.csv` with ATO, CUI, and CJIS scope flags. (2) Integrate POA&M exports via scheduled output and lookup refresh for ISSM workflows. (3) Separate LGPD and federal datasets with index- and role-level controls. (4) Quarterly control mapping workshop with ISSM, ISO, and privacy counsel. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`.
