# UC-22.14.59 · User-Installed Software Detections on Corporate Images

## Description

Lists unauthorized software packages reported by inventory agents for CM-11 user-installed software.

## Value

Lists unauthorized software packages reported by inventory agents for CM-11 user-installed software.

## Implementation

(1) Curate catalog with security exceptions; (2) version-aware matching; (3) self-service remediation tickets; (4) integrate with software blocklist.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Microsoft SCCM / Intune logs via HEC, Jamf logs.
• Ensure the following data sources are available: `index=endpoint` `sourcetype=intune:device_inventory`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Curate catalog with security exceptions; (2) version-aware matching; (3) self-service remediation tickets; (4) integrate with software blocklist.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=endpoint sourcetype=intune:device_inventory earliest=-7d
| search publisher!="Microsoft Corporation"
| lookup approved_software_catalog.csv name OUTPUT allowed
| where allowed!="true" OR isnull(allowed)
```
