# 22.20.15 — CMMC Level 3 enhanced practice — threat scenario 5

**Criticality:** low | **Difficulty:** advanced | **Status:** N/A

## Description

Demonstrates contractor cybersecurity hygiene for "CMMC Level 3 enhanced practice — threat scenario 5" using CUI-scoped telemetry mapped to CMMC 2.0 Level 3 expectations.

## Value

Gives the responsible team auditor-ready evidence for CMMC compliance by automating regulatory compliance evidence using indexes vuln and sourcetypes tenable:was. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=vuln sourcetype=tenable:was earliest=-14d
| eval uc_tag="22.20.15"
| where scan_name="CUI-Web" AND risk="High"
| stats dc(url) as urls by plugin_id
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.process_name, Processes.user, Processes.dest | sort - count
```

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=vuln`.

## Visualization

Stacked bar, Table, Time chart, Single value. Key fields: `plugin_id`, `uc_tag`, `urls`.

## Compliance

- **CMMC** 2.0: SI.L2-3.14.6 (contributing) — Vulnerability scanning on CUI systems supports attack surface monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
