# UC-22.11.3 · CDE Boundary Traffic — Unexpected Corporate-to-Payment Flows

## Description

Highlights allowed sessions from general corporate segments into payment subnets so segmentation and NSC rules can be validated between assessments, not only during annual penetration tests.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes netfw and sourcetypes pan:traffic. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Populate CIDR lookups from network diagrams approved for PCI. (2) Whitelist jump-host rules via `where NOT match(rule,"(?i)bastion|jump")`. (3) Feed hits to the segmentation validation runbook. (4) Map findings to Req 11 segmentation testing follow-up. Indexes required: `index=netfw`; Sourcetypes: `sourcetype=pan:traffic`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757).
• Ensure the following data sources are available: `index=netfw` `sourcetype="pan:traffic"` — `src`, `dest`, `src_zone`, `dest_zone`, `action`, `rule`; lookups `pci_corp_zones.csv`, `pci_payment_subnets.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate CIDR lookups from network diagrams approved for PCI. (2) Whitelist jump-host rules via `where NOT match(rule,"(?i)bastion|jump")`. (3) Feed hits to the segmentation validation runbook. (4) Map findings to Req 11 segmentation testing follow-up.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=netfw sourcetype="pan:traffic" action=allowed earliest=-24h
| lookup pci_corp_zones.csv zone AS src_zone OUTPUT corp_zone
| lookup pci_payment_subnets.csv cidr AS dest OUTPUT payment_tier
| where corp_zone="true" AND payment_tier="card_processing"
| stats count, values(dest_port) as ports, values(app) as apps by src, dest, rule
| where count>5
| sort - count
```
