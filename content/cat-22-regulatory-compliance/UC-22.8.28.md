# UC-22.8.28 · SOC 2 A1 — Disaster Recovery Test Execution and Evidence Timestamps

## Description

A1 expects recovery commitments to be tested. Indexing DR runbook executions, RTO/RPO measurements, and failover test results with immutable timestamps proves tests occurred at the required frequency and outcomes were recorded.

## Value

Gives the responsible team auditor-ready evidence for SOC 2 compliance by automating regulatory compliance evidence using indexes resilience and sourcetypes dr:test_result. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Require one test per in-scope region_pair per quarter minimum—tune to policy; (2) integrate orchestration tool webhooks; (3) store raw logs in WORM; (4) failed `pass_fail` opens severity-2 problem; (5) map to customer SLA annex. Indexes required: `index=resilience`; Sourcetypes: `sourcetype=dr:test_result`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector, Splunk Add-on for ServiceNow (Splunkbase 1928).
• Ensure the following data sources are available: `index=resilience` `sourcetype="dr:test_result"` (test_id, region_pair, rto_min, rpo_min, pass_fail, executed_by, _time); optional `snow:cmdb_ci` for scope.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require one test per in-scope region_pair per quarter minimum—tune to policy; (2) integrate orchestration tool webhooks; (3) store raw logs in WORM; (4) failed `pass_fail` opens severity-2 problem; (5) map to customer SLA annex.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=resilience sourcetype="dr:test_result" earliest=-400d
| bin _time span=90d as quarter
| stats latest(pass_fail) as outcome, latest(rto_min) as rto, latest(rpo_min) as rpo by quarter, region_pair
| eval ok=if(outcome="pass",1,0)
| eventstats sum(ok) as passes, count as quarters by region_pair
| where passes<4
| table region_pair, quarter, outcome, rto, rpo
```

**Pipeline walkthrough**
