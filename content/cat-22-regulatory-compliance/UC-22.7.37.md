# UC-22.7.37 · Anomaly on Outbound Bytes from Database Subnet

## Description

Uses statistical anomaly detection on east-west bytes for adverse event triage.

## Value

Uses statistical anomaly detection on east-west bytes for adverse event triage.

## Implementation

(1) Label DB VLANs; (2) suppress backup windows; (3) integrate with IR runbook.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Zeek `conn`.
• Ensure the following data sources are available: `zeek:conn`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Label DB VLANs; (2) suppress backup windows; (3) integrate with IR runbook.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype=zeek:conn id.orig_h=10.50.* earliest=-7d
| timechart span=1h sum(orig_bytes) as b
| anomalydetection b action=annotate
```
