# UC-22.21.9 · Feature Importance Audit Snapshot Correlation (Art. 13)

## Description

Correlates SHAP export job completion with model_version to retain feature importance audit snapshots for transparency reviews.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_serving, ml_explain and sourcetypes ai:gateway:access, ai:shap:export, ai:shap:export`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Schedule SHAP export on each release; (2) store artifact_uri in object storage with WORM; (3) alert if model_version in prod without export in 7d; (4) document in technical file. Indexes required: `index=ai_serving`, `index=ml_explain`; Sourcetypes: `sourcetype=ai:gateway:access`, `sourcetype=ai:shap:export`, `sourcetype=ai:shap:export``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Machine Learning Toolkit (2890), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: `index=ml_explain` `sourcetype=ai:shap:export` — job_id, model_version, artifact_uri.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Schedule SHAP export on each release; (2) store artifact_uri in object storage with WORM; (3) alert if model_version in prod without export in 7d; (4) document in technical file.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ml_explain sourcetype="ai:shap:export" earliest=-30d
| stats latest(_time) as last_export by model_version
| join type=left model_version [
    search index=ai_serving sourcetype="ai:gateway:access" earliest=-30d
    | stats dc(model_version) as versions_seen by model_version
  ]
| eval stale=if(isnull(last_export) OR (now()-last_export)>2592000,1,0)
| where stale=1
| table model_version, last_export, versions_seen
```
