# UC-22.15.31 · Cleartext Industrial Credentials in PCAP-Derived Metadata

## Description

Scans IDS metadata for username or password fields in cleartext industrial protocols where confidentiality must be preserved.

## Value

Scans IDS metadata for username or password fields in cleartext industrial protocols where confidentiality must be preserved.

## Implementation

(1) Never index full payload in prod—use IDS summaries; (2) rotate exposed creds; (3) enable TLS where vendor supports; (4) segment sniffers.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Claroty xDome TA, Nozomi TA.
• Ensure the following data sources are available: `claroty:alert` or `nozomi:packets_meta` (example sourcetype).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Never index full payload in prod—use IDS summaries; (2) rotate exposed creds; (3) enable TLS where vendor supports; (4) segment sniffers.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_sec (sourcetype="claroty:alert" OR sourcetype="nozomi:alerts") earliest=-7d
| where match(_raw,"(?i)password=|passwd=|user=.*pass") AND NOT match(_raw,"(?i)\\*\\*\\*")
| rex field=_raw "(?i)(?<proto>modbus|opcua|dnp3).*"
| stats count by proto, src_ip, dest_ip, signature
| sort - count
```
