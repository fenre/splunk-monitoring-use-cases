# 22.15.31 — Cleartext Industrial Credentials in PCAP-Derived Metadata

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Scans IDS metadata for username or password fields in cleartext industrial protocols where confidentiality must be preserved.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_sec and sourcetypes claroty:alert, nozomi:alerts. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_sec (sourcetype="claroty:alert" OR sourcetype="nozomi:alerts") earliest=-7d
| where match(_raw,"(?i)password=|passwd=|user=.*pass") AND NOT match(_raw,"(?i)\\*\\*\\*")
| rex field=_raw "(?i)(?<proto>modbus|opcua|dnp3).*"
| stats count by proto, src_ip, dest_ip, signature
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Intrusion_Detection.IDS_Attacks by IDS_Attacks.src, IDS_Attacks.dest, IDS_Attacks.signature | sort - count
```

## Implementation

(1) Never index full payload in prod—use IDS summaries; (2) rotate exposed creds; (3) enable TLS where vendor supports; (4) segment sniffers. Indexes required: `index=ot_sec`; Sourcetypes: `sourcetype=claroty:alert`, `sourcetype=nozomi:alerts`.
