# UC-22.6.19 · Disciplinary Process Triggers — HR Case Codes Correlated with Security Incidents

## Description

Correlates anonymized HR case creation timestamps with security investigations to evidence disciplinary process alignment without exposing PII in dashboards.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes hr_restricted, itsm and sourcetypes snow:hr_case, snow:incident. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Strict RBAC on index; (2) legal privacy review; (3) aggregate only; (4) document data minimization. Indexes required: `index=hr_restricted`, `index=itsm`; Sourcetypes: `sourcetype=snow:hr_case`, `sourcetype=snow:incident`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: ServiceNow HR case HEC (restricted index).
• Ensure the following data sources are available: `snow:hr_case` (restricted).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Strict RBAC on index; (2) legal privacy review; (3) aggregate only; (4) document data minimization.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr_restricted sourcetype=snow:hr_case earliest=-180d category=conduct
| eval day=round(_time/86400,0)
| join type=inner day [| search index=itsm sourcetype=snow:incident category=security earliest=-180d | eval day=round(_time/86400,0) | stats dc(number) as sec_cases by day]
| stats count by day
```
