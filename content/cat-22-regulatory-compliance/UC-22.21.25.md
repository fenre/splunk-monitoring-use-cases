# UC-22.21.25 · Regulatory Authority Notification Delivery Confirmation (Art. 73)

## Description

Confirms secure notification channel message_id acknowledged by authority mailbox/API within policy window.

## Value

Confirms secure notification channel message_id acknowledged by authority mailbox/API within policy window.

## Implementation

(1) Tag regulatory messages with dedicated header; (2) alert on bounce; (3) retain DKIM pass evidence; (4) legal owns recipient allowlist.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Office 365 (4055) or generic mail gateway logs.
• Ensure the following data sources are available: `index=mail` `sourcetype=o365:messaging` OR `sourcetype=sendgrid:event` — message_id, event, recipient_domain.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag regulatory messages with dedicated header; (2) alert on bounce; (3) retain DKIM pass evidence; (4) legal owns recipient allowlist.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mail (sourcetype="o365:messaging" OR sourcetype="sendgrid:event") earliest=-30d recipient_domain="*authority*"
| stats earliest(eval(if(event="queued",_time,null()))) as t_q,
        earliest(eval(if(event="delivered",_time,null()))) as t_d,
        earliest(eval(if(match(event,"bounce|dropped"),_time,null()))) as t_fail
    by message_id
| eval deliver_h=round((t_d-t_q)/3600,3)
| where isnull(t_d) OR isnotnull(t_fail)
| table message_id, t_q, t_d, t_fail, deliver_h
```
