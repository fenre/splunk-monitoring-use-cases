# UC-22.11.26 · Wireless Link Encryption Downgrade for Store WLAN Carrying Payment Terminals

## Description

Surfaces association frames or controller logs showing legacy WPA or open auth on SSIDs used by payment terminals, evidencing review of wireless encryption for account data environments.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes wireless and sourcetypes wlan:controller. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Classify SSIDs that can see payment devices. (2) Exclude guest SSIDs with captive portal if physically isolated. (3) Integrate with store opening checklist. (4) Weekly store ops digest. Indexes required: `index=wireless`; Sourcetypes: `sourcetype=wlan:controller`; Lookups: `pci_store_ssid`, `pci_store_ssid.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wireless sourcetype="wlan:controller" earliest=-24h
| lookup pci_store_ssid.csv ssid OUTPUT carries_pos
| where carries_pos="true"
| where match(_raw,"(?i)open|OWE|WEP|WPA[^2]|TKIP")
| stats count by ap_name, ssid, security_mode
| sort - count
```
