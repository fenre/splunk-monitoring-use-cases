# 22.32.6 — Privacy impact assessment (RIPD) tracking

**Criticality:** medium | **Difficulty:** advanced | **Status:** N/A

## Description

Supports Lei Geral de Proteção de Dados (LGPD) compliance evidence for privacy impact assessment (ripd) tracking in Splunk dashboards and scheduled reports.

## Value

Gives the responsible team auditor-ready evidence for LGPD compliance by automating regulatory compliance evidence using indexes privacy and sourcetypes ropa:line. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=privacy sourcetype="ropa:line" earliest=-730d
| stats dc(processing_purpose) as p by system_id
| where p=0
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - count
```

## Implementation

(1) Maintain `amer_authorization_boundary.csv` with ATO, CUI, and CJIS scope flags. (2) Integrate POA&M exports via scheduled output and lookup refresh for ISSM workflows. (3) Separate LGPD and federal datasets with index- and role-level controls. (4) Quarterly control mapping workshop with ISSM, ISO, and privacy counsel. Indexes required: `index=privacy`; Sourcetypes: `sourcetype=ropa:line`.

## Visualization

Stacked bar (control family), Table (users and events), Single value (open findings).

## Compliance

- **LGPD** Lei nº 13.709/2018: Art.46 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.46.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
