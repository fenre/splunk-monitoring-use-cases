# UC-22.11.59 · POS Terminal Tamper and Intrusion Switch Alerts

## Description

Centralizes tamper switches, lid open, and unexpected reboot signals from retail payment terminals so physical device integrity controls for card-present environments are actively monitored.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes pos and sourcetypes pos:device_health. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map terminal IDs to merchant locations. (2) Correlate with technician dispatch tickets. (3) Page on `severity=CRITICAL`. (4) Monthly store tamper rollup for PCI onsite interviews. Indexes required: `index=pos`; Sourcetypes: `sourcetype=pos:device_health`; Lookups: `pci_store_terminals`, `pci_store_terminals.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Custom POS syslog `sourcetype="verifone:vxp"` or `ingenico:telium"` HEC.
• Ensure the following data sources are available: `index=pos` `sourcetype="pos:device_health"` — `terminal_id`, `alert_type`, `severity`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map terminal IDs to merchant locations. (2) Correlate with technician dispatch tickets. (3) Page on `severity=CRITICAL`. (4) Monthly store tamper rollup for PCI onsite interviews.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=pos sourcetype="pos:device_health" earliest=-7d
| where match(alert_type,"(?i)tamper|lid|intrusion|case|seal|debug_port")
| lookup pci_store_terminals.csv terminal_id OUTPUT store_id
| stats earliest(_time) as first_alert, latest(_time) as last_alert, values(alert_type) as types by terminal_id, store_id
| sort first_alert
```
