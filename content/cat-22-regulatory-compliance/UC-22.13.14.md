# UC-22.13.14 · Background Investigation Recency for ESP and PAM Users

## Description

We flag active PAM users whose background investigation date exceeds policy limits so personnel suitability evidence stays current.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes hr, pam and sourcetypes cyberark:ptalog, hr:background`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Replace `2555` with your policy threshold lookup. (2) Weekly sync from HR. (3) Suspend access per procedure when triggered. (4) Document waivers in CIP-003 exceptions. Indexes required: `index=hr`, `index=pam`; Sourcetypes: `sourcetype=cyberark:ptalog`, `sourcetype=hr:background``.

Step 1 — Configure data collection
(1) Replace `2555` with your policy threshold lookup. (2) Weekly sync from HR. (3) Suspend access per procedure when triggered. (4) Document waivers in CIP-003 exceptions.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup esp_privileged_personnel.csv
| lookup hr_background employee_id OUTPUT investigation_date
| eval inv_epoch=strptime(investigation_date,"%Y-%m-%d")
| eval age_days=round((now()-inv_epoch)/86400,0)
| join type=left employee_id [
    search index=pam sourcetype=cyberark:ptalog action="Logon" earliest=-30d
    | stats latest(_time) as last_pam by user
    | rename user as employee_id
  ]
| where age_days>2555 AND isnotnull(last_pam)
| table employee_id, investigation_date, age_days, last_pam
```
