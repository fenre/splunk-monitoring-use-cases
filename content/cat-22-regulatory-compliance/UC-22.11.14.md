# UC-22.11.14 · Primary Account Number Pattern Discovery in Application Indexes

## Description

Identifies log fields or raw text that resemble payment card numbers in non-tokenization indexes so PAN can be purged or masked before it violates protection of stored account data requirements.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes app, web and sourcetypes java:log4j, nginx:access, payment:api:json. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Run in a restricted role; mask outputs in dashboards. (2) Validate with Luhn in a external script if false positives are high. (3) Route hits to DLP response. (4) Never store full PAN in summary indexes—store counts only. (5) Document sampling methodology for QSA. Indexes required: `index=app`, `index=web`; Sourcetypes: `sourcetype=java:log4j`, `sourcetype=nginx:access`, `sourcetype=payment:api:json`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: `index=app` OR `index=web` high-volume text; restrict to CDE-related `sourcetype` list.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Run in a restricted role; mask outputs in dashboards. (2) Validate with Luhn in a external script if false positives are high. (3) Route hits to DLP response. (4) Never store full PAN in summary indexes—store counts only. (5) Document sampling methodology for QSA.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
(index=app OR index=web) sourcetype IN ("payment:api:json","java:log4j","nginx:access") earliest=-24h
| regex _raw="\\b(?:4[0-9]{12}(?:[0-9]{3})?|5[1-5][0-9]{14}|3[47][0-9]{13}|6(?:011|5[0-9]{2})[0-9]{12})\\b"
| eval pan_like=replace(_raw,".*?(\\b(?:4[0-9]{12}(?:[0-9]{3})?|5[1-5][0-9]{14})\\b).*","\\1")
| eval pan_redacted=replace(pan_like,"(\\d{6})\\d+(\\d{4})","\\1******\\2")
| stats count by index, sourcetype, host
| sort - count
```
