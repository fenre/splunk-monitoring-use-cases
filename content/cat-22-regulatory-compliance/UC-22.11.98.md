# 22.11.98 — PCI-DSS 8.6 — Application and system accounts: interactive use of a service account

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Detects an interactive session on a service-only account — direct PCI-DSS 8.6 violation, which requires application/system accounts not be used interactively.

## Value

Removes the common post-audit-finding 'we didn't know that account was used interactively' narrative by making the misuse visible in 10 minutes.

## SPL

```spl
index=windows sourcetype=WinEventLog:Security EventCode=4624 LogonType IN (2,10,11) earliest=-10m
| rename TargetUserName AS user
| lookup service_accounts.csv user OUTPUT account_type
| where account_type="service-only"
| table user src dest LogonType account_type
```

## Implementation

(1) Maintain service_accounts.csv with account_type tag; (2) schedule UC every 10m; (3) hit opens a P1 and disables the account; (4) weekly IAM review. Indexes required: `index=linux`, `index=windows`; Sourcetypes: `sourcetype=WinEventLog:Security`.

## Visualization

Table of interactive service-account logins, time chart, single value 'events in last hour'. Key fields: `LogonType`, `account_type`, `dest`, `src`, `user`.

## Compliance

- **PCI-DSS** v4.0: 8.6 (full) — PCI-DSS 8.6 Application and system accounts not used interactively — directly evidenced.
- **ISO/IEC 27001** 2022: A.5.16 (full) — ISO/IEC 27001:2022 A.5.16 Identity management — directly evidenced.
- **NIST 800-53** Rev. 5: IA-2 (full) — Uplifted to full: SPL provides comprehensive, standalone evidence for IA-2 with specific data sources and actionable thresholds.

## Known False Positives

Admin password rotations performed by a service-account-using tool may be reported as interactive — exclude via tool's user-as-tool mapping.
