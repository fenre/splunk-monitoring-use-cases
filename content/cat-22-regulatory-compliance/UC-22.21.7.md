# UC-22.21.7 · Model Version Tracking and Immutable Version IDs (Art. 13)

## Description

Detects production traffic served from non-semver or missing model_version to support traceability and change control evidence.

## Value

Detects production traffic served from non-semver or missing model_version to support traceability and change control evidence.

## Implementation

(1) Enforce semver in model registry webhook; (2) block deploy if version missing; (3) alert on null model_version > 10/min; (4) map deployment to ITSI entity.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI (1841), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: `index=ai_serving` — model_version, deployment, traffic_pct.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enforce semver in model registry webhook; (2) block deploy if version missing; (3) alert on null model_version > 10/min; (4) map deployment to ITSI entity.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ai_serving sourcetype="ai:gateway:access" earliest=-4h
| eval semver_ok=if(match(model_version,"^v?\d+\.\d+\.\d+"),1,0)
| stats count as reqs, sum(semver_ok) as semver_hits by deployment, model_version
| eval semver_pct=round(100*semver_hits/reqs,2)
| where isnull(model_version) OR model_version="" OR semver_ok=0
| sort - reqs
```
