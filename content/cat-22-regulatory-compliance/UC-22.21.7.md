# 22.21.7 — Model Version Tracking and Immutable Version IDs (Art. 13)

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Detects production traffic served from non-semver or missing model_version to support traceability and change control evidence.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_serving and sourcetypes ai:gateway:access. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_serving sourcetype="ai:gateway:access" earliest=-4h
| eval semver_ok=if(match(model_version,"^v?\d+\.\d+\.\d+"),1,0)
| stats count as reqs, sum(semver_ok) as semver_hits by deployment, model_version
| eval semver_pct=round(100*semver_hits/reqs,2)
| where isnull(model_version) OR model_version="" OR semver_ok=0
| sort - reqs
```

## Implementation

(1) Enforce semver in model registry webhook; (2) block deploy if version missing; (3) alert on null model_version > 10/min; (4) map deployment to ITSI entity. Indexes required: `index=ai_serving`; Sourcetypes: `sourcetype=ai:gateway:access`.

## Visualization

Pie chart (semver_ok), Table (bad versions), Column chart (reqs by deployment)

## Compliance

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
