# 22.50.27 — COBIT DSS05.03 — Endpoint security management: endpoint protection coverage and compliance

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Monitors endpoint protection agent status, definition currency, and protection coverage across the estate per COBIT DSS05.03 requirements for endpoint security management.

## Value

DSS05.03 requires managing endpoint security throughout the lifecycle. Continuous coverage monitoring detects protection gaps before they become exploitable.

## SPL

```spl
index=endpoint_protection sourcetype IN (epp_status, edr_health) earliest=-24h
| stats latest(agent_version) AS agent_version latest(definitions_date) AS defs_date latest(protection_status) AS protection_status latest(last_scan) AS last_scan BY hostname, os_type
| lookup asset_inventory hostname OUTPUT business_unit, criticality
| eval defs_age_days=round((now()-strptime(defs_date, "%Y-%m-%d"))/86400)
| eval compliant=if(protection_status="active" AND defs_age_days<=3, 1, 0)
| stats count AS total_endpoints sum(compliant) AS compliant_endpoints BY business_unit
| eval coverage_pct=round(100*compliant_endpoints/total_endpoints, 1)
| where coverage_pct<100
| table business_unit, total_endpoints, compliant_endpoints, coverage_pct
```

## Implementation

(1) Ingest EPP/EDR status logs with agent_version, definitions_date, protection_status; (2) maintain asset_inventory lookup; (3) schedule daily; (4) escalate gaps to SOC. Indexes required: `index=endpoint_protection`; Sourcetypes: `sourcetype=edr_health`, `sourcetype=epp_status`.

## Visualization

Table of flagged records for investigation; single-value KPI of coverage / compliance percentage; time chart of trend over the last 90 days; drill-down per responsible owner. Key fields: `agent_version`, `business_unit`, `compliant`, `compliant_endpoints`, `coverage_pct`, `defs_age_days`.

## Compliance

- **COBIT** 2019: DSS05.03 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for DSS05.03.

## Known False Positives

Tuning prerequisites: source field mappings must be in place before the UC will produce reliable metrics; early runs will commonly flag unmapped systems until onboarding is completed.
