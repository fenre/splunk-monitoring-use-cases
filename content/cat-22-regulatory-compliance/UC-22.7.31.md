# UC-22.7.31 · EDR Heartbeat Gap Beyond Policy SLA

## Description

Detects endpoints without EDR telemetry within SLA window for continuous monitoring coverage.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes edr and sourcetypes crowdstrike:hosts. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align with policy hours; (2) auto-ticket stale agents; (3) map to CMDB ownership. Indexes required: `index=edr`; Sourcetypes: `sourcetype=crowdstrike:hosts`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: CrowdStrike or Defender HEC.
• Ensure the following data sources are available: `crowdstrike:hosts`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align with policy hours; (2) auto-ticket stale agents; (3) map to CMDB ownership.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=edr sourcetype=crowdstrike:hosts earliest=-1d
| eval last_seen_epoch=strptime(last_seen,"%Y-%m-%dT%H:%M:%SZ")
| eval gap_hours=(now()-last_seen_epoch)/3600
| where gap_hours>24
```
