# UC-22.7.50 · Status Page Update Cadence During Major Incident

## Description

Ingests statuspage.io updates to prove recovery communications cadence met customer commitments.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes comms and sourcetypes statuspage:incident. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map to customer SLAs; (2) training for incident comms role; (3) post-mortem metric. Indexes required: `index=comms`; Sourcetypes: `sourcetype=statuspage:incident`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HEC `_json`.
• Ensure the following data sources are available: `statuspage:incident`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map to customer SLAs; (2) training for incident comms role; (3) post-mortem metric.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=comms sourcetype=statuspage:incident earliest=-180d
| stats count as updates, earliest(_time) as start by incident_id
| eval duration_h=(now()-start)/3600
| where duration_h>1 AND updates<2
```
