# 22.13.8 — Low-Impact BES Cyber System Electronic Access Path Enforcement

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We catch remote access sessions to low-impact assets that bypass approved intermediaries or applications, reducing undocumented electronic access paths.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes network and sourcetypes pan:traffic, pan:traffic`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=network sourcetype=pan:traffic action=allowed earliest=-24h
| lookup bes_low_impact_hosts.csv ip AS dest OUTPUT asset_id impact
| where impact="low"
| lookup cip003_approved_remote_paths.csv app src_zone dest_zone OUTPUT approved
| where isnull(approved)
| stats sum(bytes_out) as bytes_out count by dest, app, rule, user
| sort - bytes_out
```

## CIM SPL

```spl
| tstats summariesonly=t sum(All_Traffic.bytes_out) as agg_value from datamodel=Network_Traffic.All_Traffic by All_Traffic.dest, All_Traffic.app | sort - agg_value
```

## Implementation

(1) Maintain low-impact asset IP list from CIP-002 outputs. (2) Define approved app and zone tuples. (3) Daily alert on violations. (4) Feed confirmed exceptions to the CIP-003 exception workflow. Indexes required: `index=network`; Sourcetypes: `sourcetype=pan:traffic`, `sourcetype=pan:traffic``; Lookups: `bes_low_impact_hosts`, `bes_low_impact_hosts.csv`, `cip003_approved_remote_paths`, `cip003_approved_remote_paths.csv`.
