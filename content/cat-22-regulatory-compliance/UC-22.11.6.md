# UC-22.11.6 · Wireless Client Pools Reaching CDE Hosts

## Description

Shows successful flows from wireless DHCP scopes to CDE destinations, supporting evidence that wireless is segmented from cardholder environments or that compensating detective controls exist.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes netfw, stream and sourcetypes pan:traffic. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Export DHCP superscopes per SSID from IPAM. (2) Exclude corporate VPN concentrator egress if pooled with WLAN. (3) Pair with 802.1X authentication logs when available. (4) Report zero-result runs as quarterly attestation attachment. Indexes required: `index=netfw`, `index=stream`; Sourcetypes: `sourcetype=pan:traffic`; Lookups: `pci_cde_subnets`, `pci_cde_subnets.csv`, `pci_wlan_dhcp_scopes`, `pci_wlan_dhcp_scopes.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757), Splunk Stream (1809).
• Ensure the following data sources are available: `index=netfw` `sourcetype="pan:traffic"`; optional `index=stream` for decrypted metadata tagging.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Export DHCP superscopes per SSID from IPAM. (2) Exclude corporate VPN concentrator egress if pooled with WLAN. (3) Pair with 802.1X authentication logs when available. (4) Report zero-result runs as quarterly attestation attachment.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=netfw sourcetype="pan:traffic" action=allowed earliest=-24h
| lookup pci_wlan_dhcp_scopes.csv subnet AS src OUTPUT wlan_ssid
| lookup pci_cde_subnets.csv subnet AS dest OUTPUT in_cde
| where isnotnull(wlan_ssid) AND in_cde="true"
| stats earliest(_time) as first_seen, latest(_time) as last_seen, count by wlan_ssid, src, dest, app
| eval session_span_sec=last_seen-first_seen
| sort - count
```
