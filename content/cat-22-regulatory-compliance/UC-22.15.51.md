# 22.15.51 — Engineering Workstation Access Control — RDP Skips Jump Tier

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Alerts when RDP/WinRM sessions skip the jump tier and land directly on low-Purdue controllers or Windows I/O servers.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes fw, windows and sourcetypes pan:traffic. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
(index=windows EventCode=4624 Logon_Type=10 host="PLC-WIN-IO-*") OR (index=fw sourcetype="pan:traffic" app="rdp" dest_zone="OT-L1")
| eval session_key=coalesce(src_ip, src)
| lookup ot_jump_required.csv dest_ip OUTPUT jump_required
| where jump_required="true"
| stats count by user, dest_ip, host
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.user, Authentication.dest | sort - count
```

## Implementation

(1) Disable RDP on controllers; (2) enforce PSM-only path; (3) GPO firewall; (4) weekly exception report. Indexes required: `index=fw`, `index=windows`; Sourcetypes: `sourcetype=pan:traffic`; Lookups: `ot_jump_required`, `ot_jump_required.csv`.

## Visualization

Table, Timeline, Single value (violations). Key fields: `dest_ip`, `host`, `session_key`, `user`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
