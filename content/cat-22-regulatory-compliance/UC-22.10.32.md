# UC-22.10.32 · Workstation Use — After-Hours Login from Non-Clinical IP Space

## Description

Flags Citrix/VDI launches for inpatient CPOE roles originating from residential ISP ASNs outside hospital guest WiFi, supporting workstation use policies in shared clinical environments.

## Value

Flags Citrix/VDI launches for inpatient CPOE roles originating from residential ISP ASNs outside hospital guest WiFi, supporting workstation use policies in shared clinical environments.

## Implementation

(1) Replace `Org` heuristic with ASN lookup table; (2) allow VPN concentrator ranges; (3) route to privacy for workforce policy review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Citrix (2757), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: `index=citrix` `sourcetype=citrix:session`, MaxMind `iplocation`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Replace `Org` heuristic with ASN lookup table; (2) allow VPN concentrator ranges; (3) route to privacy for workforce policy review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=citrix sourcetype="citrix:session" earliest=-7d
| eval hour=strftime(_time,"%H")
| where (hour<6 OR hour>22)
| iplocation client_ip
| lookup clinical_role_users.csv UserName OUTPUT role
| where role IN ("RN_INPATIENT","MD_INPATIENT") AND NOT match(Org,"*Health System*")
| stats count by UserName, client_ip, Country, City, Org
| sort - count
```
