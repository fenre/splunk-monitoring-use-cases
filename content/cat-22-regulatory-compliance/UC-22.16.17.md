# 22.16.17 — PLC mode changes during active OT incidents

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "PLC mode changes during active OT incidents".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes ot and sourcetypes rockwell:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot sourcetype=rockwell:audit earliest=-30d message="*Mode*Change*"
| eval uc_tag="22.16.17"
| lookup active_ir_incidents.csv start_time end_time OUTPUT incident_id
| where isnotnull(incident_id)
| stats count by incident_id, controller, user
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=ot`.
