# UC-22.3.28 · DORA TLPT — Retest and Control Effectiveness Verification Events

## Description

Retesting proves fixes actually work. Correlating retest pass events with original finding IDs closes the assurance loop expected in mature TLPT programs and avoids “fixed on paper” drift.

## Value

Retesting proves fixes actually work. Correlating retest pass events with original finding IDs closes the assurance loop expected in mature TLPT programs and avoids “fixed on paper” drift.

## Implementation

(1) Require `finding_id` foreign key; (2) block manual status=remediated without retest event; (3) store `evidence_url` out-of-band; (4) integrate with vulnerability scanners for layered proof; (5) auditor sample pack from this search.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype="tlpt:retest" earliest=-400d
| stats latest(result) as last_result, latest(executed_at) as last_exec by finding_id
| join type=left finding_id [
    search index=grc sourcetype="tlpt:finding" earliest=-400d
    | fields finding_id, status, severity
  ]
| where status="remediated" AND last_result!="pass"
| table finding_id, severity, status, last_result, last_exec
```
