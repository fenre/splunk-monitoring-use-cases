# UC-22.2.4 · NIS2 Business Continuity and Crisis Management Monitoring

## Description

Uses ITSI service health and KPI breach signals as live evidence that continuity targets (RTO/RPO expressed as service KPIs) are monitored during incidents and crises.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes itsi_summary. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Model each regulated NIS2 service in ITSI with KPIs tied to RTO/RPO (e.g. availability, transaction success, replication lag); (2) set severity thresholds so `severity_value>=3` aligns with crisis playbooks; (3) display on Glass Table / Service Analyzer for NOC/C-level crisis calls; (4) attach episode workflows for major incidents. Indexes required: `index=itsi_summary`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk IT Service Intelligence (Splunkbase 1841).
• Ensure the following data sources are available: `index=itsi_summary` (health_score, service_name, kpi_name, severity_value, severity_label, is_service_in_maintenance).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Model each regulated NIS2 service in ITSI with KPIs tied to RTO/RPO (e.g. availability, transaction success, replication lag); (2) set severity thresholds so `severity_value>=3` aligns with crisis playbooks; (3) display on Glass Table / Service Analyzer for NOC/C-level crisis calls; (4) attach episode workflows for major incidents.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsi_summary is_service_in_maintenance=0 earliest=-24h
| eval rto_rpo_risk=if(severity_value>=3 OR health_score<70, 1, 0)
| stats avg(health_score) as avg_health,
        count(eval(rto_rpo_risk=1)) as breach_events
    by service_name, kpi_name
| where breach_events>0 OR avg_health<85
| sort - breach_events
| table service_name, kpi_name, avg_health, breach_events
```
