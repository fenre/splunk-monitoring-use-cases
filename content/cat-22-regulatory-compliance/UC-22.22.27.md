# UC-22.22.27 · Customer Impact Assessment Coverage for Incidents

## Description

Supports incident governance, customer communications evidence, and supervisory reporting workflows associated with payment service continuity and fraud statistics.

## Value

Supports incident governance, customer communications evidence, and supervisory reporting workflows associated with payment service continuity and fraud statistics.

## Implementation

(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk ITSI (1841).
• Ensure the following data sources are available: `index=payments` `pay:incident:milestone` — incident_id, milestone, severity, impacted_psu_estimate.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=payments sourcetype="pay:incident:milestone" earliest=-90d
| eval inc_tag="cust"
| where milestone="impact_assessed"
| eval impact_missing=if(isnull(impacted_psu_estimate) OR impacted_psu_estimate<0,1,0)
| stats sum(impact_missing) as missing, dc(incident_id) as incidents by aspsp_id, severity
| eval uc_id="22.22.27"
| where missing>0
| sort - missing
| table uc_id, aspsp_id, severity, missing, incidents
```
