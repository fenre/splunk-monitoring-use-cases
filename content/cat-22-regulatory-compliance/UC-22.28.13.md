# UC-22.28.13 · Security concept documentation changes

## Description

Demonstrates alignment with BSI IT-Grundschutz expectations for security concept documentation changes using auditable Splunk evidence.

## Value

Gives the responsible team auditor-ready evidence for IT-Grundschutz compliance by automating regulatory compliance evidence using indexes git, wiki and sourcetypes confluence:audit, gitlab:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners. Indexes required: `index=git`, `index=wiki`; Sourcetypes: `sourcetype=confluence:audit`, `sourcetype=gitlab:audit`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742), Splunk OT Security Add-on (5151), Splunk ITSI (1841), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: Windows Security Event Logs, critical infrastructure system logs, BSI IT-Grundschutz compliance scans, SIEM correlation events, vulnerability scan outputs, change management records, incident response logs, security audit evidence.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wiki sourcetype="confluence:audit" OR index=git sourcetype="gitlab:audit" earliest=-30d
| search (object="*security concept*" OR path="*konzept*")
| stats dc(author) as authors, count by object
| where authors>5
```
