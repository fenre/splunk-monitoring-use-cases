# UC-22.7.11 · Security Role Attestation — RBAC Changes vs HR Start Dates

## Description

Links IAM role grants to HR hire events to show responsibilities are assigned only when legitimate employment exists, supporting GV.RR.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes azure and sourcetypes mscs:azure:auditlog. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ingest HR daily snapshot; (2) align UPN keys; (3) investigate hits; (4) update joiner process documentation. Indexes required: `index=azure`; Sourcetypes: `sourcetype=mscs:azure:auditlog`; Lookups: `hr_employees`, `hr_employees.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Cloud Services (Splunkbase 3110).
• Ensure the following data sources are available: `mscs:azure:auditlog` (Add member to role); `hr_employees.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest HR daily snapshot; (2) align UPN keys; (3) investigate hits; (4) update joiner process documentation.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=azure sourcetype=mscs:azure:auditlog activityDisplayName="Add member to role" earliest=-30d
| eval upn=mvindex(targetResources{}.userPrincipalName,0)
| lookup hr_employees.csv upn OUTPUT hire_date
| eval hire_epoch=strptime(hire_date,"%Y-%m-%d")
| where isnotnull(hire_epoch) AND _time < hire_epoch
```
