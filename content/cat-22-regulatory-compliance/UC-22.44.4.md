# UC-22.44.4 · Vendor access telemetry — principals active outside contracted hours/geos

## Description

Intersects vendor principal authentication events with contracted access windows (hours/days) and permitted geographies; divergence is a DORA Art.28 finding.

## Value

Materialises contractual access controls into operational telemetry.

## Implementation

(1) Maintain vendor_principals macro; (2) extract contracted windows/geos from contracts and publish to vendor_contract.csv; (3) run hourly.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: IdP authentication events for named vendor principals, vendor contracts lookup with contracted hours/geos..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain vendor_principals macro; (2) extract contracted windows/geos from contracts and publish to vendor_contract.csv; (3) run hourly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=idp earliest=-1h (sourcetype=okta:audit OR sourcetype=azuread:audit) result=success
| search user IN (`vendor_principals`)
| `get_geolocation(src_ip)`
| lookup vendor_contract.csv user OUTPUT allowed_hours allowed_countries
| eval hour=strftime(_time,"%H"), weekday=strftime(_time,"%A"), window_ok=if(match(allowed_hours,weekday.":".hour),1,0), geo_ok=if(match(allowed_countries,country),1,0)
| where window_ok=0 OR geo_ok=0
| table _time user country hour weekday allowed_countries allowed_hours
```
