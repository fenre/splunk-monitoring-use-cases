# UC-22.4.10 · CCPA/CPRA — Sensitive PI — Health Information Field Exposure in Non-PHI Indexes

## Description

Health-related sensitive PI must stay within governed stores. Pattern matches for ICD codes, procedure names, or insurance member IDs in general application logs evidence accidental sprawl that CPRA-sensitive processing rules aim to prevent.

## Value

Health-related sensitive PI must stay within governed stores. Pattern matches for ICD codes, procedure names, or insurance member IDs in general application logs evidence accidental sprawl that CPRA-sensitive processing rules aim to prevent.

## Implementation

(1) Tune regex with legal/privacy to reduce false positives; (2) exclude approved `phi_approved` indexes explicitly; (3) auto-notable in ES with restricted perms; (4) trigger log redaction tickets; (5) map findings to DPIA / ROPA updates—not legal advice in Splunk.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=app sourcetype IN ("apache:access","nginx:access","app:json") earliest=-24h NOT index=phi_approved
| regex _raw="(?i)\b(icd[- ]?10|procedure code|member[_ ]?id|diagnosis)\b"
| stats count by sourcetype, host
| sort - count
```
