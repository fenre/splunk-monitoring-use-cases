# 22.13.28 — Listening Port and Service Baseline Deviation on BES Servers

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

We compare live listening ports from host telemetry to an approved baseline so unauthorized services on BES Cyber Assets are detected quickly.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes os and sourcetypes listening_ports, netstat, netstat`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=os (sourcetype=netstat OR sourcetype=listening_ports) earliest=-24h
| lookup bes_cyber_asset_inventory.csv host OUTPUT bes_system
| where isnotnull(bes_system)
| stats latest(port) as port latest(proto) as proto by host, port
| lookup bes_port_baseline.csv host, port OUTPUT approved
| where isnull(approved)
| table host, port, proto, bes_system
```

## Implementation

(1) Collect listening port data via UF scripted input. (2) Build baseline from golden images. (3) Alert on new ports for seven days. (4) Feed changes into CIP-010 change tickets. Indexes required: `index=os`; Sourcetypes: `sourcetype=listening_ports`, `sourcetype=netstat`, `sourcetype=netstat``; Lookups: `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`, `bes_port_baseline`, `bes_port_baseline.csv`.
