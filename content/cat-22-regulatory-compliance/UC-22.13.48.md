# 22.13.48 — BES Cyber System Restore Drill — RTO Measurement

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We compute restore-time metrics from drill tickets and application availability pings to evidence recovery capability testing.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes apps, itsm and sourcetypes http:ping`, restore_drill, restore_drill`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=itsm sourcetype=restore_drill bes_system=* earliest=-400d
| eval start=strptime(started_at,"%Y-%m-%d %H:%M:%S")
| eval end=strptime(restored_at,"%Y-%m-%d %H:%M:%S")
| eval rto_min=round((end-start)/60,2)
| lookup cip009_rto_targets.csv bes_system OUTPUT target_min
| where rto_min > target_min
| table drill_id, bes_system, rto_min, target_min, restored_at
```

## Implementation

(1) Populate RTO targets from recovery plans. (2) Validate timestamps from orchestration tools. (3) Track remediation for misses. (4) Annual leadership briefing export. Indexes required: `index=apps`, `index=itsm`; Sourcetypes: `sourcetype=http:ping``, `sourcetype=restore_drill`, `sourcetype=restore_drill``; Lookups: `cip009_rto_targets`, `cip009_rto_targets.csv`.

## Visualization

Column chart (RTO vs target), Table (misses), Single value (worst RTO). Key fields: `bes_system`, `drill_id`, `end`, `restored_at`, `rto_min`, `start`.
