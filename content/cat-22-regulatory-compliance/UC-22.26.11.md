# UC-22.26.11 · Offshore platform system monitoring

## Description

Monitors historian and controller tag changes that could impact process safety on offshore installations.

## Value

Gives the responsible team auditor-ready evidence for NO Petroleumsforskriften compliance by automating regulatory compliance evidence using indexes change, ot and sourcetypes engineering:plc, snow:chg. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain supporting lookups (classified hosts, sectors, transfers) aligned to Norwegian regulatory language. (2) Schedule searches with Europe/Oslo time semantics for reporting cutoffs. (3) Route ES notables to the correct national security, OT, or privacy queues. (4) Quarterly false-positive tuning with HR, operations, and legal stakeholders. Indexes required: `index=change`, `index=ot`; Sourcetypes: `sourcetype=engineering:plc`, `sourcetype=snow:chg`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742), Splunk Add-on for Unix and Linux (833), Splunk OT Security Add-on (5151), Splunk ITSI (1841), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: Windows Security Event Logs, network infrastructure logs, OT/ICS system logs, power grid SCADA, petroleum control systems, government system audit trails, identity management systems, security incident reports.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="engineering:plc" asset="NOR-OFF-*" earliest=-7d
| eval wo=coalesce(work_order,"NONE")
| join type=left asset [| search index=change sourcetype="snow:chg" earliest=-30d | rename cmdb_ci as asset | stats latest(number) as chg by asset]
| where wo="NONE" AND isnull(chg)
| stats latest(program_hash) as phash by asset, program_name, user
```
