# UC-22.21.3 · Automated Decision Audit Trail Reconstruction (Art. 12)

## Description

Chains decision_id across orchestrator, model gateway, and business rules engine to evidence end-to-end decision audit trail required for high-risk ADM logging.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_decision and sourcetypes ai:gateway, airflow:task, rules:engine. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Propagate decision_id across pipeline stages; (2) create macro `ai_decision_index`; (3) schedule every 30m for incomplete chains; (4) attach ITSI episode template for cross-team triage. Indexes required: `index=ai_decision`; Sourcetypes: `sourcetype=ai:gateway`, `sourcetype=airflow:task`, `sourcetype=rules:engine`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI (1841), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=ai_decision` sourcetypes: `airflow:task`, `ai:gateway`, `rules:engine` — shared decision_id.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Propagate decision_id across pipeline stages; (2) create macro `ai_decision_index`; (3) schedule every 30m for incomplete chains; (4) attach ITSI episode template for cross-team triage.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ai_decision earliest=-24h decision_id=*
| eval hop=case(sourcetype="airflow:task",1,sourcetype="ai:gateway",2,sourcetype="rules:engine",3,true(),0)
| stats values(sourcetype) as path, min(_time) as t0, max(_time) as t1, range(_time) as span_sec by decision_id
| eval complete=if(mvcount(path)>=3,1,0)
| where complete=0
| eval span_min=round(span_sec/60,2)
| table decision_id, path, span_min, t0, t1
```
