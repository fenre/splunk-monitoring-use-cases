# UC-22.8.19 · SOC 2 CC6 — Timeliness of Access Removal After HR Termination Events

## Description

CC6 expects removal of access when no longer required. Measuring lag between HR termination timestamp and last successful disablement event across IdM and key SaaS apps proves offboarding SLAs are met—high-risk evidence for every SOC examination.

## Value

CC6 expects removal of access when no longer required. Measuring lag between HR termination timestamp and last successful disablement event across IdM and key SaaS apps proves offboarding SLAs are met—high-risk evidence for every SOC examination.

## Implementation

(1) Align `employee_id` across HR and IdM; (2) tune 120 minutes to contractual SLA; (3) add parallel tracks for AD disable and VPN revoke; (4) page IAM on breach; (5) monthly median lag KPI for management review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr sourcetype="workday:termination" earliest=-30d
| eval term=strptime(term_effective_ts,"%Y-%m-%dT%H:%M:%SZ")
| join type=left employee_id [
    search index=identity sourcetype="okta:system" eventType="user.lifecycle.deactivate" earliest=-30d
    | eval deactivated=strptime(published,"%Y-%m-%dT%H:%M:%SZ")
    | stats min(deactivated) as first_deact by target_user_id
    | rename target_user_id as employee_id
  ]
| eval lag_sec=first_deact-term
| eval lag_min=round(lag_sec/60,1)
| where lag_min>120 OR isnull(first_deact)
| table employee_id, term, first_deact, lag_min
```
