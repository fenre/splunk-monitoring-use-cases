# UC-22.11.34 · Critical CVE Remediation SLA Breach Tracking

## Description

Computes age in days for critical vulnerabilities against the organization’s documented remediation SLA so PCI expectations for timely flaw correction are measurable.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes vm and sourcetypes tenable:sc:vuln. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Adjust `sla_days` per risk policy (15 for internet-facing). (2) Join patch ticket state from ITSM to exclude in-progress. (3) Weekly governance email. (4) Document exceptions with compensating controls. Indexes required: `index=vm`; Sourcetypes: `sourcetype=tenable:sc:vuln`; Lookups: `pci_in_scope_hosts`, `pci_in_scope_hosts.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Tenable (4060).
• Ensure the following data sources are available: `index=vm` `tenable:sc:vuln` — `first_found`, `last_found`, `severity`, `host_fqdn`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Adjust `sla_days` per risk policy (15 for internet-facing). (2) Join patch ticket state from ITSM to exclude in-progress. (3) Weekly governance email. (4) Document exceptions with compensating controls.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vm sourcetype="tenable:sc:vuln" severity="Critical" earliest=-90d
| lookup pci_in_scope_hosts.csv fqdn AS host_fqdn OUTPUT in_scope
| where in_scope="true"
| where match(state,"(?i)open|reopened)")
| eval age_days=round((now()-first_found)/86400,1)
| eval sla_days=30
| where age_days>sla_days
| stats max(age_days) as max_age by host_fqdn, plugin_name
| sort - max_age
```
