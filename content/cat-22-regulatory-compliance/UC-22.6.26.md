# UC-22.6.26 · Privileged Access — Sudo and RunAs Usage Outside PAM Session

## Description

Surfaces local privilege elevation without corresponding PAM session ID for privileged access monitoring.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001, ISO/IEC 27001 compliance by automating regulatory compliance evidence using indexes os and sourcetypes linux:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Clock sync with PAM; (2) break-glass accounts excluded; (3) monthly privileged user review. Indexes required: `index=os`; Sourcetypes: `sourcetype=linux:audit`; Lookups: `pam_sessions`, `pam_sessions.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: CyberArk, Linux audit.
• Ensure the following data sources are available: `linux:audit` type=USER_CMD.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Clock sync with PAM; (2) break-glass accounts excluded; (3) monthly privileged user review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os sourcetype=linux:audit type=USER_CMD earliest=-24h
| where match(exe,"(?i)sudo|su|pkexec")
| lookup pam_sessions.csv user, _time OUTPUT pam_session_id
| where isnull(pam_session_id)
```
