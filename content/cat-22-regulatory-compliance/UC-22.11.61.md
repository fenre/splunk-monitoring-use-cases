# UC-22.11.61 · Audit Log Source Completeness — Missing Windows Security Events per CDE Host

## Description

Identifies in-scope Windows servers that stopped sending Security event logs so audit trails required for reconstructing access to system components remain complete and continuous.

## Value

Identifies in-scope Windows servers that stopped sending Security event logs so audit trails required for reconstructing access to system components remain complete and continuous.

## Implementation

(1) Validate `tstats` summaries are enabled for `windows` index or use metadata `| metadata type=hosts` alternative if needed. (2) Page if any CDE host missing. (3) Check forwarder connectivity. (4) Daily compliance health dashboard.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=windows` `WinEventLog:Security`; `lookup pci_windows_cde_assets.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Validate `tstats` summaries are enabled for `windows` index or use metadata `| metadata type=hosts` alternative if needed. (2) Page if any CDE host missing. (3) Check forwarder connectivity. (4) Daily compliance health dashboard.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| tstats prestats=t count WHERE index=windows sourcetype="WinEventLog:Security" earliest=-25h latest=-1h BY host
| stats count by host
| inputlookup pci_windows_cde_assets.csv
| fields hostname
| rename hostname as host
| join type=left host [
    search | tstats prestats=t count WHERE index=windows sourcetype="WinEventLog:Security" earliest=-25h latest=-1h BY host
    | stats count by host
  ]
| eval has_logs=if(isnotnull(count) AND count>0,1,0)
| where has_logs=0
| table host
```
