# UC-22.3.25 · DORA TLPT — Test Planning Milestone and Scope Lock Audit Trail

## Description

Threat-led penetration testing requires disciplined scope definition and change control. Tracking planning milestones and scope-lock timestamps shows the entity controlled test boundaries and prevented ad hoc scope creep that could invalidate results.

## Value

Gives the responsible team auditor-ready evidence for DORA compliance by automating regulatory compliance evidence using indexes grc and sourcetypes tlpt:milestone. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Integrate CISO office project template; (2) alert 14 days before `due_date`; (3) store `scope_hash` of in-scope IPs/apps; (4) map engagement to competent authority reporting year; (5) restrict index to red-team and compliance roles. Indexes required: `index=grc`; Sourcetypes: `sourcetype=tlpt:milestone`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (Splunkbase 1928), HEC from GRC.
• Ensure the following data sources are available: `index=grc` `sourcetype="tlpt:milestone"` (engagement_id, milestone, due_date, completed_date, scope_hash).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Integrate CISO office project template; (2) alert 14 days before `due_date`; (3) store `scope_hash` of in-scope IPs/apps; (4) map engagement to competent authority reporting year; (5) restrict index to red-team and compliance roles.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype="tlpt:milestone" earliest=-730d
| eval due=strptime(due_date,"%Y-%m-%d"), done=strptime(completed_date,"%Y-%m-%d")
| where milestone="scope_lock" AND (isnull(done) OR done>due)
| stats latest(scope_hash) as scope by engagement_id, milestone
| sort due
```
