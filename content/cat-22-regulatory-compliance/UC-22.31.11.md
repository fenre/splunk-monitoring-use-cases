# 22.31.11 — Multi-factor authentication coverage and failures

**Criticality:** low | **Difficulty:** expert | **Status:** N/A

## Description

Evidences ASD Essential Eight obligations for multi-factor authentication coverage and failures using endpoint, identity, and cloud telemetry.

## Value

Gives the responsible team auditor-ready evidence for ASD E8 compliance by automating regulatory compliance evidence using indexes o365 and sourcetypes ms:o365:management. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=o365 sourcetype="ms:o365:management" Operation="Set-MultiFactorAuthenticationRequirements" earliest=-7d
| stats latest(ResultStatus) as rs by UserId
| where rs!="Success"
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.process_name, Processes.user, Processes.dest | sort - count
```

## Implementation

(1) Ingest CrowdStrike policy posture via TA 5082 and map hosts to NDB and NZISM tiers. (2) Tune alerts with ACSD Essential Eight and APRA-aligned severities. (3) Coordinate privacy and security ownership for dual-reporting workflows. (4) Monthly export for privacy officer and prudential working groups. Indexes required: `index=o365`; Sourcetypes: `sourcetype=ms:o365:management`.
