# 22.50.35 — FERPA §99.31 — Disclosure-without-consent monitoring: education-record access audit trail

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Monitors education record disclosures for compliance with FERPA §99.31, which defines conditions under which disclosure without prior consent is permitted. Uses FERPA-specific exception terminology (school official, transfer, financial aid, accreditation, judicial order, health/safety, sex offender, directory information).

## Value

§99.31 limits when education records may be disclosed without consent. Continuous audit-trail monitoring ensures each disclosure is justified under a permitted exception.

## SPL

```spl
index=sis sourcetype IN (student_record_access, disclosure_log) earliest=-30d
| search record_type="education_record"
| eval has_consent=if(isnotnull(consent_ref) AND consent_ref!="", 1, 0)
| eval exception_basis=coalesce(disclosure_basis, "none")
| eval valid_exception=if(exception_basis IN ("school_official", "transfer", "financial_aid", "accreditation", "judicial_order", "health_safety", "sex_offender", "directory_info"), 1, 0)
| eval compliant_disclosure=if(has_consent=1 OR valid_exception=1, 1, 0)
| stats count AS total_disclosures sum(compliant_disclosure) AS compliant sum(has_consent) AS with_consent sum(valid_exception) AS with_exception dc(accessor_id) AS distinct_accessors BY system, exception_basis
| eval non_compliant=total_disclosures - compliant
| where non_compliant > 0
| table system, exception_basis, total_disclosures, compliant, with_consent, with_exception, non_compliant, distinct_accessors
```

## Implementation

(1) Ingest SIS access logs with disclosure_basis, consent_ref, consent_status; (2) maintain a lookup of permitted exception codes; (3) schedule daily; (4) escalate non-compliant disclosures to the Registrar. Indexes required: `index=sis_access`; Sourcetypes: `sourcetype=education_record_access`, `sourcetype=sis_audit`.

## Visualization

Table of flagged records for investigation; single-value KPI of coverage / compliance percentage; time chart of trend over the last 90 days; drill-down per responsible owner. Key fields: `compliance_pct`, `compliant`, `compliant_disclosure`, `consent_on_file`, `non_compliant`, `non_compliant_users`.

## Compliance

- **FERPA** 20 USC §1232g: §99.31 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for §99.31.

## Known False Positives

Tuning prerequisites: source field mappings must be in place before the UC will produce reliable metrics; early runs will commonly flag unmapped systems until onboarding is completed.
