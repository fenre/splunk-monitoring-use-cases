# UC-22.8.29 · SOC 2 C1 — Confidential Information Disposal and Secure Destruction Evidence

## Description

C1 addresses confidential information. Tracking destruction certificates, crypto-shredding events, and DLP “secure delete” workflows demonstrates media and logical disposal met policy when assets retired or contracts ended.

## Value

C1 addresses confidential information. Tracking destruction certificates, crypto-shredding events, and DLP “secure delete” workflows demonstrates media and logical disposal met policy when assets retired or contracts ended.

## Implementation

(1) Flip logic per policy: flag retirements without cert within 30 days; (2) restrict `classification=confidential` scope; (3) integrate ITAD vendor feed; (4) legal hold exceptions via `legal_hold_assets.csv`; (5) annual sample for auditors with cert PDF links (out of band).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect, vendor APIs (Iron Mountain, secure erase tools) via HEC.
• Ensure the following data sources are available: `index=compliance` `sourcetype="disposal:cert"` (asset_id, method, cert_number, destroyed_ts, witness); `index=endpoint` BitLocker or secure erase tool logs.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Flip logic per policy: flag retirements without cert within 30 days; (2) restrict `classification=confidential` scope; (3) integrate ITAD vendor feed; (4) legal hold exceptions via `legal_hold_assets.csv`; (5) annual sample for auditors with cert PDF links (out of band).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=compliance sourcetype="disposal:cert" earliest=-365d
| eval destroyed=strptime(destroyed_ts,"%Y-%m-%dT%H:%M:%SZ")
| join type=left asset_id [
    search index=cmdb sourcetype="cmdb:asset" earliest=-400d
    | fields asset_id, retired_date, classification
    | eval retired=strptime(retired_date,"%Y-%m-%d")
  ]
| where retired>relative_time(destroyed,"+30d@d")
| table asset_id, classification, retired, destroyed, cert_number
```
