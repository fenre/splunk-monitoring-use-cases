# UC-22.7.30 · Storage Path — Cluster Failover or Multipath Events

## Description

Surfaces storage or hypervisor failover events evidencing infrastructure resilience monitoring.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes infra and sourcetypes vmware:inv. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize messages per vendor; (2) alert spikes; (3) tie to capacity planning. Indexes required: `index=infra`; Sourcetypes: `sourcetype=vmware:inv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: VMware or SAN logs.
• Ensure the following data sources are available: `vmware:inv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize messages per vendor; (2) alert spikes; (3) tie to capacity planning.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=infra sourcetype=vmware:inv earliest=-7d failover OR "path down"
| stats count by host, Message
```

Understanding this SPL
