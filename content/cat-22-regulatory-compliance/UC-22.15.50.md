# UC-22.15.50 · DMZ Integrity Between IT and OT — Corporate Browser to OT HMI

## Description

Finds IT-originated web traffic reaching OT HMIs that should only be reachable via bastion or reverse proxy in the DMZ.

## Value

Finds IT-originated web traffic reaching OT HMIs that should only be reachable via bastion or reverse proxy in the DMZ.

## Implementation

(1) Enforce explicit proxy paths; (2) split DNS views; (3) microsegment HMI; (4) user awareness for bookmarked IPs.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Palo Alto (2757).
• Ensure the following data sources are available: `pan:traffic` url_category, zones.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enforce explicit proxy paths; (2) split DNS views; (3) microsegment HMI; (4) user awareness for bookmarked IPs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=fw sourcetype="pan:traffic" earliest=-7d
| where dest_zone="OT-Web-HMI" AND src_zone="Corporate-Users" AND app="web-browsing"
| lookup ot_hmi_exposure.csv dest_ip OUTPUT exposure
| where exposure!="via_dmz_proxy"
| stats count by src_ip, dest_ip, url_category, user
| sort - count
```
