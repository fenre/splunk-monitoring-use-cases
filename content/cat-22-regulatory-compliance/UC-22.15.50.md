# 22.15.50 — DMZ Integrity Between IT and OT — Corporate Browser to OT HMI

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Finds IT-originated web traffic reaching OT HMIs that should only be reachable via bastion or reverse proxy in the DMZ.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes fw and sourcetypes pan:traffic. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=fw sourcetype="pan:traffic" earliest=-7d
| where dest_zone="OT-Web-HMI" AND src_zone="Corporate-Users" AND app="web-browsing"
| lookup ot_hmi_exposure.csv dest_ip OUTPUT exposure
| where exposure!="via_dmz_proxy"
| stats count by src_ip, dest_ip, url_category, user
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Web.Web by Web.src, Web.dest | sort - count
```

## Implementation

(1) Enforce explicit proxy paths; (2) split DNS views; (3) microsegment HMI; (4) user awareness for bookmarked IPs. Indexes required: `index=fw`; Sourcetypes: `sourcetype=pan:traffic`; Lookups: `ot_hmi_exposure`, `ot_hmi_exposure.csv`.

## Visualization

Table, Map (src), Time chart. Key fields: `dest_ip`, `src_ip`, `url_category`, `user`.

## Compliance

- **IEC 62443** 2013-ongoing: FR 6.2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for FR 6.2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
