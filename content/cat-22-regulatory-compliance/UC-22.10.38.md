# UC-22.10.38 · Deceased Patient Records — Access After Death Date

## Description

Monitors chart access after documented death timestamps for users not on approved post-mortem workflows (HIM, organ donation, medical examiner).

## Value

Monitors chart access after documented death timestamps for users not on approved post-mortem workflows (HIM, organ donation, medical examiner).

## Implementation

(1) Refresh death index from ADT feed within SLA; (2) legal guidance on state law vs HIPAA; (3) sensitive dashboard ACLs.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686).
• Ensure the following data sources are available: `index=epic` `sourcetype=epic:audit`, `lookup patient_death_index.csv` (`PAT_ID`, `death_dttm`, `restricted_flag`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Refresh death index from ADT feed within SLA; (2) legal guidance on state law vs HIPAA; (3) sensitive dashboard ACLs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=epic sourcetype="epic:audit" AccessType="PatientChart" earliest=-90d
| lookup patient_death_index.csv PAT_ID OUTPUT death_dttm, restricted_flag
| where isnotnull(death_dttm) AND _time > strptime(death_dttm,"%Y-%m-%d %H:%M:%S")
| lookup post_mortem_allowed_roles.csv USER_ID OUTPUT allowed_role
| where isnull(allowed_role)
| stats count by USER_ID, PAT_ID
| sort - count
```
