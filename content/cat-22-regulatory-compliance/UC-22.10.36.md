# UC-22.10.36 · Bulk ePHI Export — Clarity SQL / Caboodle Extract Volume Spike

## Description

Detects unusually large SELECT/BCP operations against patient fact tables that may indicate unauthorized bulk download of ePHI for exfiltration.

## Value

Detects unusually large SELECT/BCP operations against patient fact tables that may indicate unauthorized bulk download of ePHI for exfiltration.

## Implementation

(1) Map ETL service accounts to allow list; (2) require data request tickets with correlation ID in SQL comment where feasible; (3) integrate DLP for flat file exports.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686), Splunk Add-on for Microsoft SQL Server (2648).
• Ensure the following data sources are available: `index=mssql` `sourcetype=mssql:audit` (`statement`, `rows`, `session_server_principal_name`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map ETL service accounts to allow list; (2) require data request tickets with correlation ID in SQL comment where feasible; (3) integrate DLP for flat file exports.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mssql sourcetype="mssql:audit" earliest=-24h action_class="SELECT"
| where match(lower(statement),"from\s+clarity\.(pat|patient|episodes)") OR match(lower(statement),"from\s+caboodle")
| eval row_estimate=coalesce(rows,0)
| bin _time span=1h
| stats sum(row_estimate) as est_rows by session_server_principal_name, _time, client_ip
| eventstats median(est_rows) as med by session_server_principal_name
| where est_rows > med*20 AND est_rows>100000
| sort - est_rows
```
