# UC-22.13.4 · Annual CIP-002 Categorization Review Evidence Package

## Description

We timestamp inventory validation searches and signed review workflows so annual categorization review evidence is searchable and complete.

## Value

We timestamp inventory validation searches and signed review workflows so annual categorization review evidence is searchable and complete.

## Implementation

(1) Emit HEC when the responsible manager signs the annual review. (2) Schedule the inventory recon search to write `cip:search_evidence` events. (3) Alert if `signed_ok=0` approaching the compliance date. (4) PDF the dashboard for the audit binder.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=itsm` `sourcetype=cip:review_ticket`, `index=cip_evidence` `sourcetype=cip:search_evidence`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Emit HEC when the responsible manager signs the annual review. (2) Schedule the inventory recon search to write `cip:search_evidence` events. (3) Alert if `signed_ok=0` approaching the compliance date. (4) PDF the dashboard for the audit binder.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype=cip:review_ticket review_type="CIP-002_Categorization" earliest=-400d
| eval signed_epoch=strptime(signed_at,"%Y-%m-%dT%H:%M:%SZ")
| eval annual_due=relative_time(now(),"-1y@d")
| eval signed_ok=if(signed_epoch>=annual_due,1,0)
| append [
    search index=cip_evidence sourcetype=cip:search_evidence search_name="CIP002_inventory_recon" earliest=-400d
    | stats latest(_time) as last_proof by search_name
  ]
| table review_type, owner, signed_at, signed_ok, last_proof
```
