# 22.50.44 — FERC Order 887 — Internal network security monitoring for bulk electric systems

**Criticality:** high | **Difficulty:** advanced | **Status:** verified

## Description

Monitors internal network traffic within BES Electronic Security Perimeters for anomalous or unauthorised protocols and flows per FERC Order 887 requirements for internal network security monitoring.

## Value

FERC Order 887 requires utilities to implement internal network security monitoring within ESPs beyond the perimeter. Continuous network flow analysis within control system networks provides evidence of monitoring capability.

## SPL

```spl
index=ot_network sourcetype IN (bes_network_flow, esp_firewall, ot_ids) earliest=-24h
| eval anomalous=if(action IN ("blocked", "denied", "alert") OR protocol NOT IN ("modbus", "dnp3", "iec61850", "iccp", "opc-ua", "https", "ssh"), 1, 0)
| eval internal_traffic=if(src_zone="esp" AND dest_zone="esp", 1, 0)
| stats count AS total_flows sum(anomalous) AS anomalous_flows sum(internal_traffic) AS internal_flows dc(src_ip) AS distinct_sources dc(dest_ip) AS distinct_destinations BY esp_zone, protocol
| eval anomaly_pct=round(100*anomalous_flows/total_flows, 1)
| where anomalous_flows>0
| sort -anomalous_flows
| table esp_zone, protocol, total_flows, anomalous_flows, anomaly_pct, internal_flows, distinct_sources, distinct_destinations
```

## Implementation

(1) Deploy network flow collection within ESP zones; (2) normalise protocol, action, src_zone, dest_zone; (3) schedule every 15 minutes; (4) escalate anomalous internal flows immediately. Indexes required: `index=ot_network`; Sourcetypes: `sourcetype=bes_network_flow`, `sourcetype=esp_firewall`, `sourcetype=ot_ids`.

## Visualization

Table of flagged records for investigation; single-value KPI of coverage / compliance percentage; time chart of trend over the last 90 days; drill-down per responsible owner. Key fields: `anomalous`, `anomalous_flows`, `anomaly_pct`, `distinct_destinations`, `distinct_sources`, `esp_zone`.

## Compliance

- **FERC CIP** current: Order887 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Order887.

## Known False Positives

Tuning prerequisites: source field mappings must be in place before the UC will produce reliable metrics; early runs will commonly flag unmapped systems until onboarding is completed.
