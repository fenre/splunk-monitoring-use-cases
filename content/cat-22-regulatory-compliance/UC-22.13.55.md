# 22.13.55 — TCA Scan-Before-Connect Compliance from NAC or Agent Logs

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We verify transient cyber assets received a successful posture or malware scan before being granted ESP network access.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes endpoint, network and sourcetypes cisco:ise:radius, cisco:ise:radius`, defender:scan. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=network sourcetype=cisco:ise:radius earliest=-24h Acct-Status-Type=Start
| lookup tca_device_registry.csv calling_station_id OUTPUT asset_tag
| where isnotnull(asset_tag)
| join type=left calling_station_id [
    search index=endpoint sourcetype=defender:scan earliest=-24h
    | rename device_id as calling_station_id
    | stats latest(scan_result) as scan by calling_station_id
  ]
| where scan!="clean" OR isnull(scan)
| table _time, calling_station_id, user, scan, asset_tag
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Align RADIUS calling-station IDs with endpoint inventory keys. (2) Tune for your NAC vendor sourcetype. (3) Block or quarantine integration outside Splunk. (4) Retain pass/fail for audits. Indexes required: `index=endpoint`, `index=network`; Lookups: `tca_device_registry`, `tca_device_registry.csv`.

## Visualization

Table (non-compliant TCA), Single value (count), Timeline. Key fields: `asset_tag`, `calling_station_id`, `scan`, `user`.

## Compliance

- **NERC CIP** current: CIP-010-4 R5 Part 5.3 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-010-4 R5 Part 5.3.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
