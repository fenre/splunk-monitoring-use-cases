# UC-22.13.55 · TCA Scan-Before-Connect Compliance from NAC or Agent Logs

## Description

We verify transient cyber assets received a successful posture or malware scan before being granted ESP network access.

## Value

We verify transient cyber assets received a successful posture or malware scan before being granted ESP network access.

## Implementation

(1) Align RADIUS calling-station IDs with endpoint inventory keys. (2) Tune for your NAC vendor sourcetype. (3) Block or quarantine integration outside Splunk. (4) Retain pass/fail for audits.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=network` `sourcetype=cisco:ise:radius` or `sourcetype=pan:globalprotect`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align RADIUS calling-station IDs with endpoint inventory keys. (2) Tune for your NAC vendor sourcetype. (3) Block or quarantine integration outside Splunk. (4) Retain pass/fail for audits.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype=cisco:ise:radius earliest=-24h Acct-Status-Type=Start
| lookup tca_device_registry.csv calling_station_id OUTPUT asset_tag
| where isnotnull(asset_tag)
| join type=left calling_station_id [
    search index=endpoint sourcetype=defender:scan earliest=-24h
    | rename device_id as calling_station_id
    | stats latest(scan_result) as scan by calling_station_id
  ]
| where scan!="clean" OR isnull(scan)
| table _time, calling_station_id, user, scan, asset_tag
```
