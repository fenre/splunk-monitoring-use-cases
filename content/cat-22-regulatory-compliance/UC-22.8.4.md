# UC-22.8.4 · SOC 2 Control Environment and Board-Level Attestation Workflow

## Description

CC1 and CC2 require communication and information about roles, responsibilities, and performance to support functioning of internal control. Tracking completion of quarterly control-owner attestations in ITSM demonstrates tone-at-the-top processes are operationalized with timestamps.

## Value

Gives the responsible team auditor-ready evidence for SOC 2 compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:sc_req_item. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Create catalog items per SOC2 control family with `u_control_id` matching your CCM matrix; (2) schedule quarterly auto-open tasks; (3) escalate open items after 14 days; (4) export CSV for external auditors; (5) map `assigned_to` to job titles for CC1.3 HR evidence optional. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:sc_req_item`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (Splunkbase 1928).
• Ensure the following data sources are available: `index=itsm` `sourcetype="snow:sc_req_item"` (number, cat_item, state, closed_at, opened_at, assigned_to, u_control_id).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Create catalog items per SOC2 control family with `u_control_id` matching your CCM matrix; (2) schedule quarterly auto-open tasks; (3) escalate open items after 14 days; (4) export CSV for external auditors; (5) map `assigned_to` to job titles for CC1.3 HR evidence optional.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="snow:sc_req_item" earliest=-120d (cat_item="*SOC2*Attestation*" OR short_description="*control owner attestation*")
| eval is_closed=if(match(state,"(?i)closed|resolved|complete"),1,0)
| stats count as tasks, sum(is_closed) as completed, dc(assigned_to) as owners by u_control_id, cat_item
| eval completion_pct=round(100*completed/tasks,1)
| where completion_pct<100
| sort u_control_id
```
