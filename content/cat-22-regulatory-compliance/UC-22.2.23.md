# 22.2.23 — NIS2 Incident Handling Procedure Adherence and Playbook Execution

**Criticality:** critical | **Difficulty:** intermediate | **Status:** verified

## Description

Policies on handling incidents must be effective in practice. This use case tracks whether incident tickets traverse required playbook stages (triage, containment, eradication, recovery, lessons learned) within SLA — correlating SOC workflow fields with NIS2 incident classes.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:incident. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
`notable` urgency IN ("high","critical") earliest=-30d
| eval incident_id=rule_name."|"._time
| join incident_id type=left [
    search index=itsm sourcetype="snow:incident" earliest=-30d
    | eval incident_id=correlation_id
    | eval stages_complete=mvcount(mvappend(
        if(match(work_notes,"(?i)contain"),"contain",""),
        if(match(work_notes,"(?i)erad"),"erad",""),
        if(match(work_notes,"(?i)recover"),"recover",""),
        if(match(work_notes,"(?i)lessons"),"lessons","")))
    | stats latest(stages_complete) as stages_complete by incident_id
]
| eval playbook_gap=if(stages_complete < 4, 1, 0)
| where playbook_gap=1 AND status!="Closed"
| table _time, rule_name, status, owner, stages_complete
```

## Implementation

(1) Standardise work-note keywords for each playbook stage; (2) link ES notables to ServiceNow `correlation_id`; (3) alert IR leads on missing stages before closure; (4) include lessons-learned completion in post-incident review; (5) export metrics for supervisory evidence packs. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:incident`.

## Visualization

Funnel chart (stage completion), Table (incomplete incidents), Single value (% full playbook adherence). Key fields: `incident_id`, `owner`, `playbook_gap`, `rule_name`, `stages_complete`, `status`.

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(b) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(b) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
