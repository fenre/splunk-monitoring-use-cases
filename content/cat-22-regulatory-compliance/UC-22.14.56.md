# UC-22.14.56 · Configuration Settings Compliance — CIS Benchmark Control Checks

## Description

Ingests configuration assessment scan results for CM-6 configuration settings.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes compliance and sourcetypes cis:assessment. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize control_id to CIS references; (2) join system owner lookup; (3) SLA remediations; (4) map to SC-7 for boundary controls when network-related. Indexes required: `index=compliance`; Sourcetypes: `sourcetype=cis:assessment`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Tenable Add-On for Splunk (Splunkbase 4060), SCAP/OVAL via HEC.
• Ensure the following data sources are available: `tenable:sc` compliance exports or custom `cis:assessment` sourcetype.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize control_id to CIS references; (2) join system owner lookup; (3) SLA remediations; (4) map to SC-7 for boundary controls when network-related.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=compliance sourcetype=cis:assessment earliest=-7d control_result="FAIL" benchmark="CIS_Microsoft_Windows_Server_2022"
| stats count by hostname, control_id, severity
| sort - severity, - count
```
