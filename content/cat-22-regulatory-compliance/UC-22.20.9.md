# 22.20.9 — CMMC SI.L2-3.14.6 — Real-time attack monitoring on CUI systems

**Criticality:** high | **Difficulty:** beginner | **Status:** verified

## Description

Correlates real-time attack indicators from EDR (CrowdStrike, Defender, Carbon Black) and IDS (Suricata, Snort) on CUI-classified systems. CMMC SI.L2-3.14.6 requires monitoring organisational systems to detect attacks and indicators of potential attacks.

## Value

Provides a unified attack monitoring view across CUI systems combining endpoint detection and network intrusion data — demonstrating continuous, multi-layer attack monitoring capability.

## SPL

```spl
(index=endpoint sourcetype IN ("crowdstrike:detections", "defender:alerts", "carbon_black:alerts") earliest=-24h)
OR (index=ids sourcetype IN ("suricata", "snort") earliest=-24h)
| lookup cui_hosts.csv host OUTPUT cui_flag
| where cui_flag="true"
| eval severity=coalesce(severity, alert_severity, priority)
| eval attack_type=coalesce(tactic, category, classification)
| stats count AS detections dc(host) AS affected_hosts values(attack_type) AS attack_types latest(_time) AS last_seen BY severity
| sort case(severity="critical",0, severity="high",1, severity="medium",2, true(),3)
| table severity, detections, affected_hosts, attack_types, last_seen
```

## CIM SPL

```spl
| tstats count from datamodel=Intrusion_Detection BY IDS_Attacks.severity, IDS_Attacks.dest | sort - count
```

## Implementation

(1) Verify data sources: CrowdStrike detections, Microsoft Defender alerts, Carbon Black alerts, Suricata IDS, Snort IDS, `cui_hosts.csv` lookup; (2) Maintain CUI-scoped lookups; (3) Schedule search at compliance-aligned interval; (4) Route findings to ticketing queue; (5) Retain exports per records management policy.

## Visualization

Table of detections by severity. Timeline of detection events. Single value for total critical/high severity detections in last 24 hours.

## Compliance

- **CMMC** 2.0: SI.L2-3.14.6 (partial) — Combines EDR and IDS monitoring on CUI systems for multi-layer real-time attack detection — directly satisfying SI.L2-3.14.6.

## Known False Positives

Vulnerability scanners and penetration testing tools generate detection events — correlate with approved scan schedules.
