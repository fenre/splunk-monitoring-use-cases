# UC-22.43.5 · SBOM vendor-component CVE exposure

## Description

Uses the vendor SBOM register to answer 'which vendor components in my estate are affected by the CVE published today?'.

## Value

Closes the gap between the day a CVE is published and the day a vendor responds — crucial for CRA and NIS2 supply-chain narratives.

## Implementation

(1) Ingest vendor SBOMs from the procurement workflow; (2) subscribe to NVD feed into cve_feed.csv; (3) run daily; (4) SOAR opens a case per vendor-affected component.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: Vendor SBOMs (CycloneDX/SPDX), CVE feed (NVD, vendor advisories)..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest vendor SBOMs from the procurement workflow; (2) subscribe to NVD feed into cve_feed.csv; (3) run daily; (4) SOAR opens a case per vendor-affected component.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup vendor_sbom.csv
| lookup cve_feed.csv component version OUTPUT cve cvss_v3_base
| where isnotnull(cve) AND cvss_v3_base>=7.0
| stats values(cve) as cves, max(cvss_v3_base) as max_cvss by vendor component version
| table vendor component version cves max_cvss
```
