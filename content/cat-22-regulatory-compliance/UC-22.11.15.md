# UC-22.11.15 · Key Management Operations from HSM and KMS Audit Trails

## Description

Centralizes HSM partition and cloud KMS administrative events so cryptographic key custodians can prove dual-control and logging for keys that protect stored account data.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes aws, hsm and sourcetypes aws:cloudtrail, thales:luna:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map IAM user/role to human custodian IDs. (2) Require ticket ID in CloudTrail session context where possible. (3) Alert on `ScheduleKeyDeletion` without CAB record. (4) Retain 12+ months in compliant storage tier. Indexes required: `index=aws`, `index=hsm`; Sourcetypes: `sourcetype=aws:cloudtrail`, `sourcetype=thales:luna:audit`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for AWS (1876), custom HSM syslog parser.
• Ensure the following data sources are available: `index=aws` `sourcetype="aws:cloudtrail"` (`eventName` CreateKey, DisableKey, ScheduleKeyDeletion); `index=hsm` `sourcetype="thales:luna:audit"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map IAM user/role to human custodian IDs. (2) Require ticket ID in CloudTrail session context where possible. (3) Alert on `ScheduleKeyDeletion` without CAB record. (4) Retain 12+ months in compliant storage tier.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
(index=aws sourcetype="aws:cloudtrail" eventSource="kms.amazonaws.com" earliest=-24h)
 OR (index=hsm sourcetype="thales:luna:audit" earliest=-24h)
| eval op=coalesce(eventName, hsm_command, operation)
| where match(op,"(?i)delete|disable|import|unwrap|clone|partition")
| stats count by user, op, src_ip, host
| sort - count
```
