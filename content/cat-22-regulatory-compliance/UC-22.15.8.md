# UC-22.15.8 · OT Change Management — PLC Download Without CM Ticket

## Description

Correlates controller downloads or online edits with change tickets to enforce controlled maintenance for IACS devices.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes itsm, ot_eng and sourcetypes ab:audit, plc:download, siemens:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize vendor fields to shared `ck`; (2) mandate CM tickets carry project keys; (3) tune time join if needed via subsearch; (4) weekly engineering review of unmatched. Indexes required: `index=itsm`, `index=ot_eng`; Sourcetypes: `sourcetype=ab:audit`, `sourcetype=plc:download`, `sourcetype=siemens:audit`, `sourcetype=snow:change_request`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk OT Security Add-on (5151), ServiceNow TA (1928).
• Ensure the following data sources are available: `index=ot_eng` vendor PLC audit; `snow:change_request`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize vendor fields to shared `ck`; (2) mandate CM tickets carry project keys; (3) tune time join if needed via subsearch; (4) weekly engineering review of unmatched.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_eng sourcetype="plc:download" OR sourcetype="siemens:audit" OR sourcetype="ab:audit" earliest=-14d
| eval ck=coalesce(project_id, program_name, concat("PLC-",controller))
| eval t0=_time
| join type=left ck [ search index=itsm sourcetype="snow:change_request" earliest=-30d | eval ck=coalesce(u_ot_project, number) | where state IN ("Closed Complete","Implement") | fields number, ck, opened_at ]
| where isnull(number)
| stats earliest(_time) as first_dl, latest(user) as users, values(src_ip) as src by ck, controller
```
