# UC-22.2.37 · NIS2 SCADA System Access Monitoring

## Description

Interactive access to SCADA/HMI layers is high risk. This use case monitors engineering workstation logons, HMI tag writes, and remote-desktop sessions to SCADA hosts — correlating with maintenance windows and approved technician identities.

## Value

Interactive access to SCADA/HMI layers is high risk. This use case monitors engineering workstation logons, HMI tag writes, and remote-desktop sessions to SCADA hosts — correlating with maintenance windows and approved technician identities.

## Implementation

(1) Inventory SCADA hosts with roles in `nis2_scada_asset_inventory.csv`; (2) ingest maintenance windows from plant maintenance system; (3) alert on interactive access outside windows; (4) integrate PAM session video references for investigations; (5) align with safety permit-to-work where applicable.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (Splunkbase 742), Splunk Add-on for CyberArk (Splunkbase 2891).
• Ensure the following data sources are available: `WinEventLog:Security` EventCode=4624/4648, `cyberark:session`, `nis2_scada_maintenance_windows.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Inventory SCADA hosts with roles in `nis2_scada_asset_inventory.csv`; (2) ingest maintenance windows from plant maintenance system; (3) alert on interactive access outside windows; (4) integrate PAM session video references for investigations; (5) align with safety permit-to-work where applicable.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
(index=win_sec EventCode=4624 Logon_Type=10 OR Logon_Type=2) OR sourcetype="cyberark:session" earliest=-7d
| eval host=coalesce(dest_nt_host, ComputerName, target_host)
| lookup nis2_scada_asset_inventory.csv host OUTPUT scada_role plant_site
| where scada_role IN ("HMI","ENG_WORKSTATION","SCADA_SERVER")
| join host [
    | inputlookup nis2_scada_maintenance_windows.csv
    | eval starttime=strptime(start,"%Y-%m-%d %H:%M:%S")
    | eval endtime=strptime(end,"%Y-%m-%d %H:%M:%S")
]
| where _time < starttime OR _time > endtime
| stats values(user) as users count by host plant_site
| sort - count
```
