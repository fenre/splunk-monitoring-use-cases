# 22.2.37 — NIS2 SCADA System Access Monitoring

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Interactive access to SCADA/HMI layers is high risk. This use case monitors engineering workstation logons, HMI tag writes, and remote-desktop sessions to SCADA hosts — correlating with maintenance windows and approved technician identities.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes win_sec and sourcetypes cyberark:session. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
(index=win_sec EventCode=4624 Logon_Type=10 OR Logon_Type=2) OR sourcetype="cyberark:session" earliest=-7d
| eval host=coalesce(dest_nt_host, ComputerName, target_host)
| lookup nis2_scada_asset_inventory.csv host OUTPUT scada_role plant_site
| where scada_role IN ("HMI","ENG_WORKSTATION","SCADA_SERVER")
| join host [
    | inputlookup nis2_scada_maintenance_windows.csv
    | eval starttime=strptime(start,"%Y-%m-%d %H:%M:%S")
    | eval endtime=strptime(end,"%Y-%m-%d %H:%M:%S")
]
| where _time < starttime OR _time > endtime
| stats values(user) as users count by host plant_site
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Inventory SCADA hosts with roles in `nis2_scada_asset_inventory.csv`; (2) ingest maintenance windows from plant maintenance system; (3) alert on interactive access outside windows; (4) integrate PAM session video references for investigations; (5) align with safety permit-to-work where applicable. Indexes required: `index=win_sec`; Sourcetypes: `sourcetype=cyberark:session`.

## Visualization

Table (out-of-window access), Timeline (sessions), Single value (distinct technicians), Map (plant sites). Key fields: `endtime`, `host`, `plant_site`, `starttime`, `users`.

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(a) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(a) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
