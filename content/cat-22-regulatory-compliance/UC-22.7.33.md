# UC-22.7.33 · Proxy Denies Toward Young Threat-Intel Domains

## Description

Uses threat intel first-seen dates in denied web requests for continuous monitoring of emerging C2.

## Value

Uses threat intel first-seen dates in denied web requests for continuous monitoring of emerging C2.

## Implementation

(1) Refresh intel feeds; (2) tune for CDNs; (3) escalate high-volume denies.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ES Threat Intelligence.
• Ensure the following data sources are available: `proxy` sourcetype; threat intel lookup.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Refresh intel feeds; (2) tune for CDNs; (3) escalate high-volume denies.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=proxy sourcetype=bluecoat:proxy action=BLOCK earliest=-24h
| lookup threat_intel_domain domain as dest_host OUTPUT first_seen
| eval domain_age_days=(now()-first_seen)/86400
| where domain_age_days<7
```
