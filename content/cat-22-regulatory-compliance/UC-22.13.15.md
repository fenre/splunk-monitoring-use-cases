# 22.13.15 — ESP Inbound First-Seen External Source Detection

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

We detect new external sources allowed through ESP entry rules so undocumented inbound paths are reviewed before they become normal traffic.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes network and sourcetypes pan:traffic, pan:traffic`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=network sourcetype=pan:traffic dest_zone="ESP_DMZ" action=allowed earliest=-24h
| eval ext_src=if(cidrmatch("10.0.0.0/8",src) OR cidrmatch("172.16.0.0/12",src) OR cidrmatch("192.168.0.0/16",src),null(),src)
| where isnotnull(ext_src)
| lookup esp_inbound_baseline.csv ext_src AS ext_src OUTPUT approved_vendor
| where isnull(approved_vendor)
| stats earliest(_time) as first_seen values(dest) as targets by ext_src, rule
| sort - first_seen
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```

## Implementation

(1) Seed `esp_inbound_baseline.csv` with known vendor ranges. (2) Promote reviewed sources into the baseline. (3) Create ES notable for first-seen. (4) Map rules to EAP register fields. Indexes required: `index=network`; Sourcetypes: `sourcetype=pan:traffic`, `sourcetype=pan:traffic``.

## Visualization

Table (new sources), Map (geo), Time chart (first-seen rate). Key fields: `ext_src`, `first_seen`, `rule`, `targets`.

## Compliance

- **NERC CIP** current: CIP-005-6 R1 Part 1.3 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-005-6 R1 Part 1.3.

## Known False Positives
