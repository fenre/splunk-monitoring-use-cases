# UC-22.10.7 ┬Ę Login Monitoring & Security Incident Procedures ŌĆö Brute Force to Clinical SSO

## Description

Surfaces credential stuffing and password spray against Hyperspace/SSO endpoints so incidents are detected and handled under HIPAA security incident procedures.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes ad, citrix, okta and sourcetypes OktaIM2:log`, WinEventLog:Security, WinEventLog:Security`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Baseline legitimate auth patterns during go-live; (2) integrate with ES notable or SOAR playbooks; (3) document IR steps for suspected credential compromise affecting ePHI. Indexes required: `index=ad`, `index=citrix`, `index=okta`; Sourcetypes: `sourcetype=OktaIM2:log``, `sourcetype=WinEventLog:Security`, `sourcetype=WinEventLog:Security``; Lookups: `hipaa_clinical_apps`, `hipaa_clinical_apps.csv`.

Step 2 ŌĆö Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ad sourcetype="WinEventLog:Security" EventCode="4625" earliest=-24h
| eval user=TargetUserName
| eval src=src_ip
| stats count values(EventCode) as codes dc(ComputerName) as dc_used by user, src
| where count>=15 AND dc_used>=3
| lookup hipaa_clinical_apps.csv src OUTPUT app_name
| sort - count
```
