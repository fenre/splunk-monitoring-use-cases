# 22.20.14 — CMMC Level 3 enhanced practice — threat scenario 4

**Criticality:** medium | **Difficulty:** expert | **Status:** N/A

## Description

Demonstrates contractor cybersecurity hygiene for "CMMC Level 3 enhanced practice — threat scenario 4" using CUI-scoped telemetry mapped to CMMC 2.0 Level 3 expectations.

## Value

Gives the responsible team auditor-ready evidence for CMMC compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype=WinEventLog:Security earliest=-30d EventCode=4688
| eval uc_tag="22.20.14"
| where match(Process,"(?i)procdump|mimikatz") AND cui_host="true"
| table _time, Account_Name, Process, host
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.process_name, Processes.user, Processes.dest | sort - count
```

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=wineventlog`.

## Visualization

Stacked bar, Table, Time chart, Single value. Key fields: `Account_Name`, `Process`, `host`, `uc_tag`.

## Compliance

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
