# 22.13.45 — Backup Media Integrity Test Results

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We track restore-test and checksum results for backup media containing BES configuration or security data.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes backup and sourcetypes backup:integrity_test, backup:integrity_test`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=backup sourcetype=backup:integrity_test bes_scope=1 earliest=-180d
| eval test_epoch=strptime(test_date,"%Y-%m-%d")
| eval age_days=round((now()-test_epoch)/86400,0)
| where match(result,"(?i)fail") OR age_days>120
| table media_id, test_date, result, age_days, vault
```

## Implementation

(1) Emit HEC when integrity tests complete. (2) Define maximum age without re-test. (3) Alert storage team. (4) Document failures in CIP-008 if data loss risk. Indexes required: `index=backup`; Sourcetypes: `sourcetype=backup:integrity_test`, `sourcetype=backup:integrity_test``.

## Visualization

Table (stale or failed tests), Timeline, Single value (failed). Key fields: `age_days`, `media_id`, `result`, `test_date`, `test_epoch`, `vault`.

## Compliance

- **NERC CIP** current: CIP-009-6 R2 Part 2.2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-009-6 R2 Part 2.2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
