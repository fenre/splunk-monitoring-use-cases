# UC-22.26.16 · Datatilsynet breach reporting

## Description

Tracks breach assessment milestones and Datatilsynet notification timestamps for supervisory evidence.

## Value

Tracks breach assessment milestones and Datatilsynet notification timestamps for supervisory evidence.

## Implementation

(1) Maintain supporting lookups (classified hosts, sectors, transfers) aligned to Norwegian regulatory language. (2) Schedule searches with Europe/Oslo time semantics for reporting cutoffs. (3) Route ES notables to the correct national security, OT, or privacy queues. (4) Quarterly false-positive tuning with HR, operations, and legal stakeholders.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742), Splunk Add-on for Unix and Linux (833), Splunk OT Security Add-on (5151), Splunk ITSI (1841), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: Windows Security Event Logs, network infrastructure logs, OT/ICS system logs, power grid SCADA, petroleum control systems, government system audit trails, identity management systems, security incident reports.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain supporting lookups (classified hosts, sectors, transfers) aligned to Norwegian regulatory language. (2) Schedule searches with Europe/Oslo time semantics for reporting cutoffs. (3) Route ES notables to the correct national security, OT, or privacy queues. (4) Quarterly false-positive tuning with HR, operations, and legal stakeholders.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="breach:ticket" jurisdiction="NO" earliest=-180d
| eval confirmed=strptime(confirmed_at,"%Y-%m-%dT%H:%M:%SZ")
| eval filed=strptime(datatilsynet_filed_at,"%Y-%m-%dT%H:%M:%SZ")
| eval hours=round((filed-confirmed)/3600,2)
| eval breach_sla=if(isnull(filed) OR hours>72,1,0)
| table breach_id, confirmed_at, datatilsynet_filed_at, hours, breach_sla
```
