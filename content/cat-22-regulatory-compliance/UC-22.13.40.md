# UC-22.13.40 · Incident Response Timeline Milestone Tracking

## Description

We track detection, containment, eradication, and recovery timestamps from ITSM to evidence structured incident response progression.

## Value

We track detection, containment, eradication, and recovery timestamps from ITSM to evidence structured incident response progression.

## Implementation

(1) Require milestone fields on CIP-tagged incidents. (2) Validate timestamps in UTC. (3) Dashboard for leadership. (4) Export for after-action reviews.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=itsm` `sourcetype=incident:milestone` or ServiceNow custom sourcetype.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require milestone fields on CIP-tagged incidents. (2) Validate timestamps in UTC. (3) Dashboard for leadership. (4) Export for after-action reviews.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype=incident:milestone category="CIP008" earliest=-90d
| eval det=strptime(detected_at,"%Y-%m-%d %H:%M:%S")
| eval con=strptime(contained_at,"%Y-%m-%d %H:%M:%S")
| eval rec=strptime(recovered_at,"%Y-%m-%d %H:%M:%S")
| eval contain_h=round((con-det)/3600,2)
| eval recover_h=round((rec-con)/3600,2)
| table incident_id, detected_at, contained_at, recovered_at, contain_h, recover_h
```
