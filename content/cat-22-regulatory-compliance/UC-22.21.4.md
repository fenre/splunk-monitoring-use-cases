# UC-22.21.4 · Model Performance Degradation Detection (Art. 12)

## Description

Detects sudden drops in accuracy/auroc vs baseline to support operational logging and risk management for high-risk AI performance stability.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_metrics and sourcetypes ai:model:kpi, ai:model:kpi`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ingest batch evaluation KPIs hourly; (2) tune -0.02 threshold per model risk file; (3) create ITSI KPI thresholding; (4) document rollback criteria in QMS. Indexes required: `index=ai_metrics`; Sourcetypes: `sourcetype=ai:model:kpi`, `sourcetype=ai:model:kpi``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Machine Learning Toolkit (2890), Splunk ITSI (1841).
• Ensure the following data sources are available: `index=ai_metrics` `sourcetype=ai:model:kpi` — model_id, metric_name, metric_value, window.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest batch evaluation KPIs hourly; (2) tune -0.02 threshold per model risk file; (3) create ITSI KPI thresholding; (4) document rollback criteria in QMS.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ai_metrics sourcetype="ai:model:kpi" metric_name="accuracy" earliest=-7d
| timechart span=1h perc95(metric_value) as p95_acc by model_id
| untable _time model_id p95_acc
| eventstats median(p95_acc) as baseline by model_id
| eval delta=p95_acc-baseline
| where delta < -0.02
| table _time, model_id, p95_acc, baseline, delta
```

**Pipeline walkthrough**

• Scopes the data: index=ai_metrics, sourcetype="ai:model:kpi", time bounds. Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
• `timechart` plots the metric over time using **span=1h** buckets with a separate series **by model_id** — ideal for trending and alerting on this use case.

• `eventstats` rolls up events into metrics; results are split **by model_id** so each row reflects one combination of those dimensions (useful for per-host, per-user, or per-entity comparisons for this use case).
• `eval` defines or adjusts **delta** — applying business logic to classify, threshold, or derive compliance-relevant indicators.
• Filters the current rows with `where delta < -0.02` — typically the threshold or rule expression for this monitoring goal.

Enable Data Model Acceleration (and metric indexes for `mstats`) for the models or datasets referenced above; otherwise `tstats`/`mstats` may return no results from summaries.
