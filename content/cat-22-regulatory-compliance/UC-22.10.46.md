# UC-22.10.46 · HHS Secretary Notification — 500+ Individuals Threshold Watch

## Description

Rolling count of affected individuals per incident versus the 500-person statutory threshold for timely HHS portal submission.

## Value

Rolling count of affected individuals per incident versus the 500-person statutory threshold for timely HHS portal submission.

## Implementation

(1) Dedupe subjects carefully per legal guidance; (2) separate incidents vs linked incidents; (3) alert at 450 rolling to create runway.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / HEC.
• Ensure the following data sources are available: `index=privacy` `sourcetype=hipaa:breach_subject` (`incident_id`, `subject_token`, `included_in_count`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Dedupe subjects carefully per legal guidance; (2) separate incidents vs linked incidents; (3) alert at 450 rolling to create runway.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="hipaa:breach_subject" earliest=-365d
| where included_in_count="true"
| stats dc(subject_token) as affected by incident_id
| eval hhs_required=if(affected>=500,"Yes","No")
| join incident_id [
    search index=privacy sourcetype="hipaa:hhs_filing" earliest=-365d
    | stats latest(filed_dttm) as hhs_filed by incident_id ]
| where hhs_required="Yes" AND isnull(hhs_filed)
| table incident_id, affected, hhs_required
```
