# 22.15.14 — Weak OTP or Missing MFA Step for OT VPN

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Identifies VPN authentications that skip second factor or reuse legacy token methods for paths into OT jump zones.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes vpn and sourcetypes pan:globalprotect. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=vpn sourcetype="pan:globalprotect" earliest=-7d
| eval mfa=coalesce(factor_type, auth_method, mfa_mechanism)
| where stage="login" AND (isnull(mfa) OR mfa="none" OR mfa="password-only")
| lookup ot_vpn_profiles.csv portal OUTPUT requires_mfa
| where requires_mfa="true"
| stats count by user, portal, public_ip, host
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.user, Authentication.dest | sort - count
```

## Implementation

(1) Map portals to MFA policy; (2) upgrade GP app versions enforcing MFA; (3) block legacy profiles; (4) weekly exception report to IAM. Indexes required: `index=vpn`; Sourcetypes: `sourcetype=pan:globalprotect`; Lookups: `ot_vpn_profiles`, `ot_vpn_profiles.csv`.

## Visualization

Table, Single value (violations), Time chart. Key fields: `host`, `mfa`, `portal`, `public_ip`, `user`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 1.5 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 1.5.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
