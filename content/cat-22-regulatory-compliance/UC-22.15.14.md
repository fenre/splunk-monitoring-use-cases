# UC-22.15.14 · Weak OTP or Missing MFA Step for OT VPN

## Description

Identifies VPN authentications that skip second factor or reuse legacy token methods for paths into OT jump zones.

## Value

Identifies VPN authentications that skip second factor or reuse legacy token methods for paths into OT jump zones.

## Implementation

(1) Map portals to MFA policy; (2) upgrade GP app versions enforcing MFA; (3) block legacy profiles; (4) weekly exception report to IAM.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757).
• Ensure the following data sources are available: `pan:globalprotect` or `pan:auth`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map portals to MFA policy; (2) upgrade GP app versions enforcing MFA; (3) block legacy profiles; (4) weekly exception report to IAM.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="pan:globalprotect" earliest=-7d
| eval mfa=coalesce(factor_type, auth_method, mfa_mechanism)
| where stage="login" AND (isnull(mfa) OR mfa="none" OR mfa="password-only")
| lookup ot_vpn_profiles.csv portal OUTPUT requires_mfa
| where requires_mfa="true"
| stats count by user, portal, public_ip, host
| sort - count
```
