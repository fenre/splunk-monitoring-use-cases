# UC-22.28.18 · Outsourcing management for banks and insurers

## Description

Demonstrates alignment with BAIT/KAIT expectations for outsourcing management for banks and insurers using auditable Splunk evidence.

## Value

Gives the responsible team auditor-ready evidence for BAIT/KAIT compliance by automating regulatory compliance evidence using indexes snow and sourcetypes snow:chg. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners. Indexes required: `index=snow`; Sourcetypes: `sourcetype=snow:chg`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742), Splunk OT Security Add-on (5151), Splunk ITSI (1841), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: Windows Security Event Logs, critical infrastructure system logs, BSI IT-Grundschutz compliance scans, SIEM correlation events, vulnerability scan outputs, change management records, incident response logs, security audit evidence.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=snow sourcetype="snow:chg" category="Outsourcing" earliest=-30d
| stats values(state) as st by number, vendor
| where mvcount(mvfilter(match(st,"Closed"))) = 0 AND now() > relative_time(_time,"+30d@d")
| table number, vendor, st
```
