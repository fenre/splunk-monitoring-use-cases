# UC-22.13.21 · Vendor Interactive Remote Access — Encryption and Session Recording Evidence

## Description

We verify vendor interactive remote access uses approved encryption (for example TLS versions on VPN or SSH) and that PAM session recording identifiers exist for the same sessions.

## Value

We verify vendor interactive remote access uses approved encryption (for example TLS versions on VPN or SSH) and that PAM session recording identifiers exist for the same sessions.

## Implementation

(1) Map GlobalProtect or SSL VPN field names to `tls_version`. (2) Run a companion saved search on `index=pam` for vendor sessions missing `recording_id` in `cyberark:session_recording`. (3) Alert on weak TLS or missing recording for vendor users. (4) Document compensating controls for legacy jump hosts.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for CyberArk (4295), Splunk Add-on for Palo Alto Networks (2757).
• Ensure the following data sources are available: `index=network` `sourcetype=pan:globalprotect` or SSL VPN logs, `index=pam` `sourcetype=cyberark:ptalog`, `sourcetype=cyberark:session_recording`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map GlobalProtect or SSL VPN field names to `tls_version`. (2) Run a companion saved search on `index=pam` for vendor sessions missing `recording_id` in `cyberark:session_recording`. (3) Alert on weak TLS or missing recording for vendor users. (4) Document compensating controls for legacy jump hosts.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype=pan:globalprotect earliest=-24h
| eval tls_version=coalesce(tls_version,ssl_tls_version)
| eval tls_ok=if(match(tls_version,"TLSv1\.[23]"),1,0)
| where tls_ok=0 AND isnotnull(user)
| stats count by user, public_ip, tls_version, gateway
| sort - count
```
