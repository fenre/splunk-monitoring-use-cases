# 22.13.21 — Vendor Interactive Remote Access — Encryption and Session Recording Evidence

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We verify vendor interactive remote access uses approved encryption (for example TLS versions on VPN or SSH) and that PAM session recording identifiers exist for the same sessions.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes network, pam and sourcetypes cyberark:ptalog`, cyberark:session_recording`, pan:globalprotect. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=network sourcetype=pan:globalprotect earliest=-24h
| eval tls_version=coalesce(tls_version,ssl_tls_version)
| eval tls_ok=if(match(tls_version,"TLSv1\.[23]"),1,0)
| where tls_ok=0 AND isnotnull(user)
| stats count by user, public_ip, tls_version, gateway
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```

## Implementation

(1) Map GlobalProtect or SSL VPN field names to `tls_version`. (2) Run a companion saved search on `index=pam` for vendor sessions missing `recording_id` in `cyberark:session_recording`. (3) Alert on weak TLS or missing recording for vendor users. (4) Document compensating controls for legacy jump hosts. Sourcetypes: `sourcetype=cyberark:ptalog``, `sourcetype=cyberark:session_recording``, `sourcetype=pan:globalprotect`, `sourcetype=pan:globalprotect``.

## Visualization

Table (encryption and recording gaps), Timeline, Single value (violation count). Key fields: `gateway`, `public_ip`, `tls_ok`, `tls_version`, `user`.

## Compliance

- **NERC CIP** current: CIP-005-6 R3 Part 3.2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-005-6 R3 Part 3.2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
