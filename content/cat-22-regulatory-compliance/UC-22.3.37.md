# UC-22.3.37 · DORA Exit Strategy — Runbook Step Completion and Sign-Off SLA for Critical Providers

## Description

Exit strategies are operational documents. Monitoring partial runbook executions, missing sign-offs, and tabletop drill gaps proves the organization treats exit readiness as a lifecycle control, not a one-time PDF.

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype="exit:runbook_step" earliest=-400d
| eval due=strptime(due_date,"%Y-%m-%d"), done=strptime(completed_date,"%Y-%m-%d")
| where isnull(completed_by) AND now()>due
| stats count as overdue_steps, values(step_id) as steps by provider_id
| sort - overdue_steps
```
