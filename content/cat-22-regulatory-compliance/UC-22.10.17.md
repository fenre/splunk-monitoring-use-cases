# UC-22.10.17 · Audit Controls — High-Volume ePHI Read Baseline & Anomaly

## Description

Uses statistical baselines on audited ePHI reads to detect abnormal access volumes indicative of snooping or compromised credentials while proving audit control operation.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes epic and sourcetypes epic:audit, epic:audit`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Use `anomalydetection` command alternatively for streaming; (2) exclude legitimate roles via `hipaa_role_exclusions.csv`; (3) route to privacy investigations queue. Indexes required: `index=epic`; Sourcetypes: `sourcetype=epic:audit`, `sourcetype=epic:audit``.

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=epic sourcetype="epic:audit" AccessType="PatientChart" earliest=-90d
| bin _time span=1d
| stats dc(PAT_ID) as daily_patients by USER_ID, _time
| eventstats median(daily_patients) as med by USER_ID
| eval z=(daily_patients-med)/sqrt(med+0.001)
| where z>4 AND daily_patients>30
| sort - z
```
