# 22.15.20 — HMI Session Lock Bypass — Long Idle Operator Consoles

**Criticality:** medium | **Difficulty:** intermediate | **Status:** N/A

## Description

Measures continuous HMI operator sessions without screen-lock events to evidence session lock policy effectiveness.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes hmi_audit and sourcetypes hmi:session, winlogon. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=hmi_audit (sourcetype="winlogon" OR sourcetype="hmi:session") earliest=-7d
| eval action=coalesce(EventCode, event_type)
| transaction user host maxspan=8h maxpause=15m
| eval duration_min=round(duration/60,1)
| where duration_min>240 AND mvcount(action)=1
| table starttime, endtime, user, host, duration_min
```

## Implementation

(1) Enforce GPO screen lock on HMI OS; (2) reduce `maxpause` per plant standard; (3) alert on transactions > policy; (4) physical walk-through for exceptions. Indexes required: `index=hmi_audit`; Sourcetypes: `sourcetype=hmi:session`, `sourcetype=winlogon`.

## Visualization

Timeline, Table, Single value (long sessions). Key fields: `action`, `duration_min`, `endtime`, `host`, `starttime`, `user`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 2.5 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 2.5.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
