# UC-22.15.20 · HMI Session Lock Bypass — Long Idle Operator Consoles

## Description

Measures continuous HMI operator sessions without screen-lock events to evidence session lock policy effectiveness.

## Value

Measures continuous HMI operator sessions without screen-lock events to evidence session lock policy effectiveness.

## Implementation

(1) Enforce GPO screen lock on HMI OS; (2) reduce `maxpause` per plant standard; (3) alert on transactions > policy; (4) physical walk-through for exceptions.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Windows TA or HMI vendor audit forwarder.
• Ensure the following data sources are available: `index=hmi_audit` `sourcetype="winlogon"` on HMI thin clients.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enforce GPO screen lock on HMI OS; (2) reduce `maxpause` per plant standard; (3) alert on transactions > policy; (4) physical walk-through for exceptions.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hmi_audit (sourcetype="winlogon" OR sourcetype="hmi:session") earliest=-7d
| eval action=coalesce(EventCode, event_type)
| transaction user host maxspan=8h maxpause=15m
| eval duration_min=round(duration/60,1)
| where duration_min>240 AND mvcount(action)=1
| table starttime, endtime, user, host, duration_min
```
