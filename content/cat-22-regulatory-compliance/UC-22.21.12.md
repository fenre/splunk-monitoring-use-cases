# UC-22.21.12 · Escalation to Human Review Queue Latency (Art. 14)

## Description

Tracks SLA from model_low_confidence event to human_assigned for timely human review per oversight procedures.

## Value

Tracks SLA from model_low_confidence event to human_assigned for timely human review per oversight procedures.

## Implementation

(1) Unify timestamps to UTC; (2) set SLA based on risk class; (3) ITSI threshold on median sla_sec; (4) playbook for reassignment.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI (1841), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=ai_hil` — event=low_confidence | event=human_assigned — decision_id.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Unify timestamps to UTC; (2) set SLA based on risk class; (3) ITSI threshold on median sla_sec; (4) playbook for reassignment.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ai_hil sourcetype="ai:human:queue" earliest=-7d (event="low_confidence" OR event="human_assigned")
| eval t=if(event="low_confidence",_time,null())
| eval t2=if(event="human_assigned",_time,null())
| stats min(t) as t_lc, min(t2) as t_ha by decision_id
| eval sla_sec=t_ha-t_lc
| where isnotnull(t_lc) AND (isnull(t_ha) OR sla_sec>900)
| eval breach=if(isnull(t_ha),"unassigned",if(sla_sec>900,"late","ok"))
| table decision_id, t_lc, t_ha, sla_sec, breach
```
