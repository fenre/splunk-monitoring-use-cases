# 22.34.5 — System hardening compliance within the SWIFT secure zone

**Criticality:** medium | **Difficulty:** intermediate | **Status:** N/A

## Description

Produces SWIFT Customer Security Programme evidence for system hardening compliance within the swift secure zone ahead of independent assessment.

## Value

Gives the responsible team auditor-ready evidence for SWIFT CSP compliance by automating regulatory compliance evidence using indexes os, wineventlog and sourcetypes WinEventLog:Security, package. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=os sourcetype=package OR index=wineventlog sourcetype="WinEventLog:Security" host="SWIFT-*" earliest=-24h
| stats latest(version) as ver by package, host
| lookup swift_hardening_baseline.csv package OUTPUT min_version
| where ver<min_version
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.dest | sort - count
```

## Implementation

(1) Segment SWIFT CSP evidence into dedicated indexes with strict role-based access. (2) Join operators to HR screening and vendor risk lookups refreshed weekly. (3) Validate MFA and PAM sourcetypes are CIM-mapped for Authentication models. (4) Retain evidence for at least 13 months aligned to attestation cycles. Indexes required: `index=os`, `index=wineventlog`.

## Visualization

Table (operators), Single value (sessions), Bar chart (CSCF controls). Key fields: `host`, `package`, `ver`.

## Compliance

- **SWIFT CSP** CSCF v2025: 1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
