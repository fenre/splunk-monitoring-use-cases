# UC-22.1.34 · GDPR DPIA Supervisory Authority Consultation Tracking

## Description

Prior consultation with the supervisory authority is required when a DPIA indicates high residual risk unless mitigations reduce it. This use case tracks consultation requests, authority references, response deadlines, and conditions imposed — preventing processing start dates from preceding regulatory clearance.

## Value

Gives the responsible team auditor-ready evidence for GDPR compliance by automating regulatory compliance evidence using indexes itsm. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain consultation register with planned start dates; (2) integrate legal team updates via ServiceNow or CSV upload; (3) alert when response is overdue or processing is scheduled without decision; (4) archive authority letters as external evidence with links in CSV; (5) review quarterly with DPO. Indexes required: `index=itsm`; Lookups: `gdpr_dpia_consultation_register`, `gdpr_dpia_consultation_register.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup gdpr_dpia_consultation_register.csv
| eval consultation_opened=strptime(request_date,"%Y-%m-%d")
| eval authority_due=strptime(authority_response_due,"%Y-%m-%d")
| eval processing_start=strptime(planned_processing_start,"%Y-%m-%d")
| eval overdue_response=if(now()>authority_due AND isnull(authority_decision_date),1,0)
| eval start_before_clearance=if(processing_start < strptime(authority_decision_date,"%Y-%m-%d") AND isnull(authority_decision_date),1,0)
| where overdue_response=1 OR start_before_clearance=1 OR authority_decision="prohibited"
| table processing_activity, dpa_reference, request_date, authority_response_due, authority_decision, planned_processing_start
```
