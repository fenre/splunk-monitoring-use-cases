# UC-22.14.61 · Information Exchange Agreements — Data Share Volume Anomalies

## Description

Monitors API traffic to partner enclaves for CA-3 information exchange agreement compliance.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes cloud and sourcetypes aws:apigateway, nginx:plus:access`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Tag `partner_id` at API GW stage; (2) baseline weekly seasonality; (3) alert on spikes; (4) document agreement limits. Indexes required: `index=cloud`; Sourcetypes: `sourcetype=aws:apigateway`, `sourcetype=nginx:plus:access``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for AWS (Splunkbase 1876) API Gateway, or reverse proxy logs.
• Ensure the following data sources are available: `aws:apigateway` or `sourcetype=nginx:plus:access` with partner route prefixes.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag `partner_id` at API GW stage; (2) baseline weekly seasonality; (3) alert on spikes; (4) document agreement limits.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cloud sourcetype=aws:apigateway earliest=-24h resourcePath="/partner/*"
| bin _time span=1h
| stats sum(bytes) as bytes by _time, partner_id
| eventstats median(bytes) as med by partner_id
| eval spike=if(bytes > med*5,1,0)
| where spike=1
```
