# 22.21.24 — User Feedback Collection Pipeline Health (Art. 72)

**Criticality:** medium | **Difficulty:** beginner | **Status:** N/A

## Description

Monitors volume and error rate of in-app feedback submissions for post-market user feedback collection integrity.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes app and sourcetypes app:feedback, app:feedback`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=app sourcetype="app:feedback" earliest=-7d
| eval ok=if(http_status>=200 AND http_status<300,1,0)
| timechart span=1h sum(ok) as ok_ct, count as total
| eval ok_rate=round(100*ok_ct/total,2)
| where ok_rate < 95
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Web.Web by Web.status, Web.http_method, Web.dest span=1h | sort - count
```

## Implementation

(1) Add HEC token per environment; (2) dedupe by feedback_id; (3) alert drops >20% vs baseline; (4) map to product owner. Indexes required: `index=app`; Sourcetypes: `sourcetype=app:feedback`, `sourcetype=app:feedback``.

## Visualization

Time chart (ok_rate), Single value (7d avg ok_rate), Table (hours below 95%)

## Compliance

- **EU AI Act** Regulation (EU) 2024/1689: Art.13 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.13.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
