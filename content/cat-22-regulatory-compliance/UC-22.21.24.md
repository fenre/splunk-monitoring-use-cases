# UC-22.21.24 · User Feedback Collection Pipeline Health (Art. 72)

## Description

Monitors volume and error rate of in-app feedback submissions for post-market user feedback collection integrity.

## Value

Monitors volume and error rate of in-app feedback submissions for post-market user feedback collection integrity.

## Implementation

(1) Add HEC token per environment; (2) dedupe by feedback_id; (3) alert drops >20% vs baseline; (4) map to product owner.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk ITSI (1841), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: `index=app` `sourcetype=app:feedback` — http_status, session_id.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Add HEC token per environment; (2) dedupe by feedback_id; (3) alert drops >20% vs baseline; (4) map to product owner.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=app sourcetype="app:feedback" earliest=-7d
| eval ok=if(http_status>=200 AND http_status<300,1,0)
| timechart span=1h sum(ok) as ok_ct, count as total
| eval ok_rate=round(100*ok_ct/total,2)
| where ok_rate < 95
```
