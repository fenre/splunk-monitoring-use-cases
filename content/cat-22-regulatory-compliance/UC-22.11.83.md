# UC-22.11.83 · Security Awareness Training Completion for Personnel with CDE Access

## Description

Measures annual or periodic training completion for users who can access the cardholder environment so the information security awareness program is evidenced by actual completion rates, not policy alone.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes hr and sourcetypes knowbe4:training. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Sync HR email with IAM quarterly. (2) Auto-remind via SOAR when `pct<100`. (3) Store annual completion CSV for PCI. (4) Exclude contractors with separate policy if applicable via lookup flag. Indexes required: `index=hr`; Sourcetypes: `sourcetype=knowbe4:training`.

Step 1 — Configure data collection
(1) Sync HR email with IAM quarterly. (2) Auto-remind via SOAR when `pct<100`. (3) Store annual completion CSV for PCI. (4) Exclude contractors with separate policy if applicable via lookup flag.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr sourcetype="knowbe4:training" earliest=-400d
| lookup pci_cde_users.csv email AS user OUTPUT has_cde_access
| where has_cde_access="true"
| where match(course,"(?i)PCI|security\\s*awareness|phishing)")
| eval completed=if(isnotnull(completed_date) OR match(status,"(?i)completed"),1,0)
| stats sum(completed) as done, count as assigned by user
| eval pct=if(assigned>0, round(100*done/assigned,2), null())
| where pct<100
| sort pct, user
```
