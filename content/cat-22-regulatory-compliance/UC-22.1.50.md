# UC-22.1.50 · GDPR Transparency Notice Completeness and Version Alignment

## Description

Transparency requires that data subjects receive fair processing information at collection. This use case monitors CMS and web deployments for privacy notice URL availability, version tags, and broken links — catching releases that ship without updated notices after processing changes.

## Value

Transparency requires that data subjects receive fair processing information at collection. This use case monitors CMS and web deployments for privacy notice URL availability, version tags, and broken links — catching releases that ship without updated notices after processing changes.

## Implementation

(1) Register each public site with expected notice URL and embedded version string; (2) run synthetic checks every hour from multiple regions; (3) alert privacy counsel when checks fail or version mismatches after deploy; (4) gate production deploys on successful synthetic in mature pipelines; (5) archive passing runs as Art. 12 evidence.

## Detailed Implementation

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Register each public site with expected notice URL and embedded version string; (2) run synthetic checks every hour from multiple regions; (3) alert privacy counsel when checks fail or version mismatches after deploy; (4) gate production deploys on successful synthetic in mature pipelines; (5) archive passing runs as Art. 12 evidence.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=synthetic check_type="http" earliest=-24h
| lookup gdpr_notice_registry.csv site_id OUTPUT notice_url, required_version
| eval ok=if(http_status=200 AND match(_raw, required_version),1,0)
| where ok=0
| stats latest(http_status) as http_status latest(_time) as last_fail by site_id notice_url
| join site_id [
    search index=cicd sourcetype="web:deploy" earliest=-7d
    | stats latest(release_tag) as release_tag by site_id
]
| table site_id, notice_url, http_status, release_tag, last_fail
```
