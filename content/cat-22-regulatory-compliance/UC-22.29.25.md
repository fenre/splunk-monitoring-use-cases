# UC-22.29.25 · Separate consent for sensitive personal information

## Description

Provides APAC privacy supervisory evidence for separate consent for sensitive personal information aligned with PIPL; PDPA.

## Value

Gives the responsible team auditor-ready evidence for PIPL compliance by automating regulatory compliance evidence using indexes cmp and sourcetypes consent:log. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain `apac_transfer_register.csv` and consent or DPIA lookups referenced in runbooks. (2) Map technical identifiers to records of processing activities with legal review. (3) Restrict indexes to regional privacy roles with least privilege. (4) Quarterly DPIA and consent linkage review with local counsel. Indexes required: `index=cmp`; Sourcetypes: `sourcetype=consent:log`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cmp sourcetype="consent:log" jurisdiction="CN" earliest=-30d
| stats latest(consent_type) as ct by subject_id, purpose
| where match(purpose,"(?i)biometric|sensitive") AND ct!="explicit"
```
