# UC-22.7.28 · Platform Integrity — sudoers or nsswitch Changes on Linux

## Description

Monitors file integrity events on authentication stack files for Linux platform security.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes os and sourcetypes linux:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Deploy auditd rules; (2) whitelist automation; (3) alert within minutes; (4) map to change records. Indexes required: `index=os`; Sourcetypes: `sourcetype=linux:audit`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Unix and Linux (Splunkbase 833).
• Ensure the following data sources are available: `linux:audit`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Deploy auditd rules; (2) whitelist automation; (3) alert within minutes; (4) map to change records.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os sourcetype=linux:audit type=PATH earliest=-7d
| where match(name,"(?i)/etc/(sudoers|nsswitch.conf|passwd)")
| table _time, name, exe, uid
```
