# UC-22.8.30 · SOC 2 PI1 — Processing Completeness Validation Across Multi-Stage Pipelines

## Description

PI1 expects processing to be complete, valid, accurate, timely, and authorized. Reconciling record counts and checksums between ingest, transform, and load stages catches silent drops that would undermine financial or subscriber reporting integrity.

## Value

Gives the responsible team auditor-ready evidence for SOC 2 compliance by automating regulatory compliance evidence using indexes dataops and sourcetypes pipeline:reconcile. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Emit reconcile events at stage boundaries; (2) allow known shrinkage rules via `pipeline_tolerance.csv`; (3) alert data owners on mismatch; (4) retain batch_id for auditor replay; (5) map `pipeline_id` to PI1 subservice in documentation. Indexes required: `index=dataops`; Sourcetypes: `sourcetype=pipeline:reconcile`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector from orchestration (Airflow, dbt, mainframe bridge).
• Ensure the following data sources are available: `index=dataops` `sourcetype="pipeline:reconcile"` (pipeline_id, stage, batch_id, row_count, checksum, _time).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Emit reconcile events at stage boundaries; (2) allow known shrinkage rules via `pipeline_tolerance.csv`; (3) alert data owners on mismatch; (4) retain batch_id for auditor replay; (5) map `pipeline_id` to PI1 subservice in documentation.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=dataops sourcetype="pipeline:reconcile" earliest=-7d
| stats latest(eval(if(stage="ingest",row_count,null()))) as ingest_rows,
        latest(eval(if(stage="load",row_count,null()))) as load_rows,
        latest(eval(if(stage="ingest",checksum,null()))) as c_in,
        latest(eval(if(stage="load",checksum,null()))) as c_out
  by batch_id, pipeline_id
| eval row_match=if(ingest_rows=load_rows,1,0), chk_match=if(c_in=c_out,1,0)
| where row_match=0 OR chk_match=0
| table pipeline_id, batch_id, ingest_rows, load_rows, c_in, c_out
```
