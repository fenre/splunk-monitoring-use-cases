# UC-22.1.40 · GDPR Binding Corporate Rules (BCR) Evidence and Intra-Group Transfer Monitoring

## Description

BCRs require demonstrable enforcement of group-wide policies including transfers between entities. This use case maps intra-group data flows to BCR-covered entities, verifies annual certification of policy acknowledgement, and detects flows involving non-BCR entities.

## Value

BCRs require demonstrable enforcement of group-wide policies including transfers between entities. This use case maps intra-group data flows to BCR-covered entities, verifies annual certification of policy acknowledgement, and detects flows involving non-BCR entities.

## Implementation

(1) Assign `entity_id` to sites and cloud subscriptions in `gdpr_bcr_entity_register.csv`; (2) tag internal IP ranges and cloud accounts; (3) alert when significant volumes flow to entities not listed as BCR members; (4) join HR training data for annual BCR attestation; (5) include in group privacy committee reporting.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| tstats `summariesonly` sum(All_Traffic.bytes_out) as bytes_out
  from datamodel=Network_Traffic.All_Traffic
  where earliest=-30d match(All_Traffic.src_category,"(?i)internal")
  by All_Traffic.src All_Traffic.dest
| rename All_Traffic.* as *
| lookup gdpr_bcr_entity_register.csv entity_id AS src OUTPUT bcr_covered, entity_name
| lookup gdpr_bcr_entity_register.csv entity_id AS dest OUTPUT bcr_covered AS dest_bcr, entity_name AS dest_name
| eval bcr_gap=if(bcr_covered="true" AND dest_bcr!="true", "FLOW_TO_NON_BCR", "review")
| where bcr_gap="FLOW_TO_NON_BCR"
| table src, dest, entity_name, dest_name, bytes_out, bcr_gap
```
