# UC-22.11.86 · Third-Party Service Provider Compliance Scorecard Ingest

## Description

Centralizes SOC reports and PCI AOC freshness dates for processors and cloud providers that affect account data so service-provider due diligence and monitoring obligations are continuously visible.

## Value

Centralizes SOC reports and PCI AOC freshness dates for processors and cloud providers that affect account data so service-provider due diligence and monitoring obligations are continuously visible.

## Implementation

(1) Automate monthly vendor portal scrape to HEC. (2) Map vendors to in-scope services in lookup. (3) Legal review when `days_to_pci<30`. (4) Attach artifacts in document store with Splunk link.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vendor sourcetype="vendor:compliance_scorecard" earliest=-1d
| eval pci_exp_epoch=strptime(pci_aoc_expiry,"%Y-%m-%d")
| eval days_to_pci=round((pci_exp_epoch-now())/86400,1)
| where days_to_pci<60 OR open_findings>0
| table vendor, pci_aoc_expiry, days_to_pci, soc2_expiry, open_findings
| sort days_to_pci
```
