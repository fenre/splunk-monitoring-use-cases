# UC-22.22.3 · Dynamic Linking Verification for Payee/Amount (RTS Art. 5)

## Description

Monitors PSD2 strong customer authentication telemetry against documented baselines to evidence RTS compliance and proportionate exemption usage.

## Value

Monitors PSD2 strong customer authentication telemetry against documented baselines to evidence RTS compliance and proportionate exemption usage.

## Implementation

(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: `index=payments` `sourcetype=psd2:sca` — aspsp_id, psu_id, challenge, sca_exemption, outcome, authentication_method.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=payments sourcetype="psd2:sca" earliest=-24h
| eval sca_dim="dynamic_link"
| eval challenge_flag=if(challenge="true" OR challenge="1",1,0)
| stats count as authn, sum(eval(outcome="success")) as ok, sum(challenge_flag) as ch, dc(psu_id) as psu by aspsp_id, sca_dim
| lookup psd2_sca_baseline.csv aspsp_id OUTPUT challenge_rate_baseline AS baseline
| eval challenge_rate=round(100*ch/authn,3)
| eval delta=round(challenge_rate-baseline,3)
| eval uc_id="22.22.3"
| where authn>100 AND (delta>3 OR delta<-3)
| table uc_id, aspsp_id, sca_dim, authn, challenge_rate, baseline, delta
```
