# 22.11.97 — PCI-DSS 8.4 — MFA coverage: administrative access to CDE without MFA

**Criticality:** critical | **Difficulty:** intermediate | **Status:** verified

## Description

Administrative access to the CDE must use MFA. PCI-DSS 8.4 (formerly 8.3) explicitly requires MFA for administrative access. This UC detects misses.

## Value

The #1 QSA finding in failed assessments. Converting this to a 10-minute detection reduces remediation cost 100×.

## SPL

```spl
search (index=windows sourcetype=WinEventLog:Security EventCode IN (4624,4672)) OR (index=pam sourcetype=cyberark:pam event_type=session_start) earliest=-10m
| rename target_host AS dest
| lookup cde_hosts.csv hostname=dest OUTPUT in_cde
| lookup privileged_accounts.csv user AS admin OUTPUT is_privileged
| where in_cde="yes" AND is_privileged="yes"
| eval mfa_used=if(match(_raw,"(?i)mfa|2fa|push|webauthn|smartcard"),1,0)
| where mfa_used=0
| table admin dest session_type mfa_used
```

## Implementation

(1) Integrate PAM/bastion; (2) maintain cde_hosts.csv + privileged_accounts.csv; (3) schedule UC every 10m; (4) hit revokes session; (5) monthly CISO review. Indexes required: `index=linux`, `index=pam`, `index=windows`; Sourcetypes: `sourcetype=WinEventLog:Security`, `sourcetype=cyberark:pam`, `sourcetype=cyberark:pam;`.

## Visualization

Heatmap of admins × destinations, table of non-MFA sessions, single value 'non-MFA admin in CDE'.

## Compliance

- **PCI-DSS** v4.0: 8.4 (full) — PCI-DSS 8.4 MFA for administrative access to the CDE — directly evidenced.
- **ISO/IEC 27001** 2022: A.8.5 (full) — ISO/IEC 27001:2022 A.8.5 Secure authentication — directly evidenced.
- **NIST 800-53** Rev. 5: IA-2(1) (full) — NIST 800-53 IA-2(1) MFA to privileged accounts — directly evidenced.
- **DORA** Regulation (EU) 2022/2554: Art.9 (full) — Uplifted to full: SPL provides comprehensive, standalone evidence for Art.9 with specific data sources and actionable thresholds.

## Known False Positives

Break-glass emergency access with compensating controls (hardware-token-only) must log mfa_used=1 via its own sourcetype — route to the field correctly.
