# UC-22.11.97 · PCI-DSS 8.4 — MFA coverage: administrative access to CDE without MFA

## Description

Administrative access to the CDE must use MFA. PCI-DSS 8.4 (formerly 8.3) explicitly requires MFA for administrative access. This UC detects misses.

## Value

The #1 QSA finding in failed assessments. Converting this to a 10-minute detection reduces remediation cost 100×.

## Implementation

(1) Integrate PAM/bastion; (2) maintain cde_hosts.csv + privileged_accounts.csv; (3) schedule UC every 10m; (4) hit revokes session; (5) monthly CISO review.

Step 1 — Configure data collection
(1) Integrate PAM/bastion; (2) maintain cde_hosts.csv + privileged_accounts.csv; (3) schedule UC every 10m; (4) hit revokes session; (5) monthly CISO review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
search (index=windows sourcetype=WinEventLog:Security EventCode IN (4624,4672)) OR (index=pam sourcetype=cyberark:pam event_type=session_start) earliest=-10m
| rename target_host AS dest
| lookup cde_hosts.csv hostname=dest OUTPUT in_cde
| lookup privileged_accounts.csv user AS admin OUTPUT is_privileged
| where in_cde="yes" AND is_privileged="yes"
| eval mfa_used=if(match(_raw,"(?i)mfa|2fa|push|webauthn|smartcard"),1,0)
| where mfa_used=0
| table admin dest session_type mfa_used
```
