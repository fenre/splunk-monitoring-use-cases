# 22.30.15 — Electronic payment channel security monitoring

**Criticality:** low | **Difficulty:** advanced | **Status:** N/A

## Description

Supports RBI cyber security framework supervisory examination readiness for electronic payment channel security monitoring.

## Value

Gives the responsible team auditor-ready evidence for RBI Cyber compliance by automating regulatory compliance evidence using indexes payments and sourcetypes npci:neft, upi:txn. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=payments sourcetype="npci:neft" OR sourcetype="upi:txn" earliest=-24h
| stats sum(amount) as amt by merchant, risk_flag
| where risk_flag="true" AND amt>1000000
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=payments`; Sourcetypes: `sourcetype=npci:neft`, `sourcetype=upi:txn`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones). Key fields: `amt`, `merchant`, `risk_flag`.

## Compliance

- **RBI Cyber** 2016 (as amended): Annex-A (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Annex-A.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
