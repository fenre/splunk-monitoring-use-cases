# 22.28.12 — Risk analysis per BSI methodology

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Demonstrates alignment with BSI IT-Grundschutz expectations for risk analysis per bsi methodology using auditable Splunk evidence.

## Value

Gives the responsible team auditor-ready evidence for IT-Grundschutz compliance by automating regulatory compliance evidence using indexes risk and sourcetypes bsi:bia. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=risk sourcetype="bsi:bia" earliest=-365d
| stats latest(risk_score) as rs by asset_group, threat
| where rs>75
| lookup grundschutz_ba_scope.csv asset_group OUTPUT in_scope
| where in_scope="true"
```

## CIM SPL

```spl
| tstats summariesonly=t latest(All_Changes.status) as agg_value from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - agg_value
```

## Implementation

(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners. Indexes required: `index=risk`; Sourcetypes: `sourcetype=bsi:bia`.

## Visualization

Matrix (host x module), Bar chart (severity), Table (controls). Key fields: `asset_group`, `rs`, `threat`.
