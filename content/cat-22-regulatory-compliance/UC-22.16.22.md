# 22.16.22 — PLC/RTU inventory without expected monitoring agent

**Criticality:** medium | **Difficulty:** expert | **Status:** N/A

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "PLC/RTU inventory without expected monitoring agent".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes assets, ot and sourcetypes nozomi:asset, snow:cmdb_ci_server. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot sourcetype=nozomi:asset role="PLC" earliest=-1d
| eval uc_tag="22.16.22"
| eval key=lower(mac)
| join type=left key [
    search index=assets sourcetype=snow:cmdb_ci_server earliest=-1d | eval key=lower(mac) | stats count by key | rename count as present
  ]
| where isnull(present)
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=assets`, `index=ot`.
