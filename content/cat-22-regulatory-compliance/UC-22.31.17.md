# 22.31.17 — Third-party information security assessment tracking

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Evidences APRA CPS 234 obligations for third-party information security assessment tracking using endpoint, identity, and cloud telemetry.

## Value

Gives the responsible team auditor-ready evidence for APRA CPS 234 compliance by automating regulatory compliance evidence using indexes vendor and sourcetypes cps234:third_party. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=vendor sourcetype="cps234:third_party" earliest=-180d
| stats latest(assessment_score) as s by vendor_name
| where s<75
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.process_name, Processes.user, Processes.dest | sort - count
```

## Implementation

(1) Ingest CrowdStrike policy posture via TA 5082 and map hosts to NDB and NZISM tiers. (2) Tune alerts with ACSD Essential Eight and APRA-aligned severities. (3) Coordinate privacy and security ownership for dual-reporting workflows. (4) Monthly export for privacy officer and prudential working groups. Indexes required: `index=vendor`; Sourcetypes: `sourcetype=cps234:third_party`.
