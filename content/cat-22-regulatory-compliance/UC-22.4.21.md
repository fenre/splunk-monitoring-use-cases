# UC-22.4.21 · CCPA/CPRA — Cross-Context Behavioral Advertising — Cross-Site ID Sync Pixel After GPC Signal

## Description

Global Privacy Control obligates downstream behavior for covered businesses. Detecting third-party sync pixels or bid requests carrying cross-context identifiers after a GPC-positive session start evidences broken CMP / tag enforcement for “sale/share” pathways.

## Value

Global Privacy Control obligates downstream behavior for covered businesses. Detecting third-party sync pixels or bid requests carrying cross-context identifiers after a GPC-positive session start evidences broken CMP / tag enforcement for “sale/share” pathways.

## Implementation

(1) Ensure `gpc_time` uses same clock as ad logs; (2) maintain list of `partner_id` considered cross-context; (3) integrate with tag monitoring remediation; (4) legal review for non-California visitors via geo enrichment optional; (5) document CMP version in release notes.

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=web sourcetype="cmp:gpc" gpc=1 earliest=-7d
| eval gpc_time=_time
| join type=left session_id [
    search index=web sourcetype="ads:sync" cross_context_flag=true earliest=-7d
    | fields session_id, partner_id, _time
  ]
| where _time>=gpc_time
| stats count by partner_id, session_id
| sort - count
```
