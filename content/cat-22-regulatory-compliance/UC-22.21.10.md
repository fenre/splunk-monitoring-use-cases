# 22.21.10 — Algorithm Change Documentation Linkage (Art. 13)

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Ensures each production model_version has linked change_ticket and approver for algorithm change documentation.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_governance and sourcetypes ai:deploy:gov. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_governance sourcetype="ai:deploy:gov" earliest=-90d
| stats latest(deployed_at) as deployed_at, values(approver) as approvers by model_version, change_ticket
| eval ticket_ok=if(match(change_ticket,"^(CHG|CR)-\d+"),1,0)
| where ticket_ok=0 OR mvcount(approvers)<1
| sort - deployed_at
```

## Implementation

(1) Require CI/CD to emit governance event; (2) validate ticket regex against ITSM; (3) create lookup fallback from DB Connect ITSM export; (4) quarterly audit sample. Indexes required: `index=ai_governance`; Sourcetypes: `sourcetype=ai:deploy:gov`.

## Visualization

Table (violations), Bar chart (ticket_ok rate by month), Single value (open violations)

## Compliance

- **EU AI Act** Regulation (EU) 2024/1689: Art.13 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.13.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
