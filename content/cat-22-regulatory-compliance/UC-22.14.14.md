# UC-22.14.14 · Alternate Audit Capability During Control Outages

## Description

Detects when primary logging pipelines fail and compensating syslog or packet capture taps increase, evidencing AU-15 alternate audit capability activation.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes _internal, net. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Label alternate paths consistently; (2) tune ratio for your architecture; (3) require ticket linkage when alt_ratio spikes; (4) review monthly with contingency tests (CP-10). Indexes required: `index=_internal`, `index=net`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Stream (Splunkbase 1809), Splunk Connect for Syslog.
• Ensure the following data sources are available: `_internal` pipeline errors, `index=net` tap sourcetypes.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Label alternate paths consistently; (2) tune ratio for your architecture; (3) require ticket linkage when alt_ratio spikes; (4) review monthly with contingency tests (CP-10).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_internal OR index=net earliest=-24h
| eval channel=case(match(sourcetype,"stream"),"PCAP_TAP", match(sourcetype,"syslog"),"SYSLOG_ALT",1=1,"PRIMARY")
| bin _time span=15m
| stats sum(eval(if(channel="PRIMARY",1,0))) as primary_hits,
        sum(eval(if(channel!="PRIMARY",1,0))) as alt_hits by _time, host
| eval alt_ratio=round(alt_hits/(primary_hits+alt_hits+0.001),3)
| where alt_ratio>0.3
```
