# UC-22.15.34 · Denied Exploit Attempts at OT DMZ Boundary

## Description

Aggregates blocked threats at OT firewall and IDS boundaries to prove active zone boundary protection.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_sec and sourcetypes claroty:alert, nozomi:alerts, pan:threat. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize severity scales; (2) enrich with MITRE if desired; (3) tune noisy signatures; (4) daily SOC rollup for management. Indexes required: `index=ot_sec`; Sourcetypes: `sourcetype=claroty:alert`, `sourcetype=nozomi:alerts`, `sourcetype=pan:threat`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk OT Security Add-on (5151), ES (optional).
• Ensure the following data sources are available: `pan:threat`, `nozomi:alerts`, `claroty:alert`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize severity scales; (2) enrich with MITRE if desired; (3) tune noisy signatures; (4) daily SOC rollup for management.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_sec earliest=-24h (sourcetype="pan:threat" OR sourcetype="nozomi:alerts" OR sourcetype="claroty:alert")
| eval severity=coalesce(severity, priority, level)
| where action="blocked" OR disposition="blocked" OR threat_id!="none"
| stats count by signature, src_ip, dest_zone, sourcetype
| sort - count
```
