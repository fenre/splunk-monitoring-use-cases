# UC-22.4.11 · CCPA/CPRA — Sensitive PI — Racial or Ethnic Origin Attributes in Model Training Feature Stores

## Description

Use of sensitive categories for automated decisioning is tightly constrained. Auditing feature-store schemas and ETL logs for prohibited attribute names or proxy variables supports demonstrable limits of processing for sensitive PI.

## Value

Use of sensitive categories for automated decisioning is tightly constrained. Auditing feature-store schemas and ETL logs for prohibited attribute names or proxy variables supports demonstrable limits of processing for sensitive PI.

## Implementation

(1) Expand keyword list with privacy counsel; (2) join `source_table` to approved sensitive-processing register; (3) block CI deploys on match in pre-prod mirror; (4) quarterly model governance review; (5) document lawful basis where processing is permitted.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ml sourcetype="feast:feature_view" earliest=-30d
| where match(feature_name,"(?i)race|ethnic|religion|union|orientation|immigration|citizen")
| stats dc(view_name) as views, values(feature_name) as features by source_table, created_by
| sort - views
```
