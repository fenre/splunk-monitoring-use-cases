# UC-22.6.43 · Network Segmentation — Cross-VLAN RDP Allowed by Mis-Tuned ACL

## Description

Flags RDP sessions crossing VLAN boundaries that should be blocked by segmentation policy.

## Value

Flags RDP sessions crossing VLAN boundaries that should be blocked by segmentation policy.

## Implementation

(1) Validate zone tags; (2) emergency ACL review; (3) document compensating jump hosts.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Palo Alto Networks TA.
• Ensure the following data sources are available: `pan:traffic`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Validate zone tags; (2) emergency ACL review; (3) document compensating jump hosts.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype=pan:traffic earliest=-24h app=rdp action=allow
| where src_zone!=dest_zone AND match(dest_zone,"(?i)pci|prod_db")
| stats count by src, dest, rule
```
