# 22.13.58 — BCSI Storage Location Inventory vs. Observed Disk Paths

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We compare discovered storage paths containing BCSI keywords against the approved storage location inventory.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes inventory and sourcetypes filesystem:scan, filesystem:scan`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=inventory sourcetype=filesystem:scan earliest=-7d match(path,"(?i)one-line|relay|settings|cyber")
| lookup bes_cyber_asset_inventory.csv host OUTPUT bes_system
| where isnotnull(bes_system)
| lookup bcsi_approved_locations.csv host, path_prefix OUTPUT approved
| where isnull(approved)
| stats values(path) as paths by host, bes_system
```

## Implementation

(1) Run constrained filesystem scans with legal approval. (2) Tune regex to reduce false positives. (3) Investigate unknown paths quickly. (4) Update inventory when validated. Indexes required: `index=inventory`; Sourcetypes: `sourcetype=filesystem:scan`, `sourcetype=filesystem:scan``; Lookups: `bcsi_approved_locations`, `bcsi_approved_locations.csv`, `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.
