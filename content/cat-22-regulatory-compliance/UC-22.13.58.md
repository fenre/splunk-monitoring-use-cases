# UC-22.13.58 · BCSI Storage Location Inventory vs. Observed Disk Paths

## Description

We compare discovered storage paths containing BCSI keywords against the approved storage location inventory.

## Value

We compare discovered storage paths containing BCSI keywords against the approved storage location inventory.

## Implementation

(1) Run constrained filesystem scans with legal approval. (2) Tune regex to reduce false positives. (3) Investigate unknown paths quickly. (4) Update inventory when validated.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=inventory` `sourcetype=filesystem:scan`, `bcsi_approved_locations.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Run constrained filesystem scans with legal approval. (2) Tune regex to reduce false positives. (3) Investigate unknown paths quickly. (4) Update inventory when validated.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=inventory sourcetype=filesystem:scan earliest=-7d match(path,"(?i)one-line|relay|settings|cyber")
| lookup bes_cyber_asset_inventory.csv host OUTPUT bes_system
| where isnotnull(bes_system)
| lookup bcsi_approved_locations.csv host, path_prefix OUTPUT approved
| where isnull(approved)
| stats values(path) as paths by host, bes_system
```
