# 22.34.8 — Operator screening evidence aggregation

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

Produces SWIFT Customer Security Programme evidence for operator screening evidence aggregation ahead of independent assessment.

## Value

Gives the responsible team auditor-ready evidence for SWIFT CSP compliance by automating regulatory compliance evidence using indexes hr and sourcetypes swift:operator:hr. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=hr sourcetype="swift:operator:hr" earliest=-365d
| stats latest(screening_status) as st, latest(screening_date) as sd by operator_id
| where st!="Cleared" OR sd<relative_time(now(),"-365d@d")
| table operator_id, st, sd
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Segment SWIFT CSP evidence into dedicated indexes with strict role-based access. (2) Join operators to HR screening and vendor risk lookups refreshed weekly. (3) Validate MFA and PAM sourcetypes are CIM-mapped for Authentication models. (4) Retain evidence for at least 13 months aligned to attestation cycles. Indexes required: `index=hr`.

## Visualization

Table (operators), Single value (sessions), Bar chart (CSCF controls). Key fields: `operator_id`, `sd`, `st`.

## Compliance

- **SWIFT CSP** CSCF v2025: 6.4 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 6.4.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
