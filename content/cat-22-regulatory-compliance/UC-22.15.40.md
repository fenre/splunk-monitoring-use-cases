# UC-22.15.40 · Modbus Serial Gateway Exposing Registers to Wrong TCP Subnet

## Description

Flags TCP-side Modbus requests from subnets that should not see serial-segment registers, supporting confidentiality on component channels.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_protocol and sourcetypes edge_hub_modbus. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) ACL on gateway Ethernet port; (2) VPN overlay for remote; (3) alert new src subnets; (4) document data diodes where used. Indexes required: `index=ot_protocol`; Sourcetypes: `sourcetype=edge_hub_modbus`; Lookups: `serial_gateway_allowlist`, `serial_gateway_allowlist.csv`.

(1) ACL on gateway Ethernet port; (2) VPN overlay for remote; (3) alert new src subnets; (4) document data diodes where used.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_protocol sourcetype="edge_hub_modbus" transport="tcp" earliest=-24h
| lookup serial_gateway_allowlist.csv dest_ip src_subnet OUTPUT permitted
| where permitted!="true"
| stats dc(function_code) as fcs, sum(register_count) as regs by src_ip, dest_ip
| sort - regs
```
