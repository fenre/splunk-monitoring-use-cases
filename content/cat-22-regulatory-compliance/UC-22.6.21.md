# UC-22.6.21 · Physical Perimeter — After-Hours Badge Swipes Without Matching Shift

## Description

Flags access to secured areas outside scheduled shift windows for perimeter control monitoring.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes physical and sourcetypes pacs:access. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Populate shift schedules; (2) guard tour exceptions; (3) SOC physical security watchlist. Indexes required: `index=physical`; Sourcetypes: `sourcetype=pacs:access`; Lookups: `shift_schedule`, `shift_schedule.csv`.

## Detailed Implementation

• Install and configure the required add-on or app: PACS sourcetype.
• Ensure the following data sources are available: `pacs:access`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate shift schedules; (2) guard tour exceptions; (3) SOC physical security watchlist.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype=pacs:access earliest=-30d result=granted
| lookup shift_schedule.csv badge_id day_of_week OUTPUT shift_start, shift_end
| eval tod=strftime(_time,"%H:%M")
| where tod < shift_start OR tod > shift_end
| stats count by location, badge_id
```
