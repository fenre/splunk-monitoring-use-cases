# 22.30.4 — Patch management compliance and overdue systems

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Supports MAS TRM supervisory examination readiness for patch management compliance and overdue systems.

## Value

Gives the responsible team auditor-ready evidence for MAS TRM compliance by automating regulatory compliance evidence using indexes vulns and sourcetypes tenable:sc. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=vulns sourcetype="tenable:sc" tag=mas_trm earliest=-14d
| stats max(severity_score) as sev by host
| where sev>=9
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.dest | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=vulns`; Sourcetypes: `sourcetype=tenable:sc`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones). Key fields: `host`, `sev`.

## Compliance

- **MAS TRM** 2021: §4.1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for §4.1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
