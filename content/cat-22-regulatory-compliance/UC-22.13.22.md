# 22.13.22 — Dial-Up or Serial Out-of-Band Access on ESP-Adjacent Segments

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

We detect modem, PPP, or serial console activity that could bypass documented ESP protections.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes os and sourcetypes linux:secure, linux:secure`, serial_concentrator`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=os sourcetype=linux:secure earliest=-14d (pppd OR ttyS OR ttyUSB OR "serial console")
| lookup bes_network_segments.csv host OUTPUT segment
| where segment="ESP_adjacent"
| stats count by host, process, user
| sort - count
```

## Implementation

(1) Inventory any remaining dial or OOB paths. (2) Forward concentrator logs. (3) Alert on any production hit unless maintenance window lookup matches. (4) Document removal in CIP-010. Indexes required: `index=os`; Sourcetypes: `sourcetype=linux:secure`, `sourcetype=linux:secure``, `sourcetype=serial_concentrator``.
