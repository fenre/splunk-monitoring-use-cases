# UC-22.2.21 · NIS2 Risk Analysis Evidence for Essential Entities

## Description

Essential entities must implement a comprehensive set of risk-management measures proportionate to the risks posed. This use case aggregates residual risk scores, vulnerability exposure, and control maturity for assets tagged as essential — producing continuous evidence that formal risk analysis is refreshed and acted upon, not only performed annually on paper.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes risk, tenable and sourcetypes stash, tenable:vuln. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Tag all in-scope assets with `entity_tier` in `nis2_entity_classification.csv`; (2) ensure ES risk and vuln scans cover essential services; (3) define risk thresholds with CISO sign-off; (4) alert when residual risk or critical findings breach policy; (5) export quarterly risk analysis workbook attachments from dashboard PDFs. Indexes required: `index=risk`, `index=tenable`; Sourcetypes: `sourcetype=stash`, `sourcetype=tenable:vuln`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263), Tenable Add-On for Splunk (Splunkbase 4060).
• Ensure the following data sources are available: `index=risk` `sourcetype="stash"`, `tenable:vuln`, `nis2_entity_classification.csv` (entity_tier=essential).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag all in-scope assets with `entity_tier` in `nis2_entity_classification.csv`; (2) ensure ES risk and vuln scans cover essential services; (3) define risk thresholds with CISO sign-off; (4) alert when residual risk or critical findings breach policy; (5) export quarterly risk analysis workbook attachments from dashboard PDFs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup nis2_entity_classification.csv WHERE entity_tier="essential"
| rename asset_id as risk_object
| join risk_object [
    search index=risk sourcetype="stash" earliest=-30d
    | stats latest(risk_score) as residual_risk max(risk_score) as peak_risk by risk_object
]
| join risk_object [
    search index=tenable sourcetype="tenable:vuln" severity IN ("Critical","High") earliest=-14d
    | stats dc(plugin_id) as open_critical_high by host
    | rename host as risk_object
]
| eval analysis_stale=if(residual_risk > 80 OR open_critical_high > 5, "ESCALATE", "Monitor")
| table risk_object, business_service, residual_risk, peak_risk, open_critical_high, analysis_stale
| sort - residual_risk
```
