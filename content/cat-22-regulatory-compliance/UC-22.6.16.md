# UC-22.6.16 · Compliance with Policies — Splunk Search Head Knowledge Object Violations

## Description

Detects saved searches referencing non-approved indexes or missing required index= constraint for enforced search policies.

## Value

Detects saved searches referencing non-approved indexes or missing required index= constraint for enforced search policies.

## Implementation

(1) Define approved index list separately; (2) exclude system apps; (3) quarterly SoA evidence for tooling policies.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise (`_audit`, REST).
• Ensure the following data sources are available: | rest /services/saved/searches.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Define approved index list separately; (2) exclude system apps; (3) quarterly SoA evidence for tooling policies.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| rest /services/saved/searches splunk_server=local count=0
| search disabled=0 is_scheduled=1
| where !match(search,"(?i)index\s*=\s*") OR match(search,"(?i)index\s*=\s*all")
| table title, eai:acl.owner, search
```
