# UC-22.8.25 · SOC 2 CC8 — Software Development Lifecycle Control Gates from CI/CD Telemetry

## Description

CC8 extends to software changes. Ingesting pipeline events for SAST, dependency scan, and peer review gates shows releases without required checks are rare and investigated—evidence that SDLC controls operate in production paths.

## Value

Gives the responsible team auditor-ready evidence for SOC 2 compliance by automating regulatory compliance evidence using indexes cicd and sourcetypes cicd:pipeline. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize stage names in ingestion; (2) require immutable `commit` in deploy events; (3) block deploy in CI, use Splunk for detective drift; (4) map `repo` to in-scope system; (5) monthly % deploys with all gates green. Indexes required: `index=cicd`; Sourcetypes: `sourcetype=cicd:pipeline`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector from GitHub Actions, GitLab, or Jenkins.
• Ensure the following data sources are available: `index=cicd` `sourcetype="cicd:pipeline"` (repo, commit, stage, status, build_id, _time).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize stage names in ingestion; (2) require immutable `commit` in deploy events; (3) block deploy in CI, use Splunk for detective drift; (4) map `repo` to in-scope system; (5) monthly % deploys with all gates green.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cicd sourcetype="cicd:pipeline" earliest=-30d stage="deploy_prod"
| stats values(stage) as stages by repo, commit, build_id
| where !like(stages,"(?i)sast_pass") OR !like(stages,"(?i)deps_scan_pass")
| stats count by repo
| sort - count
```
