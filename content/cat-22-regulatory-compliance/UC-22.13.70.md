# UC-22.13.70 · Unplanned Physical Security Incidents at Transmission Sites

## Description

We centralize alarms for intrusion, cut fence, or camera loss at transmission facilities for timely response and regulatory awareness.

## Value

We centralize alarms for intrusion, cut fence, or camera loss at transmission facilities for timely response and regulatory awareness.

## Implementation

(1) Map alarm feeds to station registry. (2) Page security dispatch on critical sites. (3) Correlate with law enforcement case numbers in ITSM. (4) Feed significant events to CIP-008 evaluation.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk OT Security Add-on (5151).
• Ensure the following data sources are available: `index=physical` `sourcetype=transmission:perimeter`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map alarm feeds to station registry. (2) Page security dispatch on critical sites. (3) Correlate with law enforcement case numbers in ITSM. (4) Feed significant events to CIP-008 evaluation.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype=transmission:perimeter earliest=-7d (alarm_type="intrusion" OR alarm_type="fence_cut" OR alarm_type="video_loss")
| lookup cip014_station_register station_id OUTPUT criticality
| stats count earliest(_time) as first latest(_time) as last by station_id, alarm_type, criticality
| sort - count
```
