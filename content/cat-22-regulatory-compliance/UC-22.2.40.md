# 22.2.40 — NIS2 Safety System Integrity Monitoring (SIL / SIS Interlocks)

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Cyber-physical incidents can impair safety instrumented functions. This use case monitors bypass events, force bits, interlock disables, and alarm floods from SIS and BMS historians — ensuring safety overrides are short-lived, authorised, and correlated with maintenance work orders.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes itsm, sis_audit and sourcetypes snow:sc_task. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=sis_audit earliest=-30d
| lookup nis2_sis_critical_tags.csv tag_id OUTPUT sil_level plant_area
| where match(lower(event_type),"(?i)bypass|force|override|interlock.disable")
| join tag_id [
    search index=itsm sourcetype="snow:sc_task" short_description="*permit*" earliest=-30d
    | stats latest(state) as permit_state by cmdb_ci
    | rename cmdb_ci as tag_id
]
| where permit_state!="Closed" OR isnull(permit_state)
| stats count by tag_id sil_level plant_area permit_state
| sort - count
```

## Implementation

(1) Work with functional safety officers to define critical tags; (2) ingest audit trails without exceeding vendor-supported rates; (3) alert immediately on unapproved bypass; (4) enforce maximum bypass duration timers in secondary logic where possible; (5) quarterly review with process safety and cybersecurity jointly. Indexes required: `index=itsm`, `index=sis_audit`; Sourcetypes: `sourcetype=snow:sc_task`; Lookups: `nis2_sis_critical_tags`, `nis2_sis_critical_tags.csv`.

## Visualization

Table (unauthorised overrides), Timeline (bypass events), Single value (open safety overrides), Bar chart (by plant area). Key fields: `permit_state`, `plant_area`, `sil_level`, `tag_id`.

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(c) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(c) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
