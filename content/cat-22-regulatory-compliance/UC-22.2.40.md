# UC-22.2.40 · NIS2 Safety System Integrity Monitoring (SIL / SIS Interlocks)

## Description

Cyber-physical incidents can impair safety instrumented functions. This use case monitors bypass events, force bits, interlock disables, and alarm floods from SIS and BMS historians — ensuring safety overrides are short-lived, authorised, and correlated with maintenance work orders.

## Value

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sis_audit earliest=-30d
| lookup nis2_sis_critical_tags.csv tag_id OUTPUT sil_level plant_area
| where match(lower(event_type),"(?i)bypass|force|override|interlock.disable")
| join tag_id [
    search index=itsm sourcetype="snow:sc_task" short_description="*permit*" earliest=-30d
    | stats latest(state) as permit_state by cmdb_ci
    | rename cmdb_ci as tag_id
]
| where permit_state!="Closed" OR isnull(permit_state)
| stats count by tag_id sil_level plant_area permit_state
| sort - count
```
