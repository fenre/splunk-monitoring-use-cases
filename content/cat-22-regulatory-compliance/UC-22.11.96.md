# UC-22.11.96 · PCI-DSS 8.3 — Strong authentication: password-only logins against privileged accounts

## Description

Detects a successful login to a privileged account that did not complete a second factor. PCI-DSS 8.3 requires strong authentication for all access to system components.

## Value

Catches the exact condition that has historically resulted in CDE breaches (privileged password-only sessions). MTTR shrinks from weeks to minutes.

## Implementation

(1) Maintain privileged_accounts.csv; (2) ensure IDP emits factor evidence; (3) schedule UC every 10m; (4) hit opens a P1 and disables the session; (5) monthly CISO review of patterns. Indexes required: `index=idp`, `index=linux`, `index=windows`; Sourcetypes: `sourcetype=WinEventLog:Security`, `sourcetype=auditd;`, `sourcetype=okta:system_log`, `sourcetype=okta:system_log.`.

Step 1 — Configure data collection
(1) Maintain privileged_accounts.csv; (2) ensure IDP emits factor evidence; (3) schedule UC every 10m; (4) hit opens a P1 and disables the session; (5) monthly CISO review of patterns.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
search (index=windows sourcetype=WinEventLog:Security EventCode=4624) OR (index=idp sourcetype=okta:system_log eventType=user.session.start) earliest=-10m
| lookup privileged_accounts.csv user OUTPUT is_privileged
| where is_privileged="yes"
| eval auth_method=case(isnotnull(LogonType) AND LogonType IN ("2","10"),"interactive",isnotnull(factor),factor,true(),"password")
| eval mfa_used=if(auth_method IN ("mfa_push","mfa_totp","webauthn","smartcard","token:hardware"),1,0)
| where mfa_used=0
| table user src dest auth_method mfa_used
```

## Known False Positives
