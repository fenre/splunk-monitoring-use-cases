# 22.10.9 — Periodic Evaluation — Control Test Evidence Ingest

**Criticality:** high | **Difficulty:** beginner | **Status:** verified

## Description

Centralizes periodic technical test results (MFA coverage, encryption checks) with timestamps for HIPAA periodic technical and non-technical evaluation evidence.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes grc and sourcetypes hipaa:periodic_test, hipaa:periodic_test`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=grc sourcetype="hipaa:periodic_test" earliest=-365d test_family="HIPAA_Technical"
| eval period_month=strftime(_time,"%Y-%m")
| stats latest(result) as last_result latest(_time) as last_run by control_id, period_month
| where last_result!="Pass"
| sort period_month, control_id
```

## Implementation

(1) Standardize `result` enum Pass/Fail/Partial; (2) join to asset inventory; (3) annual HIPAA evaluation deck exports from this dataset. Indexes required: `index=grc`; Sourcetypes: `sourcetype=hipaa:periodic_test`, `sourcetype=hipaa:periodic_test``.

## Visualization

Matrix (control x month), Table (failing controls), Column chart (pass rate trend). Key fields: `control_id`, `last_result`, `last_run`, `period_month`.
