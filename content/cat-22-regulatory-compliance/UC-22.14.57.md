# UC-22.14.57 · Least Functionality — Unexpected Listening Ports

## Description

Compares listening ports to approved service catalog for CM-7 least functionality.

## Value

Compares listening ports to approved service catalog for CM-7 least functionality.

## Implementation

(1) Refresh approved ports CSV by environment; (2) exclude ephemeral ranges with care; (3) alert on new listeners on servers; (4) pair with CM-8 inventory.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Universal Forwarder with `ss`/`netstat` scripted input or EDR network telemetry.
• Ensure the following data sources are available: `index=os` `sourcetype=netstat:listener`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Refresh approved ports CSV by environment; (2) exclude ephemeral ranges with care; (3) alert on new listeners on servers; (4) pair with CM-8 inventory.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os sourcetype=netstat:listener earliest=-24h state="LISTEN"
| lookup approved_listening_ports.csv port protocol OUTPUT approved
| where approved!="true" OR isnull(approved)
```
