# 22.15.54 — Remote Access Conduit Integrity — Split Tunnel on OT VPN Portal

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Detects GlobalProtect configurations that expose split routing while OT portals should force tunnel-all.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes vpn and sourcetypes pan:globalprotect. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=vpn sourcetype="pan:globalprotect" earliest=-7d
| eval split=coalesce(split_tunnel,"unknown")
| lookup ot_vpn_profiles.csv portal OUTPUT tunnel_mode_required
| where tunnel_mode_required="full" AND split!="disabled"
| stats dc(public_ip) as distinct_exit, count by user, portal, split
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.user | sort - count
```

## Implementation

(1) Push portal config from Panorama; (2) block local subnet access lists; (3) monthly config export diff; (4) user re-education. Indexes required: `index=vpn`; Sourcetypes: `sourcetype=pan:globalprotect`; Lookups: `ot_vpn_profiles`, `ot_vpn_profiles.csv`.

## Visualization

Table, Single value (misconfigured sessions), Bar. Key fields: `distinct_exit`, `portal`, `split`, `user`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 5.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 5.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
