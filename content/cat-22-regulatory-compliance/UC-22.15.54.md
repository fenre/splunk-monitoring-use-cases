# UC-22.15.54 · Remote Access Conduit Integrity — Split Tunnel on OT VPN Portal

## Description

Detects GlobalProtect configurations that expose split routing while OT portals should force tunnel-all.

## Value

Detects GlobalProtect configurations that expose split routing while OT portals should force tunnel-all.

## Implementation

(1) Push portal config from Panorama; (2) block local subnet access lists; (3) monthly config export diff; (4) user re-education.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Palo Alto (2757).
• Ensure the following data sources are available: `pan:globalprotect` config fields if logged.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Push portal config from Panorama; (2) block local subnet access lists; (3) monthly config export diff; (4) user re-education.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vpn sourcetype="pan:globalprotect" earliest=-7d
| eval split=coalesce(split_tunnel,"unknown")
| lookup ot_vpn_profiles.csv portal OUTPUT tunnel_mode_required
| where tunnel_mode_required="full" AND split!="disabled"
| stats dc(public_ip) as distinct_exit, count by user, portal, split
```
