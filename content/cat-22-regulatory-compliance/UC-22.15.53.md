# UC-22.15.53 · Wireless Zone Security Monitoring — Unknown Clients on OT SSID

## Description

Lists Wi-Fi associations to OT WLANs from unknown MAC vendors or non-instrumentation OUI prefixes.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes wireless and sourcetypes cisco:wlc, meraki:association. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) WPA3-Enterprise with cert; (2) dynamic PSK per device class; (3) block consumer OUIs; (4) RF site survey after changes. Indexes required: `index=wireless`; Sourcetypes: `sourcetype=cisco:wlc`, `sourcetype=meraki:association`; Lookups: `ot_wireless_allowlist`, `ot_wireless_allowlist.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: WLAN controller syslog.
• Ensure the following data sources are available: `meraki:airmarshal` or `cisco:wlc`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) WPA3-Enterprise with cert; (2) dynamic PSK per device class; (3) block consumer OUIs; (4) RF site survey after changes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wireless sourcetype="meraki:association" OR sourcetype="cisco:wlc" ssid="OT-PROD-*" earliest=-24h
| iplocation src_ip
| lookup ot_wireless_allowlist.csv client_mac OUTPUT asset_class
| where asset_class!="instrumentation"
| stats earliest(_time) as first_seen, latest(_time) as last_seen by client_mac, ssid, ap_name
| sort - last_seen
```
