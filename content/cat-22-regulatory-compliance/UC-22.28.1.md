# UC-22.28.1 · KRITIS operator reporting evidence

## Description

Demonstrates alignment with IT-Sicherheitsgesetz 2.0 expectations for kritis operator reporting evidence using auditable Splunk evidence.

## Value

Demonstrates alignment with IT-Sicherheitsgesetz 2.0 expectations for kritis operator reporting evidence using auditable Splunk evidence.

## Implementation

(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Windows (742), Splunk OT Security Add-on (5151), Splunk ITSI (1841), Splunk Common Information Model Add-on (1621).
• Ensure the following data sources are available: Windows Security Event Logs, critical infrastructure system logs, BSI IT-Grundschutz compliance scans, SIEM correlation events, vulnerability scan outputs, change management records, incident response logs, security audit evidence.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype="bsi:kritis:report" earliest=-365d
| eval submitted=strptime(submitted_bsi_at,"%Y-%m-%d %H:%M:%S")
| eval due=strptime(reporting_deadline,"%Y-%m-%d %H:%M:%S")
| where isnull(submitted) OR submitted>due
| table operator_id, reporting_deadline, submitted_bsi_at
```
