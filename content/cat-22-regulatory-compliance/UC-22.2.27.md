# 22.2.27 — NIS2 Vulnerability Disclosure Policy Operational Signals

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Coordinated vulnerability disclosure and cooperation with researchers support systemic resilience. This use case tracks intake of `security@` reports, HackerOne/Bugcrowd submissions, and PGP-encrypted mailbox events — measuring time-to-triage against internal policy.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes email_gateway, itsm and sourcetypes snow:incident. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=itsm sourcetype="snow:incident" category="*Vulnerability Disclosure*" earliest=-90d
| eval opened=strptime(opened_at,"%Y-%m-%d %H:%M:%S")
| eval triaged=strptime(work_start,"%Y-%m-%d %H:%M:%S")
| eval triage_hours=round((triaged-opened)/3600,2)
| eval sla_breach=if(triage_hours>48 OR isnull(triaged),1,0)
| stats count as tickets sum(sla_breach) as breaches avg(triage_hours) as avg_triage
| eval breach_pct=round(100*breaches/tickets,1)
| table tickets, breaches, breach_pct, avg_triage
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Ticket_Management.All_Ticket_Management by All_Ticket_Management.status, All_Ticket_Management.priority, All_Ticket_Management.category | sort - count
```

## Implementation

(1) Publish disclosure policy URL in security.txt and monitor referrers optionally; (2) auto-create ServiceNow incidents from email and bounty webhooks; (3) alert on SLA breaches; (4) link findings to engineering Jira; (5) annual report on researcher cooperation metrics. Indexes required: `index=email_gateway`, `index=itsm`; Sourcetypes: `sourcetype=snow:incident`.

## Visualization

Table (breached tickets), Line chart (triage time trend), Single value (breach %), Bar chart (by product team).

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(e) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(e) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
