# UC-22.14.74 · Risk Assessment Inputs — Control Deficiency Hotspots

## Description

Aggregates multiple weak signals (failed controls, vulns, policy violations) by system for RA-3 risk assessment prioritization.

## Value

Aggregates multiple weak signals (failed controls, vulns, policy violations) by system for RA-3 risk assessment prioritization.

## Implementation

(1) Replace `join` with `lookup` at scale; (2) calibrate weights annually; (3) export to enterprise risk register; (4) document assumptions.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263).
• Ensure the following data sources are available: `index=compliance`, `index=vulns`, `index=sec` — use summary joins.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Replace `join` with `lookup` at scale; (2) calibrate weights annually; (3) export to enterprise risk register; (4) document assumptions.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=compliance sourcetype=cis:assessment control_result="FAIL" earliest=-30d
| stats count as ctrl_fails by hostname
| join type=left hostname [
    search index=vulns sourcetype=tenable:vuln severity="Critical" earliest=-30d
    | stats count as crit_vulns by host
    | rename host as hostname
  ]
| eval risk_score=ctrl_fails*2 + crit_vulns*5
| sort - risk_score
| head 200
```
