# UC-22.13.27 · PSP Forced Door with Concurrent ESP Interactive Logon

## Description

We time-align severe physical alarms with interactive logons at the same site to support combined cyber-physical incident analysis.

## Value

We time-align severe physical alarms with interactive logons at the same site to support combined cyber-physical incident analysis.

## Implementation

(1) Maintain `bes_site_map.csv` from facility records. (2) Tune time bucketing (example uses join on site). (3) Prefer summary indexing for scale. (4) Attach results to CIP-008 preliminary review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk OT Security Add-on (5151), Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=physical` forced events, `index=wineventlog` RDP logons.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `bes_site_map.csv` from facility records. (2) Tune time bucketing (example uses join on site). (3) Prefer summary indexing for scale. (4) Attach results to CIP-008 preliminary review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype=pacs:access event_type="forced" earliest=-6h
| eval ptime=_time, site=bes_site
| join type=left site [
    search index=wineventlog sourcetype="WinEventLog:Security" EventCode=4624 LogonType=10 earliest=-6h
    | lookup bes_site_map.csv ComputerName OUTPUT bes_site as site
    | stats count by Account_Name, ComputerName, site
  ]
| where isnotnull(Account_Name)
| table ptime, site, employee_id, Account_Name, ComputerName, count
```
