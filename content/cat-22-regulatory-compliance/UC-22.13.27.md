# 22.13.27 — PSP Forced Door with Concurrent ESP Interactive Logon

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

We time-align severe physical alarms with interactive logons at the same site to support combined cyber-physical incident analysis.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes physical, wineventlog and sourcetypes WinEventLog:Security, pacs:access. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=physical sourcetype=pacs:access event_type="forced" earliest=-6h
| eval ptime=_time, site=bes_site
| join type=left site [
    search index=wineventlog sourcetype="WinEventLog:Security" EventCode=4624 LogonType=10 earliest=-6h
    | lookup bes_site_map.csv ComputerName OUTPUT bes_site as site
    | stats count by Account_Name, ComputerName, site
  ]
| where isnotnull(Account_Name)
| table ptime, site, employee_id, Account_Name, ComputerName, count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Maintain `bes_site_map.csv` from facility records. (2) Tune time bucketing (example uses join on site). (3) Prefer summary indexing for scale. (4) Attach results to CIP-008 preliminary review. Indexes required: `index=physical`, `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`, `sourcetype=pacs:access`.
