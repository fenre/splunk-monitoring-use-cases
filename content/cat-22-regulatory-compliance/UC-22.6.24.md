# UC-22.6.24 · Secure Disposal — Asset Decommission Wipe Confirmation Before CMDB Retire

## Description

Requires wipe or crypto-erase confirmation before CMDB retirement timestamp for media disposal chain of custody.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes itsm, mdm and sourcetypes intune:device_action, snow:cmdb_ci. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align name keys; (2) legal hold exceptions; (3) asset disposal vendor attestation upload. Indexes required: `index=itsm`, `index=mdm`; Sourcetypes: `sourcetype=intune:device_action`, `sourcetype=snow:cmdb_ci`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Intune, ServiceNow CMDB.
• Ensure the following data sources are available: `intune:device_action`; `snow:cmdb_ci`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align name keys; (2) legal hold exceptions; (3) asset disposal vendor attestation upload.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype=snow:cmdb_ci operational_status=Retired earliest=-180d
| eval retire_epoch=strptime(sys_updated_on,"%Y-%m-%d %H:%M:%S")
| join name [| search index=mdm sourcetype=intune:device_action action=wipe | stats latest(_time) as wipe_time by device_name | rename device_name as name]
| where isnull(wipe_time) OR wipe_time > retire_epoch
```
