# UC-22.11.31 · Phishing Simulation Click-Through Rates for Users with CDE Access

## Description

Measures phishing exercise outcomes for staff who can reach payment systems so security awareness training effectiveness supports malware defense and human-layer controls.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes sec_aware and sourcetypes phishme:results. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Sync HR email to `pci_cde_users.csv` after access reviews. (2) De-duplicate multiple sends per user per campaign. (3) Trigger mandatory remedial training over threshold. (4) Quarterly leadership summary. Indexes required: `index=sec_aware`; Sourcetypes: `sourcetype=phishme:results`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365 (Splunkbase 4055) if simulations logged; or `sourcetype="phishme:results"`.
• Ensure the following data sources are available: `index=sec_aware` `sourcetype="phishme:results"` — `campaign`, `user`, `clicked`, `reported`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Sync HR email to `pci_cde_users.csv` after access reviews. (2) De-duplicate multiple sends per user per campaign. (3) Trigger mandatory remedial training over threshold. (4) Quarterly leadership summary.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sec_aware sourcetype="phishme:results" earliest=-90d
| lookup pci_cde_users.csv email AS user OUTPUT has_cde_access
| where has_cde_access="true"
| eval clicked_flag=if(match(clicked,"(?i)yes|true|1"),1,0)
| stats sum(clicked_flag) as clicks, count as sent by campaign
| eval click_rate=if(sent>0, round(100*clicks/sent,2), null())
| where click_rate>15
| sort - click_rate
```
