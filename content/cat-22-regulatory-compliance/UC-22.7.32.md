# UC-22.7.32 · Administrative API Logging Volume Drop vs Baseline

## Description

Alerts when CloudTrail-style administrative logging volume drops suddenly, indicating pipeline or configuration tampering.

## Value

Alerts when CloudTrail-style administrative logging volume drops suddenly, indicating pipeline or configuration tampering.

## Implementation

(1) Seasonal adjust for holidays; (2) exclude account suspensions; (3) incident response if confirmed.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for AWS (Splunkbase 1876).
• Ensure the following data sources are available: `aws:cloudtrail`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Seasonal adjust for holidays; (2) exclude account suspensions; (3) incident response if confirmed.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=aws sourcetype=aws:cloudtrail earliest=-14d
| bucket _time span=1h as h
| stats count by h
| eventstats median(count) as med
| eval ratio=count/med
| where ratio < 0.2
```
