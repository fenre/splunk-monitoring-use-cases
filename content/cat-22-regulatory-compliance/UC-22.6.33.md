# UC-22.6.33 · Information Deletion — S3 Object Delete Storm Outside Retention Workflow

## Description

Detects mass delete operations not tagged with approved legal-hold workflow ID for controlled deletion.

## Value

Detects mass delete operations not tagged with approved legal-hold workflow ID for controlled deletion.

## Implementation

(1) Tag lifecycle jobs in lookup; (2) correlate with backup jobs; (3) insider risk review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for AWS (Splunkbase 1876).
• Ensure the following data sources are available: `aws:cloudtrail` eventName=DeleteObject.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag lifecycle jobs in lookup; (2) correlate with backup jobs; (3) insider risk review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=aws sourcetype=aws:cloudtrail eventName=DeleteObject earliest=-24h
| stats count as deletes by userIdentity.arn, bucketName
| where deletes>1000
| lookup approved_bulk_delete.csv userIdentity.arn, bucketName OUTPUT approved
| where isnull(approved)
```
