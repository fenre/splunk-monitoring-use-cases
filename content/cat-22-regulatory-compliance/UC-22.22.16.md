# UC-22.22.16 · Data Minimization Checks for TPP Response Payload Sizes

## Description

Monitors third-party provider access patterns, consent alignment, and API security controls required for PSD2 open banking operational resilience.

## Value

Monitors third-party provider access patterns, consent alignment, and API security controls required for PSD2 open banking operational resilience.

## Implementation

(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for AWS (1876).
• Ensure the following data sources are available: `index=openbanking` — tpp_id, consent_id, consent_scope, accessed_scope, http_status, environment, bytes_out.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=openbanking sourcetype="ob:api:access" earliest=-24h
| eval api_tag="min"
| eval is_prod=if(match(environment,"(?i)prod"),1,0)
| eval scope_mismatch=if(isnotnull(consent_scope) AND isnotnull(accessed_scope) AND consent_scope!=accessed_scope,1,0)
| stats count as calls, sum(eval(http_status>=400)) as err, sum(scope_mismatch) as sm, dc(tpp_id) as tpp by aspsp_id, api_tag, is_prod
| eval uc_id="22.22.16"
| eval err_rate=round(100*err/calls,2)
| where err_rate>2 OR sm>0 OR (api_tag="env" AND is_prod=0 AND tpp>50)
| sort - err_rate
```
