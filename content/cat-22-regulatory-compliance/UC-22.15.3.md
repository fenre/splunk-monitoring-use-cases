# UC-22.15.3 · OT Security Awareness Training Completion

## Description

Measures completion of OT-specific training by role so awareness obligations for personnel working on IACS are auditable.

## Value

Measures completion of OT-specific training by role so awareness obligations for personnel working on IACS are auditable.

## Implementation

(1) Ingest LMS extracts with privacy minimization; (2) map mandatory courses per role in policy; (3) join to AD only on approved fields; (4) alert managers for overdue learners.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686) or HEC from LMS.
• Ensure the following data sources are available: `index=hr_lms` `sourcetype="lms:ot_training"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest LMS extracts with privacy minimization; (2) map mandatory courses per role in policy; (3) join to AD only on approved fields; (4) alert managers for overdue learners.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr_lms sourcetype="lms:ot_training" earliest=-400d
| where like(lower(course_code),"%otsec%") OR like(lower(course_code),"%ics%")
| eval done_epoch=strptime(completed_date,"%Y-%m-%d")
| stats latest(done_epoch) as last_done by employee_id, role
| eval days_since=round((now()-last_done)/86400,0)
| where isnull(last_done) OR days_since>365
| sort - days_since
```
