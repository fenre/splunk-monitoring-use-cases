# 22.15.3 — OT Security Awareness Training Completion

**Criticality:** medium | **Difficulty:** intermediate | **Status:** N/A

## Description

Measures completion of OT-specific training by role so awareness obligations for personnel working on IACS are auditable.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes hr_lms and sourcetypes lms:ot_training. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=hr_lms sourcetype="lms:ot_training" earliest=-400d
| where like(lower(course_code),"%otsec%") OR like(lower(course_code),"%ics%")
| eval done_epoch=strptime(completed_date,"%Y-%m-%d")
| stats latest(done_epoch) as last_done by employee_id, role
| eval days_since=round((now()-last_done)/86400,0)
| where isnull(last_done) OR days_since>365
| sort - days_since
```

## Implementation

(1) Ingest LMS extracts with privacy minimization; (2) map mandatory courses per role in policy; (3) join to AD only on approved fields; (4) alert managers for overdue learners. Indexes required: `index=hr_lms`; Sourcetypes: `sourcetype=lms:ot_training`.

## Visualization

Table (overdue), Bar (completions/month), Single value (% current). Key fields: `days_since`, `done_epoch`, `employee_id`, `last_done`, `role`.

## Compliance

- **IEC 62443** 2013-ongoing: 4.3.2.6 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 4.3.2.6.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
