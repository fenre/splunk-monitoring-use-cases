# 22.15.43 — PLC CPU Load and Scan Cycle Degradation

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Monitors embedded controller resource metrics to catch resource exhaustion from misconfiguration or attack precursors.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_perf and sourcetypes snmp:plc. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_perf sourcetype="snmp:plc" earliest=-24h metric_name IN ("cpuUtil","scanCycleMs","taskOverload")
| timechart span=15m perc95(metric_value) by host, metric_name
| untable _time metric_series metric_value
| where metric_value>85 AND like(metric_series,"%cpuUtil%")
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Performance.CPU by Performance.host span=15m | sort - count
```

## Implementation

(1) Poll at safe intervals; (2) baseline per process; (3) alert above threshold; (4) tie to recent downloads. Indexes required: `index=ot_perf`; Sourcetypes: `sourcetype=snmp:plc`.

## Visualization

Time chart, Table (worst hosts), Single value (breach minutes).

## Compliance

- **IEC 62443** 2013-ongoing: CR 6.2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CR 6.2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
