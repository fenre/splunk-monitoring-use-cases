# UC-22.45.5 · Business-continuity rehearsal evidence — BCP/DR exercise execution logged

## Description

Moves BCP/DR rehearsal from a 'we did it' statement into a per-business-unit attestation log, mapped to DORA Art.24 and NIS2 Art.21(2)(c).

## Value

Converts a calendar item into evidence with a signed attestation per rehearsal.

## Implementation

(1) Maintain business_units.csv with owner + cadence; (2) emit bcp_rehearsals.csv after each exercise with a signed_by; (3) run quarterly.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: BCP rehearsal ticketing system, signed rehearsal attestations..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain business_units.csv with owner + cadence; (2) emit bcp_rehearsals.csv after each exercise with a signed_by; (3) run quarterly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup business_units.csv
| lookup bcp_rehearsals.csv bu_id OUTPUT last_rehearsal_at signed_by
| eval days_since=round((now()-strptime(last_rehearsal_at,"%Y-%m-%d"))/86400,0)
| where isnull(last_rehearsal_at) OR days_since>cadence_days
| table bu_id owner last_rehearsal_at days_since cadence_days
```
