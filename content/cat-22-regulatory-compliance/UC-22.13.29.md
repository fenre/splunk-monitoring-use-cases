# UC-22.13.29 · Security Patch Evaluation Completion Tracking

## Description

We track that each published critical patch for BES assets received a documented evaluation record within policy time.

## Value

We track that each published critical patch for BES assets received a documented evaluation record within policy time.

## Implementation

(1) Ingest patch catalog with release dates. (2) Require evaluation HEC event per KB. (3) Alert before internal evaluation SLA. (4) Export for CIP-007 evidence binders.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=patch` `sourcetype=wsus:summary` or `sourcetype=tenable:plugin`, `patch_evaluation_register.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest patch catalog with release dates. (2) Require evaluation HEC event per KB. (3) Alert before internal evaluation SLA. (4) Export for CIP-007 evidence binders.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=patch sourcetype=wsus:summary earliest=-35d classification="Critical"
| lookup bes_cyber_asset_inventory.csv ComputerName AS host OUTPUT bes_system
| where isnotnull(bes_system)
| join type=left kb_id [
    search index=grc sourcetype=cip:patch_evaluation kb_id=* earliest=-35d
    | stats latest(evaluated_at) as evaluated_at by kb_id
  ]
| where isnull(evaluated_at)
| stats dc(host) as hosts_missing_eval by kb_id, title
```
