# 22.13.29 — Security Patch Evaluation Completion Tracking

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We track that each published critical patch for BES assets received a documented evaluation record within policy time.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes grc, patch and sourcetypes cip:patch_evaluation, tenable:plugin`, wsus:summary. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=patch sourcetype=wsus:summary earliest=-35d classification="Critical"
| lookup bes_cyber_asset_inventory.csv ComputerName AS host OUTPUT bes_system
| where isnotnull(bes_system)
| join type=left kb_id [
    search index=grc sourcetype=cip:patch_evaluation kb_id=* earliest=-35d
    | stats latest(evaluated_at) as evaluated_at by kb_id
  ]
| where isnull(evaluated_at)
| stats dc(host) as hosts_missing_eval by kb_id, title
```

## Implementation

(1) Ingest patch catalog with release dates. (2) Require evaluation HEC event per KB. (3) Alert before internal evaluation SLA. (4) Export for CIP-007 evidence binders. Indexes required: `index=grc`, `index=patch`; Sourcetypes: `sourcetype=cip:patch_evaluation`, `sourcetype=tenable:plugin``, `sourcetype=wsus:summary`, `sourcetype=wsus:summary``; Lookups: `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.

## Visualization

Table (unevaluated KBs), Single value (count), Time chart (evaluations). Key fields: `evaluated_at`, `hosts_missing_eval`, `kb_id`, `title`.
