# UC-22.1.23 · GDPR Pseudonymisation Validation in Pipelines and Logs

## Description

Pseudonymisation reduces risk and supports lawful processing when combined with technical controls. This use case verifies that identifiers in downstream indexes, data lakes, and analytics topics match the organisation's pseudonymisation scheme — detecting raw direct identifiers where only tokenised or hashed substitutes should appear.

## Value

Gives the responsible team auditor-ready evidence for GDPR compliance by automating regulatory compliance evidence using indexes analytics, data_lake_events and sourcetypes pipeline_audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Document per-stream pseudonymisation rules in `gdpr_pseudonymisation_rules.csv`; (2) emit lightweight schema audit events from ETL jobs via HEC; (3) schedule hourly checks for forbidden field presence; (4) route violations to data engineering and DPO; (5) pair with UC-22.1.1 for regex-based residual PII detection. Indexes required: `index=analytics`, `index=data_lake_events`; Sourcetypes: `sourcetype=pipeline_audit`.

Step 1 — Configure data collection
(1) Document per-stream pseudonymisation rules in `gdpr_pseudonymisation_rules.csv`; (2) emit lightweight schema audit events from ETL jobs via HEC; (3) schedule hourly checks for forbidden field presence; (4) route violations to data engineering and DPO; (5) pair with UC-22.1.1 for regex-based residual PII detection.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=analytics sourcetype="pipeline_audit" earliest=-24h
| lookup gdpr_pseudonymisation_rules.csv stream_name OUTPUT expected_token_field, forbidden_raw_fields
| eval raw_hit=0
| foreach forbidden_raw_fields [ eval raw_hit=raw_hit + if(isnotnull(<<FIELD>>), 1, 0) ]
| eval violation=if(raw_hit>0 OR isnull(expected_token_field), "SCHEMA_VIOLATION", "OK")
| where violation!="OK"
| stats count by stream_name, violation, host
| sort - count
```
