# UC-22.14.13 · Session Audit for Privileged Interactive Access

## Description

Captures SSH/RDP session telemetry (logon, channel, disconnect) for AU-14 session audit on administrative jump hosts.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes os and sourcetypes WinEventLog:Security, linux:secure. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Enable verbose sshd logging per policy; (2) map EventCode meanings in a lookup; (3) alert on long sessions to sensitive assets; (4) redact passwords never present in logs. Indexes required: `index=os`; Sourcetypes: `sourcetype=WinEventLog:Security`, `sourcetype=linux:secure`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Unix and Linux (Splunkbase 833), Splunk Add-on for Microsoft Windows (Splunkbase 742).
• Ensure the following data sources are available: `linux:secure` (sshd), `WinEventLog:Security` (4624/4647/4778), Sysmon if available.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enable verbose sshd logging per policy; (2) map EventCode meanings in a lookup; (3) alert on long sessions to sensitive assets; (4) redact passwords never present in logs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os earliest=-24h (sourcetype=linux:secure OR sourcetype=WinEventLog:Security)
| search process=sshd OR EventCode IN (4624,4647,4778)
| eval session_actor=coalesce(user, Account_Name)
| stats earliest(_time) as start, latest(_time) as end, values(process) as processes by session_actor, src_ip, dest
| eval duration_sec=end-start
```
