# 22.1.46 — GDPR Consent Mechanism Audit (Lawful Basis Alignment)

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

By design, consent collection must be granular and verifiable. This use case monitors CMP and web telemetry for consent banner versions, non-essential cookies fired before consent, and mismatches between claimed purposes in the CMP and tags observed in the browser stream.

## Value

Gives the responsible team auditor-ready evidence for GDPR, UK GDPR compliance by automating regulatory compliance evidence using indexes consent, web and sourcetypes js_tag. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
(index=web sourcetype="js_tag") earliest=-24h
| eval before_consent=if(consent_state="unknown" AND match(tag_category,"(?i)marketing|analytics"),1,0)
| stats sum(before_consent) as early_nonessential_tags by site_id tag_vendor
| where early_nonessential_tags > 100
| lookup gdpr_consent_policy.csv site_id OUTPUT required_banner_version
| join site_id [
    search index=consent earliest=-24h
    | stats latest(banner_version) as live_banner by site_id
]
| where live_banner != required_banner_version
| table site_id, tag_vendor, early_nonessential_tags, live_banner, required_banner_version
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Web.Web by Web.status, Web.http_method, Web.dest | sort - count
```

## Implementation

(1) Instrument sites to emit tag and consent state events; (2) define required banner versions per site in policy lookup; (3) alert marketing and privacy when non-essential tags load pre-consent; (4) block releases that downgrade banner version without approval; (5) align with UC-22.1.5/UC-22.1.16 for withdrawal enforcement. Indexes required: `index=consent`, `index=web`; Sourcetypes: `sourcetype=js_tag`.

## Visualization

Table (sites with violations), Single value (sites off-policy), Heatmap (violations by region), Bar chart (tag vendors). Key fields: `before_consent`, `early_nonessential_tags`, `live_banner`, `required_banner_version`, `site_id`, `tag_vendor`.
