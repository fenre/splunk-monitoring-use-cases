# UC-22.11.69 ¬∑ Automated Log Review ‚Äî Correlation of Firewall Deny Bursts with IDS Signatures

## Description

Joins short-window firewall denies with IDS alerts touching the same `src_ip` so automated reviews surface coordinated attack patterns without waiting for manual cross-index pivoting.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes ids, netfw and sourcetypes pan:threat, pan:traffic, suricata:alert. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Tune `map` limits for performance; consider `join` with subsearch capped. (2) Replace with ES notable linking if preferred. (3) Document tuning for false positive rates. (4) Daily automated report to SOC leads. Indexes required: `index=ids`, `index=netfw`; Sourcetypes: `sourcetype=pan:threat`, `sourcetype=pan:traffic`, `sourcetype=suricata:alert`.

Step 2 ‚Äî Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=netfw sourcetype="pan:traffic" action=deny earliest=-1h
| stats count by src, dest
| where count>500
| join type=left max=0 src [
    search index=ids (sourcetype="suricata:alert" OR sourcetype="pan:threat") earliest=-1h
    | rename src_ip as src
    | stats dc(signature) as sigs by src
  ]
| where isnotnull(sigs) AND sigs>0
| table src, dest, count, sigs
```
