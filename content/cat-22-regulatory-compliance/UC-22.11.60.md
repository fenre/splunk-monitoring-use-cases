# UC-22.11.60 · Quarterly Physical Access List Review Exception Tracking

## Description

Tracks completion of quarterly access roster reviews for personnel with keys or badges to CDE facilities, tying physical security policy to measurable workflow deadlines.

## Value

Tracks completion of quarterly access roster reviews for personnel with keys or badges to CDE facilities, tying physical security policy to measurable workflow deadlines.

## Implementation

(1) Create recurring GRC control with quarterly `due_date`. (2) Push completion events from ServiceNow attestation. (3) Escalate to facility security lead. (4) Export closed tests for PCI policy evidence (Req 12 linkage).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (1928), `sourcetype="grc:control_test"`.
• Ensure the following data sources are available: `index=grc` `sourcetype="grc:control_test"` — `control_id`, `test_status`, `due_date`, `owner`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Create recurring GRC control with quarterly `due_date`. (2) Push completion events from ServiceNow attestation. (3) Escalate to facility security lead. (4) Export closed tests for PCI policy evidence (Req 12 linkage).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype="grc:control_test" earliest=-120d
| where control_id IN ("PCI-9.2.4-PHYS-REVIEW","PCI-9.2.4")
| eval overdue=if(now()>due_date AND NOT match(test_status,"(?i)pass|complete"),1,0)
| stats latest(test_status) as status, latest(due_date) as due by owner
| where overdue=1 OR isnull(status)
| sort due
```
