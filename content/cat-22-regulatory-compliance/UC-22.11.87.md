# UC-22.11.87 · Acceptable Use Policy Annual Attestation Completion

## Description

Tracks which employees have electronically signed the acceptable use policy each year so security policies acknowledged by users are provable for HR and PCI policy interviews.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes hr and sourcetypes hr:policy_attestation. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Version `policy_code` each revision. (2) Join workforce roster to find never-signed. (3) Manager digest weekly until 100%. (4) Export completion CSV for PCI annual evidence. Indexes required: `index=hr`; Sourcetypes: `sourcetype=hr:policy_attestation`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Workday / ServiceNow attestation HEC `sourcetype="hr:policy_attestation"`.
• Ensure the following data sources are available: `index=hr` `sourcetype="hr:policy_attestation"` — `policy_code`, `user`, `signed_date`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Version `policy_code` each revision. (2) Join workforce roster to find never-signed. (3) Manager digest weekly until 100%. (4) Export completion CSV for PCI annual evidence.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hr sourcetype="hr:policy_attestation" policy_code="AUP_V2026" earliest=-400d
| stats latest(signed_date) as last_signed by user
| eval signed_epoch=strptime(last_signed,"%Y-%m-%d")
| eval stale=if(signed_epoch<relative_time(now(),"-365d@d"),1,0)
| where stale=1 OR isnull(last_signed)
| table user, last_signed, stale
| sort user
```
