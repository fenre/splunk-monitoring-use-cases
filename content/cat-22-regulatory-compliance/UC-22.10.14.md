# UC-22.10.14 · Emergency Access Procedure — Downtime / Break-Glass Account Usage

## Description

Audits emergency and downtime accounts touching production ePHI so procedural safeguards around emergency access are reviewable and time-bound.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes epic, windows and sourcetypes epic:audit, epic:audit`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Require post-event attestation form ID in separate ticketing sourcetype and join; (2) auto-expire passwords via IAM; (3) weekly privacy review of all uses. Indexes required: `index=epic`, `index=windows`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742), Splunk Enterprise Security (263).
• Ensure the following data sources are available: `WinEventLog:Security` 4624 for `svc_epic_emergency*`, `index=epic` `sourcetype=epic:audit` `AccessContext="Downtime"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require post-event attestation form ID in separate ticketing sourcetype and join; (2) auto-expire passwords via IAM; (3) weekly privacy review of all uses.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
(index=windows EventCode="4622" OR EventCode="4624") Account_Name="svc_epic_emergency*" earliest=-30d
| append [
    search index=epic sourcetype="epic:audit" AccessContext="*" earliest=-30d
    | rex field=AccessContext "(?<ctx>Downtime|Emergency)"
    | where ctx IN ("Downtime","Emergency") ]
| eval principal=coalesce(Account_Name, USER_ID)
| stats earliest(_time) as first latest(_time) as last dc(host) as systems by principal, ctx
| eval duration_h=round((last-first)/3600,2)
| sort - last
```
