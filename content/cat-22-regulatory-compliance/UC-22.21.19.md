# UC-22.21.19 · CE Marking Evidence — Test Report Coverage (Art. 48)

## Description

Joins product_sku to test_report_id coverage from test lab feeds for CE marking evidentiary chain.

## Value

Joins product_sku to test_report_id coverage from test lab feeds for CE marking evidentiary chain.

## Implementation

(1) Normalize SKU keys; (2) nightly DB Connect sync; (3) alert coverage_pct < 100 for EU SKUs; (4) map to notified body references outside Splunk.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk DB Connect (2686), Splunk ITSI (1841).
• Ensure the following data sources are available: `index=product` `sourcetype=plm:sku` joined with `index=testlab` `sourcetype=lab:report`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize SKU keys; (2) nightly DB Connect sync; (3) alert coverage_pct < 100 for EU SKUs; (4) map to notified body references outside Splunk.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=plm sourcetype="plm:sku" earliest=-1d
| fields product_sku, model_family, hw_rev
| join type=left product_sku [
    search index=testlab sourcetype="lab:report" earliest=-400d
    | stats latest(report_id) as test_report_id by product_sku
  ]
| eval covered=if(isnotnull(test_report_id),1,0)
| stats sum(covered) as ok, count as total
| eval coverage_pct=round(100*ok/total,2)
```
