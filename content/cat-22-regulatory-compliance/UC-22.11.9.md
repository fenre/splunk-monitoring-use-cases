# UC-22.11.9 · Configuration Drift vs CIS Hardening Benchmark on Windows CDE Members

## Description

Aggregates configuration assessment findings against a documented hardening standard so you can show continuous alignment with industry-accepted secure configurations for in-scope Windows systems.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes vm and sourcetypes qualys:policy, tenable:sc:compliance. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map scanner asset names to PCI inventory FQDNs. (2) Scope policies to CIS Windows Server benchmarks used in your standard. (3) Track `fail_pct` trend weekly. (4) Attach exports to secure configuration evidence repository. Indexes required: `index=vm`; Sourcetypes: `sourcetype=qualys:policy`, `sourcetype=tenable:sc:compliance`; Lookups: `pci_windows_cde_assets`, `pci_windows_cde_assets.csv`.

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vm sourcetype IN ("tenable:sc:compliance","qualys:policy") earliest=-7d
| lookup pci_windows_cde_assets.csv fqdn AS host_fqdn OUTPUT cde_member
| where cde_member="true"
| eval fail=if(match(compliance_status,"(?i)failed|error|critical"),1,0)
| stats sum(fail) as failed_checks, count as total_checks by host_fqdn
| eval fail_pct=if(total_checks>0, round(100*failed_checks/total_checks,2), null())
| where fail_pct>5
| sort - fail_pct
```
