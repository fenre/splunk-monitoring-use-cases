# UC-22.7.34 · Database Connection Storm from Application Service Account

## Description

Statistical spike in DB connections by application service account for compromised app or insider detection.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes database and sourcetypes db:mysql:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Baseline per app; (2) correlate with deploys; (3) block at WAF optional. Indexes required: `index=database`; Sourcetypes: `sourcetype=db:mysql:audit`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: MySQL audit.
• Ensure the following data sources are available: `db:mysql:audit`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Baseline per app; (2) correlate with deploys; (3) block at WAF optional.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=database sourcetype=db:mysql:audit earliest=-6h
| stats count by user, src
| eventstats median(count) as med by user
| where count > med*10 AND count>1000
```
