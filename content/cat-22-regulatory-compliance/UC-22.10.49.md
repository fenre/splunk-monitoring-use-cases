# UC-22.10.49 · Breach Remediation — Control Implementation Evidence Post-Incident

## Description

Links post-breach corrective actions (MFA rollout, DLP rules) to measurable control telemetry so remediation is demonstrable, not narrative-only.

## Value

Links post-breach corrective actions (MFA rollout, DLP rules) to measurable control telemetry so remediation is demonstrable, not narrative-only.

## Implementation

(1) Define `metric_value` as percent complete (e.g., MFA enrolled / total); (2) nightly scripted input; (3) board-level dashboard after major breaches.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk Add-on for Microsoft Office 365 (4055).
• Ensure the following data sources are available: `index=privacy` `sourcetype=hipaa:remediation_task` (`incident_id`, `control_id`, `target_date`), `index=o365` DLP rule hits, `index=ad` MFA coverage scripted metrics.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Define `metric_value` as percent complete (e.g., MFA enrolled / total); (2) nightly scripted input; (3) board-level dashboard after major breaches.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=privacy sourcetype="hipaa:remediation_task" earliest=-180d status!="Closed"
| eval due=strptime(target_date,"%Y-%m-%d")
| join control_id [
    search index=metrics sourcetype="hipaa:control_metric" earliest=-1d
    | stats latest(metric_value) as current_value by control_id ]
| eval behind=if(now()>due AND current_value<100,1,0)
| where behind=1
| table incident_id, control_id, target_date, current_value, owner
```

Scripted input (generic example)
This use case relies on a scripted input. In the app's local/inputs.conf add a stanza such as:

```ini
[script://$SPLUNK_HOME/etc/apps/YourApp/bin/collect.sh]
interval = 300
sourcetype = your_sourcetype
index = main
disabled = 0
```

The script should print one event per line (e.g. key=value). Example minimal script (bash):

```bash
#!/usr/bin/env bash
# Output metrics or events, one per line
echo "metric=value timestamp=$(date +%s)"
```
