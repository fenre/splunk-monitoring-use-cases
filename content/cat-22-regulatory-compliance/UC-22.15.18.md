# UC-22.15.18 · Unauthorized Writes to Modbus Holding Registers

## Description

Detects Modbus function codes that modify process values from hosts not authorized for write operations.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_protocol and sourcetypes edge_hub_modbus. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Build ACL from approved EWS IPs; (2) segment read-only analytics clients; (3) test with HIL bench; (4) integrate with SOAR for host isolation. Indexes required: `index=ot_protocol`; Sourcetypes: `sourcetype=edge_hub_modbus`; Lookups: `modbus_write_acl`, `modbus_write_acl.csv`.

Prerequisites

• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Build ACL from approved EWS IPs; (2) segment read-only analytics clients; (3) test with HIL bench; (4) integrate with SOAR for host isolation.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_protocol sourcetype="edge_hub_modbus" earliest=-24h
| where function_code IN (5,6,15,16,22,23)
| lookup modbus_write_acl.csv src_ip dest_plc OUTPUT role
| where role!="engineer_approved" OR isnull(role)
| stats values(register_start) as regs, count by src_ip, dest_plc, user
| sort - count
```
