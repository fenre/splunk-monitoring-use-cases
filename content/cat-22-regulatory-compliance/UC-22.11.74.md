# UC-22.11.74 · Internal Authenticated Vulnerability Scan Coverage by CDE Subnet

## Description

Verifies each CDE /24 received at least one credentialed scan in the rolling quarter so internal vulnerability scanning frequency and scope requirements are met with machine evidence.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes vm and sourcetypes tenable:sc:vuln. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize IP field names from Tenable TA. (2) Adjust `subnet` extraction for IPv6 if in scope. (3) Alert subnets over 90 days. (4) Map exceptions to documented outage windows. Indexes required: `index=vm`; Sourcetypes: `sourcetype=tenable:sc:vuln`; Lookups: `pci_cde_subnets_summary`, `pci_cde_subnets_summary.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Tenable (4060).
• Ensure the following data sources are available: `index=vm` `tenable:sc:vuln` with `scan_name`, `host_ip`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize IP field names from Tenable TA. (2) Adjust `subnet` extraction for IPv6 if in scope. (3) Alert subnets over 90 days. (4) Map exceptions to documented outage windows.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vm sourcetype="tenable:sc:vuln" earliest=-90d
| where match(scan_name,"(?i)internal|credentialed|CDE)")
| eval subnet=replace(host_ip,"^(\\d+\\.\\d+\\.\\d+)\\.\\d+$","\\1.0/24")
| lookup pci_cde_subnets_summary.csv subnet OUTPUT required_scan
| where required_scan="true"
| stats latest(_time) as last_scan by subnet
| eval days_since=round((now()-last_scan)/86400,1)
| where days_since>90 OR isnull(last_scan)
| sort days_since
```
