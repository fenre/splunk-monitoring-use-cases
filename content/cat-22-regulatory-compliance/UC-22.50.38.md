# 22.50.38 — GLBA §314.4(c)(1) — Access controls: customer-information access governance and least-privilege enforcement

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Monitors access review completion rates and privileged session activity for GLBA §314.4(c)(1) access controls. Separates periodic access certification from per-session monitoring, fixing the logic that previously confused individual access events with periodic reviews.

## Value

§314.4(c)(1) requires implementing access controls to limit access to customer information to authorised users. Continuous monitoring of privileged access and review completeness enforces least-privilege.

## SPL

```spl
index=iam sourcetype IN (access_reviews, privileged_sessions) earliest=-30d
| eval is_review=if(sourcetype="access_reviews", 1, 0)
| eval is_session=if(sourcetype="privileged_sessions", 1, 0)
| eval review_completed=if(is_review=1 AND review_status="completed", 1, 0)
| eval privileged_access=if(is_session=1 AND privilege_level IN ("admin", "root", "elevated"), 1, 0)
| stats sum(is_review) AS total_reviews sum(review_completed) AS completed_reviews sum(privileged_access) AS priv_sessions dc(user) AS distinct_users BY business_unit
| eval review_completion_pct=if(total_reviews>0, round(100*completed_reviews/total_reviews, 1), 0)
| eval access_risk=case(review_completion_pct<80, "high", review_completion_pct<95, "medium", true(), "low")
| where access_risk!="low"
| table business_unit, total_reviews, completed_reviews, review_completion_pct, priv_sessions, distinct_users, access_risk
```

## Implementation

(1) Tag systems holding customer information with system_tag=glba_scope; (2) ingest IAM access and privilege-change logs; (3) schedule daily; (4) escalate excessive privileged access. Indexes required: `index=iam`; Sourcetypes: `sourcetype=access_log`, `sourcetype=privilege_changes`.

## Visualization

Table of flagged records for investigation; single-value KPI of coverage / compliance percentage; time chart of trend over the last 90 days; drill-down per responsible owner. Key fields: `access_reviewed`, `department`, `distinct_users`, `privileged_access`, `privileged_count`, `privileged_pct`.

## Compliance

- **GLBA** 16 CFR 314 (2023 amendments): §314.4(c)(1) (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for §314.4(c)(1).

## Known False Positives

Tuning prerequisites: source field mappings must be in place before the UC will produce reliable metrics; early runs will commonly flag unmapped systems until onboarding is completed.
