# UC-22.44.3 · Fourth-party concentration risk — shared critical dependencies across vendors

## Description

Computes a monthly concentration score for each fourth-party dependency across the vendor graph. Surfaces candidates for additional due-diligence or contractual controls.

## Value

Addresses the DORA supervisory-priorities theme that regulators ask 'who do your vendors depend on?'.

## Implementation

(1) Maintain vendor_dependency_graph.csv enriched monthly; (2) run monthly; (3) surface to risk committee.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: See subcategory preamble..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain vendor_dependency_graph.csv enriched monthly; (2) run monthly; (3) surface to risk committee.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup vendor_dependency_graph.csv
| where vendor_criticality IN ("critical","high")
| stats dc(vendor) as dependent_count, values(vendor) as vendors by fourth_party
| eval concentration_score=min(dependent_count/3,1)
| where concentration_score>=0.6
| table fourth_party dependent_count vendors concentration_score
```

## Known False Positives

Depth-1 dependencies where the upstream vendor has alternate providers should be discounted (substitutability>0).
