# UC-22.8.15 · SOC 2 CC5 — Change Impact Analysis Completeness for Production Releases

## Description

CC5 expects consideration of fraud and other changes. Verifying every production change record carries impact analysis, rollback plan, and security review flags demonstrates change-related risks were evaluated before deployment.

## Value

Gives the responsible team auditor-ready evidence for SOC 2 compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:change_request. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map Snow fields to your CMDB CI classes requiring analysis; (2) exclude standard changes via `chg_model=standard`; (3) integrate with CAB workflow to block closure in source system, Splunk mirrors drift; (4) monthly metric for % changes with complete analysis; (5) attach search export to change management KPI deck. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:change_request`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (Splunkbase 1928).
• Ensure the following data sources are available: `index=itsm` `sourcetype="snow:change_request"` (number, state, u_impact_analysis, u_rollback_plan, u_security_review, start_date, cmdb_ci).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map Snow fields to your CMDB CI classes requiring analysis; (2) exclude standard changes via `chg_model=standard`; (3) integrate with CAB workflow to block closure in source system, Splunk mirrors drift; (4) monthly metric for % changes with complete analysis; (5) attach search export to change management KPI deck.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="snow:change_request" state IN ("Implement","Closed") earliest=-60d
| eval missing=mvjoin(
    if(isnull(u_impact_analysis) OR u_impact_analysis="","impact",null()),
    if(isnull(u_rollback_plan) OR u_rollback_plan="","rollback",null()),
    if(isnull(u_security_review) OR u_security_review="false","sec_review",null()),
    ",")
| where missing!=""
| stats count by missing, cmdb_ci
| sort - count
```
