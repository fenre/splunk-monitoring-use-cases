# 22.30.24 — Third-party information security assessment tracking

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Supports APRA CPS 234 supervisory examination readiness for third-party information security assessment tracking.

## Value

Gives the responsible team auditor-ready evidence for APRA CPS 234 compliance by automating regulatory compliance evidence using indexes vendor and sourcetypes cps234:third_party. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=vendor sourcetype="cps234:third_party" earliest=-180d
| stats latest(assessment) as a by vendor_name
| where a!="Complete"
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=vendor`; Sourcetypes: `sourcetype=cps234:third_party`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones).
