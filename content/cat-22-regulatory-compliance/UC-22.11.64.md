# UC-22.11.64 · NTP Stratum and Sync Failure Events on Payment Switches and Firewalls

## Description

Surfaces NTP loss of sync or excessive stratum on devices whose timestamps anchor audit logs for network security controls, preserving accurate time for all critical systems.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes netops and sourcetypes pan:system, syslog:cisco. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Populate devices that must never drift per network standard. (2) Correlate with leap-second maintenance windows. (3) Page network ops on spikes. (4) Attach monthly zero-result report when clean. Indexes required: `index=netops`; Sourcetypes: `sourcetype=pan:system`, `sourcetype=syslog:cisco`; Lookups: `pci_time_critical_devices`, `pci_time_critical_devices.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=netops sourcetype IN ("syslog:cisco","pan:system") earliest=-24h
| where match(_raw,"(?i)ntp|clock|stratum|unsync|time\\s+set)")
| lookup pci_time_critical_devices.csv host OUTPUT requires_ntp
| where requires_ntp="true"
| stats count by host, sourcetype
| sort - count
```
