# UC-22.5.6 · MiFID II Order and Decision Data Record Integrity

## Description

Investment firms must keep orderly records of services and transactions. Detecting gaps or duplicate order IDs in OMS/EMS journals supports defensible reconstruction of the decision chain during regulatory reconstruction exercises.

## Value

Gives the responsible team auditor-ready evidence for MiFID II compliance by automating regulatory compliance evidence using indexes trading and sourcetypes _json. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ensure every order emits at least creation and terminal state events; (2) hash sensitive client fields before indexing; (3) alert on `events<2` for statuses that should be terminal within T day; (4) tune `span_sec` threshold for high-frequency desks; (5) export samples for internal audit replay tools. Indexes required: `index=trading`; Sourcetypes: `sourcetype=_json`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Financial Information eXchange (FIX) Log Parsing (Splunkbase 431), Splunk HTTP Event Collector (core platform).
• Ensure the following data sources are available: `index=trading` `sourcetype="_json"` `source="http:order_lifecycle"` (order_id, cl_ord_id, event_type, venue, decision_time_ms, _time).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ensure every order emits at least creation and terminal state events; (2) hash sensitive client fields before indexing; (3) alert on `events<2` for statuses that should be terminal within T day; (4) tune `span_sec` threshold for high-frequency desks; (5) export samples for internal audit replay tools.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=trading sourcetype="_json" source="http:order_lifecycle" earliest=-1d
| where event_type IN ("NewOrderSingle","OrderCancelReplaceRequest","ExecutionReport","OrderCancelRequest")
| stats min(_time) as first_seen, max(_time) as last_seen, dc(event_type) as event_types, count as events by order_id
| eval span_sec=last_seen-first_seen
| eventstats dc(order_id) as total_orders
| eventstats sum(events) as sum_events
| eval dup_ratio=round(events/sum_events, 6)
| where events<2 OR span_sec<0.001
| sort order_id
```
