# 22.20.6 — CMMC CM.L2-3.4.1 — Baseline configuration drift detection on CUI systems

**Criticality:** medium | **Difficulty:** expert | **Status:** verified

## Description

Detects configuration baseline drift on CUI-classified systems using configuration management tool outputs (Chef InSpec, Puppet, Ansible). CMMC CM.L2-3.4.1 requires establishing and maintaining baseline configurations and inventories of organisational systems throughout the system development life cycle.

## Value

Continuously monitors CUI systems for deviations from approved baseline configurations, providing evidence that baselines are actively enforced and drift is detected and remediated.

## SPL

```spl
index=os sourcetype IN ("chef:compliance", "puppet:report", "ansible:result") earliest=-24h
| lookup cui_hosts.csv hostname AS host OUTPUT cui_flag
| where cui_flag="true"
| eval drift=if(control_status="failed" OR status="changed" OR result="non-compliant", 1, 0)
| stats sum(drift) AS drift_count count AS total_checks latest(_time) AS last_check BY host, profile
| eval drift_pct=round(100*drift_count/total_checks, 1)
| where drift_count > 0
| sort - drift_pct
| table host, profile, total_checks, drift_count, drift_pct, last_check
```

## CIM SPL

```spl
| tstats count from datamodel=Endpoint.Processes WHERE Endpoint.dest IN (cui_hosts_lookup) BY Endpoint.dest | rename Endpoint.dest AS host
```

## Implementation

(1) Verify data sources: Chef InSpec compliance reports (`chef:compliance`), Puppet reports (`puppet:report`), Ansible results (`ansible:result`), `cui_hosts.csv` lookup; (2) Maintain CUI-scoped lookups; (3) Schedule search at compliance-aligned interval; (4) Route findings to ticketing queue; (5) Retain exports per records management policy.

## Visualization

Table of hosts with drift percentage. Heat map of drift by profile. Single value for total hosts with active drift.

## Compliance

- **CMMC** 2.0: CM.L2-3.4.1 (partial) — Directly monitors baseline configuration adherence on CUI systems — the core requirement of CM.L2-3.4.1.

## Known False Positives

Approved changes during maintenance windows may temporarily trigger drift alerts. Correlate with change management tickets.
