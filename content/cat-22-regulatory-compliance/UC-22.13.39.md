# UC-22.13.39 · Reportable Incident Classification — BES Reliability Impact Signals

## Description

We correlate OT historian or EMS quality flags with cyber alerts to help classify incidents that may affect BES reliability or operations.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes ot_hist, ot_security and sourcetypes ot:alert, scada:point, scada:point`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Normalize substation keys between historians and security tools. (2) Train analysts on EOP impact criteria. (3) Document classification decisions in ITSM. (4) Retain historian extracts per legal hold guidance. Indexes required: `index=ot_hist`, `index=ot_security`; Sourcetypes: `sourcetype=ot:alert`, `sourcetype=scada:point`, `sourcetype=scada:point``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Industrial Asset Intelligence, Splunk OT Security Add-on (5151).
• Ensure the following data sources are available: `index=ot_hist` `sourcetype=scada:point`, `index=ot_security` security events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize substation keys between historians and security tools. (2) Train analysts on EOP impact criteria. (3) Document classification decisions in ITSM. (4) Retain historian extracts per legal hold guidance.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_hist sourcetype=scada:point earliest=-24h (quality="BAD" OR quality="UNCERTAIN")
| bin _time span=15m
| stats count by substation, point_name, _time
| join type=left substation [
    search index=ot_security sourcetype="ot:alert" earliest=-24h
    | stats latest(description) as cyber_alert by substation
  ]
| where isnotnull(cyber_alert)
| table _time, substation, point_name, count, cyber_alert
```
