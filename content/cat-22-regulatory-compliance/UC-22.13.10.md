# 22.13.10 — Personnel Risk Assessment Due-Date Monitoring

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We highlight access holders whose personnel risk assessment is overdue while they still touch BES systems, reducing stale risk decisions.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes hr, wineventlog and sourcetypes WinEventLog:Security, hr:pra, hr:pra`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=hr sourcetype=hr:pra earliest=-730d
| eval next_epoch=strptime(next_due,"%Y-%m-%d")
| rename employee_id as user
| join type=left user [
    search index=wineventlog sourcetype="WinEventLog:Security" EventCode=4624 earliest=-30d
    | lookup bes_privileged_users.csv Account_Name AS user OUTPUT employee_id
    | stats dc(ComputerName) as systems by user
  ]
| where next_epoch < now() OR isnull(next_due)
| table user, assessment_date, next_due, systems
```

## Implementation

(1) Ingest PRA due dates from HR or GRC. (2) Correlate with active privileged users only. (3) Auto-ticket overdue users with recent BES logons. (4) Quarterly leadership rollup. Indexes required: `index=hr`, `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`, `sourcetype=hr:pra`, `sourcetype=hr:pra``; Lookups: `bes_privileged_users`, `bes_privileged_users.csv`.

## Visualization

Table (overdue), Timeline (assessment dates), Single value (overdue count). Key fields: `assessment_date`, `next_due`, `next_epoch`, `systems`, `user`.

## Compliance

- **NERC CIP** current: CIP-004-6 R2 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-004-6 R2.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
