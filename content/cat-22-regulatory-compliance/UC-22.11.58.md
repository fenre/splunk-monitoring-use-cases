# UC-22.11.58 · Secure Media Destruction Workflow Completion for Backup Tapes with CHD

## Description

Tracks chain-of-custody and destruction certificates for media classified as containing account data so classified media is destroyed when no longer needed for business or legal reasons.

## Value

Tracks chain-of-custody and destruction certificates for media classified as containing account data so classified media is destroyed when no longer needed for business or legal reasons.

## Implementation

(1) Ingest vendor destruction webhook as status `destroyed`. (2) Align retention to legal hold exceptions via lookup. (3) Escalate aged media weekly. (4) Attach certificates to GRC record.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (1928) or `sourcetype="ironmountain:destruction"`.
• Ensure the following data sources are available: `index=logistics` `sourcetype="media:chain"` — `barcode`, `status`, `destroyed_time`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest vendor destruction webhook as status `destroyed`. (2) Align retention to legal hold exceptions via lookup. (3) Escalate aged media weekly. (4) Attach certificates to GRC record.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=logistics sourcetype="media:chain" earliest=-90d
| where classification="PCI_CHD"
| where match(status,"(?i)in_transit|awaiting_destroy|stored")
| eval age_days=round((now()-created_time)/86400,1)
| where age_days>120
| stats max(age_days) as oldest_days by barcode, status, vault_site
| sort - oldest_days
```
