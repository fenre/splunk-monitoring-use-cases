# UC-22.11.38 · DAST and SAST Finding Density Before Payment Service Releases

## Description

Tracks high-severity static and dynamic scan findings per build of payment APIs so security testing is evidenced prior to promoting code that could affect account data.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes devsecops and sourcetypes checkmarx:scan, veracode:scan. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map scanner project names to internal `app_name`. (2) Gate release on `count=0` for Critical if policy mandates. (3) Correlate with UC-22.11.37 for same `build_id`. (4) Quarterly trending for PCI interview. Indexes required: `index=devsecops`; Sourcetypes: `sourcetype=checkmarx:scan`, `sourcetype=veracode:scan`; Lookups: `pci_payment_apps`, `pci_payment_apps.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Custom `sourcetype="veracode:scan"` / `checkmarx:scan"` HEC normalizers.
• Ensure the following data sources are available: `index=devsecops` `sourcetype IN ("veracode:scan","checkmarx:scan")` — `app_name`, `build_id`, `severity`, `status`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map scanner project names to internal `app_name`. (2) Gate release on `count=0` for Critical if policy mandates. (3) Correlate with UC-22.11.37 for same `build_id`. (4) Quarterly trending for PCI interview.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=devsecops sourcetype IN ("veracode:scan","checkmarx:scan") earliest=-30d
| lookup pci_payment_apps.csv app_name OUTPUT pci_app
| where pci_app="true"
| where severity IN ("High","Very High","Critical")
| where match(status,"(?i)open|new)")
| stats count by app_name, build_id, scan_type
| where count>0
| sort - count
```
