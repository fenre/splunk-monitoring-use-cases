# UC-22.50.10 · PIPL Art.51 — information-security measures across PRC personal-data systems

## Description

Aggregates security-control evidence across PRC-hosted systems handling personal information.

## Value

PIPL Art.51 mandates information-security management; the UC gives data processors Chinese-authority-facing evidence of Art.51 measures.

## Implementation

(1) Tag PRC-resident systems with a 'jurisdiction' field; (2) ingest control telemetry; (3) schedule this search nightly.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: Endpoint logs, crypto inventory, access control, vulnerability scanner feeds for PRC-resident systems..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag PRC-resident systems with a 'jurisdiction' field; (2) ingest control telemetry; (3) schedule this search nightly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=prc_systems earliest=-7d
| eval control_family=coalesce(control_family,"unknown")
| stats count AS events count(eval(control_state="failed")) AS failures BY system_id, control_family
| eval failure_rate=round(100*failures/events,2)
| where failure_rate > 0 AND control_family IN ("encryption","access_control","vulnerability_mgmt","logging")
```
