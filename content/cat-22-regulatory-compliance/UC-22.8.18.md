# UC-22.8.18 · SOC 2 CC6 — Encryption in Transit Policy Enforcement for Admin and API Paths

## Description

CC6 expects encryption of data in transmission. Detecting TLS policy regressions—weak cipher suites, certificate expirations, or plaintext admin sessions—provides continuous evidence that cryptographic controls remain effective.

## Value

Gives the responsible team auditor-ready evidence for SOC 2 compliance by automating regulatory compliance evidence using indexes network and sourcetypes nginx:access, zeek:ssl. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Expand to internal services behind load balancers using `server_name`; (2) join cert inventory for `cert_expires` within 30 days; (3) separate customer-facing vs admin VIPs; (4) integrate with vulnerability SLA for weak TLS; (5) document compensating controls for legacy apps with approved exceptions lookup. Indexes required: `index=network`; Sourcetypes: `sourcetype=nginx:access`, `sourcetype=zeek:ssl`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Zeek (Splunkbase 1617), Splunk Add-on for Nginx or F5 LTM (as applicable).
• Ensure the following data sources are available: `index=network` `sourcetype IN ("zeek:ssl","nginx:access")` (server_name, ssl_version, cipher, cert_expires); tag admin paths via `uri_path`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

(1) Expand to internal services behind load balancers using `server_name`; (2) join cert inventory for `cert_expires` within 30 days; (3) separate customer-facing vs admin VIPs; (4) integrate with vulnerability SLA for weak TLS; (5) document compensating controls for legacy apps with approved exceptions lookup.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="zeek:ssl" earliest=-24h
| where ssl_version IN ("TLSv1","TLSv1.0","SSLv3") OR like(cipher,"%NULL%") OR like(cipher,"%EXPORT%")
| stats count by server_name, ssl_version, cipher
| sort - count
```
