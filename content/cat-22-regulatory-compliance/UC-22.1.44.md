# UC-22.1.44 · GDPR Cross-Border Personal Data Flow Anomaly Detection

## Description

Sudden shifts in cross-border traffic can indicate misconfigured replication, insider misuse, or compromised accounts — each with transfer-law implications. This use case baselines bytes and session counts by destination country and alerts on statistically rare spikes for systems processing personal data.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| tstats `summariesonly` sum(All_Traffic.bytes_out) as bytes_out count as sessions
  from datamodel=Network_Traffic.All_Traffic
  where earliest=-60d latest=now
  by All_Traffic.dest span=1d
| rename All_Traffic.* as *
| iplocation dest
| lookup gdpr_ropa_register.csv system_name OUTPUT data_category
| where isnotnull(data_category)
| eventstats median(bytes_out) as med by Country
| eval z=round((bytes_out-med)/std(bytes_out),2)
| where abs(z) > 3
| table _time, dest, Country, bytes_out, z, sessions
```
