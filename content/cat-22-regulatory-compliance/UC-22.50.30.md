# 22.50.30 — COPPA §312.8 — Data security and confidentiality: protective-control evidence for children's data

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Monitors data security and confidentiality for children's PII per COPPA §312.8. Combines DLP monitoring with TLS encryption posture checks on systems storing children's data, addressing both access control and encryption requirements.

## Value

§312.8 requires reasonable procedures to protect the confidentiality, security, and integrity of children's personal information. Continuous access monitoring and DLP evidence demonstrates active protection.

## SPL

```spl
index=dlp sourcetype IN (dlp_events, data_access_audit) earliest=-24h
| search data_classification="children_pii" OR data_tag="coppa"
| eval access_authorised=if(action IN ("allowed", "read") AND isnotnull(authorisation_ref), 1, 0)
| eval blocked=if(action IN ("blocked", "denied", "quarantined"), 1, 0)
| stats count AS total_events sum(access_authorised) AS authorised sum(blocked) AS blocked_attempts dc(user) AS distinct_users dc(dest) AS distinct_systems BY system, data_class
| eval unauthorised=total_events - authorised - blocked_attempts
| append [search index=tls sourcetype IN ("stream:tcp", "bro:ssl") dest_port=443 earliest=-24h
  | lookup coppa_data_systems.csv dest OUTPUT coppa_system
  | where coppa_system="true"
  | eval weak_tls=if(ssl_version IN ("SSLv3","TLSv1","TLSv1.0","TLSv1.1"), 1, 0)
  | stats count AS total_conns sum(weak_tls) AS weak_conns BY dest
  | eval system=dest, data_class="encryption_posture", total_events=total_conns, unauthorised=weak_conns, authorised=total_conns-weak_conns, blocked_attempts=0, distinct_users=0, distinct_systems=1]
| where unauthorised>0 OR blocked_attempts>0
| table system, data_class, total_events, authorised, blocked_attempts, unauthorised, distinct_users, distinct_systems
```

## Implementation

(1) Tag data repositories containing children's PII with data_classification=children_pii; (2) ingest DLP and access-audit logs; (3) schedule daily; (4) escalate unauthorised access immediately. Indexes required: `index=access_logs`, `index=dlp`; Sourcetypes: `sourcetype=data_access_audit`, `sourcetype=dlp_events`.

## Visualization

Table of flagged records for investigation; single-value KPI of coverage / compliance percentage; time chart of trend over the last 90 days; drill-down per responsible owner. Key fields: `access_authorised`, `authorised`, `blocked`, `blocked_attempts`, `data_class`, `distinct_systems`.

## Compliance

- **COPPA** 16 CFR 312: §312.8 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for §312.8.

## Known False Positives

Tuning prerequisites: source field mappings must be in place before the UC will produce reliable metrics; early runs will commonly flag unmapped systems until onboarding is completed.
