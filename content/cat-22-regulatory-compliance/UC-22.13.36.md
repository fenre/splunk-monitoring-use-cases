# 22.13.36 — Shared Account Justification Review Queue

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We track shared or functional accounts that logged onto BES assets against a justification register due for periodic review.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode=4624 earliest=-30d
| lookup shared_accounts_register.csv Account_Name OUTPUT shared_owner review_due
| where isnotnull(shared_owner)
| eval due_epoch=strptime(review_due,"%Y-%m-%d")
| where due_epoch < now() OR isnull(review_due)
| stats dc(ComputerName) as systems values(bes_system) as sites by Account_Name, shared_owner, review_due
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Maintain register with review due dates. (2) Quarterly export for access governance. (3) Require re-approval tickets. (4) Remove retired shared accounts from register. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`; Lookups: `shared_accounts_register`, `shared_accounts_register.csv`.

## Visualization

Table (accounts overdue), Timeline (reviews), Single value (count). Key fields: `Account_Name`, `due_epoch`, `review_due`, `shared_owner`, `sites`, `systems`.

## Compliance

- **NERC CIP** current: CIP-007-6 R5 Part 5.3 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-007-6 R5 Part 5.3.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
