# UC-22.10.23 · FHIR / SMART on FHIR App Access to ePHI Scopes

## Description

Monitors OAuth token grants and FHIR resource access so only authorized applications and scopes can read or write clinical data exposed via interoperability APIs.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes interop and sourcetypes epic:fhir_audit, epic:fhir_audit`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ingest authorization server logs with scope strings; (2) maintain app registry with approved scopes; (3) block or revoke tokens via IAM integration for critical hits. Indexes required: `index=interop`; Sourcetypes: `sourcetype=epic:fhir_audit`, `sourcetype=epic:fhir_audit``; Lookups: `fhir_app_register`, `fhir_app_register.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742), HTTP Event Collector (HEC).
• Ensure the following data sources are available: `index=interop` `sourcetype=epic:fhir_audit` or Azure API Management / Apigee logs (`client_id`, `scope`, `patient`, `status_code`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest authorization server logs with scope strings; (2) maintain app registry with approved scopes; (3) block or revoke tokens via IAM integration for critical hits.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=interop sourcetype="epic:fhir_audit" earliest=-24h
| eval scopes=split(scope," ")
| mvexpand scopes
| lookup fhir_app_register.csv client_id OUTPUT approved_scopes_mv
| eval approved=if(isnotnull(mvfind(approved_scopes_mv,scopes)),1,0)
| where approved=0 AND status_code IN ("200","201")
| stats count by client_id, scopes, user_id, patient_id
| sort - count
```
