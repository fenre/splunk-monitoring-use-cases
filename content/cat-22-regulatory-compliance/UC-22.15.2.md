# UC-22.15.2 · IACS Cyber Risk Assessment — Critical OT Asset Exposure

## Description

Surfaces internet-facing or high-CVSS OT assets to inform documented risk treatment and residual risk acceptance for the IACS.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_assets, ot_vuln. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ingest vulnerability exports with OT asset keys; (2) refresh exposure lookup from Claroty/Nozomi topology; (3) align tiers with SL-T; (4) attach results to formal RA register. Indexes required: `index=ot_assets`, `index=ot_vuln`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Industrial Asset Intelligence, Splunk DB Connect (2686) for CMMS.
• Ensure the following data sources are available: `index=ot_assets`, `index=ot_vuln`, optional `ot_dmz_exposure.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ingest vulnerability exports with OT asset keys; (2) refresh exposure lookup from Claroty/Nozomi topology; (3) align tiers with SL-T; (4) attach results to formal RA register.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_assets OR index=ot_vuln earliest=-7d
| eval asset_id=coalesce(device_id, asset_id, host)
| stats values(cve) as cves, max(cvss_base_score) as max_cvss, latest(criticality) as tier by asset_id
| lookup ot_dmz_exposure.csv asset_id OUTPUT exposed_to_untrusted
| where max_cvss>=7.0 OR exposed_to_untrusted="true"
| eval risk_note=case(isnotnull(cves) AND exposed_to_untrusted="true","Treat first: exposure + CVE", exposed_to_untrusted="true","Network exposure", true(),"Critical vulnerability concentration")
| table asset_id, tier, max_cvss, exposed_to_untrusted, risk_note, cves
```
