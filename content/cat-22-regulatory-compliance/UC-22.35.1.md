# 22.35.1 — Audit-log continuity: detect indexing gap indicating lost evidence

**Criticality:** critical | **Difficulty:** intermediate | **Status:** draft

## Description

Detects any sourcetype that was previously producing events but has stopped indexing for more than a rolling 60-second window. A regression in event ingestion is the earliest indicator that the audit trail required by almost every modern regulation has been compromised — whether by accident (agent failure, network partition, licensing quota) or by intent (attacker clearing logs).

## Value

Without continuous audit logs, compliance claims are unfalsifiable. Auditors across GDPR, HIPAA, PCI DSS, SOC 2, and SOX all treat log-gaps as a finding. This UC lets a customer show the auditor a green dashboard: 'here is every second of the last 30 days, accounted for by source.'

## SPL

```spl
| tstats count as events where index=_internal source=*metrics.log* group=per_sourcetype_thruput earliest=-30m by sourcetype _time span=1m
| eventstats avg(events) as avg_events, stdev(events) as stdev_events by sourcetype
| where events=0 AND avg_events>0
| eval gap_seconds=60
| table _time sourcetype events avg_events stdev_events gap_seconds
| sort - _time
```

## Implementation

(1) Schedule every 5 minutes; (2) route hits to a restricted summary index (compliance_summary); (3) wire to ITSI service health or ES notable depending on deployment; (4) maintain a maintenance-window lookup (maintenance_windows.csv) listing tagged sources that should NOT alert. Indexes required: `index=_internal`.

## Visualization

Timeline of event rate per sourcetype with red bars for gap periods; heat-map of sources-by-day with green/amber/red cells; single value for '% of last 30d with full coverage'.

## Compliance

- **GDPR** 2016/679: Art.32(1)(b) (partial) — Detects interruption of the audit trail (the 'ability to ensure the ongoing ... resilience of processing systems'), but the clause also requires confidentiality and integrity guarantees not covered here.
- **HIPAA** 2013-final: §164.312(b) (full) — §164.312(b) 'Audit controls' is satisfied when the covered entity can show audit logs are captured continuously; this UC produces that evidence directly.
- **PCI-DSS** v4.0: 10.2.1 (partial) — PCI 10.2.1 requires that audit logs are enabled and active for all system components; this UC detects loss of telemetry but does not validate 10.2.1's sub-requirements (10.2.1.1-10.2.1.7) for specific event types.
- **SOC-2** 2017 TSC: CC7.2 (partial) — CC7.2 requires monitoring of system components 'to detect anomalies' — detecting audit-log gaps is one necessary signal but the criterion covers broader system monitoring.
- **SOX-ITGC** PCAOB AS 2201: ITGC.Logging.Continuity (contributing) — A material weakness finding for ITGCs often follows from unexplained gaps in the audit trail; this UC raises the signal but SOX evidence requires additional management attestation.
- **UK GDPR** post-Brexit: Art.32(1)(b) (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.32(1)(b).

## Known False Positives

Planned maintenance windows or expected cold-standby sources that receive low volume. Mitigate with a maintenance_windows.csv lookup and a `not source IN [$lookup]` clause.
