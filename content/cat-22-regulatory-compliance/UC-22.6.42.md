# UC-22.6.42 · Web and Email Filtering — Denied High-Risk Categories Toward Young Domains

## Description

Combines proxy and secure email gateway style denies for risky categories and young domains to cover web and messaging filtering controls.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes proxy and sourcetypes zscaler:web. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Feed consistent category taxonomy; (2) user awareness tie-in; (3) monthly filtering effectiveness review. Indexes required: `index=proxy`; Sourcetypes: `sourcetype=zscaler:web`; Lookups: `threat_intel_domain`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Zscaler or Proofpoint HEC.
• Ensure the following data sources are available: `zscaler:web` or `proofpoint:message`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Feed consistent category taxonomy; (2) user awareness tie-in; (3) monthly filtering effectiveness review.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=proxy sourcetype=zscaler:web action=blocked earliest=-24h
| lookup threat_intel_domain domain as hostname OUTPUT first_seen
| eval young=if((now()-first_seen)/86400<30,1,0)
| where young=1 AND match(category,"(?i)malware|phishing|newly.registered")
| stats count by hostname, user
```
