# UC-22.14.55 · Access Restrictions for Change — Privileged Route Changes

## Description

Monitors BGP or routing table changes from privileged accounts for CM-5 access restrictions for change.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes net and sourcetypes cisco:ios. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Enable AAA accounting to syslog; (2) parse `user` field; (3) alert on human-initiated changes without ticket keyword in command comment if used; (4) integrate RANCID/Oxidized diffs optional. Indexes required: `index=net`; Sourcetypes: `sourcetype=cisco:ios`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Network device TAs (Cisco IOS, Juniper), syslog.
• Ensure the following data sources are available: `cisco:ios` routing syslog messages.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enable AAA accounting to syslog; (2) parse `user` field; (3) alert on human-initiated changes without ticket keyword in command comment if used; (4) integrate RANCID/Oxidized diffs optional.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=net sourcetype=cisco:ios earliest=-7d
| search "%BGP-5-ADJCHANGE" OR "%SYS-5-CONFIG_I" OR "neighbor"
| stats count by user, host, command
| where count>0 AND NOT user IN ("svc_net_automation")
```
