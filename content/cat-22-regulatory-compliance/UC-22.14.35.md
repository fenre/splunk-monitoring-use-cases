# UC-22.14.35 · Malicious Code Protection — EDR / AV Detections Volume and Gaps

## Description

Monitors endpoint malware detection pipelines for silence or surges for SI-3 malicious code protection effectiveness.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes endpoint and sourcetypes XmlWinEventLog:Microsoft-Windows-Sysmon/Operational. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Baseline per host class; (2) exclude offline laptops with lookup; (3) alert on sustained silence; (4) cross-check agent health from MDM. Indexes required: `index=endpoint`; Sourcetypes: `sourcetype=XmlWinEventLog:Microsoft-Windows-Sysmon/Operational`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Sysmon (community/TA), CrowdStrike Falcon Event Streams (TA), or CIM Malware sources.
• Ensure the following data sources are available: `XmlWinEventLog:Microsoft-Windows-Sysmon/Operational` EventCode=1 with AV vendor logs, `malware` CIM-tagged data.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Baseline per host class; (2) exclude offline laptops with lookup; (3) alert on sustained silence; (4) cross-check agent health from MDM.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=endpoint earliest=-24h (sourcetype=XmlWinEventLog:Microsoft-Windows-Sysmon/Operational OR tag=malware)
| bin _time span=1h
| stats count by _time, host
| eventstats median(count) as med by host
| eval silent=if(count < med*0.1 AND med>10,1,0)
| where silent=1
```
