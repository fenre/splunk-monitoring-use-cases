# UC-22.14.8 · Protection of Audit Information — Tamper Detection on Audit Indexes

## Description

Monitors `_audit` for destructive actions against searches, indexes, and roles that could undermine AU-9 protection of audit information.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes _audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ensure `_audit` is indexed and retained per policy; (2) restrict who can delete `_audit`; (3) forward `_audit` to immutable store if required; (4) integrate alerts with SOC for Splunk admin changes. Indexes required: `index=_audit`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: `index=_audit` (delete, edit, permissions changes).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ensure `_audit` is indexed and retained per policy; (2) restrict who can delete `_audit`; (3) forward `_audit` to immutable store if required; (4) integrate alerts with SOC for Splunk admin changes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_audit earliest=-24h
| search object="*savedsearches*" OR object="*indexes*" OR object="*authorize*"
| stats values(action) as actions, values(user) as users, count by object
| sort - count
```
