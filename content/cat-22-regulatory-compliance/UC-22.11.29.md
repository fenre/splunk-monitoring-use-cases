# UC-22.11.29 · Scheduled Malware Scan or On-Demand Scan Failures

## Description

Captures failed or skipped AV scans on systems that can affect account data so periodic and real-time detection requirements remain demonstrable during operational reviews.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes windows and sourcetypes WinEventLog:Microsoft-Windows-Windows, WinEventLog:Microsoft-Windows-Windows Defender/Operational. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map Defender EventCodes to human-readable descriptions via lookup. (2) Include legacy AV sourcetypes in append. (3) Alert on any failure in CDE. (4) Track re-scan completion in ITSM. Indexes required: `index=windows`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=windows` `WinEventLog:Application` vendor AV EventCode; `sourcetype="WinEventLog:Microsoft-Windows-Windows Defender/Operational"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map Defender EventCodes to human-readable descriptions via lookup. (2) Include legacy AV sourcetypes in append. (3) Alert on any failure in CDE. (4) Track re-scan completion in ITSM.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=windows sourcetype="WinEventLog:Microsoft-Windows-Windows Defender/Operational" earliest=-7d
| lookup pci_windows_cde_assets.csv ComputerName OUTPUT cde_member
| where cde_member="true"
| where EventCode IN (1008,1010,5001,5007)
| stats count by ComputerName, EventCode, EventDescription
| sort - count
```
