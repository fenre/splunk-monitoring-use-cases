# UC-22.1.35 · GDPR DPIA Remediation Monitoring and Mitigation Closure

## Description

The DPIA must include measures to address risks. This use case ties DPIA mitigation items to ITSM work orders, tracks age and overdue status, and verifies technical completion signals (e.g., control deployment logs) where available.

## Value

Gives the responsible team auditor-ready evidence for GDPR compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:sc_task. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Export mitigation action list from each approved DPIA into `gdpr_dpia_mitigation_register.csv`; (2) create linked ServiceNow tasks with due dates; (3) alert owners weekly on overdue items; (4) optionally confirm technical controls via GitHub merge events or Terraform apply logs; (5) close DPIA only when all mitigations are verified. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:sc_task`.

Step 1 — Configure data collection
(1) Export mitigation action list from each approved DPIA into `gdpr_dpia_mitigation_register.csv`; (2) create linked ServiceNow tasks with due dates; (3) alert owners weekly on overdue items; (4) optionally confirm technical controls via GitHub merge events or Terraform apply logs; (5) close DPIA only when all mitigations are verified.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup gdpr_dpia_mitigation_register.csv
| rename mitigation_task as task_number
| join task_number [
    search index=itsm sourcetype="snow:sc_task" earliest=-180d
    | eval done=if(state IN ("Closed","Resolved"),1,0)
    | stats latest(state) as state latest(_time) as last_update max(done) as closed by number
    | rename number as task_number
]
| eval age_days=round((now()-strptime(due_date,"%Y-%m-%d"))/86400,0)
| where closed=0 AND age_days > 0
| table processing_activity, mitigation_description, task_number, state, due_date, age_days
| sort - age_days
```
