# UC-22.11.13 · Security Parameter Drift on In-Scope Routers from Gold Config Hash

## Description

Compares periodic configuration snapshots to an approved gold hash per device so unintended drift in security-relevant parameters is detected between formal audits.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes compliance and sourcetypes compliance:network_config_digest. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Nightly job pushes SHA-256 of normalized running-config. (2) Store `baseline_hash` only after CAB approval. (3) Auto-create incident on drift. (4) Retain hash history 13 months for PCI logging retention alignment. Indexes required: `index=compliance`; Sourcetypes: `sourcetype=compliance:network_config_digest`; Lookups: `pci_perimeter_devices`, `pci_perimeter_devices.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=compliance sourcetype="compliance:network_config_digest" earliest=-48h
| lookup pci_perimeter_devices.csv device OUTPUT pci_tier
| where pci_tier IN ("cde_border","dmz")
| eval drift=if(config_hash!=baseline_hash,1,0)
| where drift=1
| stats latest(_time) as last_drift by device, config_hash, baseline_hash
| sort device
```
