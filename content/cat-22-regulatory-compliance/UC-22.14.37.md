# UC-22.14.37 · Security Alerts Ingestion Health from Vendor Feeds

## Description

Ensures security advisory and vendor alert feeds continue arriving for SI-5 security alerts, advisories, and directives.

## Value

Ensures security advisory and vendor alert feeds continue arriving for SI-5 security alerts, advisories, and directives.

## Implementation

(1) Standardize `feed_name` at ingest; (2) alert per feed SLA; (3) document alternate notification path; (4) map to IR-4 playbooks when vendor declares incident.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for TAXII (if used), vendor JSON via HEC, RSS parser custom.
• Ensure the following data sources are available: `index=threat_intel` OR `sourcetype=vendor:security_advisory`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Standardize `feed_name` at ingest; (2) alert per feed SLA; (3) document alternate notification path; (4) map to IR-4 playbooks when vendor declares incident.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=threat_intel earliest=-7d
| stats latest(_time) as last_event, count by sourcetype, feed_name
| eval hours_since=round((now()-last_event)/3600,1)
| where hours_since>48
```

Understanding this SPL

**Pipeline walkthrough**

• Scopes the data: index=threat_intel, time bounds. Cross-check against **Data sources** above so indexes and sourcetypes match your ingestion.
• `stats` rolls up events into metrics; results are split **by sourcetype, feed_name** so each row reflects one combination of those dimensions (useful for per-host, per-user, or per-entity comparisons for this use case).
• `eval` defines or adjusts **hours_since** — often to normalize units, derive a ratio, or prepare for thresholds.
• Filters the current rows with `where hours_since>48` — typically the threshold or rule expression for this monitoring goal.
