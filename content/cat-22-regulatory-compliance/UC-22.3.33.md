# UC-22.3.33 · DORA Outsourcing Registers — Function Mapping Completeness for Each Outsourced Arrangement

## Description

Registers must reflect which functions depend on each provider. Flagging contracts without mapped internal business functions prevents hollow register entries that fail supervisory scrutiny during inspections.

## Value

Registers must reflect which functions depend on each provider. Flagging contracts without mapped internal business functions prevents hollow register entries that fail supervisory scrutiny during inspections.

## Implementation

(1) Define allowed `function_codes` taxonomy; (2) workflow assigns business owner per contract; (3) integrate procurement go-live events; (4) quarterly validation campaign; (5) tie to BIA criticality.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup dora_outsourcing_register.csv
| eval fc_len=len(function_codes)
| eval validated=strptime(last_validated,"%Y-%m-%d")
| where fc_len<5 OR isnull(function_codes) OR validated<relative_time(now(),"-365d@d")
| table contract_id, provider_id, function_codes, last_validated
```
