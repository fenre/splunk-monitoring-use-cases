# UC-22.11.42 · Access Request and Approval Completeness for CDE VPN Accounts

## Description

Verifies each new VPN entitlement for CDE users has a matching approved access request record so access is granted only with documented business justification.

## Value

Verifies each new VPN entitlement for CDE users has a matching approved access request record so access is granted only with documented business justification.

## Implementation

(1) Normalize `user` to corporate UPN. (2) Align RITM naming with join filter. (3) Alert IAM on missing ticket within 24h of first connect. (4) Monthly reconciliation report.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757), Splunk Add-on for ServiceNow (1928).
• Ensure the following data sources are available: `index=netfw` GlobalProtect or `sourcetype="pan:globalprotect"`; `index=itsm` `snow:sc_req_item`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize `user` to corporate UPN. (2) Align RITM naming with join filter. (3) Alert IAM on missing ticket within 24h of first connect. (4) Monthly reconciliation report.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=netfw sourcetype="pan:globalprotect" subtype="gateway-connected" earliest=-7d
| lookup pci_cde_vpn_profiles.csv portal OUTPUT cde_profile
| where cde_profile="true"
| eval user=lower(user)
| join type=left max=1 user [
    search index=itsm sourcetype="snow:sc_req_item" earliest=-30d short_description="*VPN*CDE*"
    | eval user=lower(caller_id)
    | stats latest(state) as req_state by user
  ]
| where isnull(req_state) OR NOT match(req_state,"(?i)closed|fulfilled")
| stats dc(_time) as connect_events by user, source
| sort - connect_events
```
