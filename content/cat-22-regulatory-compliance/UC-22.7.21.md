# UC-22.7.21 · Lessons Learned — Post-Incident Analyst Search Activity

## Description

Tracks analyst searches for IOCs after incidents to evidence continuous improvement and knowledge incorporation.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes _audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Tune regex to incident naming convention; (2) exclude bots; (3) include in AAR pack; (4) map to training gaps. Indexes required: `index=_audit`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise (`_audit`).
• Ensure the following data sources are available: `index=_audit` action=search.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tune regex to incident naming convention; (2) exclude bots; (3) include in AAR pack; (4) map to training gaps.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_audit action=search info=completed earliest=-90d
| where match(search,"(?i)ioc|threat_intel|notable")
| bucket _time span=7d as week
| stats dc(user) as analysts, count by week
```
