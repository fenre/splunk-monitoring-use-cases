# UC-22.10.15 · Automatic Logoff — Stale Clinical Workstation Sessions

## Description

Identifies unusually long Citrix/VDI sessions and unlocked Windows consoles in patient care areas where automatic logoff may be misconfigured.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes citrix and sourcetypes citrix:session. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align with published idle timeout policy (e.g., 30 min); (2) tune for legitimate long shifts using department lookup; (3) feed findings to desktop engineering for GPO fixes. Indexes required: `index=citrix`; Sourcetypes: `sourcetype=citrix:session`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Citrix (2757), Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=citrix` session start/end, `WinEventLog:Security` 4800/4801 (workstation lock).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align with published idle timeout policy (e.g., 30 min); (2) tune for legitimate long shifts using department lookup; (3) feed findings to desktop engineering for GPO fixes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=citrix sourcetype="citrix:session" earliest=-7d
| eval start=strptime(SessionStart,"%Y-%m-%d %H:%M:%S")
| eval end=if(isnotnull(SessionEnd), strptime(SessionEnd,"%Y-%m-%d %H:%M:%S"), now())
| eval session_hours=round((end-start)/3600,2)
| where session_hours>12 AND match(Application,"Hyperspace|Epic")
| stats max(session_hours) as max_hrs by UserName, client_ip, PublishedApplication
| sort - max_hrs
```
