# UC-22.14.70 · Cryptographic Key Management Events from Cloud KMS

## Description

Audits KMS key disable/delete operations for SC-12 cryptographic key establishment and management.

## Value

Audits KMS key disable/delete operations for SC-12 cryptographic key establishment and management.

## Implementation

(1) Parse `requestParameters` for key ARNs into fields; (2) alert outside maintenance CAB; (3) dual-control evidence from SNOW comments optional join; (4) map to SC-13.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for AWS (Splunkbase 1876), Splunk Add-on for GCP (TA).
• Ensure the following data sources are available: `aws:cloudtrail` KMS events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Parse `requestParameters` for key ARNs into fields; (2) alert outside maintenance CAB; (3) dual-control evidence from SNOW comments optional join; (4) map to SC-13.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cloud sourcetype=aws:cloudtrail earliest=-30d eventSource="kms.amazonaws.com" eventName IN ("DisableKey","ScheduleKeyDeletion","DeleteImportedKeyMaterial")
| stats values(eventName) as actions, latest(_time) as last by userName, requestParameters
```
