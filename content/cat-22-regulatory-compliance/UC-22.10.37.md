# UC-22.10.37 · After-Hours ePHI Access — Billing Users on Inpatient Charts

## Description

Surfaces patient chart access during overnight windows by revenue cycle roles where business need is uncommon, supporting workforce training and sanctions workflows.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes epic and sourcetypes epic:audit, epic:audit`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align dept codes with HR feed; (2) add on-call exceptions; (3) integrate with privacy investigation templates. Indexes required: `index=epic`; Sourcetypes: `sourcetype=epic:audit`, `sourcetype=epic:audit``; Lookups: `hipaa_department_roles`, `hipaa_department_roles.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=epic` `sourcetype=epic:audit`, `lookup hipaa_department_roles.csv` (`USER_ID`, `dept`, `after_hours_allowed`).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align dept codes with HR feed; (2) add on-call exceptions; (3) integrate with privacy investigation templates.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=epic sourcetype="epic:audit" AccessType="PatientChart" earliest=-30d
| eval hour=tonumber(strftime(_time,"%H"))
| where hour>=22 OR hour<=5
| lookup hipaa_department_roles.csv USER_ID OUTPUT dept, after_hours_allowed
| where dept="REVENUE_CYCLE" AND after_hours_allowed!="true"
| stats dc(PAT_ID) as patients by USER_ID, dept
| sort - patients
```
