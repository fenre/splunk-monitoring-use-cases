# UC-22.15.26 · Dual-Signature PLC Program Change Without Four-Eyes Ticket

## Description

Looks for PLC program uploads/downloads signed by only one engineering identity when policy requires dual control.

## Value

Looks for PLC program uploads/downloads signed by only one engineering identity when policy requires dual control.

## Implementation

(1) Require second approval in CM tool; (2) map PLC audit user fields; (3) integrate e-signature if available; (4) weekly compliance report.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Vendor PLC audit, ServiceNow.
• Ensure the following data sources are available: `index=ot_eng` `sourcetype="plc:audit"`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require second approval in CM tool; (2) map PLC audit user fields; (3) integrate e-signature if available; (4) weekly compliance report.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_eng sourcetype="plc:audit" action="PROGRAM_CHANGE" earliest=-30d
| eval change_id=coalesce(change_ticket, work_order)
| stats values(user) as signers, dc(user) as signer_count by change_id, controller
| where signer_count<2 AND len(change_id)>4
| table change_id, controller, signers, signer_count
```
