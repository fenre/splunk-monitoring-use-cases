# 22.15.26 — Dual-Signature PLC Program Change Without Four-Eyes Ticket

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Looks for PLC program uploads/downloads signed by only one engineering identity when policy requires dual control.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_eng and sourcetypes plc:audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_eng sourcetype="plc:audit" action="PROGRAM_CHANGE" earliest=-30d
| eval change_id=coalesce(change_ticket, work_order)
| stats values(user) as signers, dc(user) as signer_count by change_id, controller
| where signer_count<2 AND len(change_id)>4
| table change_id, controller, signers, signer_count
```

## CIM SPL

```spl
| tstats summariesonly=t dc(All_Changes.user) as agg_value from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - agg_value
```

## Implementation

(1) Require second approval in CM tool; (2) map PLC audit user fields; (3) integrate e-signature if available; (4) weekly compliance report. Indexes required: `index=ot_eng`; Sourcetypes: `sourcetype=plc:audit`.

## Visualization

Table, Timeline, Single value (violations). Key fields: `change_id`, `controller`, `signer_count`, `signers`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 2.12 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 2.12.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
