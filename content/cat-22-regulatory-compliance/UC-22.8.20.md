# UC-22.8.20 · SOC 2 CC7 — Unauthorized Production Configuration Change Detection

## Description

CC7 addresses system operations. Detecting changes to firewall rules, cloud security groups, or database parameters without a matching approved change ticket demonstrates monitoring over unauthorized modifications.

## Value

CC7 addresses system operations. Detecting changes to firewall rules, cloud security groups, or database parameters without a matching approved change ticket demonstrates monitoring over unauthorized modifications.

## Implementation

(1) Replace simplified join with CMDB-linked correlation on `requestParameters.groupId`; (2) allow emergency change workflow via `userIdentity.arn` lookup `breakglass_arns.csv`; (3) send to SOC and ITIL process owner; (4) retain CloudTrail in WORM bucket; (5) map events to CC7-2 subservice in system description.

Step 1 — Configure data collection
(1) Replace simplified join with CMDB-linked correlation on `requestParameters.groupId`; (2) allow emergency change workflow via `userIdentity.arn` lookup `breakglass_arns.csv`; (3) send to SOC and ITIL process owner; (4) retain CloudTrail in WORM bucket; (5) map events to CC7-2 subservice in system description.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=aws sourcetype="aws:cloudtrail" eventName IN ("AuthorizeSecurityGroupIngress","RevokeSecurityGroupIngress","ModifyDBClusterParameterGroup") earliest=-7d
| eval change_key=md5(_raw)
| join type=left change_key [
    search index=itsm sourcetype="snow:change_request" earliest=-14d
    | eval change_key=md5(number+cmdb_ci)
    | fields number
  ]
| where isnull(number)
| stats count by eventName, userIdentity.arn
| sort - count
```
