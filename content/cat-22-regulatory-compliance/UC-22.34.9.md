# 22.34.9 — Intrusion detection coverage within SWIFT environment

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Produces SWIFT Customer Security Programme evidence for intrusion detection coverage within swift environment ahead of independent assessment.

## Value

Gives the responsible team auditor-ready evidence for SWIFT CSP compliance by automating regulatory compliance evidence using indexes ids and sourcetypes pan:threat, suricata:event. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ids sourcetype IN ("suricata:event","pan:threat") tag=swift earliest=-24h
| stats count by signature, dest_ip
| lookup swift_vlan_map.csv dest_ip OUTPUT in_swift_vlan
| where in_swift_vlan="true"
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.signature, Authentication.dest | sort - count
```

## Implementation

(1) Segment SWIFT CSP evidence into dedicated indexes with strict role-based access. (2) Join operators to HR screening and vendor risk lookups refreshed weekly. (3) Validate MFA and PAM sourcetypes are CIM-mapped for Authentication models. (4) Retain evidence for at least 13 months aligned to attestation cycles. Indexes required: `index=ids`.

## Visualization

Table (operators), Single value (sessions), Bar chart (CSCF controls). Key fields: `dest_ip`, `signature`.

## Compliance

- **SWIFT CSP** CSCF v2025: 1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
