# UC-22.7.26 · Encryption in Transit — Deprecated TLS on Internal APIs

## Description

Parses load balancer or proxy access logs for deprecated TLS handshakes supporting data security requirements.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes proxy and sourcetypes nginx:access. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Confirm field extraction; (2) emergency patch plan for legacy clients; (3) document compensating network segmentation. Indexes required: `index=proxy`; Sourcetypes: `sourcetype=nginx:access`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: nginx, F5, or AWS ALB access logs.
• Ensure the following data sources are available: `nginx:access` ssl_protocol.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Confirm field extraction; (2) emergency patch plan for legacy clients; (3) document compensating network segmentation.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=proxy sourcetype=nginx:access earliest=-24h
| where ssl_protocol IN ("TLSv1","TLSv1.1","SSLv3")
| stats count by src_ip, ssl_protocol, uri
```
