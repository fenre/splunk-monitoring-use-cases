# UC-22.13.34 · Failed and Successful Login Attempt Monitoring on BES Assets

## Description

We summarize authentication successes and failures on BES Cyber Assets to support account lockout and misuse investigations.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ensure 4625 auditing enabled. (2) Add threshold alerts for failures. (3) Correlate with password spray use cases. (4) Retain per entity log policy. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`; Lookups: `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=wineventlog` `EventCode` 4624/4625, `bes_cyber_asset_inventory.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ensure 4625 auditing enabled. (2) Add threshold alerts for failures. (3) Correlate with password spray use cases. (4) Retain per entity log policy.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode IN (4624,4625) earliest=-24h
| lookup bes_cyber_asset_inventory.csv ComputerName AS host OUTPUT bes_system
| where isnotnull(bes_system)
| eval outcome=if(EventCode==4625,"fail","success")
| stats count by Account_Name, ComputerName, outcome, bes_system
| sort - count
```
