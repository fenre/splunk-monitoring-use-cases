# UC-22.13.16 · ESP Outbound Bytes Spike versus Rolling Baseline

## Description

We surface unusual outbound volume from ESP-internal segments that may indicate data exfiltration or misconfigured services.

## Value

We surface unusual outbound volume from ESP-internal segments that may indicate data exfiltration or misconfigured services.

## Implementation

(1) Tune multiplier and internal CIDRs. (2) Exclude CDN and update destinations via lookup. (3) Tag BES Cyber Assets on `src`. (4) Pair with PCAP workflow if available.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757).
• Ensure the following data sources are available: `index=network` `sourcetype=pan:traffic` (bytes_out, src, dest, src_zone).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tune multiplier and internal CIDRs. (2) Exclude CDN and update destinations via lookup. (3) Tag BES Cyber Assets on `src`. (4) Pair with PCAP workflow if available.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype=pan:traffic src_zone="ESP_INTERNAL" action=allowed earliest=-30d
| eval internal_dest=cidrmatch("10.0.0.0/8",dest) OR cidrmatch("172.16.0.0/12",dest) OR cidrmatch("192.168.0.0/16",dest)
| where internal_dest=0
| bin _time span=1d
| stats sum(bytes_out) as b_out by src, dest, _time
| eventstats median(b_out) as med by src, dest
| eval spike=if(b_out>med*8 AND med>0,1,0)
| where spike=1
| table _time, src, dest, b_out, med
```
