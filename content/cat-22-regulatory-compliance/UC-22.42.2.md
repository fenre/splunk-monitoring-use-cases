# UC-22.42.2 · Configuration baseline drift — regulated hosts deviating from CIS benchmark

## Description

Continuously compares live configuration to the approved CIS / osquery baseline. Only deviations on regulated systems are escalated to avoid alert fatigue.

## Value

Turns baseline drift from a quarterly scan artefact into a continuous control that auditors can review any day.

## Implementation

(1) Schedule osquery/CIS-CAT agents; (2) publish baselines via config-management tooling; (3) run daily; (4) on finding, open a P2 ticket.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=baseline sourcetype=osquery:cis OR sourcetype=ciscat earliest=-24h severity>=medium
| join host [search index=asset regulated=true | fields host tenant classification]
| table host tenant classification rule_id severity deviation
```
