# UC-22.14.32 · Re-Authentication for Sensitive Application Roles

## Description

Measures time since last strong authentication for high-risk app roles supporting IA-11 re-authentication requirements.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes o365 and sourcetypes ms:o365:management. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Replace ApplicationId tokens with production GUIDs; (2) align session length to policy; (3) pair with CASB step-up events if available; (4) document exceptions for legacy apps. Indexes required: `index=o365`; Sourcetypes: `sourcetype=ms:o365:management`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Office 365 (Splunkbase 4055).
• Ensure the following data sources are available: `ms:o365:management` sign-ins with `AuthenticationRequirement` / `SessionId` if present.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Replace ApplicationId tokens with production GUIDs; (2) align session length to policy; (3) pair with CASB step-up events if available; (4) document exceptions for legacy apps.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=o365 sourcetype=ms:o365:management earliest=-24h
| search ApplicationId IN ("*high_risk_app_guid_1*","*high_risk_app_guid_2*")
| stats earliest(_time) as first_seen, latest(_time) as last_seen by UserId, SessionId
| eval session_age_min=round((now()-last_seen)/60,1)
| where session_age_min>480
```
