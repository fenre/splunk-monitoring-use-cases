# UC-22.14.21 · System Use Notification Banner Acceptance in SSH Sessions

## Description

Verifies SSH banner or interactive MOTD markers appear in session logs for AC-8 system use notification where technically feasible.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes os and sourcetypes linux:secure. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Standardize sshd_config `Banner` path; (2) optionally emit structured JSON from bastion; (3) do not log full banner text if classified; (4) report exceptions monthly. Indexes required: `index=os`; Sourcetypes: `sourcetype=linux:secure`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Unix and Linux (Splunkbase 833).
• Ensure the following data sources are available: `linux:secure` sshd session lines.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Standardize sshd_config `Banner` path; (2) optionally emit structured JSON from bastion; (3) do not log full banner text if classified; (4) report exceptions monthly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=os sourcetype=linux:secure earliest=-7d process=sshd
| search "Accepted" OR "session opened"
| rex "(?i)banner=(?<banner_ack>yes|no)"
| stats count by banner_ack, dest
| where banner_ack="no" OR isnull(banner_ack)
```
