# 22.22.26 — Operational vs Security Incident Classification Consistency

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Supports incident governance, customer communications evidence, and supervisory reporting workflows associated with payment service continuity and fraud statistics.

## Value

Gives the responsible team auditor-ready evidence for PSD2 compliance by automating regulatory compliance evidence using indexes payments and sourcetypes pay:incident:milestone. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=payments sourcetype="pay:incident:milestone" earliest=-90d
| eval inc_tag="class"
| where isnotnull(security_class) AND isnotnull(ops_class)
| eval class_mismatch=if(security_class!=ops_class,1,0)
| stats sum(class_mismatch) as mismatches, count as events by aspsp_id, severity
| eval uc_id="22.22.26"
| where mismatches>0
| sort - mismatches
| table uc_id, aspsp_id, severity, mismatches, events
```

## Implementation

(1) Normalize field extractions and CIM tags; (2) schedule hourly or near-real-time; (3) route alerts to SOC/compliance queues; (4) export evidence packs for supervisory review. Indexes required: `index=payments`; Sourcetypes: `sourcetype=pay:incident:milestone`.

## Visualization

Timeline (milestones), Table (late NCA notifications), Single value (mismatches)

## Compliance

- **PSD2** Directive (EU) 2015/2366: Art.96 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.96.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
