# 22.31.8 — User application hardening compliance

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Evidences ASD Essential Eight obligations for user application hardening compliance using endpoint, identity, and cloud telemetry.

## Value

Gives the responsible team auditor-ready evidence for ASD E8 compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode=4673 earliest=-24h
| stats count by ProcessName, ServiceName
| lookup asd_e8_scope.csv host OUTPUT in_scope
| where in_scope="true"
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.process_name, Processes.user, Processes.dest | sort - count
```

(3) Coordinate privacy and security ownership for dual-reporting workflows. (4) Monthly export for privacy officer and prudential working groups. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`; Lookups: `asd_e8_scope`, `asd_e8_scope.csv`.
