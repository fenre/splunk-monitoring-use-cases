# 22.2.29 — NIS2 Cryptography Policy Compliance — TLS and Certificate Posture

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Appropriate use of cryptography protects network and information system security. This use case monitors TLS versions, weak cipher suites, and certificate expiry for public and internal services supporting NIS2-scoped workloads — aligning with organisational cryptography standards.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes stream and sourcetypes stream:tls. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=stream sourcetype="stream:tls" earliest=-24h
| stats latest(tls_version) as tls_version latest(cipher) as cipher values(expiry_epoch) as cert_expiry by dest_ip dest_port
| lookup nis2_scoped_service_map dest_ip OUTPUT service_name nis2_in_scope
| where nis2_in_scope="true"
| eval weak_tls=if(match(tls_version,"SSLv|TLSv1\.0|TLSv1\.1"),1,0)
| eval cert_problem=if(cert_expiry < relative_time(now(),"+30d@d"),1,0)
| where weak_tls=1 OR cert_problem=1
| table service_name, dest_ip, dest_port, tls_version, cipher, cert_problem
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Certificates.All_Certificates by All_Certificates.ssl_subject_common_name, All_Certificates.dest | sort - count
```

## Implementation

(1) Enable passive TLS inspection on critical VLAN mirrors; (2) map IPs to NIS2 services; (3) alert on weak protocols and imminent expiry; (4) integrate with PKI renewal workflow; (5) document exceptions with compensating controls. Indexes required: `index=stream`; Sourcetypes: `sourcetype=stream:tls`; Lookups: `nis2_scoped_service_map`.

## Visualization

Table (weak TLS services), Timeline (certificate expiry), Single value (critical expiries), Bar chart (by cipher family).

## Compliance

- **NIS2** Directive (EU) 2022/2555: Art.21(2)(h) (partial) — Uplifted from contributing: SPL directly addresses Art.21(2)(h) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
