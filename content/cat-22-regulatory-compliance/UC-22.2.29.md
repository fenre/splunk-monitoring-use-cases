# UC-22.2.29 · NIS2 Cryptography Policy Compliance — TLS and Certificate Posture

## Description

Appropriate use of cryptography protects network and information system security. This use case monitors TLS versions, weak cipher suites, and certificate expiry for public and internal services supporting NIS2-scoped workloads — aligning with organisational cryptography standards.

## Value

Appropriate use of cryptography protects network and information system security. This use case monitors TLS versions, weak cipher suites, and certificate expiry for public and internal services supporting NIS2-scoped workloads — aligning with organisational cryptography standards.

## Implementation

(1) Enable passive TLS inspection on critical VLAN mirrors; (2) map IPs to NIS2 services; (3) alert on weak protocols and imminent expiry; (4) integrate with PKI renewal workflow; (5) document exceptions with compensating controls.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=stream sourcetype="stream:tls" earliest=-24h
| stats latest(tls_version) as tls_version latest(cipher) as cipher values(expiry_epoch) as cert_expiry by dest_ip dest_port
| lookup nis2_scoped_service_map dest_ip OUTPUT service_name nis2_in_scope
| where nis2_in_scope="true"
| eval weak_tls=if(match(tls_version,"SSLv|TLSv1\.0|TLSv1\.1"),1,0)
| eval cert_problem=if(cert_expiry < relative_time(now(),"+30d@d"),1,0)
| where weak_tls=1 OR cert_problem=1
| table service_name, dest_ip, dest_port, tls_version, cipher, cert_problem
```
