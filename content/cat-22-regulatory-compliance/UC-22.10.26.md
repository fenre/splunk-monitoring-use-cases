# UC-22.10.26 · Transmission Security — Unencrypted SMTP with PHI Patterns

## Description

Alerts when internal mail relays accept messages on port 25 without TLS that contain medical record number or DOB patterns, evidencing safeguards for ePHI in motion.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes mail and sourcetypes proofpoint:mta, proofpoint:mta`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Prefer DLP classification fields when available instead of regex on body; (2) route to messaging team for forced TLS connectors; (3) document exceptions for legacy lab devices with compensating controls. Indexes required: `index=mail`; Sourcetypes: `sourcetype=proofpoint:mta`, `sourcetype=proofpoint:mta``.

## Detailed Implementation

Prerequisites

• Ensure the following data sources are available: `index=mail` `sourcetype=proofpoint:mta` or `ms:o365:reporting` MessageTrace, `stream:smtp` metadata.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Prefer DLP classification fields when available instead of regex on body; (2) route to messaging team for forced TLS connectors; (3) document exceptions for legacy lab devices with compensating controls.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mail sourcetype="proofpoint:mta" earliest=-24h
| eval tls_used=if(match(lower(smtp_tls_status),"none|no|plaintext"),0,1)
| regex subject="(?i)\b(MRN|medical record|DOB|date of birth)\b"
| where tls_used=0
| stats count values(rcpt) as recipients by sender, subject, client_ip
| sort - count
```
