# 22.31.13 — Office macro execution policy violation detection

**Criticality:** high | **Difficulty:** intermediate | **Status:** verified

## Description

Evidences ASD Essential Eight obligations for office macro execution policy violation detection using endpoint, identity, and cloud telemetry.

## Value

Gives the responsible team auditor-ready evidence for ASD E8 compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode=4660 earliest=-24h
| search Object_Name="*.xlsm"
| stats count by user, Object_Name
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Endpoint.Processes by Processes.user | sort - count
```

## Implementation

(1) Ingest CrowdStrike policy posture via TA 5082 and map hosts to NDB and NZISM tiers. (2) Tune alerts with ACSD Essential Eight and APRA-aligned severities. (3) Coordinate privacy and security ownership for dual-reporting workflows. (4) Monthly export for privacy officer and prudential working groups. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`.

## Visualization

Single value (% compliant hosts), Bar chart (fails by control), Table (worst hosts). Key fields: `Object_Name`, `user`.

## Compliance

- **ASD E8** Nov 2023: E8.03 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for E8.03.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
