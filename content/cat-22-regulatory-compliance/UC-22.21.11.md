# 22.21.11 — Human Override Action Logging Coverage (Art. 14)

**Criticality:** critical | **Difficulty:** intermediate | **Status:** verified

## Description

Measures fraction of automated decisions overridden by human operators with mandatory reason codes for Art. 14 human oversight.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_hil and sourcetypes ai:human:override, ai:human:override`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_hil sourcetype="ai:human:override" earliest=-24h
| eval reason_ok=if(match(reason_code,"^[A-Z]{2,4}_\d{2,4}$"),1,0)
| stats count as overrides, sum(reason_ok) as with_valid_reason by application
| eval reason_pct=round(100*with_valid_reason/overrides,2)
| where overrides>0 AND reason_pct < 100
| sort + reason_pct
```

## Implementation

(1) Enforce reason_code enum in UI; (2) map operator_id to HR directory via lookup; (3) alert on missing reason_code; (4) retain 24-month archive per policy. Indexes required: `index=ai_hil`; Sourcetypes: `sourcetype=ai:human:override`, `sourcetype=ai:human:override``.

## Visualization

Bar chart (overrides by application), Table (invalid reason), Single value (reason_pct)

## Compliance

- **EU AI Act** Regulation (EU) 2024/1689: Art.14 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Art.14.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
