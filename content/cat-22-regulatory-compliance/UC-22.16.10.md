# 22.16.10 — Vendor account usage outside approved maintenance windows

**Criticality:** medium | **Difficulty:** expert | **Status:** N/A

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "Vendor account usage outside approved maintenance windows".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype=WinEventLog:Security earliest=-30d EventCode=4624
| eval uc_tag="22.16.10"
| lookup vendor_accounts.csv Account_Name OUTPUT vendor_name, allowed_window_cron
| where isnotnull(vendor_name)
| eval ok=if(match(strftime(_time,"%u %H"),allowed_window_cron),1,0)
| where ok=0
| stats count by Account_Name, vendor_name
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=wineventlog`.

## Visualization

Time chart, Choropleth or site bar, Table, Single value. Key fields: `Account_Name`, `ok`, `uc_tag`, `vendor_name`.

## Compliance

- **TSA SD** SD02C: III.A (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for III.A.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
