# 22.30.3 — Privileged access management review evidence

**Criticality:** low | **Difficulty:** advanced | **Status:** N/A

## Description

Supports MAS TRM supervisory examination readiness for privileged access management review evidence.

## Value

Gives the responsible team auditor-ready evidence for MAS TRM compliance by automating regulatory compliance evidence using indexes bank and sourcetypes pam:session. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=bank sourcetype="pam:session" regulator="MAS" earliest=-24h
| stats dc(src_ip) as ips by privileged_user, target_system
| where ips>5
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=bank`; Sourcetypes: `sourcetype=pam:session`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones). Key fields: `ips`, `privileged_user`, `target_system`.

## Compliance

- **MAS TRM** 2021: §4.1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for §4.1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
