# UC-22.6.17 · Personnel Screening — Contractor Badge Activations Before Background Check Complete

## Description

Joins physical access grant events to HR screening completion dates to reduce insider risk from premature access.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes physical and sourcetypes pacs:access. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Align badge_id keys; (2) emergency access workflow exception lookup; (3) annual ISO internal audit evidence. Indexes required: `index=physical`; Sourcetypes: `sourcetype=pacs:access`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Lenel/CCure HEC, HR CSV.
• Ensure the following data sources are available: `pacs:access`; `hr_screening.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align badge_id keys; (2) emergency access workflow exception lookup; (3) annual ISO internal audit evidence.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=physical sourcetype=pacs:access action=grant earliest=-90d
| lookup hr_screening.csv badge_id OUTPUT screening_complete_date
| eval screen_epoch=strptime(screening_complete_date,"%Y-%m-%d")
| where isnull(screen_epoch) OR _time < screen_epoch
```
