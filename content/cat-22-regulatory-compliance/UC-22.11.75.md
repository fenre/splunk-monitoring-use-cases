# UC-22.11.75 · Unauthorized Wireless Access Point Detection by SSID and BSSID

## Description

Lists rogue or unknown BSSIDs observed on store wireless surveys so unauthorized wireless that could bypass segmentation is investigated under the wireless scanning requirement.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes wireless and sourcetypes wifi:rogue_ap. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Import approved corporate BSSIDs after each AP refresh. (2) Correlate with technician work orders. (3) Physical inspection workflow for persistent rogues. (4) Quarterly export for onsite assessor. Indexes required: `index=wireless`; Sourcetypes: `sourcetype=wifi:rogue_ap`; Lookups: `pci_approved_bssids`, `pci_approved_bssids.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Custom `sourcetype="ekahau:survey"` or `aruba:airwave:rogue"`.
• Ensure the following data sources are available: `index=wireless` `sourcetype="wifi:rogue_ap"` — `bssid`, `ssid`, `classification`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Import approved corporate BSSIDs after each AP refresh. (2) Correlate with technician work orders. (3) Physical inspection workflow for persistent rogues. (4) Quarterly export for onsite assessor.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wireless sourcetype="wifi:rogue_ap" earliest=-7d
| lookup pci_approved_bssids.csv bssid OUTPUT approved
| where isnull(approved) OR approved="false"
| stats earliest(_time) as first_seen, latest(_time) as last_seen, values(classification) as cls by bssid, ssid, site_id
| sort first_seen
```
