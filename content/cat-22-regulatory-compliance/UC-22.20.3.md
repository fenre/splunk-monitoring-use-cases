# 22.20.3 — CMMC AU.L2-3.3.1 — Audit record creation verification on CUI systems

**Criticality:** low | **Difficulty:** advanced | **Status:** verified

## Description

Verifies that all required audit record sources (Windows Security, EDR, network flow, application logs) are actively ingesting from CUI-classified systems. CMMC AU.L2-3.3.1 requires organisations to create and retain audit records to the extent needed to enable monitoring, analysis, investigation, and reporting of unlawful or unauthorized system activity.

## Value

Provides continuous verification that audit record creation is functioning on all CUI-scoped systems. Identifies gaps in log collection that would violate the CMMC requirement to create audit records for all CUI system activity.

## SPL

```spl
| tstats count WHERE index=* by index, sourcetype, host _time span=1h
| lookup cui_hosts.csv host OUTPUT cui_flag
| where cui_flag="true"
| stats dc(sourcetype) AS sourcetype_count dc(host) AS host_count latest(_time) AS last_event BY index
| lookup cui_required_sources.csv index OUTPUT required_sourcetypes
| eval missing=if(sourcetype_count < required_sourcetypes, "gap", "ok")
| where missing="gap"
| table index, sourcetype_count, required_sourcetypes, host_count, last_event
```

## CIM SPL

```spl
| tstats count from datamodel=Endpoint by Endpoint.dest, Endpoint.sourcetype | stats dc(Endpoint.sourcetype) AS src_count BY Endpoint.dest
```

## Implementation

(1) Verify data sources: All indexes feeding CUI-classified hosts, `cui_hosts.csv` lookup, `cui_required_sources.csv` lookup; (2) Maintain CUI-scoped lookups; (3) Schedule search at compliance-aligned interval; (4) Route findings to ticketing queue; (5) Retain exports per records management policy.

## Visualization

Table showing index, expected vs actual sourcetype count, host count, and last event time. Single value panel for total gaps detected.

## Compliance

- **CMMC** 2.0: AU.L2-3.3.1 (partial) — Directly verifies audit record creation on CUI systems by checking that all required event sources are actively ingesting — the core requirement of AU.L2-3.3.1.

## Known False Positives

Maintenance windows where hosts are rebooted may temporarily reduce sourcetype counts. Correlate with change management before raising severity.
