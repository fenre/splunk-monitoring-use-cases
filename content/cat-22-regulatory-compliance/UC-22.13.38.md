# UC-22.13.38 · Cyber Security Incident Identification from Correlated ESP and Endpoint Alerts

## Description

We group perimeter, authentication, and malware signals affecting BES assets to support timely identification of reportable cyber security incidents.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes notable. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Tag ES notables with BES context at correlation time. (2) Define which rules imply CIP-008 review. (3) Integrate with incident command workflow. (4) Preserve search artifacts per policy. Indexes required: `index=notable`; Lookups: `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.

## Detailed Implementation

• Ensure the following data sources are available: Splunk ES `` `notable` `` macro (or `index=notable`), BES asset tags.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag ES notables with BES context at correlation time. (2) Define which rules imply CIP-008 review. (3) Integrate with incident command workflow. (4) Preserve search artifacts per policy.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
`notable` earliest=-7d
| lookup bes_cyber_asset_inventory.csv dest as asset OUTPUT bes_system
| where isnotnull(bes_system)
| stats earliest(_time) as detected_time values(rule_name) as rules dc(asset) as assets by urgency
| sort - detected_time
```
