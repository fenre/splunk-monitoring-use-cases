# UC-22.15.46 · Zone Boundary Traffic Monitoring — Anomalous Inter-Zone Volume

## Description

Baselines permitted bytes and sessions between OT zones and flags spikes that may indicate policy drift, covert channels, or misrouted traffic at conduits.

## Value

Baselines permitted bytes and sessions between OT zones and flags spikes that may indicate policy drift, covert channels, or misrouted traffic at conduits.

## Implementation

(1) Build `ot_zone_pairs_expected.csv` for pairs that should exist; (2) exclude known maintenance windows; (3) alert on new `app` on sensitive pairs; (4) correlate with change records.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Palo Alto Networks (2757).
• Ensure the following data sources are available: `index=fw` `sourcetype="pan:traffic"` with `src_zone`, `dest_zone`, `bytes`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Build `ot_zone_pairs_expected.csv` for pairs that should exist; (2) exclude known maintenance windows; (3) alert on new `app` on sensitive pairs; (4) correlate with change records.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=fw sourcetype="pan:traffic" earliest=-14d
| eval zone_pair=concat(src_zone,"->",dest_zone)
| bin _time span=1h
| stats sum(bytes) as b, dc(dest_port) as ports by _time, zone_pair, app
| eventstats median(b) as med by zone_pair, app
| eval ratio=if(med>0, round(b/med,2), null())
| where ratio>8 AND b>10485760
| sort - b
```
