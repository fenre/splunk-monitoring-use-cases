# UC-22.11.95 · PCI-DSS 6.2 — Bespoke-software SDLC: code-review + SAST completion before CDE deploy

## Description

For every deploy to a CDE service, assert that SAST passed and a PR review was approved before deploy. PCI-DSS 6.2 mandates a documented SDLC.

## Value

Replaces the pre-release tick-box with continuous gate-enforcement evidence. Auditors see the commit→gate→deploy chain for every CDE deploy.

## Implementation

(1) Route CI/CD + SAST events; (2) maintain cde_services.csv; (3) schedule UC hourly; (4) gates_met=0 opens a security ticket; (5) pipeline policy revisions add gates.

## Detailed Implementation

Step 1 — Configure data collection
(1) Route CI/CD + SAST events; (2) maintain cde_services.csv; (3) schedule UC hourly; (4) gates_met=0 opens a security ticket; (5) pipeline policy revisions add gates.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cicd sourcetype IN (github:actions,gitlab:pipeline) event_type=deployment target_env IN ("prod-cde") earliest=-24h
| stats latest(_time) AS deployed_at latest(commit_sha) AS commit_sha BY artifact service
| join type=outer commit_sha
  [search index=security sourcetype=veracode:scan earliest=-30d
   | stats latest(overall_state) AS sast_state BY commit_sha]
| join type=outer commit_sha
  [search index=cicd event_type=pr_review state=approved earliest=-30d
   | stats latest(reviewer) AS reviewer BY commit_sha
   | eval review_state="approved"]
| eval gates_met=if(sast_state="pass" AND review_state="approved",1,0)
| where gates_met=0
| table artifact commit_sha service deployed_at sast_state review_state gates_met
```
