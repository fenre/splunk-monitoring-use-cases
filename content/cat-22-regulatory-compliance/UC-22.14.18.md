# UC-22.14.18 · Separation of Duties Violations in Change Tickets

## Description

Detects the same identity approving and implementing changes for AC-5 separation of duties.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:change_request. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Confirm field names match your SNOW transform; (2) exclude emergency changes via `chg_model` lookup; (3) alert weekly summary to ITGC; (4) document approved break-glass codes. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:change_request`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (Splunkbase 1928).
• Ensure the following data sources are available: `snow:change_request` with requested_by, assigned_to, closed_by.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Confirm field names match your SNOW transform; (2) exclude emergency changes via `chg_model` lookup; (3) alert weekly summary to ITGC; (4) document approved break-glass codes.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype=snow:change_request earliest=-30d
| eval same_actor=if(requested_by=assigned_to OR requested_by=closed_by,1,0)
| where same_actor=1 AND priority IN ("1","2")
| table number, short_description, requested_by, assigned_to, closed_by, priority, _time
```
