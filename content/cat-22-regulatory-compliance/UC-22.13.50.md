# 22.13.50 — Unauthorized Software Installation on BES Windows Servers

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We detect new software inventory entries not tied to an approved package list on in-scope Windows hosts.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes endpoint and sourcetypes inventory:software, inventory:software`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=endpoint sourcetype=inventory:software earliest=-7d
| lookup bes_cyber_asset_inventory.csv host OUTPUT bes_system
| where isnotnull(bes_system)
| lookup approved_software_catalog.csv product_name OUTPUT approved
| where isnull(approved)
| stats earliest(_time) as first_seen by host, product_name, version
| sort - first_seen
```

## Implementation

(1) Populate approved catalog from change management. (2) Deduplicate version strings. (3) Weekly review with server owners. (4) Auto-ticket unapproved installs. Indexes required: `index=endpoint`; Sourcetypes: `sourcetype=inventory:software`, `sourcetype=inventory:software``; Lookups: `approved_software_catalog`, `approved_software_catalog.csv`, `bes_cyber_asset_inventory`, `bes_cyber_asset_inventory.csv`.

## Visualization

Table (new software), Bar chart (by product), Single value (distinct hosts)

## Compliance

- **NERC CIP** current: CIP-010-4 R2 Part 2.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-010-4 R2 Part 2.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
