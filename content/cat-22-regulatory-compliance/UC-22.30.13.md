# 22.30.13 — Cyber security framework compliance monitoring

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Supports RBI cyber security framework supervisory examination readiness for cyber security framework compliance monitoring.

## Value

Gives the responsible team auditor-ready evidence for RBI Cyber compliance by automating regulatory compliance evidence using indexes rbi and sourcetypes bank:cyber:event. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=rbi sourcetype="bank:cyber:event" earliest=-30d
| eval reported=strptime(reported_cert_in_at,"%Y-%m-%d %H:%M:%S")
| eval detected=strptime(detected_at,"%Y-%m-%d %H:%M:%S")
| where isnull(reported) OR (reported-detected)>6*3600
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=rbi`; Sourcetypes: `sourcetype=bank:cyber:event`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones). Key fields: `detected`, `reported`.

## Compliance

- **RBI Cyber** 2016 (as amended): Annex-A (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for Annex-A.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
