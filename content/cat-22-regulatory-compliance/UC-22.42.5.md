# UC-22.42.5 · Infrastructure-as-code drift — applied state diverges from merged plan

## Description

Continuously reconciles the cloud resource inventory against the last-approved IaC plan checksum and alerts on drift. Drift is a canonical change-management deficiency and also a known attacker-persistence pattern.

## Value

Turns 'we use Terraform' into a continuously-verified statement.

## Implementation

(1) Export per-plan checksums from CI on merge; (2) enable cloud inventory export; (3) run every 30 min; (4) SOAR auto-remediates via `terraform plan && apply` if allowed.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: Terraform/OpenTofu state snapshots, cloud provider inventory (AWS Config, Azure Resource Graph, GCP Asset Inventory), CI/CD merge events..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Export per-plan checksums from CI on merge; (2) enable cloud inventory export; (3) run every 30 min; (4) SOAR auto-remediates via `terraform plan && apply` if allowed.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iac sourcetype=tf:state_snapshot earliest=-30m
| stats latest(plan_checksum) as expected_checksum by resource_id
| join resource_id [search index=cloud sourcetype=aws:config OR sourcetype=azure:rg OR sourcetype=gcp:asset earliest=-30m | stats latest(state_checksum) as actual_checksum by resource_id]
| where expected_checksum!=actual_checksum
| table resource_id expected_checksum actual_checksum
```

## Known False Positives

Cloud-provider control-plane metadata changes (timestamps, auto-generated IDs) can fire; exclude from checksum calculation at plan-emission time.
