# UC-22.14.40 · Information Input Validation — Web Parameter Anomalies

## Description

Surfaces WAF or app logs showing validation failures and injection patterns for SI-10 information input validation.

## Value

Surfaces WAF or app logs showing validation failures and injection patterns for SI-10 information input validation.

## Implementation

(1) Normalize field names per TA; (2) tune out scanner noise with bot lookup; (3) feed IR-4 for confirmed exploitation attempts; (4) map to RA-5 findings.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for F5 BIG-IP ASM (TA), ModSecurity, or `access_combined` with WAF headers.
• Ensure the following data sources are available: `sourcetype=f5:asm:*` or `sourcetype=apache:access` with `mod_security` message route.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Normalize field names per TA; (2) tune out scanner noise with bot lookup; (3) feed IR-4 for confirmed exploitation attempts; (4) map to RA-5 findings.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=web sourcetype=f5:asm:* OR sourcetype=modsec_json earliest=-24h
| search severity IN ("high","critical") OR response_code=406
| stats count by uri, src_ip, attack_type
| sort - count
| head 100
```
