# 22.21.2 — Model Input/Output Logging Integrity (Art. 12)

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Verifies that each inference event carries paired input feature hash and output label/probability for Art. 12 traceability and post-incident reconstruction.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes ai_serving and sourcetypes ai:inference:json. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ai_serving sourcetype="ai:inference:json" earliest=-4h
| eval has_in=if(isnotnull(input_feature_hash) AND len(input_feature_hash)>=16,1,0)
| eval has_out=if(isnotnull(output_label) OR isnotnull(output_prob),1,0)
| eval pair_ok=if(has_in=1 AND has_out=1,1,0)
| stats count as n, sum(pair_ok) as paired by model_version, deployment
| eval pair_rate=round(100*paired/n,2)
| where pair_rate < 100 OR n>0
| sort + pair_rate
```

## Implementation

(1) Require inference middleware to emit paired fields; (2) add props.conf FIELDALIAS if vendor names differ; (3) alert on pair_rate drops > 0.1% in 15m; (4) retain raw JSON in restricted index. Indexes required: `index=ai_serving`; Sourcetypes: `sourcetype=ai:inference:json`.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
