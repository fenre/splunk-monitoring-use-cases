# 22.13.42 — Incident Evidence Preservation — Splunk Search Artifact Export Audit

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

We audit exports of saved searches and large result downloads during active incidents to support evidence preservation and chain-of-custody discipline.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes _audit. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=_audit action=search info=granted earliest=-30d total_count>50000
| lookup cip008_active_incidents.csv search_name OUTPUT incident_id
| where isnotnull(incident_id)
| stats sum(total_count) as rows values(user) as users by incident_id, search_name
| sort - rows
```

## Implementation

(1) Maintain active incident lookup updated by IR. (2) Tune `total_count` threshold. (3) Pair with proxy DLP if used. (4) Retain audit logs per records program. Indexes required: `index=_audit`.

## Visualization

Table (large searches), Bar chart (by user), Single value (export events). Key fields: `incident_id`, `rows`, `search_name`, `users`.

## Compliance

- **NERC CIP** current: CIP-008-6 R5 Part 5.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for CIP-008-6 R5 Part 5.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
