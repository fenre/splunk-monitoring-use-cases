# UC-22.13.42 · Incident Evidence Preservation — Splunk Search Artifact Export Audit

## Description

We audit exports of saved searches and large result downloads during active incidents to support evidence preservation and chain-of-custody discipline.

## Value

We audit exports of saved searches and large result downloads during active incidents to support evidence preservation and chain-of-custody discipline.

## Implementation

(1) Maintain active incident lookup updated by IR. (2) Tune `total_count` threshold. (3) Pair with proxy DLP if used. (4) Retain audit logs per records program.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: `_audit` index `action=search` `info=granted`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain active incident lookup updated by IR. (2) Tune `total_count` threshold. (3) Pair with proxy DLP if used. (4) Retain audit logs per records program.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=_audit action=search info=granted earliest=-30d total_count>50000
| lookup cip008_active_incidents.csv search_name OUTPUT incident_id
| where isnotnull(incident_id)
| stats sum(total_count) as rows values(user) as users by incident_id, search_name
| sort - rows
```
