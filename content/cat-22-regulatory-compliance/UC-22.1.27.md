# UC-22.1.27 · GDPR Processor Compliance Attestation Tracking

## Description

Article 28 requires processors to demonstrate compliance and controllers to work only with processors offering sufficient guarantees. This use case tracks ISO 27001, SOC 2, and processor-signed security addenda — surfacing expired attestations and processors without current documentation before contract renewal or procurement renews access.

## Value

Article 28 requires processors to demonstrate compliance and controllers to work only with processors offering sufficient guarantees. This use case tracks ISO 27001, SOC 2, and processor-signed security addenda — surfacing expired attestations and processors without current documentation before contract renewal or procurement renews access.

## Implementation

(1) Extend processor inventory with attestation types and expiry dates; (2) integrate procurement / ServiceNow vendor records for automated updates; (3) alert procurement and DPO at 60 and 30 days before expiry; (4) block new data integrations in workflow when `EXPIRED`; (5) export quarterly attestation coverage report.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup gdpr_processor_register.csv
| eval expiry_epoch=strptime(attestation_expiry,"%Y-%m-%d")
| eval days_to_expiry=round((expiry_epoch-now())/86400,0)
| eval attestation_status=case(
    isnull(attestation_expiry), "MISSING",
    days_to_expiry < 0, "EXPIRED",
    days_to_expiry < 60, "EXPIRING",
    1=1, "CURRENT")
| where attestation_status!="CURRENT" OR dpa_signed!="true"
| table processor_name, attestation_type, attestation_expiry, days_to_expiry, attestation_status, dpa_signed
| sort attestation_status, days_to_expiry
```
