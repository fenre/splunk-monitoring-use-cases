# UC-22.4.1 · CCPA Consumer Data Access and Deletion Request Tracking

## Description

Tracks privacy request fulfillment work items end-to-end and flags requests at risk of missing the 45-day statutory response window.

## Value

Gives the responsible team auditor-ready evidence for CCPA/CPRA compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:incident, snow:sc_req_item. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Configure ServiceNow inputs for sc_req_item and/or incidents; (2) normalize catalog item names to match the filter (adjust `cat_item` strings); (3) if CCPA allows extensions, add fields for `extension_days` and update `sla_days` logic; (4) schedule daily with alert on `breach=1`. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:incident`, `sourcetype=snow:sc_req_item`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (Splunkbase 1928).
• Ensure the following data sources are available: `index=itsm` `sourcetype="snow:sc_req_item"` (number, opened_at, closed_at, state, cat_item, short_description) or `sourcetype="snow:incident"` (number, opened_at, closed_at, state, short_description).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Configure ServiceNow inputs for sc_req_item and/or incidents; (2) normalize catalog item names to match the filter (adjust `cat_item` strings); (3) if CCPA allows extensions, add fields for `extension_days` and update `sla_days` logic; (4) schedule daily with alert on `breach=1`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm (sourcetype="snow:sc_req_item" OR sourcetype="snow:incident")
    (cat_item="*CCPA*" OR cat_item="*Privacy*" OR short_description="*CCPA*" OR short_description="*Consumer Privacy*")
| eval opened_epoch=strptime(opened_at, "%Y-%m-%d %H:%M:%S")
| eval closed_epoch=if(isnotnull(closed_at), strptime(closed_at, "%Y-%m-%d %H:%M:%S"), null())
| eval age_days=round((now()-opened_epoch)/86400, 1)
| eval sla_days=45
| eval breach=if(isnull(closed_epoch) AND age_days>sla_days, 1, 0)
| eval days_remaining=if(isnull(closed_epoch), sla_days-age_days, null())
| table _time, number, state, age_days, days_remaining, breach, short_description
| sort - breach, days_remaining
```
