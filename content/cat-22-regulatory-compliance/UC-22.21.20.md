# UC-22.21.20 · Risk Management System — Hazard Log Update SLA (Art. 9/Annex IV)

## Description

Measures time from new_hazard to hazard_reviewed in risk register stream for RMS operational effectiveness.

## Value

Measures time from new_hazard to hazard_reviewed in risk register stream for RMS operational effectiveness.

## Implementation

(1) Ensure hazard_id stable; (2) escalate unreviewed >7d; (3) link to incident index; (4) quarterly RMS committee export.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263), Splunk ITSI (1841).
• Ensure the following data sources are available: `index=ai_risk` `sourcetype=rms:hazard` — hazard_id, status, transition.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ensure hazard_id stable; (2) escalate unreviewed >7d; (3) link to incident index; (4) quarterly RMS committee export.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ai_risk sourcetype="rms:hazard" earliest=-90d
| eval opened=if(status="new",_time,null())
| eval reviewed=if(status="reviewed",_time,null())
| stats min(opened) as t_open, max(reviewed) as t_rev by hazard_id
| eval review_hours=round((t_rev-t_open)/3600,2)
| where isnull(t_rev) OR review_hours>168
| sort - review_hours
```
