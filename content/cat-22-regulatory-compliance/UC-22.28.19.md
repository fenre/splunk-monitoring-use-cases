# 22.28.19 — User access management for banking systems

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Demonstrates alignment with BAIT/KAIT expectations for user access management for banking systems using auditable Splunk evidence.

## Value

Gives the responsible team auditor-ready evidence for BAIT/KAIT compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode IN (4624,4720,4738) earliest=-24h
| stats dc(ComputerName) as hosts by Account_Name, Privilege_List
| lookup bait_user_access_scope.csv Account_Name OUTPUT role
| where isnotnull(role) AND hosts>15
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Vulnerabilities.Vulnerabilities by Vulnerabilities.severity, Vulnerabilities.signature, Vulnerabilities.dest | sort - count
```

## Implementation

(1) Maintain `de_kritis_assets.csv` with sector classification and KRITIS identifiers. (2) Link ITSI services to KRITIS objects for business impact overlays. (3) Export monthly control posture snapshots for BSI engagement. (4) Tune thresholds with enterprise risk management and OT security owners. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`.

## Visualization
