# UC-22.16.27 · OT configuration changes without ITSM change record

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "OT configuration changes without ITSM change record".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes itsm, ot and sourcetypes nozomi:alert, snow:change_request. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=itsm`, `index=ot`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype=nozomi:alert earliest=-7d name="*configuration*change*"
| eval uc_tag="22.16.27"
| join type=left src_ip [
    search index=itsm sourcetype=snow:change_request earliest=-7d cmdb_ci="*OT*" | eval dev=src_ip | stats values(number) as crs by dev
  ]
| where isnull(crs)
```
