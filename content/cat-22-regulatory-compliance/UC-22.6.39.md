# UC-22.6.39 · Monitoring Activities — SOC Queue Depth vs On-Shift Analyst Headcount

## Description

Compares open notable or case backlog to scheduled SOC roster to evidence monitoring resourcing adequacy.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001, ISO/IEC 27001 compliance by automating regulatory compliance evidence. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Calibrate ratio to your SOC model; (2) management review; (3) hiring plan input. Lookups: `soc_shift_roster`, `soc_shift_roster.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security, workforce CSV.
• Ensure the following data sources are available: `notable` macro; `soc_shift_roster.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Calibrate ratio to your SOC model; (2) management review; (3) hiring plan input.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
search `notable` status!=closed earliest=-1d
| stats dc(rule_id) as open_notables
| append [| inputlookup soc_shift_roster.csv | where date=strftime(now(),"%Y-%m-%d") | stats sum(analysts_on_shift) as headcount]
| stats max(open_notables) as on max(headcount) as hc
| eval overload=if(on>hc*25,1,0)
```
