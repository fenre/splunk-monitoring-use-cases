# UC-22.1.48 · GDPR Purpose Limitation Enforcement Across Systems

## Description

Personal data must be collected for specified, explicit, and legitimate purposes and not further processed incompatibly. This use case compares declared processing purposes in ROPA to observed data flows — e.g., HR exports to marketing tools — using purpose tags on integrations where available.

## Value

Personal data must be collected for specified, explicit, and legitimate purposes and not further processed incompatibly. This use case compares declared processing purposes in ROPA to observed data flows — e.g., HR exports to marketing tools — using purpose tags on integrations where available.

## Implementation

(1) Document each integration's allowed downstream categories in `gdpr_integration_purpose_map.csv`; (2) populate `dest_category` from proxy and CASB; (3) alert when HR or health-tagged sources send data to marketing-class destinations; (4) require DPO sign-off for exceptions; (5) integrate with access reviews.

## Detailed Implementation

Step 1 — Configure data collection
(1) Document each integration's allowed downstream categories in `gdpr_integration_purpose_map.csv`; (2) populate `dest_category` from proxy and CASB; (3) alert when HR or health-tagged sources send data to marketing-class destinations; (4) require DPO sign-off for exceptions; (5) integrate with access reviews.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| tstats `summariesonly` sum(All_Traffic.bytes_out) as bytes_out
  from datamodel=Network_Traffic.All_Traffic
  where earliest=-7d
  by All_Traffic.src All_Traffic.dest All_Traffic.app
| rename All_Traffic.* as *
| lookup gdpr_integration_purpose_map.csv app OUTPUT ropa_purpose dest_category_allowed
| eval incompatible=if(NOT match(dest_category_allowed, dest_category),1,0)
| where incompatible=1
| table src, dest, app, ropa_purpose, dest_category, dest_category_allowed, bytes_out
| sort - bytes_out
```
