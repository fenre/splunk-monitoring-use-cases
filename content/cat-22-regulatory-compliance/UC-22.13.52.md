# UC-22.13.52 · Vulnerability Assessment After Material Configuration Changes

## Description

We find material firewall or OS changes without a subsequent vulnerability scan within the required follow-up window.

## Value

We find material firewall or OS changes without a subsequent vulnerability scan within the required follow-up window.

## Implementation

(1) Flag material changes in ITSM. (2) Require scanner to tag `change_ticket`. (3) Alert at fourteen days without scan. (4) Document risk acceptance if delayed.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=itsm` material changes, `index=vuln` Tenable or Qualys.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Flag material changes in ITSM. (2) Require scanner to tag `change_ticket`. (3) Alert at fourteen days without scan. (4) Document risk acceptance if delayed.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype=chg:network material=1 earliest=-35d
| eval chg_time=strptime(closed_at,"%Y-%m-%d %H:%M:%S")
| rename number as change_id
| join type=left change_id [
    search index=vuln sourcetype=tenable:scans earliest=-35d
    | rename change_ticket as change_id
    | stats latest(finished_at) as scan_done by change_id
  ]
| eval scan_epoch=strptime(scan_done,"%Y-%m-%d %H:%M:%S")
| where isnull(scan_epoch) OR (scan_epoch-chg_time)>1209600
| table change_id, closed_at, scan_done
```
