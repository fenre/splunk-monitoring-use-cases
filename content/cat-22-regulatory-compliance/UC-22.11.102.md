# UC-22.11.102 · PCI-DSS 10.7 — Log retention: CDE data-source retention + immutability attestation

## Description

Attests the retention and WORM posture required by PCI-DSS 10.7 (one-year retention, three months immediately available).

## Value

Prevents the audit-killing discovery that retention was silently reduced during an index rebuild. Daily attestation beats the quarterly spot-check.

## Implementation

(1) Name CDE indexes with cde_/pci_ prefix; (2) WORM signer writes per-index heartbeat; (3) schedule UC daily; (4) gap opens a Platform ticket; (5) monthly CISO review. Indexes required: `index=compliance`; Sourcetypes: `sourcetype=worm:signer`, `sourcetype=worm:signer.`.

Step 1 — Configure data collection

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| rest /services/data/indexes
| where match(title,"^(cde|pci)_")
| eval configured_days=round(frozenTimePeriodInSecs/86400,0)
| table title configured_days minTime maxTime
| rename title AS index
| eval oldest_event_age_d=round((now()-strptime(minTime,"%Y-%m-%dT%H:%M:%S%z"))/86400,0)
| join type=outer index
  [search index=compliance sourcetype=worm:signer earliest=-7d
   | stats latest(_time) AS worm_last_write BY index]
| where configured_days<365 OR oldest_event_age_d<365 OR worm_last_write<now()-86400
| table index configured_days oldest_event_age_d worm_last_write
```
