# 22.34.2 — Operating system privileged account control within SWIFT zone

**Criticality:** critical | **Difficulty:** beginner | **Status:** N/A

## Description

Produces SWIFT Customer Security Programme evidence for operating system privileged account control within swift zone ahead of independent assessment.

## Value

Gives the responsible team auditor-ready evidence for SWIFT CSP compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" host="SWIFT-*" EventCode IN (4720,4722,4738) earliest=-24h
| stats values(TargetUserName) as tgt by MemberName, EventCode
| lookup swift_os_account_allowlist.csv tgt OUTPUT allowed
| where isnull(allowed)
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Segment SWIFT CSP evidence into dedicated indexes with strict role-based access. (2) Join operators to HR screening and vendor risk lookups refreshed weekly. (3) Validate MFA and PAM sourcetypes are CIM-mapped for Authentication models. (4) Retain evidence for at least 13 months aligned to attestation cycles. Indexes required: `index=wineventlog`.

## Visualization

Table (operators), Single value (sessions), Bar chart (CSCF controls). Key fields: `EventCode`, `MemberName`, `tgt`.
