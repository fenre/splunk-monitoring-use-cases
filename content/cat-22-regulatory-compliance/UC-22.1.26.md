# UC-22.1.26 · GDPR Resilience and Availability Monitoring for Personal-Data Services

## Description

Integrity, confidentiality, and availability of processing systems are core Art. 32 objectives. This use case monitors uptime, error rates, and recovery objectives for applications and databases documented in the ROPA as processing personal data — evidencing resilience measures such as clustering, failover drills, and synthetic checks.

## Value

## Implementation

(1) Map ROPA `system_name` to ITSI services in `itsi_service_map.csv`; (2) define KPI thresholds aligned to internal RPO/RTO for personal-data services; (3) alert when health scores breach SLO; (4) attach failover drill references from `gdpr_security_test_register.csv`; (5) report monthly resilience posture to the DPO.

Step 1 — Configure data collection
(1) Map ROPA `system_name` to ITSI services in `itsi_service_map.csv`; (2) define KPI thresholds aligned to internal RPO/RTO for personal-data services; (3) alert when health scores breach SLO; (4) attach failover drill references from `gdpr_security_test_register.csv`; (5) report monthly resilience posture to the DPO.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup gdpr_ropa_register.csv WHERE data_category!="N/A"
| fields system_name
| lookup itsi_service_map.csv system_name OUTPUT itsi_service
| join itsi_service [
    search index=itsi_summary earliest=-24h
    | stats latest(health_score) as health_score latest(severity) as severity by itsi_service
]
| eval breach_slo=if(health_score < 70 OR match(lower(severity),"(?i)critical|high"), 1, 0)
| where breach_slo=1
| table system_name, itsi_service, health_score, severity
```
