# 22.13.37 — Password or Passphrase Policy Violations from Domain Controller Auditing

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

We surface directory events indicating weak password changes blocked by policy or Kerberos pre-authentication failures tied to account hygiene.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode IN (4723,4724,4771) earliest=-7d
| lookup bes_ad_ou_map.csv TargetUserName OUTPUT in_bes_ou
| where in_bes_ou="true"
| stats count by EventCode, TargetUserName, ComputerName
| sort - count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Enable appropriate DC auditing. (2) Map accounts to BES OUs. (3) Tune noise from service accounts. (4) Pair with IAM for password reset workflow. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`; Lookups: `bes_ad_ou_map`, `bes_ad_ou_map.csv`.
