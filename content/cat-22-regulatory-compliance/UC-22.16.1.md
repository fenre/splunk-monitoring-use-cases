# 22.16.1 — IT/OT boundary deny vs allow ratio by zone pair

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "IT/OT boundary deny vs allow ratio by zone pair".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes ot and sourcetypes pan:traffic. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot sourcetype=pan:traffic earliest=-24h zone_dest="OT-L2*"
| eval uc_tag="22.16.1"
| stats sum(eval(action=="allow")) as allows, sum(eval(action=="deny")) as denies by zone_src, app
| eval deny_pct=round(100*denies/(allows+denies+0.001),2)
| sort - allows
```

## CIM SPL

```spl
| tstats summariesonly=t sum(All_Traffic.bytes_in) as agg_value from datamodel=Network_Traffic.All_Traffic by All_Traffic.app | sort - agg_value
```
