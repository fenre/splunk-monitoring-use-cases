# UC-22.10.25 · Endpoint Controls — ePHI Clipboard/Print from VDI

## Description

Tracks clipboard redirection and client drive mapping events on clinical VDI where policies should block exfiltration paths for ePHI.

## Value

Tracks clipboard redirection and client drive mapping events on clinical VDI where policies should block exfiltration paths for ePHI.

## Implementation

(1) Align Citrix policy names with log fields; (2) validate expected exceptions (transcription vendors); (3) remediate via Citrix policy sets.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Citrix (2757), Splunk Add-on for CrowdStrike FDR (5082).
• Ensure the following data sources are available: `index=citrix` `sourcetype=citrix:session` ICA channel flags, `index=endpoint` `sourcetype=crowdstrike:hosts` or `ProcessRollup2` with clipboard tools.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Align Citrix policy names with log fields; (2) validate expected exceptions (transcription vendors); (3) remediate via Citrix policy sets.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=citrix sourcetype="citrix:session" earliest=-7d ClientDrive="On" OR Clipboard="ClientToServer"
| lookup hipaa_epic_users.csv UserName OUTPUT is_clinical_user
| where is_clinical_user="true"
| stats count by UserName, client_ip, PublishedApplication, ClientDrive, Clipboard
| sort - count
```
