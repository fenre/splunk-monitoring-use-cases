# 22.15.33 — East-West OT Traffic Crossing Intended Purdue Levels

**Criticality:** critical | **Difficulty:** advanced | **Status:** N/A

## Description

Detects flows between OT zones that should communicate only through defined conduits, evidencing segmentation policy.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes fw and sourcetypes pan:traffic. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=fw sourcetype="pan:traffic" earliest=-24h
| lookup ot_zone_pairs.csv src_zone dest_zone OUTPUT allowed_conduit
| where isnull(allowed_conduit) OR allowed_conduit="false"
| stats sum(bytes) as bytes by src_zone, dest_zone, app
| sort - bytes
```

## CIM SPL

```spl
| tstats summariesonly=t sum(All_Traffic.bytes_in) as agg_value from datamodel=Network_Traffic.All_Traffic by All_Traffic.app | sort - agg_value
```

## Implementation

(1) Maintain zone pair matrix from ISA-62443 drawings; (2) log intra-zone noise baseline; (3) alert new apps; (4) feed exceptions with expiry. Indexes required: `index=fw`; Sourcetypes: `sourcetype=pan:traffic`; Lookups: `ot_zone_pairs`, `ot_zone_pairs.csv`.

## Visualization

Sankey (zone flows), Table, Single value (new pairs/week). Key fields: `app`, `bytes`, `dest_zone`, `src_zone`.

## Compliance

- **IEC 62443** 2013-ongoing: SR 5.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 5.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
