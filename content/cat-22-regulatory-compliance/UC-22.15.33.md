# UC-22.15.33 · East-West OT Traffic Crossing Intended Purdue Levels

## Description

Detects flows between OT zones that should communicate only through defined conduits, evidencing segmentation policy.

## Value

Detects flows between OT zones that should communicate only through defined conduits, evidencing segmentation policy.

## Implementation

(1) Maintain zone pair matrix from ISA-62443 drawings; (2) log intra-zone noise baseline; (3) alert new apps; (4) feed exceptions with expiry.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Palo Alto (2757), NetFlow.
• Ensure the following data sources are available: `pan:traffic` with `src_zone`, `dest_zone`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain zone pair matrix from ISA-62443 drawings; (2) log intra-zone noise baseline; (3) alert new apps; (4) feed exceptions with expiry.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=fw sourcetype="pan:traffic" earliest=-24h
| lookup ot_zone_pairs.csv src_zone dest_zone OUTPUT allowed_conduit
| where isnull(allowed_conduit) OR allowed_conduit="false"
| stats sum(bytes) as bytes by src_zone, dest_zone, app
| sort - bytes
```
