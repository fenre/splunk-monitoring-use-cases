# 22.13.62 — Inter-Control-Center Link Availability from Synthetic Tests

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

We measure synthetic ping or application handshake success rates between control centers to evidence communications availability.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes synthetic and sourcetypes http:ping, http:ping`, tcp:check. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=synthetic (sourcetype=http:ping OR sourcetype=tcp:check) cip012_link=1 earliest=-24h
| stats count(eval(match(status,"(?i)success"))) as ok count as total by link_id
| eval pct=round(100*ok/total,2)
| where pct < 99.5
| lookup control_center_links.csv link_id OUTPUT src_cc dest_cc
| table link_id, src_cc, dest_cc, pct, ok, total
```

## Implementation

(1) Deploy lightweight synthetic from each center. (2) Tune SLA percentage. (3) Page network operations on breach. (4) Retain raw samples for monthly report. Indexes required: `index=synthetic`; Sourcetypes: `sourcetype=http:ping`, `sourcetype=http:ping``, `sourcetype=tcp:check`, `sourcetype=tcp:check``; Lookups: `control_center_links`, `control_center_links.csv`.
