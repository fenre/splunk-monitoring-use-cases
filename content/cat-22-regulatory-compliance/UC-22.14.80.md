# UC-22.14.80 · System Recovery Time Objective Tracking from DR Drills

## Description

Captures measured recovery time metrics from DR drills for CP-10 system recovery and reconstitution.

## Value

Gives the responsible team auditor-ready evidence for NIST 800-53 compliance by automating regulatory compliance evidence using indexes grc and sourcetypes dr:drill, dr:drill`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Ensure epoch fields populated at drill completion; (2) store raw drill packets in secure share; (3) feed POA&M if missed; (4) integrate with IR-8 updates. Indexes required: `index=grc`; Sourcetypes: `sourcetype=dr:drill`, `sourcetype=dr:drill``; Lookups: `app_tiers`, `app_tiers.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: HTTP Event Collector (HEC), Splunk Add-on for ServiceNow (Splunkbase 1928).
• Ensure the following data sources are available: `index=grc` `sourcetype=dr:drill` or `snow:change_request` DR template.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Ensure epoch fields populated at drill completion; (2) store raw drill packets in secure share; (3) feed POA&M if missed; (4) integrate with IR-8 updates.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype=dr:drill earliest=-365d
| eval rto_min=recovery_end_epoch - failover_start_epoch
| lookup app_tiers.csv application OUTPUT rto_target_min
| where rto_min > rto_target_min
| table drill_id, application, rto_min, rto_target_min, notes
```
