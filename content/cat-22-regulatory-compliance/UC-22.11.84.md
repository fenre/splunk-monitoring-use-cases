# UC-22.11.84 · Incident Response Plan Tabletop and Live Test Execution Logging

## Description

Indexes IR test outcomes and scenario coverage so annual (or more frequent) response exercises for payment breaches are tracked with timestamps, participants, and lessons learned for PCI readiness reviews.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes grc and sourcetypes grc:ir_test. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Push test completion from GRC after each tabletop. (2) Adjust `days_since` threshold if policy requires semi-annual tests. (3) Link `test_id` to remediation tickets. (4) Board briefing attachment optional. Indexes required: `index=grc`; Sourcetypes: `sourcetype=grc:ir_test`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (1928), `sourcetype="grc:ir_test"`.
• Ensure the following data sources are available: `index=grc` `sourcetype="grc:ir_test"` — `test_id`, `scenario`, `executed_date`, `result`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Push test completion from GRC after each tabletop. (2) Adjust `days_since` threshold if policy requires semi-annual tests. (3) Link `test_id` to remediation tickets. (4) Board briefing attachment optional.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=grc sourcetype="grc:ir_test" earliest=-400d
| where match(scenario,"(?i)payment|breach|PAN|CDE|PCI)")
| stats latest(executed_date) as last_run, latest(result) as outcome by test_id
| eval days_since=round((now()-strptime(last_run,"%Y-%m-%d"))/86400,1)
| where days_since>365 OR isnull(last_run)
| table test_id, last_run, days_since, outcome
| sort days_since
```
