# 22.34.10 — Annual KYC-SA attestation evidence pack

**Criticality:** low | **Difficulty:** beginner | **Status:** N/A

## Description

Produces SWIFT Customer Security Programme evidence for annual kyc-sa attestation evidence pack ahead of independent assessment.

## Value

Gives the responsible team auditor-ready evidence for SWIFT CSP compliance by automating regulatory compliance evidence using indexes grc and sourcetypes swift:kycsa. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=grc sourcetype="swift:kycsa" earliest=-400d
| stats latest(attestation_status) as st, latest(attestation_year) as yr by bic
| where st!="Submitted" AND yr=strftime(now(),"%Y")
| table bic, st, yr
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.action, Authentication.user, Authentication.src | sort - count
```

## Implementation

(1) Segment SWIFT CSP evidence into dedicated indexes with strict role-based access. (2) Join operators to HR screening and vendor risk lookups refreshed weekly. (3) Validate MFA and PAM sourcetypes are CIM-mapped for Authentication models. (4) Retain evidence for at least 13 months aligned to attestation cycles. Indexes required: `index=grc`.

## Visualization

Table (operators), Single value (sessions), Bar chart (CSCF controls). Key fields: `bic`, `st`, `yr`.

## Compliance

- **SWIFT CSP** CSCF v2025: 1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 1.1.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
