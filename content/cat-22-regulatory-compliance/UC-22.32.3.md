# 22.32.3 — ANPD personal data incident notification evidence

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Supports Lei Geral de Proteção de Dados (LGPD) compliance evidence for anpd personal data incident notification evidence in Splunk dashboards and scheduled reports.

## Value

Gives the responsible team auditor-ready evidence for LGPD compliance by automating regulatory compliance evidence using indexes privacy and sourcetypes anpd:notification. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=privacy sourcetype="anpd:notification" earliest=-365d
| where isnull(anpd_filed_at) AND confirmed="true"
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - count
```

## Implementation

(1) Maintain `amer_authorization_boundary.csv` with ATO, CUI, and CJIS scope flags. (2) Integrate POA&M exports via scheduled output and lookup refresh for ISSM workflows. (3) Separate LGPD and federal datasets with index- and role-level controls. (4) Quarterly control mapping workshop with ISSM, ISO, and privacy counsel. Indexes required: `index=privacy`; Sourcetypes: `sourcetype=anpd:notification`.
