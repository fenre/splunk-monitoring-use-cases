# UC-22.15.32 · Deprecated TLS or SSH Algorithms on OT Jump Hosts

## Description

Lists sessions negotiating weak ciphers for OT administrative channels where cryptography policy must be enforced.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes fw and sourcetypes pan:ssl. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Disable legacy TLS on servers; (2) upgrade PanOS profiles; (3) exception process for legacy PLC web UIs; (4) monthly compliance scan. Indexes required: `index=fw`; Sourcetypes: `sourcetype=pan:ssl`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Palo Alto (2757) decrypted metadata or Stream.
• Ensure the following data sources are available: `pan:ssl` or Splunk Stream TLS fields.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Disable legacy TLS on servers; (2) upgrade PanOS profiles; (3) exception process for legacy PLC web UIs; (4) monthly compliance scan.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=fw sourcetype="pan:ssl" dest_zone="OT-Jump" earliest=-7d
| where cipher IN ("RC4","3DES","DES") OR tls_version="TLSv1.0" OR tls_version="TLSv1.1"
| stats count by dest_ip, cipher, tls_version, app
| sort - count
```
