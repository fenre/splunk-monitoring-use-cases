# UC-22.6.12 · Project Security Gate — Production Deploys Without Security CAB Tag

## Description

Flags production deployments missing required security project gate metadata for new or changed systems.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes cicd and sourcetypes github:workflow. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Require change correlation ID in pipeline; (2) block deploy on missing tag optional; (3) map to secure SDLC KPIs. Indexes required: `index=cicd`; Sourcetypes: `sourcetype=github:workflow`; Lookups: `snow_change_tags`, `snow_change_tags.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for GitHub or Jenkins, ServiceNow change.
• Ensure the following data sources are available: `github:workflow` or `jenkins:build`; `snow:change_request`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require change correlation ID in pipeline; (2) block deploy on missing tag optional; (3) map to secure SDLC KPIs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cicd sourcetype=github:workflow earliest=-30d conclusion=success branch=main
| lookup snow_change_tags.csv deploy_sha OUTPUT security_gate
| where isnull(security_gate) OR security_gate!='approved'
```
