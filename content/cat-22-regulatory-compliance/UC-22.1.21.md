# UC-22.1.21 · GDPR Encryption at Rest Audit Evidence

## Description

Article 32(1)(a) requires pseudonymisation and encryption of personal data where appropriate. This use case aggregates technical evidence that storage volumes, databases, object stores, and backup targets that process personal data are encrypted at rest — surfacing misconfigured buckets, unencrypted snapshots, and hosts lacking disk encryption so the DPO can prove the measure is operational, not aspirational.

## Value

Article 32(1)(a) requires pseudonymisation and encryption of personal data where appropriate. This use case aggregates technical evidence that storage volumes, databases, object stores, and backup targets that process personal data are encrypted at rest — surfacing misconfigured buckets, unencrypted snapshots, and hosts lacking disk encryption so the DPO can prove the measure is operational, not aspirational.

## Implementation

(1) Maintain `gdpr_data_store_register.csv` linking cloud resource IDs and on-prem volumes to systems of record for personal data; (2) ingest CloudTrail / Azure audit events that reflect encryption configuration changes; (3) join nightly Tenable or CSPM exports for at-rest encryption posture; (4) alert on `NON_COMPLIANT` or stale `UNKNOWN` assets; (5) retain dashboard exports for Art. 32 accountability evidence packs.

Step 1 — Configure data collection
(1) Maintain `gdpr_data_store_register.csv` linking cloud resource IDs and on-prem volumes to systems of record for personal data; (2) ingest CloudTrail / Azure audit events that reflect encryption configuration changes; (3) join nightly Tenable or CSPM exports for at-rest encryption posture; (4) alert on `NON_COMPLIANT` or stale `UNKNOWN` assets; (5) retain dashboard exports for Art. 32 accountability evidence packs.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup gdpr_data_store_register.csv WHERE processes_personal_data="true"
| lookup cloud_asset_encryption_lookup resource_id OUTPUT encryption_at_rest, last_scan_date, kms_key_arn
| eval scan_age_days=if(isnotnull(last_scan_date), round((now()-strptime(last_scan_date,"%Y-%m-%d"))/86400,0), 9999)
| eval compliance_status=case(
    encryption_at_rest="true", "COMPLIANT",
    encryption_at_rest="false", "NON_COMPLIANT",
    isnull(encryption_at_rest), "UNKNOWN")
| where compliance_status!="COMPLIANT" OR scan_age_days > 90
| table resource_id, system_name, data_category, encryption_at_rest, compliance_status, scan_age_days, kms_key_arn
| sort compliance_status
```
