# 22.30.5 — Security testing evidence aggregation

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Supports MAS TRM supervisory examination readiness for security testing evidence aggregation.

## Value

Gives the responsible team auditor-ready evidence for MAS TRM compliance by automating regulatory compliance evidence using indexes grc and sourcetypes mas:incident. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=grc sourcetype="mas:incident" earliest=-365d
| eval delta_hours=(now()-strptime(detected_at,"%Y-%m-%d %H:%M:%S"))/3600
| where isnull(notified_mas_at) AND delta_hours>4
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - count
```

## Implementation

(1) Maintain `apac_material_services.csv` for important business services and regulator tags. (2) Send weekly digests to CRO and technology risk committees. (3) Validate DB Connect concurrency for audit peak windows. (4) Archive results to a restricted summary index for retention governance. Indexes required: `index=grc`; Sourcetypes: `sourcetype=mas:incident`.

## Visualization

Column chart (events by regulator), Table (privileged actions), Timeline (notification milestones). Key fields: `delta_hours`.

## Compliance

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
