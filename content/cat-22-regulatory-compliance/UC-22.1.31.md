# UC-22.1.31 · GDPR Processor Audit Evidence — Right to Audit and Inspection Logs

## Description

Controllers must be able to audit processors where contractually agreed. This use case logs scheduled and ad hoc audit events, findings severity, remediation due dates, and closure — creating a defensible audit trail that complements static PDF reports stored elsewhere.

## Value

Gives the responsible team auditor-ready evidence for GDPR compliance by automating regulatory compliance evidence using indexes itsm. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Record each on-site or remote audit in `gdpr_processor_audit_log.csv`; (2) create ServiceNow tasks per finding with linkage to processor; (3) alert when biennial audit cycle is missed; (4) dashboard open findings for steering committee; (5) export evidence package before supervisory inspections. Indexes required: `index=itsm`.

Step 1 — Configure data collection
(1) Record each on-site or remote audit in `gdpr_processor_audit_log.csv`; (2) create ServiceNow tasks per finding with linkage to processor; (3) alert when biennial audit cycle is missed; (4) dashboard open findings for steering committee; (5) export evidence package before supervisory inspections.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup gdpr_processor_audit_log.csv
| eval audit_date_epoch=strptime(audit_date,"%Y-%m-%d")
| eval days_since_audit=round((now()-audit_date_epoch)/86400,0)
| join processor_name type=left [
    search index=itsm short_description="*processor audit*" earliest=-365d
    | stats count as open_findings by vendor_name
    | rename vendor_name as processor_name
]
| eval next_audit_due=if(days_since_audit > 730, "OVERDUE", if(days_since_audit > 600, "DUE_SOON", "OK"))
| where next_audit_due!="OK" OR open_findings > 0
| table processor_name, audit_date, days_since_audit, next_audit_due, open_findings
```
