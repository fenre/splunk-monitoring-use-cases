# UC-22.15.36 · PLC Local Console Login Without Corporate Identity

## Description

Finds controller-local or maintenance port logins that cannot be mapped to a named human, weakening component-level identification.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_eng and sourcetypes plc:access. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Enable user management on intelligent devices; (2) disable default maintenance accounts; (3) physical locks on USB; (4) quarterly audit export. Indexes required: `index=ot_eng`; Sourcetypes: `sourcetype=plc:access`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Rockwell/Siemens vendor forwarders → `index=ot_eng`.
• Ensure the following data sources are available: `sourcetype="plc:access"` or vendor-specific.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Enable user management on intelligent devices; (2) disable default maintenance accounts; (3) physical locks on USB; (4) quarterly audit export.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot_eng sourcetype="plc:access" earliest=-30d
| eval user_id=coalesce(user, operator_slot)
| where user_id IN ("0","anonymous","MAINT","") OR match(user_id, "^\*+$")
| stats count by controller, user_id, src_interface
| sort - count
```
