# UC-22.45.3 · Backup completeness — unprotected workloads with regulated data

## Description

Cross-checks the backup inventory against the regulated-workload asset list. Any covered workload without a successful backup in its window becomes a finding.

## Value

Coverage gaps are the single most common backup finding at audit; continuous evidence eliminates surprise findings.

## Implementation

(1) Tag regulated workloads; (2) publish dr_tiers.csv; (3) run daily; (4) escalate via SOAR.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: See subcategory preamble..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag regulated workloads; (2) publish dr_tiers.csv; (3) run daily; (4) escalate via SOAR.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| rest /services/backup/inventory splunk_server=* | where regulated="true"
| stats max(last_success) as last_ok by workload tier
| lookup dr_tiers.csv tier OUTPUT expected_interval_hours
| eval hours_since=round((now()-last_ok)/3600,0)
| where hours_since>expected_interval_hours
| table tier workload last_ok hours_since expected_interval_hours
```
