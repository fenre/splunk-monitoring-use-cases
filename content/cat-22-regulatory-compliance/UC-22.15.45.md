# UC-22.15.45 · Historian Query Saturation During Incident

## Description

Tracks historian and OPC query rates to ensure analytical consumers do not starve operators during sustained attacks or mis-tuned reports.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes hist_queries and sourcetypes hist:sql. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Separate reporting VLAN; (2) cap concurrent ODBC; (3) throttle BI tools; (4) burst alerts to ops. Indexes required: `index=hist_queries`; Sourcetypes: `sourcetype=hist:sql`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Historian ODBC/API logs.
• Ensure the following data sources are available: `index=hist_queries`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Separate reporting VLAN; (2) cap concurrent ODBC; (3) throttle BI tools; (4) burst alerts to ops.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=hist_queries sourcetype="hist:sql" earliest=-24h
| bin _time span=5m
| stats count as qps by _time, client_app, historian_cluster
| eventstats perc95(qps) as p95_global
| where qps > p95_global*3
| sort - qps
```

**Pipeline walkthrough**

• Discretizes time or numeric ranges with `bin`/`bucket`.
