# UC-22.6.31 · Technical Vulnerability Management — Exploitable CVEs with Public PoC on In-Scope Hosts

## Description

Joins vulnerability scanner output to exploit availability tags to prioritize technical vulnerability response.

## Value

Gives the responsible team auditor-ready evidence for ISO 27001 compliance by automating regulatory compliance evidence using indexes vuln and sourcetypes tenable:vuln. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain exploit feed; (2) map to patch CAB; (3) document risk acceptance in SoA. Indexes required: `index=vuln`; Sourcetypes: `sourcetype=tenable:vuln`; Lookups: `exploit_available`, `exploit_available.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Tenable Add-On for Splunk (Splunkbase 4060).
• Ensure the following data sources are available: `tenable:vuln`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain exploit feed; (2) map to patch CAB; (3) document risk acceptance in SoA.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vuln sourcetype=tenable:vuln earliest=-7d severity IN ("Critical","High")
| lookup exploit_available.csv cve OUTPUT has_public_poc
| where has_public_poc="true"
| stats dc(host) as hosts, values(cve) as cves by plugin_name
| sort - hosts
```
