# UC-22.13.60 · BCSI Destruction and Sanitization Evidence from ITAD Tickets

## Description

We correlate IT asset disposal tickets with wipe certificates indexed as structured events for sanitization evidence.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes itad:disposal, itad:disposal`, itad:wipe_cert. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Require vendors to upload certificate metadata to ITSM. (2) Alert when disposal closes without cert. (3) Store PDF hashes out-of-band if needed. (4) Annual sampling by internal audit. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=itad:disposal`, `sourcetype=itad:disposal``, `sourcetype=itad:wipe_cert`, `sourcetype=itad:wipe_cert``.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (263).
• Ensure the following data sources are available: `index=itsm` `sourcetype=itad:disposal`, `sourcetype=itad:wipe_cert`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require vendors to upload certificate metadata to ITSM. (2) Alert when disposal closes without cert. (3) Store PDF hashes out-of-band if needed. (4) Annual sampling by internal audit.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype=itad:disposal status="completed" earliest=-365d
| rename asset_serial as serial
| join type=left serial [
    search index=itsm sourcetype=itad:wipe_cert earliest=-365d
    | stats latest(wipe_method) as method latest(cert_id) as cert by serial
  ]
| where bcsi_stored="true" AND isnull(cert)
| table disposal_id, serial, asset_tag, completed_at, cert
```
