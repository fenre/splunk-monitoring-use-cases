# UC-22.5.11 · MiFID II Algo Trading — Message Rate Throttle Breaches vs Exchange Limits

## Description

Throttle breaches can indicate malfunctioning algos or reckless co-location usage. Continuous comparison of outbound order rates to venue-specific caps evidences real-time surveillance aligned with Art. 17 controls.

## Value

Gives the responsible team auditor-ready evidence for MiFID II compliance by automating regulatory compliance evidence using indexes trading and sourcetypes gw:rate. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain `venue_limit_ops` from exchange rule files; (2) map sessions to `algo_id`; (3) auto-throttle at gateway where possible—Splunk is detective; (4) alert desk head; (5) weekly summary to market conduct. Indexes required: `index=trading`; Sourcetypes: `sourcetype=gw:rate`; Lookups: `algo_session_map`, `algo_session_map.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Gateway logs (FIX, binary protocol) via HEC.
• Ensure the following data sources are available: `index=trading` `sourcetype="gw:rate"` (session_id, venue, orders_per_sec, venue_limit_ops, breach_flag, _time).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `venue_limit_ops` from exchange rule files; (2) map sessions to `algo_id`; (3) auto-throttle at gateway where possible—Splunk is detective; (4) alert desk head; (5) weekly summary to market conduct.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=trading sourcetype="gw:rate" breach_flag=true earliest=-30d
| stats count as breaches, perc95(orders_per_sec) as p95_ops by venue, session_id
| lookup algo_session_map.csv session_id OUTPUT algo_id, desk
| sort - breaches
```
