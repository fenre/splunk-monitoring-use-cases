# 22.8.38 — SOC 2 C1.1 — Confidentiality: sensitive-data exposure at the egress boundary

**Criticality:** high | **Difficulty:** advanced | **Status:** verified

## Description

Surfaces DLP detections for confidential-class data whose destination is not in the approved-egress registry.

## Value

C1.1 confidentiality assurance hinges on continuous detection of data leaving the boundary. This UC gives the DPO a queryable control output.

## SPL

```spl
index=dlp sourcetype IN (symantec:dlp,microsoft:purview:dlp) severity IN ("high","critical") earliest=-30m
| rename destination_ip AS dest policy_name AS dlp_policy
| lookup approved_egress.csv dest OUTPUT approved
| where approved!="yes" OR isnull(approved)
| stats count BY dlp_policy severity user dest approved
```

## Implementation

(1) Onboard DLP; (2) maintain approved_egress.csv; (3) schedule UC every 30m; (4) hit opens a P1 to DPO + user's manager; (5) weekly DPO report. Indexes required: `index=dlp`; Sourcetypes: `sourcetype=microsoft:purview:dlp`, `sourcetype=microsoft:purview:dlp;`, `sourcetype=symantec:dlp`.

## Visualization

Table of unapproved egress detections, bar chart by policy, single value 'new unapproved egress (1h)'.

## Compliance

- **SOC-2** 2017 TSC: C1.1 (full) — C1.1 Confidentiality — continuous DLP boundary detection is the direct evidence.
- **ISO/IEC 27001** 2022: A.8.12 (full) — ISO/IEC 27001:2022 A.8.12 Data leakage prevention — directly evidenced.
- **HIPAA Security Rule** 2013-final: §164.312(e)(1) (full) — Uplifted to full: SPL provides comprehensive, standalone evidence for §164.312(e)(1) with specific data sources and actionable thresholds.
- **PCI-DSS** v4.0: 10.6.2 (partial) — PCI-DSS 10.6.2 Review logs for anomalies (applied to DLP) — contributing.

## Known False Positives

Legitimate cross-tenant file shares with approved partners can produce detections — allowlist by partner in approved_egress.csv.
