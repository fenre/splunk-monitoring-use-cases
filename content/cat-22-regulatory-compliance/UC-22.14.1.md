# UC-22.14.1 · Centralized Audit Event Logging Policy Coverage

## Description

Shows which systems and sourcetypes are forwarding security-relevant events so auditors can evidence that logging policy scope matches AU-2 event types.

## Value

Shows which systems and sourcetypes are forwarding security-relevant events so auditors can evidence that logging policy scope matches AU-2 event types.

## Implementation

(1) Maintain `nist_au2_event_coverage.csv` mapping sourcetypes to AU-2 event families; (2) schedule daily and alert on sourcetypes with `required=true` and zero volume; (3) onboard missing forwarders or enable missing channels; (4) attach evidence export for assessors.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263), Splunk Common Information Model Add-on (Splunkbase 1621).
• Ensure the following data sources are available: `index=sec` OR `index=os` — Windows Security, Linux auditd, cloud control planes (sourcetypes vary by TA).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Maintain `nist_au2_event_coverage.csv` mapping sourcetypes to AU-2 event families; (2) schedule daily and alert on sourcetypes with `required=true` and zero volume; (3) onboard missing forwarders or enable missing channels; (4) attach evidence export for assessors.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index IN (sec, os, cloud) earliest=-24h
| stats dc(host) as hosts, dc(sourcetype) as sourcetypes, count as events by index, sourcetype
| lookup nist_au2_event_coverage.csv sourcetype OUTPUT au2_event_family, required
| eval gap=if(required="true" AND events=0, "No_Data", "OK")
| sort index, sourcetype
```
