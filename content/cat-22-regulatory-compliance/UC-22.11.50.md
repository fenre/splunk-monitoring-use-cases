# UC-22.11.50 · Account Lifecycle — New AD Users with Immediate CDE Group Assignment

## Description

Detects user creations followed within a short window by addition to CDE-privileged groups so provisioning follows lifecycle controls and management approval for access to account data.

## Value

Gives the responsible team auditor-ready evidence for PCI DSS compliance by automating regulatory compliance evidence using indexes windows and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Map `TargetUserName` alignment between events—may require `rename` on 4720 field `TargetUserName` as new user and 4728 `MemberName`. (2) Adjust `transaction` keys per environment testing. (3) Alert IAM for out-of-band adds. (4) Evidence for user identification procedures. Indexes required: `index=windows`; Sourcetypes: `sourcetype=WinEventLog:Security`; Lookups: `pci_cde_ad_groups`, `pci_cde_ad_groups.csv`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=windows` EventCode 4720 (user created), 4728 (member added to security-enabled global group).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Map `TargetUserName` alignment between events—may require `rename` on 4720 field `TargetUserName` as new user and 4728 `MemberName`. (2) Adjust `transaction` keys per environment testing. (3) Alert IAM for out-of-band adds. (4) Evidence for user identification procedures.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=windows sourcetype="WinEventLog:Security" EventCode IN (4720,4728) earliest=-24h
| transaction TargetUserName maxspan=60m startswith=EventCode=4720 endswith=EventCode=4728
| lookup pci_cde_ad_groups.csv GroupName OUTPUT cde_group
| where cde_group="true"
| table duration, TargetUserName, GroupName, SubjectUserName
```
