# 22.27.5 — Security of essential services mapping

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Supports UK UK NIS Regulations supervisory expectations by correlating technical telemetry with governance records for security of essential services mapping.

## Value

Gives the responsible team auditor-ready evidence for UK NIS compliance by automating regulatory compliance evidence using indexes cmdb, itsi_grouped_alerts and sourcetypes service:dependency. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=cmdb sourcetype="service:dependency" OR index=itsi_grouped_alerts earliest=-7d
| stats values(parent_service) as upstream, values(child_service) as downstream by business_service
| search business_service="*essential*"
| mvexpand downstream
| stats dc(downstream) as dep_count by business_service
| sort - dep_count
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Change.All_Changes by All_Changes.action, All_Changes.object_category, All_Changes.user | sort - count
```

## Implementation

(1) Maintain `uk_oes_inventory.csv` with OES identifiers, essential services, and CAF mappings. (2) Tag correlation searches with `nis_relevant` or `fca_relevant` for workflow routing. (3) Integrate IR ticketing milestones for competent authority and FCA clocks. (4) Monthly review with compliance, technology risk, and resilience forums. Indexes required: `index=cmdb`, `index=itsi_grouped_alerts`; Sourcetypes: `sourcetype=service:dependency`.

## Visualization

Time chart (event volume), Heatmap (service x control), Table (top risk drivers). Key fields: `business_service`, `dep_count`, `downstream`, `upstream`.

## Compliance

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
