# 22.21.16 — Quality Management System Control Evidence Freshness (Art. 17)

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Monitors last_review date for each QMS control_id to ensure Art. 17 quality management system remains current.

## Value

Gives the responsible team auditor-ready evidence for EU AI Act compliance by automating regulatory compliance evidence using indexes qms and sourcetypes qms:control:event, qms:control:event`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=qms sourcetype="qms:control:event" earliest=-400d
| stats latest(last_review) as lr_epoch by control_id, owner
| eval days_since=round((now()-lr_epoch)/86400,1)
| where days_since > 365 OR isnull(lr_epoch)
| sort - days_since
| table control_id, owner, days_since
```

## Implementation

(1) Ingest QMS export nightly; (2) map owners to RACI; (3) alert >365d; (4) attach evidence artifact hash field when available. Indexes required: `index=qms`; Sourcetypes: `sourcetype=qms:control:event`, `sourcetype=qms:control:event``.

## Visualization

Table (stale controls), Bar chart (days_since distribution), Single value (stale count)

## Known False Positives
