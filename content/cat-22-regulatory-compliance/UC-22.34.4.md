# 22.34.4 — Operator MFA and session integrity monitoring

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

Produces SWIFT Customer Security Programme evidence for operator mfa and session integrity monitoring ahead of independent assessment.

## Value

Gives the responsible team auditor-ready evidence for SWIFT CSP compliance by automating regulatory compliance evidence using indexes cyberark, o365 and sourcetypes cyberark:pta, ms:o365:management. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=cyberark sourcetype="cyberark:pta" OR index=o365 sourcetype="ms:o365:management" earliest=-7d
| stats count by user, action, src
| search user="swift*" (action="Logon" OR Operation="UserLoggedIn")
| eval mfa=if(match(_raw,"MFA"),1,0)
| where mfa=0
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.user, Authentication.action, Authentication.src | sort - count
```

## Implementation

(1) Segment SWIFT CSP evidence into dedicated indexes with strict role-based access. (2) Join operators to HR screening and vendor risk lookups refreshed weekly. (3) Validate MFA and PAM sourcetypes are CIM-mapped for Authentication models. (4) Retain evidence for at least 13 months aligned to attestation cycles. Indexes required: `index=cyberark`, `index=o365`.

## Visualization

Table (operators), Single value (sessions), Bar chart (CSCF controls). Key fields: `action`, `mfa`, `src`, `user`.

## Compliance

- **SWIFT CSP** CSCF v2025: 6.4 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 6.4.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
