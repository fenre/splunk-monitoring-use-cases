# 22.10.8 — Contingency Plan — Backup Job Success for ePHI Databases

**Criticality:** critical | **Difficulty:** intermediate | **Status:** N/A

## Description

Monitors backup completion for Clarity/Caboodle and clinical data stores so availability safeguards in the contingency plan are continuously evidenced.

## Value

Gives the responsible team auditor-ready evidence for HIPAA Security compliance by automating regulatory compliance evidence using indexes backup and sourcetypes commvault:job, commvault:job`. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=backup sourcetype="commvault:job" earliest=-48h (database="CLARITY" OR database="CABOODLE" OR app="Epic*")
| eval success=if(match(lower(status),"success|completed"),1,0)
| stats sum(success) as ok_jobs count as total_jobs latest(_time) as last_job by database, client_name
| eval health=if(ok_jobs=total_jobs,"OK","FAIL")
| where health!="OK"
```

## Implementation

(1) Tag databases containing ePHI in backup catalog; (2) page DBAs on failure; (3) quarterly restore test outcomes ingested as separate sourcetype for correlation. Indexes required: `index=backup`.

## Visualization

Single value (failed jobs), Table (DB, last success), Timeline (job duration anomalies).

## Compliance

- **HIPAA Security** 2013-final: §164.308(a)(7) (partial) — Uplifted from contributing: SPL directly addresses §164.308(a)(7) requirements with specific, actionable monitoring.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
