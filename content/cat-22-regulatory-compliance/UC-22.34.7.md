# 22.34.7 — External transmission data protection monitoring

**Criticality:** critical | **Difficulty:** expert | **Status:** N/A

## Description

Produces SWIFT Customer Security Programme evidence for external transmission data protection monitoring ahead of independent assessment.

## Value

Gives the responsible team auditor-ready evidence for SWIFT CSP compliance by automating regulatory compliance evidence using indexes network, ssl and sourcetypes pan:traffic, stream:tls. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=network sourcetype="pan:traffic" OR index=ssl sourcetype="stream:tls" earliest=-24h
| stats dc(cipher) as ciphers, values(ssl_version) as tls by dest_ip, dest_port
| where dest_port=443 AND mvfind(tls,"TLS1_0")>=0
| lookup swift_external_endpoints.csv dest_ip OUTPUT approved
| where approved!="true"
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.dest | sort - count
```

## Implementation

(1) Segment SWIFT CSP evidence into dedicated indexes with strict role-based access. (2) Join operators to HR screening and vendor risk lookups refreshed weekly. (3) Validate MFA and PAM sourcetypes are CIM-mapped for Authentication models. (4) Retain evidence for at least 13 months aligned to attestation cycles. Indexes required: `index=network`, `index=ssl`.

## Visualization

Table (operators), Single value (sessions), Bar chart (CSCF controls). Key fields: `ciphers`, `dest_ip`, `dest_port`, `tls`.

## Compliance

- **SWIFT CSP** CSCF v2025: 6.4 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 6.4.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
