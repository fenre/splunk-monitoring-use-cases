# 22.15.15 — Short or Default Password Patterns in OT Authentication Logs

**Criticality:** high | **Difficulty:** advanced | **Status:** N/A

## Description

Uses policy checks on authentication metadata (where available) and heuristics on failed logon bursts to catch weak password hygiene in OT accounts.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes fw, windows and sourcetypes fortigate_event. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
(index=windows EventCode="4625" Status!="0x0") OR (index=fw sourcetype="fortigate_event" subtype="vpn")
| eval user=coalesce(Account_Name, user, vpnuser)
| where match(user,"(?i)hmi|operator|ot_")
| bin _time span=1h
| stats count as fails by _time, user, src_ip
| where fails>30
| sort - fails
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Authentication.Authentication by Authentication.user, Authentication.src span=1h | sort - count
```

## Implementation

(1) Never log cleartext passwords—use failure reason codes only; (2) pair with AD password policy audit offline; (3) enforce lockout thresholds on jump hosts; (4) reset compromised accounts via PAM. Indexes required: `index=fw`, `index=windows`; Sourcetypes: `sourcetype=fortigate_event`.

## Visualization

Time chart (fail spikes), Table (user, src), Single value (distinct users).

## Compliance

- **IEC 62443** 2013-ongoing: SR 1.7 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 1.7.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
