# UC-22.7.38 · Risk Index Spike for Privileged Accounts

## Description

Surfaces elevated Splunk ES risk scores for privileged users for layered adverse event analysis.

## Value

Surfaces elevated Splunk ES risk scores for privileged users for layered adverse event analysis.

## Implementation

(1) Calibrate thresholds; (2) enrich with geo and MFA; (3) pair with analyst workflow.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise Security (Splunkbase 263).
• Ensure the following data sources are available: `from datamodel Risk.All_Risk`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Calibrate thresholds; (2) enrich with geo and MFA; (3) pair with analyst workflow.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| from datamodel Risk.All_Risk earliest=-7d
| search risk_object_type=user risk_object=*admin*
| stats sum(risk_score) as score by risk_object
| where score>50
```
