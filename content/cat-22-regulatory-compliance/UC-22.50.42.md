# 22.50.42 — UN R156 7.1.1 — Software update management: OTA deployment tracking and integrity verification

**Criticality:** high | **Difficulty:** advanced | **Status:** verified

## Description

Monitors OTA software update campaign success rates, integrity verification, and deployment coverage per UN R156 7.1.1 requirements for software update management system processes.

## Value

UN R156 requires a validated SUMS (Software Update Management System). Tracking deployment success and integrity verification provides auditable evidence that the update process is secure and complete.

## SPL

```spl
index=ota_updates sourcetype IN (ota_campaign, ota_vehicle_status) earliest=-30d
| eval update_successful=if(status IN ("installed", "verified", "activated"), 1, 0)
| eval integrity_verified=if(signature_check="pass" AND hash_check="pass", 1, 0)
| stats count AS total_updates sum(update_successful) AS successful sum(integrity_verified) AS integrity_pass dc(vin) AS vehicles_targeted latest(campaign_date) AS last_campaign BY campaign_id, software_version, vehicle_model
| eval success_pct=round(100*successful/total_updates, 1)
| eval integrity_pct=round(100*integrity_pass/total_updates, 1)
| where success_pct<100 OR integrity_pct<100
| table campaign_id, vehicle_model, software_version, total_updates, success_pct, integrity_pct, vehicles_targeted, last_campaign
```

## Implementation

(1) Ingest OTA campaign and vehicle update status events; (2) normalise signature_check, hash_check, status fields; (3) schedule per-campaign; (4) escalate integrity failures immediately. Indexes required: `index=ota_updates`; Sourcetypes: `sourcetype=ota_campaign`, `sourcetype=ota_vehicle_status`.

## Visualization

Table of flagged records for investigation; single-value KPI of coverage / compliance percentage; time chart of trend over the last 90 days; drill-down per responsible owner. Key fields: `campaign_id`, `integrity_pass`, `integrity_pct`, `integrity_verified`, `last_campaign`, `software_version`.

## Compliance

- **UN R156** 2021: 7.1.1 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for 7.1.1.

## Known False Positives

Tuning prerequisites: source field mappings must be in place before the UC will produce reliable metrics; early runs will commonly flag unmapped systems until onboarding is completed.
