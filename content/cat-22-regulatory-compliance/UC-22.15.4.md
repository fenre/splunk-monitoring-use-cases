# UC-22.15.4 ┬À Privileged OT Account Changes vs Personnel Screening

## Description

Flags new or elevated OT privileged accounts that lack matching screening status, supporting personnel security for IACS access paths.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes cyberark, hr, windows and sourcetypes cyberark:cpm, cyberark:pta, hr:screening. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Enable AD auditing for sensitive OT groups; (2) ingest CyberArk for break-glass and vendor IDs; (3) HR feed with legal basis documented; (4) remediate within SLA. Indexes required: `index=cyberark`, `index=hr`, `index=windows`; Sourcetypes: `sourcetype=cyberark:cpm`, `sourcetype=cyberark:pta`, `sourcetype=hr:screening`.

Step 2 ÔÇö Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
((index=windows EventCode IN ("4720","4728","4732") Message="*OT_*") OR (index=cyberark sourcetype IN ("cyberark:pta","cyberark:cpm")))
| where index="windows" OR match(_raw,"(?i)OT_|PLC_|SCADA")
| eval account=coalesce(Account_Name, user, dest_user)
| stats earliest(_time) as first_evt, latest(_time) as last_evt by account, EventCode, host
| join type=left account [ search index=hr sourcetype="hr:screening" | stats latest(status) as screening by samaccountname | rename samaccountname as account ]
| where screening!="Cleared" OR isnull(screening)
| table first_evt, last_evt, account, EventCode, screening, host
```
