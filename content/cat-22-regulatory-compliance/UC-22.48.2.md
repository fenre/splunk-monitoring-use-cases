# UC-22.48.2 · SoD violations via break-glass usage — emergency role abuse

## Description

Detects break-glass sessions that executed actions which would normally violate segregation of duties. Unless there is an emergency-exception ticket, the session breaches SoD even if the break-glass approval is otherwise valid.

## Value

Break-glass is often abused to step around SoD; this UC catches that pattern without waiting for audit.

## Implementation

(1) Tag PAM session actions into SoD categories; (2) maintain sod_matrix.csv; (3) require emergency exceptions to log to GRC; (4) run hourly.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Enterprise / Splunk Cloud Platform.
• Ensure the following data sources are available: See subcategory preamble..
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Tag PAM session actions into SoD categories; (2) maintain sod_matrix.csv; (3) require emergency exceptions to log to GRC; (4) run hourly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=pam breakglass=true earliest=-24h
| stats values(action_category) as actions by session_id account
| mvexpand actions
| lookup sod_matrix.csv action_a=actions OUTPUT toxic_against
| mvexpand toxic_against
| join session_id toxic_against [search index=pam breakglass=true earliest=-24h | eval hit=action_category]
| join session_id [search index=grc sourcetype=emergency:exception | fields session_id exception_id]
| where isnull(exception_id)
| table session_id account actions toxic_against
```
