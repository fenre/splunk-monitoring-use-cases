# UC-22.13.18 Â· Interactive Remote Access Session Duration on Jump Hosts

## Description

We track RDP or vendor remote sessions through registered jump hosts so interactive remote access is visible for ESP monitoring.

## Value

Gives the responsible team auditor-ready evidence for NERC CIP compliance by automating regulatory compliance evidence using indexes wineventlog and sourcetypes WinEventLog:Security. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Maintain jump host list aligned with CIP-005 EAP documentation. (2) Ensure logoff events are audited. (3) Cross-check CyberArk session metadata. (4) Alert on sessions exceeding policy without active work order. Indexes required: `index=wineventlog`; Sourcetypes: `sourcetype=WinEventLog:Security`; Lookups: `bes_jump_hosts`, `bes_jump_hosts.csv`.

Step 2 â€” Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode IN (4624,4634) LogonType=10 earliest=-24h
| lookup bes_jump_hosts.csv ComputerName OUTPUT jump_id esp_segment
| where isnotnull(jump_id)
| transaction Account_Name maxspan=7200 startswith=(EventCode=4624) endswith=(EventCode=4634)
| eval duration_min=round(duration/60,2)
| table _time, Account_Name, ComputerName, esp_segment, duration_min
```
