# UC-22.6.13 · Cloud Shared Responsibility Control Coverage Map

## Description

Joins cloud configuration findings to a shared-responsibility matrix so use of cloud services shows which Annex controls are organization-operated.

## Value

Joins cloud configuration findings to a shared-responsibility matrix so use of cloud services shows which Annex controls are organization-operated.

## Implementation

(1) Populate lookup from CSP matrix; (2) prioritize CRITICAL; (3) supplier review input.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for AWS (Splunkbase 1876), Splunk App for AWS Security Hub.
• Ensure the following data sources are available: `aws:securityhub_finding`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate lookup from CSP matrix; (2) prioritize CRITICAL; (3) supplier review input.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=aws sourcetype=aws:securityhub_finding earliest=-7d RecordState=ACTIVE
| lookup csr_shared_responsibility.csv control_id OUTPUT annex_owner
| stats count by annex_owner, Severity.Label
```
