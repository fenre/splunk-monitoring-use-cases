# 22.16.21 — Control-loop latency before vs after hardening window

**Criticality:** high | **Difficulty:** beginner | **Status:** N/A

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "Control-loop latency before vs after hardening window".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes ot and sourcetypes edge_hub:opcua. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot sourcetype=edge_hub:opcua metric_name="*LoopLatency*" earliest=-30d
| eval uc_tag="22.16.21"
| eval phase=if(_time<relative_time(now(),"-14d@d"),"before","after")
| stats perc95(metric_value) as p95 by device_id, phase
```

## CIM SPL

```spl
| tstats summariesonly=t count from datamodel=Network_Traffic.All_Traffic by All_Traffic.action, All_Traffic.src, All_Traffic.dest, All_Traffic.dest_port | sort - count
```
