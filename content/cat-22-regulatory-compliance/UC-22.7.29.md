# UC-22.7.29 · DNS Resolver Error Rate SLO for Internal Resolvers

## Description

Tracks DNS infrastructure error rates as resilience signal for critical name resolution dependency.

## Value

Gives the responsible team auditor-ready evidence for NIST CSF compliance by automating regulatory compliance evidence using indexes dns and sourcetypes dns:query. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Set SLO thresholds; (2) correlate with DDoS; (3) include in DR tabletop; (4) document failover. Indexes required: `index=dns`; Sourcetypes: `sourcetype=dns:query`.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: BIND or CoreDNS logs.
• Ensure the following data sources are available: `dns:query`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Set SLO thresholds; (2) correlate with DDoS; (3) include in DR tabletop; (4) document failover.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=dns sourcetype=dns:query earliest=-24h
| eval fail=if(match(_raw,"(?i)SERVFAIL|REFUSED"),1,0)
| timechart span=1h sum(fail) as failures
```
