# 22.15.28 — PLC Firmware Verification Job Failures

**Criticality:** high | **Difficulty:** intermediate | **Status:** N/A

## Description

Tracks scheduled integrity or attestation jobs on controllers that fail or never run, weakening verification of security functionality.

## Value

Gives the responsible team auditor-ready evidence for IEC 62443 compliance by automating regulatory compliance evidence using indexes ot_integrity and sourcetypes plc:integrity_scan. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## SPL

```spl
index=ot_integrity sourcetype="plc:integrity_scan" earliest=-30d
| stats latest(result) as last_res, latest(_time) as last_run by controller, firmware_version
| eval days_since=round((now()-last_run)/86400,0)
| where last_res!="PASS" OR days_since>30
| table controller, firmware_version, last_res, days_since
```

## Implementation

(1) Schedule vendor integrity checks post-patch; (2) ingest pass/fail JSON; (3) integrate with patch CMDB; (4) block production promotion on FAIL. Indexes required: `index=ot_integrity`; Sourcetypes: `sourcetype=plc:integrity_scan`.

## Visualization

Table, Single value (controllers overdue), Time chart.

## Compliance

- **IEC 62443** 2013-ongoing: SR 3.3 (partial) — Uplifted from contributing to partial: UC provides meaningful monitoring evidence for SR 3.3.

## Known False Positives

Administrative tasks, scheduled jobs or platform updates can match this pattern — correlate with change management, maintenance windows and user role before raising severity.
