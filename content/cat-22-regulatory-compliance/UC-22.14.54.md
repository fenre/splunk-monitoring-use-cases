# UC-22.14.54 · Security Impact Analysis Signals for Emergency Changes

## Description

Flags emergency changes with subsequent security alerts for CM-4 security impact analysis feedback.

## Value

Flags emergency changes with subsequent security alerts for CM-4 security impact analysis feedback.

## Implementation

(1) Require CHG id in ES annotations via workflow; (2) tune join window; (3) monthly emergency change review deck; (4) document false positive handling.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for ServiceNow (Splunkbase 1928), Splunk Enterprise Security (Splunkbase 263).
• Ensure the following data sources are available: `snow:change_request` type=emergency, `` `notable` ``.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Require CHG id in ES annotations via workflow; (2) tune join window; (3) monthly emergency change review deck; (4) document false positive handling.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype=snow:change_request type=emergency earliest=-30d
| eval cid=number
| join type=left cid [
    search `notable` earliest=-30d
    | rex field=annotations "(?i)chg:(?<cid>CHG[0-9]+)"
    | stats count by cid
  ]
| where count>0
```
