# UC-22.36.3 · Portability export integrity — signed manifest verification

## Description

Validates that the bytes delivered to a data subject match the signed manifest produced when the export was generated. Any divergence is evidence of in-transit tampering or a broken CDN path; either way the export cannot be trusted as a lawful Art.20 response.

## Value

Adds cryptographic evidence to what is otherwise a 'we generated the file' audit trail.

## Implementation

(1) Sign manifests at export time with an HSM key scoped to the privacy program; (2) log manifest_hash and delivered_hash at each delivery; (3) run hourly; (4) on hit, auto-open an Art.12 remediation ticket. Indexes required: `index=dsar`; Sourcetypes: `sourcetype=portability:export`, `sourcetype=portability:manifest`.

## Known False Positives

Content encoding transforms can change the on-wire hash. Require manifest to record pre-compression hash and compare apples to apples.
