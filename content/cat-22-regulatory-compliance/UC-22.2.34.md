# UC-22.2.34 · NIS2 Cross-Border Incident Coordination Task Tracking

## Description

When incidents affect multiple Member States, parallel coordination tasks multiply. This use case tracks per-country CSIRT notification tickets, language-specific attachments, and follow-up deadlines — complementing UC-22.2.19's technical cross-border detection.

## Value

Gives the responsible team auditor-ready evidence for NIS2 compliance by automating regulatory compliance evidence using indexes itsm and sourcetypes snow:sc_task. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) When cross-border flag is set, auto-spawn tasks per Member State template; (2) track filing confirmations as attachments; (3) alert incident commander on open_tasks; (4) post-incident merge lessons into UC-22.2.23; (5) legal owns register accuracy. Indexes required: `index=itsm`; Sourcetypes: `sourcetype=snow:sc_task`; Lookups: `nis2_cross_border_incident_register`, `nis2_cross_border_incident_register.csv`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
| inputlookup nis2_cross_border_incident_register.csv
| join incident_id [
    search index=itsm sourcetype="snow:sc_task" short_description="*CSIRT*" earliest=-90d
    | stats dc(short_description) as tasks_created sum(eval(case(state!="Closed",1,0))) as open_tasks by parent_incident
    | rename parent_incident as incident_id
]
| eval expected_tasks=affected_member_states*2
| where open_tasks > 0 OR tasks_created < expected_tasks
| table incident_id, affected_member_states, tasks_created, open_tasks, expected_tasks
```
