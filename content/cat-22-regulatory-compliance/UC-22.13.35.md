# UC-22.13.35 · Default, Built-In, or Generic Account Usage on BES Hosts

## Description

We flag logons using default or generic accounts that should be disabled or renamed per system hardening standards.

## Value

We flag logons using default or generic accounts that should be disabled or renamed per system hardening standards.

## Implementation

(1) Populate blocklist with Administrator, Guest, vendor defaults. (2) Tune for renamed admin accounts via SID lookup if available. (3) Page IR on hits. (4) Document approved break-glass in lookup.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk Add-on for Microsoft Windows (742).
• Ensure the following data sources are available: `index=wineventlog` `EventCode=4624`, `generic_accounts_blocklist.csv`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
(1) Populate blocklist with Administrator, Guest, vendor defaults. (2) Tune for renamed admin accounts via SID lookup if available. (3) Page IR on hits. (4) Document approved break-glass in lookup.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=wineventlog sourcetype="WinEventLog:Security" EventCode=4624 earliest=-7d
| lookup bes_cyber_asset_inventory.csv ComputerName AS host OUTPUT bes_system
| where isnotnull(bes_system)
| lookup generic_accounts_blocklist.csv Account_Name OUTPUT blocked_account
| where isnotnull(blocked_account)
| stats count by Account_Name, ComputerName, bes_system
| sort - count
```
