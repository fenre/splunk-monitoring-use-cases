# UC-22.16.11 · Shared OT maintenance account attribution via PAM

## Description

Supports pipeline cybersecurity program oversight under TSA Pipeline Security Directive by correlating operational telemetry with governance records for "Shared OT maintenance account attribution via PAM".

## Value

Gives the responsible team auditor-ready evidence for TSA SD compliance by automating regulatory compliance evidence using indexes pam, wineventlog and sourcetypes WinEventLog:Security, cyberark:pta. Replaces manual evidence gathering with a continuous Splunk search that produces structured, time-stamped artefacts suitable for regulatory examination and internal audit review.

## Implementation

(1) Confirm field extractions and CIM tags for the sourcetypes in scope; (2) Load or maintain the referenced lookups/KVStore collections with owner attestation dates; (3) Schedule the search at an interval aligned to the control’s materiality; (4) Route positive findings to the compliance ticketing queue with required evidence fields; (5) Retain scheduled search exports per records management and legal hold procedures. Indexes required: `index=pam`, `index=wineventlog`.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=pam sourcetype=cyberark:pta earliest=-7d safe="OT-Maint"
| eval uc_tag="22.16.11"
| stats earliest(_time) as t0, latest(_time) as t1, values(device) as targets by user, session_id
| join type=left user [
    search index=wineventlog sourcetype=WinEventLog:Security EventCode=4624 Account_Name="OTMAINT" earliest=-7d | stats values(IpAddress) as ips by _time, user
  ]
| table user, session_id, targets, ips
```
