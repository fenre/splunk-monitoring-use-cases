# UC-18.2.11 · NAT Rule Utilization

## Description

Tracks NAT session and port allocation per rule — prevents exhaustion on busy Edge gateways.

## Value

Tracks NAT session and port allocation per rule — prevents exhaustion on busy Edge gateways.

## Implementation

Session limits depend on Edge form factor; load from capacity sheet via lookup. Alert at 85%.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `vmware_nsx_addon`.
• Ensure the following data sources are available: `vmware:nsx:nat_stats`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Session limits depend on Edge form factor; load from capacity sheet via lookup. Alert at 85%.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vmware sourcetype="vmware:nsx:nat_stats" earliest=-1h
| eval used_pct=round(100*active_sessions/session_limit,1)
| where used_pct>85
| table edge_node, rule_id, active_sessions, session_limit, used_pct
| sort -used_pct
```
