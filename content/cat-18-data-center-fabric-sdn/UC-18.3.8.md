# UC-18.3.8 · SDN Configuration Change and Rollback Audit

## Description

Unauthorized fabric or policy changes can cause outages or security gaps. Auditing changes and rollbacks supports change control and incident analysis.

## Value

Unauthorized fabric or policy changes can cause outages or security gaps. Auditing changes and rollbacks supports change control and incident analysis.

## Implementation

Ingest controller and fabric audit logs. Alert on change to critical objects (e.g., tenant, contract, segment) without change ticket. Report on change frequency and rollback rate. Integrate with change management.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: APIC/NSX/controller audit logs.
• Ensure the following data sources are available: Configuration change events, user, object, before/after.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest controller and fabric audit logs. Alert on change to critical objects (e.g., tenant, contract, segment) without change ticket. Report on change frequency and rollback rate. Integrate with change management.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=sdn sourcetype="sdn:audit"
| search (action="modified" OR action="deleted" OR action="rollback")
| table _time, user, object_type, object_name, action, change_summary
| sort -_time
```
