# UC-18.4.7 · Nexus Dashboard Orchestrator Cross-Fabric Consistency

## Description

Nexus Dashboard Orchestrator (NDO, formerly MSO) manages policies across multiple ACI fabrics or NDFC-managed NX-OS fabrics. Configuration inconsistencies between sites cause asymmetric routing, broken inter-site connectivity, and policy enforcement gaps.

## Value

Nexus Dashboard Orchestrator (NDO, formerly MSO) manages policies across multiple ACI fabrics or NDFC-managed NX-OS fabrics. Configuration inconsistencies between sites cause asymmetric routing, broken inter-site connectivity, and policy enforcement gaps.

## Implementation

Poll NDO deployment status via REST API. Detect schema deployment failures and pending diffs between sites. Alert on any site showing stale or failed deployment. Cross-reference with ACI multisite health (UC-18.1.17).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Cisco DC Networking Application (Splunkbase 7777), NDO REST API scripted input.
• Ensure the following data sources are available: NDO audit logs, schema/template deployment status, `cisco:ndo:audit`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Poll NDO deployment status via REST API. Detect schema deployment failures and pending diffs between sites. Alert on any site showing stale or failed deployment. Cross-reference with ACI multisite health (UC-18.1.17).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_dc sourcetype="cisco:ndo:audit" earliest=-24h
| stats count by action, template_name, site_name, status, user
| where status!="deployed" OR action IN ("failed","conflict")
| table template_name, site_name, action, status, user, count
| sort -count
```
