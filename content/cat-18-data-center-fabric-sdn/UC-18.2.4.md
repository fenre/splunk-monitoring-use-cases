# UC-18.2.4 · NSX Edge Performance

## Description

NSX Edge nodes handle north-south traffic, load balancing, and NAT. Performance bottlenecks on Edge nodes directly impact application availability and throughput for any workload communicating outside the NSX fabric.

## Value

NSX Edge nodes handle north-south traffic, load balancing, and NAT. Performance bottlenecks on Edge nodes directly impact application availability and throughput for any workload communicating outside the NSX fabric.

## Implementation

Collect Edge node metrics via NSX Manager API every 60 seconds. Monitor both management plane and datapath CPU separately. Track interface throughput on uplinks. Set thresholds: datapath CPU >80% critical, >60% warning. Plan Edge node scale-out when sustained utilization exceeds thresholds.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `vmware_nsx_addon`, NSX Edge metrics.
• Ensure the following data sources are available: NSX Edge node metrics (CPU, memory, datapath), NSX Manager API.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Collect Edge node metrics via NSX Manager API every 60 seconds. Monitor both management plane and datapath CPU separately. Track interface throughput on uplinks. Set thresholds: datapath CPU >80% critical, >60% warning. Plan Edge node scale-out when sustained utilization exceeds thresholds.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vmware sourcetype="vmware:nsx:edge_metrics"
| stats avg(cpu_pct) as avg_cpu, max(cpu_pct) as peak_cpu, avg(mem_pct) as avg_mem, avg(datapath_cpu_pct) as avg_dp_cpu by edge_node, cluster
| eval status=case(peak_cpu>90 OR avg_dp_cpu>80, "Critical", peak_cpu>75 OR avg_dp_cpu>60, "Warning", 1==1, "Healthy")
| table edge_node, cluster, avg_cpu, peak_cpu, avg_mem, avg_dp_cpu, status
| sort -peak_cpu
```
