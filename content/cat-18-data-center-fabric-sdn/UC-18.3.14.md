# UC-18.3.14 · Spine-Leaf Topology Anomalies

## Description

Detects unexpected BGP neighbor loss, missing spine links, or asymmetric ECMP paths in Clos topology.

## Value

Detects unexpected BGP neighbor loss, missing spine links, or asymmetric ECMP paths in Clos topology.

## Implementation

Maintain `expected_peers.csv` from design. Alert on any spine-leaf session not Established.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: BGP syslog, LLDP.
• Ensure the following data sources are available: `bgp:neighbor`, `lldp:topology`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Maintain `expected_peers.csv` from design. Alert on any spine-leaf session not Established.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="bgp:neighbor" earliest=-4h
| where state!="Established"
| lookup expected_bgp_peers.csv local_host peer_ip OUTPUT 1 as expected
| where isnotnull(expected)
| stats count by local_host, peer_ip, state, reason
| sort -count
```
