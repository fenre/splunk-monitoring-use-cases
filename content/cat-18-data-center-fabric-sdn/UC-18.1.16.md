# UC-18.1.16 · ACI Tenant Configuration Compliance Audit

## Description

Checks tenants for required objects (vzAny restrictions, monitoring policies, SNMP/Syslog) — extends change audit (UC-18.1.5) with **policy completeness** scoring.

## Value

Checks tenants for required objects (vzAny restrictions, monitoring policies, SNMP/Syslog) — extends change audit (UC-18.1.5) with **policy completeness** scoring.

## Implementation

Build `tenant_summary` from scheduled API pulls (`fvTenant` + children). Adjust required attributes to your standards.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `TA_cisco-ACI`.
• Ensure the following data sources are available: APIC config export or `cisco:aci:audit`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Build `tenant_summary` from scheduled API pulls (`fvTenant` + children). Adjust required attributes to your standards.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_aci sourcetype="cisco:aci:tenant_summary"
| eval has_mon=isnotnull(mon_policy)
| eval has_snmp=isnotnull(snmp_group)
| where has_mon=0 OR has_snmp=0
| table tenant, has_mon, has_snmp
```
