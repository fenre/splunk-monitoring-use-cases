# UC-18.2.2 · Micro-Segmentation Enforcement

## Description

NSX micro-segmentation is a key Zero Trust control. Monitoring enforcement validates that workloads are properly isolated, detects lateral movement attempts, and proves compliance with segmentation policies during audits.

## Value

NSX micro-segmentation is a key Zero Trust control. Monitoring enforcement validates that workloads are properly isolated, detects lateral movement attempts, and proves compliance with segmentation policies during audits.

## Implementation

Define security groups in NSX aligned with application tiers. Enable DFW logging for inter-group and intra-group traffic. Enrich logs with security group membership. Track allowed vs denied inter-group communication. Alert on intra-group denials or unexpected inter-group allows.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `vmware_nsx_addon`, NSX DFW logs.
• Ensure the following data sources are available: NSX DFW logs, NSX security group membership.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Define security groups in NSX aligned with application tiers. Enable DFW logging for inter-group and intra-group traffic. Enrich logs with security group membership. Track allowed vs denied inter-group communication. Alert on intra-group denials or unexpected inter-group allows.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vmware sourcetype="vmware:nsx:dfw"
| lookup nsx_security_groups vm_name OUTPUT security_group
| stats count as hits, dc(dst_ip) as unique_destinations by security_group, action, direction
| eval compliance=if(action="DROP" AND direction="intra-group", "Violation", "Expected")
| sort -hits
```
