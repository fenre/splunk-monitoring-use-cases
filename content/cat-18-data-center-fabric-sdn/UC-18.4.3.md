# UC-18.4.3 · Nexus Dashboard Advisory and Field Notice Alerts

## Description

Nexus Dashboard Insights identifies field notices, PSIRTs, and hardware advisories that affect switches in the fabric by matching device serial numbers and software versions. Forwarding these to Splunk provides a centralised risk view alongside other infrastructure advisories.

## Value

Nexus Dashboard Insights identifies field notices, PSIRTs, and hardware advisories that affect switches in the fabric by matching device serial numbers and software versions. Forwarding these to Splunk provides a centralised risk view alongside other infrastructure advisories.

## Implementation

Export NDI advisories to Splunk via webhook or scheduled API poll. Correlate with asset inventory to calculate exposure percentage. Alert on critical PSIRTs or field notices affecting production fabrics.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Cisco DC Networking Application (Splunkbase 7777), NDI webhook.
• Ensure the following data sources are available: NDI advisory exports, `cisco:ndi:advisory`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Export NDI advisories to Splunk via webhook or scheduled API poll. Correlate with asset inventory to calculate exposure percentage. Alert on critical PSIRTs or field notices affecting production fabrics.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_dc sourcetype="cisco:ndi:advisory"
| stats count by advisory_id, advisory_type, severity, affected_switch_count
| where severity IN ("critical","high")
| table advisory_id, advisory_type, severity, affected_switch_count
| sort -severity
```
