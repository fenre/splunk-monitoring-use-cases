# UC-18.1.12 · Fabric Node Decommission Events

## Description

Audits leaf/spine/APIC removal or disable operations for change control and capacity reconciliation.

## Value

Audits leaf/spine/APIC removal or disable operations for change control and capacity reconciliation.

## Implementation

Tune search terms to APIC messages when nodes are drained from fabric. Correlate with maintenance windows.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `TA_cisco-ACI`, APIC audit/syslog.
• Ensure the following data sources are available: `cisco:aci:audit`, APIC syslog.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Tune search terms to APIC messages when nodes are drained from fabric. Correlate with maintenance windows.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_aci (sourcetype="cisco:aci:audit" OR sourcetype="cisco:aci:system")
| search decommission OR "node-remove" OR "unregister" OR "fabricDecommission"
| table _time, user, affected, descr
| sort -_time
```
