# UC-18.3.16 · VTEP Reachability and Loss

## Description

Packet loss and latency between VTEP peers — augments UC-18.3.9 state-only checks.

## Value

Packet loss and latency between VTEP peers — augments UC-18.3.9 state-only checks.

## Implementation

Run periodic probes between TEP IPs from automation. Correlate with underlay QoS drops.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `Splunk_TA_nix`, `TA-cisco_ios`, `arista:eos` via SC4S, ICMP probes, SNMP.
• Ensure the following data sources are available: `vtep:probe` or synthetic tests.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Run periodic probes between TEP IPs from automation. Correlate with underlay QoS drops.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=network sourcetype="vtep:probe" earliest=-24h
| where loss_pct>2 OR latency_ms>10
| stats avg(loss_pct) as avg_loss, avg(latency_ms) as avg_lat by src_vtep, dst_vtep
| sort -avg_loss
| head 20
```
