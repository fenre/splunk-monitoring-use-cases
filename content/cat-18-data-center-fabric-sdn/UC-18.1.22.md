# UC-18.1.22 ┬À Fabric Port-Channel and Member Link Imbalance

## Description

Uneven distribution across port-channel members defeats ECMP assumptions and can saturate individual uplinks while siblings stay idle. Detecting imbalance protects spine-leaf oversubscription models and avoids tail latency for storage and replication traffic.

## Value

Uneven distribution across port-channel members defeats ECMP assumptions and can saturate individual uplinks while siblings stay idle. Detecting imbalance protects spine-leaf oversubscription models and avoids tail latency for storage and replication traffic.

## Implementation

(1) Ensure `port_channel` is extracted from interface DN or API; map orphan physical members. (2) Alert when member spread exceeds policy (for example 35%). (3) Validate hashing and down-members with operational state feed.

Step 2 ÔÇö Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_aci sourcetype="cisco:aci:interface_stats" earliest=-2h
| where isnotnull(port_channel)
| stats sum(bytesRate) as br by node, port_channel, interface
| eventstats sum(br) as pc_total by node, port_channel
| eval member_pct=round(100*br/pc_total,2)
| eventstats range(member_pct) as spread by node, port_channel
| where spread > 35
| table node, port_channel, interface, member_pct, spread
| sort node, port_channel, -member_pct
```
