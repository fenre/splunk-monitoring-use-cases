# UC-18.1.13 · Bridge Domain Subnet Utilization

## Description

Tracks IP usage vs subnet size per BD to prevent exhaustion of gateway pools and VM mobility issues.

## Value

Tracks IP usage vs subnet size per BD to prevent exhaustion of gateway pools and VM mobility issues.

## Implementation

Ingest BD statistics from periodic API poll (`fvBD` subnets vs endpoint counts). Alert at 85%/95% thresholds.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `TA_cisco-ACI`, APIC BD API.
• Ensure the following data sources are available: `cisco:aci:bd_stats` or scripted API.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest BD statistics from periodic API poll (`fvBD` subnets vs endpoint counts). Alert at 85%/95% thresholds.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cisco_aci sourcetype="cisco:aci:bd_stats"
| eval used_pct=round(100*ip_in_use/total_ips,1)
| where used_pct > 85
| table tenant, bd, subnet, used_pct, ip_in_use, total_ips
| sort -used_pct
```
