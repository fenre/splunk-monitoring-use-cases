# UC-18.2.12 · T0/T1 Gateway Failover Events

## Description

Captures active/standby transitions for Tier-0/Tier-1 logical routers for incident correlation and HA validation.

## Value

Captures active/standby transitions for Tier-0/Tier-1 logical routers for incident correlation and HA validation.

## Implementation

Normalize syslog/API messages for HA events. Alert on any unplanned failover. Pager for T0.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `vmware_nsx_addon`.
• Ensure the following data sources are available: `vmware:nsx:lr_events`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Normalize syslog/API messages for HA events. Alert on any unplanned failover. Pager for T0.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=vmware sourcetype="vmware:nsx:lr_events" earliest=-7d
| search failover OR "HA switch" OR "role change"
| stats latest(_time) as last_seen, count by lr_name, prev_role, new_role, edge_node
| sort -last_seen
```
