# UC-12.4.3 · Ansible Playbook Outcomes

## Description

Tracking Ansible run results ensures configuration management is working. Failed tasks indicate systems in unknown state.

## Value

Tracking Ansible run results ensures configuration management is working. Failed tasks indicate systems in unknown state.

## Implementation

Configure Ansible Splunk callback plugin to send results to HEC. Track ok/changed/failed/unreachable counts per playbook and host. Alert on failed or unreachable hosts. Report on configuration management coverage.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Ansible callback plugin (Splunk HEC callback).
• Ensure the following data sources are available: Ansible callback output (play results, task results).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Configure Ansible Splunk callback plugin to send results to HEC. Track ok/changed/failed/unreachable counts per playbook and host. Alert on failed or unreachable hosts. Report on configuration management coverage.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iac sourcetype="ansible:result"
| stats sum(ok) as ok, sum(changed) as changed, sum(failed) as failed, sum(unreachable) as unreachable by playbook, host
| where failed > 0 OR unreachable > 0
```
