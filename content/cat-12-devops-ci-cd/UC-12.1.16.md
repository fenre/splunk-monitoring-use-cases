# UC-12.1.16 · Large File Commit Detection

## Description

Large blobs bloat repos, bypass review in diff viewers, and may hide embedded data.

## Value

Large blobs bloat repos, bypass review in diff viewers, and may hide embedded data.

## Implementation

Threshold 5MB default; tune per repo. Require LFS for binaries. Alert on repeated violations.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: GitHub `push` hook with size, Git LFS audit.
• Ensure the following data sources are available: Commit statistics (`size`, `distinct_size`), LFS upload events.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Threshold 5MB default; tune per repo. Require LFS for binaries. Alert on repeated violations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=devops sourcetype="github:webhook" event="push"
| where commits{}.size > 5242880 OR commits{}.distinct_size > 5242880
| table _time, repository, commits{}.id, commits{}.size, pusher
```
