# UC-12.3.4 · License Compliance Tracking

## Description

Open-source license violations create legal risk. Automated tracking ensures compliance before code reaches production.

## Value

Open-source license violations create legal risk. Automated tracking ensures compliance before code reaches production.

## Implementation

Ingest SCA license scan results. Track license types across all projects. Alert on copyleft licenses in commercial products. Report on license distribution for legal review. Block deployments with policy violations.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: SCA tool output.
• Ensure the following data sources are available: SCA license scan results (Snyk, FOSSA, WhiteSource).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest SCA license scan results. Track license types across all projects. Alert on copyleft licenses in commercial products. Report on license distribution for legal review. Block deployments with policy violations.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=devops sourcetype="sca:license"
| where license_risk IN ("high","critical") OR license IN ("GPL-3.0","AGPL-3.0")
| stats count by project, package_name, license, license_risk
| sort -license_risk
```
