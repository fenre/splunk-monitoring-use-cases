# UC-12.2.6 · Pipeline Queue Time

## Description

Long queue times indicate insufficient CI/CD runner capacity, slowing developer feedback loops and delivery velocity.

## Value

Long queue times indicate insufficient CI/CD runner capacity, slowing developer feedback loops and delivery velocity.

## Implementation

Track job queue wait times and queue lengths. Alert when average wait exceeds 5 minutes. Monitor runner/agent utilization. Report on peak hours to guide scaling decisions.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk App for Jenkins, CI/CD system metrics.
• Ensure the following data sources are available: CI/CD queue metrics (time in queue, pending jobs).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Track job queue wait times and queue lengths. Alert when average wait exceeds 5 minutes. Monitor runner/agent utilization. Report on peak hours to guide scaling decisions.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cicd sourcetype="jenkins:queue"
| timechart span=15m avg(wait_time_sec) as avg_wait, max(queue_length) as max_queue
| where avg_wait > 300
```
