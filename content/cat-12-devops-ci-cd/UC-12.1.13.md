# UC-12.1.13 · PR Review Bypass Detection

## Description

Merges with zero approvals or with only self-approval violate four-eyes policies.

## Value

Merges with zero approvals or with only self-approval violate four-eyes policies.

## Implementation

Ingest PR merged payload with review tally from API enrichment. Exclude bots via label. Alert in Splunk as backstop to branch protection.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: GitHub GraphQL / pull_request webhook.
• Ensure the following data sources are available: `pull_request` `closed` merged=true with `review_count`, `merge_method`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest PR merged payload with review tally from API enrichment. Exclude bots via label. Alert in Splunk as backstop to branch protection.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=devops sourcetype="github:pull_request" action="closed" merged="true"
| where review_count=0 OR (review_count=1 AND merger=author)
| search base_ref IN ("main","master","production")
| table _time, repository, author, merger, review_count, base_ref
```
