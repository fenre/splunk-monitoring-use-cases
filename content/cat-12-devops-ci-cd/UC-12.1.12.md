# UC-12.1.12 · Repository Permission Changes

## Description

Team `maintain`/`admin` grants and visibility changes to `public` are higher risk than member adds (UC-12.1.5).

## Value

Team `maintain`/`admin` grants and visibility changes to `public` are higher risk than member adds (UC-12.1.5).

## Implementation

Alert on public visibility toggles and admin grants. Quarterly access review export.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: GitHub audit log.
• Ensure the following data sources are available: `repo.update`, `team.add_repository`, `org.update_member`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Alert on public visibility toggles and admin grants. Quarterly access review export.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=devops sourcetype="github:audit" action IN ("repo.update","team.add_repository","repo.access")
| where visibility="public" OR permission IN ("admin","maintain")
| table _time, actor, repo, action, permission, visibility
```
