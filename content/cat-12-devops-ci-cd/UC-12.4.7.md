# UC-12.4.7 · Container Image Build and Push Audit

## Description

Unauthorized or untagged image pushes can introduce risk. Auditing build and push events supports supply chain security and compliance.

## Value

Unauthorized or untagged image pushes can introduce risk. Auditing build and push events supports supply chain security and compliance.

## Implementation

Ingest registry audit and CI build events. Alert on push from unexpected identity or to production repo without tag policy. Report on image provenance and push frequency.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Registry and CI logs (ECR, ACR, Harbor, Docker).
• Ensure the following data sources are available: Image push events, build logs, registry audit.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest registry audit and CI build events. Alert on push from unexpected identity or to production repo without tag policy. Report on image provenance and push frequency.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=registry sourcetype="registry:audit"
| search action=push
| stats latest(_time) as last_push, count by image_name, actor, tag
| table image_name, tag, actor, last_push, count
```
