# UC-12.3.12 · Registry Storage Growth

## Description

Daily growth rate of container/NPM registry—forecast disk exhaustion (extends UC-12.3.1).

## Value

Daily growth rate of container/NPM registry—forecast disk exhaustion (extends UC-12.3.1).

## Implementation

Alert when weekly growth >20% or forecast crosses 85% capacity in <90 days. Recommend GC tuning.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Harbor/ECR/Artifactory storage API.
• Ensure the following data sources are available: Total bytes, per-repo breakdown.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Alert when weekly growth >20% or forecast crosses 85% capacity in <90 days. Recommend GC tuning.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=devops sourcetype="registry:storage"
| timechart span=1d sum(size_bytes) as total by registry_name
| predict total as forecast
```
