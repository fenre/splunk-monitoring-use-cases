# UC-12.2.7 · Test Coverage Trending

## Description

Declining test coverage increases risk of undetected bugs. Trending ensures quality standards are maintained.

## Value

Declining test coverage increases risk of undetected bugs. Trending ensures quality standards are maintained.

## Implementation

Parse test coverage reports from CI/CD pipelines. Send to Splunk via HEC. Track coverage per project. Alert when coverage drops below minimum (e.g., <80%). Block merges when coverage decreases (enforce in CI).

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Custom test report input.
• Ensure the following data sources are available: Test result reports (JUnit XML, coverage reports).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Parse test coverage reports from CI/CD pipelines. Send to Splunk via HEC. Track coverage per project. Alert when coverage drops below minimum (e.g., <80%). Block merges when coverage decreases (enforce in CI).

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cicd sourcetype="test_coverage"
| timechart span=1d latest(coverage_pct) as coverage by project
| where coverage < 80
```
