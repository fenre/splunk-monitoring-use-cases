# UC-12.2.19 · Flaky Test Detection

## Description

Tests that pass/fail without code changes waste time and hide real failures; quarantine candidates identified by pass rate variance.

## Value

Tests that pass/fail without code changes waste time and hide real failures; quarantine candidates identified by pass rate variance.

## Implementation

Minimum 10 runs for statistics. File Jira for quarantine when flake_rate >25%. Track fix SLA.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: JUnit XML ingest, Buildkite/GitHub Actions annotations.
• Ensure the following data sources are available: Test case name, suite, result per run.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Minimum 10 runs for statistics. File Jira for quarantine when flake_rate >25%. Track fix SLA.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=cicd sourcetype="junit:result"
| stats count(eval(result="SUCCESS")) as pass, count(eval(result="FAILURE")) as fail, count as runs by test_case, class_name
| eval flake_rate=round(fail/runs*100,1)
| where runs>10 AND flake_rate>10 AND flake_rate<90
| sort -flake_rate
```
