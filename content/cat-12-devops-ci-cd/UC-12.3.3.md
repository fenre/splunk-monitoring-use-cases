# UC-12.3.3 · Package Download Anomalies

## Description

Unusual package download patterns may indicate dependency confusion attacks or compromised internal packages.

## Value

Unusual package download patterns may indicate dependency confusion attacks or compromised internal packages.

## Implementation

Monitor package download patterns. Baseline normal download volumes per package. Alert on statistical outliers. Watch for downloads of internal packages from external IPs. Track new/unknown packages being introduced.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Artifactory/Nexus access logs.
• Ensure the following data sources are available: Repository access logs (download events).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Monitor package download patterns. Baseline normal download volumes per package. Alert on statistical outliers. Watch for downloads of internal packages from external IPs. Track new/unknown packages being introduced.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=devops sourcetype="artifactory:access"
| stats count by package_name, client_ip
| eventstats avg(count) as avg_downloads, stdev(count) as stdev_downloads by package_name
| where count > avg_downloads + 3*stdev_downloads
```
