# UC-12.4.16 · IaC Module Version Compliance

## Description

Terraform module sources below approved minimum semver—reduces stale or vulnerable module usage.

## Value

Terraform module sources below approved minimum semver—reduces stale or vulnerable module usage.

## Implementation

Weekly compliance report. Enforce minimum via Sentinel/OPA in pipeline. Pair with private registry pinning.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: `terraform-config-inspect`, CI parse of resolved modules.
• Ensure the following data sources are available: Module `name`, `version` from `terraform init -json`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Weekly compliance report. Enforce minimum via Sentinel/OPA in pipeline. Pair with private registry pinning.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=iac sourcetype="terraform:modules"
| lookup terraform_module_allowed module_name OUTPUT min_version
| where semver_compare(module_version, min_version) < 0
| table workspace, module_name, module_version, min_version
```
