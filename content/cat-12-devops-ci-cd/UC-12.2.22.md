# UC-12.2.22 · Mean Time to Recovery (MTTR)

## Description

Time from incident detection or deploy failure to successful restore—core DORA metric.

## Value

Time from incident detection or deploy failure to successful restore—core DORA metric.

## Implementation

Correlate PD incidents with service. For deploy failures, measure time to healthy deploy. Executive review monthly.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: PagerDuty/Opsgenie, deployment events.
• Ensure the following data sources are available: Incident `created_at`, `resolved_at`; rollback `deploy_time`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Correlate PD incidents with service. For deploy failures, measure time to healthy deploy. Executive review monthly.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=itsm sourcetype="pagerduty:incident"
| eval mttr_min=(resolved_epoch-ack_epoch)/60
| stats avg(mttr_min) as avg_mttr, p95(mttr_min) as p95_mttr by service
| where avg_mttr > 60
| sort -avg_mttr
```
