# UC-12.4.9 · Feature Flag and Experiment Rollout Monitoring

## Description

Feature flags and experiments affect user experience. Monitoring rollout percentage and error rate per flag supports safe rollouts and rollback.

## Value

Feature flags and experiments affect user experience. Monitoring rollout percentage and error rate per flag supports safe rollouts and rollback.

## Implementation

Ingest flag evaluation and error data. Track rollout % and error rate per flag/variant. Alert on error rate spike after rollout. Report on flag adoption and performance by variant.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Feature flag provider logs, app telemetry.
• Ensure the following data sources are available: Flag evaluation logs, rollout events, error rates by flag.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest flag evaluation and error data. Track rollout % and error rate per flag/variant. Alert on error rate spike after rollout. Report on flag adoption and performance by variant.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=app sourcetype="feature_flag:eval"
| bin _time span=1h
| stats count, sum(eval(if(error="true",1,0))) as errors by flag_name, variant, _time
| eval error_rate=round((errors/count)*100, 2)
| where error_rate > 5
```
