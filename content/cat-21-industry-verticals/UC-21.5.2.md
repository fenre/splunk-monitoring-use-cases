# UC-21.5.2 · Wellhead Telemetry Data Gap Monitoring

## Description

Missing wellhead measurements indicate communication failures. Quick detection ensures SCADA and field teams can restore data continuity.

## Value

Missing wellhead measurements indicate communication failures. Quick detection ensures SCADA and field teams can restore data continuity.

## Implementation

Include expected interval on each event or use 3x scan rate as threshold. Verify against planned shutdowns via a maintenance lookup. **Domain context:** Wellhead SCADA gaps can mask unsafe operating conditions or production loss—align thresholds with RTU poll schedule (minutes vs seconds). **Splunk:** `now()-last_seen` in scheduled searches uses search time; use `latest(_time)` from a summarization for fleet-wide freshness.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="wellhead:telemetry"
| stats latest(_time) as last_seen by well_id
| eval gap_sec=now()-last_seen
| where gap_sec > 900
| eval gap_min=round(gap_sec/60,1)
| sort - gap_sec
| table well_id, last_seen, gap_min
```
