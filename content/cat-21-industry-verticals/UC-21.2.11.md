# UC-21.2.11 · Robotic Cell Cycle Time Deviation

## Description

Drift from takt-time signals tooling wear or program changes; catching deviation early avoids quality escapes and unplanned maintenance.

## Value

Drift from takt-time signals tooling wear or program changes; catching deviation early avoids quality escapes and unplanned maintenance.

## Implementation

Ingest cycle completion messages from robot OEM or PLC via Edge Hub syslog. Baseline `target_sec` per program revision in lookup when recipes change. Alert maintenance when sustained positive delta exceeds policy. **Domain context:** Robot takt-time drift often tracks EOAT wear, TCP calibration, or upstream material variation—correlate with CMMS on the same `cell_id`. **Splunk:** OEM syslog formats vary; use `props.conf` to extract `cycle_sec`/`program_id` consistently across firmware versions.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: Splunk OT Intelligence, robot controller HEC.
• Ensure the following data sources are available: `index=mfg` `sourcetype="robot:cycle"` (cell_id, program_id, cycle_sec, target_sec).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Ingest cycle completion messages from robot OEM or PLC via Edge Hub syslog. Baseline `target_sec` per program revision in lookup when recipes change. Alert maintenance when sustained positive delta exceeds policy. **Domain context:** Robot takt-time drift often tracks EOAT wear, TCP calibration, or upstream material variation—correlate with CMMS on the same `cell_id`. **Splunk:** OEM syslog formats vary; use `props.conf` to extract `cycle_sec`/`program_id` consistently across firmware versions.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=mfg sourcetype="robot:cycle"
| eval delta_sec=cycle_sec-target_sec, delta_pct=if(target_sec>0, 100*delta_sec/target_sec, null)
| bin _time span=15m
| stats avg(cycle_sec) as avg_cycle, avg(target_sec) as avg_target, stdev(cycle_sec) as cycle_jitter by cell_id, program_id, _time
| eval avg_delta_pct=if(avg_target>0, 100*(avg_cycle-avg_target)/avg_target, null)
| where avg_delta_pct>10 OR cycle_jitter>3
| table _time, cell_id, program_id, avg_cycle, avg_target, avg_delta_pct, cycle_jitter
```
