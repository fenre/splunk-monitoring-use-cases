# UC-21.9.3 · Distribution System Pressure Zone Monitoring

## Description

Low pressure risks contamination and service complaints; high pressure stresses mains. Zone-level analytics isolate PRV faults and demand spikes faster than single-point alarms.

## Value

Low pressure risks contamination and service complaints; high pressure stresses mains. Zone-level analytics isolate PRV faults and demand spikes faster than single-point alarms.

## Implementation

Multiple sensors per zone—use `stats` to aggregate. Join PRV asset IDs via lookup for work orders. Pair with demand forecasts during fire flow tests. **Domain context:** Pressure zones must maintain minimum service pressure per utility design standards—breaches can indicate main breaks, PRV faults, or pump/VFD issues. **Splunk:** `min(pressure_psi)` / `max` over 5m hides transient dips—add `max(breach)` or shorter `bin` for critical zones.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: SCADA pressure telemetry via HEC.
• Ensure the following data sources are available: `index=water` `sourcetype="pressure:zone"` (zone_id, pressure_psi, min_target_psi, max_target_psi, sensor_id).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Multiple sensors per zone—use `stats` to aggregate. Join PRV asset IDs via lookup for work orders. Pair with demand forecasts during fire flow tests. **Domain context:** Pressure zones must maintain minimum service pressure per utility design standards—breaches can indicate main breaks, PRV faults, or pump/VFD issues. **Splunk:** `min(pressure_psi)` / `max` over 5m hides transient dips—add `max(breach)` or shorter `bin` for critical zones.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=water sourcetype="pressure:zone"
| eval breach=if(pressure_psi < min_target_psi OR pressure_psi > max_target_psi, 1, 0)
| bin _time span=5m
| stats min(pressure_psi) as min_p, max(pressure_psi) as max_p, max(breach) as any_breach by zone_id, _time
| where any_breach=1
| table _time, zone_id, min_p, max_p, any_breach
```
