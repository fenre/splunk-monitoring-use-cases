# UC-21.5.9 · Environmental Compliance Effluent Monitoring

## Description

Tracking effluent parameters against permit limits enables timely corrective action and audit-ready documentation for environmental regulations.

## Value

Tracking effluent parameters against permit limits enables timely corrective action and audit-ready documentation for environmental regulations.

## Implementation

Align sampling frequency with permit requirements. Add separate searches for daily max vs monthly average if your permit uses both formats. **Domain context:** NPDES-type permits specify limits, averaging periods, and bypass reporting—mirror legal definitions in Splunk (instantaneous vs composite samples). **Splunk:** `exceed=1` per row may over-count—use duration above limit for chronic breaches.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: LIMS / online analyzer via HEC.
• Ensure the following data sources are available: `index=ot` `sourcetype="effluent:monitor"` fields `outfall_id`, `parameter`, `value_mg_l`, `limit_mg_l`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Align sampling frequency with permit requirements. Add separate searches for daily max vs monthly average if your permit uses both formats. **Domain context:** NPDES-type permits specify limits, averaging periods, and bypass reporting—mirror legal definitions in Splunk (instantaneous vs composite samples). **Splunk:** `exceed=1` per row may over-count—use duration above limit for chronic breaches.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=ot sourcetype="effluent:monitor"
| eval exceed=if(value_mg_l > limit_mg_l, 1, 0)
| where exceed=1
| stats earliest(_time) as first_exceed, max(value_mg_l) as peak_value by outfall_id, parameter
| table outfall_id, parameter, first_exceed, peak_value, limit_mg_l
```
