# UC-21.4.8 · Warehouse Management System Order Accuracy

## Description

Pick accuracy directly impacts customer satisfaction, returns, and operational costs. Tracking accuracy by zone and picker enables targeted training.

## Value

Pick accuracy directly impacts customer satisfaction, returns, and operational costs. Tracking accuracy by zone and picker enables targeted training.

## Implementation

Capture pick confirmation events from WMS. Calculate accuracy rates by zone and picker. Alert on accuracy below threshold. Identify systemic issues vs individual training needs. **Domain context:** Pick accuracy impacts OTIF and returns; voice-pick and scan-to-light systems have different baseline error modes. **Splunk:** `stats avg(correct)` on binary picks yields accuracy per aggregation grain—ensure enough `total_picks` per cell before ranking pickers.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: WMS application logs via HEC.
• Ensure the following data sources are available: `index=logistics` `sourcetype="wms:order"` fields `order_id`, `pick_correct`, `zone`, `picker_id`.
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Capture pick confirmation events from WMS. Calculate accuracy rates by zone and picker. Alert on accuracy below threshold. Identify systemic issues vs individual training needs. **Domain context:** Pick accuracy impacts OTIF and returns; voice-pick and scan-to-light systems have different baseline error modes. **Splunk:** `stats avg(correct)` on binary picks yields accuracy per aggregation grain—ensure enough `total_picks` per cell before ranking pickers.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=logistics sourcetype="wms:order"
| eval correct=if(pick_correct="true" OR pick_correct=1, 1, 0)
| stats avg(correct) as accuracy_pct, count as total_picks by zone, picker_id
| eval accuracy_pct=round(accuracy_pct*100,2)
| where accuracy_pct < 99.5
| sort accuracy_pct
| table zone, picker_id, accuracy_pct, total_picks
```
