# UC-21.3.20 · DIPS Arena User Authentication and SSO Monitoring

## Description

DIPS Arena authentication flows through the DIPS Federation Service using OAuth/OIDC, with support for DIPS credentials, Microsoft Entra ID, and SMART on FHIR SSO. Authentication failures lock clinicians out of the EHR during critical care moments. Monitoring authentication success rates, latency, and failure patterns detects identity infrastructure issues before they cascade into widespread access disruptions.

## Value

## Implementation

Collect authentication event logs from the DIPS Federation Service. The service handles Authorization Code Flow (interactive clinician login), Client Credentials Flow (system integrations), and SMART on FHIR (embedded applications with SSO). Parse each event for authentication flow type, result (success/failure), error code, and response time. Alert on: authentication failure rate exceeding 10% within a 10-minute window, SSO response time exceeding 5 seconds (causes UI timeouts), repeated failures from a single client application (broken integration), or complete authentication service unavailability. Correlate with Microsoft Entra ID status if using federated identity.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=healthcare sourcetype="dips:federation:auth"
| bin _time span=10m
| stats sum(eval(if(result="success", 1, 0))) as success,
  sum(eval(if(result="failure", 1, 0))) as failures,
  dc(client_id) as unique_clients,
  avg(response_time_ms) as avg_auth_ms by auth_flow, _time
| eval fail_pct=round(failures/(success+failures)*100,1)
| where fail_pct > 10 OR avg_auth_ms > 5000
| table _time, auth_flow, success, failures, fail_pct, avg_auth_ms, unique_clients
```
