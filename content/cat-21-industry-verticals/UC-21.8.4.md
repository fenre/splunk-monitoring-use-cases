# UC-21.8.4 · Network Capacity Planning (Spectrum Utilization Trending)

## Description

Rising PRB or downlink utilization trends drive sector splits and carrier adds; long-window trending supports RF engineering without duplicating CDR-based traffic analytics (Cat 5.12).

## Value

Rising PRB or downlink utilization trends drive sector splits and carrier adds; long-window trending supports RF engineering without duplicating CDR-based traffic analytics (Cat 5.12).

## Implementation

Aggregate busy-hour samples per operator policy; store `sample_period_sec` for weighting. Join sector metadata (band, azimuth) via lookup for planning reports. Use weekly baselines to smooth day-of-week noise. **Domain context:** LTE PRB utilization drives capacity; LTE vs NR reporting periods differ—align `bin` span with PM file granularity. **Splunk:** `eventstats median(avg_peak) as med_site by site_id` compares cells to site median—use `cell_id` if site-level median masks hot sectors.

## Detailed Implementation

Prerequisites
• Install and configure the required add-on or app: RAN performance management feed via HEC.
• Ensure the following data sources are available: `index=telecom` `sourcetype="spectrum:utilization"` (site_id, cell_id, dl_prb_util_pct, ul_prb_util_pct, sample_period_sec).
• For app installation, inputs.conf, and Splunk directory layout, see the Implementation guide: docs/implementation-guide.md

Step 1 — Configure data collection
Aggregate busy-hour samples per operator policy; store `sample_period_sec` for weighting. Join sector metadata (band, azimuth) via lookup for planning reports. Use weekly baselines to smooth day-of-week noise. **Domain context:** LTE PRB utilization drives capacity; LTE vs NR reporting periods differ—align `bin` span with PM file granularity. **Splunk:** `eventstats median(avg_peak) as med_site by site_id` compares cells to site median—use `cell_id` if site-level median masks hot sectors.

Step 2 — Create the search and alert
Run the following SPL in Search (then save as report or alert; adjust time range and threshold as needed):

```spl
index=telecom sourcetype="spectrum:utilization"
| eval peak_util=max(dl_prb_util_pct, ul_prb_util_pct)
| bin _time span=1d
| stats avg(peak_util) as avg_peak, perc95(peak_util) as p95_peak by site_id, cell_id, _time
| eventstats median(avg_peak) as med_site by site_id
| eval growth_ratio=if(med_site>0, round(avg_peak/med_site,2), null)
| where p95_peak > 85 OR growth_ratio > 1.15
| table _time, site_id, cell_id, avg_peak, p95_peak, growth_ratio
```
